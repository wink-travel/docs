[רכיבי ווב](https://developer.mozilla.org/en-US/docs/Web/API/Web_components) הוא תקן שמאפשר לך לשלב תכונות JavaScript שלמות באתר שלך מבלי לדעת לתכנת.
תודות לטכנולוגיה המגניבה הזו, ניתן לשלב את מלאי הנסיעות של Wink בקלות רבה. מאמר זה מדריך אותך דרך אוסף רכיבי הווב שלנו ומראה כיצד להשתמש בהם.

## שילוב Wink

יש שלושה דברים שעליך לכלול בכל עמוד שבו תרצה להציג אחד מרכיבי הווב שלנו.

1. כלול את גיליון הסגנונות שלנו.
2. כלול את קוד ה-Javascript שלנו.
3. כלול את טוען היישום שלנו.

### גיליון סגנונות

כלול את סגנונות ה-CSS שלנו בתוך תג ה-head של המסמך שלך.

```html {3}
<html>
  <head>
    <link rel="stylesheet" href="https://elements.wink.travel/styles.css"></link>
  </head>
</html>
```

### Javascript

כלול את קוד ה-Javascript שלנו בתחתית המסמך שלך. (*מומלץ למקם מיד מעל תג ה-</body>*).

```html {3}
<html>
  <body>
    <script src="https://elements.wink.travel/elements.js" type="module" defer></script>
  </body>
</html>
```

האתר שלך מוכן כעת ויכול להציג את רכיבי הווב שלנו.

## רכיבים

קרא על ספריית הרכיבים שלנו למטה.

### טוען יישום

הטוען אחראי על שמירת מצב וניהול האינטראקציה בין הרכיבים שלנו. (*כלול אותו מתחת ל-Javascript שלנו*).

```html {3-6}
<html>
  <body>
    <wink-app-loader
      client-id="YOUR CLIENT ID GOES HERE"
      configuration-id="YOUR CUSTOMIZATION ID GOES HERE"
    ></wink-app-loader>
  </body>
</html>
```

### טוען תוכן

טוען התוכן הוא הלב של ספריית רכיבי הווב שלנו. הוא אחראי על הצגת המלאי שלך (כרטיסים, רשתות, מפות).

תכונות זמינות:

- layout
- id
- sort

*התכונה `sort` זמינה רק כאשר layout הוא `RANKED` ואתה לא רוצה להשתמש ברשת מדורגת קיימת. במקרים אלו, השאר את `id` ריק.*

סוגי פריסות זמינות:

- `AD_BANNER`
- `MAP`
- `HOTEL`
- `GUEST_ROOM`
- `MEETING_ROOM`
- `SPA`
- `RESTAURANT`
- `ACTIVITY`
- `ATTRACTION`
- `PLACE`
- `ADD_ON`
- `LIST`
- `SEARCH`
- `RANKED`

סוגי מיון זמינים:

- `MEMBER`
- `PRICE_LOW_TO_HIGH`
- `PRICE_HIGH_TO_LOW`
- `PRICE`
- `POPULARITY`
- `ECO`
- `EXPERIENCE`
- `PERK`
- `LOYALTY`
- `PACKAGE`

```html {3-6}
<html>
  <body>
    <wink-content-loader
       layout="GUEST_ROOM"
       id="2de7ee9c-61c9-11ef-9722-42004e494300"
    ></wink-content-loader>
  </body>
</html>
```

דוגמאות ניתן למצוא עבור [כרטיסים](/studio/cards), [רשתות](/studio/grids) ו-[מפות](/studio/maps).

## רכיבים

### תשלום

רכיב התשלום מאפשר לך ליידע את TripPay כי נוסע רוצה לרכוש משהו ואת הכללים והמחירים עבור אותם פריטים.

יש תכונה אחת חובה לווידג'ט:

- `id` מזהה של חוזה ההזמנה שברצונך לבצע.

```html {3}
<html>
  <body>
    <trip-pay id="<INSERT_BOOKING_CONTRACT_ID>"></trip-pay>
  </body>
</html>
```

:::note
כדי ללמוד כיצד ליצור את מזהה חוזה ההזמנה `id`, קרא את המדריך: [אינטגרציה עם TripPay](/guides/integrating-with-trip-pay).
:::

הווידג'ט מכין את החוזה לביצוע ומציג פרטי תשלום *(איור 1)* למשתמש לסיום ההזמנה.

## שגיאות

אם מופיעה לך הודעת שגיאה בעת שילוב אחד מרכיבי הווב שלנו, בין אם ב-WinkLinks או באתר שלך, יש כמה דברים שעשויים להיות לא תקינים:

### זמינות

ייתכן שהמלאי כבר אינו זמין. עבור ל-[Wink Studio](/studio/cards) ובדוק אם סטטוס ה"כרטיס" זמין.
אם לא, הוא יוצג באדום. במקרה כזה, תוכל להמתין ואולי הספק יחזיר אותו לזמינות או להסיר אותו מהרשימה שלך.

:::note
`זמינות` אינה מתייחסת לזמינות יומית של המלאי להזמנה. היא מתייחסת לכך שהמלאי אינו זמין כלל עבורך ועבור אף אחד אחר.
:::

### הוסר

ייתכן שהמלאי הוסר. עבור ל-[Wink Studio](/studio/cards) ובדוק אם סטטוס ה"כרטיס" זמין.
אם לא, הוא יוצג באדום. במקרה כזה, תוכל להמתין ואולי הספק יחזיר אותו לזמינות או להסיר אותו מהרשימה שלך.

### התאמה אישית

ייתכן שהסרת בטעות את ההתאמה האישית שהגדרת עבור המלאי שלך. ודא שההתאמה האישית עדיין זמינה והגדר חדשה אם חסרה.

### יישום

זה מיועד למפתחי ווב בלבד. אם הסרת בטעות את [היישום](/developer/applications); מה שגרם למזהה הלקוח להיות לא זמין, אנא צור יישום חדש והשתמש במזהה הלקוח החדש לשילוב רכיבי הווב שלנו.
