Wink משתמש במיקום גאוגרפי באופן נרחב כדי להפעיל חיפוש מבוסס מיקום, זיהוי אוטומטי של מטבע ושפה, וסינון מלאי גאוגרפי.

## זיהוי GeoIP

כאשר משתמש מבקר בפלטפורמה, Wink מזהה אוטומטית את מיקומו על בסיס כתובת ה-IP שלו. זה מאפשר:

- **בחירת מטבע אוטומטית** — הצגת מחירים במטבע המקומי של המשתמש.
- **העדפת שפה** — הצעת תוכן בשפה הסבירה של המשתמש.
- **תוצאות בקרבת מקום** — עדיפות למלאי הקרוב למיקום המשתמש.

## חיפוש מרחבי

Wink תומך בחיפוש גאוגרפי באמצעות ארבעה סוגי שאילתות מרחביות. אלו מפעילים את החיפוש המבוסס מפה, חיפוש ברדיוס, וסינון מבוסס אזור ברחבי הפלטפורמה.

### נקודה

קו רוחב ואורך יחיד (קו רוחב, קו אורך). משמש לסימון מיקום ספציפי — לדוגמה, כתובת נכס או מיקום נוכחי של משתמש.

```json
{
  "point": {
    "x": -73.9857,
    "y": 40.7484
  }
}
```

### מעגל

נקודת מרכז עם רדיוס. משמש לשאילתות "חפש בקרבתי" או מבוססות רדיוס — לדוגמה, "מלונות בטווח של 10 ק"מ מ-Times Square."

```json
{
  "point": {
    "x": -73.9857,
    "y": 40.7484
  },
  "radius": 10000
}
```

### מלבן

תיבת גבול המוגדרת על ידי פינות דרום-מערב וצפון-מזרח. משמש לחיפוש מבוסס מפה — כאשר משתמש מזיז ומגדיל מפה, האזור הנראה מגדיר שאילתת מלבן.

```json
{
  "southWest": {
    "x": -74.0060,
    "y": 40.7128
  },
  "northEast": {
    "x": -73.9350,
    "y": 40.7831
  }
}
```

### מצולע

צורה מותאמת המוגדרת על ידי סדרת נקודות. משמש לחיפוש בתוך גבולות גאוגרפיים לא סדירים — לדוגמה, שכונה, רובע עירוני, או אזור שצויר בהתאמה על מפה.

```json
{
  "polygon": {
    "points": [
      { "x": -74.0060, "y": 40.7128 },
      { "x": -73.9350, "y": 40.7128 },
      { "x": -73.9350, "y": 40.7831 },
      { "x": -74.0060, "y": 40.7831 },
      { "x": -74.0060, "y": 40.7128 }
    ]
  }
}
```

:::note
המצולעים חייבים להיות סגורים — הנקודה האחרונה חייבת להתאים לנקודה הראשונה.
:::

## מקרי שימוש

| מקרה שימוש | סוג צורה | דוגמה |
|---|---|---|
| חיפוש בקרבתי | מעגל | מלונות בטווח של 5 ק"מ מהמיקום שלי |
| חיפוש בתצוגת מפה | מלבן | נכסים הנראים בתצוגת המפה הנוכחית |
| חיפוש בשכונה | מצולע | מלונות ברובע עירוני מסוים |
| מיקום נכס | נקודה | מיקום מדויק של נכס ברשימה |

## למפתחים

פרמטרים של שאילתות מרחביות מתקבלים על ידי נקודות הקצה של Lookup API ו-Inventory API. העבר את צורת GeoJSON המתאימה בבקשת החיפוש שלך כדי לסנן תוצאות לפי מיקום.

עיין ב-[תיעוד ה-APIs](/he/developers/apis) לפרטים ספציפיים לנקודות הקצה.
