## סקירה כללית
שרת MCP של Wink המתארח זמין בכתובת:

```
https://docs.mcp.wink.travel/mcp
```

כוון לקוחות AI תואמים (VS Code, Cursor, אחרים המיישמים MCP HTTP transport) לנקודת הקצה הזו כדי לאפשר להם:
- לגלוש בתיעוד מוצר ופלטפורמה (`wink://docs/...`)
- לגלוש במפרטי OpenAPI חיים ובתיעוד פעולות (`wink://api/...`)
- לבצע חיפוש סמנטי / מילות מפתח קלות במשאבים וב-APIs חיים

## תחבורה
השתמש בנקודת הקצה הראשית הניתנת לזרימה ב-HTTP:

```
https://docs.mcp.wink.travel/mcp
```

אם הלקוח שלך תומך רק ב-SSE, השתמש בכתובת:

```
https://docs.mcp.wink.travel/sse
```

### Claude Code
תיעוד רשמי: [חבר את Claude Code לכלים דרך MCP](https://code.claude.com/docs/en/mcp)

Anthropic ממליצה על תחבורה HTTP מרוחקת עבור שרתי MCP המתארחים בענן:
```bash
claude mcp add --transport http wink-docs https://docs.mcp.wink.travel/mcp
claude mcp list
```

בתוך Claude Code, הרץ `/mcp` כדי לבדוק את מצב השרת. אם ברצונך לשתף את השרת עם כל הריפו, הוסף `--scope project` כדי ש-Claude Code יכתוב את ההגדרה ל-`.mcp.json`.

### Codex
תיעוד רשמי: [התחלת עבודה מהירה עם Docs MCP עבור Codex](https://developers.openai.com/resources/docs-mcp)

OpenAI מתעדת את הגדרת MCP דרך CLI של Codex, ומציינת שההגדרה משותפת עם תוסף ה-IDE:
```bash
codex mcp add wink-docs --url https://docs.mcp.wink.travel/mcp
codex mcp list
```

### Cursor
תיעוד MCP של Cursor: https://cursor.com/docs/context/mcp
הגדרה: הוסף שרת מרוחק דרך הגדרות MCP.
```json
{
  "mcpServers": {
    "wink-docs": {
      "url": "https://docs.mcp.wink.travel/mcp",
    }
  }
}
```

### VS Code
תיעוד רשמי: https://code.visualstudio.com/docs/copilot/customization/mcp-servers
הגדרה מרחוק מינימלית (קובץ `mcp.json` למשתמש או לסביבת עבודה):
```json
{
  "servers": {
    "wink-docs": {
      "type": "http",
      "url": "https://docs.mcp.wink.travel/mcp"
    }
  }
}
```

טיפים:
- השתמש ב-Command Palette: “MCP: List Servers” כדי לאשר חיבור ולגלוש בכלים.
- בוחר הכלים מאפשר להפעיל/לבטל `api_search` ו-`docs_search`.
- הוסף הקשר: תצוגת שיחה → הוסף הקשר → משאבי MCP כדי לצרף דפי תיעוד.

## כתובות URI עיקריות למשאבים

| URI | תיאור |
|-----|-------------|
| `wink://docs` | אינדקס תיעוד (ממוין לפי קטגוריה) |
| `wink://docs/{path}` | מאמר תיעוד ספציפי (Markdown / MDX) |
| `wink://api` | קטלוג OpenAPI חי שהתגלה מתוך קונפיגורציית swagger של Wink |
| `wink://api/{apiName}` | סיכום API קומפקטי עם אימות, תגיות, כותרות ודוגמאות לפעולות |
| `wink://api/{apiName}/operations` | רשימת פעולות מלאה ממוין לפי תגית |
| `wink://api/{apiName}/operation/{key}` | פירוט פעולה בודדת (פרמטרים, גוף בקשה, תגובות) |
| `wink://api/{apiName}/schema/{schemaName}` | סכמת רכיב מפורטת עם שדות מקוננים, סוגים ודוגמאות |
| `wink://api/{apiName}/operation/{key}/request` | תצוגת בקשה עם קלטים נדרשים ודוגמת payload מינימלית |
| `wink://api/{apiName}/operation/{key}/response/{status}` | תצוגת תגובה עבור קוד סטטוס ספציפי |
| `wink://api/{apiName}/operation/{key}/recipe` | זרימת משימה לקריאה נכונה של הפעולה |

## כלי חיפוש (התייחסות מהירה)
| כלי | מטרה | דוגמאות לשאילתות טיפוסיות |
|------|---------|-----------------------|
| `api_search` | מצא את ה-API או הפעולה החיה לשילוב | `create affiliate`, `payment receipt`, `lookup inventory`, `invite manager` |
| `docs_search` | מצא תיעוד מושגי / מדריכים | `authentication`, `webhooks`, `wordPress` |

השאר את `query` ריק כדי לקבל הצעות. כל תוצאה כוללת את כתובת ה-MCP שניתן לפתוח.
לפעולות שמצאו, `api_search` מחזיר גם כתובות URI מקושרות של בקשה, תגובה, סכמות ומתכונים כדי שהסוכנים יוכלו לקפוץ ישירות לתת-המשאב השימושי ביותר.

## דפוס שימוש טיפוסי
1. הרץ `api_search` עם הכוונה שלך (למשל *"create affiliate"* או *"payment receipt"*).
2. פתח את כתובת ה-URI שהוחזרה `wink://api/...`, או עקוב אחרי שדות ה-`request_uri`, `response_uris`, `request_schema_uris`, `response_schema_uris`, ו-`recipe_uri`.
3. השתמש ב-`docs_search` עבור מושגים תומכים (אימות, webhooks, זרימות עבודה).

## הערות
- כל URI שהוחזר ניתן לפתיחה על ידי הסוכן לתוכן מלא.
- עץ `wink://api` חי ועוקב אחרי קטלוג OpenAPI העדכני במקום אינדקס ריפו שנוצר.
- הצעות מופיעות אוטומטית כאשר השאילתות ריקות.
