The developer section lets you connect your own systems to TripPay via API applications and webhooks.

## Applications

Applications let you connect to TripPay APIs and SDKs, embed web components on your website, and enable authenticated user sessions with the platform.

To manage your applications, navigate to `Developer > Applications`.

### Creating an application

Click `Create application` and fill in:

- **Name** — A unique name for your application (auto-converted to lowercase with hyphens).
- **Redirect URIs** — One or more URLs where users are sent after login.
- **Post-logout redirect URIs** — Where users are sent after logging out.

After creation, you will be shown your **Client ID** and **Secret Key**.

:::caution
The secret key is only displayed once. Copy and store it securely before closing the dialog.
:::

### Revoking credentials

If your credentials are compromised, click `Revoke` to generate a new Client ID and Secret Key. This immediately invalidates the current credentials.

## Webhooks

Webhooks deliver real-time event notifications so your systems stay in sync with TripPay as bookings, payments, and other changes occur.

To manage your webhooks, navigate to `Developer > Webhooks`.

### Creating a webhook

Click `Create webhook` and configure:

- **Name** — A descriptive name for the webhook.
- **Webhook URL** — The endpoint on your server that will receive event notifications.
- **Events** — Select specific events to subscribe to, or leave empty to receive all events.
- **Enabled** — Toggle the webhook on or off without deleting it.

### Managing webhooks

From the webhook list you can:

- Edit the endpoint URL or subscribed events.
- Enable or disable a webhook to pause notifications temporarily.
- Delete a webhook when it is no longer needed.

:::note
Deleting a webhook means any integration currently listening will no longer receive real-time notifications.
:::
