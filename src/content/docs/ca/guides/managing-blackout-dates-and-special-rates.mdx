No tots els dies tenen el mateix preu. Els esdeveniments, festius, temporades baixes i les últimes places disponibles requereixen tarifes diferents. Aquesta guia t’ensenya com utilitzar tarifes especials i restriccions de plans tarifaris per gestionar aquests casos.

## Preus per esdeveniments

Quan una conferència, festival o esdeveniment local augmenta la demanda, voldràs incrementar les tarifes per a aquestes dates.

<Steps>
1. Ves a `Monetize > Special Rates` al teu Extranet.
2. Fes clic a `Create a new special rate`.
3. Selecciona la tarifa principal i el tipus d’habitació que vols sobreescriure.
4. Estableix l’interval de dates que cobreixi el període de l’esdeveniment.
5. Introdueix els preus diaris ajustats (més alts).
6. Fes clic a `Save`.
</Steps>

La tarifa especial sobreescriu la tarifa principal només per a aquestes dates. La teva tarifa habitual es manté sense canvis fora del període de l’esdeveniment.

## Ajustos estacionals

Per a períodes més llargs (temporada alta, temporada mitjana, temporada baixa), pots:

- **Utilitzar tarifes especials** — Crear sobreescriptures per a cada temporada amb preus ajustats.
- **Actualitzar les tarifes principals** — Ajustar directament les tarifes base per a aquests períodes.

Les tarifes especials són millors per ajustos temporals, mentre que actualitzar les tarifes principals té sentit per a patrons anuals previsibles.

## Tarifes d’última hora

Per omplir habitacions no venudes a prop de la data d’arribada:

<Steps>
1. Crea un nou **pla tarifari** amb una restricció de reserva anticipada (per exemple, s’ha de reservar com a màxim 3 dies abans).
2. Combina’l amb una **tarifa especial** que ofereixi un descompte per a les dates objectiu.
3. Aquest pla tarifari només apareixerà als viatgers que reservin dins de la finestra d’última hora.
</Steps>

## Dates bloquejades

Per bloquejar certes dates d’un pla tarifari específic (per exemple, sense tarifes promocionals durant la temporada alta):

<Steps>
1. Ves a `Monetize > Rate Plans` i selecciona el pla tarifari.
2. Navega a la secció **Restrictions**.
3. Afegeix una restricció de **Sell date** per limitar quan es pot reservar la tarifa.
4. O utilitza restriccions de **Stay date** per limitar quan cobreix la tarifa.
</Steps>

Per exemple, pots restringir el teu pla tarifari "Early Bird" perquè no estigui disponible per a reserves durant el desembre (temporada alta), mentre que el teu pla tarifari habitual es manté obert.

## Combinació de restriccions

Les restriccions es poden combinar per a un control precís:

| Restricció | Què controla |
| -- | -- |
| **Sell dates** | Quan es pot reservar la tarifa |
| **Stay dates** | Quines dates d’estada cobreix la tarifa |
| **Available days** | Quins dies de la setmana apareix la tarifa |
| **Arrival days** | En quins dies ha d’arribar el client |
| **Departure days** | En quins dies ha de marxar el client |
| **Required days** | El client ha d’estar almenys un d’aquests dies |
| **Advance booking** | Amb quanta antelació s’ha de fer la reserva |
| **Length of stay** | Nombre mínim i màxim de nits |
