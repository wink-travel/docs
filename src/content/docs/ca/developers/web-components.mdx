[Components web](https://developer.mozilla.org/en-US/docs/Web/API/Web_components)és un estàndard que us permet integrar funcions completes de Javascript al vostre lloc web sense saber programar.
Gràcies a aquesta tecnologia tan interessant, us permet integrar l'inventari de viatges de Wink amb molt poca feina. Aquest article us guiarà a través de la nostra col·lecció de components web i us mostrarà com utilitzar-los.

## Incrustar Wink

Hi ha tres coses que heu d'incloure a qualsevol pàgina on vulgueu mostrar un dels nostres components web.

1. Inclou el nostre full d'estil.
2. Inclou el nostre Javascript.
3. Inclou el nostre carregador d'aplicacions.

### Full d'estils

Inclou els nostres estils CSS a la capçalera del document.

```html {3}
<html>
  <head>
    <link rel="stylesheet" href="https://elements.wink.travel/styles.css"></link>
  </head>
</html>
```

### Javascript

Incloeu el nostre Javascript a la part inferior del document. (*Recomanem que estigui immediatament a sobre de l'etiqueta final del cos*).

```html {3}
<html>
  <body>
    <script src="https://elements.wink.travel/elements.js" type="module" defer></script>
  </body>
</html>
```

El vostre lloc web ja està a punt i pot mostrar els nostres components web.

## components

Llegiu sobre la nostra biblioteca de components a continuació.

### Carregador d'aplicacions

El carregador és responsable de mantenir l'estat i gestionar la interacció entre els nostres components.*Inclou-ho sota el nostre Javascript*).

```html {3-6}
<html>
  <body>
    <wink-app-loader
      client-id="YOUR CLIENT ID GOES HERE"
      configuration-id="YOUR CUSTOMIZATION ID GOES HERE"
    ></wink-app-loader>
  </body>
</html>
```

### Carregador de contingut

El carregador de contingut és el cor de la nostra biblioteca de components web. És responsable de mostrar el vostre inventari (targetes, quadrícules, mapes).

Atributs disponibles:

* disseny
* identificador
* ordenar

*L'atribut`sort`només està disponible quan el disseny és`RANKED`i no voleu utilitzar una graella classificada existent. Per a aquests casos, deixeu`id`buit.*

Tipus de disseny disponibles:

* `AD_BANNER`
* `MAP`
* `HOTEL`
* `GUEST_ROOM`
* `MEETING_ROOM`
* `SPA`
* `RESTAURANT`
* `ACTIVITY`
* `ATTRACTION`
* `PLACE`
* `ADD_ON`
* `LIST`
* `SEARCH`
* `RANKED`

Tipus d'ordenació disponibles:

* `MEMBER`
* `PRICE_LOW_TO_HIGH`
* `PRICE_HIGH_TO_LOW`
* `PRICE`
* `POPULARITY`
* `ECO`
* `EXPERIENCE`
* `PERK`
* `LOYALTY`
* `PACKAGE`

```html {3-6}
<html>
  <body>
    <wink-content-loader
       layout="GUEST_ROOM"
       id="2de7ee9c-61c9-11ef-9722-42004e494300"
    ></wink-content-loader>
  </body>
</html>
```

Es poden trobar mostres per a[cartes](/studio/cards),[quadrícules](/studio/grids)i[mapes](/studio/maps).

## components

### Pagament

El component de pagament et permet informar a TripPay que un viatger vol comprar alguna cosa i les normes i els preus d'aquests articles.

Hi ha un atribut obligatori per al widget:

* `id`L'identificador del contracte de reserva que voleu executar.

```html {3}
<html>
  <body>
    <trip-pay id="<INSERT_BOOKING_CONTRACT_ID>"></trip-pay>
  </body>
</html>
```

:::nota
Per aprendre com generar el contracte de reserva`id`, llegeix la guia:[Integració amb TripPay](/guides/integrating-with-trip-pay).
:::
