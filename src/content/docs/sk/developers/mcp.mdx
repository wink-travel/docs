## Prehľad
Hostovaný Wink MCP server je dostupný na:

```
https://docs.mcp.wink.travel/mcp
```

Nasmerujte kompatibilných AI klientov (VS Code, Cursor, ďalšie implementujúce MCP HTTP transport) na túto adresu, aby mohli:
- Prehliadať dokumentáciu produktov a platformy (`wink://docs/...`)
- Prehliadať živé OpenAPI špecifikácie a dokumentáciu operácií (`wink://api/...`)
- Vykonávať ľahké sémantické / kľúčové vyhľadávanie v dokumentácii a živých API

## Transport
Použite primárny streamovateľný HTTP endpoint:

```
https://docs.mcp.wink.travel/mcp
```

Ak váš klient podporuje iba SSE, použite:

```
https://docs.mcp.wink.travel/sse
```

### Claude Code
Oficiálna dokumentácia: [Pripojenie Claude Code k nástrojom cez MCP](https://code.claude.com/docs/en/mcp)

Anthropic odporúča vzdialený HTTP transport pre cloud-hostované MCP servery:
```bash
claude mcp add --transport http wink-docs https://docs.mcp.wink.travel/mcp
claude mcp list
```

V Claude Code spustite `/mcp` pre kontrolu stavu servera. Ak chcete zdieľať server s celým repozitárom, pridajte `--scope project`, aby Claude Code zapísal konfiguráciu do `.mcp.json`.

### Codex
Oficiálna dokumentácia: [Docs MCP quickstart pre Codex](https://developers.openai.com/resources/docs-mcp)

OpenAI dokumentuje nastavenie MCP cez Codex CLI a poznamenáva, že konfigurácia sa zdieľa s IDE rozšírením:
```bash
codex mcp add wink-docs --url https://docs.mcp.wink.travel/mcp
codex mcp list
```

### Cursor
Cursor MCP dokumentácia: https://cursor.com/docs/context/mcp
Nastavenie: Pridajte vzdialený server cez nastavenia MCP.
```json
{
  "mcpServers": {
    "wink-docs": {
      "url": "https://docs.mcp.wink.travel/mcp",
    }
  }
}
```

### VS Code
Oficiálna dokumentácia: https://code.visualstudio.com/docs/copilot/customization/mcp-servers
Minimálna vzdialená konfigurácia (používateľský alebo pracovný priestor `mcp.json`):
```json
{
  "servers": {
    "wink-docs": {
      "type": "http",
      "url": "https://docs.mcp.wink.travel/mcp"
    }
  }
}
```

Tipy:
- Použite Command Palette: „MCP: List Servers“ pre potvrdenie pripojenia a prehliadanie nástrojov.
- Výber nástrojov umožňuje zapnúť/vypnúť `api_search` a `docs_search`.
- Pridajte kontext: Zobrazenie chatu → Pridať kontext → MCP Resources pre pripojenie stránok dokumentácie.

## Hlavné URI zdrojov

| URI | Popis |
|-----|-------------|
| `wink://docs` | Index dokumentácie (zoskupený podľa kategórie) |
| `wink://docs/{path}` | Konkrétny článok dokumentácie (Markdown / MDX) |
| `wink://api` | Živý OpenAPI katalóg zistený z Wink swagger konfigurácie |
| `wink://api/{apiName}` | Kompaktné zhrnutie API s autentifikáciou, tagmi, hlavičkami a ukážkovými operáciami |
| `wink://api/{apiName}/operations` | Kompletný zoznam operácií zoskupených podľa tagu |
| `wink://api/{apiName}/operation/{key}` | Detail jednotlivých operácií (parametre, telo požiadavky, odpovede) |
| `wink://api/{apiName}/schema/{schemaName}` | Riešená schéma komponentu s vnorenými poľami, typmi a príkladmi |
| `wink://api/{apiName}/operation/{key}/request` | Pohľad zameraný na požiadavku s povinnými vstupmi a minimálnym príkladom payloadu |
| `wink://api/{apiName}/operation/{key}/response/{status}` | Pohľad zameraný na odpoveď pre konkrétny stavový kód |
| `wink://api/{apiName}/operation/{key}/recipe` | Úlohový tok pre správne volanie danej operácie |

## Nástroje vyhľadávania (rýchla referencia)
| Nástroj | Účel | Typické príklady dotazov |
|------|---------|-----------------------|
| `api_search` | Nájsť živé API alebo operáciu na integráciu | `create affiliate`, `payment receipt`, `lookup inventory`, `invite manager` |
| `docs_search` | Nájsť konceptuálnu dokumentáciu / návody | `authentication`, `webhooks`, `wordPress` |

Nechajte `query` prázdne pre získanie návrhov. Každý výsledok obsahuje MCP URI, ktoré môžete otvoriť.
Pri nájdení operácií `api_search` tiež vracia prepojené URI požiadaviek, odpovedí, schém a receptov, aby agenti mohli priamo prejsť na najpoužívanejšie podzdroje.

## Typický spôsob použitia
1. Spustite `api_search` s vaším zámerom (napr. *"create affiliate"* alebo *"payment receipt"*).
2. Otvorte vrátené `wink://api/...` URI alebo nasledujte jeho prepojené polia `request_uri`, `response_uris`, `request_schema_uris`, `response_schema_uris` a `recipe_uri`.
3. Použite `docs_search` pre podporné koncepty (autentifikácia, webhooks, workflowy).

## AI schopnosti

Predpripravené schopnosti agentov používajú tento MCP server ako svoj zdroj pravdy. Schopnosti dávajú vášmu kódovaciemu agentovi štruktúrovaný pracovný tok pre vývoj proti Wink API — namiesto hádania endpointov agent číta živé kontrakty priamo z MCP servera.

Pozrite si [AI Skills](/sk/developers/ai-skills) pre dostupné schopnosti a inštalačné pokyny.

## Poznámky
- Každé vrátené URI môže agent otvoriť pre plný obsah.
- Strom `wink://api` je živý a sleduje aktuálny upstream OpenAPI katalóg namiesto generovaného indexu repozitára.
- Návrhy sa zobrazujú automaticky, keď sú dotazy prázdne.
