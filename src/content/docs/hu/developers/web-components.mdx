A [Webkomponensek](https://developer.mozilla.org/en-US/docs/Web/API/Web_components) egy szabvány, amely lehetővé teszi, hogy teljes JavaScript funkciókat integráljon a weboldalába anélkül, hogy tudnia kellene kódolni.
Ennek a menő technológiának köszönhetően nagyon egyszerűen beágyazhatja a Wink utazási készletét. Ez a cikk végigvezeti Önt webkomponens-gyűjteményünkön, és megmutatja, hogyan használhatja őket.

## Wink beágyazása

Három dolgot kell beillesztenie bármely oldalra, ahol meg szeretné jeleníteni valamelyik webkomponensünket.

1. Illessze be a stíluslapunkat.
2. Illessze be a JavaScriptünket.
3. Illessze be az alkalmazás betöltőjét.

### Stíluslap

Illessze be CSS stílusainkat a dokumentum fej részébe.

```html {3}
<html>
  <head>
    <link rel="stylesheet" href="https://elements.wink.travel/styles.css"></link>
  </head>
</html>
```

### Javascript

Illessze be JavaScriptünket a dokumentum aljára. (*Javasoljuk közvetlenül a záró body tag elé*).

```html {3}
<html>
  <body>
    <script src="https://elements.wink.travel/elements.js" type="module" defer></script>
  </body>
</html>
```

Az oldala most már készen áll, és képes megjeleníteni webkomponenseinket.

## Komponensek

Olvassa el az alábbiakban a komponenskönyvtárunkról szóló információkat.

### Alkalmazás betöltő

A betöltő felelős az állapot megőrzéséért és a komponenseink közötti interakció kezeléséért. (*Illessze be a JavaScriptünk alá*).

```html {3-6}
<html>
  <body>
    <wink-app-loader
      client-id="AZ ÖN ÜGYFÉL AZONOSÍTÓJA IDE KERÜL"
      configuration-id="AZ ÖN TESTRESZABÁSI AZONOSÍTÓJA IDE KERÜL"
    ></wink-app-loader>
  </body>
</html>
```

### Tartalom betöltő

A tartalom betöltő a webkomponens könyvtárunk szíve. Ez felelős az Ön készletének megjelenítéséért (kártyák, rácsok, térképek).

Elérhető attribútumok:

- layout
- id
- sort

*A `sort` attribútum csak akkor érhető el, ha a layout `RANKED`, és nem szeretne egy meglévő rangsorolt rácsot használni. Ezekben az esetekben hagyja az `id` mezőt üresen.*

Elérhető layout típusok:

- `AD_BANNER`
- `MAP`
- `HOTEL`
- `GUEST_ROOM`
- `MEETING_ROOM`
- `SPA`
- `RESTAURANT`
- `ACTIVITY`
- `ATTRACTION`
- `PLACE`
- `ADD_ON`
- `LIST`
- `SEARCH`
- `RANKED`

Elérhető rendezési típusok:

- `MEMBER`
- `PRICE_LOW_TO_HIGH`
- `PRICE_HIGH_TO_LOW`
- `PRICE`
- `POPULARITY`
- `ECO`
- `EXPERIENCE`
- `PERK`
- `LOYALTY`
- `PACKAGE`

```html {3-6}
<html>
  <body>
    <wink-content-loader
       layout="GUEST_ROOM"
       id="2de7ee9c-61c9-11ef-9722-42004e494300"
    ></wink-content-loader>
  </body>
</html>
```

Mintákat talál [kártyákhoz](/studio/cards), [rácsokhoz](/studio/grids) és [térképekhez](/studio/maps).

```html {3}
<html>
  <body>
    <wink-lookup></wink-lookup>
  </body>
```

## TripPay beágyazása

Két dolgot kell beillesztenie bármely oldalra, ahol használni szeretné a TripPay fizetési webkomponenst.

1. Illessze be a stíluslapunkat.
2. Illessze be a JavaScriptünket.

### Stíluslap

Illessze be CSS stílusainkat a dokumentum fej részébe.

```html {3}
<html>
  <head>
    <link rel="stylesheet" href="https://elements.trippay.io/styles.css"></link>
  </head>
</html>
```

### Javascript

Illessze be JavaScriptünket a dokumentum aljára. (*Javasoljuk közvetlenül a záró body tag elé*).

```html {3}
<html>
  <body>
    <script src="https://elements.trippay.io/elements.js" type="module" defer></script>
  </body>
</html>
```

Az oldala most már készen áll, és képes megjeleníteni webkomponenseinket.

## Komponensek

### Fizetés

A fizetési komponens lehetővé teszi, hogy a TripPay-nek jelezze, hogy egy utazó vásárolni szeretne valamit, valamint a szabályokat és árakat az adott tételekre.

A widgetnek egy kötelező attribútuma van:

- `id` A foglalási szerződés azonosítója, amelyet végre szeretne hajtani.

```html {3}
<html>
  <body>
    <trip-pay id="<INSERT_BOOKING_CONTRACT_ID>"></trip-pay>
  </body>
</html>
```

:::note
A foglalási szerződés `id` generálásának módját megtudhatja a következő útmutatóban: [Integrálás a TripPay-jel](/guides/integrating-with-trip-pay).
:::

A widget előkészíti a szerződést végrehajtásra, és megjeleníti a fizetési adatokat *(1. ábra)* a felhasználónak a foglalás véglegesítéséhez.

## Hibák

Ha hibaüzenetet kap, amikor beágyaz egy webkomponenst, akár WinkLinks-en, akár a saját webhelyén, néhány dolog történhetett:

### Elérhetőség

Lehet, hogy a készlet már nem elérhető. Látogasson el a [Wink Studio](/studio/cards) oldalra, és ellenőrizze, hogy a "kártya" státusza elérhető-e.
Ha nem, pirosan jelenik meg. Ilyen esetben várhat, hátha a szolgáltató újra elérhetővé teszi, vagy eltávolíthatja a listájából.

:::note
Az `Elérhetőség` nem a napi foglalhatóságot jelenti. Ez azt jelenti, hogy a készlet teljesen nem elérhető Önnek és/vagy bárki másnak.
:::

### Eltávolítva

Lehet, hogy a készletet eltávolították. Látogasson el a [Wink Studio](/studio/cards) oldalra, és ellenőrizze, hogy a "kártya" státusza elérhető-e.
Ha nem, pirosan jelenik meg. Ilyen esetben várhat, hátha a szolgáltató újra elérhetővé teszi, vagy eltávolíthatja a listájából.

### Testreszabás

Lehet, hogy véletlenül eltávolította a készletéhez társított testreszabást. Győződjön meg róla, hogy a testreszabás még elérhető, és állítson be újat, ha hiányzik.

### Alkalmazás

Ez csak webfejlesztőknek szól. Ha véletlenül eltávolította az [Alkalmazást](/developer/applications), így az ügyfélazonosító már nem elérhető, kérjük, hozzon létre egy új alkalmazást, és használja az új ügyfélazonosítót a Webkomponensek beágyazásához.
