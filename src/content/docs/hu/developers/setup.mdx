Ez a cikk megmutatja, hogyan állhatsz be gyorsan fejlesztőként, és kezdheted el tesztelni API-jainkat a Wink-en.

:::note
A tesztelést mindig a [staging környezetünkben](/hu/getting-started/environments) kell végezni. Ott sokkal kevesebb adat van, de korlátlanul tesztelhetsz anélkül, hogy a termelési környezetet terhelnéd.

Amint készen állsz az éles indulásra, hitelesítjük a fiókodat és az alkalmazásodat, és elindulhatsz a termelési környezetünkben.
:::

Kövesd az alábbi lépéseket:

<Steps>
1. [Regisztrálj](/hu/user/register) egy felhasználói fiókot a Wink-en
2. Hozz létre egy [utazási ügynöki fiókot](/hu/travel-agent/what-is-agent) vagy egy [partneri fiókot](/hu/studio/what-is-studio).
3. [Hozz létre egy alkalmazást](/hu/developer/applications), amelyet összekapcsolsz a 2. lépésben létrehozott fiókkal.
4. Miután létrehoztad az `Application`-t, mentsd el biztonságos helyre a client-id-t és a client secret-et.
</Steps>

## CDN

A Wink-nél a Cloudinary-t használjuk minden kép- és videóanyagunkhoz. Amikor hozzáférsz az utazási készletünkhöz, teljes hozzáférésed lesz a Cloudinary összes fejlett funkciójához.

A `cloud_name`-ünk: `traveliko`.

### Cloudinary

- [Kezdő lépések](https://cloudinary.com/documentation/dev_kickstart)
- [SDK-k](https://cloudinary.com/documentation/cloudinary_sdks)
- [Cloudinary a GitHubon](https://github.com/cloudinary)
-	[Video oktatóanyagok](https://cloudinary.com/documentation/cloudinary_sdks_tutorials)

### Példa

Minden általunk támogatott utazási készlettípushoz tartozik egy beágyazott JSON tömb `multimedias` néven. Egy multimédia bejegyzés JSON-je így néz ki:

```json {3}
{
  "multimediaIdentifier": "multimedia-1",
  "identifier": "partners/radisson_blu_logo",
  "type": "IMAGE",
  "width": "1024",
  "height": "768",
  "category": "1",
  "descriptions": [
    {
      "name": "Lobby",
      "description": "A hall képe",
      "language": "en"
    }
  ],
  "lifestyleType": "LIFESTYLE_BUSINESS",
  "attribution": [
    {
      "url": "https://www.hilton.com",
      "name": "Hilton"
    }
  ]
}
```

A kiemelt sor mutatja a Cloudinary egyedi azonosítót `partners/radisson_blu_logo`. Ez az egyetlen, amire szükséged van, hogy bármilyen formátumban, minőségben és méretben lekérd a képet.

## Tesztelés

Látogass el az [API-k](/hu/developer/apis) szekciónkba, és válaszd ki a használni kívánt API-t.

### API dokumentáció

A dokumentációs oldalunk úgy van kialakítva, hogy az API parancsokat közvetlenül az oldalon belül futtathatod.

Ebben a példában a `Search Lookups (OAuth2)` API hívást szeretnéd használni.

:::tip[Pro Tipp]
A könnyebb használat érdekében azt javasoljuk, hogy az integrátorok azokat a végpontokat használják, amelyeket `OAuth2`-ként jelölünk. Ezek a végpontok tudják, hogy API hívók érkeznek, és már ismerik, melyik fiókhoz tartozol, így nem kell megadnod a `companyIdentifier`-t az URL-ekben.
:::

<Steps>
1. Használd a [Lookup API-t](https://developer.wink.travel/lookup) úti célok vagy szállodák lekéréséhez.
2. A bal oldali navigációs sávban kattints az `Authentication` menüpontra.
3. Írd be a client-id-t és a client secret-et, majd kattints a `Get TOKEN` gombra.
4. Ha a hitelesítő adatok érvényesek, látni fogod, hogy az API kulcs alkalmazva lett.
5. Most már tesztelheted a lookup végpontot.
</Steps>

### Postman
Ugyanezt megteheted Postman-nel is.

<Steps>
1. Töltsd le a [Postman-t](https://www.postman.com/downloads).
2. Kattints a `Download OpenAPI spec` gombra a [Lookup API](https://developer.wink.travel/lookup) oldal tetején.
3. Importáld be a Postman-be.
4. Ha új gyűjteményként importáltad `Wink API` néven, kattints a gyűjtemény neve melletti vízszintes három pontra, majd válaszd az `Edit` opciót.
5. Az `Authorization` fülön állítsd az `Auth type`-ot OAuth2-re.
6. Állítsd az `Add auth data to` mezőt `Request headers`-re.
7. Állítsd a `Header prefix` mezőt `Bearer`-re.
8. Add meg a `Token Name` mezőben, hogy milyen néven szeretnéd elmenteni a tokent a Postman-ben, pl. Wink Staging.
9. Állítsd a `Grant type` mezőt `Client Credentials`-re.
10. Állítsd az `Access token URL` mezőt erre: https://staging-iam.wink.travel/oauth2/token.
11. Állítsd a `Client ID` mezőt az alkalmazásod által generált Client ID-re.
12. Állítsd a `Client Secret` mezőt az alkalmazásod által generált Client Secret-re.
13. Állítsd a `Scope` mezőt `inventory.read inventory.write inventory.remove` értékre. Ezek a Wink-hez szükséges engedélyek.
14. Állítsd a `Client Authentication` mezőt `Send as Basic Auth header`-re.
15. Kattints a `Get New Access Token` gombra, hogy lekérd és elmentsd a tokent.
16. Minden Wink API gyűjtemény alatti hívásnál győződj meg róla, hogy az authentikációs fülön az `Inherit auth from parent` van beállítva, így a token minden híváshoz hozzáadódik.
</Steps>

:::tip[Pro Tipp]
Ajánlott a szerver domainjét, a client ID-t és a client secret-et környezeti változókká tenni. Így könnyen válthatsz a staging és a termelési környezet között, és újra felhasználhatod az összes létrehozott műveletet.
:::
