Ez a cikk megmutatja, hogyan állhatsz be gyorsan fejlesztőként, és kezdheted el tesztelni API-jainkat a Wink-en.

:::note
A tesztelést mindig a [staging környezetünkben](/hu/getting-started/environments) kell végezni. Ott sokkal kevesebb adat van, de korlátlanul tesztelhetsz anélkül, hogy a termelési környezetet terhelnéd.

Amint készen állsz az éles indulásra, hitelesítjük a fiókodat és az alkalmazásodat, és elindulhatsz a termelési környezetünkben.
:::

Kövesd az alábbi lépéseket:

<Steps>
1. [Regisztrálj](/hu/my-account/register) felhasználói fiókot a Wink-en.
2. [Hozz létre egy affiliate fiókot](/hu/my-account/create-account). Erre lesz szükséged, hogy az alkalmazásodat hozzákapcsold. További információért lásd a [Mi az a Studio](/hu/studio/what-is-studio) részt.
3. [Hozz létre egy alkalmazást](/hu/my-account/applications), és társítsd az előző lépésben létrehozott affiliate fiókhoz. Szerver-szerver API integrációk esetén válaszd a **MACHINE_2_MACHINE** kliens típust.
4. Miután létrehoztad az `Application`-t, azonnal másold ki és mentsd el a client-id-t és a titkos kulcsot — a titkos kulcs csak egyszer jelenik meg, és később nem kérhető le újra.
</Steps>

## CDN

A Wink-nél a Cloudinary-t használjuk minden kép- és videóanyagunkhoz. Amikor hozzáférsz az utazási készletünkhöz, teljes hozzáférést kapsz a Cloudinary fejlett funkcióihoz.

A `cloud_name`-ünk: `traveliko`.

### Cloudinary

- [Kezdő lépések](https://cloudinary.com/documentation/dev_kickstart)
- [SDK-k](https://cloudinary.com/documentation/cloudinary_sdks)
- [Cloudinary a GitHubon](https://github.com/cloudinary)
-	[Video oktatóanyagok](https://cloudinary.com/documentation/cloudinary_sdks_tutorials)

### Példa

Minden általunk támogatott utazási készlettípushoz tartozik egy beágyazott JSON tömb `multimedias` néven. Egy multimédia bejegyzés JSON-je így néz ki:

```json {3}
{
  "multimediaIdentifier": "multimedia-1",
  "identifier": "partners/radisson_blu_logo",
  "type": "IMAGE",
  "width": "1024",
  "height": "768",
  "category": "1",
  "descriptions": [
    {
      "name": "Lobby",
      "description": "A hall képe",
      "language": "en"
    }
  ],
  "lifestyleType": "LIFESTYLE_BUSINESS",
  "attribution": [
    {
      "url": "https://www.hilton.com",
      "name": "Hilton"
    }
  ]
}
```

A kiemelt sor mutatja a Cloudinary egyedi azonosítót `partners/radisson_blu_logo`. Ezzel azonosítóval bármilyen formátumban, minőségben és méretben lekérheted a képet.

## Tesztelés

Látogass el az [API-k](/hu/developers/apis) szekciónkba, és válaszd ki a használni kívánt API-t.

### API dokumentáció

Dokumentációs oldalunk úgy van kialakítva, hogy az API parancsokat közvetlenül az oldalon belül futtathatod.

Ebben a példában a `Search Lookups (OAuth2)` API hívást szeretnéd használni.

:::tip[Pro Tipp]
A könnyebb használat érdekében azt javasoljuk, hogy az integrátorok az általunk `OAuth2`-ként megjelölt végpontokat használják. Ezek a végpontok tudják, hogy API hívók érkeznek, és már ismerik, melyik fiókban vagy, így nem kell a `companyIdentifier`-t megadnod az URL-ekben.
:::

<Steps>
1. Használd a [Lookup API-t](https://developer.wink.travel/lookup) úti célok vagy szállodák lekéréséhez.
2. A bal oldali navigációs sávban kattints az `Authentication` fülre.
3. Írd be a client-id-det és a titkos kulcsodat, majd kattints a `Get TOKEN` gombra.
4. Ha a hitelesítő adataid érvényesek, látni fogod, hogy az API kulcsod alkalmazásra került.
5. Most már tesztelheted a lookup végpontot.
</Steps>

### Postman
Ugyanezt megteheted Postman-nel is.

<Steps>
1. Töltsd le a [Postman-t](https://www.postman.com/downloads).
2. Kattints a `Download OpenAPI spec` gombra a [Lookup API](https://developer.wink.travel/lookup) oldal tetején.
3. Importáld be a Postman-be.
4. Ha új kollekcióként importáltad `Wink API` néven, kattints a kollekció neve melletti vízszintes három pontra, majd válaszd az `Edit` opciót.
5. Az `Authorization` fülön állítsd az `Auth type`-ot OAuth2-re.
6. Állítsd be az `Add auth data to` mezőt `Request headers`-re.
7. Állítsd be a `Header prefix` mezőt `Bearer`-re.
8. Add meg a `Token Name` mezőben, hogy milyen néven szeretnéd menteni a tokent a Postman-ben, pl. Wink Staging.
9. Állítsd be a `Grant type` mezőt `Client Credentials`-re.
10. Állítsd be az `Access token URL` mezőt erre: https://staging-iam.wink.travel/oauth2/token.
11. Állítsd be a `Client ID` mezőt az alkalmazásod által generált Client ID-re.
12. Állítsd be a `Client Secret` mezőt az alkalmazásod által generált Client Secret-re.
13. Állítsd be a `Scope` mezőt `inventory.read inventory.write inventory.remove` értékre. Ezek a Wink-hez szükséges engedélyek.
14. Állítsd be a `Client Authentication` mezőt `Send as Basic Auth header`-re.
15. Kattints a `Get New Access Token` gombra, hogy lekérd és elmentsd a tokent.
16. A Wink API kollekció összes hívásánál győződj meg róla, hogy az authentication fülön az `Inherit auth from parent` van beállítva, így minden hívásnál elküldi a tokent.
</Steps>

:::tip[Pro Tipp]
Ajánlott a szerver domainjét, a client ID-t és a client secret-et környezeti változókká tenni. Így könnyen válthatsz a staging és a termelési környezet között, és újrahasznosíthatod az összes létrehozott műveletet.
:::
