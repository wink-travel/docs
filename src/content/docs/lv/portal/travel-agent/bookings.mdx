Kad platformā Wink tiek veikta rezervācija, notiek sekojošais:

<Steps>
1. Jūsu kanālu pārvaldnieks, CRS vai PMS tiek informēts.
2. Jūs saņemat apstiprinājuma e-pastu *(rezervāciju nodaļas kontakti)* ar saiti, lai pārvaldītu jauno rezervāciju.
3. Ceļotājs saņem apstiprinājuma e-pastu, kurā ir arī informācija, kā sazināties ar jūsu rezervāciju nodaļu.
4. Ceļojumu aģents saņem to pašu rezervācijas apstiprinājumu kā ceļotājs.
4. Izstrādātāji var klausīties `booking.created` webhook notikumu un saņemt no mums detalizētu rezervācijas informāciju reāllaikā.
</Steps>

Seko līdzi saitei e-pastā vai dodies uz `Account > Bookings` galvenajā navigācijas joslā, lai sāktu.

## Rezervāciju režģis

Rezervāciju režģis parāda visas jūsu pagātnes un nākotnes rezervācijas, sakārtotas pēc rezervācijas datuma augošā secībā.

### Filtri

Jūs varat izmantot filtrus, lai atrastu meklētās rezervācijas.

- Rezervācijas datums [diapazons]
- Ierašanās datums [diapazons]
- Izbraukšanas datums [diapazons]
- Rezervācijas kods
- Ceļotāja uzvārds
- Ceļotāja vārds
- Galvenā tarifa nosaukums
- Naktsmītnes nosaukums

### Eksports

Jūs varat eksportēt rezervācijas, kas atbilst jūsu pašreizējiem filtru kritērijiem, uz CSV.

Lai eksportētu savas rezervācijas:

<Steps>
1. Pārliecinieties, ka ir aktīvi vēlamie filtri.
2. Gaidiet, kamēr rezervāciju režģis atsvaidzinās.
2. Noklikšķiniet uz 🗂️ (*Faila eksports*) ikonas pogas režģa apakšējā labajā stūrī.
</Steps>

## Rezervācija

Izvēlieties rezervāciju, ar kuru vēlaties strādāt, no rezervāciju režģa, un jūs tiksiet novirzīts uz `Booking` lapu.
Šajā lapā ir visi nepieciešamie dati, lai pienācīgi sagaidītu savu viesi ierašanās dienā.

Lapa arī satur dažas svarīgas darbības, kas nepieciešamas, lai pienācīgi apkalpotu jūsu viesus. Visas pieejamās darbības var atrast nolaižamajā izvēlnē `Actions` lapas augšpusē.

### Pārsūtīt e-pastu

Lai pārsūtītu apstiprinājuma e-pastu ceļotājam, noklikšķiniet uz `Actions > Resend confirmation email`.

### Atcelt rezervāciju

Lai atceltu rezervāciju kā naktsmītne, noklikšķiniet uz `Actions > Cancel booking`.

Tiks parādīts uznirstošais logs, kurā varat:

<Steps>
1. Izvēlēties atcelšanas veidu. *piemēram, Nepareizi datumi*
2. Ierakstīt atcelšanas iemeslu.
</Steps>

*Šī poga darbojas saskaņā ar jūsu atcelšanas politiku un tāpēc ne vienmēr būs pieejama.*

## API

Izstrādātāji, kas vēlas pārvaldīt `Bookings` kā ceļojumu aģenti, var doties uz [Developers > API > Travel Agent](/lv/api).
