## Pārskats
Mitinātais Wink MCP serveris ir pieejams šeit:

```
https://docs.mcp.wink.travel/mcp
```

Norādiet saderīgus AI klientus (VS Code, Cursor, citi, kas īsteno MCP HTTP transportu) uz šo galapunktu, lai tie varētu:
- Pārlūkot produktu un platformas dokumentāciju (`wink://docs/...`)
- Pārlūkot tiešraides OpenAPI specifikācijas un darbību dokumentus (`wink://api/...`)
- Veikt vieglu semantisko / atslēgvārdu meklēšanu dokumentācijā un tiešraides API

## Transports
Izmantojiet galveno straumējamo HTTP galapunktu:

```
https://docs.mcp.wink.travel/mcp
```

Ja jūsu klients atbalsta tikai SSE, izmantojiet:

```
https://docs.mcp.wink.travel/sse
```

### Claude Code
Oficiālā dokumentācija: [Savienojiet Claude Code ar rīkiem, izmantojot MCP](https://code.claude.com/docs/en/mcp)

Anthropic iesaka attālināto HTTP transportu mākoņos mitinātiem MCP serveriem:
```bash
claude mcp add --transport http wink-docs https://docs.mcp.wink.travel/mcp
claude mcp list
```

Claude Code iekšienē palaidiet `/mcp`, lai pārbaudītu servera statusu. Ja vēlaties koplietot serveri ar visu repozitoriju, pievienojiet `--scope project`, lai Claude Code ierakstītu konfigurāciju `.mcp.json`.

### Codex
Oficiālā dokumentācija: [Docs MCP ātrā sākšana Codex](https://developers.openai.com/resources/docs-mcp)

OpenAI dokumentē MCP iestatīšanu caur Codex CLI un norāda, ka konfigurācija tiek koplietota ar IDE paplašinājumu:
```bash
codex mcp add wink-docs --url https://docs.mcp.wink.travel/mcp
codex mcp list
```

### Cursor
Cursor MCP dokumentācija: https://cursor.com/docs/context/mcp
Iestatīšana: Pievienojiet attālināto serveri, izmantojot MCP iestatījumus.
```json
{
  "mcpServers": {
    "wink-docs": {
      "url": "https://docs.mcp.wink.travel/mcp",
    }
  }
}
```

### VS Code
Oficiālā dokumentācija: https://code.visualstudio.com/docs/copilot/customization/mcp-servers
Minimāla attālinātā konfigurācija (lietotāja vai darba vietas `mcp.json`):
```json
{
  "servers": {
    "wink-docs": {
      "type": "http",
      "url": "https://docs.mcp.wink.travel/mcp"
    }
  }
}
```

Padomi:
- Izmantojiet Komandu paleti: “MCP: List Servers”, lai apstiprinātu savienojumu un pārlūkotu rīkus.
- Rīku izvēlētājs ļauj iespējot/atspējot `api_search` un `docs_search`.
- Pievienojiet kontekstu: Čata skats → Pievienot kontekstu → MCP resursi, lai pievienotu dokumentu lapas.

## Galvenie resursu URI

| URI | Apraksts |
|-----|----------|
| `wink://docs` | Dokumentācijas indekss (grupēts pēc kategorijas) |
| `wink://docs/{path}` | Konkrēts dokumentācijas raksts (Markdown / MDX) |
| `wink://api` | Tiešraides OpenAPI katalogs, atklāts no Wink swagger konfigurācijas |
| `wink://api/{apiName}` | Kompakts API kopsavilkums ar autentifikāciju, tagiem, galvenēm un paraugu operācijām |
| `wink://api/{apiName}/operations` | Pilns darbību saraksts, grupēts pēc taga |
| `wink://api/{apiName}/operation/{key}` | Atsevišķas darbības detaļas (parametri, pieprasījuma saturs, atbildes) |
| `wink://api/{apiName}/schema/{schemaName}` | Atrisināta komponentes shēma ar iekšējiem laukiem, tipiem un piemēriem |
| `wink://api/{apiName}/operation/{key}/request` | Pieprasījuma skats ar nepieciešamajiem ievades datiem un minimālu piemēru |
| `wink://api/{apiName}/operation/{key}/response/{status}` | Atbildes skats konkrētam statusa kodam |
| `wink://api/{apiName}/operation/{key}/recipe` | Uzdevuma orientēts plūsmas apraksts, kā pareizi izsaukt šo darbību |

## Meklēšanas rīki (ātrā atsauce)
| Rīks | Mērķis | Tipiski vaicājumu piemēri |
|------|--------|-------------------------|
| `api_search` | Atrod tiešraides API vai darbību integrācijai | `create affiliate`, `payment receipt`, `lookup inventory`, `invite manager` |
| `docs_search` | Atrod konceptuālus dokumentus / ceļvežus | `authentication`, `webhooks`, `wordPress` |

Atstājiet `query` tukšu, lai saņemtu ieteikumus. Katrs rezultāts ietver MCP URI, ko var atvērt.
Darbību rezultātos `api_search` arī atgriež saistītos pieprasījuma, atbildes, shēmas un receptes URI, lai aģenti varētu tieši pāriet uz visnoderīgāko apakšresursu.

## Tipiska lietošanas kārtība
1. Izpildiet `api_search` ar savu nodomu (piemēram, *"create affiliate"* vai *"payment receipt"*).
2. Atveriet atgriezto `wink://api/...` URI vai sekojiet tā saistītajiem `request_uri`, `response_uris`, `request_schema_uris`, `response_schema_uris` un `recipe_uri` laukiem.
3. Izmantojiet `docs_search`, lai atrastu atbalstošas koncepcijas (autentifikācija, webhooks, darbplūsmas).

## AI prasmes

Iepriekš izveidotās aģentu prasmes izmanto šo MCP serveri kā savu patiesības avotu. Prasmes nodrošina jūsu kodēšanas aģentam strukturētu darbplūsmu, lai veidotu risinājumus, balstoties uz Wink API — nevis minot galapunktus, aģents tieši lasa tiešraides līgumus no MCP servera.

Skatiet [AI Skills](/lv/developers/ai-skills) pieejamās prasmes un instalēšanas instrukcijas.

## Piezīmes
- Katru atgriezto URI aģents var atvērt pilnai satura apskatei.
- `wink://api` koks ir tiešraides un seko pašreizējam OpenAPI katalogam, nevis ģenerētam repozitorija indeksam.
- Ieteikumi parādās automātiski, ja vaicājumi ir tukši.
