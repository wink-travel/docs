## Pārskats
Mitinātais Wink MCP serveris ir pieejams šeit:

```
https://docs.mcp.wink.travel/mcp
```

Norādiet saderīgus AI klientus (VS Code, Cursor, citi, kas īsteno MCP HTTP transportu) uz šo galapunktu, lai tie varētu:
- Pārlūkot produktu un platformas dokumentāciju (`wink://docs/...`)
- Atklāt SDK un izpētīt API klases / modeļus (`wink://sdk/...` un `wink://sdk/.../docs`)
- Veikt vieglu semantisko / atslēgvārdu meklēšanu dokumentācijā un SDK kopsavilkumos

## Transports
Izmantojiet galveno straumējamo HTTP galapunktu:

```
https://docs.mcp.wink.travel/mcp
```

Ja jūsu klients atbalsta tikai SSE, izmantojiet:

```
https://docs.mcp.wink.travel/sse
```

### Cursor
Cursor MCP dokumentācija: https://cursor.com/docs/context/mcp
Iestatīšana: Pievienojiet attālo serveri, izmantojot MCP iestatījumus.
```json
{
  "mcpServers": {
    "wink-docs": {
      "url": "https://docs.mcp.wink.travel/mcp",
    }
  }
}
```

### VS Code
Oficiālā dokumentācija: https://code.visualstudio.com/docs/copilot/customization/mcp-servers
Minimāla attālā konfigurācija (lietotāja vai darba vietas `mcp.json`):
```json
{
  "servers": {
    "wink-docs": {
      "type": "http",
      "url": "https://docs.mcp.wink.travel/mcp"
    }
  }
}
```

Padomi:
- Izmantojiet Komandu paleti: “MCP: List Servers”, lai apstiprinātu savienojumu un pārlūkotu rīkus.
- Rīku izvēlne ļauj iespējot/atspējot `sdk_search` un `docs_search`.
- Pievienojiet kontekstu: Čata skats → Pievienot kontekstu → MCP resursi, lai pievienotu dokumentu lapas.

## Galvenie resursu URI

| URI | Apraksts |
|-----|----------|
| `wink://docs` | Dokumentācijas indekss (grupēts pēc kategorijas) |
| `wink://docs/{path}` | Konkrēts dokumentācijas raksts (Markdown / MDX) |
| `wink://sdk` | SDK pārskats (katrs SDK + dokumentācija) |
| `wink://sdk/{sdkPath}` | SDK README / moduļa dokumentu mape / konkrēts moduļa dokuments |

### Moduļu dokumentācija
Atverot SDK dokumentu mapi (piemēram, `wink://sdk/python/wink-sdk-booking/docs`), tiek atgriezts grupēts saturs:
- **API klases** (metožu skaits + piemēru metožu nosaukumi)
- **Modeļi un tipi** (datu struktūras, ko izmanto šie API)

## Meklēšanas rīki (ātra atsauce)
| Rīks | Mērķis | Tipiski vaicājumu piemēri |
|------|--------|--------------------------|
| `sdk_search` | Atrod, kuru SDK / API klasi izmantot | `cancel booking`, `list properties`, `affiliate inventory` |
| `docs_search` | Atrod konceptuālos dokumentus / ceļvežus | `authentication`, `webhooks`, `wordPress` |

Atstājiet `query` tukšu, lai saņemtu ieteikumus. Katrs rezultāts ietver MCP URI, ko var atvērt.

## Tipiska lietošanas kārtība
1. Izpildiet `sdk_search` ar savu nodomu (piemēram, *"list properties"*).
2. Atveriet saņemto SDK README vai tā `docs_uri`, lai izpētītu API / modeļus.
3. Izmantojiet `docs_search`, lai atrastu atbalstošos konceptus (autentifikācija, webhooks).

## Piezīmes
- Katru saņemto URI aģents var atvērt pilnai saturam.
- Ieteikumi parādās automātiski, ja vaicājumi ir tukši.
