[Tīmekļa komponentes](https://developer.mozilla.org/en-US/docs/Web/API/Web_components) ir standarts, kas ļauj integrēt pilnas JavaScript funkcijas jūsu vietnē, nezinot programmēšanu.
Pateicoties šai foršajai tehnoloģijai, varat ļoti vienkārši iegult Wink ceļojumu inventāru. Šis raksts iepazīstina ar mūsu tīmekļa komponentu kolekciju un parāda, kā tos izmantot.

## Iegult Wink

Ir trīs lietas, kas jāiekļauj jebkurā lapā, kur vēlaties parādīt kādu no mūsu tīmekļa komponentēm.

1. Iekļaut mūsu stilu lapu.
2. Iekļaut mūsu JavaScript.
3. Iekļaut mūsu lietojumprogrammas ielādētāju.

### Stilu lapa

Iekļaujiet mūsu CSS stilus dokumenta galvā.

```html {3}
<html>
  <head>
    <link rel="stylesheet" href="https://elements.wink.travel/styles.css"></link>
  </head>
</html>
```

### JavaScript

Iekļaujiet mūsu JavaScript dokumenta apakšā. (*Mēs iesakām tieši virs &lt;/body&gt; beigu taga*).

```html {3}
<html>
  <body>
    <script src="https://elements.wink.travel/elements.js" type="module" defer></script>
  </body>
</html>
```

Jūsu vietne tagad ir gatava un spēj parādīt mūsu tīmekļa komponentes.

## Komponentes

Lasiet par mūsu komponentu bibliotēku zemāk.

### Lietojumprogrammas ielādētājs

Ielādētājs ir atbildīgs par stāvokļa uzturēšanu un mijiedarbības pārvaldību starp mūsu komponentēm. (*Iekļaujiet to zem mūsu JavaScript*).

```html {3-6}
<html>
  <body>
    <wink-app-loader
      client-id="JŪSU KLIENTA ID ŠEIT"
      configuration-id="JŪSU PIELĀGOJUMA ID ŠEIT"
    ></wink-app-loader>
  </body>
</html>
```

### Satura ielādētājs

Satura ielādētājs ir mūsu tīmekļa komponentu bibliotēkas sirds. Tas ir atbildīgs par jūsu inventāra attēlošanu (kartes, režģi, kartes).

Pieejamie atribūti:

- layout
- id
- sort

*Atribūts `sort` ir pieejams tikai tad, ja layout ir `RANKED` un jūs nevēlaties izmantot esošu rangotu režģi. Šādos gadījumos atstājiet `id` tukšu.*

Pieejamie izkārtojuma veidi:

- `AD_BANNER`
- `MAP`
- `HOTEL`
- `GUEST_ROOM`
- `MEETING_ROOM`
- `SPA`
- `RESTAURANT`
- `ACTIVITY`
- `ATTRACTION`
- `PLACE`
- `ADD_ON`
- `LIST`
- `SEARCH`
- `RANKED`

Pieejamie kārtošanas veidi:

- `MEMBER`
- `PRICE_LOW_TO_HIGH`
- `PRICE_HIGH_TO_LOW`
- `PRICE`
- `POPULARITY`
- `ECO`
- `EXPERIENCE`
- `PERK`
- `LOYALTY`
- `PACKAGE`

```html {3-6}
<html>
  <body>
    <wink-content-loader
       layout="GUEST_ROOM"
       id="2de7ee9c-61c9-11ef-9722-42004e494300"
    ></wink-content-loader>
  </body>
</html>
```

Paraugus var atrast [kartēm](/lv/studio/cards), [režģiem](/lv/studio/grids) un [kartēm](/lv/studio/maps).

## Iegult TripPay

Ir divas lietas, kas jāiekļauj jebkurā lapā, kur vēlaties izmantot TripPay maksājumu tīmekļa komponenti.

1. Iekļaut mūsu stilu lapu.
2. Iekļaut mūsu JavaScript.

### Stilu lapa

Iekļaujiet mūsu CSS stilus dokumenta galvā.

```html {3}
<html>
  <head>
    <link rel="stylesheet" href="https://elements.trippay.io/styles.css"></link>
  </head>
</html>
```

### JavaScript

Iekļaujiet mūsu JavaScript dokumenta apakšā. (*Mēs iesakām tieši virs &lt;body&gt; beigu taga*).

```html {3}
<html>
  <body>
    <script src="https://elements.trippay.io/elements.js" type="module" defer></script>
  </body>
</html>
```

Jūsu vietne tagad ir gatava un spēj parādīt mūsu tīmekļa komponentes.

## Komponentes

### Maksājums

Maksājumu komponente ļauj informēt TripPay, ka ceļotājs vēlas kaut ko iegādāties, kā arī noteikt noteikumus un cenas šiem priekšmetiem.

Ir viens obligāts atribūts logrīkā:

- `id` Rezervācijas līguma identifikators, kuru vēlaties izpildīt.

```html {3}
<html>
  <body>
    <trip-pay id="<INSERT_BOOKING_CONTRACT_ID>"></trip-pay>
  </body>
</html>
```

:::note
Lai uzzinātu, kā ģenerēt rezervācijas līguma `id`, izlasiet ceļvedi: [Integrācija ar TripPay](/lv/guides/integrating-with-trip-pay).
:::

Logrīks sagatavo līgumu izpildei un parāda maksājuma detaļas *(1. attēls)* lietotājam, lai pabeigtu rezervāciju.

## Kļūdas

Ja, ieguldot kādu no mūsu tīmekļa komponentēm, gan WinkLinks, gan jūsu pašu vietnē, saņemat kļūdas ziņojumu, var būt dažas iespējamās problēmas:

### Pieejamība

Inventārs var vairs nebūt pieejams. Dodieties uz [Wink Studio](/lv/studio/cards) un pārbaudiet, vai "kartes" statuss ir pieejams.
Ja nav, tas tiks parādīts sarkanā krāsā. Šādā gadījumā varat pagaidīt, un varbūt piegādātājs to atkal padarīs pieejamu, vai arī noņemt to no sava saraksta.

:::note
`Pieejamība` nenozīmē ikdienas rezervējamo inventāra pieejamību. Tas nozīmē, ka inventārs jums un/vai visiem ir pilnīgi nepieejams.
:::

### Noņemts

Inventārs var būt noņemts. Dodieties uz [Wink Studio](/lv/studio/cards) un pārbaudiet, vai "kartes" statuss ir pieejams.
Ja nav, tas tiks parādīts sarkanā krāsā. Šādā gadījumā varat pagaidīt, un varbūt piegādātājs to atkal padarīs pieejamu, vai arī noņemt to no sava saraksta.

### Pielāgošana

Varbūt nejauši esat noņēmis pielāgošanu, ko esat definējis kopā ar savu inventāru. Pārliecinieties, ka pielāgošana joprojām ir pieejama, un, ja kāda trūkst, iestatiet jaunu.

### Lietojumprogramma

Tas attiecas tikai uz tīmekļa izstrādātājiem. Ja nejauši esat noņēmis [Lietojumprogrammu](/lv/developer/applications), tādējādi klienta ID vairs nav pieejams, lūdzu, izveidojiet jaunu lietojumprogrammu un izmantojiet jauno klienta ID, lai iegultu mūsu tīmekļa komponentes.
