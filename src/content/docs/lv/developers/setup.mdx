Šis raksts parāda, kā ātri uzstādīties kā izstrādātājam un sākt testēt mūsu API vietnē Wink.

:::note
Testēšana vienmēr jāveic mūsu [testēšanas vidē](/getting-started/environments). Tur ir daudz mazāk datu, taču jūs varat brīvi testēt tik daudz, cik vēlaties, neietekmējot ražošanas vidi.

Kad esat gatavs palaist tiešraidē, mēs sertificēsim jūsu kontu un jūsu lietotni, un jūs varēsiet palaist to mūsu ražošanas vidē.
:::

Izpildiet šos soļus:

<Steps>
1. [Reģistrējiet](/user/register) savu lietotāja kontu vietnē Wink
2. Izveidojiet [ceļojumu aģenta kontu](/travel-agent/what-is-agent) vai [afilācijas kontu](/studio/what-is-studio).
3. [Izveidojiet lietotni](/developer/applications), kurā saistāt to ar kontu, ko izveidojāt 2. solī.
4. Kad esat izveidojis savu `Application`, saglabājiet klienta ID un klienta slepeno atslēgu drošā vietā.
</Steps>

## CDN

Wink izmanto Cloudinary visiem mūsu attēlu un video resursiem. Piekļūstot mūsu ceļojumu inventāram, jums būs pilnīga piekļuve visām Cloudinary uzlabotajām funkcijām.

Mūsu `cloud_name` ir `traveliko`.

### Cloudinary

- [Sākt darbu](https://cloudinary.com/documentation/dev_kickstart)
- [SDK](https://cloudinary.com/documentation/cloudinary_sdks)
- [Cloudinary GitHub](https://github.com/cloudinary)
-	[Video pamācības](https://cloudinary.com/documentation/cloudinary_sdks_tutorials)

### Piemērs

Katram ceļojumu inventāra veidam, ko atbalstām, būs iekšējs JSON masīvs ar nosaukumu `multimedias`. Multimediju ieraksta JSON izskatās šādi:

```json {3}
{
  "multimediaIdentifier": "multimedia-1",
  "identifier": "partners/radisson_blu_logo",
  "type": "IMAGE",
  "width": "1024",
  "height": "768",
  "category": "1",
  "descriptions": [
    {
      "name": "Lobby",
      "description": "Picture of the lobby",
      "language": "en"
    }
  ],
  "lifestyleType": "LIFESTYLE_BUSINESS",
  "attribution": [
    {
      "url": "https://www.hilton.com",
      "name": "Hilton"
    }
  ]
}
```

Augstāk izceltais rindiņa rāda Cloudinary unikālo ID `partners/radisson_blu_logo`. Tas ir viss, kas jums nepieciešams, lai izgūtu attēlu jebkurā formātā, kvalitātē un izmērā, kādu vēlaties.

## Testēšana

Dodieties uz mūsu [API](/developer/apis) sadaļu un izvēlieties API, ar kuru vēlaties strādāt.

### API dokumentācija

Mūsu dokumentācijas vietne ir izveidota tā, lai jūs varētu izpildīt API komandas tieši no vietnes.

Šajā piemērā vēlaties izmantot `Search Lookups (OAuth2)` API izsaukumu.

:::tip[Pro padoms]
Lai atvieglotu lietošanu, mēs iesakām integrētājiem izmantot galapunktus, kurus mēs apzīmējam kā `OAuth2`. Šie galapunkti zina, ka gaida API izsaucējus un jau zina, kurā kontā atrodaties, tāpēc jums nav jānorāda `companyIdentifier` jūsu URL.
:::

<Steps>
1. Izmantojiet [Lookup API](https://developer.wink.travel/lookup), lai izgūtu galamērķus vai viesnīcas.
2. Kreisajā navigācijas joslā noklikšķiniet uz `Authentication`.
3. Ievadiet savu klienta ID un klienta slepeno atslēgu un noklikšķiniet uz pogas `Get TOKEN`.
4. Ja jūsu akreditācijas dati ir derīgi, redzēsiet, ka jūsu API atslēga ir piemērota.
5. Tagad varat testēt lookup galapunktu.
</Steps>

### Postman
To pašu var izdarīt ar Postman.

<Steps>
1. Lejupielādējiet [Postman](https://www.postman.com/downloads).
2. Noklikšķiniet uz `Download OpenAPI spec` lapas augšpusē [Lookup API](https://developer.wink.travel/lookup) lapā.
3. Importējiet to Postman.
4. Ja importējāt to kā jaunu kolekciju ar nosaukumu `Wink API`, noklikšķiniet uz horizontālās elipses (3 punkti) blakus kolekcijas nosaukumam un izvēlieties `Edit`.
5. Cilnē `Authorization` iestatiet `Auth type` uz OAuth2.
6. Iestatiet lauku `Add auth data to` uz `Request headers`.
7. Iestatiet lauku `Header prefix` uz `Bearer`.
8. Iestatiet lauku `Token Name` uz jebkuru nosaukumu, ko vēlaties piešķirt saglabātajam tokenam Postman. Piemēram, Wink Staging.
9. Iestatiet lauku `Grant type` uz `Client Credentials`.
10. Iestatiet lauku `Access token URL` uz https://staging-iam.wink.travel/oauth2/token.
11. Iestatiet lauku `Client ID` uz klienta ID, ko jūsu lietotne palīdzēja ģenerēt.
12. Iestatiet lauku `Client Secret` uz klienta slepeno atslēgu, ko jūsu lietotne palīdzēja ģenerēt.
13. Iestatiet lauku `Scope` uz `inventory.read inventory.write inventory.remove`. Tie ir visi nepieciešamie piekļuves līmeņi Wink.
14. Iestatiet lauku `Client Authentication` uz `Send as Basic Auth header`.
15. Noklikšķiniet uz pogas `Get New Access Token`, lai izgūtu savu tokenu un saglabātu to.
16. Visos API izsaukumos Wink API kolekcijā pārliecinieties, ka autorizācijas cilnē ir norādīts `Inherit auth from parent`, lai katrā izsaukumā tiktu iekļauts jūsu tokens.
</Steps>

:::tip[Pro padoms]
Ieteicams servera domēnu, klienta ID un klienta slepeno atslēgu padarīt par vides mainīgajiem. Tā jūs varat viegli pārslēgties starp testēšanas un ražošanas vidēm un atkārtoti izmantot visus izveidotos operācijas.
:::
