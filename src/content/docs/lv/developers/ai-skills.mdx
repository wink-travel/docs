AI prasmes ir pārnēsājamas instrukciju pakas, kas māca kodēšanas aģentiem, kā strādāt ar Wink. Tā vietā, lai paļautos uz apmācības datiem vai novecojušu dokumentāciju, katra prasme norāda aģentam izmantot [Wink MCP serveri](/lv/developers/mcp) kā tiešo patiesības avotu API līgumiem un platformas dokumentācijai.

Prasmes darbojas ar jebkuru aģenta klientu, kas tās atbalsta — Claude Code, Codex, Cursor un citiem.

## Pieejamās prasmes

### wink-api-builder

Ieviest, pārskatīt vai atkļūdot Wink API integrācijas. Prasme vada aģentu caur strukturētu darba plūsmu:

1. Meklēt pareizo API operāciju, izmantojot `api_search`
2. Atvērt tiešos pieprasījuma/atbildes līgumus no MCP servera
3. Iegūt autentifikācijas iestatījumus no dokumentācijas
4. Rakstīt kodu, balstoties uz pārbaudīto līgumu
5. Atzīmēt jebkādas neatbilstības starp esošo kodu un pašreizējo API

Noderīgi:
- Jaunu klienta metožu izveidei jebkuram Wink API
- Līgumu neatbilstību atkļūdošanai (piemēram, pieprasījums atgriež HTTP 400)
- Esošo integrāciju pārskatīšanai pret tiešo specifikāciju

## Instalācija

Kopējiet prasmes mapi savas aģenta prasmju direktorijā.

:::tip
Wink MCP serverim jābūt pieslēgtam tajā pašā vidē. Skatīt [MCP Server](/lv/developers/mcp) iestatīšanas instrukcijas.
:::

**Claude Code**

```bash
# repozitorija līmenī
cp -r skills/wink-api-builder .claude/skills/

# vai lietotāja līmenī
cp -r skills/wink-api-builder ~/.claude/skills/
```

Izsauciet ar:
```
/wink-api-builder add a TypeScript client for the Wink affiliate create flow
```

**Codex**

```bash
# repozitorija līmenī
cp -r skills/wink-api-builder .agents/skills/

# vai lietotāja līmenī
cp -r skills/wink-api-builder $CODEX_HOME/skills/
```

Izsauciet ar:
```
Use $wink-api-builder to add a TypeScript client for the Wink affiliate create flow.
```

**Citi aģenti**

Novietojiet prasmes mapi tur, no kurienes jūsu aģents ielādē prasmes. Ieejas punkts ir `SKILL.md`.
