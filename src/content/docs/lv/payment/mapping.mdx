Ja esat integrators, piemēram, Wink, un vēlaties kartēt savus ārējos maksājumu saņēmējus lietošanai ar TripPay, varat to izdarīt vienā no diviem veidiem:

* Manuāla kartēšana
* Izmantojot[Maksājumu API](/developers/apis/#payment-api)(*Skatīt`Account Mappings`*)

:::tip
Ja vien jums nav darīšana tikai ar dažiem vienādiem īpašumiem, iesakām izmantot API, lai automatizētu šo darbību.
:::

Tālāk mēs soli pa solim parādīsim, kā manuāli kartēt ārējo maksājuma saņēmēju.*Tas izmanto API zem pārsega.*

<Steps>
  1. Pieslēgties [TripPay](https://pay.wink.travel).
  2. Atlasiet savu integratora kontu.
  3. Klikšķis `Settings > Mappings` no galvenās navigācijas joslas.
  4. Noklikšķiniet uz `Create mapping` pogu. Jūs redzēsiet kartēšanas veidlapu.
  5. **Lokālais konts** Sāciet rakstīt esošā TripPay nosaukumu, kuru vēlaties piesaistīt.
  6. **Ārējais identifikators** Ielīmējiet šīs entītijas ārējo ID tā, kā tas pastāv attālajā sistēmā.
  7. Klikšķis `Save` lai turpinātu. Jūs tiksiet novirzīts atpakaļ uz savu kartējumu sarakstu.
</Steps>


Kad jūsu kartējumi ir ieviesti, TripPay tagad zina, kurus kontus ieskaitīt kā rezervācijas saņēmējus, ja uz tiem atsaucaties, izmantojot savus ārējos identifikatorus.

## API

Izstrādātāji, kuri vēlas pārvaldīt`Payments`, dodieties uz[Izstrādātāji > API > Maksājumi](/developers/apis/#payment-api).
