Customizations control how your inventory is displayed to travelers — the theme colors, default currency, language, itinerary, and more. They apply across shareable links, embedded cards, grids, maps, and the booking engine.

## What is a customization?

A customization is a configuration profile that defines:

- **Display currency** — What currency prices are shown in.
- **Display language** — The default language for content.
- **Default itinerary** — Pre-filled check-in/out dates and guest count.
- **Theme** — Colors and branding applied to embedded components.
- **Configuration** — How inventory is presented (layout, sorting).

You can create multiple customizations for different markets or use cases.

## Creating a customization

<Steps>
1. Go to https://studio.wink.travel (or your account on https://my.wink.travel).
2. Navigate to `Customization` from the sidebar.
3. Click `Create a new customization`.
4. Give it a descriptive name (e.g., "Europe - EUR", "Japan Market", "Default").
5. Set the display currency, language, and default itinerary.
6. Configure theme colors to match your branding.
7. Click `Save`.
</Steps>

## Where customizations are used

Once created, customizations are referenced across the platform:

| Feature | How customization applies |
| -- | -- |
| **Shareable links** | The link uses the customization's currency, language, and itinerary as defaults |
| **Embedded cards** | Cards render with your theme colors and display settings |
| **Grids and maps** | Inventory grids and map markers follow the customization |
| **Booking engine** | The entire booking flow uses the customization's settings |
| **Social posts** | AI-generated posts can use a specific customization for pricing display |

## Multiple customizations

A common pattern is to create one customization per target market:

- **"US Market"** — USD, English, default 2-night stay.
- **"Europe"** — EUR, English, default 3-night stay.
- **"Japan"** — JPY, Japanese, default 1-night stay.

When you create a shareable link or embed a card, you select which customization to use. This means the same property can be presented differently depending on the audience.

## Overriding via URL

You can also override customization defaults using URL parameters on shareable links:

- `l=es` — Override the language to Spanish.
- `c=EUR` — Override the currency to EUR.
- `sd=2024-08-24&n=2` — Override the itinerary to a specific date and number of nights.

See [The Book Now Button](/guides/the-book-now-button) for the full list of URL parameters.
