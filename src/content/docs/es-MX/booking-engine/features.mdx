El motor de reservas viene en dos versiones para apoyar las diferentes necesidades de un afiliado, una empresa de administración de propiedades y un hotel.

## Agencia de viajes en línea

Para un afiliado, tiene más sentido usar la versión de agencia de viajes en línea. La razón es que el sitio está configurado para acomodar a viajeros que desean ver múltiples propiedades.

Puedes encontrar la página principal de esta aplicación a continuación.

<LinkButton href="https://ota.wink.travel/home" variant="secondary" icon="external" target="_blank">
  Wink OTA
</LinkButton>

## Motor de reservas por internet

Para un hotel o una empresa de administración de propiedades, tiene más sentido usar la versión del motor de reservas por internet. Este tipo de usuario está interesado en destacar una sola propiedad o múltiples propiedades bajo administración.

Puedes encontrar la página principal de esta aplicación a continuación.

## Configuración

La mayoría de las veces, no es necesario compartir directamente ninguna de estas aplicaciones. Hemos creado varias formas para que compartas inventario de viajes, que puedes descubrir en [Wink Studio](/studio/what-is-studio) o a través de [WinkLinks](/link-manager/wink-links).

Sin embargo, si deseas enlazar manualmente a uno de estos sitios web, asegúrate de agregar tu ID de cliente y el ID de personalización a la URL como parámetros.

Puedes encontrar tu ID de cliente haciendo lo siguiente:

<Steps>
1. Inicia sesión en [Wink Extranet](/extranet/what-is-extranet) o en [Wink Studio](/studio/what-is-studio).
2. Haz clic en el ícono de tu perfil en la esquina superior derecha y selecciona [Aplicaciones](/developers/applications).
3. Usa la aplicación que esté vinculada a la cuenta Wink con la que quieres trabajar.
4. Copia el ID de Cliente de esa aplicación y pégalo en la URL como un parámetro de solicitud. Ejemplo: ?client-id=MI_ID_DE_CLIENTE
</Steps>

### Extranet

Puedes encontrar tu ID de personalización como propiedad haciendo lo siguiente:

<Steps>
1. Inicia sesión en [Wink Extranet](/extranet/what-is-extranet).
2. Selecciona la propiedad con la que quieres trabajar.
3. En la barra de navegación principal, haz clic en `Cuenta > Branding`
4. Copia el ID de Personalización y pégalo en la URL como un parámetro de solicitud. Ejemplo: ?configuration-id=MI_ID_DE_PERSONALIZACIÓN
</Steps>

### Studio

Puedes encontrar tu ID de personalización como afiliado haciendo lo siguiente:

<Steps>
1. Inicia sesión en [Wink Studio](/studio/what-is-studio).
2. Selecciona la cuenta con la que quieres trabajar.
3. En la barra de navegación principal, haz clic en `Herramientas > Personalización`
4. Copia el ID de Personalización de la personalización que quieres usar y pégalo en la URL como un parámetro de solicitud. Ejemplo: ?configuration-id=MI_ID_DE_PERSONALIZACIÓN
</Steps>

## OTA

Lo que obtienes es una URL que se ve algo así:

https://ota.wink.travel?client-id=MI_ID_DE_CLIENTE&configuration-id=MI_ID_DE_PERSONALIZACIÓN

Si quieres ir directamente a la página de un hotel, la URL se vería algo así:

https://ota.wink.travel/hotel/hotel-x?client-id=MI_ID_DE_CLIENTE&configuration-id=MI_ID_DE_PERSONALIZACIÓN

:::note
Si no incluyes un ID de cliente y/o de configuración, Wink OTA usará una configuración predeterminada.
:::

## IBE

Si eres una empresa de administración de propiedades, quieres enlazar a la página principal de Wink IBE. La página principal mostrará tu tema designado junto con las [propiedades que representas](/studio/agency-representation).

Lo que obtienes es una URL que se ve algo así:

https://book.wink.travel?client-id=MI_ID_DE_CLIENTE&configuration-id=MI_ID_DE_PERSONALIZACIÓN

Si eres una propiedad, quieres enlazar directamente a la página de aterrizaje de la propiedad en Wink IBE.

Esa URL se vería algo así:

https://book.wink.travel/hotel/hotel-x?client-id=MI_ID_DE_CLIENTE&configuration-id=MI_ID_DE_PERSONALIZACIÓN


:::note
Si no incluyes un ID de cliente y/o de configuración, Wink IBE NO cargará. Estos parámetros son obligatorios.
:::

## Ejemplo: Página de propiedad

Cada propiedad en Wink obtiene su propia página de aterrizaje única en cualquiera de los sitios web. Todos los enlaces externos dirigen a los viajeros a una de estas páginas de aterrizaje.

La página de aterrizaje contiene toda la información que un viajero necesita para tomar una decisión informada sobre si hospedarse en esta propiedad.
