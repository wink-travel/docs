:::tip
Il server di autenticazione IAM di Wink supporta i comuni endpoint OAuth2:

- /oauth2/authorize
- /oauth2/token
- /oauth2/introspect
- /oauth2/revoke
- /oauth2/userinfo
:::

Un'applicazione ti fornisce un `Client ID` e una `Secret Key` necessari per creare una sessione OAuth2 autenticata che puoi utilizzare per comunicare con le API di Wink e TripPay.

Ecco i passaggi per creare una sessione autenticata.

**Passo 1**. Recupera un token di accesso nell'ambiente di staging o in produzione:

Molto probabilmente utilizzerai una libreria OAuth2 robusta per il tuo linguaggio, che farà tutto il lavoro pesante per te.
I nostri esempi mostreranno l'uso più basilare da riga di comando usando `curl`.

### Staging

```bash

curl -X POST https://staging-iam.wink.travel/oauth2/token \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=client_credentials" \
  -d "client_id=<YOUR_CLIENT_ID>" \
  -d "client_secret=<YOUR_SECRET_KEY>"

```

### Produzione

```bash

curl -X POST https://iam.wink.travel/oauth2/token \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=client_credentials" \
  -d "client_id=<YOUR_CLIENT_ID>" \
  -d "client_secret=<YOUR_SECRET_KEY>"

```

Questo restituirà un token di accesso, insieme ad altri dati di risposta OAuth2:

```json
{
  "access_token": "abc123"
}
```

Quando effettui una chiamata a uno qualsiasi dei nostri endpoint, includi quanto segue nell'intestazione:

- `Wink-Version` = `2.0` Ultima versione - *Consulta la nostra documentazione API per altre versioni disponibili.*
- `Authentication` = `Bearer: ${access_token}` *Inserisci il tuo token di accesso.*

:::tip
Puoi testare tutte le nostre API senza scrivere una sola riga di codice prima di iniziare.

Prendi un 'Client ID' e una 'Secret Key' da una delle tue app e naviga in una qualsiasi delle nostre [API](/it/developers/apis).
:::
