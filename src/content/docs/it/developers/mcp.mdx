## Panoramica
Il server MCP Wink ospitato è disponibile all’indirizzo:

```
https://docs.mcp.wink.travel/mcp
```

Indirizza i client AI compatibili (VS Code, Cursor, altri che implementano il trasporto MCP HTTP) a questo endpoint per permettere loro di:
- Consultare la documentazione di prodotto e piattaforma (`wink://docs/...`)
- Consultare specifiche OpenAPI live e documenti delle operazioni (`wink://api/...`)
- Eseguire ricerche semantiche / per parola chiave leggere su documenti e API live

## Trasporto
Usa l’endpoint HTTP principale e streamabile:

```
https://docs.mcp.wink.travel/mcp
```

Se il tuo client supporta solo SSE, usa:

```
https://docs.mcp.wink.travel/sse
```

### Claude Code
Documentazione ufficiale: [Collega Claude Code agli strumenti tramite MCP](https://code.claude.com/docs/en/mcp)

Anthropic consiglia il trasporto HTTP remoto per server MCP ospitati in cloud:
```bash
claude mcp add --transport http wink-docs https://docs.mcp.wink.travel/mcp
claude mcp list
```

All’interno di Claude Code, esegui `/mcp` per controllare lo stato del server. Se vuoi condividere il server con tutto il repository, aggiungi `--scope project` così Claude Code scrive la configurazione in `.mcp.json`.

### Codex
Documentazione ufficiale: [Quickstart Docs MCP per Codex](https://developers.openai.com/resources/docs-mcp)

OpenAI documenta la configurazione MCP tramite la CLI Codex, e segnala che la configurazione è condivisa con l’estensione IDE:
```bash
codex mcp add wink-docs --url https://docs.mcp.wink.travel/mcp
codex mcp list
```

### Cursor
Documentazione MCP Cursor: https://cursor.com/docs/context/mcp
Configurazione: aggiungi il server remoto tramite le impostazioni MCP.
```json
{
  "mcpServers": {
    "wink-docs": {
      "url": "https://docs.mcp.wink.travel/mcp",
    }
  }
}
```

### VS Code
Documentazione ufficiale: https://code.visualstudio.com/docs/copilot/customization/mcp-servers
Configurazione remota minima (file `mcp.json` utente o workspace):
```json
{
  "servers": {
    "wink-docs": {
      "type": "http",
      "url": "https://docs.mcp.wink.travel/mcp"
    }
  }
}
```

Suggerimenti:
- Usa la Command Palette: “MCP: List Servers” per confermare la connessione e navigare tra gli strumenti.
- Il selettore degli strumenti ti permette di abilitare/disabilitare `api_search` e `docs_search`.
- Aggiungi contesto: vista Chat → Aggiungi Contesto → Risorse MCP per allegare pagine di documentazione.

## URI delle Risorse Core

| URI | Descrizione |
|-----|-------------|
| `wink://docs` | Indice della documentazione (raggruppato per categoria) |
| `wink://docs/{path}` | Articolo specifico della documentazione (Markdown / MDX) |
| `wink://api` | Catalogo OpenAPI live scoperto dalla configurazione swagger di Wink |
| `wink://api/{apiName}` | Sommario compatto dell’API con autenticazione, tag, header e operazioni di esempio |
| `wink://api/{apiName}/operations` | Elenco completo delle operazioni raggruppate per tag |
| `wink://api/{apiName}/operation/{key}` | Dettaglio di una singola operazione (parametri, corpo richiesta, risposte) |
| `wink://api/{apiName}/schema/{schemaName}` | Schema componente risolto con campi annidati, tipi ed esempi |
| `wink://api/{apiName}/operation/{key}/request` | Vista focalizzata sulla richiesta con input obbligatori ed esempio minimo di payload |
| `wink://api/{apiName}/operation/{key}/response/{status}` | Vista focalizzata sulla risposta per un codice di stato specifico |
| `wink://api/{apiName}/operation/{key}/recipe` | Flusso orientato al compito per chiamare correttamente quell’operazione |

## Strumenti di Ricerca (Riferimento Rapido)
| Strumento | Scopo | Esempi Tipici di Query |
|-----------|-------|-----------------------|
| `api_search` | Trova l’API live o l’operazione da integrare | `create affiliate`, `payment receipt`, `lookup inventory`, `invite manager` |
| `docs_search` | Trova documenti concettuali / guide | `authentication`, `webhooks`, `wordPress` |

Lascia `query` vuoto per ricevere suggerimenti. Ogni risultato include l’URI MCP che puoi aprire.
Per i risultati sulle operazioni, `api_search` restituisce anche URI collegati di richiesta, risposta, schema e ricetta così gli agenti possono saltare direttamente alla sotto-risorsa più utile.

## Modalità d’Uso Tipica
1. Esegui `api_search` con la tua intenzione (es. *"create affiliate"* o *"payment receipt"*).
2. Apri l’URI `wink://api/...` restituito, o segui i suoi campi collegati `request_uri`, `response_uris`, `request_schema_uris`, `response_schema_uris` e `recipe_uri`.
3. Usa `docs_search` per concetti di supporto (autenticazione, webhook, workflow).

## Note
- Ogni URI restituito può essere aperto dall’agente per il contenuto completo.
- L’albero `wink://api` è live e segue il catalogo OpenAPI corrente a monte invece di un indice generato dal repository.
- I suggerimenti appaiono automaticamente quando le query sono vuote.
