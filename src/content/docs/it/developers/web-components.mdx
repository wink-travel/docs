[I Componenti Web](https://developer.mozilla.org/en-US/docs/Web/API/Web_components) sono uno standard che ti permette di integrare intere funzionalità Javascript nel tuo sito web senza dover sapere programmare.
Grazie a questa tecnologia innovativa, puoi incorporare l'inventario viaggi di Wink con pochissimo sforzo. Questo articolo ti guida attraverso la nostra collezione di componenti web e ti mostra come usarli.

## Incorpora Wink

Ci sono tre cose che devi includere in ogni pagina dove vuoi mostrare uno dei nostri componenti web.

1. Includi il nostro foglio di stile.
2. Includi il nostro Javascript.
3. Includi il nostro loader dell'applicazione.

### Foglio di stile

Includi i nostri stili CSS nell'head del tuo documento.

```html {3}
<html>
  <head>
    <link rel="stylesheet" href="https://elements.wink.travel/styles.css"></link>
  </head>
</html>
```

### Javascript

Includi il nostro Javascript in fondo al tuo documento. (*Consigliamo subito prima del tag di chiusura body*).

```html {3}
<html>
  <body>
    <script src="https://elements.wink.travel/elements.js" type="module" defer></script>
  </body>
</html>
```

Il tuo sito è ora pronto e in grado di mostrare i nostri componenti web.

## Componenti

Leggi qui sotto della nostra libreria di componenti.

### Loader dell'applicazione

Il loader è responsabile di mantenere lo stato e gestire l'interazione tra i nostri componenti. (*Includilo sotto il nostro Javascript*).

```html {3-6}
<html>
  <body>
    <wink-app-loader
      client-id="IL TUO CLIENT ID QUI"
      configuration-id="IL TUO ID DI PERSONALIZZAZIONE QUI"
    ></wink-app-loader>
  </body>
</html>
```

### Loader dei contenuti

Il loader dei contenuti è il cuore della nostra libreria di componenti web. È responsabile di mostrare il tuo inventario (card, griglie, mappe).

Attributi disponibili:

- layout
- id
- sort

*L'attributo `sort` è disponibile solo quando il layout è `RANKED` e non vuoi usare una griglia ranked esistente. In questi casi, lascia `id` vuoto.*

Tipi di layout disponibili:

- `AD_BANNER`
- `MAP`
- `HOTEL`
- `GUEST_ROOM`
- `MEETING_ROOM`
- `SPA`
- `RESTAURANT`
- `ACTIVITY`
- `ATTRACTION`
- `PLACE`
- `ADD_ON`
- `LIST`
- `SEARCH`
- `RANKED`

Tipi di ordinamento disponibili:

- `MEMBER`
- `PRICE_LOW_TO_HIGH`
- `PRICE_HIGH_TO_LOW`
- `PRICE`
- `POPULARITY`
- `ECO`
- `EXPERIENCE`
- `PERK`
- `LOYALTY`
- `PACKAGE`

```html {3-6}
<html>
  <body>
    <wink-content-loader
       layout="GUEST_ROOM"
       id="2de7ee9c-61c9-11ef-9722-42004e494300"
    ></wink-content-loader>
  </body>
</html>
```

Puoi trovare esempi per [card](/studio/cards), [griglie](/studio/grids) e [mappe](/studio/maps).

## Componenti

### Pagamento

Il componente di pagamento ti permette di informare TripPay che un viaggiatore vuole acquistare qualcosa e le regole e i prezzi per quegli articoli.

C'è un attributo obbligatorio per il widget:

- `id` L'identificatore del contratto di prenotazione che vuoi eseguire.

```html {3}
<html>
  <body>
    <trip-pay id="<INSERT_BOOKING_CONTRACT_ID>"></trip-pay>
  </body>
</html>
```

:::note
Per imparare come generare l'`id` del contratto di prenotazione, leggi la guida: [Integrazione con TripPay](/guides/integrating-with-trip-pay).
:::

Il widget prepara il contratto per l'esecuzione e mostra i dettagli di pagamento *(Figura 1)* all'utente per finalizzare la prenotazione.

## Errori

Se ricevi un messaggio di errore quando incorpori uno dei nostri componenti web, sia su WinkLinks che sul tuo sito web, ci sono alcune cose che potrebbero essere andate storte:

### Disponibilità

L'inventario potrebbe non essere più disponibile. Vai su [Wink Studio](/studio/cards) e verifica se lo stato della "card" è disponibile.
Se non lo è, sarà evidenziato in rosso. In tal caso, puoi aspettare e magari il fornitore lo renderà nuovamente disponibile oppure rimuoverlo dalla tua lista.

:::note
`Disponibilità` non significa la disponibilità giornaliera dell'inventario da prenotare. Significa che l'inventario è completamente non disponibile per te e/o per chiunque.
:::

### Rimosso

L'inventario potrebbe essere stato rimosso. Vai su [Wink Studio](/studio/cards) e verifica se lo stato della "card" è disponibile.
Se non lo è, sarà evidenziato in rosso. In tal caso, puoi aspettare e magari il fornitore lo renderà nuovamente disponibile oppure rimuoverlo dalla tua lista.

### Personalizzazione

Potresti aver accidentalmente rimosso la personalizzazione che avevi definito per accompagnare il tuo inventario. Assicurati che la personalizzazione sia ancora disponibile e imposta una nuova se manca.

### Applicazione

Questo è solo per sviluppatori web. Se hai accidentalmente rimosso la [Applicazione](/developer/applications); rendendo il client ID non più disponibile, crea una nuova applicazione e usa il nuovo client ID per incorporare i nostri Componenti Web.
