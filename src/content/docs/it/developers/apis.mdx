## Consumare gli endpoint

Gli endpoint Consume sono destinati agli sviluppatori che desiderano trovare un inventario di viaggi esistente e prenotarlo o utilizzarlo per fare pubblicità tramite uno dei loro account affiliati Wink.

### API di configurazione

Un singolo endpoint per recuperare informazioni whitelabel + personalizzazione per il motore di prenotazione.

<LinkButton href="https://docs.wink.travel/engine-client" variant="secondary" icon="external" target="_blank">
  Saperne di più
</LinkButton>


### API di ricerca

Un modo semplice da usare per i programmatori per cercare un inventario di viaggi personalizzato in diverse aree geografiche.

<LinkButton href="https://docs.wink.travel/lookup" variant="secondary" icon="external" target="_blank">
  Saperne di più
</LinkButton>


### API di inventario

Un modo semplice da usare per i programmatori per recuperare l'inventario di viaggio noto così come era ufficialmente localizzato utilizzando l'API Lookup.

  Saperne di più
</LinkButton>


### API di prenotazione

Un modo semplice da usare per programmare per prenotare l'inventario, trovato sulla nostra piattaforma.

  Saperne di più

## API di pagamento

Gli endpoint di pagamento sono destinati agli sviluppatori che desiderano acquistare inventario di viaggi. Questo può essere fatto tramite l'API come agente di viaggio registrato o utilizzando la nostra API insieme al nostro widget di pagamento conforme allo standard PCI per tutte le altre entità.
L'API di pagamento espone endpoint per affiliati e hotel per monitorare prenotazioni, analisi e disponibilità di fondi. Possono anche scegliere di prelevare i fondi disponibili sul proprio conto bancario.

<LinkButton href="https://docs.wink.travel/payment" variant="secondary" icon="external" target="_blank">
  Saperne di più
</LinkButton>
