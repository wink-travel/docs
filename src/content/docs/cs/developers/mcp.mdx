## Přehled
Hostovaný Wink MCP server je dostupný na:

```
https://docs.mcp.wink.travel/mcp
```

Nasměrujte kompatibilní AI klienty (VS Code, Cursor, další implementující MCP HTTP transport) na tento endpoint, aby mohli:
- Procházet dokumentaci produktů a platformy (`wink://docs/...`)
- Procházet živé specifikace OpenAPI a dokumentaci operací (`wink://api/...`)
- Provádět lehké sémantické / klíčové vyhledávání v dokumentaci a živých API

## Transport
Použijte primární streamovatelný HTTP endpoint:

```
https://docs.mcp.wink.travel/mcp
```

Pokud váš klient podporuje pouze SSE, použijte:

```
https://docs.mcp.wink.travel/sse
```

### Claude Code
Oficiální dokumentace: [Připojení Claude Code k nástrojům přes MCP](https://code.claude.com/docs/en/mcp)

Anthropic doporučuje vzdálený HTTP transport pro cloud-hostované MCP servery:
```bash
claude mcp add --transport http wink-docs https://docs.mcp.wink.travel/mcp
claude mcp list
```

Uvnitř Claude Code spusťte `/mcp` pro kontrolu stavu serveru. Pokud chcete sdílet server s celým repozitářem, přidejte `--scope project`, aby Claude Code zapsal konfiguraci do `.mcp.json`.

### Codex
Oficiální dokumentace: [Docs MCP rychlý start pro Codex](https://developers.openai.com/resources/docs-mcp)

OpenAI dokumentuje nastavení MCP přes Codex CLI a uvádí, že konfigurace je sdílená s IDE rozšířením:
```bash
codex mcp add wink-docs --url https://docs.mcp.wink.travel/mcp
codex mcp list
```

### Cursor
Cursor MCP dokumentace: https://cursor.com/docs/context/mcp
Nastavení: Přidejte vzdálený server přes MCP nastavení.
```json
{
  "mcpServers": {
    "wink-docs": {
      "url": "https://docs.mcp.wink.travel/mcp",
    }
  }
}
```

### VS Code
Oficiální dokumentace: https://code.visualstudio.com/docs/copilot/customization/mcp-servers
Minimální vzdálená konfigurace (uživatelský nebo workspace `mcp.json`):
```json
{
  "servers": {
    "wink-docs": {
      "type": "http",
      "url": "https://docs.mcp.wink.travel/mcp"
    }
  }
}
```

Tipy:
- Použijte Command Palette: „MCP: List Servers“ pro potvrzení připojení a procházení nástrojů.
- Výběr nástrojů umožňuje povolit/zakázat `api_search` a `docs_search`.
- Přidejte kontext: Zobrazení chatu → Přidat kontext → MCP Resources pro připojení stránek dokumentace.

## Hlavní URI zdrojů

| URI | Popis |
|-----|-------------|
| `wink://docs` | Index dokumentace (seskupený podle kategorií) |
| `wink://docs/{path}` | Konkrétní článek dokumentace (Markdown / MDX) |
| `wink://api` | Živý katalog OpenAPI získaný z Wink swagger konfigurace |
| `wink://api/{apiName}` | Kompaktní shrnutí API s autentizací, tagy, hlavičkami a ukázkovými operacemi |
| `wink://api/{apiName}/operations` | Kompletní seznam operací seskupený podle tagů |
| `wink://api/{apiName}/operation/{key}` | Detail jednotlivé operace (parametry, tělo požadavku, odpovědi) |
| `wink://api/{apiName}/schema/{schemaName}` | Vyřešené schéma komponenty s vnořenými poli, typy a příklady |
| `wink://api/{apiName}/operation/{key}/request` | Pohled zaměřený na požadavek s povinnými vstupy a minimálním příkladem payloadu |
| `wink://api/{apiName}/operation/{key}/response/{status}` | Pohled zaměřený na odpověď pro konkrétní stavový kód |
| `wink://api/{apiName}/operation/{key}/recipe` | Úkolově orientovaný postup pro správné volání dané operace |

## Nástroje pro vyhledávání (rychlá reference)
| Nástroj | Účel | Typické příklady dotazů |
|------|---------|-----------------------|
| `api_search` | Najít živé API nebo operaci pro integraci | `create affiliate`, `payment receipt`, `lookup inventory`, `invite manager` |
| `docs_search` | Najít konceptuální dokumentaci / návody | `authentication`, `webhooks`, `wordPress` |

Nechte `query` prázdné pro získání návrhů. Každý výsledek obsahuje MCP URI, které můžete otevřít.
U zásahů do operací `api_search` také vrací propojené URI požadavků, odpovědí, schémat a postupů, takže agenti mohou přímo přejít na nejvhodnější podzdroje.

## Typický způsob použití
1. Spusťte `api_search` s vaším záměrem (např. *"create affiliate"* nebo *"payment receipt"*).
2. Otevřete vrácené `wink://api/...` URI, nebo následujte jeho propojená pole `request_uri`, `response_uris`, `request_schema_uris`, `response_schema_uris` a `recipe_uri`.
3. Použijte `docs_search` pro podpůrné koncepty (autentizace, webhooks, workflow).

## AI dovednosti

Předpřipravené dovednosti agentů používají tento MCP server jako svůj zdroj pravdy. Dovednosti dávají vašemu kódovacímu agentovi strukturovaný pracovní postup pro práci s Wink API — místo hádání endpointů agent čte živé kontrakty přímo z MCP serveru.

Viz [AI Skills](/cs/developers/ai-skills) pro dostupné dovednosti a instrukce k instalaci.

## Poznámky
- Každé vrácené URI může agent otevřít pro plný obsah.
- Strom `wink://api` je živý a sleduje aktuální upstream OpenAPI katalog místo generovaného indexu repozitáře.
- Návrhy se zobrazují automaticky, když jsou dotazy prázdné.
