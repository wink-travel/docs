:::tip
Wink IAM autentizační server podporuje běžné OAuth2 koncové body:

- /oauth2/authorize
- /oauth2/token
- /oauth2/introspect
- /oauth2/revoke
- /oauth2/userinfo
:::

Aplikace vám poskytne `Client ID` a `Secret Key`, které potřebujete k vytvoření autentizované OAuth2 relace, kterou můžete použít ke komunikaci s Wink a TripPay API.

Zde jsou kroky k vytvoření autentizované relace.

**Krok 1**. Získejte přístupový token na stagingu nebo v našem produkčním prostředí:

Pravděpodobně budete pracovat s robustní OAuth2 knihovnou pro váš jazyk, která za vás udělá veškerou těžkou práci.
Naše příklady ukazují nejzákladnější použití z příkazové řádky pomocí `curl`.

### Staging

```bash

curl -X POST https://staging-iam.wink.travel/oauth2/token \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=client_credentials" \
  -d "client_id=<YOUR_CLIENT_ID>" \
  -d "client_secret=<YOUR_SECRET_KEY>"

```

### Produkce

```bash

curl -X POST https://iam.wink.travel/oauth2/token \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=client_credentials" \
  -d "client_id=<YOUR_CLIENT_ID>" \
  -d "client_secret=<YOUR_SECRET_KEY>"

```

Toto vrátí přístupový token spolu s dalšími daty odpovědi OAuth2:

```json
{
  "access_token": "abc123"
}
```

Když voláte jakýkoli z našich koncových bodů, zahrňte do hlavičky následující:

- `Wink-Version` = `2.0` Nejnovější - *Podívejte se do naší dokumentace API pro další dostupné verze.*
- `Authentication` = `Bearer: ${access_token}` *Vložte svůj přístupový token.*

:::tip
Můžete otestovat všechna naše API, aniž byste napsali jediný řádek kódu, než začnete.

Získejte 'Client ID' a 'Secret Key' z jedné ze svých aplikací a přejděte na jakékoli z našich [API](/cs/developers/apis).
:::
