Připojením svého Google Analytics 4 účtu k Wink se vaše rezervační události — od zobrazení pokojů až po dokončené nákupy — automaticky přeposílají do vašeho vlastního GA4 dashboardu spolu s analytikou Wink. Žádný kód, žádné technické nastavení na vaší straně.

<Aside type="tip">
Vše, co potřebujete, je GA4 Measurement ID. Po zadání do portálu Wink jsou vaše analytiky ihned aktivní.
</Aside>

---

## Jak to funguje

```
Zadáte svůj G-XXXXXXXXXX do portálu Wink
       ↓
Wink přeposílá vaše rezervační události do vašeho GA4 účtu
       ↓
Přihlásíte se na analytics.google.com a vidíte své rezervace
```

---

## Co musíte udělat

<Steps>

1. **Vytvořte GA4 účet**

   Přejděte na [analytics.google.com](https://analytics.google.com): Admin → Vytvořit účet. Nastavte časové pásmo a měnu odpovídající vašemu trhu.

2. **Vytvořte webový datový stream**

   Admin → Data Streams → Přidat stream → Web. Zadejte `wink.travel` jako URL. Tím se vygeneruje vaše **Measurement ID** (`G-XXXXXXXXXX`). Zkopírujte ho.

3. **Zadejte své Measurement ID do portálu Wink**

   Přihlaste se na [my.wink.travel](https://my.wink.travel), vyberte svůj účet a přejděte na záložku **Customizations**. Vložte tam své Measurement ID a uložte. To je vše — vaše integrace GA4 je nyní aktivní.

4. **Označte `purchase` jako klíčovou událost**

   Vraťte se do svého GA4 účtu: Admin → Události → najděte `purchase` → přepněte **Označit jako klíčovou událost**. Tím se dokončené rezervace zobrazí ve vašem přehledu Konverzí.

5. **Prohlédněte si svá data**

   Přihlaste se na [analytics.google.com](https://analytics.google.com) a uvidíte své rezervace, příjmy a cestu hosta — vše automaticky sledované přes Wink.

</Steps>

<Aside type="note">
Pro tuto integraci nemusíte na svých webových stránkách instalovat žádný sledovací kód. O vše se postará Wink. Vaše Measurement ID je jednoduše cílové místo, kam se odesílají data o rezervacích.
</Aside>

## Co můžete sledovat

Po připojení váš GA4 účet přijímá kompletní sadu Enhanced E-commerce událostí pro vaši nemovitost:

### Objevování

| Událost | Význam |
|---|---|
| `view_item_list` | Host viděl vaši nemovitost ve vyhledávání nebo v seznamu kategorií |
| `select_item` | Host klikl na vaši nemovitost ze seznamu |
| `view_item` | Host si prohlédl detailní stránku hotelu |

### Zvažování

| Událost | Význam |
|---|---|
| `view_promotion` | Host viděl propagační nabídku vaší nemovitosti |
| `select_promotion` | Host klikl na propagační nabídku |
| `add_to_wishlist` | Host si uložil vaši nemovitost do seznamu přání |

### Rezervační trychtýř

| Událost | Význam |
|---|---|
| `add_to_cart` | Host vybral pokoj nebo balíček |
| `view_cart` | Host si prohlédl shrnutí rezervace |
| `remove_from_cart` | Host odstranil pokoj nebo balíček |
| `begin_checkout` | Host zahájil proces rezervace |
| `add_payment_info` | Host zadal platební údaje |
| `add_shipping_info` | Host zadal údaje o hostovi |
| `purchase` | Host dokončil rezervaci |
| `refund` | Rezervace byla zrušena a refundována |

Všechny události obsahují relevantní dimenze jako příjmy, měnu, typ pokoje a hodnotu rezervace — což vám poskytuje kompletní přehled o cestě hosta od prvního dojmu až po dokončenou rezervaci. Pro odemknutí dalších dimenzí na úrovni nemovitosti a pokojů, jako jsou `city_name`, `cancellation_policy`, `bed_type` a další, viz [Vlastní dimenze v GA4](/cs/guides/gtm-dual-tagging#custom-dimensions-in-ga4).

## Sledování napříč doménami (pokročilé)

Standardní integrace výše sleduje vše, co se děje v rámci Wink. Pokud však chcete sledovat hosty od vašeho vlastního webu až po dokončení rezervace na Wink — a správně přiřadit původní zdroj návštěvnosti (Google Ads, e-mailové kampaně, organické vyhledávání atd.) — je potřeba provést ještě jeden krok na obou stranách.

<Aside type="caution">
Tato část platí pouze pokud máte GA4 nainstalované na svém vlastním webu. Pokud GA4 na svém webu nepoužíváte, standardní integrace výše vám stačí.
</Aside>

### Proč je to důležité

Bez sledování napříč doménami se host, který nejprve navštíví váš web a poté provede rezervaci na Wink, v GA4 zobrazí jako **Přímá** návštěva bez spojení s předchozí návštěvou. Jakékoliv kliknutí z Google Ads, e-mailové kampaně nebo organického vyhledávání, které vedly k rezervaci, tak nezískají žádné uznání.

Se správně nastaveným sledováním napříč doménami vidíte kompletní cestu hosta:

```
google.com (placená reklama)
       ↓
váš web (host prohlíží pokoje)
       ↓
wink.travel (host vybere pokoj a rezervuje)
       ↓
nákup správně přiřazen původnímu kliknutí z Google Ads
```

### Co musíte udělat vy

Nastavte svůj GA4 účet tak, aby rozpoznal `wink.travel` jako součást vaší vlastní domény, nikoli jako externí odkaz. To GA4 umožní spojit dvě relace do jedné kontinuální cesty hosta.

<Steps>
1. GA4 Admin → Data Streams → váš stream
2. Nastavení tagu → **Nastavte své domény**
3. Přidejte `wink.travel` jako propojenou doménu
4. Uložte
</Steps>

### Co udělá Wink

Kontaktujte [Wink Support](https://my.wink.travel) a poskytněte doménu svého webu (např. `yourhotel.com`). Wink přidá vaši doménu do seznamu vyloučených odkazů v GA4 účtu Wink. Tím se zabrání tomu, aby váš web byl zaznamenán jako zdroj odkazu, což by jinak přepsalo původní atribuci hosta a zahájilo novou relaci.

<Aside type="tip">
Standardní integrace výše je dostačující pro většinu nemovitostí a poskytuje okamžitý přehled o rezervacích a příjmech na Wink. Sledování napříč doménami je pokročilá možnost, kterou stojí za to nastavit, jakmile budete mít s daty GA4 větší zkušenosti a budete chtít hlubší přehled o atribuci.
</Aside>
