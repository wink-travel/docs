Pokud jste integrátor, například Wink, a chcete mapovat své externí příjemce pro použití s TripPay, můžete to udělat jedním ze dvou způsobů:

- Ruční mapování
- Použití [Payment API](/cs/developers/apis/#payment-api) (*viz `Account Mappings`*)

:::tip
Pokud nepracujete pouze s několika stejnými objekty, doporučujeme použít API k automatizaci tohoto kroku.
:::

Níže vás provedeme ručním způsobem mapování externího příjemce. *Pod kapotou používá API.*

<Steps>
1. Přihlaste se do [TripPay](https://pay.wink.travel).
2. Vyberte svůj integrátorský účet.
3. Klikněte na `Settings > Mappings` v hlavním navigačním panelu.
4. Klikněte na tlačítko `Create mapping`. Zobrazí se formulář pro mapování.
5. **Local account** Začněte psát název existujícího TripPay účtu, který chcete mapovat.
6. **External identifier** Vložte svůj externí ID pro tuto entitu, jak existuje v vzdáleném systému.
7. Klikněte na `Save` pro pokračování. Budete přesměrováni zpět na seznam mapování.
</Steps>

S nastavenými mapováními nyní TripPay ví, které účty má připisovat jako příjemce rezervace, když na ně odkazujete pomocí svých externích identifikátorů.

## API

Vývojáři, kteří chtějí spravovat `Payments`, přejděte na [Developers > API > Payments](/cs/developers/apis/#payment-api).
