Remix는 기본적으로 서버 렌더링을 하는 풀스택 프레임워크입니다. 루트 컴포넌트에서 `useEffect`를 사용하여 클라이언트에서만 Wink CDN을 로드하세요.

## 설치

```bash
npm install @wink/elements
```

## app/root.tsx에서 로드하기

`useEffect`는 브라우저에서만 실행되며 서버 사이드 렌더링 중에는 절대 실행되지 않습니다:

```tsx
// app/root.tsx
import { useEffect } from 'react';
import {
  Links, Meta, Outlet, Scripts, ScrollRestoration,
} from '@remix-run/react';
import { load } from '@wink/elements';

export default function App() {
  useEffect(() => {
    load({ clientId: import.meta.env.VITE_WINK_CLIENT_ID });
  }, []);

  return (
    <html lang="en">
      <head>
        <Meta />
        <Links />
      </head>
      <body>
        <Outlet />
        <ScrollRestoration />
        <Scripts />
      </body>
    </html>
  );
}
```

## 환경 변수

Remix (Vite 기반, v2.8+)는 브라우저에 노출되는 변수를 위해 `import.meta.env`를 지원합니다. 브라우저 번들에 노출하려면 `VITE_` 접두사를 붙이세요:

```
# .env
VITE_WINK_CLIENT_ID=your-client-id
```

Node.js 어댑터를 사용하는 이전 Remix 설정의 경우, 루트 로더에서 클라이언트 ID를 반환하고 컴포넌트에서 읽어오세요:

```tsx
// app/root.tsx
import { json } from '@remix-run/node';
import { useLoaderData } from '@remix-run/react';

export async function loader() {
  return json({ winkClientId: process.env.WINK_CLIENT_ID });
}

export default function App() {
  const { winkClientId } = useLoaderData<typeof loader>();

  useEffect(() => {
    if (winkClientId) {
      load({ clientId: winkClientId });
    }
  }, [winkClientId]);

  // ...
}
```

## 라우트에서 사용하기

```tsx
// app/routes/hotels.tsx
export default function HotelsRoute() {
  return (
    <main>
      <wink-content-loader layout="HOTEL" id="YOUR_LAYOUT_ID" />
    </main>
  );
}
```

<Aside type="caution" title="Hydration 불일치">
Remix는 서버 렌더링된 HTML을 클라이언트에서 하이드레이션합니다. 서버에서 `<wink-*>` 요소가 렌더링된 경우, CDN 스크립트가 등록되기 전 브라우저가 잠시 미해결 커스텀 요소를 표시할 수 있습니다. 이는 단지 시각적인 문제이며, 스크립트가 로드되면 요소가 정상적으로 업그레이드됩니다.
</Aside>
