Wink와 같은 통합자가 TripPay에서 사용할 외부 수취인을 매핑하려는 경우 다음 두 가지 방법 중 하나를 사용할 수 있습니다.

* 수동 매핑
* 사용하여[결제 API](/developers/apis/#payment-api)(*보다 `Account Mappings`*)

:::팁
동일한 부동산을 몇 개만 다루는 경우가 아니라면 API를 사용하여 이 단계를 자동화하는 것이 좋습니다.
:::

아래에서는 외부 수취인을 수동으로 매핑하는 방법을 안내해 드리겠습니다.*이는 내부적으로 API를 사용합니다.*

<Steps>
  1. 로그인 [트립페이](https://pay.wink.travel).
  2. 통합자 계정을 선택하세요.
  3. 딸깍 하는 소리 `Settings > Mappings` 메인 탐색 바에서.
  4. 클릭 `Create mapping` 버튼을 클릭하세요. 매핑 양식이 나타납니다.
  5. **로컬 계정** 매핑하려는 기존 TripPay의 이름을 입력하세요.
  6. **외부 식별자** 원격 시스템에 있는 이 엔터티의 외부 ID를 붙여넣습니다.
  7. 딸깍 하는 소리 `Save` 계속하려면 를 클릭하세요. 매핑 목록으로 돌아갑니다.
</Steps>


매핑이 완료되면 이제 TripPay에서 외부 식별자를 사용하여 참조하는 계정에 대해 예약 수혜자로 어떤 계정을 적립할지 알 수 있습니다.

## API

관리하고 싶은 개발자`Payments`, 가다[개발자 > API > 결제](/developers/apis/#payment-api).
