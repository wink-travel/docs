Wink와 같은 통합업체이고 TripPay와 함께 사용할 외부 수취인을 매핑하려는 경우, 두 가지 방법 중 하나로 할 수 있습니다:

- 수동 매핑
- [Payment API](/ko/developers/apis/#payment-api) 사용 (*`Account Mappings` 참조*)

:::tip
동일한 숙소가 몇 개뿐인 경우가 아니라면, 이 단계를 자동화하기 위해 API 사용을 권장합니다.
:::

아래에서는 외부 수취인을 수동으로 매핑하는 방법을 단계별로 안내합니다. *내부적으로는 API를 사용합니다.*

<Steps>
1. [TripPay](https://pay.wink.travel)에 로그인합니다.
2. 통합업체 계정을 선택합니다.
3. 메인 내비게이션 바에서 `Settings > Mappings`를 클릭합니다.
4. `Create mapping` 버튼을 클릭합니다. 매핑 양식이 표시됩니다.
5. **Local account** 매핑하려는 기존 TripPay 이름을 입력하기 시작합니다.
6. **External identifier** 원격 시스템에 존재하는 이 엔터티의 외부 ID를 붙여넣습니다.
7. 계속하려면 `Save`를 클릭합니다. 매핑 목록으로 리디렉션됩니다.
</Steps>

매핑이 완료되면, TripPay는 외부 식별자를 사용하여 예약의 수취인으로 지정할 계정을 인식합니다.

## API

`Payments`를 관리하려는 개발자는 [Developers > API > Payments](/ko/developers/apis/#payment-api)로 이동하세요.
