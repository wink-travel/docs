맞춤 설정은 코드를 작성하지 않고도 Wink와의 통합의 모든 측면을 제어할 수 있는 강력한 기능입니다. 판매하는 모든 재고 항목을 맞춤 설정할 수 있습니다. 기능을 추가하거나 제거하고, 색상 구성표를 변경하는 등 다양한 작업이 가능합니다.

`Customization` 페이지에서 작업하려는 맞춤 설정 아래의 `Actions` 버튼을 클릭한 후 `Update`를 클릭하세요.
*계정을 생성할 때 기본 항목이 이미 제공됩니다. 이 항목은 삭제할 수 없습니다.*

이렇게 하면 5~7개의 탭(*계정 유형에 따라 다름*)이 있는 맞춤 설정 양식으로 이동합니다.

## 일반

`General` 탭에서 사용할 수 있는 옵션은 다음과 같습니다.

- **이름** 맞춤 설정을 쉽게 기억할 수 있도록 이름을 지정하세요. *예: 내 여행 블로그용*
- **표시 통화** 사용자가 예약 가능한 여행 재고 옆에서 처음 보게 될 통화를 선택하세요. *예: USD*
- **표시 언어** 사용자가 처음 접하게 될 언어를 선택하세요. *예: 영어*
- **페이지 제목에 추가** 페이지 제목(*브라우저 탭에서 보이는 제목*)에 추가할 내용을 제어할 수 있습니다. 주로 예약 엔진에 사용됩니다. *예: https://ota.wink.travel로 이동하면 기본적으로 `Home | Traveliko - Hotels with Benefits`가 표시됩니다.* '|' 파이프 이후의 모든 내용을 변경할 수 있습니다.

## 테마

`Theme` 탭에서는 [예약 엔진](/booking-engine/in-a-nutshell) 웹사이트, [WinkLinks](/link-manager/wink-links) 웹사이트 및 모든 [웹 컴포넌트](/developers/web-components) 의 모양과 느낌을 맞춤 설정할 수 있습니다.

### 구성

이 섹션에서는 디자인과 카드 레이아웃을 구성할 수 있습니다. 현재는 Wink라는 단일 디자인만 제공됩니다.

카드 레이아웃은 사용자가 수평 또는 수직 정렬된 거래 카드 중 어떤 것을 볼지 결정합니다. 정보는 동일합니다.

:::note
모바일 기기에서 거래 카드를 볼 때 카드 레이아웃에 차이가 없습니다. 그러나 iPad mini 이상 해상도에서는 차이를 확인할 수 있습니다.
:::

:::tip[전문가 팁]
여행 재고(예: 객실 유형)가 4개 미만인 경우 수평 레이아웃을 사용하세요... 더 보기 좋습니다.
:::

### 색상

## 분석

기본적으로 GA4 분석을 지원합니다. 시작부터 끝까지 전체 향상된 전자상거래 전환 경로를 추적하려면 측정 ID를 준비하여 `Analytics` 탭의 필드에 입력하세요.

<Aside type="caution" title="알림">
  GA4 속성 스트림에 `i.trvl.as`, `book.wink.travel` 및 `ota.wink.travel`을 추가하는 것을 잊지 마세요.
</Aside>

## 지도

[WordPress 플러그인](/developers/wordpress) 또는 [웹 컴포넌트](/developers/web-components) 를 사용하여 사이트에 지도를 삽입하려면 Google Maps API 키를 제공해야 합니다.

:::note
  [여기에서 Google Maps API 키 생성](https://console.developers.google.com/projectselector2/apis/credentials?supportedpurview=project)
:::

키를 생성하고 사이트에 맞게 구성한 후 `Analytics` 탭에 입력하세요.

## 일정

`Itinerary` 탭은 선택한 재고에 대한 가격 표시를 보장하는 데 가장 흥미로운 부분일 것입니다.
여행 재고 가격 책정이 복잡하기 때문에 Wink는 정확한 가격을 표시하기 위해 사용자의 일정을 알아야 합니다.

이 섹션에서는 다음과 같은 모든 디지털 공간에 가격을 표시할 기본 일정을 설정할 수 있습니다.

- 가격 오버레이가 있는 소셜 친화적 이미지
- WinkLinks 그리드, 카드 및 링크
- 웹 컴포넌트
- 예약 엔진

### 날짜

정적 날짜 범위 또는 롤링 날짜 범위를 설정할지 선택할 수 있습니다.

- 특정 날짜에 발생하는 이벤트에 맞춤 설정을 사용할 경우 정적 날짜 범위를 선택하세요.
- 다음 중 하나를 원할 경우 롤링 날짜 범위를 선택하세요:
    1. 대부분 OTA가 사용하는 오늘 + 1박 기본값.
    2. 항상 이용 가능해야 하는 미래 날짜 기본값.

### 객실 구성

기본적으로 일정은 대부분 여행자에게 가장 일반적인 설정인 성인 2명으로 설정되어 있습니다.
이 설정은 쉽게 변경할 수 있으며, 여러 객실과 여러 명의 성인 및 어린이를 추가할 수 있습니다.

### 프로모션

프로모션은 사용자의 시간을 절약해 주는 유용한 기능입니다. 숙소에서 제공한 프로모션 코드를 친구와 공유할 수 있다면 여기에 입력할 수 있습니다.
링크를 통해 들어오는 모든 사용자는 자동으로 프로모션 코드가 일정에 추가되고 할인 가격이 자동으로 적용된 것을 볼 수 있습니다.

## 통합

`Integration` 탭은 호텔, 통합업체 및 여행사가 Wink에서 예약 분배 방식을 더 세밀하게 제어하고자 할 때 사용합니다.

### 알림

이 기능은 호텔과 통합업체가 예약 발생 시 관련 모든 당사자에게 알림을 보내도록 Wink에 요청할 수 있게 합니다.
채널 관리자에게 알림을 보내는 것도 포함됩니다.

기본적으로 이 옵션들은 모두 활성화되어 있습니다. 통합업체가 이 기능 중 하나 이상을 비활성화하려는 경우는 다음과 같습니다:

- 여행사가 API 수준에서 통합되어 자체 확인 이메일을 보내고자 할 때.
- 호텔이 통합 테스트 중이며 영업 직원에게 방해가 되지 않도록 할 때.
- 호텔이 이메일 알림에 관심이 없고 PMS를 통해서만 알림을 받고자 할 때.

### 이메일 로고

이메일 알림 서비스를 사용하려는 통합업체는 이메일을 자신들의 브랜드처럼 보이도록 맞춤 설정할 수 있습니다.

:::note
  권장 크기는 가로 300px, 세로 75px입니다.
:::

## 호텔 전용

`Hotel-specific` 탭에는 호텔이 예약 엔진 내 랜딩 페이지에 표시할 내용과 요청한 일정에 재고가 없을 때 표시할 내용을 제어할 수 있는 옵션이 있습니다.

- **순위**: 의미 있는 순위를 아직 획득하지 못한 숙소는 이 기능을 비활성화할 수 있습니다.

:::note
  숙소는 제휴사가 호텔 랜딩 페이지를 어떻게 표시하는지 제어할 수 없습니다... 이 둘은 별개의 설정입니다.
:::

- **재고 상태**: 재고가 없더라도 계속 표시할지 여부를 숙소가 선택할 수 있습니다.

:::note
  대부분의 경우, 숙소는 사용자가 `예약` 버튼을 클릭하여 일정 변경 후 이용 가능한 날짜를 찾을 수 있도록 허용하려고 합니다.
  사용자가 계속 진행하지 못하도록 차단하려면 이 스위치를 활성화하세요.
:::

## 사용 방법

대부분의 경우, 링크, 카드, 지도 및 그리드에 적용할 `Customization`을 선택합니다.

다음과 같은 경우에 수동으로 구성을 적용할 수 있습니다:

- **수동 재정의** 맞춤 구성이 적용된 [공유 가능한 링크](/studio/shareable-links) 기능을 사용하지 않고 예약 엔진 링크를 직접 공유하려는 경우.
- **WordPress 플러그인** Wink [WordPress 플러그인](/developers/wordpress) `설정`에 구성을 추가할 때.
- **맞춤 통합** [wink-app-loader](/developers/web-components/#application-loader) 웹 컴포넌트를 사용하여 맞춤 통합에 수동으로 적용할 때.

### 수동 재정의

다음은 모든 설정이 포함된 구성 ID를 주요 예약 엔진 URL에 적용하는 간단한 예입니다.

<h4 align="center"> [https://ota.wink.travel?configurationId=MY-UNIQUE-ID](https://ota.wink.travel?configurationId=MY-UNIQUE-ID) </h4>

:::note
  사용자가 이 링크를 클릭하면 제휴사 추적 쿠키도 설정됩니다.
:::

### WordPress 플러그인

자세한 내용은 [Wink WordPress 플러그인](/developers/wordpress) 으로 이동하세요.

### 맞춤 통합

페이지 내 모든 임베디드 Wink 웹 컴포넌트를 [wink-app-loader](/developers/web-components/#application-loader) 웹 컴포넌트로 제어하세요.

아래는 `MY-UNIQUE-ID`를 구성 ID로 사용하는 예시입니다.

```html {11}
<html>
  <head>
    <link rel="stylesheet" href="https://elements.wink.travel/styles.css"></link>
  </head>
  <body>

    <script src="https://elements.wink.travel/elements.js" type="module" defer></script>

    <wink-app-loader
      client-id="YOUR CLIENT ID GOES HERE"
      configuration-id="MY-UNIQUE-ID"
    ></wink-app-loader>
  </body>
</html>
```
