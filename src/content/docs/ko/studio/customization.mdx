*계정을 생성할 때 이미 기본 항목이 설정되어 있습니다. 이 항목은 삭제할 수 없습니다.*

이렇게 하면 5~7개의 탭이 있는 사용자 정의 양식으로 이동합니다.*귀하의 계정 유형에 따라*).

## 일반적인

다음은 귀하가 사용할 수 있는 옵션입니다.`General`탭.

* **이름**쉽게 기억할 수 있도록 사용자 정의에 이름을 지정하세요.*예를 들어, 내 여행 블로그의 경우*
* **표시 통화**예약 가능한 여행 목록 옆에 표시되는 초기 통화를 선택하세요.*예: 미국 달러*
* **표시 언어**사용자가 처음 접하게 될 언어를 선택하세요.*예를 들어 영어*
* **페이지 제목에 추가**페이지 제목에 추가되는 내용을 제어할 수 있습니다.*브라우저 탭에 보이는 제목*). 이는 주로 예약 엔진에 사용됩니다.*예를 들어 https://ota.wink.travel로 이동하면 기본적으로 다음이 표시됩니다.`Home | Traveliko - Hotels with Benefits`*'|' 파이프 뒤의 모든 내용을 변경할 수 있습니다.

### 구성

이 섹션에서는 디자인과 카드 레이아웃을 구성할 수 있습니다. 현재로서는 Wink라는 디자인 하나만 있습니다.

카드 레이아웃은 사용자에게 거래 카드를 가로 또는 세로로 정렬하여 보여줄지 여부를 결정합니다. 정보는 동일합니다.

:::참고
모바일 기기에서 거래 카드를 볼 때는 카드 레이아웃에 차이가 없습니다. 하지만 iPad mini 이상의 해상도에서는 차이를 확인하실 수 있습니다.
:::

:::팁\[전문가 팁]
목록에 여행 상품(예: 객실 유형)이 4개 미만인 경우 가로 레이아웃을 사용하세요. 보기에도 더 좋습니다.
:::

## 여정

그`Itinerary`탭은 선택한 재고의 가격이 표시되는지 확인하는 측면에서 가장 흥미로운 기능일 것입니다.
여행 재고 가격은 다소 복잡하기 때문에 Wink는 정확한 가격을 표시하기 전에 사용자의 여정을 알아야 합니다.

이 섹션에서는 다음과 같은 모든 디지털 부동산의 가격을 표시하는 기본 여정을 설정할 수 있습니다.

* 가격이 오버레이된 소셜 친화적인 이미지입니다.
* WinkLinks 그리드, 카드 및 링크.
* 웹 구성 요소
* 예약 엔진

### 날짜

고정된 날짜 범위를 설정할지, 아니면 변동되는 날짜 범위를 설정할지 선택할 수 있습니다.

* 지정된 날짜에 발생하는 특정 이벤트에 대한 사용자 정의를 사용하려면 고정된 날짜 범위를 선택하세요.
* 다음 중 하나를 원하는 경우 롤링 날짜 범위를 선택하세요.
  1. 오늘 + 1박으로 기본 설정하는데, 대부분 OTA가 이렇게 합니다. 또는.
  2. 항상 예약 가능한 미래의 날짜를 기본으로 설정합니다.

### 객실 구성

기본적으로 여정은 대부분의 여행객이 선호하는 성인 2명으로 설정되어 있습니다.
이 설정은 쉽게 변경할 수 있으며, 성인과 어린이를 포함하여 여러 명이 함께 투숙하는 여러 객실을 추가할 수 있습니다.

### 프로모션

프로모션은 사용자의 시간을 절약해 주는 편리한 기능입니다. 숙소에서 친구와 공유할 수 있는 프로모션 코드를 제공했다면 여기에 입력할 수 있습니다.
링크나 유사한 링크를 통해 들어오는 모든 사람은 여정에 프로모션 코드가 자동으로 추가되고 할인된 가격이 자동으로 적용됩니다.

## 완성

그`Integration`이 탭은 Wink에서 예약이 배포되는 방식을 보다 효과적으로 제어하고자 하는 호텔, 통합업체 및 여행사를 위한 것입니다.

### 알림

이를 통해 호텔과 통합업체는 예약 발생 시 Wink가 모든 관련자에게 알림을 보내도록 설정할 수 있습니다.
여기에는 채널 관리자에게 알리는 것도 포함됩니다.

기본적으로 이러한 옵션은 모두 활성화되어 있습니다. 통합자가 다음 기능 중 하나 이상을 비활성화하려는 사용 사례는 다음과 같습니다.

* 여행사는 API 수준에서 통합되어 있으며 자체적으로 확인 이메일을 보내려고 합니다.
* 호텔에서는 통합을 테스트 중이며 영업 직원을 귀찮게 하고 싶어하지 않습니다.
* 호텔은 이메일에는 관심이 없고 PMS를 통해서만 알림을 받고 싶어합니다.

### 이메일 로고

당사의 이메일 알림 서비스를 이용하고자 하는 통합자의 경우, 마치 자신이 보낸 것처럼 보이도록 이메일을 사용자 지정할 수 있습니다.

:::참고
권장 크기는 가로 300px x 세로 75px입니다.
:::

## 호텔별

아래에`Hotel-specific`탭은 호텔이 예약 엔진의 랜딩 페이지에 표시되는 내용과 요청된 여정에 대한 객실이 없을 때 표시되는 내용을 제어할 수 있는 옵션입니다.

* **랭킹**: 아직 의미 있는 순위를 얻지 못한 속성은 해당 기능을 비활성화하기로 선택할 수 있습니다.

:::참고
숙박 시설은 제휴사가 호텔 랜딩 페이지를 표시하는 방식을 제어할 수 없습니다... 이 둘은 서로 다른 구성입니다.
:::

* **유효성**: 예약 가능한 객실이 없더라도 해당 숙박 시설을 계속 표시할지 여부를 선택할 수 있습니다.

:::note
대부분의 경우, 해당 숙박 시설은 사용자에게 클릭 방법을 제공하고자 합니다.`Book`버튼을 클릭하면 랜딩 페이지로 이동하여 여정을 변경하고 이용 가능한 날짜를 확인할 수 있습니다.
사용자가 계속 진행하지 못하도록 하려면 이 스위치를 활성화하세요.
:::

## 사용 방법

대부분의 시나리오에서는 다음을 선택하게 됩니다.`Customization`링크, 카드, 지도, 그리드에 적용하세요.

구성을 수동으로 적용해야 할 경우는 다음과 같습니다.

* **수동 오버라이드**당사 예약 엔진에 대한 링크를 직접 공유하고 싶지만 당사를 사용하고 싶지 않습니다.[공유 가능한 링크](/studio/shareable-links)사용자 정의 구성이 적용된 기능입니다.
* **워드프레스 플러그인**Wink에 구성을 추가하세요[워드프레스 플러그인](/developers/wordpress) `Settings`.
* **맞춤형 통합**당사의 사용으로 사용자 정의 통합기에 수동으로 적용합니다.[윙크 앱 로더](/developers/web-components/#application-loader)웹 컴포넌트.

### 수동 오버라이드

모든 설정이 포함된 구성 ID를 주요 예약 엔진 URL에 적용하는 방법에 대한 간단한 예는 다음과 같습니다.

<h4 align="center">[https://ota.wink.travel?configurationId=MY-UNIQUE-ID](https://ota.wink.travel?configurationId=MY-UNIQUE-ID)</h4>


:::참고
사용자가 이 링크를 클릭하면 제휴사 귀속 쿠키도 설정됩니다.
:::

### 워드프레스 플러그인

로 이동합니다[윙크 워드프레스 플러그인](/developers/wordpress)자세히 알아보려면.

### 맞춤형 통합

페이지에서 내장된 모든 Wink 웹 구성 요소를 제어하세요.[윙크 앱 로더](/developers/web-components/#application-loader)웹 컴포넌트.

아래는 사용하는 샘플입니다.`MY-UNIQUE-ID`구성 ID로.

```html {11}
<html>
  <head>
    <link rel="stylesheet" href="https://elements.wink.travel/styles.css"></link>
  </head>
  <body>

    <script src="https://elements.wink.travel/elements.js" type="module" defer></script>

    <wink-app-loader
      client-id="YOUR CLIENT ID GOES HERE"
      configuration-id="MY-UNIQUE-ID"
    ></wink-app-loader>
  </body>
</html>
```
