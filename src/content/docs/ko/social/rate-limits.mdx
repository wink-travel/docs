Wink가 연결된 소셜 미디어 계정에 콘텐츠를 게시할 때 각 플랫폼에서 부과하는 요율 제한을 준수합니다. 이를 통해 계정이 좋은 상태를 유지하고 콘텐츠가 안정적으로 전달됩니다.

## 플랫폼 제한

각 소셜 플랫폼은 자체 게시 제한을 시행합니다. Wink는 실시간으로 사용량을 추적하며 이러한 제한에 근접할 경우 자동으로 게시 속도를 조절합니다.

| 플랫폼 | 시간당 제한 | 비고 |
|---|---|---|
| **Instagram** | 180 요청 | 게시물, 스토리 및 API 상호작용 포함. |
| **Facebook** | 4,500 요청 | 페이지 게시물, 댓글 및 API 상호작용 포함. |

:::note
이 제한은 소셜 플랫폼 자체에서 설정하며 변경될 수 있습니다. 플랫폼 제한이 변경되면 Wink가 자동으로 조정합니다.
:::

## Wink가 요율 제한을 관리하는 방법

Wink는 플랫폼별 API 사용량을 추적하기 위해 슬라이딩 시간 창을 사용합니다:

1. **각 요청은 전송 시점에 타임스탬프가 찍힙니다.**
2. **1시간이 지난 요청은 자동으로 시간 창에서 제거됩니다.**
3. **새 요청 전마다**, Wink는 현재 시간 창에 남은 용량이 있는지 확인합니다.
4. **제한에 도달하면**, Wink는 요청을 대기열에 넣고 용량이 생기면 전달합니다.

즉, 플랫폼 제한을 초과하거나 계정이 플래그 지정되는 것을 걱정할 필요 없이 Wink가 투명하게 처리합니다.

## 모범 사례

- **게시물을 분산하세요.** 많은 게시물을 한 번에 예약하기보다는 하루 종일 분산시키세요. 이는 플랫폼과 청중에게 더 자연스럽게 보입니다.
- **캘린더를 사용하세요.** [Social Calendar](/ko/social/calendar)를 사용하면 게시 일정을 시각화하고 집중을 피할 수 있습니다.
- **분석을 모니터링하세요.** [Social Analytics](/ko/social/analytics)에서 게시된 게시물 수와 요율 제한으로 인해 지연된 게시물이 있는지 확인하세요.
