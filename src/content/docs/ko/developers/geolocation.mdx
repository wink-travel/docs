Wink는 위치 기반 검색, 자동 통화 및 언어 감지, 지리적 인벤토리 필터링을 위해 지리 위치를 광범위하게 사용합니다.

## GeoIP 감지

사용자가 플랫폼을 방문하면 Wink는 IP 주소를 기반으로 자동으로 위치를 감지합니다. 이를 통해 다음이 가능합니다:

- **자동 통화 선택** — 사용자의 현지 통화로 가격 표시.
- **언어 선호도** — 사용자가 선호할 가능성이 높은 언어로 콘텐츠 제안.
- **근처 결과** — 사용자의 위치에 가까운 인벤토리 우선 표시.

## 공간 검색

Wink는 네 가지 공간 쿼리 유형을 사용하여 지리적 검색을 지원합니다. 이는 플랫폼 전반의 지도 기반 검색, 반경 검색 및 영역 기반 필터링을 구동합니다.

### 점(Point)

단일 지리 좌표(위도, 경도). 특정 위치를 표시하는 데 사용 — 예를 들어, 숙소 주소나 사용자의 현재 위치.

```json
{
  "point": {
    "x": -73.9857,
    "y": 40.7484
  }
}
```

### 원(Circle)

중심점과 반경. "내 주변 검색" 또는 반경 기반 쿼리에 사용 — 예: "타임스퀘어에서 10km 이내 호텔."

```json
{
  "point": {
    "x": -73.9857,
    "y": 40.7484
  },
  "radius": 10000
}
```

### 사각형(Rectangle)

남서쪽과 북동쪽 모서리로 정의된 경계 상자. 지도 기반 검색에 사용 — 사용자가 지도를 이동하거나 확대/축소할 때, 보이는 영역이 사각형 쿼리를 정의합니다.

```json
{
  "southWest": {
    "x": -74.0060,
    "y": 40.7128
  },
  "northEast": {
    "x": -73.9350,
    "y": 40.7831
  }
}
```

### 다각형(Polygon)

일련의 점으로 정의된 맞춤형 형태. 불규칙한 지리 경계 내에서 검색할 때 사용 — 예: 특정 동네, 도시 구역, 지도에 직접 그린 맞춤 영역.

```json
{
  "polygon": {
    "points": [
      { "x": -74.0060, "y": 40.7128 },
      { "x": -73.9350, "y": 40.7128 },
      { "x": -73.9350, "y": 40.7831 },
      { "x": -74.0060, "y": 40.7831 },
      { "x": -74.0060, "y": 40.7128 }
    ]
  }
}
```

:::note
다각형은 닫혀 있어야 합니다 — 마지막 점은 첫 번째 점과 일치해야 합니다.
:::

## 사용 사례

| 사용 사례 | 형태 유형 | 예시 |
|---|---|---|
| 내 주변 검색 | 원(Circle) | 내 위치에서 5km 이내 호텔 |
| 지도 뷰포트 검색 | 사각형(Rectangle) | 현재 지도 뷰에 보이는 숙소 |
| 동네 검색 | 다각형(Polygon) | 특정 도시 구역 내 호텔 |
| 숙소 위치 | 점(Point) | 숙소 목록의 정확한 위치 |

## 개발자를 위한 정보

공간 쿼리 매개변수는 Lookup API 및 Inventory API 엔드포인트에서 허용됩니다. 검색 요청에 적절한 GeoJSON 형태를 전달하여 위치별로 결과를 필터링하세요.

엔드포인트별 세부 정보는 [APIs documentation](/ko/developers/apis)에서 확인하세요.
