Wink에서 예약이 생성, 업데이트 또는 취소되면 플랫폼은 해당 변경 사항을 연결된 외부 시스템—숙박 관리 시스템(PMS), 채널 매니저, 중앙 예약 시스템(CRS)—과 동기화합니다.

## 작동 방식

각 예약 변경은 동기화 상태를 추적하는 **동기화 항목**을 생성합니다. 플랫폼은 동기화 항목을 비동기적으로 처리하며, 실패 시 재시도하여 안정적인 전달을 보장합니다.

## 동기화 수명 주기

모든 동기화 항목은 상태 머신을 따릅니다:

```
┌─────────────┐     success     ┌───────────┐
│  TO_CREATE   │ ──────────────→ │ PROCESSED │
└─────────────┘                  └───────────┘
       │
       │ failure
       ▼
┌──────────────────┐
│ TO_CREATE_FAILED │
└──────────────────┘
```

업데이트 및 취소에도 동일한 패턴이 적용됩니다:

| 작업 | 대기 상태 | 성공 상태 | 실패 상태 |
|---|---|---|---|
| 신규 예약 | `TO_CREATE` | `PROCESSED` | `TO_CREATE_FAILED` |
| 예약 업데이트 | `TO_UPDATE` | `PROCESSED` | `TO_UPDATE_FAILED` |
| 취소 | `TO_CANCEL` | `PROCESSED` | `TO_CANCEL_FAILED` |

## 동기화를 유발하는 경우

- **신규 예약** — 여행자가 Wink 또는 제휴 채널을 통해 예약할 때.
- **예약 업데이트** — 예약 세부 정보가 변경될 때(날짜, 투숙객 정보, 객실 배정).
- **취소** — 여행자 또는 숙소에서 예약을 취소할 때.

## 모니터링

연결된 채널 매니저 또는 PMS 대시보드를 통해 동기화 상태를 모니터링할 수 있습니다. 실패한 동기화 항목은 자동으로 재시도됩니다. 동기화가 계속 실패하면 조사 목적으로 `*_FAILED` 상태로 유지됩니다.

## 문제 해결

예약이 PMS에 나타나지 않는 경우:

1. **채널 매니저 연결 확인** — PMS 또는 채널 매니저 통합이 활성화되어 있고 올바르게 구성되었는지 확인하세요.
2. **웹훅 전달 확인** — 웹훅을 사용하는 경우, 엔드포인트가 `booking.create` 및 `booking.cancelled` 이벤트를 수신하는지 확인하세요. 자세한 내용은 [Webhook Events Catalog](/ko/developers/webhook-events#booking-events)를 참조하세요.
3. **실패한 항목 검토** — 실패한 동기화 항목은 대상 시스템과의 연결 또는 데이터 문제를 나타냅니다.
4. **지원팀에 문의** — 문제가 지속되면 Wink 팀이 실패한 동기화 항목을 대신 조사할 수 있습니다.
