AI 스킬은 코딩 에이전트에게 Wink와 작업하는 방법을 가르치는 휴대 가능한 지침 패키지입니다. 훈련 데이터나 오래된 문서에 의존하는 대신, 각 스킬은 에이전트가 API 계약 및 플랫폼 문서의 실시간 진실의 원천인 [Wink MCP 서버](/ko/developers/mcp)를 사용하도록 안내합니다.

스킬은 Claude Code, Codex, Cursor 등 스킬을 지원하는 모든 에이전트 클라이언트와 함께 작동합니다.

## 사용 가능한 스킬

### wink-api-builder

Wink API 통합을 구현, 검토 또는 디버깅합니다. 이 스킬은 에이전트를 구조화된 워크플로우로 안내합니다:

1. `api_search`를 통해 적절한 API 작업 검색
2. MCP 서버에서 실시간 요청/응답 계약 열기
3. 문서에서 인증 설정 가져오기
4. 검증된 계약에 따라 코드 작성
5. 기존 코드와 현재 API 간 불일치 표시

유용한 경우:
- 모든 Wink API에 대한 새로운 클라이언트 메서드 구축
- 계약 불일치 디버깅(예: HTTP 400 반환 요청)
- 실시간 사양에 따른 기존 통합 검토

## 설치

스킬 폴더를 에이전트의 스킬 디렉터리에 복사하세요.

:::tip
Wink MCP 서버는 동일한 환경에 연결되어 있어야 합니다. 설정 지침은 [MCP 서버](/ko/developers/mcp)를 참조하세요.
:::

**Claude Code**

```bash
# 저장소 로컬
cp -r skills/wink-api-builder .claude/skills/

# 또는 사용자 레벨
cp -r skills/wink-api-builder ~/.claude/skills/
```

다음과 같이 호출하세요:
```
/wink-api-builder add a TypeScript client for the Wink affiliate create flow
```

**Codex**

```bash
# 저장소 로컬
cp -r skills/wink-api-builder .agents/skills/

# 또는 사용자 레벨
cp -r skills/wink-api-builder $CODEX_HOME/skills/
```

다음과 같이 호출하세요:
```
Use $wink-api-builder to add a TypeScript client for the Wink affiliate create flow.
```

**기타 에이전트**

스킬 폴더를 에이전트가 스킬을 해석하는 위치에 배치하세요. 진입점은 `SKILL.md`입니다.
