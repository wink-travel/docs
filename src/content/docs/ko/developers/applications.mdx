애플리케이션을 사용하면 Wink와 TripPay의 모든 기능을 이용할 수 있으며, Facebook이나 Google처럼 사용자가 OAuth2 인증 서버를 사용하여 사이트에 인증하고 로그인할 수 있습니다.
Wink를 사용하여 나만의 OTA를 구축할 수 있습니다. 모든 것은 애플리케이션에서 시작됩니다.

*기본적으로 새로운 Wink 제휴 계정이나 TripPay 계정을 등록하면 앱이 제공됩니다.`client ID`일부 시나리오에서는.*

:::참고
Wink 또는 TripPay에서 생성된 애플리케이션은 모든 API 엔드포인트에 사용할 수 있습니다.
:::

*이 예에서는 다음을 통해 인증합니다.[윙크 스튜디오](https://studio.wink.travel).*

<Steps>
  1. 인증이 완료되면 오른쪽 상단에 있는 프로필 아이콘을 클릭하면 메뉴가 나타납니다.
  2. 클릭 `Applications` 링크.
  3. 앱 페이지로 리디렉션됩니다.
</Steps>


## 앱 만들기

접근할 수 없으므로`secret key`우리가 당신을 위해 만든 앱이 있지만, 당신도 당신만의 앱을 만들어야 합니다.

:::주의
앱을 만들기 전에 Wink에 제휴 계정을 만들거나 TripPay에 계정을 만들어야 합니다. 앱을 특정 계정에 연결하게 됩니다.
:::

신청서를 만드는 단계는 다음과 같습니다.

<Steps>
  1. 앱 페이지에서 다음을 클릭하세요. `Create new application` 단추.
  2. ***이름*** 앱에 이름을 지정하세요. *예: 쿨 앱*
  3. **엔터티 이름** 앱을 기존 계정 중 하나와 연결하세요. *예: 쿨 계정*
  4. **리디렉션 URI** 최소 하나의 리디렉션 URI가 필요합니다. 이는 인증 성공 후 리디렉션 도메인의 유효성을 검사하는 데 사용됩니다. *예를 들어 https://www.cool-site.com*
  5. 클릭 `Save` 계속하려면 버튼을 클릭하세요.
</Steps>


애플리케이션 목록으로 돌아갑니다. 목록에 새 앱이 표시되며, 앱 비밀 키는 이번 한 번만 접근할 수 있습니다. 키를 복사하여 안전한 곳에 보관하세요.

## 앱 업데이트

리디렉션 URI 중 하나가 변경되거나 URI를 추가 또는 제거하려면 앱을 업데이트해야 합니다.

기존 앱을 업데이트하려면 다음을 수행하세요.

<Steps>
  1. 클릭 `Actions` 업데이트하려는 앱에 대한 링크입니다.
  2. 클릭 `Update` 작업 아래의 버튼을 클릭하세요.
  3. 앱을 변경하세요.
  4. 클릭 `Save` 계속하려면 버튼을 클릭하세요.
</Steps>


귀하의 신청서 목록으로 다시 돌아가게 됩니다.

## 앱 제거

:::주의
계정의 모든 기능이 정상적으로 작동하도록 자동 생성된 앱을 삭제하지 마세요.
:::

더 이상 앱을 사용할 필요가 없다면 삭제해도 됩니다.

앱을 제거하려면 다음을 수행하세요.

<Steps>
  1. 클릭 `Actions` 업데이트하려는 앱에 대한 링크입니다.
  2. 클릭 `Remove` 작업 아래의 버튼을 클릭하세요.
  3. 딸깍 하는 소리 `OK` 삭제를 확인하려면 버튼을 클릭하세요.
</Steps>


귀하의 신청서 목록으로 다시 돌아가게 됩니다.
