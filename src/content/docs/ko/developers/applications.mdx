애플리케이션은 Wink와 TripPay의 기능에 완전한 접근 권한을 제공하며, 사용자가 당사의 OAuth2 인증 서버를 사용하여 귀하의 사이트에 로그인하고 인증할 수 있게 합니다; Facebook과 Google과 같은 방식입니다.  
Wink로 자체 OTA를 구축할 수 있습니다. 모든 것은 애플리케이션에서 시작됩니다.

*기본적으로, 새로운 Wink 제휴 계정이나 TripPay 계정을 등록하면 앱이 제공됩니다. 일부 시나리오에서는 `client ID`를 사용합니다.*

:::note
Wink 또는 TripPay에서 생성된 애플리케이션은 모든 API 엔드포인트에 사용할 수 있습니다.
:::

*이 예제에서는 [Wink Studio](https://studio.wink.travel)를 통해 인증합니다.*

<Steps>
1. 인증이 완료되면, 오른쪽 상단의 프로필 아이콘을 클릭하면 메뉴가 나타납니다.
2. `Applications` 링크를 클릭합니다.
3. 앱 페이지로 리디렉션됩니다.
</Steps>

## 앱 생성

당신을 위해 생성된 앱의 `secret key`에 접근할 수 없으므로, 직접 앱을 생성해야 합니다.

:::caution
애플리케이션을 특정 계정에 연결해야 하므로, 앱을 생성하기 전에 Wink 제휴 계정이나 TripPay 계정을 만들어야 합니다.
:::

애플리케이션 생성 단계는 다음과 같습니다:

<Steps>
1. 앱 페이지에서 `Create new application` 버튼을 클릭합니다.
2. ***이름*** 앱 이름을 지정합니다. *예: Cool App*
3. **엔터티 이름** 기존 계정 중 하나와 앱을 연결합니다. *예: Cool Account*
4. **리디렉션 URI** 최소 하나의 리디렉션 URI가 필요합니다. 인증 성공 후 리디렉션 도메인을 검증하는 데 사용됩니다. *예: https://www.cool-site.com*
5. 계속하려면 `Save` 버튼을 클릭합니다.
</Steps>

애플리케이션 목록으로 다시 리디렉션됩니다. 새 앱이 목록에 표시되며, 앱 비밀 키는 이때만 확인할 수 있습니다. 반드시 복사하여 안전한 곳에 보관하세요.

## 앱 업데이트

리디렉션 URI가 변경되었거나 URI를 추가 또는 제거하려면 앱을 업데이트해야 합니다.

기존 앱을 업데이트하려면 다음을 수행하세요:

<Steps>
1. 업데이트할 앱의 `Actions` 링크를 클릭합니다.
2. Actions 아래의 `Update` 버튼을 클릭합니다.
3. 앱 정보를 수정합니다.
4. 계속하려면 `Save` 버튼을 클릭합니다.
</Steps>

애플리케이션 목록으로 다시 리디렉션됩니다.

## 앱 삭제

:::caution
계정의 모든 기능이 정상 작동하도록 자동 생성된 앱은 삭제하지 마세요.
:::

더 이상 사용하지 않는 앱은 삭제할 수 있습니다.

앱을 삭제하려면 다음을 수행하세요:

<Steps>
1. 삭제할 앱의 `Actions` 링크를 클릭합니다.
2. Actions 아래의 `Remove` 버튼을 클릭합니다.
3. 삭제 확인을 위해 `OK` 버튼을 클릭합니다.
</Steps>

애플리케이션 목록으로 다시 리디렉션됩니다.
