웹후크는 Wink 및 TripPay에서 귀하의 계정과 관련된 실시간 이벤트를 수신하고 반응할 수 있는 방법입니다.

## 예시

<Steps>
1. Wink에서 예약이 발생합니다.
1. Wink 이벤트 `booking.created`가 전송됩니다.
2. 귀하의 시스템이 이 이벤트를 적극적으로 수신 대기합니다.
3. 귀하의 시스템은 자체 분석을 추적하고 이 이벤트를 사용하여 시계열 데이터베이스를 업데이트합니다.
</Steps>

:::note
Wink 웹후크는 TripPay 웹후크와 다릅니다.

- Wink 이벤트를 수신하려면 [Extranet](https://extranet.wink.travel) 또는 [Studio](https://studio.wink.travel)에서 웹후크를 생성할 수 있습니다.
- TripPay 이벤트를 수신하려면 [Payment](https://pay.wink.travel)에서 웹후크를 생성할 수 있습니다.
:::

*이 예시에서는 [Studio](https://studio.wink.travel)를 통해 인증합니다.*

<Steps>
1. 인증이 완료되면, 오른쪽 상단의 프로필 아이콘을 클릭하면 메뉴가 나타납니다.
2. `Webhooks` 링크를 클릭합니다.
3. 웹후크 페이지로 리디렉션됩니다.
</Steps>

## 웹후크 생성

:::caution
웹후크를 생성하기 전에 Wink에서 제휴사 계정을 만들거나 TripPay에서 계정을 만들어야 합니다. 웹후크가 특정 계정에 연결되기 때문입니다.
:::

웹후크 생성 절차는 다음과 같습니다:

<Steps>
1. 웹후크 페이지에서 `Create new webhook` 버튼을 클릭합니다.
2. ***이름*** 웹후크 이름을 지정합니다. *예: Wink Stalker*
3. ***활성화*** 활성 상태로 둡니다.
4. **엔터티 이름** 기존 계정 중 하나와 웹후크를 연결합니다. *예: Cool Account*
5. **이벤트** 관심 있는 이벤트를 선택합니다. *예: `booking.created`*
6. **웹후크 URL** 이벤트를 수신할 URL을 입력합니다. *예: `https://my.system.com/webhook/wink`*
7. 계속하려면 `Save` 버튼을 클릭합니다.
</Steps>

웹후크 목록으로 리디렉션되며 새 웹후크가 준비됩니다.

## 웹후크 업데이트

다음과 같은 경우 앱을 업데이트해야 합니다:

- 웹후크를 비활성화하려는 경우.
- 더 많거나 적은 이벤트를 수신하려는 경우.
- 수신할 웹후크 URL을 변경하려는 경우.

기존 웹후크를 업데이트하려면 다음을 수행하세요:

<Steps>
1. 업데이트할 웹후크의 `Actions` 링크를 클릭합니다.
2. Actions 아래의 `Update` 버튼을 클릭합니다.
3. 웹후크를 수정합니다.
4. 계속하려면 `Save` 버튼을 클릭합니다.
</Steps>

웹후크 목록으로 리디렉션됩니다.

## 웹후크 제거

더 이상 웹후크가 필요하지 않으면 제거할 수 있습니다.

웹후크를 제거하려면 다음을 수행하세요:

<Steps>
1. 제거할 웹후크의 `Actions` 링크를 클릭합니다.
2. Actions 아래의 `Remove` 버튼을 클릭합니다.
3. 삭제 확인을 위해 `OK` 버튼을 클릭합니다.
</Steps>

웹후크 목록으로 리디렉션됩니다.
