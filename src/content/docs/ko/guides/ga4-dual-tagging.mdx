Google Analytics 4 속성을 Wink에 연결하면 객실 조회부터 완료된 구매까지의 예약 이벤트가 Wink 자체 애널리틱스와 함께 자동으로 귀하의 GA4 대시보드로 전달됩니다. 코딩이나 기술적 설정이 필요 없습니다.

<Aside type="tip">
필요한 것은 GA4 측정 ID뿐입니다. Wink 포털에 입력하면 즉시 애널리틱스가 활성화됩니다.
</Aside>

---

## 작동 방식

```
Wink 포털에 G-XXXXXXXXXX를 입력합니다
       ↓
Wink가 예약 이벤트를 귀하의 GA4 속성으로 전달합니다
       ↓
analytics.google.com에 로그인하여 예약 내역을 확인합니다
```

---

## 해야 할 일

<Steps>

1. **GA4 속성 생성**

   [analytics.google.com](https://analytics.google.com)으로 이동: 관리자 → 속성 생성. 시장에 맞게 시간대와 통화를 설정하세요.

2. **웹 데이터 스트림 생성**

   관리자 → 데이터 스트림 → 스트림 추가 → 웹. URL에 `wink.travel`을 입력하세요. 그러면 **측정 ID**(`G-XXXXXXXXXX`)가 생성됩니다. 복사하세요.

3. **Wink 포털에 측정 ID 입력**

   [my.wink.travel](https://my.wink.travel)에 로그인하여 계정을 선택한 후 **사용자 지정** 탭으로 이동합니다. 측정 ID를 붙여넣고 저장하세요. 이제 GA4 통합이 활성화됩니다.

4. **`purchase`를 주요 이벤트로 표시**

   GA4 속성에서: 관리자 → 이벤트 → `purchase` 찾기 → **주요 이벤트로 표시** 토글 활성화. 완료된 예약이 전환 보고서에 나타납니다.

5. **데이터 확인**

   [analytics.google.com](https://analytics.google.com)에 로그인하면 예약, 수익, 고객 여정이 Wink를 통해 자동으로 추적되어 표시됩니다.

</Steps>

<Aside type="note">
이 통합을 위해 귀하의 웹사이트에 추적 코드를 설치할 필요가 없습니다. Wink가 모든 것을 처리합니다. 측정 ID는 예약 데이터가 전송되는 목적지일 뿐입니다.
</Aside>

## 추적 가능한 항목

연결되면 귀하의 GA4 속성은 귀하의 숙소에 대한 향상된 전자상거래 이벤트 전체를 수신합니다:

### 발견

| 이벤트 | 의미 |
|---|---|
| `view_item_list` | 고객이 검색 또는 카테고리 목록에서 귀하의 숙소를 봄 |
| `select_item` | 고객이 목록에서 귀하의 숙소를 클릭함 |
| `view_item` | 고객이 호텔 상세 페이지를 봄 |

### 고려

| 이벤트 | 의미 |
|---|---|
| `view_promotion` | 고객이 귀하의 숙소 프로모션을 봄 |
| `select_promotion` | 고객이 프로모션을 클릭함 |
| `add_to_wishlist` | 고객이 숙소를 위시리스트에 저장함 |

### 예약 퍼널

| 이벤트 | 의미 |
|---|---|
| `add_to_cart` | 고객이 객실 또는 패키지를 선택함 |
| `view_cart` | 고객이 예약 요약을 봄 |
| `remove_from_cart` | 고객이 객실 또는 패키지를 제거함 |
| `begin_checkout` | 고객이 결제 과정을 시작함 |
| `add_payment_info` | 고객이 결제 정보를 입력함 |
| `add_shipping_info` | 고객이 게스트 정보를 입력함 |
| `purchase` | 고객이 예약을 완료함 |
| `refund` | 예약이 취소되고 환불됨 |

모든 이벤트에는 수익, 통화, 객실 유형, 예약 가치 등 관련 차원이 포함되어 있어 첫 인상부터 완료된 예약까지 고객 여정을 완벽하게 파악할 수 있습니다. `city_name`, `cancellation_policy`, `bed_type` 등 추가 숙소 및 객실 수준 차원을 활성화하려면 [GA4의 맞춤 차원](/ko/guides/gtm-dual-tagging#custom-dimensions-in-ga4)을 참조하세요.

## 교차 도메인 추적 (고급)

위의 표준 통합은 Wink 내에서 발생하는 모든 것을 추적합니다. 그러나 고객이 귀하의 웹사이트에서 시작해 Wink에서 예약을 완료할 때까지 전체 여정을 추적하고 원래 트래픽 출처(구글 광고, 이메일 캠페인, 유기적 검색 등)를 올바르게 귀속하려면 양쪽에서 추가 설정이 필요합니다.

<Aside type="caution">
이 섹션은 귀하의 웹사이트에 GA4가 설치된 경우에만 해당됩니다. 사이트에 GA4를 운영하지 않는 경우 위의 표준 통합만으로 충분합니다.
</Aside>

### 왜 중요한가요

교차 도메인 추적이 없으면, 고객이 먼저 귀하의 웹사이트를 방문하고 Wink에서 예약할 경우 GA4에서는 이전 방문과 연결되지 않은 **직접 방문** 세션으로 나타납니다. 예약에 이르게 한 구글 광고 클릭, 이메일 캠페인, 유기적 검색 방문은 인정을 받지 못합니다.

교차 도메인 추적이 설정되면 전체 고객 여정을 볼 수 있습니다:

```
google.com (유료 광고)
       ↓
귀하의 웹사이트 (고객이 객실을 탐색)
       ↓
wink.travel (고객이 객실 선택 및 예약)
       ↓
구매가 원래 구글 광고 클릭에 올바르게 귀속됨
```

### 귀하가 해야 할 일

GA4 속성에서 `wink.travel`을 외부 추천이 아닌 귀하 도메인의 일부로 인식하도록 구성하세요. 이렇게 하면 GA4가 두 세션을 하나의 연속된 고객 여정으로 연결합니다.

<Steps>
1. GA4 관리자 → 데이터 스트림 → 해당 스트림 선택
2. 태그 설정 구성 → **도메인 구성**
3. `wink.travel`을 연결된 도메인으로 추가
4. 저장
</Steps>

### Wink가 해야 할 일

[Wink 지원](https://my.wink.travel)에 연락하여 귀하의 웹사이트 도메인(예: `yourhotel.com`)을 알려주세요. Wink가 Wink GA4 속성의 추천 제외 목록에 귀하 도메인을 추가합니다. 이렇게 하면 귀하의 웹사이트가 추천 출처로 기록되는 것을 방지하여 고객의 원래 귀속 정보가 덮어쓰여지고 새 세션이 시작되는 것을 막습니다.

<Aside type="tip">
위의 표준 통합은 대부분의 숙소에 충분하며 Wink에서 예약 및 수익을 즉시 확인할 수 있습니다. 교차 도메인 추적은 GA4 데이터를 충분히 이해하고 더 깊은 귀속 인사이트가 필요할 때 설정할 가치가 있는 고급 옵션입니다.
</Aside>
