[Wink IAM](https://iam.wink.travel)는 Wink **사용자 계정**의 아이덴티티 서비스입니다 — [Portal](/ko/portal/overview)에서 관리하는 **조직 계정**(숙소, 제휴 계정 등)과는 별개입니다.

## 여기서 관리하는 항목

- **`/my-account/general`** — 사용자 프로필: 이름, 성, 사용자 이름, 이메일, 프로필 사진, 비밀번호.
- **`/my-account/security`** — 비밀번호 없는 로그인용 패스키(WebAuthn).
- **`/my-account/apps`** — Wink 계정 접근 권한을 부여한 OAuth2 애플리케이션. 언제든지 접근 권한을 취소할 수 있습니다.

## 사용자 설정과 Portal의 차이점

| 개념 | 위치 | 예시 |
|---|---|---|
| **사용자 아이덴티티** (사람, 사용자 본인) | `iam.wink.travel` | 이메일, 비밀번호, 패스키, 프로필 사진 |
| **조직 계정** (운영하는 숙소, 제휴 조직 등) | `app.wink.travel` | 브랜딩, 관리자, 정산, 애플리케이션, 웹훅 |

한 명의 사용자가 여러 조직 계정을 운영할 수 있으며, 한 조직 계정은 여러 사용자가 ([관리자](/ko/portal/settings/managers)를 통해) 운영할 수 있습니다.

## 로그인 흐름

로그인은 항상 IAM에서 시작합니다. 인증이 완료되면 Portal로 이동하여 작업할 조직 계정을 선택합니다.
