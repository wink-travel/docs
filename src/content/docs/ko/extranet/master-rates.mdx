:::note
첫 번째 마스터 요금 생성은 온보딩 마법사의 일부입니다.
:::

:::note
`마스터 요금`은 고유한 `객실 / 요금제` 조합입니다.

*대부분의 여행 요금 제공자는 이 맥락에서 객실과 요금제만 사용합니다. OTA나 예약 엔진에서 객실이 판매 목록에 올라가면 여행객들은 요금제를 비교하게 됩니다.*

Wink는 고유 조합에 더 많은 유연성을 부여하기 위해 `마스터 요금`이라는 용어를 추가했습니다.
:::

:::note
객실이나 요금제가 추가되거나 제거될 때 마스터 요금도 자동으로 생성/삭제됩니다.
:::

마스터 요금을 관리하려면 메인 내비게이션 바에서 `Monetize > Master Rate`를 클릭하세요.

숙소는 모든 객실 / 요금제 조합에 대해 마스터 요금을 가집니다.

*예: 숙소에 객실 1개와 요금제 2개가 있다면, 모든 조건이 충족될 경우 여행객은 2개의 마스터 요금을 구매할 수 있습니다.*

마스터 요금 항목은 두 가지 관리 옵션이 있습니다:

- 마스터 요금 사용 가능 여부 전환.
- 혜택 설정.

## 사용 가능 여부

마스터 요금을 활성화하거나 비활성화하면 모든 판매 채널에서 사용 가능 여부에 영향을 줍니다. 이 스위치는 특정 마스터 요금을 모든 곳에서 즉시 끌 수 있는 빠른 방법입니다.

## 혜택

여행객이 객실을 예약할 때 객실 자체와 직접 상호작용하지 않습니다. 대신 해당 객실의 마스터 요금과 상호작용합니다.
마스터 요금에는 특정 객실 / 요금제 조합에만 적용되는 고유 혜택을 부여할 수 있어, 숙소 소유자가 해당 조합을 유인하는 또 다른 방법을 제공합니다.

혜택을 선택할 때 기억해야 할 두 가지:

- 보장된 혜택인지, 사용 가능 여부에 따른 혜택인지.
- 수치적 가치.

보장된 혜택은 고객에게 더 큰 가치를 제공하므로 더 높은 수치적 가치를 가집니다.
모든 호텔은 모든 마스터 요금의 모든 혜택의 수치적 가치를 합산한 `혜택 점수`를 받습니다.
할인을 찾는 여행객은 혜택 점수가 높은 호텔을 기준으로 정렬할 수 있습니다.

혜택은 다음을 위한 수단입니다:

- 객실 / 요금제 조합을 차별화.
- 더 높은 가시성 달성. *예: 해당 지역 내 다른 호텔보다 혜택 점수에서 더 높은 점수를 받음.*

:::tip
혜택은 호텔이 자신을 차별화하고 `시스템을 활용`하려는 욕구에서 나왔습니다.
이전에는 호텔들이 더 높은 수수료를 주고 OTA에서 더 좋은 노출을 받으려다 결국 수익을 모두 잃는 결과를 낳았습니다.
역사적으로 수수료가 호텔이 활용할 수 있는 유일한 수단이었습니다.

혜택은 호텔이 실제 비용 없이 여행객에게 보상할 수 있는 방법입니다.

다른 유형의 혜택에 대한 제안이 있으면 [문의해 주세요](mailto:hi@wink.travel).
:::

판매 채널에 재고를 활성화하는 두 가지 방법:

- 카드에서 재고 사용 가능 여부 전환. *위 그림 참조*
- 메인 내비게이션 바에서 `Distribution > Inventory`로 이동하여 모든 판매 채널의 재고를 관리.

## API

`Master Rates`를 관리하려는 개발자는 [Developers > APIs > Monetize](/ko/developers/apis/#monetize-api)로 이동하세요.
