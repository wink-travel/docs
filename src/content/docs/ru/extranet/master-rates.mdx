:::note
Создание вашей первой основной ставки является частью мастера адаптации.
:::

:::примечание
А`master rate`является уникальным`guest room / rate plan` пара.

*Большинство поставщиков тарифов на поездки используют в этом контексте только гостевой номер и тарифный план. Когда вы видите свой гостевой номер, выставленный на продажу на OTA или в системах бронирования, путешественники будут сравнивать тарифные планы.*

Wink добавил термин`master rate`чтобы иметь возможность придать уникальной комбинации большую гибкость.
:::

:::note
Основные тарифы создаются/удаляются автоматически при добавлении или удалении гостевого номера или тарифного плана.
:::

Чтобы управлять своими основными ставками, нажмите`Monetize > Master Rate`из главной навигационной панели.

В объекте размещения будет действовать общая ставка для каждой комбинации номера/тарифного плана.

*Пример: если в вашем объекте размещения есть один гостевой номер и два тарифных плана, гость увидит два основных тарифа, доступных для продажи (при соблюдении всех условий).*

Запись основной ставки имеет два варианта управления:

* Переключить доступность основной ставки.
* Установите льготы.

## Доступность

Когда вы включаете/отключаете основную ставку, это влияет на доступность по всем вашим каналам продаж. Этот переключатель — быстрый способ отключить определенную основную ставку везде, немедленно.

## Перки

Когда путешественники хотят забронировать гостевой номер, они не взаимодействуют с гостевым номером напрямую. Вместо этого они взаимодействуют с основным тарифом этого гостевого номера.
Основной тариф может быть загружен уникальными льготами, которые доступны только для определенной комбинации гостевого номера/тарифного плана; тем самым предоставляя владельцам недвижимости еще один способ стимулировать эту комбинацию.

При выборе бонуса следует учитывать две вещи:

* Гарантировано ли это или зависит от наличия.
* Его числовое значение.

Гарантированный бонус имеет более высокое числовое значение, поскольку он представляет большую ценность для клиентов.
Каждый отель получит`perk score`это основано на совокупной числовой стоимости всех льгот по всем основным тарифам.
Путешественники, ищущие выгодное предложение, могут сортировать отели по высоким оценкам льгот.

Льготы — это способ:

* Различайте комбинацию номера/тарифного плана.
* Достичь большей видимости.*т. е. получить более высокий балл по шкале преимуществ, чем другие отели в вашем районе.*

:::tip
Преимущества появились из-за желания отелей выделиться и`game the system`.
В прошлый раз, когда они это сделали, они буквально убили себя, отдав всю свою прибыль OTA для лучшего размещения за счет более высоких комиссий.
Исторически комиссии были единственным, на чем отели могли геймифицировать.

Бонусы — это способ для отелей вознаградить путешественников, не обязательно подразумевающий реальные расходы на проживание в отеле.

Если у вас есть предложения по другим типам привилегий, которые можно раздать,[свяжитесь с нами](mailto:hi@wink.travel).
:::

* Переключить доступность инвентаря на карте.*См. рисунок выше.*
* Перейти к`Distribution > Inventory`из главной навигационной панели, предназначенной для управления запасами по всем каналам продаж.

## API

Разработчики, которые хотят управлять`Master Rates`можете направиться к[Разработчики > API > Монетизация](/developers/apis/#monetize-api).
