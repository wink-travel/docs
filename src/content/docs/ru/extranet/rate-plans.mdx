Тарифные планы — это фильтр, через который гостевые номера представлены пользователям.
Они также содержат настройки, управляющие структурой ценообразования в определённых сценариях.

Тарифные планы позволяют:

- Устанавливать политику отмены [и исключения из политики].
- Устанавливать цены для дополнительных детей, взрослых и одиночных путешественников.
- Устанавливать питание.
- Добавлять сборы за раннее заселение и поздний выезд.
- Устанавливать дополнительные сборы.
- Устанавливать ограничения.

Чтобы создать тарифный план, нажмите кнопку `Create a new rate plan`.

## Конфигурация

Вкладка `Configurations` выбрана по умолчанию при начале работы.

Этот раздел охватывает основные параметры для настройки вашего тарифного плана.

- **Название тарифного плана** Название тарифного плана, под которым вы его используете внутренне. *например, BAR RO*
- **Включён** Включите или отключите этот тарифный план. При отключении тарифный план не будет доступен для бронирования на любом канале продаж.
- **Предоплата** Установите, требует ли этот тарифный план предоплату в момент бронирования.
- **Бонусные баллы** Учитываются ли бонусные баллы вашей программы лояльности по этому тарифному плану. *например, Нет*
- **Модификатор цены для одного проживающего** Установите ценовой модификатор для одного проживающего в номере с несколькими местами. *например, скидка 5% от обычной цены*
- **Модификатор цены для дополнительного взрослого** Установите ценовой модификатор для дополнительного взрослого в номере. *например, +10 / взрослый*
- **Модификатор цены для дополнительного ребёнка** Установите ценовой модификатор для дополнительного ребёнка в номере. *например, +5 / ребёнок*

### Политика отмены

- **Политика отмены** Установите политику отмены для этого тарифного плана. *например, Возвратный*
- **Исключения** Установите дополнительные политики отмены, которые вступают в силу для определённых диапазонов дат. *например, на декабрь используйте политику `Возвратный с дополнительными условиями`.*

### Питание

Выберите планы питания, которые включены в этот тарифный план. Доступны следующие варианты:

- **Завтрак**
- **Бранч**
- **Обед**
- **Ужин**
- **Все включено**
- **Все включено + алкоголь**

## Сборы

Установите дополнительные сборы, сопровождающие тарифный план.

- **Раннее заселение** Установите дополнительный сбор, если гость хочет заселиться раньше. *Это только для информационных целей*
- **Поздний выезд** Установите дополнительный сбор, если гость хочет выехать позже. *Это только для информационных целей*

### Дополнительные сборы

Установите дополнительные сборы вместе с тарифным планом.

Чтобы создать дополнительный сбор, нажмите кнопку `Create extra charge`.

- **Тип ценообразования** Выберите, как рассчитывается этот сбор. Доступные типы:
  - *За проживание* — взимается один раз за бронирование.
  - *За день* — взимается за каждую ночь проживания.
  - *За человека / За проживание* — взимается за каждого гостя за бронирование.
  - *За человека / За день* — взимается за каждого гостя за ночь.
- **Сумма** Установите фиксированную сумму сбора. *например, 100*

Заполните название и описание этого сбора.

:::note
Необходимо написать хотя бы одно описание, и оно должно быть на английском языке.
:::

- **Название** Название сбора. *например, Одноразовый сбор за уборку*
- **Описание** Опишите сбор в одном-двух абзацах. *например, Вилла будет убрана нашей профессиональной командой уборщиков перед вашим приездом.*

## Ограничения

Установите ограничения, которые контролируют доступность гостевого номера / тарифного плана.

Поддерживаемые ограничения:

- Длительность проживания
- Заблаговременное бронирование
- Заполняемость
- Возраст
- Даты продажи / проживания / доступные дни / прибытия / отъезда / обязательные даты

### Длительность проживания

При добавлении этого ограничения вы требуете, чтобы гости "проживали не менее" и "не более" указанного количества дней.

*Пример: Этот тип номера доступен только гостям, которые проживают более 10 дней. Примечание: Максимальная длительность проживания необязательна.*

### Заблаговременное бронирование

При добавлении этого ограничения вы требуете, чтобы гости "бронировали не менее" и "не более" указанного количества дней заранее.

*Пример: Эта акция доступна только гостям, которые бронируют более чем за 10 дней. Примечание: Также требуется указать максимальный срок заблаговременного бронирования.*

### Заполняемость

При добавлении этого ограничения вы требуете, чтобы прибывающая группа "состояла не менее" и "не более" указанного количества человек.

### Возраст

При добавлении этого ограничения вы требуете, чтобы гости "были не младше" и "не старше" указанного возраста.

### Даты продажи

При добавлении этого ограничения вы хотите сделать этот тариф доступным для бронирования только в определённом диапазоне дат. Гость увидит этот тариф только если он бронирует номер В указанную дату в этом диапазоне.

### Даты проживания

При добавлении этого ограничения вы хотите сделать этот тариф доступным для проживания только в определённом диапазоне дат. Гость увидит этот тариф только если он бронирует номер НА указанную дату в этом диапазоне.

### Доступные дни недели

:::note
Это ограничение хорошо сочетается с ограничением "Даты продажи", где вы можете ограничить диапазон дат и дополнительно ограничить его днями недели с помощью этого ограничения.
:::

Включив это ограничение, вы разрешаете поиск этого тарифа ТОЛЬКО в указанные дни недели.

### Даты прибытия

:::note
Это ограничение хорошо сочетается с ограничением "Даты проживания", где вы можете ограничить диапазон дат и дополнительно ограничить дни недели с помощью этого ограничения.
:::

Включив это ограничение, гости ДОЛЖНЫ прибыть в этот день, чтобы увидеть этот тариф.

*Пример: Включение понедельника требует, чтобы гость прибыл в понедельник.*

### Даты отъезда

:::note
Это ограничение хорошо сочетается с ограничением "Даты проживания", где вы можете ограничить диапазон дат и дополнительно ограничить дни недели с помощью этого ограничения.
:::

Включив это ограничение, гости ДОЛЖНЫ уехать в этот день, чтобы увидеть этот тариф.

*Пример: Включение понедельника требует, чтобы гость уехал в понедельник.*

### Обязательные даты

:::note
Это ограничение хорошо сочетается с ограничением "Даты проживания", где вы можете ограничить диапазон дат и дополнительно ограничить дни недели с помощью этого ограничения.
:::

Включив это ограничение, гости ДОЛЖНЫ проживать "хотя бы в один" из этих дней, чтобы увидеть этот тариф.

*Пример: Включение понедельника требует, чтобы гости проживали в понедельник.*
