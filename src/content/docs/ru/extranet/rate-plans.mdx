Тарифные планы — это фильтр, через который гостевые номера представлены пользователям.
Они также содержат настройки, которые контролируют структуру ценообразования в определённых сценариях.

Тарифные планы позволяют:

- Установить политику отмены [и исключения из политики].
- Учитывать доплату за дополнительных детей, взрослых и одиночных путешественников.
- Установить питание.
- Добавить плату за раннее заселение и поздний выезд.
- Установить дополнительные сборы.
- Установить ограничения.

Чтобы создать тарифный план, нажмите кнопку `Create a new rate plan`.

## Конфигурация

Вкладка `Configurations` выбрана по умолчанию при начале работы.

В этом разделе описаны основные параметры для настройки вашего тарифного плана.

- **Название тарифного плана** Название тарифного плана, под которым вы его используете внутренне. *например, BAR RO*
- **Бонусные баллы** Начисляются ли гостю ваши бонусные баллы по этому тарифному плану. *например, Нет*
- **Одиночное размещение** Установите модификатор цены для одного гостя в номере с несколькими местами. *например, скидка 5% от обычной цены*
- **Дополнительный взрослый** Установите модификатор цены за дополнительного взрослого в номере. *например, +10 / взрослый*
- **Дополнительный ребенок** Установите модификатор цены за дополнительного ребенка в номере. *например, +5 / ребенок*

### Политика отмены

- **Политика отмены** Установите политику отмены для этого тарифного плана. *например, Возвратный*
- **Исключения** Установите дополнительные политики отмены, которые вступают в силу для определённых диапазонов дат. *например, Для декабря используйте политику `Возвратный с дополнительными условиями`.*

### Питание

Установите питание, включённое в этот тарифный план. *например, Завтрак включён*

## Дополнительные сборы

Установите дополнительные сборы, сопровождающие тарифный план.

- **Раннее заселение** Установите дополнительный сбор, если гость хочет заселиться раньше. *Только для информационных целей*
- **Поздний выезд** Установите дополнительный сбор, если гость хочет выехать позже. *Только для информационных целей*

### Дополнительные сборы

Установите дополнительные сборы в тарифном плане.

Чтобы создать дополнительный сбор, нажмите кнопку `Create extra charge`.

- **Тип ценообразования** Выберите, как рассчитывается этот сбор. *например, За весь период*
- **Сумма** Установите фиксированную сумму сбора. *например, 100*

Заполните название и описание этого сбора.

:::note
Необходимо написать хотя бы одно описание, и оно должно быть на английском языке.
:::

- **Название** Название сбора. *например, Одноразовый сбор за уборку*
- **Описание** Опишите сбор в одном-двух абзацах. *например, Вилла будет убрана нашей профессиональной командой уборщиков до вашего приезда.*

## Ограничения

Установите ограничения, которые контролируют доступность гостевого номера / тарифного плана.

Поддерживаемые ограничения:

- Длительность проживания
- Заблаговременное бронирование
- Заполняемость
- Возраст
- Даты продажи / проживания / бронирования / прибытия / отъезда / обязательные даты

### Длительность проживания

При добавлении этого ограничения вы требуете, чтобы гости "проживали не менее" и "не более" указанного количества дней.

*Пример: Этот тип номера доступен только гостям, которые проживают более 10 дней. Примечание: Максимальная длительность проживания необязательна.*

### Заблаговременное бронирование

При добавлении этого ограничения вы требуете, чтобы гости "бронировали не менее" и "не более" указанного количества дней заранее.

*Пример: Эта акция доступна только гостям, которые бронируют более чем за 10 дней. Примечание: Также требуется указать максимальный срок заблаговременного бронирования.*

### Заполняемость

При добавлении этого ограничения вы требуете, чтобы прибывающая группа "состояла не менее" и "не более" из указанного количества человек.

### Возраст

При добавлении этого ограничения вы требуете, чтобы гости "были не моложе" и "не старше" указанного возраста.

### Даты продажи

При добавлении этого ограничения вы хотите сделать этот тариф доступным для бронирования только в определённом диапазоне дат. Гость увидит этот тариф только если он бронирует номер В указанную дату в этом диапазоне.

### Даты проживания

При добавлении этого ограничения вы хотите сделать этот тариф доступным для проживания только в определённом диапазоне дат. Гость увидит этот тариф только если он бронирует номер НА указанную дату в этом диапазоне.

### Даты бронирования

:::note
Это ограничение хорошо сочетается с ограничением "Даты продажи", где вы можете ограничить диапазон дат и дополнительно ограничить дни недели с помощью этого ограничения.
:::

Включив это ограничение, вы разрешаете искать этот тариф ТОЛЬКО в указанные дни.

### Даты прибытия

:::note
Это ограничение хорошо сочетается с ограничением "Даты проживания", где вы можете ограничить диапазон дат и дополнительно ограничить дни недели с помощью этого ограничения.
:::

Включив это ограничение, гости ДОЛЖНЫ прибыть в этот день, чтобы увидеть этот тариф.

*Пример: Включение понедельника требует, чтобы гость прибыл в понедельник.*

### Даты отъезда

:::note
Это ограничение хорошо сочетается с ограничением "Даты проживания", где вы можете ограничить диапазон дат и дополнительно ограничить дни недели с помощью этого ограничения.
:::

Включив это ограничение, гости ДОЛЖНЫ уехать в этот день, чтобы увидеть этот тариф.

*Пример: Включение понедельника требует, чтобы гость уехал в понедельник.*

### Обязательные даты

:::note
Это ограничение хорошо сочетается с ограничением "Даты проживания", где вы можете ограничить диапазон дат и дополнительно ограничить дни недели с помощью этого ограничения.
:::

Включив это ограничение, гости ДОЛЖНЫ проживать "хотя бы в один" из этих дней, чтобы увидеть этот тариф.

*Пример: Включение понедельника требует, чтобы гости проживали в понедельник.*
