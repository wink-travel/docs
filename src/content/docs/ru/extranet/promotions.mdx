Акции — это способ для отеля создать уникальные условия, при которых путешественник получает дополнительное снижение цены [или надбавку].
:::

Для управления вашими акциями нажмите `Monetize > Single Promotions` в главном навигационном меню.

## Конфигурация

Этот раздел отвечает за установку фактической денежной стоимости акции.

- **Name** Введите название этой акции, которое будете видеть только вы.
- **Type** Выберите, хотите ли вы, чтобы акция содержала скидку или надбавку.
- **Pricing type** Выберите, как вы хотите рассчитывать этот пакет.
    - `Per stay`: Сумма применяется к последней ночи. *например, 100% скидка > Останьтесь 2 ночи — 3-я ночь бесплатно.*
    - `Per night`: Сумма умножается на количество ночей.
    - `Per use`: Дополнительных расчетов не происходит.
    - `Per person`: Сумма умножается на количество гостей.
    - `Per person per night`: Сумма умножается на количество ночей и количество гостей.
- **Amount type** Выберите, хотите ли вы, чтобы модификатор суммы был фиксированной суммой или процентом.
- **Status** Переключите тумблер, чтобы контролировать доступность акции во всех каналах продаж.

## Описания

:::note
Необходимо написать хотя бы одно описание, и оно должно быть на английском языке.
:::

Нажмите на вкладку `Descriptions`, чтобы продолжить.

Локализованное описание объясняет путешественнику на его языке, для чего предназначена эта акция.
Вы можете писать на любом количестве языков. Мы добавим переводы для всех самых популярных языков.
Это текст, который увидят путешественники при просмотре вашей акции.

- **Description** Дайте акции название. *например, Early bird - 20% OFF*

## Ограничения

:::note
Имейте в виду, что это ограничения... Включайте их только если хотите сделать акцию менее доступной.
:::

Нажмите на вкладку `Restrictions`, чтобы продолжить.

Движок акций Wink имеет не менее 20 уникальных способов запуска изменения цены.

Мы поддерживаем следующие критерии:

- Ограничить акцию определёнными дополнительными услугами.
- Ограничить акцию определёнными основными тарифами.
- Ограничить акцию определёнными тарифными планами.
- Ограничить акцию определёнными условиями отмены.
- Ограничить акцию по длительности проживания.
- Ограничить акцию по сроку предварительного бронирования.
- Ограничить акцию по количеству комнат.
- Ограничить акцию по промокоду.
- Ограничить акцию по диапазону IP путешественника.
- Ограничить акцию по последней минуте.
- Ограничить акцию по городу путешественника.
- Ограничить акцию по стране путешественника.
- Ограничить акцию по континенту путешественника.
- Ограничить акцию по часовому поясу путешественника.
- Ограничить акцию по дате продажи.
- Ограничить акцию по дате проживания.
- Ограничить акцию по дням недели бронирования.
- Ограничить акцию по дням недели прибытия.
- Ограничить акцию по дням недели отъезда.
- Ограничить акцию по обязательным дням недели.

Читайте дальше, чтобы узнать, как использовать каждое из них.

### Дополнительная услуга

:::note
Для использования этой функции необходимо создать хотя бы одну дополнительную услугу.
:::

Чтобы акция применялась только к выбранным вами дополнительным услугам, выполните следующие шаги:

<Steps>
1. Включите переключатель `Add-on restriction`.
2. Отметьте флажок рядом с каждой дополнительной услугой, которую хотите включить.
</Steps>

### Основной тариф

:::note
Для использования этой функции необходимо создать хотя бы один основной тариф.
:::

Чтобы акция применялась только к выбранным основным тарифам, выполните следующие шаги:

<Steps>
1. Включите переключатель `Master rate restriction`.
2. Отметьте флажок рядом с каждым основным тарифом, который хотите включить.
</Steps>

### Тарифный план

:::note
Для использования этой функции необходимо создать хотя бы один тарифный план.
:::

Чтобы акция применялась только к выбранным тарифным планам, выполните следующие шаги:

<Steps>
1. Включите переключатель `Rate plan restriction`.
2. Отметьте флажок рядом с каждым тарифным планом, который хотите включить.
</Steps>

### Возвратный

:::note
Для использования этой функции необходимо создать хотя бы одну политику отмены.
:::

Чтобы акция применялась только к выбранным политикам отмены, выполните следующие шаги:

<Steps>
1. Включите переключатель `Cancellation policy restriction`.
2. Отметьте флажок рядом с каждой политикой отмены, которую хотите включить.
</Steps>

### Длительность проживания

:::note
Это ограничение обычно используется для вашей акции «долгосрочное проживание».
:::

При добавлении этого ограничения вы требуете, чтобы гости `проживали не менее` и `не более` указанного количества дней.

Выполните следующие шаги, чтобы включить `Length of Stay`:

<Steps>
1. Включите переключатель `Length of stay restriction`.
2. Введите минимальное количество дней, которое гость должен прожить. *например, 7*
3. По желанию введите максимальное количество дней проживания. *например, 90*
</Steps>

*Пример: Эта акция доступна только гостям, которые проживают более 10 дней. Примечание: Максимальная длительность проживания необязательна.*

### Предварительное бронирование

:::note
Это ограничение обычно используется для вашей акции «раннее бронирование».
:::

При добавлении этого ограничения вы требуете, чтобы гости `бронировали не менее` и `не более` указанного количества дней заранее.

Выполните следующие шаги, чтобы включить `Advance booking`:

<Steps>
1. Включите переключатель `Advance booking restriction`.
2. Введите минимальное количество дней, за которое гость должен забронировать. *например, 10*
3. По желанию введите максимальное количество дней для предварительного бронирования. *например, 180*
</Steps>

*Пример: Эта акция доступна только гостям, которые бронируют более чем за 10 дней. Примечание: Мы также требуем указать максимальный срок предварительного бронирования.*

### Количество комнат

:::note
Акция доступна только если гость бронирует минимальное количество комнат.
:::

При добавлении этого ограничения вы требуете, чтобы гости `бронировали не менее` и `не более` указанного количества комнат в рамках бронирования.

Выполните следующие шаги, чтобы включить `Number of rooms`:

<Steps>
1. Включите переключатель `Room restriction`.
2. Введите минимальное количество комнат, которое гость должен забронировать. *например, 3*
3. Введите максимальное количество комнат, которое гость может забронировать. *например, 8*
</Steps>

*Пример: Эта акция доступна только гостям, которые бронируют более 3 комнат.*

### Промокод

:::note
Используйте это ограничение, если хотите, чтобы гость вводил код скидки для активации акции.
:::

Выполните следующие шаги, чтобы включить `Promo code`:

<Steps>
1. Включите переключатель `Promotion restriction`.
2. Нажмите кнопку `Add promo code`.
3. Введите промокод. *например, ABC123*
</Steps>

*Пример: Эта акция доступна только гостям, которые вводят промокод `ABC123`.*

### Диапазон IP

:::note
Это хорошее ограничение для проведения кросс-промоакций с мобильными операторами или крупными компаниями, у которых есть собственные диапазоны IP.
:::

Выполните следующие шаги, чтобы включить `IP range`:

<Steps>
1. Включите переключатель `IP range restriction`.
2. Нажмите кнопку `Add IP range restriction`.
3. Введите начальный [глобально доступный] диапазон IP. *например, 203.0.113.50*
4. Введите конечный [глобально доступный] диапазон IP. *например, 203.0.113.59*
</Steps>

*Пример: Эта акция доступна только гостям с IP-адресами в диапазоне от `203.0.113.50` до `203.0.113.59`.*

### Последняя минута

:::note
Это временной интервал для акции последней минуты, на который вы хотите наложить ограничение.
:::

Выполните следующие шаги, чтобы включить `Last minute`:

<Steps>
1. Включите переключатель `Last minute restriction`.
2. Нажмите кнопку `Add IP range restriction`.
3. Введите количество часов до даты прибытия. *например, 24 часа*
</Steps>

*Пример: Эта акция доступна только гостям, которые бронируют за 24 часа до даты прибытия.*

### Город

:::note
Позволяет ограничить акцию по городу, в котором в данный момент находится путешественник. Создайте ограничение для каждого города, который хотите включить.

3. Выберите город из результатов поиска. *например, New York City*
</Steps>

*Пример: Эта акция доступна только гостям, которые бронируют из `New York City`.*

### Страна

:::note
Позволяет ограничить акцию по стране, в которой в данный момент находится путешественник. Создайте ограничение для каждой страны, которую хотите включить.

Выполните следующие шаги, чтобы включить `Country`:

<Steps>

2. Начните вводить название страны.
3. Выберите страну из результатов поиска. *например, United States of America*
</Steps>

*Пример: Эта акция доступна только гостям, которые бронируют из `United States of America`.*

### Континент

:::note
Позволяет ограничить тариф по континенту, в котором в данный момент находится гость. Создайте ограничение для каждого континента, который хотите включить.

### Дата продажи

:::note
Позволяет ограничить акцию на определённый диапазон дат. Путешественник будет иметь право на акцию при бронировании номера в указанные даты.

4. Введите дату окончания. *например, 1 декабря 2024*

*Пример: Эта акция доступна только гостям, которые бронируют с `19 сентября 2024` по `1 декабря 2024`.*

## Даты блокировки

Нажмите на вкладку `Blackout dates`, чтобы продолжить.

Дополнительно контролируйте акцию, указав даты, когда эта акция не должна быть доступна.

Чтобы добавить дату блокировки, нажмите кнопку `Add blackout date`.

- **Start date** Выберите дату начала периода блокировки.
- **End date** Выберите дату окончания периода блокировки.

Если даты бронирования пересекаются с этим диапазоном, акция не будет применена.

Путешественник может одновременно иметь право на несколько акций, например долгосрочную + раннее бронирование.
Без пакетной акции путешественник имел бы право только на одну из них — ту, которая предлагает лучшую сделку.

Создавая пакетную акцию, вы учитываете право путешественника на обе акции и можете дополнительно её настроить.

Чтобы создать пакетную акцию, нажмите кнопку `Create bundled promotion`.

### Конфигурация

- **Name** Дайте пакетной акции название, которое видите только вы.
- **Status** Переключите тумблер, чтобы контролировать доступность пакета во всех каналах продаж.
- **Promotions** Выберите не менее 2 одиночных акций, чтобы сформировать пакет.

### Переопределение суммы

Вы можете выбрать переопределение общей применяемой скидки. Если оставить это выключенным, скидки каждой выбранной акции будут объединены в одну.

- **Override amount** Хотите ли вы агрегировать все скидки или переопределить своей суммой.
- **Type** Выберите, хотите ли вы увеличить или уменьшить цену. *например, выберите `Discount` для снижения цены.*
- **Pricing type** Выберите, как вы хотите рассчитывать этот пакет.
    - `Per stay`: Сумма применяется к последней ночи. *например, 100% скидка > Останьтесь 2 ночи — 3-я ночь бесплатно.*
    - `Per night`: Сумма умножается на количество ночей.
    - `Per use`: Дополнительных расчетов не происходит.
    - `Per person`: Сумма умножается на количество гостей.
    - `Per person per night`: Сумма умножается на количество ночей и количество гостей.
- **Amount type** Выберите, хотите ли вы, чтобы модификатор суммы был фиксированной суммой или процентом.
