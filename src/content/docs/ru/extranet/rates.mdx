Чтобы управлять тарифами, нажмите `Distribution > Master rate calendar` в главном навигационном меню.

В этой статье вы узнаете всё, что нужно знать о работе тарифов в Wink.

## Особенности

:::note
Wink управляет тарифами на уровне основного тарифа, а не на уровне гостевой комнаты. Это даёт вам дополнительную гибкость в распределении комнат.
:::

Тариф в Wink поддерживает следующие параметры:

- **Количество** Количество доступных единиц.
- **Сумма тарифа** Базовая цена.
- **Доступен** Доступен ли тариф на эту дату.
- **Закрыт на заезд (COA)** Можно ли заезжать в этот день.
- **Закрыт на выезд (COD)** Можно ли выезжать в этот день.
- **Минимальная продолжительность пребывания (Min LoS)** Есть ли требование к минимальному сроку пребывания.
- **Максимальная продолжительность пребывания (Max LoS)** Есть ли требование к максимальному сроку пребывания.
- **Минимальная заполняемость** Есть ли требование к минимальной заполняемости.
- **Максимальная заполняемость** Есть ли требование к максимальной заполняемости.

Нижеуказанные параметры доступны только через поставщика тарифов:

*Если ваш поставщик тарифов не поддерживает эти параметры, вы всё равно можете управлять ими через тарифные планы в Wink.*

- Модификатор тарифа для одного проживающего
- Модификатор тарифа для дополнительного взрослого
- Модификатор тарифа для дополнительного ребёнка

Тарифы — это основа Wink. Объекты могут добавлять тарифы вручную или подключать своего поставщика тарифов.

## Ручной ввод

:::caution
Создавайте или обновляйте тарифы вручную только если вы не продаёте свои типы комнат на других платформах, кроме Wink.
:::

:::caution
Если вы обновляете тарифы вручную при подключённом поставщике тарифов, тарифы изменятся снова, как только поставщик пришлёт обновление за тот же период.
:::

## Через поставщика тарифов

Чтобы начать отправлять тарифы через вашего поставщика, выполните следующие шаги:

- Сначала убедитесь, что мы поддерживаем вашего [поставщика тарифов](/ru/extranet/rate-providers).
- Установите вашего поставщика тарифов в разделе `Distribution > Channel manager`.
- Войдите в аккаунт вашего поставщика тарифов.
- Добавьте канал `Wink` в панель управления поставщика тарифов.
- Свяжите ваши типы комнат / тарифные планы с теми, что созданы в Wink.
- Инициируйте полную отправку тарифов в Wink [если это не сделано автоматически поставщиком].

Подождите около 3-5 минут, чтобы увидеть заполнение календарей основных тарифов Wink.

## Проверка

Вы можете протестировать настройку тарифов, перейдя в `Distribution > Verifier` в главном меню.

На этой странице можно смоделировать любой запрос цены, который может сделать путешественник.

:::tip
Если не можете найти нужную настройку, нажмите кнопку `Advanced search` внизу страницы.
:::

### Пример

Ниже пример маршрута по одному из ваших существующих каналов продаж:

- **Канал** Выберите канал продаж, через который хотите продавать. *например, Traveliko OTA*.
- **Валюта** Выберите валюту отображения. *например, USD*
- **Дата начала пребывания** Введите дату заезда. *например, 10 сентября 2024*
- **Ночи** Введите количество ночей. *например, 1*
- **Гости** Введите количество взрослых / детей. *например, 2 взрослых*
- Нажмите кнопку `Search` для продолжения.

Вы можете протестировать весь процесс бронирования, создав тестовое бронирование.
Тестовое бронирование помечается как `тестовое` и не требует подтверждения с помощью кредитной карты или другого способа оплаты.

Опции тестового бронирования включают:

- **Уведомить объект по электронной почте** Отправляет объекту подтверждение по e-mail.
- **Уведомить channel manager через API** Отправляет бронирование выбранному поставщику тарифов.
- **Уведомить бронирующего по электронной почте** Отправляет бронирующему подтверждение по e-mail.

По умолчанию бронирующим в этом сценарии являетесь вы, аутентифицированный пользователь. Вы также можете добавить данные бронирующего, включив переключатель `Add custom booker details` и заполнив форму.

*Если вы выбрали уведомление поставщика тарифов, он будет обрабатывать это бронирование как обычное и не узнает, что это тест. Рекомендуем тестировать с возвратными тарифами для удобной отмены после тестирования.*
