Vue 3 требует одного подсказки компилятору, чтобы обрабатывать теги `<wink-*>` как нативные кастомные элементы, а не как нераспознанные компоненты Vue. После этого они работают как любые другие HTML-элементы.

/>

## Установка

```bash
npm install @wink/elements
```

## Настройка компилятора

### Vite + Vue (`vite.config.ts`)

```ts
import { defineConfig } from 'vite';
import vue from '@vitejs/plugin-vue';

export default defineConfig({
  plugins: [
    vue({
      template: {
        compilerOptions: {
          // Обрабатывать все теги, начинающиеся с "wink-", как кастомные элементы
          isCustomElement: (tag) => tag.startsWith('wink-'),
        },
      },
    }),
  ],
});
```

### Vue CLI (`vue.config.js`)

```js
module.exports = {
  chainWebpack(config) {
    config.module
      .rule('vue')
      .use('vue-loader')
      .tap((options) => ({
        ...options,
        compilerOptions: {
          isCustomElement: (tag) => tag.startsWith('wink-'),
        },
      }));
  },
};
```

## Загрузка CDN один раз при запуске приложения

```ts
// src/main.ts
import { createApp } from 'vue';
import App from './App.vue';
import { load } from '@wink/elements';

load({ clientId: import.meta.env.VITE_WINK_CLIENT_ID });

createApp(App).mount('#app');
```

## Использование в шаблонах

```vue
<template>
  <main>
    <wink-content-loader layout="HOTEL" :id="layoutId" />
    <wink-lookup />
  </main>
</template>

<script setup lang="ts">
const layoutId = 'YOUR_LAYOUT_ID';
</script>
```

## TypeScript — расширение глобальных типов элементов

Vue 3 + TypeScript считывает типы кастомных элементов из глобального `HTMLElementTagNameMap`. Добавьте файл деклараций:

```ts
// src/wink-elements.d.ts
import type {
  WinkContentLoaderAttributes,
  WinkLookupAttributes,
  WinkSearchButtonAttributes,
  WinkAccountButtonAttributes,
  WinkItineraryButtonAttributes,
  WinkShoppingCartButtonAttributes,
  WinkAppLoaderAttributes,
} from '@wink/elements';

declare global {
  interface HTMLElementTagNameMap {
    'wink-content-loader': HTMLElement & WinkContentLoaderAttributes;
    'wink-lookup': HTMLElement & WinkLookupAttributes;
    'wink-search-button': HTMLElement & WinkSearchButtonAttributes;
    'wink-account-button': HTMLElement & WinkAccountButtonAttributes;
    'wink-itinerary-button': HTMLElement & WinkItineraryButtonAttributes;
    'wink-shopping-cart-button': HTMLElement & WinkShoppingCartButtonAttributes;
    'wink-app-loader': HTMLElement & WinkAppLoaderAttributes;
  }
}
```

## Переменные окружения

Добавьте в `.env.local`:

```
VITE_WINK_CLIENT_ID=your-client-id
```

Vite предоставляет клиентскому бандлу только переменные с префиксом `VITE_`.
