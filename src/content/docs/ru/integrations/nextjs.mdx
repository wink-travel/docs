Компоненты Wink — это кастомные элементы, работающие только в браузере — скрипты CDN должны загружаться на стороне клиента. Ниже описаны паттерны для **App Router** (Next.js 13+) и устаревшего **Pages Router**.

<LinkCard
  title="@wink/elements npm package"

/>

## Установка

```bash
npm install @wink/elements
```

## App Router (Next.js 13+)

Создайте клиентский компонент, который вызовет `load()` один раз, и отрендерьте его внутри корневого лейаута:

```tsx
// components/WinkLoader.tsx
'use client';
import { useEffect } from 'react';
import { load } from '@wink/elements';

export function WinkLoader() {
  useEffect(() => {
    load({ clientId: process.env.NEXT_PUBLIC_WINK_CLIENT_ID! });
  }, []);
  return null;
}
```

```tsx
// app/layout.tsx
import { WinkLoader } from '@/components/WinkLoader';

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body>
        <WinkLoader />
        {children}
      </body>
    </html>
  );
}
```

Используйте компоненты Wink в любом **Client Component**:

```tsx
// app/hotels/page.tsx
'use client';

export default function HotelsPage() {
  return (
    <main>
      <wink-content-loader layout="HOTEL" id="YOUR_LAYOUT_ID" />
    </main>
  );
}
```

<Aside type="caution" title="Server Components">
Теги `<wink-*>` не могут рендериться в Server Components, так как реестр кастомных элементов доступен только в браузере. Всегда добавляйте `'use client'` в любой компонент, который рендерит элементы Wink.
</Aside>

## Pages Router

Загрузите один раз в `_app.tsx`:

```tsx
// pages/_app.tsx
import type { AppProps } from 'next/app';
import { useEffect } from 'react';
import { load } from '@wink/elements';

export default function App({ Component, pageProps }: AppProps) {
  useEffect(() => {
    load({ clientId: process.env.NEXT_PUBLIC_WINK_CLIENT_ID! });
  }, []);
  return <Component {...pageProps} />;
}
```

Затем используйте компоненты Wink на любой странице:

```tsx
// pages/hotels.tsx
export default function HotelsPage() {
  return <wink-content-loader layout="HOTEL" id="YOUR_LAYOUT_ID" />;
}
```

## Переменные окружения

Добавьте в `.env.local`:

```
NEXT_PUBLIC_WINK_CLIENT_ID=your-client-id
```

Префикс `NEXT_PUBLIC_` обязателен, чтобы переменная была доступна в браузерном бандле.

## Типы TypeScript

Добавьте файл деклараций, чтобы TypeScript распознавал имена кастомных элементов в JSX — смотрите [страницу интеграции с React](/ru/integrations/react#typescript--declare-jsx-intrinsics) для полного примера `wink-elements.d.ts`.
