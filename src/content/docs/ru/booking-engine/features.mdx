Система бронирования представлена в двух вариантах, чтобы поддерживать разные потребности аффилиатов, управляющих компаний и отелей.

## Онлайн-турагентство

Для аффилиата наиболее логично использовать версию онлайн-турагентства. Причина в том, что сайт настроен для путешественников, желающих просмотреть несколько объектов размещения.

Ниже вы можете найти главную страницу этого приложения.

<LinkButton href="https://ota.wink.travel/home" variant="secondary" icon="external" target="_blank">
  Wink OTA
</LinkButton>

## Интернет-система бронирования

Для отеля или управляющей компании наиболее логично использовать версию интернет-системы бронирования. Такой пользователь заинтересован в выделении одного объекта или нескольких объектов под управлением.

Ниже вы можете найти главную страницу этого приложения.

## Конфигурация

В большинстве случаев нет необходимости напрямую делиться этими приложениями. Мы создали различные способы для обмена туристическим инвентарём, которые вы можете изучить в [Wink Studio](/studio/what-is-studio) или через [WinkLinks](/link-manager/wink-links).

Однако, если вы хотите вручную создать ссылку на один из этих сайтов, убедитесь, что вы добавили ваш client ID и customization ID в URL в виде параметров.

Вы можете найти ваш client ID следующим образом:

<Steps>
1. Войдите в [Wink Extranet](/extranet/what-is-extranet) или [Wink Studio](/studio/what-is-studio).
2. Нажмите на иконку профиля в правом верхнем углу и выберите [Applications](/developers/applications).
3. Используйте приложение, связанное с аккаунтом Wink, с которым вы хотите работать.
4. Скопируйте Client ID из этого приложения и вставьте его в URL как параметр запроса. Пример: ?client-id=MY_CLIENT_ID
</Steps>

### Extranet

Вы можете найти ваш customization ID как объект размещения следующим образом:

<Steps>
1. Войдите в [Wink Extranet](/extranet/what-is-extranet).
2. Выберите объект размещения, с которым хотите работать.
3. В главном меню нажмите `Account > Branding`
4. Скопируйте Customization ID и вставьте его в URL как параметр запроса. Пример: ?configuration-id=MY_CUSTOMIZATION_ID
</Steps>

### Studio

Вы можете найти ваш customization ID как аффилиат следующим образом:

<Steps>
1. Войдите в [Wink Studio](/studio/what-is-studio).
2. Выберите аккаунт, с которым хотите работать.
3. В главном меню нажмите `Tools > Customization`
4. Скопируйте Customization ID из нужной настройки и вставьте его в URL как параметр запроса. Пример: ?configuration-id=MY_CUSTOMIZATION_ID
</Steps>

## OTA

В итоге у вас получится URL, похожий на этот:

https://ota.wink.travel?client-id=MY_CLIENT_ID&configuration-id=MY_CUSTOMIZATION_ID

Если вы хотите перейти напрямую на страницу отеля, URL будет выглядеть примерно так:

https://ota.wink.travel/hotel/hotel-x?client-id=MY_CLIENT_ID&configuration-id=MY_CUSTOMIZATION_ID

:::note
Если вы не укажете client и/или configuration ID, Wink OTA будет использовать конфигурацию по умолчанию.
:::

## IBE

Если вы управляющая компания, вы хотите ссылаться на главную страницу Wink IBE. Главная страница покажет вашу назначенную тему вместе с [объектами, которые вы представляете](/studio/agency-representation).

В итоге у вас получится URL, похожий на этот:

https://book.wink.travel?client-id=MY_CLIENT_ID&configuration-id=MY_CUSTOMIZATION_ID

Если вы объект размещения, вы хотите ссылаться напрямую на страницу объекта Wink IBE.

Этот URL будет выглядеть примерно так:

https://book.wink.travel/hotel/hotel-x?client-id=MY_CLIENT_ID&configuration-id=MY_CUSTOMIZATION_ID


:::note
Если вы не укажете client и/или configuration ID, Wink IBE НЕ загрузится. Эти параметры обязательны.
:::

Целевая страница содержит всю информацию, необходимую путешественнику для обоснованного выбора, стоит ли останавливаться в этом объекте.
