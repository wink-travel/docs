Система бронирования представлена в двух вариантах, которые отвечают различным потребностям партнерских компаний, компаний по управлению недвижимостью и отелей.

## Онлайн туристическое агентство

Для партнера наиболее разумно использовать версию онлайн-турагентства. Причина в том, что сайт создан для путешественников, желающих просмотреть несколько объектов недвижимости.

Домашнюю страницу этого приложения вы можете найти ниже.

<LinkButton href="https://ota.wink.travel/home" variant="secondary" icon="external" target="_blank">
  Подмигивание ОТА
</LinkButton>


## Система интернет-бронирования

Для отеля или компании по управлению недвижимостью наиболее разумно использовать версию интернет-бронирования. Этот тип пользователей заинтересован в выделении одного объекта недвижимости или нескольких объектов недвижимости под управлением.

## Конфигурация

В большинстве случаев нет необходимости делиться ни одним из этих приложений напрямую. Мы создали множество способов для вас поделиться инвентарем для путешествий, которые вы можете найти в[Студия Винк](/studio/what-is-studio)или через[WinkLinks](/link-manager/wink-links).

Однако если вы хотите вручную создать ссылку на один из этих веб-сайтов, обязательно добавьте свой идентификатор клиента и идентификатор настройки в URL-адрес в качестве параметров.

Вы можете найти свой идентификатор клиента следующим образом:

<Steps>
  1. Войдите в систему [Wink Экстранет](/extranet/what-is-extranet) или [Студия Винк](/studio/what-is-studio).
  2. Нажмите на значок вашего профиля в правом верхнем углу и нажмите [Приложения](/developers/applications).
  3. Используйте приложение, привязанное к учетной записи Wink, с которой вы хотите работать.
  4. Скопируйте Client ID из этого приложения и вставьте его в URL как параметр запроса. Пример: ?client-id=MY\_CLIENT\_ID
</Steps>


### Экстранет

Вы можете найти свой идентификатор настройки как свойство:

<Steps>
  1. Войти в систему [Wink Экстранет](/extranet/what-is-extranet).
  2. Выберите объект недвижимости, с которым вы хотите работать.
  3. На главной панели навигации нажмите `Account > Branding`
  4. Скопируйте идентификатор настройки и вставьте его в URL как параметр запроса. Пример: ?configuration-id=MY\_CUSTOMIZATION\_ID
</Steps>


### Студия

Вы можете найти свой идентификатор настройки как партнера следующим образом:

<Steps>
  1. Войти в систему [Студия Винк](/studio/what-is-studio).
  2. Выберите учетную запись, с которой вы хотите работать.
  3. На главной панели навигации нажмите `Tools > Customization`
  4. Скопируйте идентификатор настройки из настройки, которую вы хотите использовать, и вставьте его в URL в качестве параметра запроса. Пример: ?configuration-id=MY\_CUSTOMIZATION\_ID
</Steps>


## ОТА

В итоге вы получите URL-адрес, который выглядит примерно так:

https://ota.wink.travel?client-id=MY\_CLIENT\_ID\&configuration-id=MY\_CUSTOMIZATION\_ID

Если вы хотите перейти непосредственно на страницу отеля, URL-адрес будет выглядеть примерно так:

https://ota.wink.travel/hotel/hotel-x?client-id=MY\_CLIENT\_ID\&configuration-id=MY\_CUSTOMIZATION\_ID

:::note
Если вы не укажете идентификатор клиента и/или конфигурации, Wink OTA по умолчанию будет использовать конфигурацию по умолчанию.
:::

## МБП

Если вы компания по управлению недвижимостью, вы хотите сделать ссылку на домашнюю страницу Wink IBE. Домашняя страница будет показывать вашу назначенную тему вместе с[свойства, которые вы представляете](/studio/agency-representation).

В итоге вы получите URL-адрес, который выглядит примерно так:

https://book.wink.travel?client-id=MY\_CLIENT\_ID\&configuration-id=MY\_CUSTOMIZATION\_ID

Если вы владелец недвижимости, вам необходимо разместить прямую ссылку на целевую страницу недвижимости Wink IBE.

Этот URL будет выглядеть примерно так:

https://book.wink.travel/hotel/hotel-x?client-id=MY\_CLIENT\_ID\&configuration-id=MY\_CUSTOMIZATION\_ID

:::note
Если вы не укажете идентификатор клиента и/или конфигурации, Wink IBE НЕ загрузится. Эти параметры являются обязательными.
:::

## Пример: Страница свойств

Каждый объект недвижимости на Wink получает свою собственную уникальную целевую страницу на любом из веб-сайтов. Все внешние ссылки направляют путешественников на одну из этих целевых страниц.

Целевая страница содержит всю информацию, необходимую путешественнику для принятия обоснованного решения об остановке в данном объекте размещения.
