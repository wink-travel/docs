## Обзор
Размещённый сервер Wink MCP доступен по адресу:

```
https://docs.mcp.wink.travel/mcp
```

Направьте совместимых AI-клиентов (VS Code, Cursor, других, реализующих MCP HTTP транспорт) на эту конечную точку, чтобы они могли:
- Просматривать документацию по продукту и платформе (`wink://docs/...`)
- Просматривать живые спецификации OpenAPI и документацию по операциям (`wink://api/...`)
- Выполнять лёгкий семантический / ключевой поиск по документации и живым API

## Транспорт
Используйте основной потоковый HTTP-эндпоинт:

```
https://docs.mcp.wink.travel/mcp
```

Если ваш клиент поддерживает только SSE, используйте:

```
https://docs.mcp.wink.travel/sse
```

### Claude Code
Официальная документация: [Подключение Claude Code к инструментам через MCP](https://code.claude.com/docs/en/mcp)

Anthropic рекомендует использовать удалённый HTTP транспорт для облачных MCP серверов:
```bash
claude mcp add --transport http wink-docs https://docs.mcp.wink.travel/mcp
claude mcp list
```

В Claude Code выполните `/mcp` для проверки статуса сервера. Если хотите поделиться сервером со всем репозиторием, добавьте `--scope project`, чтобы Claude Code записал конфигурацию в `.mcp.json`.

### Codex
Официальная документация: [Быстрый старт Docs MCP для Codex](https://developers.openai.com/resources/docs-mcp)

OpenAI описывает настройку MCP через Codex CLI и отмечает, что конфигурация используется также расширением IDE:
```bash
codex mcp add wink-docs --url https://docs.mcp.wink.travel/mcp
codex mcp list
```

### Cursor
Документация Cursor MCP: https://cursor.com/docs/context/mcp
Настройка: Добавьте удалённый сервер через настройки MCP.
```json
{
  "mcpServers": {
    "wink-docs": {
      "url": "https://docs.mcp.wink.travel/mcp",
    }
  }
}
```

### VS Code
Официальная документация: https://code.visualstudio.com/docs/copilot/customization/mcp-servers
Минимальная удалённая конфигурация (пользовательский или рабочий `mcp.json`):
```json
{
  "servers": {
    "wink-docs": {
      "type": "http",
      "url": "https://docs.mcp.wink.travel/mcp"
    }
  }
}
```

Советы:
- Используйте Command Palette: «MCP: List Servers» для подтверждения подключения и просмотра инструментов.
- В селекторе инструментов можно включать/отключать `api_search` и `docs_search`.
- Добавляйте контекст: Вид чата → Добавить контекст → MCP Resources для прикрепления страниц документации.

## Основные URI ресурсов

| URI | Описание |
|-----|----------|
| `wink://docs` | Индекс документации (сгруппирован по категориям) |
| `wink://docs/{path}` | Конкретная статья документации (Markdown / MDX) |
| `wink://api` | Живой каталог OpenAPI, обнаруженный из конфигурации swagger Wink |
| `wink://api/{apiName}` | Краткое описание API с информацией об аутентификации, тегах, заголовках и примерах операций |
| `wink://api/{apiName}/operations` | Полный список операций, сгруппированных по тегам |
| `wink://api/{apiName}/operation/{key}` | Детали отдельной операции (параметры, тело запроса, ответы) |
| `wink://api/{apiName}/schema/{schemaName}` | Разрешённая схема компонента с вложенными полями, типами и примерами |
| `wink://api/{apiName}/operation/{key}/request` | Вид, ориентированный на запрос, с обязательными входными данными и минимальным примером полезной нагрузки |
| `wink://api/{apiName}/operation/{key}/response/{status}` | Вид, ориентированный на ответ, для конкретного кода статуса |
| `wink://api/{apiName}/operation/{key}/recipe` | Ориентированный на задачу поток для правильного вызова этой операции |

## Инструменты поиска (Краткое руководство)
| Инструмент | Назначение | Типичные примеры запросов |
|------------|------------|--------------------------|
| `api_search` | Найти живое API или операцию для интеграции | `create affiliate`, `payment receipt`, `lookup inventory`, `invite manager` |
| `docs_search` | Найти концептуальную документацию / руководства | `authentication`, `webhooks`, `wordPress` |

Оставьте `query` пустым, чтобы получить предложения. Каждый результат включает MCP URI, который можно открыть.
Для совпадений по операциям `api_search` также возвращает связанные URI запросов, ответов, схем и рецептов, чтобы агенты могли перейти напрямую к наиболее полезным подресурсам.

## Типичный сценарий использования
1. Выполните `api_search` с вашим намерением (например, *"create affiliate"* или *"payment receipt"*).
2. Откройте возвращённый URI `wink://api/...` или перейдите по связанным полям `request_uri`, `response_uris`, `request_schema_uris`, `response_schema_uris` и `recipe_uri`.
3. Используйте `docs_search` для поиска вспомогательных концепций (аутентификация, вебхуки, рабочие процессы).

## Примечания
- Каждый возвращённый URI может быть открыт агентом для полного просмотра содержимого.
- Дерево `wink://api` живое и следует текущему каталогу OpenAPI, а не сгенерированному индексу репозитория.
- Предложения появляются автоматически, когда запросы пусты.
