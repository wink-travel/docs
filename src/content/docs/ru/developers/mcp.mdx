## Обзор
Размещённый сервер Wink MCP доступен по адресу:

```
https://docs.mcp.wink.travel/mcp
```

Направьте совместимых AI-клиентов (VS Code, Cursor, другие, реализующие MCP HTTP транспорт) на эту конечную точку, чтобы они могли:
- Просматривать документацию по продукту и платформе (`wink://docs/...`)
- Открывать SDK и изучать классы API / модели (`wink://sdk/...` и `wink://sdk/.../docs`)
- Выполнять лёгкий семантический / ключевой поиск по документации и сводкам SDK

## Транспорт
Используйте основной потоковый HTTP-эндпоинт:

```
https://docs.mcp.wink.travel/mcp
```

Если ваш клиент поддерживает только SSE, используйте:

```
https://docs.mcp.wink.travel/sse
```

### Cursor
Документация Cursor MCP: https://cursor.com/docs/context/mcp  
Настройка: добавьте удалённый сервер через настройки MCP.
```json
{
  "mcpServers": {
    "wink-docs": {
      "url": "https://docs.mcp.wink.travel/mcp",
    }
  }
}
```

### VS Code
Официальная документация: https://code.visualstudio.com/docs/copilot/customization/mcp-servers  
Минимальная удалённая конфигурация (пользовательский или рабочий `mcp.json`):
```json
{
  "servers": {
    "wink-docs": {
      "type": "http",
      "url": "https://docs.mcp.wink.travel/mcp"
    }
  }
}
```

Советы:
- Используйте Command Palette: «MCP: List Servers» для подтверждения подключения и просмотра инструментов.
- Выбор инструментов позволяет включать/отключать `sdk_search` и `docs_search`.
- Добавляйте контекст: Вид чата → Add Context → MCP Resources для прикрепления страниц документации.

## Основные URI ресурсов

| URI | Описание |
|-----|----------|
| `wink://docs` | Индекс документации (сгруппирован по категориям) |
| `wink://docs/{path}` | Конкретная статья документации (Markdown / MDX) |
| `wink://sdk` | Обзор SDK (каждый SDK + документация) |
| `wink://sdk/{sdkPath}` | README SDK / папка с документацией модуля / конкретная документация модуля |

### Документация модулей
Открытие папки с документацией SDK (например, `wink://sdk/python/wink-sdk-booking/docs`) возвращает сгруппированное:
- **Классы API** (количество методов + примеры имён методов)
- **Модели и типы** (структуры данных, используемые этими API)

## Инструменты поиска (быстрая справка)
| Инструмент | Назначение | Типичные примеры запросов |
|------------|------------|---------------------------|
| `sdk_search` | Найти, какой SDK / класс API использовать | `cancel booking`, `list properties`, `affiliate inventory` |
| `docs_search` | Найти концептуальную документацию / руководства | `authentication`, `webhooks`, `wordPress` |

Оставьте `query` пустым, чтобы получить предложения. Каждый результат включает MCP URI, который можно открыть.

## Типичный сценарий использования
1. Запустите `sdk_search` с вашим запросом (например, *"list properties"*).
2. Откройте возвращённый README SDK или его `docs_uri` для изучения API / моделей.
3. Используйте `docs_search` для поиска вспомогательных концепций (аутентификация, вебхуки).

## Примечания
- Каждый возвращённый URI может быть открыт агентом для полного просмотра содержимого.
- Предложения появляются автоматически, если запросы пусты.
