Настройка — это мощная функция, которую вы можете использовать для управления каждым аспектом интеграции с Wink без необходимости писать код. Каждый элемент инвентаря, который вы продаёте, можно настроить. Добавляйте или удаляйте функции, меняйте цветовую схему и многое другое.

На странице `Customization` нажмите кнопку `Actions` под нужной настройкой и выберите `Update`.
*При создании аккаунта у вас уже есть доступная запись по умолчанию. Эту запись нельзя удалить.*

Это приведёт вас к форме настройки, где вы увидите 5–7 вкладок (*в зависимости от типа вашего аккаунта*).

## Общие

Вот доступные вам опции на вкладке `General`.

- **Name** Дайте вашей настройке имя, чтобы легко её запомнить. *например, Для моего туристического блога*
- **Display currency** Выберите начальную валюту, которая будет отображаться рядом с доступным для бронирования туристическим инвентарём. *например, USD*
- **Display language** Выберите начальный язык, с которым столкнутся пользователи. *например, English*
- **Append to page title** Вы можете контролировать, что добавляется к заголовку страницы (*заголовок, который вы видите на вкладке браузера*). Это в основном используется для нашего booking engine. *например, если вы перейдёте на https://ota.wink.travel, по умолчанию вы увидите `Home | Traveliko - Hotels with Benefits`*. Вы можете изменить всё после символа '|'.

## Тема

Вкладка `Theme` позволяет настроить внешний вид нашего [booking engine](/ru/booking-engine/in-a-nutshell), сайта [WinkLinks](/ru/link-manager/wink-links) и всех наших [Web Components](/ru/developers/web-components).

### Конфигурация

Этот раздел позволяет настроить дизайн и расположение карточек. На данный момент доступен только один дизайн — Wink.

Расположение карточек контролирует, хотите ли вы, чтобы пользователи видели транзакционные карточки горизонтально или вертикально. Информация при этом остаётся одинаковой.

:::note
При просмотре транзакционных карточек с мобильного устройства различий в расположении нет. Однако разница видна на iPad mini и устройствах с большим разрешением.
:::

:::tip[Полезный совет]
Используйте горизонтальное расположение, если у вас в списке менее 4 единиц туристического инвентаря (например, типов номеров)... так выглядит лучше.
:::

### Цвета

## Аналитика

Мы поддерживаем GA4 аналитику из коробки. Если вы хотите отслеживать весь путь конверсии расширенной электронной коммерции от начала до конца, подготовьте ваш идентификатор измерения и введите его в поле на вкладке `Analytics`.

<Aside type="caution" title="Напоминание">
  Не забудьте добавить `i.trvl.as`, `book.wink.travel` и `ota.wink.travel` в потоки вашей GA4-свойства.
</Aside>

## Маршрут

Вкладка `Itinerary` вероятно самая интересная с точки зрения отображения цен для выбранного инвентаря.
Поскольку ценообразование туристического инвентаря достаточно сложное, Wink должен знать маршрут пользователя, чтобы корректно показать цену.

Этот раздел позволяет задать маршрут по умолчанию для отображения цен на всех наших цифровых площадках, таких как:

- Наши социально-ориентированные изображения с наложением цены.
- Сетки, карточки и ссылки WinkLinks.
- Web Components
- Booking engine

### Даты

Вы можете выбрать, хотите ли вы установить статический или скользящий диапазон дат.

- Выберите статический диапазон, если настройка используется для конкретного события, которое происходит в определённую дату.
- Выберите скользящий диапазон, если хотите:
    1. По умолчанию использовать сегодня + 1 ночь, как делают большинство OTA, ИЛИ
    2. По умолчанию использовать дату в будущем, которая всегда должна иметь доступность.

### Конфигурация комнат

По умолчанию маршрут установлен на 2 взрослых, так как это наиболее распространённая настройка для большинства путешественников.
Это легко изменить, добавив несколько комнат с разным количеством взрослых и детей.

### Акции

Акции — удобная функция, которая экономит время вашим пользователям. Если объект предоставил вам промокод, которым вы можете поделиться с друзьями, вы можете ввести его здесь.
Любой, кто перейдёт по вашей ссылке или аналогичной, автоматически получит промокод в своём маршруте и увидит применённую скидку.

## Интеграция

Вкладка `Integration` предназначена для отелей, интеграторов и туристических агентов, которые хотят иметь больший контроль над распределением бронирований на Wink.

### Уведомления

Это даёт отелю и интегратору возможность позволить Wink уведомлять все заинтересованные стороны при бронировании.
Включая уведомление менеджера каналов.

По умолчанию все эти опции включены. Случаи, когда интегратор может захотеть отключить одну или несколько функций:

- Туристический агент интегрирован на уровне API и хочет отправлять собственные подтверждающие письма.
- Отель тестирует интеграцию и не хочет беспокоить свой отдел продаж.
- Отелю не важны электронные письма, и он хочет получать уведомления только через PMS.

### Логотип в письмах

Для интеграторов, которые хотят использовать нашу службу уведомлений по электронной почте, можно настроить письмо так, чтобы оно выглядело, как отправленное от них.

:::note
  Рекомендуемые размеры: 300px в ширину и 75px в высоту
:::

## Специфично для отеля

На вкладке `Hotel-specific` находятся опции, позволяющие отелю контролировать, что отображается на их целевой странице в нашем booking engine и что показывать, если для запрошенного маршрута нет доступного инвентаря.

- **Rankings**: Объект, который ещё не получил значимых рейтингов, может отключить эту функцию.

:::note
  Объект не может контролировать, как аффилиат отображает целевую страницу отеля... Это две разные настройки.
:::

- **Availability**: Позволяет объекту выбрать, продолжать ли отображать инвентарь, даже если нет доступности.

:::note
  В большинстве случаев объект захочет дать пользователям возможность нажать кнопку `Book` и перейти на их целевую страницу, где можно изменить маршрут и попытаться найти доступную дату.
  Включите этот переключатель, если хотите заблокировать пользователя от продолжения.
:::

## Как использовать

В большинстве случаев вы выбираете, какую `Customization` применять к вашим ссылкам, карточкам, картам и сеткам.

Вот случаи, когда вы можете захотеть применить конфигурацию вручную:

- **Ручное переопределение** Вы хотите поделиться ссылкой на наш booking engine напрямую, не используя функцию [Shareable Links](/ru/studio/shareable-links), с применённой вашей пользовательской конфигурацией.
- **Плагин WordPress** Добавьте конфигурацию в настройки вашего Wink [плагина для WordPress](/ru/developers/wordpress).
- **Пользовательская интеграция** Примените её вручную к вашему интегратору с помощью нашего Web Component [wink-app-loader](/ru/developers/web-components/#application-loader).

### Ручное переопределение

Вот простой пример, как применить ваш ID конфигурации со всеми настройками к основным URL нашего booking engine.

<h4 align="center"> [https://ota.wink.travel?configurationId=MY-UNIQUE-ID](https://ota.wink.travel?configurationId=MY-UNIQUE-ID) </h4>

:::note
  При переходе по этой ссылке также будет установлен ваш cookie для аффилиатской атрибуции.
:::

### Плагин WordPress

Перейдите к [плагину Wink для WordPress](/ru/developers/wordpress), чтобы узнать больше.

### Пользовательская интеграция

Управляйте всеми встроенными Wink Web Components на странице с помощью Web Component [wink-app-loader](/ru/developers/web-components/#application-loader).

Ниже пример с использованием `MY-UNIQUE-ID` в качестве ID конфигурации.

```html {11}
<html>
  <head>
    <link rel="stylesheet" href="https://elements.wink.travel/styles.css"></link>
  </head>
  <body>

    <script src="https://elements.wink.travel/elements.js" type="module" defer></script>

    <wink-app-loader
      client-id="YOUR CLIENT ID GOES HERE"
      configuration-id="MY-UNIQUE-ID"
    ></wink-app-loader>
  </body>
</html>
```
