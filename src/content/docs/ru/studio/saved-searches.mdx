Сохраненные поиски делают то, что подразумевает их название. При поиске туристического инвентаря у вас есть возможность сохранить свой поисковый запрос для дальнейшего использования.

## Сохранение вашего поискового запроса

Когда вы начинаете поиск инвентаря (см.[Поиск](/studio/search)), у вас есть возможность тонкой настройки фильтров поиска.
В нижней части формы запроса есть возможность`Save search`.

<figure>
  <CloudinaryImage width="350" height="329" resize="scale" quality="best" alt="Сохранить поиск" src="academy/save-search" errorImage="noimage_opaque_nyrtl0.png" />

  <figcaption>*Показывает ссылку для сохранения поискового запроса*</figcaption>
</figure>


Нажав на эту ссылку, вы сможете дать этому поиску имя (*например, Моя библиотека волшебных отелей*) что-то, что вы запомните на будущее.

<figure>
  <CloudinaryImage width="350" height="329" resize="scale" quality="best" alt="Дайте вашему поиску имя" src="academy/name-save-search" errorImage="noimage_opaque_nyrtl0.png" />

  <figcaption>*Дайте вашему поиску имя*</figcaption>
</figure>


Дайте ему имя и нажмите на`Save search`кнопку. После этого вы можете перейти к`Inventory > Saves Searches`из главной навигационной панели, чтобы увидеть вашу вновь созданную запись.

## Список действий

Сохраненный поиск можно использовать для предоставления пользователям туристических предложений в виде сетки или карты, а также для отображения результатов поиска в удобном для вас виде.

<figure>
  <CloudinaryImage width="700" height="939" resize="scale" quality="best" alt="Сохраненные действия поиска" src="academy/saved-search-actions" errorImage="noimage_opaque_nyrtl0.png" />

  <figcaption>*Сохраненный поисковый запрос без применения фильтров*</figcaption>
</figure>


Для каждого сохраненного поискового запроса есть действия, которые вы можете выполнить:

* Преобразовать поисковый запрос в[карта](/studio/maps).
* Преобразовать поисковый запрос в[сетка](/studio/grids).
* Удалить сохраненный поиск.

## API

Разработчики, желающие управлять сохраненными поисковыми запросами, могут перейти по ссылке[Разработчики > API > Обзор](/developers/apis/#browse-api).

## Дальнейшее чтение
