Если вы интегратор, например Wink, и хотите сопоставить своих внешних получателей для использования с TripPay, вы можете сделать это одним из двух способов:

- Ручное сопоставление
- Использование [Payment API](/ru/developers/apis/#payment-api) (*См. `Account Mappings`*)

:::tip
Если вы работаете не только с несколькими одинаковыми объектами, мы рекомендуем использовать API для автоматизации этого шага.
:::

Ниже мы пошагово покажем вам, как вручную сопоставить внешнего получателя. *Под капотом используется API.*

<Steps>
1. Войдите в [TripPay](https://pay.wink.travel).
2. Выберите свой аккаунт интегратора.
3. Нажмите `Settings > Mappings` в главном навигационном меню.
4. Нажмите кнопку `Create mapping`. Появится форма сопоставления.
5. **Local account** Начните вводить название существующего аккаунта TripPay, который хотите сопоставить.
6. **External identifier** Вставьте ваш внешний ID для этого объекта, как он существует в удалённой системе.
7. Нажмите `Save`, чтобы продолжить. Вы будете перенаправлены обратно к списку сопоставлений.
</Steps>

С установленными сопоставлениями TripPay теперь знает, какие аккаунты кредитовать как бенефициаров бронирования, когда вы ссылаетесь на них, используя ваши внешние идентификаторы.

## API

Разработчики, которые хотят управлять `Payments`, переходите в [Developers > API > Payments](/ru/developers/apis/#payment-api).
