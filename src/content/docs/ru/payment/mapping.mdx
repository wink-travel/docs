Если вы являетесь интегратором, как Wink, и хотите сопоставить внешних получателей платежей для использования с TripPay, вы можете сделать это одним из двух способов:

* Ручное картирование
* Используя[API-интерфейс для платежей](/developers/apis/#payment-api)(*Видеть `Account Mappings`*)

:::tip
Если вы не имеете дело только с несколькими одинаковыми свойствами, мы рекомендуем вам использовать API для автоматизации этого шага.
:::

Ниже мы пошагово расскажем вам, как вручную сопоставить внешнего получателя платежа.*Он использует API-интерфейс изнутри.*

<Steps>
  1. Войти в систему [TripPay](https://pay.wink.travel).
  2. Выберите учетную запись интегратора.
  3. Нажмите `Settings > Mappings` из главной навигационной панели.
  4. Нажмите на `Create mapping` Кнопка. Вы увидите форму сопоставления.
  5. **Локальный счет** Начните вводить название существующей карты TripPay, которую вы хотите сопоставить.
  6. **Внешний идентификатор** Вставьте внешний идентификатор для этой сущности, который существует в удаленной системе.
  7. Нажмите `Save` для продолжения. Вы будете перенаправлены обратно к списку сопоставлений.
</Steps>


После настройки сопоставлений TripPay теперь знает, какие счета следует зачислять в качестве бенефициаров бронирования, когда вы ссылаетесь на них с помощью внешних идентификаторов.

## API

Разработчики, которые хотят управлять`Payments`, перейти к[Разработчики > API > Платежи](/developers/apis/#payment-api).
