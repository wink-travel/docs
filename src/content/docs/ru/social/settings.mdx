# Настройки

Когда вы впервые начинаете работать с Social, мы спросим, как вы хотите, чтобы мы создавали ваш контент. Эти параметры доступны вам в любое время, чтобы вы могли легко изменить их позже.

Перейдите в `Preferences` в боковом меню, чтобы управлять своими настройками.

Раздел предпочтений содержит основные настройки генерации контента:

1. **Предпочитаемая кастомизация** Выберите, какую кастомизацию использовать при создании контента. Кастомизации управляют цветами темы, стандартным маршрутом, валютой, языком и многим другим. Вы создаёте кастомизации в своей учетной записи на [my.wink.travel](https://my.wink.travel).
2. **Автоматическое одобрение** При включении сгенерированные посты публикуются автоматически без вашего одобрения. Включайте эту опцию, когда будете уверены в качестве создаваемого нами контента.
3. **Имя утверждающего** Имя человека или команды, которые будут получать письма с запросами на одобрение.
4. **E-mail утверждающего** Адрес электронной почты, на который будут отправляться запросы на одобрение предстоящих постов.

## Создатели контента

  <figcaption>*Рис. 2 - Создатели контента*</figcaption>

В Social доступно множество создателей контента на выбор. Каждый создатель фокусируется на одном конкретном способе создания уникального контента для вашего объекта с помощью ИИ. У каждого создателя есть атрибут **Weight** — более высокий вес увеличивает вероятность выбора этого создателя при генерации постов.

Вот все доступные создатели контента:

| Создатель | Описание |
| -- | -- |
| **Destination** | Использует ваше местоположение для создания полезного текста об окрестностях |
| **Room type** | Создаёт бронируемый пост с одним из ваших типов номеров с текстом, сгенерированным ИИ, и изображениями номера |
| **Activity** | Создаёт бронируемый пост с одной из ваших активностей |
| **Add-On** | Создаёт бронируемый пост с одним из ваших дополнительных сервисов |
| **Attraction** | Создаёт бронируемый пост с одной из ваших достопримечательностей |
| **Place** | Создаёт бронируемый пост с одним из ваших мест |
| **Meeting room** | Создаёт бронируемый пост с одной из ваших переговорных комнат |
| **Restaurant** | Создаёт бронируемый пост с одним из ваших ресторанов |
| **Spa** | Создаёт бронируемый пост с одним из ваших спа |
| **Property** | Создаёт бронируемый пост с вашим лучшим по цене номером с текстом, сгенерированным ИИ, и изображениями объекта |
| **User sentiment** | Собирает отзывы гостей из интернета в пост, основанный на настроениях, который подчёркивает, что гости любят в вашем объекте |
| **Funny** | Создаёт лёгкие, развлекательные посты, чтобы поддерживать интерес к вашей ленте |

:::tip[Полезный совет]
Убедитесь, что у вас есть нужный тип инвентаря для выбранных создателей контента. Например, если вы выбрали «Restaurant», у вас должен быть хотя бы один ресторан, указанный в вашем объекте.
:::

## Тональность

Тональность задаёт стиль языка, который ИИ использует при создании контента. Выберите тональности, соответствующие личности вашего бренда. Как и у создателей контента, у каждого тона есть **Weight**, чтобы контролировать частоту его использования.

Доступные тональности:

| Тон | Стиль |
| -- | -- |
| **Neutral** | Спокойный, авторитетный, фактический. Без преувеличений и шуток — просто надёжные, хорошо структурированные знания |
| **Friendly** | Тёплый, дружелюбный, с энергией улыбки в тексте, с повседневными сокращениями и иногда эмодзи |
| **Luxury** | Изысканный, утончённый. Слова вроде «изысканный» и «индивидуальный», создающие ощущение эксклюзивности |
| **Foodie** | Чувственные, аппетитные описания. Язык «фуд-порно», вызывающий аппетит |
| **Adventurous** | Смелый, отважный, с энергией исследователя. Яркие сенсорные описания, делающие всё эпичным |
| **Family** | Добрый, подходящий для детей восторг. Тёплая родительская энергия о безопасных и весёлых впечатлениях |
| **Romantic** | Поэтичный, мечтательный, трепетный. Закаты, мягкий язык и любовь |
| **Humorous** | Саркастичный, быстрые шутки, остроумная игра слов. Читается как стендап |
| **Eco** | Искренний и праведный. Пропагандирует зелёные альтернативы и устойчивый выбор |
| **Local insider** | Очень конкретные местные ссылки, предупреждения о туристических ловушках, использование местного сленга |
| **Inclusive** | Приветливый, праздничный и поддерживающий. Заставляет каждого чувствовать себя замеченным и ценным |
| **Mystical** | Эфирный, духовный, космические истины. Мягкие приглашения заглянуть внутрь себя |
| **Futuristic** | Стильный, ориентированный в будущее. Любит термины вроде «парадигмальный сдвиг» и оптимистичные видения будущего |
| **Budget** | Хитрый, находчивый, гордо экономный. Пропагандирует доступные находки и бесплатные впечатления |

## Типы постов

Типы постов определяют цель и структуру создаваемого контента. У каждого типа поста есть **Weight**, чтобы контролировать частоту его использования.

| Тип поста | Описание |
| -- | -- |
| **Transactional** | Чёткий, лаконичный, ориентированный на конверсию. Стимулирует бронирования и прямые действия с фактами, ценами и следующими шагами |
| **Awareness** | Контент для построения бренда, предназначенный для знакомства с вашим объектом и увеличения видимости среди потенциальных гостей |
| **Sentiment** | Эмоционально резонансный контент, создающий положительные ассоциации и строящий эмоциональную связь с аудиторией |

Возможно, вы захотите сосредоточить маркетинговые усилия на определённом языке и регионе. Укажите языки, для которых хотите создавать контент, и интересующий вас часовой пояс — мы сделаем остальное. Посты публикуются в запланированное время в выбранном часовом поясе.

*Большинство объектов используют один язык и по умолчанию выбирают свой местный язык.*

Подключив ваш профиль Google Business, мы сможем получить более полные данные об объекте для более эффективного контента. Ваш объект должен иметь подтверждённый профиль Google Business, чтобы мы могли генерировать контент на основе вашего листинга. Некоторые наши создатели контента **требуют** наличия аккаунта Google Business.

:::tip[Полезный совет]
Начинайте с малого

Не будьте слишком амбициозны при первой попытке. Рекомендуем начать с создания контента для вашего объекта на одном языке.
:::
