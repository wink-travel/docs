Не каждый день имеет одинаковую цену. События, праздники, межсезонье и последние свободные даты требуют разных тарифов. В этом руководстве показано, как использовать специальные тарифы и ограничения тарифных планов для решения таких ситуаций.

## Ценообразование на мероприятия

Когда конференция, фестиваль или местное событие увеличивает спрос, вы захотите повысить тарифы на эти даты.

<Steps>
1. Перейдите в `Monetize > Special Rates` в вашем Extranet.
2. Нажмите `Create a new special rate`.
3. Выберите основной тариф и тип номера для переопределения.
4. Установите диапазон дат, охватывающий период мероприятия.
5. Введите скорректированные (повышенные) дневные цены.
6. Нажмите `Save`.
</Steps>

Специальный тариф переопределяет основной тариф только на эти даты. Ваши обычные цены остаются без изменений вне периода мероприятия.

## Сезонные корректировки

Для более длительных периодов (высокий сезон, переходный сезон, низкий сезон) вы можете:

- **Использовать специальные тарифы** — Создавать переопределения для каждого сезона с изменёнными ценами.
- **Обновлять основные тарифы** — Напрямую корректировать базовые тарифы для этих периодов.

Специальные тарифы лучше подходят для временных корректировок, а обновление основных тарифов имеет смысл для предсказуемых ежегодных изменений.

## Тарифы в последний момент

Чтобы заполнить непроданные номера ближе к дате заезда:

<Steps>
1. Создайте новый **тарифный план** с ограничением по предварительному бронированию (например, бронирование не более чем за 3 дня).
2. Свяжите его со **специальным тарифом**, предлагающим скидку на целевые даты.
3. Этот тарифный план будет виден только путешественникам, бронирующим в последний момент.
</Steps>

## Даты блокировки

Чтобы заблокировать определённые даты для конкретного тарифного плана (например, без промо-тарифов в высокий сезон):

<Steps>
1. Перейдите в `Monetize > Rate Plans` и выберите тарифный план.
2. Перейдите в раздел **Restrictions**.
3. Добавьте ограничение **Sell date**, чтобы ограничить периоды, когда тариф доступен для бронирования.
4. Или используйте ограничения **Stay date**, чтобы ограничить периоды проживания по тарифу.
</Steps>

Например, вы можете ограничить тарифный план "Early Bird", чтобы он не был доступен для бронирований в декабре (высокий сезон), в то время как ваш обычный тарифный план останется доступен.

## Комбинирование ограничений

Ограничения можно комбинировать для точного контроля:

| Ограничение | Что контролирует |
| -- | -- |
| **Sell dates** | Когда тариф можно забронировать |
| **Stay dates** | Какие даты проживания покрывает тариф |
| **Available days** | В какие дни недели тариф отображается |
| **Arrival days** | В какие дни гость должен заезжать |
| **Departure days** | В какие дни гость должен выезжать |
| **Required days** | Гость должен проживать хотя бы один из этих дней |
| **Advance booking** | За сколько времени до заезда должно быть сделано бронирование |
| **Length of stay** | Минимальное и максимальное количество ночей |
