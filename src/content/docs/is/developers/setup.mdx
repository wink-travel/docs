Þessi grein sýnir þér hvernig þú getur fljótt sett þig upp sem forritari og byrjað að prófa API-in okkar á Wink.

:::note
Prófanir ættu alltaf að fara fram í okkar [staging environment](/is/getting-started/environments). Þar er mun minna magn af gögnum en þú getur prófað eins mikið og þú vilt án þess að hafa áhrif á framleiðsluumhverfið.

Þegar þú ert tilbúinn að fara í loftið munum við votta reikninginn þinn og umsóknina þína svo þú getir hafið notkun í framleiðsluumhverfinu.
:::

Fylgdu þessum skrefum:

<Steps>
1. [Skráðu þig](/is/my-account/register) með notendareikningnum þínum á Wink.
2. [Búðu til tengdan reikning](/is/my-account/create-account). Þú þarft þennan til að tengja Umsóknina þína við. Sjáðu [hvað er Studio](/is/studio/what-is-studio) fyrir frekari upplýsingar.
3. [Búðu til Umsókn](/is/my-account/applications) og tengdu hana við tengda reikninginn sem þú bjóst til í skrefi 2. Fyrir server-to-server API samþættingar, veldu **MACHINE_2_MACHINE** sem viðskiptavinatýpu.
4. Þegar þú hefur búið til `Application`, afritaðu og vistaðu client-id og leynilykilinn strax — leynilykillinn er sýndur aðeins einu sinni og er ekki hægt að nálgast aftur.
</Steps>

## CDN

Hjá Wink notum við Cloudinary fyrir öll mynd- og myndbandsgögn okkar. Þegar þú nálgast ferðatöflu okkar færðu fullan aðgang að öllum háþróuðum eiginleikum Cloudinary.

`cloud_name` okkar er `traveliko`.

### Cloudinary

- [Byrjaðu hér](https://cloudinary.com/documentation/dev_kickstart)
- [SDKs](https://cloudinary.com/documentation/cloudinary_sdks)
- [Cloudinary á GitHub](https://github.com/cloudinary)
-	[Myndbandskennsluefni](https://cloudinary.com/documentation/cloudinary_sdks_tutorials)

### Dæmi

Fyrir hvern ferðatöflu-flokk sem við styðjum verður til staðar innfelld JSON fylki sem kallast `multimedias`. JSON fyrir fjölmiðlaatriði lítur svona út:

```json {3}
{
  "multimediaIdentifier": "multimedia-1",
  "identifier": "partners/radisson_blu_logo",
  "type": "IMAGE",
  "width": "1024",
  "height": "768",
  "category": "1",
  "descriptions": [
    {
      "name": "Lobby",
      "description": "Picture of the lobby",
      "language": "en"
    }
  ],
  "lifestyleType": "LIFESTYLE_BUSINESS",
  "attribution": [
    {
      "url": "https://www.hilton.com",
      "name": "Hilton"
    }
  ]
}
```

Línan sem er merkt sýnir einstakt Cloudinary auðkenni `partners/radisson_blu_logo`. Þetta er allt sem þú þarft til að sækja myndina í hvaða sniði, gæðum og stærð sem þú vilt.

## Prófanir

Farðu á [API-svæðið okkar](/is/developers/apis) og veldu API-ið sem þú vilt vinna með.

### API skjöl

Skjalsíðan okkar er uppsett þannig að þú getur keyrt API skipanir beint innan síðunnar.

Í þessu dæmi viltu nota `Search Lookups (OAuth2)` API kallið.

:::tip[Pro ráð]
Til að auðvelda notkun mælum við með að samþættarar noti þau endapunkta sem við merkjum sem `OAuth2`. Þessir endapunktar vita að þeir eiga að taka við API köllum og vita þegar hvaða reikningur þú ert á svo þú þarft ekki að tilgreina `companyIdentifier` í URL-inum þínum.
:::

<Steps>
1. Notaðu [Lookup API](https://developer.wink.travel/lookup) til að sækja áfangastaði eða hótel.
2. Í vinstri valmynd, smelltu á `Authentication`.
3. Sláðu inn client-id og client secret og smelltu á `Get TOKEN` hnappinn.
4. Ef auðkenni þín eru gild sérðu að API lykillinn þinn hefur verið virkjaður.
5. Þú getur nú prófað lookup endapunktinn.
</Steps>

### Postman
Þú getur gert það sama með Postman.

<Steps>
1. Sæktu [Postman](https://www.postman.com/downloads).
2. Smelltu á `Download OpenAPI spec` efst á [Lookup API](https://developer.wink.travel/lookup) síðunni.
3. Flytja það inn í Postman.
4. Ef þú fluttir það inn sem nýja safn sem heitir `Wink API`, smelltu á lárétta þrennu (3 punktar) við hlið safnsins og veldu `Edit`.
5. Í `Authorization` flipanum, stilltu `Auth type` á OAuth2.
6. Stilltu reitinn `Add auth data to` á `Request headers`.
7. Stilltu reitinn `Header prefix` á `Bearer`.
8. Stilltu reitinn `Token Name` á það sem þú vilt kalla vistaða tokenið þitt í Postman, t.d. Wink Staging.
9. Stilltu reitinn `Grant type` á `Client Credentials`.
10. Stilltu reitinn `Access token URL` á https://staging-iam.wink.travel/oauth2/token.
11. Stilltu reitinn `Client ID` á Client ID sem umsóknin þín hjálpaði til að búa til.
12. Stilltu reitinn `Client Secret` á Client Secret sem umsóknin þín hjálpaði til að búa til.
13. Stilltu reitinn `Scope` á `inventory.read inventory.write inventory.remove`. Þetta eru öll réttindin sem þú þarft fyrir Wink.
14. Stilltu reitinn `Client Authentication` á `Send as Basic Auth header`.
15. Smelltu á `Get New Access Token` hnappinn til að sækja tokenið þitt og vista það.
16. Fyrir öll API köll undir Wink API safninu, vertu viss um að auðkenningaflipinn segi `Inherit auth from parent` svo tokenið þitt fylgi með í hverju kalli.
</Steps>

:::tip[Pro ráð]
Mælt er með að gera server lén, client ID og client secret að umhverfisbreytum. Þannig getur þú auðveldlega skipt á milli staging og framleiðsluumhverfa og endurnýtt allar aðgerðir sem þú hefur búið til.
:::
