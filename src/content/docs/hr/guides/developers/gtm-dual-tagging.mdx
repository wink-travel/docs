Ako vodite oglašivačke kampanje putem Sojerna, možete povezati svoj Sojern račun s Winkom unosom vašeg Sojern GTM Container ID-a u Wink portal. Nakon povezivanja, Sojernov tracking pixel se automatski aktivira za vašu nekretninu — pokrivajući cijelo putovanje gosta od pretraživanja do završene rezervacije.

<Aside type="tip">
Ovo pretpostavlja da već imate aktivan Sojern račun i da ste od svog Sojern account managera dobili GTM Container ID. Ako još nemate Sojern račun, posjetite [sojern.com/contact](https://www.sojern.com/contact) za početak.
</Aside>

---

## Što trebate učiniti

<Steps>

1. **Nabavite svoj GTM Container ID od Sojerna**

   Kontaktirajte svog Sojern account managera i zatražite svoj **GTM Container ID**. Ima format `GTM-XXXXXXX`. Ovo se razlikuje od vašeg Sojern Pixel ID-a — pazite da tražite točno GTM Container ID.

2. **Unesite ga u Wink portal**

   Prijavite se na [my.wink.travel](https://my.wink.travel), odaberite svoj račun i idite na karticu **Customizations**. Zalijepite svoj Sojern GTM Container ID u polje za Sojern i spremite. Vaša integracija je sada aktivna.

3. **Potvrdite sa svojim Sojern account managerom**

   Obavijestite svog Sojern account managera da je integracija aktivna. Oni mogu provjeriti s njihove strane da pixel ispravno radi na `wink.travel` za vašu nekretninu i da podaci teku u vaš Sojern račun.

</Steps>

<Aside type="note">
Za ovu integraciju nije potrebno postavljati nikakav pixel kod na vlastitu web stranicu. Wink automatski učitava vaš Sojern container na svim stranicama gdje je vaša nekretnina aktivna.

Ova konfiguracija radi za bilo koji prilagođeni GTM Container ID. Sojern smo koristili samo kao primjer jer je vrlo česta integracija.
</Aside>

## Što Wink radi s naše strane

Radi transparentnosti, evo što Wink konfigurira da bi ova integracija funkcionirala:

- **Polje za Sojern GTM Container ID** dostupno je u kartici Customizations vašeg Wink portala. Kada unesete svoj Container ID, Wink provjerava format i sigurno ga pohranjuje za vašu nekretninu.
- **Na svakoj stranici gdje je vaša nekretnina aktivna** na Winku, vaš Sojern GTM Container ID se ubacuje u stranicu, a Winkov sustav za upravljanje tagovima dinamički učitava vaš Sojern container zajedno sa svojim. To se događa automatski — bez ručnih koraka od Wink tima po nekretnini.
- **Vaš Sojern container se učitava asinkrono**, što znači da ne utječe na performanse učitavanja stranice za vaše goste.
- **Winkovo vlastito praćenje i vaš Sojern pixel rade paralelno**, čitajući iz istih zajedničkih podataka o događajima. Svaki događaj rezervacije koji Wink prati istovremeno je dostupan vašem Sojern containeru.
- **Kada vaša nekretnina nije pregledavana** — na primjer, kada gost pregledava drugi hotel na Winku — vaš Sojern container se ne učitava. Time se osigurava da Sojern prima samo podatke relevantne za vašu nekretninu.

## Referenca DataLayer-a

Wink šalje GA4-kompatibilne događaje u `dataLayer` tijekom cijelog putovanja gosta. Vaš GTM container čita iz istog zajedničkog `dataLayer`-a, tako da svaki trigger ili varijabla koju konfigurirate ima pristup svim svojstvima opisanima u nastavku.

Svaki događaj slijedi standardnu GA4 Enhanced Ecommerce strukturu: string `event` na vrhu i objekt `ecommerce`. Prije svakog push-a, Wink briše prethodni ecommerce payload kako bi spriječio da zastarjeli podaci utječu na nove triggere.

<Aside type="note">
**Obrazac resetiranja dataLayer-a**

Wink resetira ecommerce objekt prije svakog push-a događaja:

```js
dataLayer.push({ ecommerce: null });   // briše prethodni događaj
dataLayer.push({ event: '...', ecommerce: { ... } });
```

Nemojte se oslanjati na to da ecommerce svojstva ostaju između događaja. Uvijek ih čitajte u trenutku triggera unutar istog događaja.
</Aside>

### Događaji putovanja rezervacije

Ovi događaji se automatski aktiviraju dok se gost kreće kroz lijevak rezervacije. Konfigurirajte svoje GTM triggere kao **Custom Event** triggere koji odgovaraju imenima događaja u nastavku.

| Korak | Događaj | Kada se aktivira |
|-------|---------|------------------|
| 1 | `view_item_list` | Učitava se mreža ili lista hotela |
| 2 | `view_item` | Gost pregledava stranicu određene nekretnine |
| 3 | `add_to_cart` | Gost dodaje sobu u košaricu |
| 4 | `begin_checkout` | Učitava se stranica za naplatu / plaćanje |
| 5 | `add_payment_info` | Gost odabire način plaćanja |
| 6 | `login` | Gost se autentificira putem SSO-a |
| 7 | `purchase` | Rezervacija je potvrđena |
| 8 | `refund` | Rezervacija je otkazana |

#### `view_item_list`

Aktivira se kada se prikaže mreža ili lista hotela.

```json
{
  "event": "view_item_list",
  "ecommerce": {
    "item_list_id": "featured_hotels",
    "item_list_name": "Featured Hotels",
    "items": [ ]
  }
}
```

### Prilagođene dimenzije u GA4

GA4 ignorira prilagođene parametre stavki u izvještajima dok ih eksplicitno ne registrirate kao dimenzije vezane uz stavku. Potrebno je registrirati samo one parametre koje planirate koristiti u izvještajima — nije potrebno registrirati svih 20.

<Steps>

1. **Otvorite Custom Definitions**

   U GA4 idite na **Admin** (ikona zupčanika dolje lijevo) → odaberite svoju svojinu → **Data display** → **Custom definitions**.

2. **Kreirajte novu prilagođenu dimenziju**

   Kliknite **Create custom dimensions**.

3. **Ispunite detalje dimenzije**

   - **Dimension name** — čitljiv naziv koji se pojavljuje u GA4 izvještajima (pogledajte tablicu dolje)
   - **Scope** — odaberite **Item**
   - **Event parameter** — točan naziv parametra kako se pojavljuje u dataLayer-u (pogledajte tablicu dolje)
   - **Description** — opcionalno, ali preporučeno

4. **Spremite i ponovite**

   Kliknite **Save**, zatim ponovite za svaki parametar koji želite pratiti.

</Steps>

<Aside type="note">
Prilagođene dimenzije mogu se početi prikazivati u GA4 izvještajima do 24 sata nakon registracije. Povijesni podaci prikupljeni prije registracije se ne nadopunjuju.

GA4 dopušta do 50 prilagođenih dimenzija vezanih uz stavku po svojini na standardnim svojinama (360 svojine dopuštaju više).
</Aside>

Koristite sljedeću tablicu kao referencu pri kreiranju prilagođenih dimenzija. Vrijednost **Event parameter** mora biti točna — GA4 razlikuje velika i mala slova.

| Naziv dimenzije | Event parameter | Tip |

| Property name | `property_name` | Tekst |
| Property brand | `property_brand` | Tekst |

### `virtualPageView`

Ovo je prilagođeni Wink događaj — nije dio standardnog GA4 seta događaja. Aktivira se pri svakoj navigaciji stranice i nosi kontekst uređaja i sesije. Ne uključuje `ecommerce` objekt.

```json
{
  "event": "virtualPageView",
  "device": "Desktop",
  "deviceAgent": "Mozilla/5.0 ...",
  "os": "MacOS",
  "path": "/hotels/grand-resort",
  "title": "Grand Resort — Wink",
  "userId": "user_abc123",
  "startDate": "2026-06-01",
  "endDate": "2026-06-05",
  "guests": "2",
  "rooms": "1"
}
```

| Svojstvo | Opis |
|----------|------|
| `device` | Detektirani tip uređaja — `Desktop`, `Android`, `iPhone`, `iPad` ili slično |
| `deviceAgent` | Izvorni user agent string |
| `os` | Detektirani operativni sustav — `Windows`, `MacOS`, `Linux`, `UNIX` |
| `path` | Trenutna putanja stranice |
| `title` | Trenutni naslov stranice |
| `userId` | Identifikator autentificiranog korisnika (izostavljen ako nije prijavljen) |
| `startDate` | Datum prijave iz trenutne sesije pretraživanja |
| `endDate` | Datum odjave iz trenutne sesije pretraživanja |
| `guests` | Broj gostiju iz trenutne sesije pretraživanja |
| `rooms` | Broj soba iz trenutne sesije pretraživanja |
