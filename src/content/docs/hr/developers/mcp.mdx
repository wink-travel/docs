## Pregled
Hostani Wink MCP poslužitelj dostupan je na:

```
https://docs.mcp.wink.travel/mcp
```

Usmjerite kompatibilne AI klijente (VS Code, Cursor, drugi koji implementiraju MCP HTTP transport) na ovu krajnju točku kako bi mogli:
- Pregledavati dokumentaciju proizvoda i platforme (`wink://docs/...`)
- Pregledavati aktivne OpenAPI specifikacije i dokumentaciju operacija (`wink://api/...`)
- Izvoditi laganu semantičku / ključnu pretragu kroz dokumente i aktivne API-je

## Transport
Koristite primarnu streamabilnu HTTP krajnju točku:

```
https://docs.mcp.wink.travel/mcp
```

Ako vaš klijent podržava samo SSE, koristite:

```
https://docs.mcp.wink.travel/sse
```

### Claude Code
Službena dokumentacija: [Povežite Claude Code s alatima putem MCP](https://code.claude.com/docs/en/mcp)

Anthropic preporučuje udaljeni HTTP transport za cloud-hostane MCP poslužitelje:
```bash
claude mcp add --transport http wink-docs https://docs.mcp.wink.travel/mcp
claude mcp list
```

Unutar Claude Code pokrenite `/mcp` za pregled statusa poslužitelja. Ako želite podijeliti poslužitelj s cijelim repozitorijem, dodajte `--scope project` kako bi Claude Code zapisao konfiguraciju u `.mcp.json`.

### Codex
Službena dokumentacija: [Docs MCP brzi početak za Codex](https://developers.openai.com/resources/docs-mcp)

OpenAI dokumentira postavljanje MCP-a putem Codex CLI-ja, te napominje da se konfiguracija dijeli s IDE ekstenzijom:
```bash
codex mcp add wink-docs --url https://docs.mcp.wink.travel/mcp
codex mcp list
```

### Cursor
Cursor MCP dokumentacija: https://cursor.com/docs/context/mcp
Postavljanje: Dodajte udaljeni poslužitelj putem MCP postavki.
```json
{
  "mcpServers": {
    "wink-docs": {
      "url": "https://docs.mcp.wink.travel/mcp",
    }
  }
}
```

### VS Code
Službena dokumentacija: https://code.visualstudio.com/docs/copilot/customization/mcp-servers
Minimalna udaljena konfiguracija (korisnička ili radna `mcp.json`):
```json
{
  "servers": {
    "wink-docs": {
      "type": "http",
      "url": "https://docs.mcp.wink.travel/mcp"
    }
  }
}
```

Savjeti:
- Koristite Command Palette: “MCP: List Servers” za potvrdu veze i pregled alata.
- Alat za odabir omogućuje uključivanje/isključivanje `api_search` i `docs_search`.
- Dodajte kontekst: Prikaz chata → Add Context → MCP Resources za pridruživanje stranica dokumentacije.

## Osnovni URI resursi

| URI | Opis |
|-----|-------------|
| `wink://docs` | Indeks dokumentacije (grupiran po kategorijama) |
| `wink://docs/{path}` | Specifični članak dokumentacije (Markdown / MDX) |
| `wink://api` | Aktivni OpenAPI katalog otkriven iz Wink swagger konfiguracije |
| `wink://api/{apiName}` | Sažetak API-ja s autentifikacijom, oznakama, zaglavljima i primjerima operacija |
| `wink://api/{apiName}/operations` | Potpuni popis operacija grupiran po oznakama |
| `wink://api/{apiName}/operation/{key}` | Detalji pojedinačne operacije (parametri, tijelo zahtjeva, odgovori) |
| `wink://api/{apiName}/schema/{schemaName}` | Riješena komponenta sheme s ugniježđenim poljima, tipovima i primjerima |
| `wink://api/{apiName}/operation/{key}/request` | Prikaz usmjeren na zahtjev s potrebnim unosima i minimalnim primjerom podataka |
| `wink://api/{apiName}/operation/{key}/response/{status}` | Prikaz usmjeren na odgovor za određeni statusni kod |
| `wink://api/{apiName}/operation/{key}/recipe` | Tok zadatka za ispravan poziv te operacije |

## Alati za pretraživanje (brzi pregled)
| Alat | Svrha | Primjeri tipičnih upita |
|------|---------|-----------------------|
| `api_search` | Pronađite aktivni API ili operaciju za integraciju | `create affiliate`, `payment receipt`, `lookup inventory`, `invite manager` |
| `docs_search` | Pronađite konceptualnu dokumentaciju / vodiče | `authentication`, `webhooks`, `wordPress` |

Ostavite `query` praznim za primanje prijedloga. Svaki rezultat uključuje MCP URI koji možete otvoriti.
Za rezultate operacija, `api_search` također vraća povezane URI-je za zahtjev, odgovor, shemu i recept kako bi agenti mogli izravno pristupiti najkorisnijim podresursima.

## Tipični obrazac korištenja
1. Pokrenite `api_search` s vašom namjerom (npr. *"create affiliate"* ili *"payment receipt"*).
2. Otvorite vraćeni `wink://api/...` URI ili slijedite njegove povezane `request_uri`, `response_uris`, `request_schema_uris`, `response_schema_uris` i `recipe_uri` polja.
3. Koristite `docs_search` za podržavajuće koncepte (autentifikacija, webhooks, workflowi).

## AI vještine

Unaprijed izrađene vještine agenata koriste ovaj MCP poslužitelj kao svoj izvor istine. Vještine daju vašem kodirajućem agentu strukturirani tijek rada za razvoj prema Wink API-jima — umjesto nagađanja krajnjih točaka, agent čita aktivne ugovore izravno s MCP poslužitelja.

Pogledajte [AI Skills](/hr/developers/ai-skills) za dostupne vještine i upute za instalaciju.

## Napomene
- Svaki vraćeni URI agent može otvoriti za puni sadržaj.
- Stablo `wink://api` je aktivno i prati trenutni upstream OpenAPI katalog umjesto generiranog indeksa repozitorija.
- Prijedlozi se automatski pojavljuju kada su upiti prazni.
