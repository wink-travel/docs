AI vještine su prenosivi paketi uputa koji podučavaju agente za kodiranje kako raditi s Winkom. Umjesto oslanjanja na podatke za obuku ili zastarjelu dokumentaciju, svaka vještina usmjerava agenta da koristi [Wink MCP server](/hr/developers/mcp) kao aktivni izvor istine za API ugovore i dokumentaciju platforme.

Vještine rade s bilo kojim agent klijentom koji ih podržava — Claude Code, Codex, Cursor i drugi.

## Dostupne vještine

### wink-api-builder

Implementirajte, pregledajte ili otklonite pogreške u Wink API integracijama. Vještina vodi agenta kroz strukturirani tijek rada:

1. Pretražite odgovarajuću API operaciju putem `api_search`
2. Otvorite aktivne ugovore zahtjeva/odgovora s MCP servera
3. Preuzmite postavke autentifikacije iz dokumentacije
4. Napišite kod prema verificiranom ugovoru
5. Označite sve nesukladnosti između postojećeg koda i trenutnog API-ja

Koristan za:
- Izgradnju novih metoda klijenta za bilo koji Wink API
- Otklanjanje pogrešaka u nesukladnostima ugovora (npr. zahtjev koji vraća HTTP 400)
- Pregled postojećih integracija prema aktivnoj specifikaciji

## Instalacija

Kopirajte mapu vještine u direktorij vještina vašeg agenta.

:::tip
Wink MCP server mora biti povezan u istoj okolini. Pogledajte [MCP Server](/hr/developers/mcp) za upute o postavljanju.
:::

**Claude Code**

```bash
# lokalno u repozitoriju
cp -r skills/wink-api-builder .claude/skills/

# ili na razini korisnika
cp -r skills/wink-api-builder ~/.claude/skills/
```

Pozovite s:
```
/wink-api-builder add a TypeScript client for the Wink affiliate create flow
```

**Codex**

```bash
# lokalno u repozitoriju
cp -r skills/wink-api-builder .agents/skills/

# ili na razini korisnika
cp -r skills/wink-api-builder $CODEX_HOME/skills/
```

Pozovite s:
```
Use $wink-api-builder to add a TypeScript client for the Wink affiliate create flow.
```

**Ostali agenti**

Postavite mapu vještine gdje god vaš agent učitava vještine. Ulazna točka je `SKILL.md`.
