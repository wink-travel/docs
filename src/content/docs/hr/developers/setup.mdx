Ovaj članak pokazuje kako se brzo postaviti kao developer i početi testirati naše API-je na Wink.

:::note
Testiranje bi uvijek trebalo obavljati u našem [staging okruženju](/hr/getting-started/environments). Tamo ima puno manje podataka, ali slobodni ste testirati koliko god želite bez utjecaja na produkcijsko okruženje.

Kada ste spremni za lansiranje, certificirat ćemo vaš račun i vašu aplikaciju te možete pokrenuti u našem produkcijskom okruženju.
:::

Slijedite ove korake:

<Steps>
1. [Registrirajte](/hr/user/register) svoj korisnički račun na Wink
2. Kreirajte [račun turističkog agenta](/hr/travel-agent/what-is-agent) ili [račun partnera](/hr/studio/what-is-studio).
3. [Kreirajte aplikaciju](/hr/developer/applications) i povežite je s računom koji ste kreirali u koraku 2.
4. Nakon što ste kreirali `Application`, spremite client-id i client secret na sigurno mjesto.
</Steps>

## CDN

U Wink-u koristimo Cloudinary za sve naše slike i video materijale. Kada pristupite našem inventaru putovanja, imat ćete potpuni pristup svim naprednim značajkama Cloudinaryja.

Naš `cloud_name` je `traveliko`.

### Cloudinary

- [Početak rada](https://cloudinary.com/documentation/dev_kickstart)
- [SDK-ovi](https://cloudinary.com/documentation/cloudinary_sdks)
- [Cloudinary na GitHubu](https://github.com/cloudinary)
-	[Video tutorijali](https://cloudinary.com/documentation/cloudinary_sdks_tutorials)

### Primjer

Za svaki tip inventara putovanja koji podržavamo, postoji ugniježđeni JSON niz nazvan `multimedias`. JSON za jedan multimedijski unos izgleda ovako:

```json {3}
{
  "multimediaIdentifier": "multimedia-1",
  "identifier": "partners/radisson_blu_logo",
  "type": "IMAGE",
  "width": "1024",
  "height": "768",
  "category": "1",
  "descriptions": [
    {
      "name": "Lobby",
      "description": "Slika lobija",
      "language": "en"
    }
  ],
  "lifestyleType": "LIFESTYLE_BUSINESS",
  "attribution": [
    {
      "url": "https://www.hilton.com",
      "name": "Hilton"
    }
  ]
}
```

Istaknuti redak iznad prikazuje jedinstveni Cloudinary ID `partners/radisson_blu_logo`. To je sve što vam treba za dohvat slike u bilo kojem formatu, kvaliteti i veličini koju želite.

## Testiranje

Idite na našu sekciju [API-jevi](/hr/developer/apis) i odaberite API s kojim želite raditi.

### API dokumentacija

Naša dokumentacija je postavljena tako da možete izvoditi API naredbe direktno sa stranice.

U ovom primjeru želite koristiti API poziv `Search Lookups (OAuth2)`.

:::tip[Pro savjet]
Radi lakše upotrebe, preporučujemo integratorima da koriste krajnje točke koje označavamo kao `OAuth2`. Te krajnje točke prepoznaju API pozivatelje i već znaju na kojem ste računu, tako da ne morate navoditi `companyIdentifier` u URL-ovima.
:::

<Steps>
1. Koristite [Lookup API](https://developer.wink.travel/lookup) za dohvat destinacija ili hotela.
2. U lijevom izborniku kliknite na `Authentication`.
3. Unesite svoj client-id i client secret te kliknite na gumb `Get TOKEN`.
4. Ako su vaše vjerodajnice valjane, vidjet ćete da je vaš API ključ primijenjen.
5. Sada možete testirati lookup endpoint.
</Steps>

### Postman
Isto možete napraviti i s Postmanom.

<Steps>
1. Preuzmite [Postman](https://www.postman.com/downloads).
2. Kliknite na `Download OpenAPI spec` na vrhu stranice [Lookup API](https://developer.wink.travel/lookup).
3. Uvezite ga u Postman.
4. Ako ste ga uvezli kao novu kolekciju pod nazivom `Wink API`, kliknite na horizontalne tri točke pored imena kolekcije i odaberite `Edit`.
5. U kartici `Authorization` postavite `Auth type` na OAuth2.
6. Postavite polje `Add auth data to` na `Request headers`.
7. Postavite polje `Header prefix` na `Bearer`.
8. Postavite polje `Token Name` na naziv koji želite za svoj spremljeni token u Postmanu, npr. Wink Staging.
9. Postavite polje `Grant type` na `Client Credentials`.
10. Postavite polje `Access token URL` na https://staging-iam.wink.travel/oauth2/token.
11. Postavite polje `Client ID` na Client ID koji je vaša aplikacija generirala.
12. Postavite polje `Client Secret` na Client Secret koji je vaša aplikacija generirala.
13. Postavite polje `Scope` na `inventory.read inventory.write inventory.remove`. To su svi opsezi koji su vam potrebni za Wink.
14. Postavite polje `Client Authentication` na `Send as Basic Auth header`.
15. Kliknite gumb `Get New Access Token` da dohvatite i spremite svoj token.
16. Za sve API pozive unutar Wink API kolekcije, provjerite da kartica za autentifikaciju kaže `Inherit auth from parent` kako bi se vaš token uključivao u svaki poziv.
</Steps>

:::tip[Pro savjet]
Preporučuje se da domena servera, client ID i client secret budu postavljeni kao varijable okruženja. Tako možete lako prebacivati između staging i produkcijskih okruženja i ponovno koristiti sve operacije koje ste kreirali.
:::
