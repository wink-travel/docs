:::note
建立您的第一個主費率是入門精靈的一部分。
:::

:::note
`主費率` 是獨特的 `客房 / 費率方案` 配對。

*大多數旅遊費率供應商在此情境下僅使用客房和費率方案。當您看到您的客房在 OTA 或訂房引擎上販售時，旅客會比較不同的費率方案。*

Wink 新增了 `主費率` 這個術語，以便讓這個獨特組合擁有更多彈性。
:::

:::note
當新增或移除客房或費率方案時，主費率會自動建立或移除。
:::

要管理您的主費率，請從主選單點擊 `Monetize > Master Rate`。

每個物業會有每個客房 / 費率方案組合的主費率。

*範例：如果您的物業有一間客房和兩個費率方案，旅客將看到兩個可販售的主費率（若所有條件符合）。*

主費率條目有兩個管理選項：

- 切換主費率的可用性。
- 設定優惠。

## 可用性

啟用或停用主費率時，會影響所有銷售通路的可用性。此開關是快速在所有地方立即關閉特定主費率的方式。

## 優惠

當旅客想訂房時，他們不會直接與客房互動，而是與該客房的主費率互動。
主費率可以附加獨特的優惠，這些優惠僅適用於特定的客房 / 費率方案組合，從而為物業業主提供另一種激勵該組合的方式。

選擇優惠時需注意兩件事：

- 是保證提供還是視可用性而定。
- 其數值大小。

保證提供的優惠數值較高，因為它代表對顧客更大的價值。
每家飯店會根據所有主費率中所有優惠的總數值，獲得一個 `優惠分數`。
尋找優惠的旅客可以依據優惠分數排序飯店。

優惠的作用是：

- 區別客房 / 費率方案組合。
- 獲得更高的曝光度。*例如，在您所在地區的其他飯店中，優惠分數排名更高。*

:::tip
優惠的出現源自飯店想要區別自己並「遊戲化系統」的需求。
上一次他們這麼做時，實際上是自我傷害，因為他們為了更好的排名，透過提高佣金將所有利潤讓給了 OTA。
歷史上，佣金是飯店唯一能用來遊戲化的項目。

優惠是飯店用來獎勵旅客的一種方式，且不一定會產生實際成本。

如果您有其他想要提供的優惠類型建議，[請與我們聯繫](mailto:hi@wink.travel)。
:::

## API

想管理 `主費率` 的開發者可以前往 [Developers > APIs > Monetize](/zh-TW/developers/apis/#monetize-api)。
