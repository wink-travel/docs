應用程式讓您能完全存取 Wink 和 TripPay 的功能，並讓您的使用者使用我們的 OAuth2 授權伺服器在您的網站上進行身份驗證和登入；就像 Facebook 和 Google 一樣。
您可以使用 Wink 建立自己的 OTA。這一切都從一個應用程式開始。

*預設情況下，當您註冊新的 Wink 聯盟帳號或 TripPay 帳號時，會自動獲得一個應用程式。我們在某些情境下會使用 `client ID`。*

:::note
在 Wink 或 TripPay 產生的應用程式可用於所有 API 端點。
:::

*在此範例中，我們將透過 [Wink Studio](https://studio.wink.travel) 進行身份驗證。*

<Steps>
1. 身份驗證後，點擊右上角的個人資料圖示，會出現選單。
2. 點擊 `Applications` 連結。
3. 您將被導向您的應用程式頁面。
</Steps>

## 建立應用程式

由於您無法存取我們為您建立的應用程式的 `secret key`，您需要自行建立一個。

:::caution
您必須先在 Wink 建立聯盟帳號或在 TripPay 建立帳號，才能建立應用程式，因為應用程式會綁定至特定帳號。
:::

建立應用程式的步驟如下：

<Steps>
1. 在應用程式頁面，點擊 `Create new application` 按鈕。
2. ***名稱*** 為您的應用程式命名。*例如 Cool App*
3. **實體名稱** 將您的應用程式與您現有的帳號連結。*例如 Cool Account*
4. **重新導向 URI** 至少需要一個重新導向 URI。此 URI 用於驗證成功身份驗證後的重新導向網域。*例如 https://www.cool-site.com*
5. 點擊 `Save` 按鈕繼續。
</Steps>

您將被導回應用程式列表頁面。您會在列表中看到新建立的應用程式，且只會這一次看到應用程式的 secret key。請務必複製並妥善保存。

## 更新應用程式

如果您的重新導向 URI 有變更，或想新增或移除 URI，您需要更新應用程式。

更新現有應用程式的步驟如下：

<Steps>
1. 點擊您想更新的應用程式旁的 `Actions` 連結。
2. 點擊 Actions 下的 `Update` 按鈕。
3. 進行應用程式的修改。
4. 點擊 `Save` 按鈕繼續。
</Steps>

您將被導回應用程式列表頁面。

## 移除應用程式

:::caution
請勿刪除我們自動為您建立的應用程式，以確保您帳號的所有功能正常運作。
:::

如果您不再需要某個應用程式，可以將其移除。

移除應用程式的步驟如下：

<Steps>
1. 點擊您想移除的應用程式旁的 `Actions` 連結。
2. 點擊 Actions 下的 `Remove` 按鈕。
3. 點擊 `OK` 按鈕確認刪除。
</Steps>

您將被導回應用程式列表頁面。
