Wink 與領先的通路管理系統、物業管理系統（PMS）及中央訂房系統（CRS）整合，確保您的價格與可用性在所有銷售通路中保持同步。

## 運作方式

通路管理整合可實現您的物業管理系統與 Wink 之間的雙向資料流：

1. **價格推送** — 您的通路管理系統將價格更新發送至 Wink。當您在 PMS 中更新價格時，這些變更會自動反映在 Wink 上。
2. **可用性同步** — 您的通路管理系統將可用性更新發送至 Wink。當房間在 PMS 中被預訂或釋出時，Wink 的庫存會即時更新。
3. **訂單通知** — 當在 Wink 上完成訂房時，您的通路管理系統會收到通知，讓訂單顯示在您的 PMS 中。

## 支援的通路管理系統

Wink 目前整合以下系統：

| 通路管理系統 | 說明 |
|---|---|
| **Channex** | 全球分銷與通路管理平台。 |
| **TravelClick** | 飯店解決方案與通路管理（Amadeus）。 |
| **Omnibees** | 飯店分銷與收益管理。 |
| **D-Edge** | 旅宿科技與中央訂房解決方案。 |
| **Allotz** | 庫存分配與通路管理。 |
| **Ezee / Yanolja** | 以亞洲市場為主的雲端 PMS 與通路管理。 |
| **RateGain** | 價格情報與分銷技術。 |
| **RateTiger** | 即時價格與可用性分銷。 |
| **CloudBeds** | 雲端 PMS、通路管理與訂房引擎。 |
| **SiteMinder** | 領先的飯店通路管理與分銷平台。 |
| **Exely** | 飯店價格與可用性同步。 |

:::note
如果您的通路管理系統未列出，請聯繫 Wink 團隊。可根據 Wink 支援的 OTA（Open Travel Alliance）協議標準新增整合。
:::

## 開發者專區

如果您正在開發與 Wink 的通路管理系統或 PMS 整合，請使用通路管理 API 端點：

- **價格更新** — 將價格變更推送至 Wink。
- **可用性更新** — 將可用性變更推送至 Wink。
- **訂單擷取** — 從 Wink 拉取訂單至您的系統。

通路管理 API 遵循 OTA 協議標準（SOAP/XML），以確保與現有旅宿系統的相容性。

## Webhook 通知

您可以訂閱通路管理系統的 webhook 事件，以接收即時通知：

- `channel-manager.update.rate` — 收到價格更新。
- `channel-manager.update.availability` — 收到可用性更新。
- `channel-manager.update` — 一般通路管理更新。

詳情請參閱 [Webhook 事件目錄](/zh-TW/developers/webhook-events#channel-manager-events)。
