## 概覽
託管的 Wink MCP 伺服器位於：

```
https://docs.mcp.wink.travel/mcp
```

將相容的 AI 用戶端（VS Code、Cursor、其他實作 MCP HTTP 傳輸的工具）指向此端點，可讓它們：
- 瀏覽產品與平台文件（`wink://docs/...`）
- 瀏覽即時 OpenAPI 規格與操作文件（`wink://api/...`）
- 對文件與即時 API 執行輕量語意／關鍵字搜尋

## 傳輸
使用主要可串流的 HTTP 端點：

```
https://docs.mcp.wink.travel/mcp
```

如果您的用戶端僅支援 SSE，請使用：

```
https://docs.mcp.wink.travel/sse
```

### Claude Code
官方文件：[透過 MCP 連接 Claude Code 到工具](https://code.claude.com/docs/en/mcp)

Anthropic 建議對雲端託管的 MCP 伺服器使用遠端 HTTP 傳輸：
```bash
claude mcp add --transport http wink-docs https://docs.mcp.wink.travel/mcp
claude mcp list
```

在 Claude Code 中，執行 `/mcp` 以檢查伺服器狀態。若想與整個專案共用伺服器，請加上 `--scope project`，讓 Claude Code 將設定寫入 `.mcp.json`。

### Codex
官方文件：[Codex 的 Docs MCP 快速入門](https://developers.openai.com/resources/docs-mcp)

OpenAI 透過 Codex CLI 文件說明 MCP 設定，並指出此設定會與 IDE 擴充功能共用：
```bash
codex mcp add wink-docs --url https://docs.mcp.wink.travel/mcp
codex mcp list
```

### Cursor
Cursor MCP 文件：https://cursor.com/docs/context/mcp
設定：透過 MCP 設定新增遠端伺服器。
```json
{
  "mcpServers": {
    "wink-docs": {
      "url": "https://docs.mcp.wink.travel/mcp",
    }
  }
}
```

### VS Code
官方文件：https://code.visualstudio.com/docs/copilot/customization/mcp-servers
最簡遠端設定（使用者或工作區的 `mcp.json`）：
```json
{
  "servers": {
    "wink-docs": {
      "type": "http",
      "url": "https://docs.mcp.wink.travel/mcp"
    }
  }
}
```

提示：
- 使用命令面板：「MCP: List Servers」確認連線並瀏覽工具。
- 工具選擇器可啟用／停用 `api_search` 和 `docs_search`。
- 新增上下文：聊天視圖 → 新增上下文 → MCP 資源，附加文件頁面。

## 核心資源 URI

| URI | 說明 |
|-----|-------------|
| `wink://docs` | 文件索引（依類別分組） |
| `wink://docs/{path}` | 特定文件文章（Markdown / MDX） |
| `wink://api` | 從 Wink swagger 設定發現的即時 OpenAPI 目錄 |
| `wink://api/{apiName}` | 含認證、標籤、標頭及範例操作的精簡 API 摘要 |
| `wink://api/{apiName}/operations` | 依標籤分組的完整操作列表 |
| `wink://api/{apiName}/operation/{key}` | 單一操作細節（參數、請求主體、回應） |
| `wink://api/{apiName}/schema/{schemaName}` | 已解析的元件結構，含巢狀欄位、類型與範例 |
| `wink://api/{apiName}/operation/{key}/request` | 以請求為主的檢視，含必要輸入與最小範例負載 |
| `wink://api/{apiName}/operation/{key}/response/{status}` | 針對特定狀態碼的回應檢視 |
| `wink://api/{apiName}/operation/{key}/recipe` | 呼叫該操作的任務導向流程 |

## 搜尋工具（快速參考）
| 工具 | 目的 | 典型查詢範例 |
|------|---------|-----------------------|
| `api_search` | 尋找可整合的即時 API 或操作 | `create affiliate`、`payment receipt`、`lookup inventory`、`invite manager` |
| `docs_search` | 尋找概念性文件／指南 | `authentication`、`webhooks`、`wordPress` |

留空 `query` 可獲得建議。每個結果都包含可開啟的 MCP URI。
對於操作命中，`api_search` 也會回傳連結的請求、回應、結構與流程 URI，讓代理能直接跳轉到最有用的子資源。

## 典型使用模式
1. 使用您的意圖執行 `api_search`（例如 *「create affiliate」* 或 *「payment receipt」*）。
2. 開啟回傳的 `wink://api/...` URI，或依其連結的 `request_uri`、`response_uris`、`request_schema_uris`、`response_schema_uris` 和 `recipe_uri` 欄位操作。
3. 使用 `docs_search` 查找支援概念（認證、webhooks、工作流程）。

## AI 技能

預建的代理技能使用此 MCP 伺服器作為其真實資料來源。技能為您的程式代理提供結構化工作流程，以便針對 Wink API 建構 — 不必猜測端點，代理直接從 MCP 伺服器讀取即時合約。

請參閱 [AI Skills](/zh-TW/developers/ai-skills) 了解可用技能及安裝說明。

## 注意事項
- 每個回傳的 URI 代理都能開啟以取得完整內容。
- `wink://api` 樹狀結構是即時的，遵循目前上游 OpenAPI 目錄，而非產生的 repo 索引。
- 查詢為空時會自動顯示建議。
