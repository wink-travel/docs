這樣的平台不是孤立建造的，而是許多運作部件的總和；包含舊的與新的。我們希望繼續服務後來者。

> 「沒必要重新發明破損的輪子。」

旅遊科技分散且多為舊有系統。我們嘗試採用有效的舊系統精華，整合了多個渠道管理系統、物業管理系統等。我們也加入了一些新元素，並將所有這些包裝在業界標準和通用協議中……讓你不必自己去做。

:::note[API 驅動方法]
大致上意味著，我們為自己建置的功能應該是：

- 易於理解
- 靈活
- 可擴展
- 可重用，且...
- 可擴充

……供其他也想使用這些 API 的人使用。
:::

Wink 是使用這種 `API 驅動方法` 建構的。基於這種設計理念，所有 Wink 的功能都可以像是系統屬於你一樣，讓外部開發者進行互動。平台還有其他面向，供你在整合時利用。

## AI 驅動內容

Wink 使用 AI 生成物業描述、房型細節、餐廳文案等，涵蓋 9 種庫存類型與 12 種語氣。AI 也支援一鍵物業註冊，透過擷取網站 URL 自動生成所有內容。
