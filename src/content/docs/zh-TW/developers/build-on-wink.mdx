> 「如果我看得更遠，那是因為我站在巨人的肩膀上。」

這樣的平台不是孤立建造的，而是許多舊有與新興元素的綜合體。我們希望繼續服務後來者。

> 「沒必要重新發明破損的輪子。」

旅遊科技分散且多為舊有系統。我們嘗試採用有效的舊系統精華，整合了多個渠道管理器、物業管理系統等。我們也加入了一些新功能，並將所有這些包裝在業界標準和通用協議中……讓你不必自己去做。

:::note[API 驅動方法]
大致上意味著，我們為自己打造的功能應該是：

- 易於理解
- 靈活
- 可擴展
- 可重用，且…
- 可擴充

…供其他想使用這些 API 的人使用。
:::

Wink 採用這種 `API 驅動方法` 建構。基於此設計理念，所有 Wink 功能都能讓外部開發者像操作自己的系統一樣互動。平台還有其他可供整合時利用的面向。
