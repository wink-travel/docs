當 Wink 將內容發佈到您連結的社群媒體帳戶時，會遵守各平台所施加的速率限制。這確保您的帳戶保持良好狀態，且內容能可靠地傳遞。

## 平台限制

每個社群平台都有其自身的發佈限制。Wink 會即時追蹤使用情況，並在您接近這些限制時自動調節發佈速度。

| 平台 | 每小時限制 | 備註 |
|---|---|---|
| **Instagram** | 180 次請求 | 包含貼文、限時動態及 API 互動。 |
| **Facebook** | 4,500 次請求 | 包含粉絲專頁貼文、留言及 API 互動。 |

:::note
這些限制由社群平台自行設定，可能會有所變動。當平台限制改變時，Wink 會自動調整。
:::

## Wink 如何管理速率限制

Wink 使用滑動時間視窗來追蹤您在每個平台的 API 使用量：

1. **每次請求發送時都會加上時間戳記**。
2. **超過 1 小時的請求會自動從視窗中移除**。
3. **在每次新請求前**，Wink 會檢查目前視窗是否還有剩餘容量。
4. **若已達限制**，Wink 會將請求排入佇列，並在有容量時再送出。

這表示您不必擔心超過平台限制或帳戶被標記 — Wink 會透明地處理這些狀況。

## 最佳實踐

- **分散您的貼文。** 與其同時排程大量貼文，不如將它們分散在一天中。這對平台和您的觀眾來說看起來更自然。
- **使用行事曆。** [社群行事曆](/zh-TW/social/calendar) 幫助您視覺化發佈排程，避免貼文集中。
- **監控分析。** 查看您的 [社群分析](/zh-TW/social/analytics)，了解發佈了多少貼文，以及是否有因速率限制而延遲的情況。
