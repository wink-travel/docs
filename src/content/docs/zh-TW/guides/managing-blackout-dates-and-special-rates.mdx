並非每天的價格都相同。活動、假期、淡季及最後一刻空房等情況，都需要不同的定價。本指南將示範如何使用特殊價格和價格方案限制來應對這些情況。

## 活動定價

當會議、節慶或當地活動帶動需求時，您會想要提高這些日期的價格。

<Steps>
1. 前往您的 Extranet 中的 `Monetize > Special Rates`。
2. 點擊 `Create a new special rate`。
3. 選擇要覆蓋的主價格和房型。
4. 設定涵蓋活動期間的日期範圍。
5. 輸入調整後（較高）的每日價格。
6. 點擊 `Save`。
</Steps>

特殊價格只會在這些日期覆蓋主價格。活動期間外，您的常規定價保持不變。

## 季節性調整

針對較長期間（旺季、肩季、淡季），您可以：

- **使用特殊價格** — 為每個季節建立調整後的覆蓋價格。
- **更新主價格** — 直接調整該期間的基礎價格。

特殊價格適合臨時調整，而更新主價格則適合可預測的年度模式。

## 最後一刻價格

為了填補接近入住日的未售房間：

<Steps>
1. 建立新的 **價格方案**，並設定提前預訂限制（例如，必須在入住前三天內預訂）。
2. 搭配提供目標日期折扣的 **特殊價格**。
3. 此價格方案只會顯示給在最後一刻時間窗內預訂的旅客。
</Steps>

## 停用日期

若要阻擋特定日期使用某價格方案（例如，旺季期間不提供促銷價格）：

<Steps>
1. 前往 `Monetize > Rate Plans` 並選擇該價格方案。
2. 移至 **Restrictions** 區段。
3. 新增 **Sell date** 限制，限制該價格可被預訂的時間。
4. 或使用 **Stay date** 限制，限制該價格適用的住宿日期。
</Steps>

例如，您可以限制「早鳥」價格方案在十二月（旺季）期間不可預訂，而您的常規價格方案仍然開放。

## 限制條件組合

限制條件可以疊加以達到精確控制：

| 限制條件 | 控制內容 |
| -- | -- |
| **Sell dates** | 價格可被預訂的時間 |
| **Stay dates** | 價格適用的住宿日期 |
| **Available days** | 價格顯示的星期幾 |
| **Arrival days** | 旅客必須抵達的星期幾 |
| **Departure days** | 旅客必須離開的星期幾 |
| **Required days** | 旅客必須包含至少一天的住宿日期 |
| **Advance booking** | 預訂必須提前多久完成 |
| **Length of stay** | 最短及最長住宿天數 |
