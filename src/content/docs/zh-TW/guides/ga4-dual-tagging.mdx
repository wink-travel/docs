透過將您的 Google Analytics 4 帳戶連接到 Wink，您的訂單事件 — 從房型瀏覽到完成購買 — 將自動轉發到您自己的 GA4 儀表板，與 Wink 自有的分析數據並行。無需程式碼，也不需您端的技術設定。

<Aside type="tip">
您只需要一組 GA4 測量 ID。輸入到 Wink 後台後，您的分析數據即刻生效。
</Aside>

---

## 運作原理

```
您在 Wink 後台輸入 G-XXXXXXXXXX
       ↓
Wink 將您的訂單事件轉發到您的 GA4 帳戶
       ↓
您登入 analytics.google.com，查看您的訂單數據
```

---

## 您需要做的事

<Steps>

1. **建立 GA4 帳戶**

   前往 [analytics.google.com](https://analytics.google.com)：管理員 → 建立帳戶。設定您的時區與貨幣，與您的市場相符。

2. **建立網站資料串流**

   管理員 → 資料串流 → 新增串流 → 網站。輸入 `wink.travel` 作為網址。系統會產生您的 **測量 ID**（`G-XXXXXXXXXX`）。請複製它。

3. **在 Wink 後台輸入您的測量 ID**

   登入 [my.wink.travel](https://my.wink.travel)，選擇您的帳戶，前往 **自訂設定** 分頁。將測量 ID 貼上並儲存。完成後，您的 GA4 整合即啟用。

4. **將 `purchase` 標記為關鍵事件**

   回到您的 GA4 帳戶：管理員 → 事件 → 找到 `purchase` → 切換 **標記為關鍵事件**。這會讓完成的訂單出現在您的轉換報告中。

5. **查看您的數據**

   登入 [analytics.google.com](https://analytics.google.com)，您將看到訂單、收入和旅客漏斗 — 全部由 Wink 自動追蹤。

</Steps>

<Aside type="note">
此整合不需您在自己網站安裝任何追蹤程式碼。Wink 全權處理。您的測量 ID 僅是訂單數據的接收端。
</Aside>

---

## 您可以追蹤的項目

連接後，您的 GA4 帳戶會接收完整的增強型電子商務事件，涵蓋您的物業：

### 發現階段

| 事件 | 意義 |
|---|---|
| `view_item_list` | 旅客在搜尋或分類列表中看到您的物業 |
| `select_item` | 旅客從列表中點擊您的物業 |
| `view_item` | 旅客瀏覽您的飯店詳細頁面 |

### 考慮階段

| 事件 | 意義 |
|---|---|
| `view_promotion` | 旅客看到您的促銷優惠 |
| `select_promotion` | 旅客點擊促銷優惠 |
| `add_to_wishlist` | 旅客將您的物業加入願望清單 |

### 訂單漏斗

| 事件 | 意義 |
|---|---|
| `add_to_cart` | 旅客選擇房型或方案 |
| `view_cart` | 旅客查看訂單摘要 |
| `remove_from_cart` | 旅客移除房型或方案 |
| `begin_checkout` | 旅客開始結帳流程 |
| `add_payment_info` | 旅客輸入付款資訊 |
| `add_shipping_info` | 旅客輸入旅客資料 |
| `purchase` | 旅客完成訂單 |
| `refund` | 訂單取消並退款 |

所有事件都包含相關維度，如收入、貨幣、房型和訂單價值 — 讓您完整掌握旅客從初次印象到完成訂單的旅程。若要解鎖更多物業及房型層級維度，如 `city_name`、`cancellation_policy`、`bed_type` 等，請參考 [GA4 中的自訂維度](/zh-TW/guides/gtm-dual-tagging#custom-dimensions-in-ga4)。

## 跨網域追蹤（進階）

上述標準整合會追蹤所有在 Wink 內發生的行為。但如果您想追蹤旅客從您自己網站一路到 Wink 完成訂單的全程 — 並正確歸因原始流量來源（Google Ads、電子郵件行銷、有機搜尋等） — 則雙方都需額外設定一個步驟。

<Aside type="caution">
此段僅適用於您自己網站已安裝 GA4 的情況。若您網站未使用 GA4，標準整合即足夠。
</Aside>

### 為什麼這很重要

若無跨網域追蹤，旅客先造訪您網站再到 Wink 訂房，GA4 會將此視為 **直接流量**，與先前造訪無關。任何 Google Ads 點擊、電子郵件行銷或有機搜尋導流都不會被歸因。

設定跨網域追蹤後，您將看到完整旅客旅程：

```
google.com（付費廣告）
       ↓
您的網站（旅客瀏覽房型）
       ↓
wink.travel（旅客選房並訂房）
       ↓
訂單正確歸因至原始 Google Ads 點擊
```

### 您需要做的事

設定您的 GA4 帳戶，將 `wink.travel` 視為您自有網域的一部分，而非外部推薦來源。這會讓 GA4 將兩次造訪串接成一個連續旅程。

<Steps>
1. GA4 管理員 → 資料串流 → 您的串流
2. 設定標籤設定 → **設定您的網域**
3. 新增 `wink.travel` 為連結網域
4. 儲存
</Steps>

### Wink 需要做的事

聯絡 [Wink 支援](https://my.wink.travel)，提供您的網站網域（例如 `yourhotel.com`）。Wink 會將您的網域加入 Wink GA4 帳戶的推薦排除清單，避免您的網站被記錄為推薦來源，進而覆蓋旅客原始歸因並啟動新會話。

<Aside type="tip">
上述標準整合對大多數物業已足夠，能立即讓您在 Wink 上看到訂單與收入。跨網域追蹤是進階選項，適合您熟悉 GA4 數據並想獲得更深入歸因洞察時設定。
</Aside>
