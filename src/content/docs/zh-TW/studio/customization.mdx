客製化是一項強大的功能，您可以使用它來控制與 Wink 整合的各個方面，而無需編寫任何程式碼。您銷售的每件庫存都可以客製化。新增或刪除功能、更改配色方案等等。

在`Customization` 頁面上，點擊`Actions` 按鈕，然後按一下`Update`。
*當您建立帳戶時，您已經有一個可用的預設條目。無法刪除該條目。*

這將帶您進入自訂表單，您可以在其中看到 5 - 7 個選項卡（*取決於您的帳戶類型*）。

## 一般的

以下是您可用的選項`General` 選項卡。

* **姓名** 為您的定制提供一個容易記住的名稱。 *例如對於我的旅遊博客*
* **顯示貨幣** 選擇用戶將在可預訂旅行庫存旁邊看到的初始貨幣。 *例如美元*
* **顯示語言** 選擇使用者最初遇到的語言。 *例如英語*
* **附加到頁面標題** 您可以控制新增到頁面標題的內容（*您在瀏覽器標籤中看到的標題*）。這主要用於我們的預訂引擎。 *例如如果您導航至 https://ota.wink.travel，預設情況下，您將看到`Home | Traveliko - Hotels with Benefits`*。您可以在“|”之後更改所有內容管道。

## 主題

這`Theme` 選項卡可讓您自訂我們的外觀和感覺[預訂引擎](/booking-engine/in-a-nutshell) 網站，[WinkLinks](/link-manager/wink-links) 網站和我們所有的[Web 元件](/developers/web-components)。

### 配置

此部分可讓您配置設計和卡片佈局。截至目前，只有一種設計；即 Wink。

卡片佈局控制您是否希望使用者看到水平或垂直對齊的交易卡。訊息是一樣的。

：：：筆記
從行動裝置查看我們的交易卡時，卡片佈局沒有區別。但是，您可以在 iPad mini 和更大的解析度上看到差異。

如果清單中只有不到 4 個旅行清單（即房間類型），請使用水平佈局...這樣看起來更好。

## 行程

這`Itinerary` 在確保顯示您選擇的庫存價格方面，標籤可能是最有趣的。
由於旅行庫存定價相當複雜，Wink 需要了解用戶的行程才能顯示準確的價格。

此部分可讓您設定預設行程以顯示我們所有數位房地產的價格，例如：

* 我們的社交友善圖片帶有價格疊加。
* WinkLinks 網格、卡片和連結。
* Web 元件
* 預訂引擎

### 日期

您可以選擇設定靜態日期範圍還是捲動日期範圍。

* 如果您要對給定日期發生的特定事件進行自訂，請選擇靜態日期範圍。
* 如果您想要執行下列操作之一，請選擇捲動日期範圍：
  1. 預設為今天 + 1 晚，大多數 OTA 都這樣做。
  2. 預設為將來始終可用的日期。

### 房間配置

預設情況下，行程設定為 2 名成人，因為這是大多數旅客最常見的設定。
這可以輕鬆改變，您可以添加多個可容納多名居住者的房間；成人和兒童均可使用。

### 促銷

促銷是一項巧妙的功能，可以節省用戶的時間。如果飯店為您提供了可與朋友分享的促銷代碼，您可以選擇在此輸入。
透過您的連結或類似連結進入的任何人都會自動將促銷代碼添加到他們的行程中，並且他們會看到自動應用的折扣價。

## 一體化

這`Integration` 標籤適用於希望更好地控制 Wink 上的預訂分配方式的飯店、整合商和旅行社。

### 通知

這使得飯店和整合商可以選擇讓 Wink 在預訂發生時通知所有相關方。
這包括通知頻道經理。

預設情況下，這些選項都是啟用的。整合商可能想要停用以下一個或多個功能的用例包括：

* 旅行社在 API 層級進行集成，並希望發送自己的確認電子郵件。
* 酒店正在測試其集成，並且不想打擾他們的銷售人員。
* 飯店並不關心電子郵件，只想透過 PMS 獲得通知。

### 電子郵件標誌

對於希望使用我們的電子郵件通知服務的整合商，他們可以自訂電子郵件，使其看起來像是來自他們。

：：：筆記
建議尺寸為 300px 寬 x 75px 高
:::

## 飯店專用

根據`Hotel-specific` 選項卡是讓飯店控制在我們的預訂引擎的登入頁面上顯示的內容以及在所要求的行程沒有庫存時顯示的內容的選項。

* **排名**：尚未獲得任何有意義的排名的酒店可能會選擇停用該功能。

：：：筆記
該酒店無法控制聯屬酒店如何選擇顯示酒店登陸頁面......這是兩種不同的配置。
:::

* **可用性**：讓飯店選擇是否繼續顯示庫存，即使沒有可用庫存。

：：：筆記
在大多數情況下，該屬性會希望讓使用者能夠點擊`Book` 按鈕繼續進入他們的登陸頁面，他們可以在那裡更改行程並嘗試找到可用的日期。
當您想要阻止使用者繼續時，請啟用此開關。
:::

## 如何使用

在大多數情況下，你會選擇`Customization` 應用於您的連結、卡片、地圖和網格。

以下是您可能需要手動套用配置的情況：

* **手動操作** 您想直接分享我們預訂引擎的鏈接，而不是使用我們的[可共享連結](/studio/shareable-links) 功能，並套用您的自訂配置。
* **WordPress 外掛** 將配置新增至您的 Wink[WordPress外掛](/developers/wordpress) `Settings`。
* **自訂集成** 使用我們的[wink-app-loader](/developers/web-components/#application-loader) Web 元件。

### 手動操作

這是一個簡單範例，說明如何將您的設定 ID 及其所有設定套用到我們的主要預訂引擎 URL。

<h4 align="center">[https://ota.wink.travel?configurationId=MY-UNIQUE-ID](https://ota.wink.travel?configurationId=MY-UNIQUE-ID)</h4>


：：：筆記
當使用者點擊此連結時，您也會設定您的聯盟歸屬 cookie。
:::

### WordPress 外掛

導航至[Wink WordPress 插件](/developers/wordpress) 了解更多。

### 自訂集成

使用下列方式控制頁面上所有嵌入的 Wink Web 元件[wink-app-loader](/developers/web-components/#application-loader) Web 元件。

下面是一個使用`MY-UNIQUE-ID` 作為配置 ID。

```html {11}
<html>
  <head>
    <link rel="stylesheet" href="https://elements.wink.travel/styles.css"></link>
  </head>
  <body>

    <script src="https://elements.wink.travel/elements.js" type="module" defer></script>

    <wink-app-loader
      client-id="YOUR CLIENT ID GOES HERE"
      configuration-id="MY-UNIQUE-ID"
    ></wink-app-loader>
  </body>
</html>
```
