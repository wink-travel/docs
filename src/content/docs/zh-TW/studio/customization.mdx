在 `Customization` 頁面中，點擊您想操作的自訂化項目下方的 `Actions` 按鈕，然後點選 `Update`。
*當您建立帳號時，系統已提供一個預設項目。此項目無法刪除。*

這將帶您進入自訂化表單，您會看到 5 到 7 個標籤頁（*視您的帳號類型而定*）。

## 一般設定

以下是在 `General` 標籤頁下可用的選項。

- **名稱** 為您的自訂化命名，方便記憶。*例如：我的旅遊部落格*
- **顯示貨幣** 選擇使用者初次看到的貨幣，會顯示在可預訂旅遊庫存旁。*例如：美元（USD）*
- **顯示語言** 選擇使用者初次遇到的語言。*例如：英文*
- **附加至頁面標題** 您可以控制頁面標題（*瀏覽器分頁上看到的標題*）中附加的內容。此功能主要用於我們的訂房引擎。*例如：當您瀏覽 https://ota.wink.travel 時，預設會看到 `Home | Traveliko - Hotels with Benefits`*。您可以更改 '|' 管道符號後的所有內容。

## 主題

`Theme` 標籤頁讓您自訂我們的 [訂房引擎](/booking-engine/in-a-nutshell) 網站、 [訂房引擎](/booking-engine/in-a-nutshell) 網站以及所有 [訂房引擎](/booking-engine/in-a-nutshell) 的外觀與感受。

### 配置

此區段讓您配置設計與卡片佈局。目前僅有一種設計：Wink。

卡片佈局控制您希望使用者看到水平或垂直排列的交易卡片。資訊內容相同。

:::note
在行動裝置上查看我們的交易卡片時，卡片佈局沒有差異。但在 iPad mini 及更大解析度裝置上，您可以看到差異。
:::

:::tip[專家提示]
如果您的旅遊庫存（例如房型）少於 4 項，建議使用水平佈局……看起來會更好。
:::

### 顏色

顏色控制使用者看到我們任何 UI 元件和網站的色彩。用它來搭配您現有的配色方案（如果有的話）。

我們已準備大量預設色彩供您試用。預設主題會改變下方表單欄位的顏色。您也可以完全使用自己的顏色。

## 分析

我們內建支援 GA4 分析。如果您想追蹤完整的強化電子商務轉換路徑，請準備好您的測量 ID，並在 `Analytics` 標籤頁的欄位中輸入。

<Aside type="caution" title="提醒">
  請勿忘將 `i.trvl.as`、`book.wink.travel` 和 `ota.wink.travel` 加入您的 GA4 產物串流中。
</Aside>

## 地圖

若要透過我們的 [WordPress 外掛](/developers/wordpress) 或 [WordPress 外掛](/developers/wordpress) 將地圖嵌入您的網站，您需要提供自己的 Google Maps API 金鑰。

:::note
  [在此建立 Google Maps API 金鑰](https://console.developers.google.com/projectselector2/apis/credentials?supportedpurview=project)
:::

產生金鑰並為您的網站設定後，請在 `Analytics` 標籤頁輸入。

## 行程

`Itinerary` 標籤頁在確保所選庫存價格正確顯示方面非常重要。
由於旅遊庫存定價相當複雜，Wink 需要知道使用者的行程，才能顯示準確價格。

此區段讓您設定預設行程，以便在我們所有數位資產中顯示價格，例如：

- 我們帶有價格覆蓋的社群友善圖片。
- WinkLinks 的格子、卡片與連結。
- 網頁元件
- 訂房引擎

### 日期

您可以選擇設定靜態日期範圍或滾動日期範圍。

- 若您要針對特定日期發生的活動使用此自訂化，請選擇靜態日期範圍。
- 若您想要：
    1. 預設為今天加 1 晚（大多數 OTA 採用此方式），或
    2. 預設為未來某個應始終有庫存的日期，
  請選擇滾動日期範圍。

### 房間配置

預設行程設定為 2 位成人，因為這是大多數旅客的常見設定。
您可以輕鬆更改，並新增多間房間及多位入住者，包括成人與兒童。

### 促銷

促銷是一項方便的功能，能為您的使用者節省時間。如果某個物業提供了促銷代碼，您可以在此輸入並分享給朋友。
任何透過您的連結或類似方式進入的使用者，促銷代碼會自動加入其行程，並自動套用折扣價格。

## 整合

`Integration` 標籤頁適用於飯店、整合商及旅行社，讓他們能更精細控制 Wink 上的訂單分配方式。

### 通知

此功能讓飯店與整合商選擇是否在訂單發生時通知所有相關方。
包括通知頻道管理員。

預設這些選項皆為啟用。整合商可能想停用一項或多項功能的情境包括：

- 旅行社透過 API 層級整合，想自行發送確認郵件。
- 飯店正在測試整合，不想打擾銷售人員。
- 飯店不在意電子郵件，只想透過 PMS 收到通知。

### 電子郵件標誌

想使用我們電子郵件通知服務的整合商，可以自訂郵件外觀，使其看起來像是來自自己。

:::note
  建議尺寸為寬 300px x 高 75px
:::

## 飯店專屬

`Hotel-specific` 標籤頁提供飯店控制其在我們訂房引擎上的登陸頁面顯示內容，以及在無庫存時顯示的內容。

- **排名**：尚未取得任何有效排名的物業，可能會選擇停用此功能。

:::note
  物業無法控制聯盟如何顯示飯店登陸頁……這是兩種不同的設定。
:::

- **可用性**：讓物業決定是否在無庫存時繼續顯示庫存。

:::note
  大多數物業會希望讓使用者能點擊 `Book` 按鈕，繼續前往其登陸頁面，修改行程並嘗試尋找可用日期。
  若您想阻止使用者繼續，請啟用此開關。
:::

## 使用方式

在大多數情況下，您會選擇要套用到連結、卡片、地圖和格子的 `Customization`。

以下是您可能想手動套用設定的時機：

- **手動覆寫** 您想直接分享訂房引擎連結，而不使用我們的 [可分享連結](/studio/shareable-links) 功能，並套用您的自訂設定。
- **WordPress 外掛** 在 Wink [WordPress 外掛](/developers/wordpress) 的 `設定` 中加入設定。
- **自訂整合** 使用我們的 [wink-app-loader](/developers/web-components/#application-loader) 網頁元件，手動套用至您的自訂整合。

### 手動覆寫

以下是如何將您的設定 ID（包含所有設定）套用到我們主要訂房引擎 URL 的簡單範例。

<h4 align="center"> [https://ota.wink.travel?configurationId=MY-UNIQUE-ID](https://ota.wink.travel?configurationId=MY-UNIQUE-ID) </h4>

:::note
  使用者點擊此連結時，您也會設定您的聯盟歸因 Cookie。
:::

### WordPress 外掛

請前往 [Wink WordPress 外掛](/developers/wordpress) 了解更多。

### 自訂整合

使用 [wink-app-loader](/developers/web-components/#application-loader) 網頁元件，控制頁面上所有嵌入的 Wink 網頁元件。

以下範例使用 `MY-UNIQUE-ID` 作為設定 ID。

```html {11}
<html>
  <head>
    <link rel="stylesheet" href="https://elements.wink.travel/styles.css"></link>
  </head>
  <body>

    <script src="https://elements.wink.travel/elements.js" type="module" defer></script>

    <wink-app-loader
      client-id="YOUR CLIENT ID GOES HERE"
      configuration-id="MY-UNIQUE-ID"
    ></wink-app-loader>
  </body>
</html>
```
