在 `Customization` 頁面中，點擊您想操作的自訂化項目下方的 `Actions` 按鈕，然後點選 `Update`。
*當您建立帳號時，系統已提供一個預設項目。此項目無法刪除。*

這將帶您進入自訂化表單，您會看到 5 到 7 個標籤頁（*視您的帳號類型而定*）。

## General

以下是 `General` 標籤頁下可用的選項。

- **Name** 為您的自訂化命名，方便記憶。*例如：我的旅遊部落格*
- **Display currency** 選擇使用者初次看到的貨幣，會顯示在可預訂的旅遊庫存旁。*例如：USD*
- **Display language** 選擇使用者初次遇到的語言。*例如：English*
- **Append to page title** 您可以控制頁面標題（*瀏覽器分頁上看到的標題*）後方要加上的內容。此功能主要用於我們的訂房引擎。*例如：若您瀏覽 https://ota.wink.travel，預設會看到 `Home | Traveliko - Hotels with Benefits`*。您可以更改 '|' 管道符號後的所有內容。

## Theme

`Theme` 標籤頁讓您自訂我們的 [booking engine](/zh-TW/booking-engine/in-a-nutshell) 網站、[WinkLinks](/zh-TW/link-manager/wink-links) 網站及所有 [Web Components](/zh-TW/developers/web-components) 的外觀與感受。

### Configuration

此區段讓您配置設計與卡片佈局。目前僅有一種設計：Wink。

卡片佈局控制您希望使用者看到水平或垂直排列的交易卡片。資訊內容相同。

:::note
在行動裝置上查看我們的交易卡片時，卡片佈局沒有差異。但在 iPad mini 及更大解析度裝置上，您可以看到差異。
:::

:::tip[專家提示]
如果您的旅遊庫存（例如房型）少於 4 項，建議使用水平佈局……看起來會更好。
:::

### Colors

顏色控制使用者看到我們任何 UI 元件和網站的色彩。用它來配合您現有的配色方案（如果有的話）。

我們已準備大量預設色彩供您嘗試。預設主題會改變下方表單欄位的顏色。您也可以完全使用自己的顏色。

## Analytics

我們內建支援 GA4 分析。如果您想追蹤完整的強化電子商務轉換路徑，請準備好您的測量 ID，並在 `Analytics` 標籤頁的欄位中輸入。

<Aside type="caution" title="提醒">
  請勿忘記將 `i.trvl.as`、`book.wink.travel` 和 `ota.wink.travel` 加入您的 GA4 產物串流中。
</Aside>

## Maps

若要透過我們的 [WordPress 外掛](/zh-TW/developers/wordpress) 或 [Web Components](/zh-TW/developers/web-components) 將地圖嵌入您的網站，您需要提供自己的 Google Maps API 金鑰。

:::note
  [在此建立 Google Maps API 金鑰](https://console.developers.google.com/projectselector2/apis/credentials?supportedpurview=project)
:::

產生金鑰並為您的網站設定後，請在 `Analytics` 標籤頁輸入。

## Itinerary

`Itinerary` 標籤頁在確保所選庫存價格正確顯示方面最為重要。
由於旅遊庫存定價相當複雜，Wink 需要知道使用者的行程，才能顯示準確價格。

此區段讓您設定預設行程，以便在我們所有數位資產中顯示價格，例如：

- 我們帶有價格覆蓋的社群友善圖片。
- WinkLinks 的格子、卡片與連結。
- Web Components
- 訂房引擎

### Dates

您可以選擇設定靜態日期範圍或滾動日期範圍。

- 若您要為特定日期的活動使用此自訂化，請選擇靜態日期範圍。
- 若您想要：
    1. 預設為今天加 1 晚（大多數 OTA 採用此方式），或
    2. 預設為未來某個應始終有庫存的日期，
  請選擇滾動日期範圍。

### Room configuration

預設行程設定為 2 位成人，因為這是大多數旅客的常見設定。
您可以輕鬆更改，並新增多間房間及多位入住者，包括成人與兒童。

### Promotions

促銷是一項方便的功能，能為您的使用者節省時間。如果某個物業提供了您可以與朋友分享的促銷代碼，您可以選擇在此輸入。
任何透過您的連結或類似方式進入的使用者，促銷代碼會自動加入其行程，並自動套用折扣價格。

## Integration

`Integration` 標籤頁適用於飯店、整合商及旅行社，讓他們能更掌控 Wink 上的訂單分配方式。

### Notifications

此功能讓飯店與整合商可選擇在訂單發生時，讓 Wink 通知所有相關方。
包括通知渠道管理者。

預設這些選項皆已啟用。整合商可能想停用其中一項或多項功能的情境包括：

- 旅行社在 API 層級整合，想自行發送確認郵件。
- 飯店正在測試整合，不想打擾銷售人員。
- 飯店不在意電子郵件，只想透過 PMS 接收通知。

### E-mail logo

想使用我們電子郵件通知服務的整合商，可以自訂郵件外觀，使其看起來像是由他們發出。

:::note
  建議尺寸為寬 300px x 高 75px
:::

## Hotel-specific

`Hotel-specific` 標籤頁提供飯店控制其在我們訂房引擎上的登陸頁面顯示內容，以及在所請求行程無庫存時的顯示設定。

- **Rankings**：尚未取得任何有效排名的物業，可能會選擇停用此功能。

:::note
  物業無法控制聯盟如何顯示飯店登陸頁……這是兩種不同的設定。
:::

- **Availability**：讓物業選擇是否在無庫存時繼續顯示庫存。

:::note
  大多數情況下，物業會希望讓使用者能點擊 `Book` 按鈕，繼續前往其登陸頁面，修改行程並嘗試尋找可用日期。
  若您想阻止使用者繼續，請啟用此開關。
:::

## How to use

在大多數情況下，您會選擇要套用哪個 `Customization` 到您的連結、卡片、地圖和格子。

以下是您可能想手動套用設定的時機：

- **手動覆寫** 您想直接分享訂房引擎的連結，而不使用我們的 [Shareable Links](/zh-TW/studio/shareable-links) 功能，並套用您的自訂設定。
- **WordPress 外掛** 將設定加入您的 Wink [WordPress 外掛](/zh-TW/developers/wordpress) `設定`。
- **自訂整合** 使用我們的 [wink-app-loader](/zh-TW/developers/web-components/#application-loader) Web Component，手動套用到您的自訂整合器。

### Manual override

以下是如何將您的設定 ID（含所有設定）套用到我們主要訂房引擎 URL 的簡單範例。

<h4 align="center"> [https://ota.wink.travel?configurationId=MY-UNIQUE-ID](https://ota.wink.travel?configurationId=MY-UNIQUE-ID) </h4>

:::note
  使用者點擊此連結時，您也會設定您的聯盟歸因 cookie。
:::

### WordPress plugin

請前往 [Wink WordPress 外掛](/zh-TW/developers/wordpress) 了解更多。

### Custom integration

使用 [wink-app-loader](/zh-TW/developers/web-components/#application-loader) Web Component，控制頁面上所有嵌入的 Wink Web Components。

以下範例使用 `MY-UNIQUE-ID` 作為設定 ID。

```html {11}
<html>
  <head>
    <link rel="stylesheet" href="https://elements.wink.travel/styles.css"></link>
  </head>
  <body>

    <script src="https://elements.wink.travel/elements.js" type="module" defer></script>

    <wink-app-loader
      client-id="YOUR CLIENT ID GOES HERE"
      configuration-id="MY-UNIQUE-ID"
    ></wink-app-loader>
  </body>
</html>
```
