已儲存的搜尋功能正如其名稱所暗示的那樣。在搜尋旅行清單時，您可以選擇儲存搜尋查詢以供日後使用。

## 儲存您的搜尋查詢

當您開始搜尋庫存時（請參閱 [搜尋](/studio/search) )，您可以微調您的搜尋過濾器。
查詢表單底部有一個選項`Save search`。

<figure>
  <CloudinaryImage width="350" height="329" resize="scale" quality="best" alt="儲存搜尋" src="academy/save-search" errorImage="noimage_opaque_nyrtl0.png" />

  <figcaption>*顯示儲存搜尋查詢的連結*</figcaption>
</figure>


點擊此連結將使您有機會命名此搜尋（*例如我的魔法圖書館飯店*) 一些你以後會記得的事情。

<figure>
  <CloudinaryImage width="350" height="329" resize="scale" quality="best" alt="為搜尋命名" src="academy/name-save-search" errorImage="noimage_opaque_nyrtl0.png" />

  <figcaption>*為搜尋命名*</figcaption>
</figure>


給它命名並點擊`Save search` 按鈕。之後，您可以導航至`Inventory > Saves Searches` 從主導覽列查看您新建立的記錄。

## 列出操作

您儲存的搜尋可用於以網格或地圖的形式向您的使用者顯示旅行清單，並以您想要的方式顯示您的搜尋結果。

<figure>
  <CloudinaryImage width="700" height="939" resize="scale" quality="best" alt="已儲存的搜尋操作" src="academy/saved-search-actions" errorImage="noimage_opaque_nyrtl0.png" />

  <figcaption>*已儲存的搜尋查詢，未套用任何篩選器*</figcaption>
</figure>


每個已儲存的搜尋查詢都有可以執行的操作：

* 將搜尋查詢轉換為 [地圖](/studio/maps) 。
* 將搜尋查詢轉換為 [網格](/studio/grids) 。
* 刪除已儲存的搜尋。

## API

想要管理已儲存搜尋的開發者可以前往 [開發人員 > API > 瀏覽](/developers/apis/#browse-api) 。
