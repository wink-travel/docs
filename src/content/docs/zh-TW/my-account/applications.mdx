作為開發者，應用程式讓您能使用 Wink 的所有功能。您可以存取相同的資料，甚至擴充 Wink 自身的功能。
您可以選擇使用 Wink 的 OAuth2 伺服器來驗證您的旅遊網站，就像使用 Google 或 Facebook 一樣。
如此一來，您就能將 Wink 的功能擴展到您自己的網站或應用程式中。

:::note
當您[建立新帳號](/zh-TW/my-account/create-account)時，系統已經為您建立了第一個應用程式。我們會在某些訂房情境中使用該應用程式。

此應用程式無法重複使用。請依需求建立新的應用程式。
:::

<Steps>
 1. 從 [Accounts](https://my.wink.travel) 選擇您想操作的帳號。
 2. 點擊帳號卡片底部的 `Actions`，選擇 `Manage account`。
 3. 在接下來的頁面點選 `Applications` 分頁。
 4. 此區會列出您所有的應用程式。
 5. 選擇編輯現有應用程式或建立新的應用程式。
 </Steps>

## 建立應用程式

由於您無法存取我們為您建立的應用程式的 `secret key`，因此您需要自行建立應用程式。

:::caution
在建立應用程式前，您需要先在 Wink 建立聯盟帳號或在 TripPay 建立帳號，因為您的應用程式將綁定至特定帳號。
:::

建立應用程式的步驟如下：

<Steps>
1. 在應用程式頁面，點擊 `Create new application` 按鈕。
2. ***名稱*** 為您的應用程式命名。*例如 Cool App*
3. **實體名稱** 將您的應用程式連結至您現有的帳號之一。*例如 Cool Account*
4. **重新導向 URI** 至少需要一個重新導向 URI。此 URI 用於驗證成功驗證後的重新導向網域。*例如 https://www.cool-site.com*
5. 點擊 `Save` 按鈕繼續。
</Steps>

系統會將您導回應用程式列表。您會在列表中看到新建立的應用程式，且只會顯示一次應用程式的密鑰。請務必複製並妥善保存。

## 更新應用程式

如果您的重新導向 URI 有變更，或想新增或移除 URI，您需要更新應用程式。

更新現有應用程式的步驟如下：

<Steps>
1. 點擊您想更新的應用程式的 `Actions` 連結。
2. 點擊 Actions 下的 `Update` 按鈕。
3. 進行應用程式的修改。
4. 點擊 `Save` 按鈕繼續。
</Steps>

系統會將您導回應用程式列表。

## 移除應用程式

:::caution
請勿刪除我們自動為您建立的應用程式，以確保您帳號的所有功能正常運作。
:::

如果您不再需要某個應用程式，可以將其移除。

移除應用程式的步驟如下：

<Steps>
1. 點擊您想移除的應用程式的 `Actions` 連結。
2. 點擊 Actions 下的 `Remove` 按鈕。
3. 點擊 `OK` 按鈕確認刪除。
</Steps>

系統會將您導回應用程式列表。
