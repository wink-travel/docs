應用程式讓您作為開發者能使用 Wink 的所有功能。您可以存取所有相同的資料，甚至擴充 Wink 自身的功能。
您可以選擇使用 Wink 的 OAuth2 伺服器來驗證您的旅遊網站，就像使用 Google 或 Facebook 一樣。
這樣您就能將 Wink 的功能擴展到您自己的網站或應用程式中。

:::note
當您[建立新帳號](/zh-TW/my-account/create-account)時，系統已經為您建立了第一個應用程式。我們會在某些訂房情境中使用該應用程式。

您無法重複使用此應用程式。請根據需要建立新的應用程式。
:::

 1. 從[帳號](https://my.wink.travel)中選擇您想操作的帳號。
 2. 點擊帳號卡片底部的 `操作`，選擇 `管理帳號`。

 4. 此區域會列出您所有的應用程式。
 5. 選擇編輯現有應用程式或建立新的應用程式。

## 建立應用程式

由於您無法存取我們為您建立的應用程式的 `密鑰`，您需要自行建立應用程式。

:::caution
您必須先在 Wink 建立聯盟帳號或在 TripPay 建立帳號，才能建立應用程式，因為應用程式會綁定到特定帳號。
:::

### 用戶端類型

建立應用程式時，您需要選擇用戶端類型，決定應用程式如何驗證：

- **SPA**（單頁應用程式）— 適用於使用 OAuth2 授權碼流程搭配 PKCE 的瀏覽器前端應用程式。這是網頁應用程式最常見的類型。
- **MACHINE_2_MACHINE** — 適用於伺服器對伺服器的整合，直接使用用戶端憑證驗證。當您的後端服務需要在無使用者介入的情況下呼叫 Wink API 時使用。

以下是建立應用程式的步驟：

<Steps>
1. 在應用程式頁面，點擊 `建立新應用程式` 按鈕。
2. ***名稱*** 為您的應用程式命名。*例如 Cool App*
3. **用戶端類型** 選擇 `SPA` 用於網頁應用程式，或 `MACHINE_2_MACHINE` 用於伺服器整合。
4. **實體名稱** 將您的應用程式連結到您現有的帳號之一。*例如 Cool Account*
5. **重新導向 URI** 至少需要一個重新導向 URI。用於驗證成功驗證後的重新導向網域。*例如 https://www.cool-site.com*
6. **登出後重新導向 URI** *(選填)* 使用者登出後可被導向的 URL，確保使用者在會話結束後會停留在已知頁面。
7. 點擊 `儲存` 按鈕繼續。
</Steps>

系統會將您導回應用程式列表。您會在列表中看到新建立的應用程式，且只會顯示一次應用程式密鑰。請務必複製並妥善保存。

## 更新應用程式

如果您的重新導向 URI 有變更，或想新增或移除 URI，您需要更新應用程式。

更新現有應用程式的步驟如下：

<Steps>
1. 點擊您想更新的應用程式的 `操作` 連結。
2. 點擊操作下的 `更新` 按鈕。
3. 進行應用程式的修改。
4. 點擊 `儲存` 按鈕繼續。
</Steps>

系統會將您導回應用程式列表。

## 移除應用程式

:::caution
請勿刪除我們自動為您建立的應用程式，以確保您的帳號所有功能正常運作。
:::

如果您不再需要某個應用程式，可以將其移除。

移除應用程式的步驟如下：

<Steps>
1. 點擊您想移除的應用程式的 `操作` 連結。
2. 點擊操作下的 `移除` 按鈕。
3. 點擊 `確定` 按鈕以確認刪除。
</Steps>

系統會將您導回應用程式列表。
