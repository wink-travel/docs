并非每天的价格都相同。活动、节假日、淡季和临时空档都需要不同的定价。本指南将向您展示如何使用特殊价格和价格计划限制来应对这些情况。

## 活动定价

当会议、节日或本地活动带动需求时，您会希望提高这些日期的价格。

<Steps>
1. 在您的 Extranet 中进入 `Monetize > Special Rates`。
2. 点击 `Create a new special rate`。
3. 选择要覆盖的主价格和房型。
4. 设置覆盖活动期间的日期范围。
5. 输入调整后的（更高的）每日价格。
6. 点击 `Save`。
</Steps>

特殊价格仅在这些日期覆盖主价格。活动窗口之外，您的常规定价保持不变。

## 季节性调整

对于较长时间段（旺季、肩季、淡季），您可以：

- **使用特殊价格** — 为每个季节创建调整后的覆盖价格。
- **更新主价格** — 直接调整这些期间的基础价格。

特殊价格更适合临时调整，而更新主价格适合可预测的年度模式。

## 临时价格

为了填补临近入住日期的未售房间：

<Steps>
1. 创建一个带有提前预订限制的新**价格计划**（例如，必须在入住前不超过3天预订）。
2. 配合提供目标日期折扣的**特殊价格**。
3. 该价格计划只会向在临时窗口内预订的旅客显示。
</Steps>

## 停用日期

要阻止某些日期使用特定价格计划（例如，旺季期间不提供促销价格）：

<Steps>
1. 进入 `Monetize > Rate Plans` 并选择价格计划。
2. 导航到**限制**部分。
3. 添加**销售日期**限制，限制价格可预订的时间。
4. 或使用**入住日期**限制，限制价格覆盖的入住时间。
</Steps>

例如，您可以限制“早鸟”价格计划在12月（旺季）不可预订，而您的常规价格计划仍然开放。

## 组合限制

限制可以叠加以实现精确控制：

| 限制类型 | 控制内容 |
| -- | -- |
| **销售日期** | 价格可预订的时间 |
| **入住日期** | 价格覆盖的入住日期 |
| **可用天数** | 价格显示的星期几 |
| **到店日** | 客人必须到店的日期 |
| **离店日** | 客人必须离店的日期 |
| **必需天数** | 客人必须至少入住这些天中的一天 |
| **提前预订** | 预订必须提前多久完成 |
| **入住时长** | 最短和最长入住夜数 |
