通过将您的 Google Analytics 4 属性连接到 Wink，您的预订事件——从房间浏览到完成购买——会自动转发到您自己的 GA4 仪表板，与 Wink 自身的分析数据并行显示。无需编码，也无需您这边进行技术设置。

<Aside type="tip">
您只需要一个 GA4 测量 ID。输入到 Wink 门户后，您的分析数据即可立即生效。
</Aside>

---

## 工作原理

```
您在 Wink 门户中输入 G-XXXXXXXXXX
       ↓
Wink 将您的预订事件转发到您的 GA4 属性
       ↓
您登录 analytics.google.com，查看您的预订数据
```

---

## 您需要做什么

<Steps>

1. **创建 GA4 属性**

   访问 [analytics.google.com](https://analytics.google.com)：管理 → 创建属性。设置您的时区和货币以匹配您的市场。

2. **创建网站数据流**

   管理 → 数据流 → 添加流 → 网站。输入 `wink.travel` 作为网址。系统会生成您的**测量 ID**（`G-XXXXXXXXXX`）。复制该 ID。

3. **在 Wink 门户中输入测量 ID**

   登录 [my.wink.travel](https://my.wink.travel)，选择您的账户，进入**自定义**标签页。粘贴您的测量 ID 并保存。完成后，您的 GA4 集成即已激活。

4. **将 `purchase` 标记为关键事件**

   返回您的 GA4 属性：管理 → 事件 → 找到 `purchase` → 切换 **标记为关键事件**。这样完成的预订会出现在您的转化报告中。

5. **查看您的数据**

   登录 [analytics.google.com](https://analytics.google.com)，您将看到通过 Wink 自动跟踪的预订、收入和客人漏斗数据。

</Steps>

<Aside type="note">
此集成无需您在自己的网站上安装任何跟踪代码。Wink 会处理所有内容。您的测量 ID 只是预订数据发送的目标地址。
</Aside>

---

## 您可以跟踪的内容

连接后，您的 GA4 属性将接收完整的增强型电子商务事件套件：

### 发现阶段

| 事件 | 含义 |
|---|---|
| `view_item_list` | 客人在搜索或分类列表中看到您的房源 |
| `select_item` | 客人在列表中点击了您的房源 |
| `view_item` | 客人查看了您的酒店详情页 |

### 考虑阶段

| 事件 | 含义 |
|---|---|
| `view_promotion` | 客人看到您的促销优惠 |
| `select_promotion` | 客人点击了促销优惠 |
| `add_to_wishlist` | 客人将您的房源添加到心愿单 |

### 预订漏斗

| 事件 | 含义 |
|---|---|
| `add_to_cart` | 客人选择了房间或套餐 |
| `view_cart` | 客人查看了预订摘要 |
| `remove_from_cart` | 客人移除了房间或套餐 |
| `begin_checkout` | 客人开始结账流程 |
| `add_payment_info` | 客人输入了支付信息 |
| `add_shipping_info` | 客人输入了入住信息 |
| `purchase` | 客人完成了预订 |
| `refund` | 预订被取消并退款 |

所有事件均包含相关维度，如收入、货币、房型和预订价值——为您提供从首次印象到完成预订的完整客人旅程视图。要解锁更多房源和房型级别的维度，如 `city_name`、`cancellation_policy`、`bed_type` 等，请参见 [GA4 中的自定义维度](/zh-CN/guides/gtm-dual-tagging#custom-dimensions-in-ga4)。

## 跨域跟踪（高级）

上述标准集成跟踪所有在 Wink 内发生的操作。但如果您想跟踪客人从自己的网站一直到在 Wink 完成预订的全过程——并正确归因原始流量来源（Google Ads、邮件活动、自然搜索等）——则需要在双方进行额外配置。

<Aside type="caution">
本节仅适用于您在自己网站上安装了 GA4 的情况。如果您未在网站上运行 GA4，以上标准集成已足够。
</Aside>

### 重要性说明

没有跨域跟踪时，先访问您网站后在 Wink 预订的客人在 GA4 中会被视为**直接访问**，与之前的访问无关联。任何导致预订的 Google Ads 点击、邮件活动或自然搜索都不会被归因。

配置跨域跟踪后，您将看到完整的客人旅程：

```
google.com（付费广告）
       ↓
您的网站（客人浏览房间）
       ↓
wink.travel（客人选择房间并预订）
       ↓
购买正确归因于最初的 Google Ads 点击
```

### 您需要做的操作

配置您的 GA4 属性，将 `wink.travel` 识别为您自己域名的一部分，而非外部推荐来源。这样 GA4 会将两个会话合并为一个连续的客人旅程。

<Steps>
1. GA4 管理 → 数据流 → 选择您的数据流
2. 配置标签设置 → **配置您的域名**
3. 添加 `wink.travel` 作为关联域名
4. 保存
</Steps>

### Wink 需要做的操作

联系 [Wink 支持](https://my.wink.travel)，提供您的网站域名（例如 `yourhotel.com`）。Wink 会将您的域名添加到 Wink GA4 属性的推荐排除列表中，防止您的网站被记录为推荐来源，否则会覆盖客人的原始归因并启动新会话。

<Aside type="tip">
上述标准集成对大多数房源来说已足够，能让您立即看到 Wink 上的预订和收入情况。跨域跟踪是一个高级选项，适合在您熟悉 GA4 数据并希望获得更深入归因洞察时设置。
</Aside>
