自定义控制您的库存如何向旅客展示——主题颜色、默认货币、语言、行程等。它们适用于可分享链接、嵌入卡片、网格、地图和预订引擎。

## 什么是自定义？

自定义是一个配置档案，定义了：

- **显示货币** — 价格显示的货币。
- **显示语言** — 内容的默认语言。
- **默认行程** — 预填的入住/退房日期和客人数。
- **主题** — 应用于嵌入组件的颜色和品牌。
- **配置** — 库存的展示方式（布局、排序）。

您可以为不同市场或用例创建多个自定义。

## 创建自定义

<Steps>
1. 访问 https://studio.wink.travel （或您的 https://my.wink.travel 账户）。
2. 从侧边栏导航到 `Customization`。
3. 点击 `Create a new customization`。
4. 给它一个描述性名称（例如，“Europe - EUR”，“Japan Market”，“Default”）。
5. 设置显示货币、语言和默认行程。
6. 配置主题颜色以匹配您的品牌。
7. 点击 `Save`。
</Steps>

## 自定义的使用场景

创建后，自定义会在平台各处被引用：

| 功能 | 自定义的应用方式 |
| -- | -- |
| **可分享链接** | 链接使用自定义的货币、语言和行程作为默认值 |
| **嵌入卡片** | 卡片以您的主题颜色和显示设置渲染 |
| **网格和地图** | 库存网格和地图标记遵循自定义设置 |
| **预订引擎** | 整个预订流程使用自定义的设置 |
| **社交帖子** | AI生成的帖子可使用特定自定义显示价格 |

## 多个自定义

常见做法是为每个目标市场创建一个自定义：

- **“US Market”** — USD，英语，默认2晚住宿。
- **“Europe”** — EUR，英语，默认3晚住宿。
- **“Japan”** — JPY，日语，默认1晚住宿。

当您创建可分享链接或嵌入卡片时，选择使用哪个自定义。这意味着同一物业可以根据受众不同而呈现不同的方式。

## 通过 URL 覆盖

您还可以通过可分享链接的 URL 参数覆盖自定义默认值：

- `l=es` — 将语言覆盖为西班牙语。
- `c=EUR` — 将货币覆盖为 EUR。
- `sd=2024-08-24&n=2` — 将行程覆盖为特定日期和夜数。

详见 [The Book Now Button](/zh-CN/guides/the-book-now-button) 获取完整的 URL 参数列表。
