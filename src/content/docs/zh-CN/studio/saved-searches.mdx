已保存搜索功能顾名思义。在搜索旅行清单时，您可以选择保存搜索查询，以供日后使用。

## 保存您的搜索查询

当您开始搜索库存时（请参阅[搜索](/studio/search)），您可以微调搜索筛选条件。
在查询表单的底部，您可以选择`Save search`。

<figure>
  <CloudinaryImage width="350" height="329" resize="scale" quality="best" alt="保存搜索" src="academy/save-search" errorImage="noimage_opaque_nyrtl0.png" />

  <figcaption>*显示保存搜索查询的链接*</figcaption>
</figure>


点击此链接将使您有机会命名此搜索（*例如我的魔法酒店图书馆*) 一些你以后会记住的事情。

<figure>
  <CloudinaryImage width="350" height="329" resize="scale" quality="best" alt="为搜索命名" src="academy/name-save-search" errorImage="noimage_opaque_nyrtl0.png" />

  <figcaption>*为搜索命名*</figcaption>
</figure>


给它命名并点击`Save search`按钮。之后，您可以导航到`Inventory > Saves Searches`从主导航栏查看您新创建的记录。

## 列出操作

您保存的搜索可用于以网格或地图的形式向您的用户展示旅行清单，并以您想要的方式显示您的搜索结果。

<figure>
  <CloudinaryImage width="700" height="939" resize="scale" quality="best" alt="已保存的搜索操作" src="academy/saved-search-actions" errorImage="noimage_opaque_nyrtl0.png" />

  <figcaption>*已保存的搜索查询，未应用任何过滤器*</figcaption>
</figure>


每个已保存的搜索查询都有可以对其执行的操作：

* 将搜索查询转换为[地图](/studio/maps)。
* 将搜索查询转换为[网格](/studio/grids)。
* 删除已保存的搜索。

## API

想要管理已保存搜索的开发者可以前往[开发人员 > API > 浏览](/developers/apis/#browse-api)。
