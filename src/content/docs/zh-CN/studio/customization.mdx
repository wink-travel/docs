在 `Customization` 页面，点击您想操作的自定义项下的 `Actions` 按钮，然后点击 `Update`。
*当您创建账户时，系统会为您提供一个默认条目。此条目无法删除。*

这将带您进入自定义表单，您可以看到 5 到 7 个标签页（*取决于您的账户类型*）。

## General

以下是在 `General` 标签页下可用的选项。

- **名称** 给您的自定义命名，方便记忆。*例如：为我的旅游博客*
- **显示货币** 选择用户初次看到的货币，显示在可预订的旅游库存旁边。*例如：USD*
- **显示语言** 选择用户初次遇到的语言。*例如：英语*
- **附加到页面标题** 您可以控制添加到页面标题（*浏览器标签页中看到的标题*）的内容。这主要用于我们的预订引擎。*例如：如果您访问 https://ota.wink.travel，默认情况下，您会看到 `Home | Traveliko - Hotels with Benefits`*。您可以更改“|”管道符号后的所有内容。

## Theme

`Theme` 标签页允许您自定义我们的[预订引擎](/zh-CN/booking-engine/in-a-nutshell)网站、[WinkLinks](/zh-CN/link-manager/wink-links)网站以及所有[Web 组件](/zh-CN/developers/web-components)的外观和感觉。

### Configuration

本节允许您配置设计和卡片布局。目前只有一种设计；即 Wink。

卡片布局控制您希望用户看到水平排列还是垂直排列的交易卡片。信息内容相同。

:::note
在移动设备上查看我们的交易卡片时，卡片布局没有区别。但在 iPad mini 及更大分辨率设备上可以看到差异。
:::

:::tip[专业提示]
如果列表中只有少于 4 个旅游库存（即房型），建议使用水平布局……看起来更美观。
:::

## Analytics

我们开箱即用支持 GA4 分析。如果您想跟踪整个增强型电商转化路径，从开始到结束，请准备好您的测量 ID，并在 `Analytics` 标签页下输入。

<Aside type="caution" title="提醒">
  别忘了将 `i.trvl.as`、`book.wink.travel` 和 `ota.wink.travel` 添加到您的 GA4 属性流中。
</Aside>

## Maps

为了通过我们的[WordPress 插件](/zh-CN/developers/wordpress)或[Web 组件](/zh-CN/developers/web-components)将地图嵌入您的网站，您需要提供自己的 Google Maps API 密钥。

:::note
  [在此创建 Google Maps API 密钥](https://console.developers.google.com/projectselector2/apis/credentials?supportedpurview=project)
:::

生成密钥并为您的网站配置后，请在 `Analytics` 标签页下输入。

## Itinerary

`Itinerary` 标签页可能是确保所选库存价格正确显示的最重要部分。
由于旅游库存定价较为复杂，Wink 需要知道用户的行程，才能显示准确价格。

本节允许您设置默认行程，以便在我们所有数字资产中显示价格，例如：

- 我们带价格叠加的社交友好图片。
- WinkLinks 网格、卡片和链接。
- Web 组件
- 预订引擎

### Dates

您可以选择设置静态日期范围或滚动日期范围。

- 如果您为特定日期的活动使用自定义，选择静态日期范围。
- 如果您想要：
    1. 默认设置为今天加 1 晚（大多数 OTA 采用此方式），或者
    2. 默认设置为未来某个始终有可用性的日期，
  请选择滚动日期范围。

### Room configuration

默认行程设置为 2 位成人，因为这是大多数旅行者的常见设置。
您可以轻松更改，并添加多个房间及多位入住者，包括成人和儿童。

### Promotions

促销是一个方便的功能，可以为您的用户节省时间。如果某个物业给了您一个可与朋友分享的促销代码，您可以选择在此输入。
通过您的链接或类似方式进入的任何人，都会自动将促销代码添加到行程中，并自动看到折扣价。

## Integration

`Integration` 标签页适用于希望更好控制 Wink 预订分发的酒店、集成商和旅行社。

### Notifications

此功能允许酒店和集成商在预订发生时，让 Wink 通知所有相关方。
包括通知渠道管理器。

默认情况下，这些选项均已启用。集成商可能想禁用其中一项或多项功能的场景包括：

- 旅行社通过 API 级别集成，想发送自己的确认邮件。
- 酒店正在测试集成，不想打扰销售人员。
- 酒店不关心邮件，只想通过 PMS 接收通知。

### E-mail logo

希望使用我们的邮件通知服务的集成商，可以自定义邮件，使其看起来像是他们自己发送的。

:::note
  推荐尺寸为宽 300px x 高 75px
:::

## Hotel-specific

`Hotel-specific` 标签页下的选项允许酒店控制其在我们预订引擎上的落地页显示内容，以及在所请求行程无库存时显示的内容。

- **排名**：尚未获得任何有效排名的物业，可能选择禁用此功能。

:::note
  物业无法控制联盟如何展示酒店落地页……这两个是不同的配置。
:::

- **可用性**：允许物业选择是否在无库存时继续显示库存。

:::note
  大多数情况下，物业希望用户能点击 `Book` 按钮，继续进入其落地页，在那里更改行程并尝试找到可用日期。
  当您想阻止用户继续时，请启用此开关。
:::

## How to use

在大多数场景中，您会选择应用哪个 `Customization` 到您的链接、卡片、地图和网格。

以下是您可能想手动应用配置的情况：

- **手动覆盖** 您想直接分享我们的预订引擎链接，而不使用我们的[可分享链接](/zh-CN/studio/shareable-links)功能，并应用您的自定义配置。
- **WordPress 插件** 在 Wink [WordPress 插件](/zh-CN/developers/wordpress)的 `Settings` 中添加配置。
- **自定义集成** 使用我们的 [wink-app-loader](/zh-CN/developers/web-components/#application-loader) Web 组件，手动应用到您的自定义集成。

### Manual override

以下是如何将您的配置 ID（包含所有设置）应用到我们主预订引擎 URL 的简单示例。

<h4 align="center"> [https://ota.wink.travel?configurationId=MY-UNIQUE-ID](https://ota.wink.travel?configurationId=MY-UNIQUE-ID) </h4>

:::note
  当用户点击此链接时，您也会设置您的联盟归因 cookie。
:::

### WordPress plugin

请访问 [Wink WordPress 插件](/zh-CN/developers/wordpress) 了解更多。

### Custom integration

使用 [wink-app-loader](/zh-CN/developers/web-components/#application-loader) Web 组件控制页面上所有嵌入的 Wink Web 组件。

下面是一个使用 `MY-UNIQUE-ID` 作为配置 ID 的示例。

```html {11}
<html>
  <head>
    <link rel="stylesheet" href="https://elements.wink.travel/styles.css"></link>
  </head>
  <body>

    <script src="https://elements.wink.travel/elements.js" type="module" defer></script>

    <wink-app-loader
      client-id="YOUR CLIENT ID GOES HERE"
      configuration-id="MY-UNIQUE-ID"
    ></wink-app-loader>
  </body>
</html>
```
