自定义是一项强大的功能，您可以用它来控制与 Wink 集成的各个方面，而无需编写任何代码。您出售的每一件商品都可以自定义。您可以添加或删除功能、更改配色方案等等。

在`Customization`页面上，点击`Actions`按钮，然后单击`Update`。
*创建帐户时，您已有一个默认条目可用。此条目无法删除。*

这将带您进入自定义表单，您可以在其中看到 5 - 7 个选项卡（*取决于您的帐户类型*）。

## 一般的

以下是您可用的选项`General`选项卡。

* **姓名**为您的定制提供一个容易记住的名称。*例如，我的旅行博客*
* **显示货币**选择用户将在可预订旅行库存旁边看到的初始货币。*例如美元*
* **显示语言**选择用户最初遇到的语言。*例如英语*
* **附加到页面标题**您可以控制添加到页面标题的内容（*您在浏览器标签中看到的标题*）。这主要用于我们的预订引擎。*例如，如果您导航至 https://ota.wink.travel，默认情况下，您将看到`Home | Traveliko - Hotels with Benefits`*。您可以更改“|”管道后面的所有内容。

## 主题

这 `Theme`选项卡可让您自定义我们的外观和感觉[预订引擎](/booking-engine/in-a-nutshell) 网站， [WinkLinks](/link-manager/wink-links)网站和我们所有的[Web 组件](/developers/web-components)。

### 配置

此部分允许您配置设计和卡片布局。目前只有一种设计：Wink。

卡片布局决定了您希望用户看到水平对齐还是垂直对齐的交易卡。信息是相同的。

:::注意
在移动设备上查看我们的交易卡时，卡片布局没有区别。但是，在 iPad mini 和更大的分辨率下，您可以看到差异。
:::

:::tip\[专业提示]
如果列表中的旅行房源（例如房型）少于 4 个，请使用水平布局……这样看起来更美观。
:::

## 行程

这 `Itinerary`在确保显示所选库存价格方面，“价格标签”可能是最有趣的。
由于旅行库存定价相当复杂，Wink 需要了解用户的行程才能显示准确的价格。

此部分可让您设置默认行程以显示我们所有数字房地产的价格，例如：

* 我们的社交友好型图片带有价格叠加。
* WinkLinks 网格、卡片和链接。
* Web 组件
* 预订引擎

### 日期

您可以选择设置静态日期范围还是滚动日期范围。

* 如果您要对给定日期发生的特定事件进行自定义，请选择静态日期范围。
* 如果您想要执行以下操作之一，请选择滚动日期范围：
  1. 默认为今天 + 1 晚，大多数 OTA 都这样做。
  2. 默认为将来始终可用的日期。

### 房间配置

默认情况下，行程设置为 2 名成人，因为这是大多数旅行者最常见的设置。
您可以轻松更改此设置，并添加多个可容纳多名成人和儿童的房间。

### 促销

促销是一项实用功能，可以节省用户的时间。如果酒店为您提供了可分享给好友的促销代码，您可以选择在此处输入。
任何通过您的链接或类似方式进入的客人都会自动将促销代码添加到行程中，并自动看到折扣价。

## 一体化

这 `Integration`标签适用于希望更好地控制 Wink 上的预订分配方式的酒店、集成商和旅行社。

### 通知

这使得酒店和集成商可以选择让 Wink 在发生预订时通知所有相关方。
这包括通知渠道经理。

默认情况下，这些选项均处于启用状态。集成商可能需要禁用以下一个或多个功能：

* 旅行社在 API 级别进行集成，并希望发送自己的确认电子邮件。
* 酒店正在测试其集成，并且不想打扰他们的销售人员。
* 酒店并不关心电子邮件，只想通过 PMS 获得通知。

### 电子邮件徽标

对于希望使用我们的电子邮件通知服务的集成商，他们可以自定义电子邮件，使其看起来像是来自他们。

:::注意
建议尺寸为宽 300px x 高 75px
:::

## 酒店专用

根据`Hotel-specific`选项卡是让酒店控制在我们的预订引擎的登录页面上显示的内容以及在所请求的行程没有库存时显示的内容的选项。

* **排名**：尚未获得任何有意义的排名的酒店可能会选择禁用该功能。

:::注意
酒店无法控制联盟会员如何显示酒店落地页……这是两种不同的配置。
:::

* **可用性**：让酒店选择是否继续显示库存，即使没有可用库存。

:::注意
在大多数情况下，该属性会允许用户点击`Book`按钮，让他们继续前往着陆页，在那里他们可以更改行程并尝试查找可用的日期。
如果您想阻止用户继续，请启用此开关。
:::

## 如何使用

在大多数情况下，你会选择`Customization`应用于您的链接、卡片、地图和网格。

以下是您可能需要手动应用配置的情况：

* **手动操作**您想直接分享我们预订引擎的链接，而不是使用我们的[可共享链接](/studio/shareable-links)功能，并应用您的自定义配置。
* **WordPress 插件**将配置添加到您的 Wink[WordPress插件](/developers/wordpress) `Settings`。
* **自定义集成**使用我们的[wink-app-loader](/developers/web-components/#application-loader)Web 组件。

### 手动操作

这是一个简单示例，说明如何将您的配置 ID 及其所有设置应用到我们的主要预订引擎 URL。

<h4 align="center">[https://ota.wink.travel?configurationId=MY-UNIQUE-ID](https://ota.wink.travel?configurationId=MY-UNIQUE-ID)</h4>


:::注意
当用户点击此链接时，您还将设置您的联属归因 Cookie。
:::

### WordPress 插件

导航至[Wink WordPress 插件](/developers/wordpress)了解更多信息。

### 自定义集成

使用以下方式控制页面上所有嵌入的 Wink Web 组件[wink-app-loader](/developers/web-components/#application-loader)Web 组件。

下面是一个使用`MY-UNIQUE-ID`作为配置 ID。

```html {11}
<html>
  <head>
    <link rel="stylesheet" href="https://elements.wink.travel/styles.css"></link>
  </head>
  <body>

    <script src="https://elements.wink.travel/elements.js" type="module" defer></script>

    <wink-app-loader
      client-id="YOUR CLIENT ID GOES HERE"
      configuration-id="MY-UNIQUE-ID"
    ></wink-app-loader>
  </body>
</html>
```
