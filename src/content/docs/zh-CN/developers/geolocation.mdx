Wink 广泛使用地理定位来支持基于位置的搜索、自动货币和语言检测，以及地理库存过滤。

## GeoIP 检测

当用户访问平台时，Wink 会根据其 IP 地址自动检测其位置。这使得：

- **自动货币选择** — 以用户本地货币显示价格。
- **语言偏好** — 推荐用户可能使用的语言内容。
- **附近结果** — 优先显示靠近用户位置的库存。

## 空间搜索

Wink 支持使用四种空间查询类型进行地理搜索。这些查询支持平台上的基于地图的搜索、半径搜索和基于区域的过滤。

### 点

单个地理坐标（纬度，经度）。用于标记特定位置——例如，房产地址或用户当前位置。

```json
{
  "point": {
    "x": -73.9857,
    "y": 40.7484
  }
}
```

### 圆

一个中心点和半径。用于“搜索我附近”或基于半径的查询——例如，“时代广场10公里内的酒店”。

```json
{
  "point": {
    "x": -73.9857,
    "y": 40.7484
  },
  "radius": 10000
}
```

### 矩形

由西南角和东北角定义的边界框。用于基于地图的搜索——当用户平移和缩放地图时，当前可见区域定义了一个矩形查询。

```json
{
  "southWest": {
    "x": -74.0060,
    "y": 40.7128
  },
  "northEast": {
    "x": -73.9350,
    "y": 40.7831
  }
}
```

### 多边形

由一系列点定义的自定义形状。用于在不规则地理边界内搜索——例如，某个社区、城市区或地图上自定义绘制的区域。

```json
{
  "polygon": {
    "points": [
      { "x": -74.0060, "y": 40.7128 },
      { "x": -73.9350, "y": 40.7128 },
      { "x": -73.9350, "y": 40.7831 },
      { "x": -74.0060, "y": 40.7831 },
      { "x": -74.0060, "y": 40.7128 }
    ]
  }
}
```

:::note
多边形必须闭合——最后一个点必须与第一个点相同。
:::

## 使用场景

| 使用场景 | 形状类型 | 示例 |
|---|---|---|
| 搜索我附近 | 圆 | 我位置5公里内的酒店 |
| 地图视口搜索 | 矩形 | 当前地图视图内可见的房产 |
| 社区搜索 | 多边形 | 某个特定城市区的酒店 |
| 房产位置 | 点 | 房产列表的精确位置 |

## 面向开发者

空间查询参数被 Lookup API 和 Inventory API 端点接受。在搜索请求中传递相应的 GeoJSON 形状，以按位置过滤结果。

详情请参阅[API 文档](/zh-CN/developers/apis)了解端点特定信息。
