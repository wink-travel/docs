Webhooks 是一种让您接收并响应来自 Wink 和 TripPay 关于您账户的实时事件的方式。

## 示例

<Steps>
1. 在 Wink 上发生一次预订。
1. 触发 Wink 事件 `booking.created`。
2. 您的系统正在主动监听此事件。
3. 您的系统跟踪自己的分析数据，并使用此事件更新其时间序列数据库。
</Steps>

:::note
Wink webhook 与 TripPay webhook 不同。

- 如果您想监听 Wink 事件，可以从 [Extranet](https://extranet.wink.travel) 或 [Studio](https://studio.wink.travel) 创建 webhook。
- 如果您想监听 TripPay 事件，可以从 [Payment](https://pay.wink.travel) 创建 webhook。
:::

*在本示例中，我们将通过 [Studio](https://studio.wink.travel) 进行身份验证。*

<Steps>
1. 认证后，点击右上角的个人资料图标，会出现一个菜单。
2. 点击 `Webhooks` 链接。
3. 您将被重定向到您的 webhook 页面。
</Steps>

## 创建 webhook

:::caution
在创建 webhook 之前，您需要先在 Wink 创建一个分销商账户或在 TripPay 创建一个账户，因为您将把 webhook 绑定到特定账户。
:::

创建 webhook 的步骤如下：

<Steps>
1. 在 webhook 页面，点击 `Create new webhook` 按钮。
2. ***名称*** 给您的 webhook 命名。*例如 Wink Stalker*
3. ***启用*** 保持默认启用状态。
4. **实体名称** 将您的 webhook 关联到您已有的账户之一。*例如 Cool Account*
5. **事件** 选择您感兴趣的事件。*例如 `booking.created`*
6. **Webhook URL** 输入您用于接收事件的 URL。*例如 `https://my.system.com/webhook/wink`*
7. 点击 `Save` 按钮继续。
</Steps>

您将被重定向回 webhook 列表，您的新 webhook 已准备就绪。

## 更新 webhook

如果您想要：

- 禁用您的 webhook。
- 监听更多或更少的事件。
- 更改监听的 webhook URL。

您需要更新您的应用。

更新现有 webhook 的步骤如下：

<Steps>
1. 点击您想更新的 webhook 的 `Actions` 链接。
2. 点击 Actions 下的 `Update` 按钮。
3. 对 webhook 进行修改。
4. 点击 `Save` 按钮继续。
</Steps>

您将被重定向回 webhook 列表。

## 删除 webhook

如果您不再需要该 webhook，可以将其删除。

删除 webhook 的步骤如下：

<Steps>
1. 点击您想删除的 webhook 的 `Actions` 链接。
2. 点击 Actions 下的 `Remove` 按钮。
3. 点击 `OK` 按钮确认删除。
</Steps>

您将被重定向回 webhook 列表。
