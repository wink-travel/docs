## 概览
托管的 Wink MCP 服务器地址为：

```
https://docs.mcp.wink.travel/mcp
```

将兼容的 AI 客户端（VS Code、Cursor 及其他实现 MCP HTTP 传输的客户端）指向此端点，以便它们能够：
- 浏览产品和平台文档（`wink://docs/...`）
- 浏览实时 OpenAPI 规范和操作文档（`wink://api/...`）
- 对文档和实时 API 进行轻量级语义/关键词搜索

## 传输
使用主要的可流式 HTTP 端点：

```
https://docs.mcp.wink.travel/mcp
```

如果您的客户端仅支持 SSE，请使用：

```
https://docs.mcp.wink.travel/sse
```

### Claude Code
官方文档：[通过 MCP 连接 Claude Code 到工具](https://code.claude.com/docs/en/mcp)

Anthropic 推荐对云托管的 MCP 服务器使用远程 HTTP 传输：
```bash
claude mcp add --transport http wink-docs https://docs.mcp.wink.travel/mcp
claude mcp list
```

在 Claude Code 中，运行 `/mcp` 查看服务器状态。如果想与整个仓库共享服务器，添加 `--scope project`，这样 Claude Code 会将配置写入 `.mcp.json`。

### Codex
官方文档：[Codex 的 Docs MCP 快速入门](https://developers.openai.com/resources/docs-mcp)

OpenAI 通过 Codex CLI 记录了 MCP 的设置，并指出配置与 IDE 扩展共享：
```bash
codex mcp add wink-docs --url https://docs.mcp.wink.travel/mcp
codex mcp list
```

### Cursor
Cursor MCP 文档：https://cursor.com/docs/context/mcp
设置：通过 MCP 设置添加远程服务器。
```json
{
  "mcpServers": {
    "wink-docs": {
      "url": "https://docs.mcp.wink.travel/mcp",
    }
  }
}
```

### VS Code
官方文档：https://code.visualstudio.com/docs/copilot/customization/mcp-servers
最简远程配置（用户或工作区的 `mcp.json`）：
```json
{
  "servers": {
    "wink-docs": {
      "type": "http",
      "url": "https://docs.mcp.wink.travel/mcp"
    }
  }
}
```

提示：
- 使用命令面板：“MCP: List Servers” 确认连接并浏览工具。
- 工具选择器允许启用/禁用 `api_search` 和 `docs_search`。
- 添加上下文：聊天视图 → 添加上下文 → MCP 资源，附加文档页面。

## 核心资源 URI

| URI | 描述 |
|-----|-------------|
| `wink://docs` | 文档索引（按类别分组） |
| `wink://docs/{path}` | 具体文档文章（Markdown / MDX） |
| `wink://api` | 从 Wink 的 swagger 配置发现的实时 OpenAPI 目录 |
| `wink://api/{apiName}` | 包含认证、标签、头信息和示例操作的简洁 API 概览 |
| `wink://api/{apiName}/operations` | 按标签分组的完整操作列表 |
| `wink://api/{apiName}/operation/{key}` | 单个操作详情（参数、请求体、响应） |
| `wink://api/{apiName}/schema/{schemaName}` | 解析后的组件模式，含嵌套字段、类型和示例 |
| `wink://api/{apiName}/operation/{key}/request` | 以请求为中心的视图，包含必填输入和最小示例负载 |
| `wink://api/{apiName}/operation/{key}/response/{status}` | 针对特定状态码的响应视图 |
| `wink://api/{apiName}/operation/{key}/recipe` | 调用该操作的任务导向流程 |

## 搜索工具（快速参考）
| 工具 | 目的 | 典型查询示例 |
|------|---------|-----------------------|
| `api_search` | 查找要集成的实时 API 或操作 | `create affiliate`，`payment receipt`，`lookup inventory`，`invite manager` |
| `docs_search` | 查找概念性文档/指南 | `authentication`，`webhooks`，`wordPress` |

留空 `query` 可获得建议。每个结果都包含可打开的 MCP URI。
对于操作命中，`api_search` 还返回关联的请求、响应、模式和流程 URI，方便代理直接跳转到最有用的子资源。

## 典型使用模式
1. 使用您的意图运行 `api_search`（例如 *“create affiliate”* 或 *“payment receipt”*）。
2. 打开返回的 `wink://api/...` URI，或跟随其关联的 `request_uri`、`response_uris`、`request_schema_uris`、`response_schema_uris` 和 `recipe_uri` 字段。
3. 使用 `docs_search` 查找支持性概念（认证、webhooks、工作流）。

## AI 技能

预构建的代理技能使用此 MCP 服务器作为其事实来源。技能为您的编码代理提供了一个结构化的工作流程，用于基于 Wink API 构建——代理无需猜测端点，而是直接从 MCP 服务器读取实时合约。

请参阅 [AI Skills](/zh-CN/developers/ai-skills) 了解可用技能及安装说明。

## 注意事项
- 每个返回的 URI 代理都可以打开以获取完整内容。
- `wink://api` 树是实时的，遵循当前上游 OpenAPI 目录，而非生成的仓库索引。
- 查询为空时会自动显示建议。
