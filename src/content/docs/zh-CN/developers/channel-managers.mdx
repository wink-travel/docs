Wink 与领先的渠道管理系统、物业管理系统（PMS）和中央预订系统（CRS）集成，确保您的价格和可用性在所有销售渠道中保持同步。

## 工作原理

渠道管理集成实现了您的物业管理系统与 Wink 之间的数据双向流动：

1. **价格推送** — 您的渠道管理系统将价格更新发送到 Wink。当您在 PMS 中更新价格时，这些更改会自动反映在 Wink 上。
2. **可用性同步** — 您的渠道管理系统将可用性更新发送到 Wink。当房间在 PMS 中被预订或释放时，Wink 的库存会实时更新。
3. **预订通知** — 当在 Wink 上完成预订时，您的渠道管理系统会收到通知，以便预订信息出现在您的 PMS 中。

## 支持的渠道管理系统

Wink 目前集成了以下系统：

| 渠道管理系统 | 描述 |
|---|---|
| **Channex** | 全球分销和渠道管理平台。 |
| **TravelClick** | 酒店解决方案和渠道管理（Amadeus）。 |
| **Omnibees** | 酒店分销和收益管理。 |
| **D-Edge** | 酒店技术和中央预订解决方案。 |
| **Allotz** | 库存分配和渠道管理。 |
| **Ezee / Yanolja** | 专注于亚洲市场的云端 PMS 和渠道管理。 |
| **RateGain** | 价格智能和分销技术。 |
| **RateTiger** | 实时价格和可用性分销。 |
| **CloudBeds** | 基于云的 PMS、渠道管理和预订引擎。 |
| **SiteMinder** | 领先的酒店渠道管理和分销平台。 |
| **Exely** | 酒店价格和可用性同步。 |

:::note
如果您的渠道管理系统未列出，请联系 Wink 团队。可以根据 Wink 支持的 OTA（开放旅游联盟）协议标准添加新的集成。
:::

## 面向开发者

如果您正在构建与 Wink 的渠道管理或 PMS 集成，请使用渠道管理 API 端点：

- **价格更新** — 向 Wink 推送价格变更。
- **可用性更新** — 向 Wink 推送可用性变更。
- **预订获取** — 从 Wink 拉取预订到您的系统。

渠道管理 API 遵循 OTA 协议标准（SOAP/XML），以兼容现有的酒店系统。

## Webhook 通知

您可以订阅渠道管理的 webhook 事件以接收实时通知：

- `channel-manager.update.rate` — 收到价格更新。
- `channel-manager.update.availability` — 收到可用性更新。
- `channel-manager.update` — 一般渠道管理更新。

详情请参见 [Webhook 事件目录](/zh-CN/developers/webhook-events#channel-manager-events)。
