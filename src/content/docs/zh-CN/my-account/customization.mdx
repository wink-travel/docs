自定义是一项强大的功能，您可以使用它来控制与 Wink 集成的各个方面，而无需编写任何代码。您销售的每一件库存都可以进行自定义。添加或移除功能，更改配色方案等等。

以下是入门步骤：

<Steps>
 1. 从[账户](https://my.wink.travel)中选择您想要操作的账户。
 2. 点击账户卡底部的`操作`，选择`管理账户`。
 3. 在接下来的页面点击`自定义`标签。
 4. 此部分将列出您所有的自定义项。
 5. 选择编辑已有的自定义项或创建新的。
 </Steps>

在`自定义`页面，点击您想操作的自定义项下的`操作`按钮，然后点击`更新`。

:::note
创建账户时，您已经拥有一个默认条目。此条目无法删除。
:::

这将带您进入自定义表单，您可以使用以下功能。

## 通用

以下是在`通用`标签下可用的选项。

- **名称** 给您的自定义命名，方便记忆。*例如：我的旅游博客*
- **显示货币** 选择用户初次看到的可预订旅游库存旁显示的货币。*例如：美元（USD）*
- **显示语言** 选择用户初次遇到的语言。*例如：英语*
- **附加到页面标题** 您可以控制添加到页面标题（*浏览器标签中看到的标题*）的内容。此功能主要用于我们的预订引擎。*例如：如果您访问 https://ota.wink.travel，默认显示为 `Home | Traveliko - Hotels with Benefits`*。您可以更改“|”管道符号后的所有内容。

## 主题

`主题`标签允许您自定义我们的[预订引擎](/zh-CN/booking-engine/in-a-nutshell)网站、[WinkLinks](/zh-CN/link-manager/wink-links)网站以及所有[Web 组件](/zh-CN/developers/web-components)的外观和感觉。

### 配置

此部分允许您配置设计和卡片布局。目前只有一种设计，即 Wink。

卡片布局控制用户看到的交易卡是水平排列还是垂直排列。信息内容相同。

:::note
在移动设备上查看我们的交易卡时，卡片布局无差异。但在 iPad mini 及更大分辨率设备上可以看到区别。
:::

:::tip[专业提示]
如果列表中旅游库存（如房型）少于4个，使用水平布局效果更佳。
:::

### 颜色

颜色控制用户看到的所有 UI 组件和网站的配色。用它来匹配您现有的配色方案（如果有的话）。

我们已经准备了大量预定义颜色供您尝试。预定义主题会更改下面表单字段中的颜色。您也可以完全使用自己的颜色。

## 分析

我们开箱即用支持 GA4 分析。如果您想追踪完整的增强型电商转化路径，请准备好您的测量 ID，并在`分析`标签下输入。

<Aside type="caution" title="提醒">
  别忘了将 `i.trvl.as`、`book.wink.travel` 和 `ota.wink.travel` 添加到您的 GA4 属性流中。
</Aside>

## 地图

要通过我们的[WordPress 插件](/zh-CN/developers/wordpress)或[Web 组件](/zh-CN/developers/web-components)将地图嵌入您的网站，您需要提供自己的 Google Maps API 密钥。

:::note
  [在此创建 Google Maps API 密钥](https://console.developers.google.com/projectselector2/apis/credentials?supportedpurview=project)
:::

生成密钥并为您的网站配置后，请在`分析`标签下输入。

## 行程

`行程`标签可能是确保所选库存价格正确显示的最重要部分。
由于旅游库存定价较为复杂，Wink 需要知道用户的行程，才能显示准确价格。

此部分允许您设置默认行程，用于显示所有数字资产的价格，例如：

- 我们带价格叠加的社交友好图片。
- WinkLinks 网格、卡片和链接。
- Web 组件
- 预订引擎

### 日期

您可以选择设置静态日期范围或滚动日期范围。

- 如果自定义用于特定日期的活动，请选择静态日期范围。
- 如果想要：
    1. 默认设置为今天加1晚（大多数 OTA 采用此方式），或
    2. 默认设置为未来某个始终有库存的日期，
  请选滚动日期范围。

### 房间配置

默认行程设置为2位成人，因为这是大多数旅客的常见设置。
您可以轻松更改，并添加多间房及多位入住者，包括成人和儿童。

### 促销

促销是一个方便的功能，能为用户节省时间。如果某物业提供了您可以分享给朋友的促销码，您可以在此输入。
通过您的链接或类似方式进入的用户，促销码会自动添加到行程中，折扣价格也会自动生效。

## 集成

`集成`标签适用于酒店、集成商和旅行社，帮助他们更好地控制 Wink 上的预订分发。

### 通知

此功能允许酒店和集成商在预订发生时通知所有相关方。
包括通知渠道管理器。

默认情况下，这些选项均已启用。集成商可能想禁用其中一项或多项的场景包括：

- 旅行社通过 API 级别集成，想发送自己的确认邮件。
- 酒店正在测试集成，不想打扰销售人员。
- 酒店不关心邮件，只想通过 PMS 接收通知。

### 邮件标志

希望使用我们的邮件通知服务的集成商，可以自定义邮件，使其看起来像是来自他们自己。

:::note
  推荐尺寸为宽300px，高75px
:::

## 酒店专属

`酒店专属`标签下的选项允许酒店控制其在我们预订引擎上的落地页显示内容，以及当请求行程无库存时的显示内容。

- **排名**：尚未获得有效排名的物业，可能选择禁用此功能。

:::note
  物业无法控制联盟如何展示酒店落地页……这两个配置是独立的。
:::

- **可用性**：允许物业选择是否在无库存时继续显示库存。

:::note
  大多数情况下，物业希望用户能点击`预订`按钮，继续进入其落地页，在那里更改行程并尝试查找可用日期。
  当您想阻止用户继续时，请启用此开关。
:::

## 使用方法

在大多数场景中，您会选择应用哪个`自定义`到您的链接、卡片、地图和网格。

以下是您可能想手动应用配置的情况：

- **手动覆盖** 您想直接分享我们的预订引擎链接，而不使用我们的[可分享链接](/zh-CN/studio/shareable-links)功能，并应用您的自定义配置。
- **WordPress 插件** 在 Wink [WordPress 插件](/zh-CN/developers/wordpress)的`设置`中添加配置。
- **自定义集成** 使用我们的 [wink-app-loader](/zh-CN/developers/web-components/#application-loader) Web 组件，手动应用到您的自定义集成器。

### 手动覆盖

以下是如何将您的配置 ID（包含所有设置）应用到我们主预订引擎 URL 的简单示例。

<h4 align="center"> [https://ota.wink.travel?configurationId=MY-UNIQUE-ID](https://ota.wink.travel?configurationId=MY-UNIQUE-ID) </h4>

:::note
  用户点击此链接时，您也会设置您的联盟归因 cookie。
:::

### WordPress 插件

请访问[Wink WordPress 插件](/zh-CN/developers/wordpress)了解更多。

### 自定义集成

使用 [wink-app-loader](/zh-CN/developers/web-components/#application-loader) Web 组件控制页面上所有嵌入的 Wink Web 组件。

以下示例使用 `MY-UNIQUE-ID` 作为配置 ID。

```html {11}
<html>
  <head>
    <link rel="stylesheet" href="https://elements.wink.travel/styles.css"></link>
  </head>
  <body>

    <script src="https://elements.wink.travel/elements.js" type="module" defer></script>

    <wink-app-loader
      client-id="YOUR CLIENT ID GOES HERE"
      configuration-id="MY-UNIQUE-ID"
    ></wink-app-loader>
  </body>
</html>
```
