当在 Wink 上完成预订时，会发生以下情况：

<Steps>
1. 您的渠道管理器、CRS 或 PMS 会收到通知。
2. 您会收到一封确认邮件 *(预订台联系方式)*，其中包含管理新预订的链接。
3. 旅客会收到一封确认邮件，邮件中还包含如何联系您的预订台的信息。
4. 旅行社会收到与旅客相同的预订确认邮件。
4. 开发者可以监听 `booking.created` webhook 事件，实时接收我们提供的详细预订信息。
</Steps>

您可以点击邮件中的链接，或从主导航栏进入 `账户 > 预订` 开始操作。

## 预订列表

预订列表显示您所有的历史和未来预订，按预订日期升序排列。

### 筛选条件

您可以应用筛选条件来查找所需的预订。

- 预订日期 [范围]
- 到达日期 [范围]
- 离开日期 [范围]
- 预订代码
- 旅客姓氏
- 旅客名字
- 主费率名称
- 酒店名称

### 导出

您可以将符合当前筛选条件的预订导出为 CSV 文件。

导出预订步骤：

<Steps>
1. 确保所需的筛选条件已启用。
2. 等待预订列表刷新。
2. 点击列表右下角的 🗂️ (*文件导出*) 图标按钮。
</Steps>

## 预订详情

从预订列表中选择您要操作的预订，系统将跳转到 `预订` 页面。
该页面包含您在客人到达当天妥善接待所需的所有详细信息。

页面还包含一些重要操作，帮助您为客人提供优质服务。您可以在页面顶部的 `操作` 下拉菜单中找到所有可用操作。

### 重新发送邮件

要重新发送确认邮件给旅客，请点击 `操作 > 重新发送确认邮件`。

### 取消预订

作为酒店方，若要取消预订，请点击 `操作 > 取消预订`。

将弹出一个窗口，允许您：

<Steps>
1. 选择取消类型。*例如：日期错误*
2. 输入取消原因。
</Steps>

*此按钮遵循您的取消政策规则，因此并非始终可用。*

## API

希望作为旅行社管理 `预订` 的开发者，可以访问 [开发者 > API > 旅行社](/zh-CN/developers/apis/#travel-agent-api)。
