在 Wink 上进行预订时，会发生以下情况：

<Steps>
  1. 您的渠道经理、CRS 或 PMS 已收到通知。
  2. 您会收到一封确认电子邮件 *（预订台联系方式）* 并附有管理新预订的链接。
  3. 旅行者会收到一封确认电子邮件，其中还包括如何联系预订处的信息。
  4. 旅行社收到与旅行者相同的预订确认。
  5. 开发人员可以听取 `booking.created` webhook 事件并实时接收我们的详细预订信息。
</Steps>


请点击电子邮件中的链接，或者前往`Account > Bookings`从主导航栏开始。

## 预订网格

预订网格显示您所有过去和未来的预订，按预订日期升序排列。

### 过滤器

您可以应用过滤器来查找您正在寻找的预订。

* 预订日期\[范围]
* 到达日期\[范围]
* 出发日期\[范围]
* 预订代码
* 旅行者姓氏
* 旅行者的名字
* 主费率名称
* 属性名称

### 出口

您可以将符合当前过滤条件的预订导出为 CSV。

要导出您的预订：

<Steps>
  1. 确保您想要的过滤器处于活动状态。
  2. 等待预订网格刷新。
  3. 点击🗂️（*文件导出*) 图标按钮。
</Steps>


## 预订

从预订网格中选择您想要处理的预订，您将被重定向到`Booking`页面。
本页面包含您在客人抵达当天妥善接待所需了解的所有详细信息。

该页面还包含一些您需要采取的重要措施，以便更好地服务您的客人。您可以在`Actions`页面顶部的下拉菜单。

### 重新发送电子邮件

要向旅行者重新发送确认电子邮件，请点击`Actions > Resend confirmation email`。

### 取消预订

要取消预订，请点击`Actions > Cancel booking`。

将显示一个弹出窗口，您可以：

<Steps>
  1. 选择取消类型。 *例如：日期不正确*
  2. 输入取消的原因。
</Steps>


*此按钮遵循您的取消政策规则，因此并不总是可用。*

## API

想要管理的开发人员`Bookings`作为旅行社代理人，可以前往[开发人员 > API > 旅行社](/developers/apis/#travel-agent-api)。
