Jei esate integratorius, pavyzdžiui, „Wink“, ir norite susieti išorinius gavėjus, kad juos būtų galima naudoti su „TripPay“, galite tai padaryti vienu iš dviejų būdų:

* Rankinis žemėlapių sudarymas
* Naudojant[Mokėjimo API](/developers/apis/#payment-api)(*Žr.`Account Mappings`*)

:::tip
Rekomenduojame naudoti API, kad automatizuotumėte šį veiksmą, nebent dirbate tik su keliomis tomis pačiomis savybėmis.
:::

Žemiau pateikiame išsamų išorinio gavėjo susiejimo rankiniu būdu procesą.*Jis naudoja paslėptą API.*

<Steps>
  1. Prisijungti [TripPay](https://pay.wink.travel).
  2. Pasirinkite savo integratoriaus paskyrą.
  3. Spustelėkite `Settings > Mappings` iš pagrindinės naršymo juostos.
  4. Spustelėkite `Create mapping` mygtuką. Pamatysite susiejimo formą.
  5. **Vietinė paskyra** Pradėkite rašyti esamo „TripPay“ paskyros, kurią norite susieti, pavadinimą.
  6. **Išorinis identifikatorius** Įklijuokite šio objekto išorinį ID tokį, koks jis yra nuotolinėje sistemoje.
  7. Spustelėkite `Save` tęsti. Jūs būsite nukreipti atgal į savo susiejimų sąrašą.
</Steps>


Sukūrus susiejimus, „TripPay“ dabar žino, kurias sąskaitas įtraukti kaip užsakymo gavėjus, kai į jas nurodote naudodami išorinius identifikatorius.

## API

Kūrėjai, norintys valdyti`Payments`, eikite į[Kūrėjai > API > Mokėjimai](/developers/apis/#payment-api).
