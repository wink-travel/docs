## Apžvalga
Talpinamas Wink MCP serveris pasiekiamas adresu:

```
https://docs.mcp.wink.travel/mcp
```

Nukreipkite suderinamus AI klientus (VS Code, Cursor, kitus, įgyvendinančius MCP HTTP transportą) į šį galinį tašką, kad jie galėtų:
- Naršyti produkto ir platformos dokumentaciją (`wink://docs/...`)
- Naršyti tiesiogines OpenAPI specifikacijas ir operacijų dokumentus (`wink://api/...`)
- Atlikti lengvą semantinę / raktinių žodžių paiešką dokumentuose ir tiesioginiuose API

## Transportas
Naudokite pagrindinį srautinį HTTP galinį tašką:

```
https://docs.mcp.wink.travel/mcp
```

Jei jūsų klientas palaiko tik SSE, naudokite:

```
https://docs.mcp.wink.travel/sse
```

### Claude Code
Oficiali dokumentacija: [Prijunkite Claude Code prie įrankių per MCP](https://code.claude.com/docs/en/mcp)

Anthropic rekomenduoja nuotolinį HTTP transportą debesyje talpinamiems MCP serveriams:
```bash
claude mcp add --transport http wink-docs https://docs.mcp.wink.travel/mcp
claude mcp list
```

Claude Code viduje paleiskite `/mcp`, kad patikrintumėte serverio būseną. Jei norite bendrinti serverį su visa saugykla, pridėkite `--scope project`, kad Claude Code įrašytų konfigūraciją į `.mcp.json`.

### Codex
Oficiali dokumentacija: [Docs MCP greitas startas Codex](https://developers.openai.com/resources/docs-mcp)

OpenAI dokumentuoja MCP nustatymą per Codex CLI ir pažymi, kad konfigūracija dalijamasi su IDE plėtiniu:
```bash
codex mcp add wink-docs --url https://docs.mcp.wink.travel/mcp
codex mcp list
```

### Cursor
Cursor MCP dokumentacija: https://cursor.com/docs/context/mcp
Nustatymas: pridėkite nuotolinį serverį per MCP nustatymus.
```json
{
  "mcpServers": {
    "wink-docs": {
      "url": "https://docs.mcp.wink.travel/mcp",
    }
  }
}
```

### VS Code
Oficiali dokumentacija: https://code.visualstudio.com/docs/copilot/customization/mcp-servers
Minimalus nuotolinis konfigas (vartotojo arba darbo srities `mcp.json`):
```json
{
  "servers": {
    "wink-docs": {
      "type": "http",
      "url": "https://docs.mcp.wink.travel/mcp"
    }
  }
}
```

Patarimai:
- Naudokite komandų paletę: „MCP: List Servers“, kad patvirtintumėte ryšį ir naršytumėte įrankius.
- Įrankių pasirinkiklis leidžia įjungti/išjungti `api_search` ir `docs_search`.
- Pridėkite kontekstą: Pokalbio vaizdas → Pridėti kontekstą → MCP ištekliai, kad pridėtumėte dokumentų puslapius.

## Pagrindiniai išteklių URI

| URI | Aprašymas |
|-----|-----------|
| `wink://docs` | Dokumentacijos indeksas (grupuotas pagal kategoriją) |
| `wink://docs/{path}` | Konkretus dokumentacijos straipsnis (Markdown / MDX) |
| `wink://api` | Tiesioginis OpenAPI katalogas, rastas iš Wink swagger konfigūracijos |
| `wink://api/{apiName}` | Kompaktiška API santrauka su autentifikacija, žymomis, antraštėmis ir pavyzdinėmis operacijomis |
| `wink://api/{apiName}/operations` | Pilnas operacijų sąrašas, grupuotas pagal žymą |
| `wink://api/{apiName}/operation/{key}` | Atitinkamos operacijos detalės (parametrai, užklausos turinys, atsakymai) |
| `wink://api/{apiName}/schema/{schemaName}` | Išspręsta komponento schema su įdėtinėmis laukų struktūromis, tipais ir pavyzdžiais |
| `wink://api/{apiName}/operation/{key}/request` | Užklausai skirta peržiūra su privalomais įvesties duomenimis ir minimaliu pavyzdiniu užklausos turiniu |
| `wink://api/{apiName}/operation/{key}/response/{status}` | Atsakymui skirta peržiūra konkrečiam statuso kodui |
| `wink://api/{apiName}/operation/{key}/recipe` | Užduoties orientuotas srautas, kaip teisingai iškviesti tą operaciją |

## Paieškos įrankiai (greita nuoroda)
| Įrankis | Paskirtis | Tipiniai užklausų pavyzdžiai |
|---------|-----------|------------------------------|
| `api_search` | Rasti tiesioginį API arba operaciją integracijai | `create affiliate`, `payment receipt`, `lookup inventory`, `invite manager` |
| `docs_search` | Rasti konceptualius dokumentus / gaires | `authentication`, `webhooks`, `wordPress` |

Palikite `query` tuščią, kad gautumėte pasiūlymus. Kiekvienas rezultatas apima MCP URI, kurį galite atidaryti.
Operacijų atvejais `api_search` taip pat grąžina susietus užklausos, atsakymo, schemos ir recepto URI, kad agentai galėtų tiesiogiai pereiti prie naudingiausių posistemų.

## Tipinis naudojimo modelis
1. Paleiskite `api_search` su savo ketinimu (pvz., *„create affiliate“* arba *„payment receipt“*).
2. Atidarykite grąžintą `wink://api/...` URI arba sekite susietus laukus `request_uri`, `response_uris`, `request_schema_uris`, `response_schema_uris` ir `recipe_uri`.
3. Naudokite `docs_search` papildomoms sąvokoms (autentifikacija, webhooks, darbo srautai).

## AI įgūdžiai

Iš anksto paruošti agentų įgūdžiai naudoja šį MCP serverį kaip savo tiesos šaltinį. Įgūdžiai suteikia jūsų kodavimo agentui struktūruotą darbo eigą, kaip kurti naudojant Wink API — vietoje spėjimo apie galinius taškus agentas tiesiogiai skaito tiesioginius sutartinius duomenis iš MCP serverio.

Žr. [AI Skills](/lt/developers/ai-skills) dėl prieinamų įgūdžių ir diegimo instrukcijų.

## Pastabos
- Kiekvieną grąžintą URI agentas gali atidaryti pilnam turiniui.
- `wink://api` medis yra tiesioginis ir seka dabartinį aukštesnio lygio OpenAPI katalogą, o ne sugeneruotą saugyklos indeksą.
- Pasiūlymai rodomi automatiškai, kai užklausos yra tuščios.
