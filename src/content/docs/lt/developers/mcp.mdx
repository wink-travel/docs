## Apžvalga
Talpinamas Wink MCP serveris yra pasiekiamas adresu:

```
https://docs.mcp.wink.travel/mcp
```

Nukreipkite suderinamus AI klientus (VS Code, Cursor, kiti, įgyvendinantys MCP HTTP transportą) į šį galinį tašką, kad jie galėtų:
- Naršyti produkto ir platformos dokumentaciją (`wink://docs/...`)
- Atrasti SDK ir gilintis į API klases / modelius (`wink://sdk/...` ir `wink://sdk/.../docs`)
- Atlikti lengvą semantinę / raktinių žodžių paiešką dokumentacijoje ir SDK santraukoje

## Transportas
Naudokite pagrindinį srautinį HTTP galinį tašką:

```
https://docs.mcp.wink.travel/mcp
```

Jei jūsų klientas palaiko tik SSE, naudokite:

```
https://docs.mcp.wink.travel/sse
```

### Cursor
Cursor MCP dokumentacija: https://cursor.com/docs/context/mcp
Nustatymas: pridėkite nuotolinį serverį per MCP nustatymus.
```json
{
  "mcpServers": {
    "wink-docs": {
      "url": "https://docs.mcp.wink.travel/mcp",
    }
  }
}
```

### VS Code
Oficiali dokumentacija: https://code.visualstudio.com/docs/copilot/customization/mcp-servers
Minimalus nuotolinis konfigūravimas (vartotojo arba darbo srities `mcp.json`):
```json
{
  "servers": {
    "wink-docs": {
      "type": "http",
      "url": "https://docs.mcp.wink.travel/mcp"
    }
  }
}
```

Patarimai:
- Naudokite komandų paletę: „MCP: List Servers“, kad patvirtintumėte ryšį ir naršytumėte įrankius.
- Įrankių pasirinkiklis leidžia įjungti/išjungti `sdk_search` ir `docs_search`.
- Pridėkite kontekstą: Pokalbių vaizdas → Pridėti kontekstą → MCP ištekliai, kad prijungtumėte dokumentacijos puslapius.

## Pagrindiniai išteklių URI

| URI | Aprašymas |
|-----|-----------|
| `wink://docs` | Dokumentacijos indeksas (grupuotas pagal kategoriją) |
| `wink://docs/{path}` | Konkretus dokumentacijos straipsnis (Markdown / MDX) |
| `wink://sdk` | SDK apžvalga (kiekvienas SDK + dokumentacijos skaičius) |
| `wink://sdk/{sdkPath}` | SDK README / modulio dokumentacijos aplankas / konkretus modulio dokumentas |

### Modulinių dokumentų aplankas
Atidarius SDK dokumentacijos aplanką (pvz., `wink://sdk/python/wink-sdk-booking/docs`) grąžinami grupuoti:
- **API klasės** (metodų skaičius + pavyzdiniai metodų pavadinimai)
- **Modeliai ir tipai** (duomenų struktūros, naudojamos šiose API)

## Paieškos įrankiai (greita nuoroda)
| Įrankis | Paskirtis | Tipiniai užklausų pavyzdžiai |
|---------|-----------|------------------------------|
| `sdk_search` | Rasti, kurį SDK / API klasę naudoti | `atšaukti užsakymą`, `sąrašas su nekilnojamuoju turtu`, `partnerių inventorius` |
| `docs_search` | Rasti konceptualią dokumentaciją / gaires | `autentifikacija`, `webhook'ai`, `WordPress` |

Palikite `query` tuščią, kad gautumėte pasiūlymus. Kiekvienas rezultatas apima MCP URI, kurį galite atidaryti.

## Tipinis naudojimo modelis
1. Paleiskite `sdk_search` su savo ketinimu (pvz., *„sąrašas su nekilnojamuoju turtu“*).
2. Atidarykite grąžintą SDK README arba jo `docs_uri`, kad peržiūrėtumėte API / modelius.
3. Naudokite `docs_search` papildomoms sąvokoms (autentifikacija, webhook'ai).

## Pastabos
- Kiekvieną grąžintą URI agentas gali atidaryti pilnam turiniui.
- Pasiūlymai rodomi automatiškai, kai užklausos yra tuščios.
