## Consume endpoints

Consume endpoints are for developers who want to find existing travel inventory and either book it or use it to advertise through one of their Wink affiliate accounts.

### Configuration API

A single endpoint to retrieve whitelabel + customization information for the booking engine.

Learn more
</LinkButton>

### Lookup API

A programmer-friendly way to search for bespoke travel inventory across geographic regions.

<LinkButton href="https://docs.wink.travel/lookup" variant="secondary" icon="external" target="_blank">

### Inventory API

The Analytics API exposes endpoints for anyone who wants insight into track leaderboard rankings as well as deep insight into time series, platform-level, data; hosting a wide variety of metrics.

## Payment API

Payment endpoints are for developers who want to purchase travel inventory. This can be done via the API as a registered Travel Agent or using our API in conjunction with our PCI compliant payment widget for all other entities.
The Payment API exposes endpoints for affiliates and hotels to track bookings, analytics and funds availability. They can also choose to withdraw available funds to their bank account.

<LinkButton href="https://docs.wink.travel/payment" variant="secondary" icon="external" target="_blank">
Learn more
</LinkButton>
