Όταν μια κράτηση δημιουργείται, ενημερώνεται ή ακυρώνεται στο Wink, η πλατφόρμα συγχρονίζει αυτή την αλλαγή με τα συνδεδεμένα εξωτερικά σας συστήματα — συστήματα διαχείρισης καταλυμάτων (PMS), channel managers και κεντρικά συστήματα κρατήσεων (CRS).

## Πώς λειτουργεί

Κάθε αλλαγή κράτησης δημιουργεί μια **εγγραφή συγχρονισμού** που παρακολουθεί την κατάσταση του συγχρονισμού. Η πλατφόρμα επεξεργάζεται τις εγγραφές συγχρονισμού ασύγχρονα, επαναλαμβάνοντας σε περίπτωση αποτυχίας για να εξασφαλίσει αξιόπιστη παράδοση.

## Κύκλος ζωής συγχρονισμού

Κάθε εγγραφή συγχρονισμού ακολουθεί μια μηχανή καταστάσεων:

```
┌─────────────┐     success     ┌───────────┐
│  TO_CREATE   │ ──────────────→ │ PROCESSED │
└─────────────┘                  └───────────┘
       │
       │ failure
       ▼
┌──────────────────┐
│ TO_CREATE_FAILED │
└──────────────────┘
```

Το ίδιο μοτίβο ισχύει για ενημερώσεις και ακυρώσεις:

| Ενέργεια | Εκκρεμής κατάσταση | Κατάσταση επιτυχίας | Κατάσταση αποτυχίας |
|---|---|---|---|
| Νέα κράτηση | `TO_CREATE` | `PROCESSED` | `TO_CREATE_FAILED` |
| Ενημέρωση κράτησης | `TO_UPDATE` | `PROCESSED` | `TO_UPDATE_FAILED` |
| Ακύρωση | `TO_CANCEL` | `PROCESSED` | `TO_CANCEL_FAILED` |

## Τι ενεργοποιεί τον συγχρονισμό

- **Νέα κράτηση** — Ένας ταξιδιώτης κάνει κράτηση μέσω Wink ή ενός συνεργαζόμενου καναλιού.
- **Ενημέρωση κράτησης** — Αλλαγή στα στοιχεία της κράτησης (ημερομηνίες, πληροφορίες επισκέπτη, ανάθεση δωματίου).
- **Ακύρωση** — Μια κράτηση ακυρώνεται από τον ταξιδιώτη ή το κατάλυμα.

## Παρακολούθηση

Μπορείτε να παρακολουθείτε την κατάσταση συγχρονισμού μέσω του συνδεδεμένου channel manager ή του πίνακα ελέγχου PMS. Οι αποτυχημένες εγγραφές συγχρονισμού επαναλαμβάνονται αυτόματα. Εάν ο συγχρονισμός συνεχίσει να αποτυγχάνει, παραμένει στην κατάσταση `*_FAILED` για διερεύνηση.

## Αντιμετώπιση προβλημάτων

Εάν οι κρατήσεις δεν εμφανίζονται στο PMS σας:

1. **Ελέγξτε τη σύνδεση με τον channel manager** — Βεβαιωθείτε ότι η ενσωμάτωση PMS ή channel manager είναι ενεργή και σωστά ρυθμισμένη.
2. **Επαληθεύστε την παράδοση webhook** — Εάν χρησιμοποιείτε webhooks, ελέγξτε ότι το endpoint σας λαμβάνει τα γεγονότα `booking.create` και `booking.cancelled`. Δείτε το [Κατάλογο Γεγονότων Webhook](/el/developers/webhook-events#booking-events).
3. **Εξετάστε τις αποτυχημένες εγγραφές** — Οι αποτυχημένες εγγραφές συγχρονισμού υποδεικνύουν πρόβλημα σύνδεσης ή δεδομένων με το σύστημα προορισμού.
4. **Επικοινωνήστε με την υποστήριξη** — Εάν τα προβλήματα συνεχίζονται, η ομάδα του Wink μπορεί να διερευνήσει τις αποτυχημένες εγγραφές συγχρονισμού για λογαριασμό σας.
