Αυτό το άρθρο σας δείχνει πώς να ρυθμιστείτε γρήγορα ως προγραμματιστής και να ξεκινήσετε να δοκιμάζετε τα APIs μας στο Wink.

:::note
Οι δοκιμές πρέπει πάντα να γίνονται στο [περιβάλλον staging](/el/getting-started/environments). Υπάρχουν πολύ λιγότερα δεδομένα εκεί, αλλά μπορείτε να δοκιμάσετε όσο θέλετε χωρίς να επηρεάσετε το περιβάλλον παραγωγής.

Μόλις είστε έτοιμοι να πάτε live, θα πιστοποιήσουμε τον λογαριασμό και την εφαρμογή σας και μπορείτε να ξεκινήσετε στο περιβάλλον παραγωγής μας.
:::

Ακολουθήστε αυτά τα βήματα:

<Steps>
1. [Εγγραφείτε](/el/user/register) για τον λογαριασμό χρήστη σας στο Wink
2. Δημιουργήστε έναν [λογαριασμό ταξιδιωτικού πράκτορα](/el/travel-agent/what-is-agent) ή έναν [λογαριασμό συνεργάτη](/el/studio/what-is-studio).
3. [Δημιουργήστε μια Εφαρμογή](/el/developer/applications) όπου θα τη συσχετίσετε με τον λογαριασμό που δημιουργήσατε στο Βήμα 2.
4. Μόλις δημιουργήσετε την `Εφαρμογή`, αποθηκεύστε το client-id και το client secret σε ασφαλές μέρος.
</Steps>

## CDN

Στο Wink, χρησιμοποιούμε το Cloudinary για όλα τα αρχεία εικόνων και βίντεο. Όταν έχετε πρόσβαση στο ταξιδιωτικό μας απόθεμα, θα έχετε πλήρη πρόσβαση σε όλες τις προηγμένες λειτουργίες του Cloudinary.

Το `cloud_name` μας είναι `traveliko`.

### Παράδειγμα

Για κάθε τύπο ταξιδιωτικού αποθέματος που υποστηρίζουμε, υπάρχει ένας εμφωλευμένος πίνακας JSON με όνομα `multimedias`. Το JSON για μια εγγραφή multimedia μοιάζει ως εξής:

```json {3}
{
  "multimediaIdentifier": "multimedia-1",
  "identifier": "partners/radisson_blu_logo",
  "type": "IMAGE",
  "width": "1024",
  "height": "768",
  "category": "1",
  "descriptions": [
    {
      "name": "Lobby",
      "description": "Picture of the lobby",
      "language": "en"
    }
  ],
  "lifestyleType": "LIFESTYLE_BUSINESS",
  "attribution": [
    {
      "url": "https://www.hilton.com",
      "name": "Hilton"
    }
  ]
}
```

Η επισημασμένη γραμμή παραπάνω δείχνει το μοναδικό ID του Cloudinary `partners/radisson_blu_logo`. Αυτό είναι όλο ό,τι χρειάζεστε για να ανακτήσετε την εικόνα σε οποιαδήποτε μορφή, ποιότητα και μέγεθος θέλετε.

## Δοκιμές

Μεταβείτε στην ενότητα [APIs](/el/developer/apis) και επιλέξτε το API με το οποίο θέλετε να εργαστείτε.

### Τεκμηρίωση API

Ο ιστότοπος τεκμηρίωσής μας είναι διαμορφωμένος έτσι ώστε να μπορείτε να εκτελείτε εντολές API απευθείας από τον ιστότοπο.

Σε αυτό το παράδειγμα, θέλετε να χρησιμοποιήσετε την κλήση API `Search Lookups (OAuth2)`.

:::tip[Συμβουλή]
Για ευκολία χρήσης, προτείνουμε στους ενσωματωτές να χρησιμοποιούν τα endpoints που ορίζουμε ως `OAuth2`. Αυτά τα endpoints γνωρίζουν να περιμένουν κλήσεις API και ήδη γνωρίζουν σε ποιον λογαριασμό βρίσκεστε, οπότε δεν χρειάζεται να καθορίσετε το `companyIdentifier` στα URLs σας.
:::

<Steps>
1. Χρησιμοποιήστε το [Lookup API](https://developer.wink.travel/lookup) για να ανακτήσετε προορισμούς ή ξενοδοχεία.
2. Στο αριστερό navbar, κάντε κλικ στο `Authentication`.
3. Εισάγετε το client-id και το client secret και πατήστε το κουμπί `Get TOKEN`.
4. Αν τα διαπιστευτήριά σας είναι έγκυρα, θα δείτε ότι το κλειδί API σας έχει εφαρμοστεί.
5. Τώρα μπορείτε να δοκιμάσετε το endpoint lookup.
</Steps>

### Postman
Μπορείτε να κάνετε το ίδιο με το Postman.

<Steps>
1. Κατεβάστε το [Postman](https://www.postman.com/downloads).
2. Κάντε κλικ στο `Download OpenAPI spec` από την κορυφή της σελίδας [Lookup API](https://developer.wink.travel/lookup).
3. Εισάγετέ το στο Postman.
4. Αν το εισαγάγατε ως νέα συλλογή με όνομα `Wink API`, κάντε κλικ στις οριζόντιες ελλείψεις (3 τελείες) δίπλα στο όνομα της συλλογής και επιλέξτε `Edit`.
5. Στην καρτέλα `Authorization`, ορίστε τον `Auth type` σε OAuth2.
6. Ορίστε το πεδίο `Add auth data to` σε `Request headers`.
7. Ορίστε το πεδίο `Header prefix` σε `Bearer`.
8. Ορίστε το πεδίο `Token Name` σε όποιο όνομα θέλετε να δώσετε στο αποθηκευμένο token στο Postman, π.χ. Wink Staging.
9. Ορίστε το πεδίο `Grant type` σε `Client Credentials`.
10. Ορίστε το πεδίο `Access token URL` σε https://staging-iam.wink.travel/oauth2/token.
11. Ορίστε το πεδίο `Client ID` στο Client ID που δημιούργησε η εφαρμογή σας.
12. Ορίστε το πεδίο `Client Secret` στο Client Secret που δημιούργησε η εφαρμογή σας.
13. Ορίστε το πεδίο `Scope` σε `inventory.read inventory.write inventory.remove`. Αυτά είναι όλα τα scopes που θα χρειαστείτε για το Wink.
14. Ορίστε το πεδίο `Client Authentication` σε `Send as Basic Auth header`.
15. Πατήστε το κουμπί `Get New Access Token` για να ανακτήσετε το token σας και να το αποθηκεύσετε.
16. Για όλες τις κλήσεις API στη συλλογή Wink API, βεβαιωθείτε ότι η καρτέλα authentication λέει `Inherit auth from parent` ώστε να περιλαμβάνει το token σας σε κάθε κλήση.
</Steps>

:::tip[Συμβουλή]
Συνιστάται να κάνετε το domain του server, το client ID και το client secret μεταβλητές περιβάλλοντος. Με αυτόν τον τρόπο μπορείτε εύκολα να εναλλάσσεστε μεταξύ των περιβαλλόντων staging και παραγωγής και να επαναχρησιμοποιείτε όλες τις λειτουργίες που έχετε δημιουργήσει.
:::
