Εάν πραγματοποιείτε διαφημιστικές καμπάνιες μέσω Sojern, μπορείτε να συνδέσετε τον λογαριασμό σας Sojern με το Wink εισάγοντας το Sojern GTM Container ID σας στην πύλη Wink. Μόλις συνδεθεί, το pixel παρακολούθησης του Sojern ενεργοποιείται αυτόματα για το ακίνητό σας — καλύπτοντας ολόκληρο το ταξίδι του επισκέπτη από την αναζήτηση έως την ολοκληρωμένη κράτηση.

<Aside type="tip">
Αυτό προϋποθέτει ότι έχετε ήδη ενεργό λογαριασμό Sojern και έχετε λάβει το GTM Container ID από τον διαχειριστή του λογαριασμού σας Sojern. Εάν δεν έχετε ακόμη λογαριασμό Sojern, επισκεφθείτε το [sojern.com/contact](https://www.sojern.com/contact) για να ξεκινήσετε.
</Aside>

---

## Τι Πρέπει να Κάνετε

<Steps>

1. **Λάβετε το GTM Container ID από το Sojern**

   Επικοινωνήστε με τον διαχειριστή του λογαριασμού σας Sojern και ζητήστε το **GTM Container ID** σας. Ακολουθεί τη μορφή `GTM-XXXXXXX`. Αυτό διαφέρει από το Sojern Pixel ID — βεβαιωθείτε ότι ζητάτε συγκεκριμένα το GTM Container ID.

2. **Εισάγετε το στην Πύλη Wink**

   Συνδεθείτε στο [my.wink.travel](https://my.wink.travel), επιλέξτε τον λογαριασμό σας και μεταβείτε στην καρτέλα **Προσαρμογές**. Επικολλήστε το Sojern GTM Container ID στο πεδίο Sojern και αποθηκεύστε. Η ενσωμάτωσή σας είναι πλέον ενεργή.

3. **Επιβεβαιώστε με τον Διαχειριστή του Λογαριασμού Sojern**

   Ενημερώστε τον διαχειριστή του λογαριασμού σας Sojern ότι η ενσωμάτωση είναι ενεργή. Μπορούν να επαληθεύσουν από την πλευρά τους ότι το pixel ενεργοποιείται σωστά στο `wink.travel` για το ακίνητό σας και ότι τα δεδομένα ρέουν στον λογαριασμό σας Sojern.

</Steps>

<Aside type="note">
Δεν χρειάζεται να τοποθετήσετε κανέναν κώδικα pixel στην ιστοσελίδα σας για αυτήν την ενσωμάτωση. Το Wink φορτώνει αυτόματα το κοντέινερ Sojern σε όλες τις σελίδες όπου το ακίνητό σας είναι ενεργό.

Αυτή η ρύθμιση λειτουργεί για οποιοδήποτε προσαρμοσμένο GTM Container ID. Χρησιμοποιήσαμε το Sojern ως παράδειγμα επειδή είναι μια πολύ κοινή ενσωμάτωση.
</Aside>

## Τι Κάνει το Wink από την Πλευρά μας

Για διαφάνεια, εδώ είναι τι ρυθμίζει το Wink για να λειτουργήσει αυτή η ενσωμάτωση:

- **Διαθέσιμο πεδίο Sojern GTM Container ID** στην καρτέλα Προσαρμογές της πύλης Wink. Όταν εισάγετε το Container ID, το Wink ελέγχει τη μορφή και το αποθηκεύει με ασφάλεια για το ακίνητό σας.
- **Σε κάθε σελίδα όπου το ακίνητό σας είναι ενεργό** στο Wink, το Sojern GTM Container ID σας αποδίδεται στη σελίδα και το σύστημα διαχείρισης ετικετών του Wink φορτώνει δυναμικά το κοντέινερ Sojern παράλληλα με το δικό του. Αυτό γίνεται αυτόματα — δεν απαιτούνται χειροκίνητα βήματα από την ομάδα Wink ανά ακίνητο.
- **Το κοντέινερ Sojern φορτώνεται ασύγχρονα**, που σημαίνει ότι δεν επηρεάζει την απόδοση φόρτωσης της σελίδας για τους επισκέπτες σας.
- **Η δική μας παρακολούθηση και το pixel Sojern τρέχουν παράλληλα**, διαβάζοντας από τα ίδια κοινά δεδομένα συμβάντων. Κάθε συμβάν κράτησης που παρακολουθεί το Wink είναι ταυτόχρονα διαθέσιμο στο κοντέινερ Sojern.
- **Όταν το ακίνητό σας δεν προβάλλεται** — για παράδειγμα, όταν ένας επισκέπτης περιηγείται σε διαφορετικό ξενοδοχείο στο Wink — το κοντέινερ Sojern δεν φορτώνεται. Αυτό διασφαλίζει ότι το Sojern λαμβάνει μόνο δεδομένα σχετικά με το ακίνητό σας.

## Αναφορά DataLayer

Το Wink στέλνει συμβάντα συμβατά με GA4 στο `dataLayer` καθ’ όλη τη διάρκεια του ταξιδιού του επισκέπτη. Το κοντέινερ GTM σας διαβάζει από το ίδιο κοινό `dataLayer`, οπότε κάθε ενεργοποιητής ή μεταβλητή που ρυθμίζετε έχει πρόσβαση σε όλες τις ιδιότητες που περιγράφονται παρακάτω.

Κάθε συμβάν ακολουθεί τη δομή του GA4 Enhanced Ecommerce: μια κορυφαία συμβολοσειρά `event` και ένα αντικείμενο `ecommerce`. Πριν από κάθε αποστολή, το Wink καθαρίζει το προηγούμενο φορτίο ecommerce για να αποτρέψει τη διαρροή παλαιών δεδομένων σε νέους ενεργοποιητές.

<Aside type="note">
**Πρότυπο επαναφοράς dataLayer**

Το Wink επαναφέρει το αντικείμενο ecommerce πριν από κάθε αποστολή συμβάντος:

```js
dataLayer.push({ ecommerce: null });   // καθαρισμός προηγούμενου συμβάντος
dataLayer.push({ event: '...', ecommerce: { ... } });
```

Μην βασίζεστε στην επιμονή των ιδιοτήτων ecommerce μεταξύ των συμβάντων. Διαβάζετε πάντα τις ιδιότητες κατά το χρόνο ενεργοποίησης μέσα στο ίδιο συμβάν.
</Aside>

### Συμβάντα Ταξιδιού Κράτησης

Αυτά τα συμβάντα ενεργοποιούνται αυτόματα καθώς ο επισκέπτης προχωρά μέσα στον χωνί κράτησης. Ρυθμίστε τους ενεργοποιητές GTM ως **Custom Event** ενεργοποιητές που ταιριάζουν με τα ονόματα συμβάντων παρακάτω.

| Βήμα | Συμβάν | Πότε Ενεργοποιείται |
|------|--------|--------------------|
| 1 | `view_item_list` | Φόρτωση πλέγματος ή λίστας ξενοδοχείων |
| 2 | `view_item` | Ο επισκέπτης βλέπει συγκεκριμένη σελίδα ακινήτου |
| 3 | `add_to_cart` | Ο επισκέπτης προσθέτει δωμάτιο στο καλάθι |
| 4 | `begin_checkout` | Φόρτωση σελίδας ολοκλήρωσης αγοράς / πληρωμής |
| 5 | `add_payment_info` | Ο επισκέπτης επιλέγει μέθοδο πληρωμής |
| 6 | `login` | Ο επισκέπτης αυθεντικοποιείται μέσω SSO |
| 7 | `purchase` | Η κράτηση επιβεβαιώνεται |
| 8 | `refund` | Η κράτηση ακυρώνεται |

---

#### `view_item_list`

Ενεργοποιείται όταν αποδίδεται πλέγμα ή λίστα ξενοδοχείων.

```json
{
  "event": "view_item_list",
  "ecommerce": {
    "item_list_id": "featured_hotels",
    "item_list_name": "Featured Hotels",
    "items": [ ]
  }
}
```

---

#### `view_item`

Ενεργοποιείται όταν ο επισκέπτης βλέπει συγκεκριμένο ακίνητο.

```json
{
  "event": "view_item",
  "ecommerce": {
    "currency": "USD",
    "value": 299.00,
    "items": [ ]
  }
}
```

---

#### `add_to_cart`

Ενεργοποιείται όταν ο επισκέπτης προσθέτει δωμάτιο στο καλάθι.

```json
{
  "event": "add_to_cart",
  "ecommerce": {
    "currency": "USD",
    "value": 299.00,
    "items": [ ]
  }
}
```

---

#### `begin_checkout`

Ενεργοποιείται όταν φορτώνει η σελίδα ολοκλήρωσης αγοράς / πληρωμής.

```json
{
  "event": "begin_checkout",
  "ecommerce": {
    "currency": "USD",
    "value": 299.00,
    "coupon": "SUMMER10",
    "items": [ ]
  }
}
```

Το `coupon` υπάρχει μόνο όταν έχει εφαρμοστεί κωδικός έκπτωσης.

---

#### `add_payment_info`

Ενεργοποιείται όταν ο επισκέπτης επιλέγει ή επιβεβαιώνει μέθοδο πληρωμής.

```json
{
  "event": "add_payment_info",
  "ecommerce": {
    "currency": "USD",
    "value": 299.00,
    "coupon": "SUMMER10",
    "payment_type": "STRIPE",
    "items": [ ]
  }
}
```

Το `payment_type` προέρχεται από τον τύπο του αποκτώντα πληρωμής. Το `coupon` υπάρχει μόνο όταν έχει εφαρμοστεί κωδικός έκπτωσης.

---

#### `login`

Ενεργοποιείται όταν ο επισκέπτης αυθεντικοποιείται μέσω single sign-on.

```json
{
  "event": "login",
  "ecommerce": {
    "method": "IAM"
  }
}
```

---

#### `purchase`

Ενεργοποιείται όταν επιβεβαιώνεται μια κράτηση.

```json
{
  "event": "purchase",
  "ecommerce": {
    "currency": "USD",
    "transaction_id": "bc-a1b2c3d4",
    "value": 299.00,
    "coupon": "SUMMER10",
    "shipping": 0.00,
    "tax": 0.00,
    "items": [
      {
        "item_id": "rr-xyz789",
        "item_name": "Deluxe Ocean Suite",
        "item_brand": "Grand Resort",
        "item_category": "GUEST_ROOM",
        "price": 299.00,
        "quantity": 1,
        "property_id": "prop-abc123",
        "property_name": "Grand Resort Maldives",
        "property_brand": "Grand Collection",
        "property_chain": "Luxury Hotels International",
        "city_name": "Malé",
        "country_code": "MV",
        "start_date": "2026-06-01",
        "end_date": "2026-06-05",
        "num_adults": 2,
        "num_children": 0,
        "cancellation_policy": "Refundable",
        "rate_plan_id": "rp-summer2026",
        "room_rate_id": "rr-xyz789",
        "room_rate_name": "Summer Escape — Bed & Breakfast",
        "num_perks": 2,
        "room_location": "Overwater",
        "room_view": "Ocean",
        "room_classification": "Suite"
      }
    ]
  }
}
```

Το `transaction_id` είναι ο μοναδικός αναγνωριστικός αριθμός σύμβασης κράτησης. Τα `coupon`, `shipping` και `tax` υπάρχουν μόνο όταν ισχύουν.

---

#### `refund`

Ενεργοποιείται όταν ακυρώνεται μια κράτηση.

```json
{
  "event": "refund",
  "ecommerce": {
    "currency": "USD",
    "transaction_id": "bc-a1b2c3d4",
    "value": 299.00,
    "coupon": "SUMMER10",
    "shipping": 0.00,
    "tax": 0.00,
    "items": [ ]
  }
}
```

Το `transaction_id` ταιριάζει με το συμβάν `purchase` για την ίδια κράτηση. Τα `coupon`, `shipping` και `tax` υπάρχουν μόνο όταν ισχύουν.

### Ιδιότητες Αντικειμένων

Όλα τα συμβάντα που περιλαμβάνουν πίνακα `items` μοιράζονται το ίδιο σχήμα αντικειμένου. Κάθε ιδιότητα είναι προαιρετική εκτός αν σημειωθεί ως υποχρεωτική.

#### Τυπικά πεδία GA4

Αυτά τα πεδία αναγνωρίζονται εγγενώς από το GA4 και εμφανίζονται σε τυπικές αναφορές ecommerce χωρίς επιπλέον ρύθμιση.

| Ιδιότητα | Τύπος | Υποχρεωτικό | Περιγραφή |
|----------|-------|-------------|-----------|
| `item_id` | string | Ναι | Μοναδικός αναγνωριστικός αριθμός για το δωμάτιο ή προϊόν |
| `item_name` | string | — | Όνομα δωματίου ή προϊόντος |
| `affiliation` | string | — | Όνομα συνεργάτη ή πωλητή |
| `coupon` | string | — | Κωδικός κουπονιού που εφαρμόστηκε στο αντικείμενο |
| `currency` | string | — | Κωδικός νομίσματος ISO 4217 (π.χ. `USD`) |
| `creative_name` | string | — | Όνομα δημιουργικού που χρησιμοποιήθηκε σε προώθηση |
| `creative_slot` | string | — | Θέση δημιουργικού στην προώθηση |
| `discount` | number | — | Ποσό έκπτωσης που εφαρμόστηκε |
| `index` | number | — | Θέση στη λίστα (ξεκινά από 1) |
| `item_brand` | string | — | Όνομα ξενοδοχείου ή μάρκας |
| `item_category` | string | — | Τύπος προϊόντος — π.χ. `GUEST_ROOM`, `ACTIVITY`, `ATTRACTION`, `MEETING_ROOM`, `RESTAURANT`, `SPA` |
| `item_list_id` | string | — | Αναγνωριστικό της λίστας στην οποία ανήκει το αντικείμενο |
| `item_list_name` | string | — | Εμφανιζόμενο όνομα της λίστας |
| `location_id` | string | — | Γεωγραφικές συντεταγμένες του ακινήτου |
| `price` | number | — | Τιμή μονάδας |
| `promotion_id` | string | — | Αναγνωριστικό προώθησης |
| `promotion_name` | string | — | Εμφανιζόμενο όνομα προώθησης |
| `quantity` | number | — | Ποσότητα (προεπιλογή `1`) |

#### Προσαρμοσμένες παράμετροι σε επίπεδο αντικειμένου

Αυτές οι παράμετροι μεταφέρουν το πλαίσιο κράτησης ειδικό για το Wink. Το GA4 δεν τις εμφανίζει αυτόματα στις αναφορές — πρέπει να τις καταχωρίσετε ως **προσαρμοσμένες διαστάσεις σε επίπεδο αντικειμένου** στην ιδιοκτησία GA4 σας πριν εμφανιστούν σε εξερευνήσεις ή τυπικές αναφορές. Δείτε [Προσαρμοσμένες Διαστάσεις στο GA4](#custom-dimensions-in-ga4) παρακάτω για οδηγίες ρύθμισης.

| Ιδιότητα | Τύπος | Περιγραφή |
|----------|-------|-----------|
| `property_id` | string | Αναγνωριστικό ακινήτου Wink |
| `property_name` | string | Εμφανιζόμενο όνομα ακινήτου |
| `property_brand` | string | Μάρκα στην οποία ανήκει το ακίνητο |
| `property_chain` | string | Αλυσίδα στην οποία ανήκει το ακίνητο |
| `property_url_name` | string | URL-friendly slug για το ακίνητο |
| `city_name` | string | Πόλη όπου βρίσκεται το ακίνητο |
| `country_code` | string | Κωδικός χώρας ISO 3166-1 alpha-2 |
| `start_date` | string | Ημερομηνία check-in σε μορφή `YYYY-MM-DD` |
| `end_date` | string | Ημερομηνία check-out σε μορφή `YYYY-MM-DD` |
| `num_adults` | number | Αριθμός ενηλίκων στην κράτηση |
| `num_children` | number | Αριθμός παιδιών στην κράτηση |
| `cancellation_policy` | string | Ένα από: `Refundable`, `Non-Refundable`, `Refundable-With-Restrictions`, `Non-Refundable-With-Restrictions` |
| `rate_plan_id` | string | Αναγνωριστικό τιμολογιακού πλάνου |
| `room_rate_id` | string | Αναγνωριστικό συγκεκριμένης τιμής δωματίου |
| `room_rate_name` | string | Εμφανιζόμενο όνομα τιμής δωματίου |
| `num_perks` | number | Αριθμός παροχών που περιλαμβάνονται στην τιμή |
| `room_location` | string | Περιγραφή τοποθεσίας δωματίου (π.χ. `Overwater`, `Garden`) |
| `room_view` | string | Περιγραφή θέας δωματίου (π.χ. `Ocean`, `City`) |
| `room_classification` | string | Κατάταξη ή κατηγορία δωματίου (π.χ. `Suite`, `Standard`) |
| `bed_type` | string | Διαμόρφωση κρεβατιού για το δωμάτιο (π.χ. `King`, `Twin`) |

### Προσαρμοσμένες Διαστάσεις στο GA4

Το GA4 αγνοεί τις προσαρμοσμένες παραμέτρους αντικειμένων στις αναφορές μέχρι να τις καταχωρίσετε ρητά ως προσαρμοσμένες διαστάσεις σε επίπεδο αντικειμένου. Πρέπει να καταχωρίσετε μόνο τις παραμέτρους που σκοπεύετε να χρησιμοποιήσετε στις αναφορές — δεν χρειάζεται να καταχωρίσετε και τις 20.

<Steps>

1. **Ανοίξτε τις Προσαρμοσμένες Ορισμούς**

   Στο GA4, πηγαίνετε στο **Διαχειριστής** (εικονίδιο γραναζιού κάτω αριστερά) → επιλέξτε την ιδιοκτησία σας → **Εμφάνιση δεδομένων** → **Προσαρμοσμένοι ορισμοί**.

2. **Δημιουργήστε μια νέα προσαρμοσμένη διάσταση**

   Κάντε κλικ στο **Δημιουργία προσαρμοσμένων διαστάσεων**.

3. **Συμπληρώστε τις λεπτομέρειες της διάστασης**

   - **Όνομα διάστασης** — η ετικέτα που εμφανίζεται στις αναφορές GA4 (δείτε τον πίνακα παρακάτω)
   - **Πεδίο εφαρμογής** — επιλέξτε **Αντικείμενο**
   - **Παράμετρος συμβάντος** — το ακριβές όνομα παραμέτρου όπως εμφανίζεται στο dataLayer (δείτε τον πίνακα παρακάτω)
   - **Περιγραφή** — προαιρετική αλλά συνιστάται

4. **Αποθηκεύστε και επαναλάβετε**

   Κάντε κλικ στο **Αποθήκευση** και επαναλάβετε για κάθε παράμετρο που θέλετε να παρακολουθείτε.

</Steps>

<Aside type="note">
Οι προσαρμοσμένες διαστάσεις μπορεί να χρειαστούν έως και 24 ώρες για να αρχίσουν να εμφανίζονται στις αναφορές GA4 μετά την καταχώριση. Τα ιστορικά δεδομένα που συλλέχθηκαν πριν την καταχώριση δεν συμπληρώνονται αναδρομικά.

Το GA4 επιτρέπει έως 50 προσαρμοσμένες διαστάσεις σε επίπεδο αντικειμένου ανά ιδιοκτησία σε τυπικές ιδιοκτησίες (οι ιδιοκτησίες 360 επιτρέπουν περισσότερες).
</Aside>

Χρησιμοποιήστε τον παρακάτω πίνακα ως αναφορά κατά τη δημιουργία προσαρμοσμένων διαστάσεων. Η τιμή **Παράμετρος συμβάντος** πρέπει να ταιριάζει ακριβώς — το GA4 είναι ευαίσθητο σε πεζά-κεφαλαία.

| Όνομα διάστασης | Παράμετρος συμβάντος | Τύπος |
|-----------------|----------------------|-------|
| Αναγνωριστικό ακινήτου | `property_id` | Κείμενο |
| Όνομα ακινήτου | `property_name` | Κείμενο |
| Μάρκα ακινήτου | `property_brand` | Κείμενο |
| Αλυσίδα ακινήτου | `property_chain` | Κείμενο |
| Όνομα URL ακινήτου | `property_url_name` | Κείμενο |
| Όνομα πόλης | `city_name` | Κείμενο |
| Κωδικός χώρας | `country_code` | Κείμενο |
| Ημερομηνία check-in | `start_date` | Κείμενο |
| Ημερομηνία check-out | `end_date` | Κείμενο |
| Αριθμός ενηλίκων | `num_adults` | Αριθμός |
| Αριθμός παιδιών | `num_children` | Αριθμός |
| Πολιτική ακύρωσης | `cancellation_policy` | Κείμενο |
| Αναγνωριστικό τιμολογιακού πλάνου | `rate_plan_id` | Κείμενο |
| Αναγνωριστικό τιμής δωματίου | `room_rate_id` | Κείμενο |
| Όνομα τιμής δωματίου | `room_rate_name` | Κείμενο |

### Επιπλέον Διαθέσιμα Συμβάντα

Τα παρακάτω συμβάντα είναι διαθέσιμα στο dataLayer του Wink αλλά δεν αποτελούν μέρος του τυπικού χωνιού κράτησης. Μπορεί να ενεργοποιούνται από άλλες αλληλεπιδράσεις στην πλατφόρμα.

#### `add_to_wishlist`

```json
{
  "event": "add_to_wishlist",
  "ecommerce": {
    "currency": "USD",
    "value": 299.00,
    "items": [ ]
  }
}
```

#### `remove_from_cart`

```json
{
  "event": "remove_from_cart",
  "ecommerce": {
    "currency": "USD",
    "value": 299.00,
    "items": [ ]
  }
}
```

```json
{
  "event": "select_item",
  "ecommerce": {
    "item_list_id": "featured_hotels",
```

#### `view_cart`


### `virtualPageView`

Αυτό είναι ένα προσαρμοσμένο συμβάν Wink — δεν αποτελεί μέρος του τυπικού συνόλου συμβάντων GA4. Στέλνεται σε κάθε πλοήγηση σελίδας και μεταφέρει πληροφορίες συσκευής και συνεδρίας. Δεν περιλαμβάνει αντικείμενο `ecommerce`.

```json
{
  "event": "virtualPageView",
  "device": "Desktop",
  "deviceAgent": "Mozilla/5.0 ...",
  "os": "MacOS",
  "path": "/hotels/grand-resort",
  "title": "Grand Resort — Wink",
  "userId": "user_abc123",
  "startDate": "2026-06-01",
  "endDate": "2026-06-05",
  "guests": "2",
  "rooms": "1"
}
```

| Ιδιότητα | Περιγραφή |
|----------|-----------|
| `device` | Ανιχνευμένος τύπος συσκευής — `Desktop`, `Android`, `iPhone`, `iPad` ή παρόμοιο |
| `deviceAgent` | Ακατέργαστη συμβολοσειρά user agent |
| `os` | Ανιχνευμένο λειτουργικό σύστημα — `Windows`, `MacOS`, `Linux`, `UNIX` |
| `path` | Τρέχουσα διαδρομή σελίδας |
| `title` | Τρέχων τίτλος σελίδας |
| `userId` | Αναγνωριστικό αυθεντικοποιημένου χρήστη (παραλείπεται αν δεν είναι συνδεδεμένος) |
| `startDate` | Ημερομηνία check-in από την τρέχουσα συνεδρία αναζήτησης |
| `endDate` | Ημερομηνία check-out από την τρέχουσα συνεδρία αναζήτησης |
| `guests` | Αριθμός επισκεπτών από την τρέχουσα συνεδρία αναζήτησης |
| `rooms` | Αριθμός δωματίων από την τρέχουσα συνεδρία αναζήτησης |
