Εάν πραγματοποιείτε διαφημιστικές καμπάνιες μέσω Sojern, μπορείτε να συνδέσετε τον λογαριασμό σας Sojern με το Wink εισάγοντας το Sojern GTM Container ID σας στην πύλη Wink. Μόλις συνδεθεί, το pixel παρακολούθησης του Sojern ενεργοποιείται αυτόματα για το ακίνητό σας — καλύπτοντας ολόκληρο το ταξίδι του επισκέπτη από την αναζήτηση έως την ολοκλήρωση της κράτησης.

<Aside type="tip">
Αυτό προϋποθέτει ότι έχετε ήδη ενεργό λογαριασμό Sojern και έχετε λάβει το GTM Container ID από τον διαχειριστή του λογαριασμού σας Sojern. Εάν δεν έχετε ακόμη λογαριασμό Sojern, επισκεφθείτε το [sojern.com/contact](https://www.sojern.com/contact) για να ξεκινήσετε.
</Aside>

---

## Τι Πρέπει να Κάνετε

<Steps>

1. **Λάβετε το GTM Container ID από το Sojern**

   Επικοινωνήστε με τον διαχειριστή του λογαριασμού σας Sojern και ζητήστε το **GTM Container ID** σας. Ακολουθεί τη μορφή `GTM-XXXXXXX`. Αυτό διαφέρει από το Sojern Pixel ID — βεβαιωθείτε ότι ζητάτε συγκεκριμένα το GTM Container ID.

2. **Εισάγετε το στην Πύλη Wink**

   Συνδεθείτε στο [my.wink.travel](https://my.wink.travel), επιλέξτε τον λογαριασμό σας και μεταβείτε στην καρτέλα **Customizations**. Επικολλήστε το Sojern GTM Container ID στο πεδίο Sojern και αποθηκεύστε. Η ενσωμάτωσή σας είναι πλέον ενεργή.

3. **Επιβεβαιώστε με τον Διαχειριστή του Λογαριασμού Sojern**

   Ενημερώστε τον διαχειριστή του λογαριασμού σας Sojern ότι η ενσωμάτωση είναι ενεργή. Μπορούν να επαληθεύσουν από την πλευρά τους ότι το pixel ενεργοποιείται σωστά στο `wink.travel` για το ακίνητό σας και ότι τα δεδομένα ρέουν στον λογαριασμό σας Sojern.

</Steps>

<Aside type="note">
Δεν χρειάζεται να τοποθετήσετε κανέναν κώδικα pixel στην ιστοσελίδα σας για αυτήν την ενσωμάτωση. Το Wink φορτώνει αυτόματα το container Sojern σε όλες τις σελίδες όπου το ακίνητό σας είναι ενεργό.

Αυτή η ρύθμιση λειτουργεί για οποιοδήποτε προσαρμοσμένο GTM Container ID. Χρησιμοποιήσαμε το Sojern ως παράδειγμα επειδή είναι μια πολύ κοινή ενσωμάτωση.
</Aside>

## Τι Κάνει το Wink από τη Δική μας Πλευρά

Για διαφάνεια, εδώ είναι τι ρυθμίζει το Wink για να λειτουργήσει αυτή η ενσωμάτωση:

- **Υπάρχει πεδίο Sojern GTM Container ID** στην καρτέλα Customizations της πύλης Wink σας. Όταν εισάγετε το Container ID, το Wink ελέγχει τη μορφή και το αποθηκεύει με ασφάλεια για το ακίνητό σας.
- **Σε κάθε σελίδα όπου το ακίνητό σας είναι ενεργό** στο Wink, το Sojern GTM Container ID σας αποδίδεται στη σελίδα και το σύστημα διαχείρισης ετικετών του Wink φορτώνει δυναμικά το container Sojern παράλληλα με το δικό του. Αυτό γίνεται αυτόματα — δεν απαιτούνται χειροκίνητα βήματα από την ομάδα Wink ανά ακίνητο.
- **Το container Sojern φορτώνεται ασύγχρονα**, που σημαίνει ότι δεν επηρεάζει την απόδοση φόρτωσης της σελίδας για τους επισκέπτες σας.
- **Η δική μας παρακολούθηση και το pixel Sojern τρέχουν παράλληλα**, διαβάζοντας από τα ίδια κοινά δεδομένα συμβάντων. Κάθε συμβάν κράτησης που παρακολουθεί το Wink είναι ταυτόχρονα διαθέσιμο στο container Sojern.
- **Όταν το ακίνητό σας δεν προβάλλεται** — για παράδειγμα, όταν ένας επισκέπτης περιηγείται σε διαφορετικό ξενοδοχείο στο Wink — το container Sojern δεν φορτώνεται. Αυτό διασφαλίζει ότι το Sojern λαμβάνει μόνο δεδομένα σχετικά με το ακίνητό σας.

## Αναφορά DataLayer

Το Wink στέλνει συμβάντα συμβατά με GA4 στο `dataLayer` καθ’ όλη τη διάρκεια του ταξιδιού του επισκέπτη. Το container GTM σας διαβάζει από το ίδιο κοινό `dataLayer`, οπότε οποιοδήποτε trigger ή μεταβλητή ρυθμίσετε έχει πρόσβαση σε όλες τις ιδιότητες που περιγράφονται παρακάτω.

Κάθε συμβάν ακολουθεί τη δομή του GA4 Enhanced Ecommerce: μια κορυφαία συμβολοσειρά `event` και ένα αντικείμενο `ecommerce`. Πριν από κάθε push, το Wink καθαρίζει το προηγούμενο payload ecommerce για να αποτρέψει τη διαρροή παλαιών δεδομένων σε νέα triggers.

<Aside type="note">
**Πρότυπο επαναφοράς dataLayer**

Το Wink επαναφέρει το αντικείμενο ecommerce πριν από κάθε push συμβάντος:

```js
dataLayer.push({ ecommerce: null });   // καθαρισμός προηγούμενου συμβάντος
dataLayer.push({ event: '...', ecommerce: { ... } });
```

Μην βασίζεστε στην επιμονή των ιδιοτήτων ecommerce μεταξύ των συμβάντων. Διαβάζετε πάντα τις ιδιότητες κατά το χρόνο ενεργοποίησης μέσα στο ίδιο συμβάν.
</Aside>

### Συμβάντα Ταξιδιού Κράτησης

Αυτά τα συμβάντα ενεργοποιούνται αυτόματα καθώς ο επισκέπτης προχωρά μέσα στο funnel κράτησης. Ρυθμίστε τα triggers GTM σας ως **Custom Event** triggers που ταιριάζουν με τα ονόματα των συμβάντων παρακάτω.

| Βήμα | Συμβάν | Πότε Ενεργοποιείται |
|------|--------|--------------------|
| 1 | `view_item_list` | Φόρτωση πλέγματος ή λίστας ξενοδοχείων |
| 2 | `view_item` | Ο επισκέπτης βλέπει συγκεκριμένη σελίδα ακινήτου |
| 3 | `add_to_cart` | Ο επισκέπτης προσθέτει δωμάτιο στο καλάθι |
| 4 | `begin_checkout` | Φόρτωση σελίδας ολοκλήρωσης αγοράς / πληρωμής |
| 5 | `add_payment_info` | Ο επισκέπτης επιλέγει μέθοδο πληρωμής |
| 6 | `login` | Ο επισκέπτης αυθεντικοποιείται μέσω SSO |
| 7 | `purchase` | Η κράτηση επιβεβαιώνεται |
| 8 | `refund` | Η κράτηση ακυρώνεται |

---

#### `view_item_list`

Ενεργοποιείται όταν αποδίδεται πλέγμα ή λίστα ξενοδοχείων.

```json
{
  "event": "view_item_list",
  "ecommerce": {
    "item_list_id": "featured_hotels",
    "item_list_name": "Featured Hotels",
    "items": [ ]
  }
}
```

---

#### `view_item`

Ενεργοποιείται όταν ο επισκέπτης βλέπει συγκεκριμένο ακίνητο.

```json
{
  "event": "view_item",
  "ecommerce": {
    "currency": "USD",
    "value": 299.00,
    "items": [ ]
  }
}
```

---

#### `add_to_cart`

Ενεργοποιείται όταν ο επισκέπτης προσθέτει δωμάτιο στο καλάθι.

```json
{
  "event": "add_to_cart",
  "ecommerce": {
    "currency": "USD",
    "value": 299.00,
    "items": [ ]
  }
}
```

---

#### `begin_checkout`

Ενεργοποιείται όταν φορτώνει η σελίδα ολοκλήρωσης αγοράς / πληρωμής.

```json
{
  "event": "begin_checkout",
  "ecommerce": {
    "currency": "USD",
    "value": 299.00,
    "coupon": "SUMMER10",
    "items": [ ]
  }
}
```

Το `coupon` υπάρχει μόνο όταν έχει εφαρμοστεί κωδικός έκπτωσης.

---

#### `add_payment_info`

Ενεργοποιείται όταν ο επισκέπτης επιλέγει ή επιβεβαιώνει μέθοδο πληρωμής.

```json
{
  "event": "add_payment_info",
  "ecommerce": {
    "currency": "USD",
    "value": 299.00,
    "coupon": "SUMMER10",
    "payment_type": "STRIPE",
    "items": [ ]
  }
}
```

Το `payment_type` προέρχεται από τον τύπο του αποδέκτη πληρωμής. Το `coupon` υπάρχει μόνο όταν έχει εφαρμοστεί κωδικός έκπτωσης.

---

#### `login`

Ενεργοποιείται όταν ο επισκέπτης αυθεντικοποιείται μέσω single sign-on.

```json
{
  "event": "login",
  "ecommerce": {
    "method": "IAM"
  }
}
```

---

#### `purchase`

Ενεργοποιείται όταν επιβεβαιώνεται μια κράτηση.

```json
{
  "event": "purchase",
  "ecommerce": {
    "currency": "USD",
    "transaction_id": "bc-a1b2c3d4",
    "value": 299.00,
    "coupon": "SUMMER10",
    "shipping": 0.00,
    "tax": 0.00,
    "items": [
      {
        "item_id": "rr-xyz789",
        "item_name": "Deluxe Ocean Suite",
        "item_brand": "Grand Resort",
        "item_category": "GUEST_ROOM",
        "price": 299.00,
        "quantity": 1,
        "property_id": "prop-abc123",
        "property_name": "Grand Resort Maldives",
        "property_brand": "Grand Collection",
        "property_chain": "Luxury Hotels International",
        "city_name": "Malé",
        "country_code": "MV",
        "start_date": "2026-06-01",
        "end_date": "2026-06-05",
        "num_adults": 2,
        "num_children": 0,
        "cancellation_policy": "Refundable",
        "rate_plan_id": "rp-summer2026",
        "room_rate_id": "rr-xyz789",
        "room_rate_name": "Summer Escape — Bed & Breakfast",
        "num_perks": 2,
        "room_location": "Overwater",
        "room_view": "Ocean",
        "room_classification": "Suite"
      }
    ]
  }
}
```

Το `transaction_id` είναι ο μοναδικός αναγνωριστικός αριθμός σύμβασης κράτησης. Τα `coupon`, `shipping` και `tax` υπάρχουν μόνο όταν ισχύουν.

---

#### `refund`

Ενεργοποιείται όταν ακυρώνεται μια κράτηση.

```json
{
  "event": "refund",
  "ecommerce": {
    "currency": "USD",
    "transaction_id": "bc-a1b2c3d4",
    "value": 299.00,
    "coupon": "SUMMER10",
    "shipping": 0.00,
    "tax": 0.00,
    "items": [ ]
  }
}
```

Το `transaction_id` ταιριάζει με το συμβάν `purchase` για την ίδια κράτηση. Τα `coupon`, `shipping` και `tax` υπάρχουν μόνο όταν ισχύουν.

### Ιδιότητες Αντικειμένων

Όλα τα συμβάντα που περιλαμβάνουν πίνακα `items` μοιράζονται το ίδιο σχήμα αντικειμένου. Κάθε ιδιότητα είναι προαιρετική εκτός αν σημειώνεται ως υποχρεωτική.

#### Τυπικά πεδία GA4

Αυτά τα πεδία αναγνωρίζονται εγγενώς από το GA4 και εμφανίζονται σε τυπικές αναφορές ecommerce χωρίς επιπλέον ρύθμιση.

| Ιδιότητα | Τύπος | Υποχρεωτικό | Περιγραφή |
|----------|-------|-------------|-----------|
| `item_id` | string | Ναι | Μοναδικός αναγνωριστικός αριθμός για το δωμάτιο ή προϊόν |
| `item_name` | string | — | Όνομα δωματίου ή προϊόντος |
| `affiliation` | string | — | Όνομα συνεργάτη ή πωλητή |
| `coupon` | string | — | Κωδικός κουπονιού που εφαρμόστηκε στο αντικείμενο |
| `currency` | string | — | Κωδικός νομίσματος ISO 4217 (π.χ. `USD`) |
| `creative_name` | string | — | Όνομα δημιουργικού που χρησιμοποιήθηκε σε προώθηση |
| `creative_slot` | string | — | Θέση του δημιουργικού στην προώθηση |
| `discount` | number | — | Ποσό έκπτωσης που εφαρμόστηκε |
| `index` | number | — | Θέση στη λίστα (ξεκινά από 1) |
| `item_brand` | string | — | Όνομα ξενοδοχείου ή μάρκας |
| `item_category` | string | — | Τύπος προϊόντος — π.χ. `GUEST_ROOM`, `ACTIVITY`, `ATTRACTION`, `MEETING_ROOM`, `RESTAURANT`, `SPA` |
| `item_list_id` | string | — | Αναγνωριστικό της λίστας στην οποία ανήκει το αντικείμενο |
| `item_list_name` | string | — | Εμφανιζόμενο όνομα της λίστας |
| `location_id` | string | — | Γεωγραφικές συντεταγμένες του ακινήτου |
| `price` | number | — | Τιμή μονάδας |
| `promotion_id` | string | — | Αναγνωριστικό της προώθησης |
| `promotion_name` | string | — | Εμφανιζόμενο όνομα της προώθησης |
| `quantity` | number | — | Ποσότητα (προεπιλογή `1`) |

| Ιδιότητα | Τύπος | Περιγραφή |

| `num_children` | number | Αριθμός παιδιών στην κράτηση |
| `cancellation_policy` | string | Ένα από: `Refundable`, `Non-Refundable`, `Refundable-With-Restrictions`, `Non-Refundable-With-Restrictions` |

| `room_location` | string | Περιγραφή τοποθεσίας δωματίου (π.χ. `Overwater`, `Garden`) |
| `room_view` | string | Περιγραφή θέας δωματίου (π.χ. `Ocean`, `City`) |
| `room_classification` | string | Κατάταξη ή κατηγορία δωματίου (π.χ. `Suite`, `Standard`) |

### Προσαρμοσμένες Διαστάσεις στο GA4

Το GA4 αγνοεί τις προσαρμοσμένες παραμέτρους αντικειμένων στις αναφορές μέχρι να τις καταχωρήσετε ρητά ως προσαρμοσμένες διαστάσεις σε επίπεδο αντικειμένου. Πρέπει να καταχωρήσετε μόνο τις παραμέτρους που σκοπεύετε να χρησιμοποιήσετε στις αναφορές — δεν χρειάζεται να καταχωρήσετε και τις 18.

<Steps>

1. **Ανοίξτε τις Προσαρμοσμένες Ορισμούς**

   Στο GA4, πηγαίνετε στο **Admin** (εικονίδιο γραναζιού κάτω αριστερά) → επιλέξτε την ιδιοκτησία σας → **Data display** → **Custom definitions**.

2. **Δημιουργήστε μια νέα προσαρμοσμένη διάσταση**

   Κάντε κλικ στο **Create custom dimensions**.

3. **Συμπληρώστε τις λεπτομέρειες της διάστασης**

   - **Dimension name** — η ετικέτα που εμφανίζεται στις αναφορές GA4 (δείτε τον πίνακα παρακάτω)
   - **Scope** — επιλέξτε **Item**
   - **Event parameter** — το ακριβές όνομα παραμέτρου όπως εμφανίζεται στο dataLayer (δείτε τον πίνακα παρακάτω)
   - **Description** — προαιρετικό αλλά συνιστάται

4. **Αποθηκεύστε και επαναλάβετε**

   Κάντε κλικ στο **Save** και επαναλάβετε για κάθε παράμετρο που θέλετε να παρακολουθείτε.

</Steps>

<Aside type="note">
Οι προσαρμοσμένες διαστάσεις μπορεί να χρειαστούν έως και 24 ώρες για να αρχίσουν να εμφανίζονται στις αναφορές GA4 μετά την καταχώρηση. Τα ιστορικά δεδομένα που συλλέχθηκαν πριν την καταχώρηση δεν συμπληρώνονται αναδρομικά.

Το GA4 επιτρέπει έως 50 προσαρμοσμένες διαστάσεις σε επίπεδο αντικειμένου ανά ιδιοκτησία σε τυπικές ιδιοκτησίες (οι ιδιοκτησίες 360 επιτρέπουν περισσότερες).
</Aside>

Χρησιμοποιήστε τον παρακάτω πίνακα ως αναφορά κατά τη δημιουργία προσαρμοσμένων διαστάσεων. Η τιμή **Event parameter** πρέπει να ταιριάζει ακριβώς — το GA4 είναι case-sensitive.

| Όνομα διάστασης | Event parameter | Τύπος |
|-----------------|-----------------|-------|
| Property ID | `property_id` | Κείμενο |
| Property name | `property_name` | Κείμενο |
| Property brand | `property_brand` | Κείμενο |
| Property chain | `property_chain` | Κείμενο |
| City name | `city_name` | Κείμενο |
| Country code | `country_code` | Κείμενο |
| Check-in date | `start_date` | Κείμενο |
| Check-out date | `end_date` | Κείμενο |
| Number of adults | `num_adults` | Αριθμός |
| Number of children | `num_children` | Αριθμός |
| Cancellation policy | `cancellation_policy` | Κείμενο |
| Rate plan ID | `rate_plan_id` | Κείμενο |
| Room rate ID | `room_rate_id` | Κείμενο |
| Room rate name | `room_rate_name` | Κείμενο |
| Number of perks | `num_perks` | Αριθμός |
| Room location | `room_location` | Κείμενο |
| Room view | `room_view` | Κείμενο |
| Room classification | `room_classification` | Κείμενο |

### Επιπλέον Διαθέσιμα Συμβάντα

Τα παρακάτω συμβάντα είναι διαθέσιμα στο dataLayer του Wink αλλά δεν αποτελούν μέρος του τυπικού funnel κράτησης. Μπορεί να ενεργοποιούνται από άλλες αλληλεπιδράσεις στην πλατφόρμα.

#### `add_to_wishlist`

```json
{
  "event": "add_to_wishlist",
  "ecommerce": {
    "currency": "USD",
    "value": 299.00,
    "items": [ ]
  }
}
```

#### `remove_from_cart`

```json
{
  "event": "remove_from_cart",
  "ecommerce": {
    "currency": "USD",
    "value": 299.00,
    "items": [ ]
  }
}
```

#### `select_item`

```json
{
  "event": "select_item",
  "ecommerce": {
    "item_list_id": "featured_hotels",
    "item_list_name": "Featured Hotels",
    "items": [ ]
  }
}
```

#### `view_cart`

```json
{
  "event": "view_cart"
}
```

Δεν περιλαμβάνεται payload `ecommerce` με αυτό το συμβάν.

#### `view_promotion`

```json
{
  "event": "view_promotion",
  "ecommerce": {
    "creative_name": "Summer Banner",
    "creative_slot": "hero",

  }
}
```

### `virtualPageView`

Αυτό είναι ένα προσαρμοσμένο συμβάν Wink — δεν αποτελεί μέρος του τυπικού συνόλου συμβάντων GA4. Στέλνεται σε κάθε πλοήγηση σελίδας και μεταφέρει πληροφορίες συσκευής και συνεδρίας. Δεν περιλαμβάνει αντικείμενο `ecommerce`.

```json
{
  "event": "virtualPageView",
  "device": "Desktop",
  "deviceAgent": "Mozilla/5.0 ...",
  "os": "MacOS",
  "path": "/hotels/grand-resort",
  "title": "Grand Resort — Wink",
  "userId": "user_abc123",
  "startDate": "2026-06-01",
  "endDate": "2026-06-05",
  "guests": "2",
  "rooms": "1"
}
```

| Ιδιότητα | Περιγραφή |
|----------|-----------|
| `device` | Τύπος ανιχνευμένης συσκευής — `Desktop`, `Android`, `iPhone`, `iPad` ή παρόμοιο |
| `deviceAgent` | Ακατέργαστη συμβολοσειρά user agent |
| `os` | Ανιχνευμένο λειτουργικό σύστημα — `Windows`, `MacOS`, `Linux`, `UNIX` |
| `path` | Τρέχουσα διαδρομή σελίδας |
| `title` | Τρέχων τίτλος σελίδας |
| `userId` | Αναγνωριστικό αυθεντικοποιημένου χρήστη (παραλείπεται αν δεν έχει συνδεθεί) |
| `startDate` | Ημερομηνία check-in από την τρέχουσα συνεδρία αναζήτησης |
| `endDate` | Ημερομηνία check-out από την τρέχουσα συνεδρία αναζήτησης |
| `guests` | Αριθμός επισκεπτών από την τρέχουσα συνεδρία αναζήτησης |
| `rooms` | Αριθμός δωματίων από την τρέχουσα συνεδρία αναζήτησης |
