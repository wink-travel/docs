Ако сте интегратор, као што је Wink, и желите да мапирате своје екстерне примаоце плаћања за коришћење са TripPay-ом, то можете учинити на један од два начина:

* Ручно мапирање
* Коришћење [API за плаћање](/developers/apis/#payment-api) (*Види`Account Mappings`*)

:::савет
Осим ако не радите само са неколико истих својстава, препоручујемо вам да користите API за аутоматизацију овог корака.
:::

У наставку ћемо вас провести кроз ручни начин мапирања екстерног примаоца плаћања.*Користи API испод хаубе.*

<Steps>
  1. Пријавите се [ТрипПеј](https://pay.wink.travel).
  2. Изаберите свој интеграторски налог.
  3. Кликните `Settings > Mappings` из главне навигационе траке.
  4. Кликните на `Create mapping` дугме. Видећете образац за мапирање.
  5. **Локални налог** Почните да куцате име постојећег TripPay-а који желите да мапирате.
  6. **Спољни идентификатор** Налепите свој екстерни ИД за овај ентитет онако како постоји у удаљеном систему.
  7. Кликните `Save` да бисте наставили. Бићете преусмерени назад на вашу листу мапирања.
</Steps>


Када су ваша мапирања на месту, TripPay сада зна које налоге треба да припише као кориснике резервације када их позовете користећи своје екстерне идентификаторе.

## API

Програмери који желе да управљају`Payments`, иди на[Програмери > API > Плаћања](/developers/apis/#payment-api).
