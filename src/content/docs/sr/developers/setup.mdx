Ovaj članak vam pokazuje kako brzo da se podesite kao programer i počnete da testirate naše API-je na Wink-u.

:::note
Testiranje bi uvek trebalo da se odvija u našem [staging okruženju](/getting-started/environments). Tamo ima mnogo manje podataka, ali slobodno možete testirati koliko god želite bez ometanja produkcionog okruženja.

Kada budete spremni za lansiranje, mi ćemo sertifikovati vaš nalog i vašu aplikaciju i moći ćete da pokrenete u našem produkcionom okruženju.
:::

Pratite ove korake:

<Steps>
1. [Registrujte se](/user/register) kao korisnik na Wink-u
2. Kreirajte [nalog turističkog agenta](/travel-agent/what-is-agent) ili [partnerski nalog](/studio/what-is-studio).
3. [Kreirajte aplikaciju](/developer/applications) i povežite je sa nalogom koji ste napravili u koraku 2.
4. Kada kreirate `Application`, sačuvajte client-id i client secret na sigurnom mestu.
</Steps>

## CDN

Na Wink-u koristimo Cloudinary za sve naše slike i video sadržaje. Kada pristupite našem turističkom inventaru, imaćete potpuni pristup svim naprednim funkcijama Cloudinary-ja.

Naš `cloud_name` je `traveliko`.

### Cloudinary

- [Početak rada](https://cloudinary.com/documentation/dev_kickstart)
- [SDK-ovi](https://cloudinary.com/documentation/cloudinary_sdks)
- [Cloudinary na GitHub-u](https://github.com/cloudinary)
-	[Video tutorijali](https://cloudinary.com/documentation/cloudinary_sdks_tutorials)

### Primer

Za svaki tip turističkog inventara koji podržavamo, postoji ugnježdeni JSON niz pod nazivom `multimedias`. JSON za jedan multimedijalni unos izgleda ovako:

```json {3}
{
  "multimediaIdentifier": "multimedia-1",
  "identifier": "partners/radisson_blu_logo",
  "type": "IMAGE",
  "width": "1024",
  "height": "768",
  "category": "1",
  "descriptions": [
    {
      "name": "Lobby",
      "description": "Slika lobija",
      "language": "en"
    }
  ],
  "lifestyleType": "LIFESTYLE_BUSINESS",
  "attribution": [
    {
      "url": "https://www.hilton.com",
      "name": "Hilton"
    }
  ]
}
```

Istaknuti red prikazuje jedinstveni Cloudinary ID `partners/radisson_blu_logo`. To je sve što vam treba da preuzmete sliku u bilo kom formatu, kvalitetu i veličini koju želite.

## Testiranje

Idite u našu sekciju [API-je](/developer/apis) i izaberite API sa kojim želite da radite.

### API dokumentacija

Naš sajt za dokumentaciju je postavljen tako da možete izvršavati API komande direktno sa sajta.

U ovom primeru, želite da koristite API poziv `Search Lookups (OAuth2)`.

:::tip[Pro savet]
Za lakšu upotrebu, preporučujemo integratorima da koriste krajnje tačke koje označavamo kao `OAuth2`. Te krajnje tačke znaju da očekuju API pozivaoca i već znaju na kom ste nalogu, tako da ne morate da navodite `companyIdentifier` u URL-ovima.
:::

<Steps>
1. Koristite [Lookup API](https://developer.wink.travel/lookup) da pretražite destinacije ili hotele.
2. U levom meniju kliknite na `Authentication`.
3. Unesite vaš client-id i client secret i kliknite na dugme `Get TOKEN`.
4. Ako su vaše akreditive validne, videćete da je vaš API ključ primenjen.
5. Sada možete testirati lookup endpoint.
</Steps>

### Postman
Isto možete uraditi i pomoću Postman-a.

<Steps>
1. Preuzmite [Postman](https://www.postman.com/downloads).
2. Kliknite na `Download OpenAPI spec` na vrhu stranice [Lookup API](https://developer.wink.travel/lookup).
3. Uvezite ga u Postman.
4. Ako ste ga uvezli kao novu kolekciju pod imenom `Wink API`, kliknite na horizontalne tri tačke pored imena kolekcije i izaberite `Edit`.
5. U tabu `Authorization`, podesite `Auth type` na OAuth2.
6. Podesite polje `Add auth data to` na `Request headers`.
7. Podesite polje `Header prefix` na `Bearer`.
8. Podesite polje `Token Name` na ime koje želite da dodelite vašem sačuvanom tokenu u Postman-u, npr. Wink Staging.
9. Podesite polje `Grant type` na `Client Credentials`.
10. Podesite polje `Access token URL` na https://staging-iam.wink.travel/oauth2/token.
11. Podesite polje `Client ID` na Client ID koji je vaša aplikacija generisala.
12. Podesite polje `Client Secret` na Client Secret koji je vaša aplikacija generisala.
13. Podesite polje `Scope` na `inventory.read inventory.write inventory.remove`. To su svi opsezi koji su vam potrebni za Wink.
14. Podesite polje `Client Authentication` na `Send as Basic Auth header`.
15. Kliknite na dugme `Get New Access Token` da preuzmete token i sačuvate ga.
16. Za sve API pozive unutar Wink API kolekcije, proverite da li tab za autentifikaciju kaže `Inherit auth from parent` kako bi vaš token bio uključen u svaki poziv.
</Steps>

:::tip[Pro savet]
Preporučuje se da domen servera, client ID i client secret budu postavljeni kao promenljive okruženja. Na taj način lako možete menjati između staging i produkcionih okruženja i ponovo koristiti sve operacije koje ste kreirali.
:::
