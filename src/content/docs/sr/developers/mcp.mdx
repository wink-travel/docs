## Pregled
Hostovani Wink MCP server dostupan je na:

```
https://docs.mcp.wink.travel/mcp
```

Usmerite kompatibilne AI klijente (VS Code, Cursor, i druge koje implementiraju MCP HTTP transport) na ovu adresu da bi mogli:
- Pregledati dokumentaciju proizvoda i platforme (`wink://docs/...`)
- Pregledati aktivne OpenAPI specifikacije i dokumentaciju operacija (`wink://api/...`)
- Izvršiti laganu semantičku / ključnu pretragu kroz dokumentaciju i aktivne API-je

## Transport
Koristite primarni HTTP endpoint koji podržava streaming:

```
https://docs.mcp.wink.travel/mcp
```

Ako vaš klijent podržava samo SSE, koristite:

```
https://docs.mcp.wink.travel/sse
```

### Claude Code
Zvanična dokumentacija: [Povežite Claude Code sa alatima putem MCP](https://code.claude.com/docs/en/mcp)

Anthropic preporučuje udaljeni HTTP transport za cloud-hostovane MCP servere:
```bash
claude mcp add --transport http wink-docs https://docs.mcp.wink.travel/mcp
claude mcp list
```

U Claude Code pokrenite `/mcp` da proverite status servera. Ako želite da delite server sa celim repozitorijumom, dodajte `--scope project` da Claude Code upiše konfiguraciju u `.mcp.json`.

### Codex
Zvanična dokumentacija: [Docs MCP quickstart za Codex](https://developers.openai.com/resources/docs-mcp)

OpenAI dokumentuje MCP podešavanje preko Codex CLI, i napominje da se konfiguracija deli sa IDE ekstenzijom:
```bash
codex mcp add wink-docs --url https://docs.mcp.wink.travel/mcp
codex mcp list
```

### Cursor
Cursor MCP dokumentacija: https://cursor.com/docs/context/mcp
Podešavanje: Dodajte udaljeni server preko MCP podešavanja.
```json
{
  "mcpServers": {
    "wink-docs": {
      "url": "https://docs.mcp.wink.travel/mcp",
    }
  }
}
```

### VS Code
Zvanična dokumentacija: https://code.visualstudio.com/docs/copilot/customization/mcp-servers
Minimalna udaljena konfiguracija (korisnička ili radna `mcp.json`):
```json
{
  "servers": {
    "wink-docs": {
      "type": "http",
      "url": "https://docs.mcp.wink.travel/mcp"
    }
  }
}
```

Saveti:
- Koristite Command Palette: “MCP: List Servers” da potvrdite vezu i pregledate alate.
- Izbor alata vam omogućava da uključite/isključite `api_search` i `docs_search`.
- Dodajte kontekst: Prikaz četa → Add Context → MCP Resources da prikačite stranice dokumentacije.

## Osnovni URI resursi

| URI | Opis |
|-----|-------------|
| `wink://docs` | Indeks dokumentacije (grupisan po kategorijama) |
| `wink://docs/{path}` | Specifičan dokumentacioni članak (Markdown / MDX) |
| `wink://api` | Aktivni OpenAPI katalog otkriven iz Wink swagger konfiguracije |
| `wink://api/{apiName}` | Kompaktni API rezime sa autentifikacijom, tagovima, headerima i primerima operacija |
| `wink://api/{apiName}/operations` | Kompletan spisak operacija grupisanih po tagu |
| `wink://api/{apiName}/operation/{key}` | Detalji pojedinačne operacije (parametri, telo zahteva, odgovori) |
| `wink://api/{apiName}/schema/{schemaName}` | Razrešena šema komponente sa ugnježdenim poljima, tipovima i primerima |
| `wink://api/{apiName}/operation/{key}/request` | Prikaz fokusiran na zahtev sa obaveznim ulazima i minimalnim primerom payload-a |
| `wink://api/{apiName}/operation/{key}/response/{status}` | Prikaz odgovora za određeni status kod |
| `wink://api/{apiName}/operation/{key}/recipe` | Tok zadatka za pravilno pozivanje te operacije |

## Alati za pretragu (brzi pregled)
| Alat | Svrha | Tipični primeri upita |
|------|---------|-----------------------|
| `api_search` | Pronađite aktivni API ili operaciju za integraciju | `create affiliate`, `payment receipt`, `lookup inventory`, `invite manager` |
| `docs_search` | Pronađite konceptualnu dokumentaciju / vodiče | `authentication`, `webhooks`, `wordPress` |

Ostavite `query` prazno da biste dobili predloge. Svaki rezultat uključuje MCP URI koji možete otvoriti.
Za rezultate operacija, `api_search` takođe vraća povezane URI-je za zahtev, odgovor, šemu i recepte, tako da agenti mogu direktno skočiti na najkorisniji podresurs.

## Tipičan obrazac korišćenja
1. Pokrenite `api_search` sa vašom namerom (npr. *"create affiliate"* ili *"payment receipt"*).
2. Otvorite vraćeni `wink://api/...` URI, ili pratite njegove povezane `request_uri`, `response_uris`, `request_schema_uris`, `response_schema_uris` i `recipe_uri` polja.
3. Koristite `docs_search` za podržavajuće koncepte (autentifikacija, webhooks, tokovi rada).

## AI veštine

Predefinisane veštine agenata koriste ovaj MCP server kao svoj izvor istine. Veštine daju vašem kodirajućem agentu strukturirani tok rada za rad sa Wink API-jima — umesto da pogađa krajnje tačke, agent direktno čita aktivne ugovore sa MCP servera.

Pogledajte [AI Skills](/sr/developers/ai-skills) za dostupne veštine i uputstva za instalaciju.

## Napomene
- Svaki vraćeni URI agent može otvoriti za kompletan sadržaj.
- `wink://api` stablo je aktivno i prati trenutni OpenAPI katalog umesto generisanog indeksa repozitorijuma.
- Predlozi se automatski pojavljuju kada su upiti prazni.
