:::note
最初のマスターレートの作成はオンボーディングウィザードの一部です。
:::

:::note
`マスターレート`とは、ユニークな`客室 / レートプラン`の組み合わせのことです。

*ほとんどの旅行料金提供者はこの文脈で客室とレートプランのみを使用します。OTAや予約エンジンで客室が販売されているのを見ると、旅行者はレートプランを比較しています。*

Winkはこのユニークな組み合わせにより柔軟性を持たせるために`マスターレート`という用語を追加しました。
:::

:::note
マスターレートは、客室またはレートプランが追加・削除されると自動的に作成・削除されます。
:::

マスターレートを管理するには、メインナビバーの`Monetize > Master Rate`をクリックしてください。

物件には、すべての客室 / レートプランの組み合わせごとにマスターレートがあります。

*例：物件に1つの客室と2つのレートプランがある場合、すべての条件が満たされていれば、旅行者は2つのマスターレートが販売されているのを確認できます。*

マスターレートのエントリーには、管理用の2つのオプションがあります：

- マスターレートの利用可能性の切り替え。
- 特典の設定。

## 利用可能性

マスターレートを有効または無効にすると、すべての販売チャネルでの利用可能性に影響します。このスイッチは特定のマスターレートをすぐにすべての場所でオフにするための簡単な方法です。

## 特典

旅行者が客室を予約しようとするとき、直接客室とやり取りするのではなく、その客室のマスターレートとやり取りします。
マスターレートには、その特定の客室 / レートプランの組み合わせにのみ適用されるユニークな特典を設定でき、物件オーナーにとってその組み合わせを促進する別の方法となります。

特典を選ぶ際に覚えておくべき2つのポイント：

- 保証されているか、利用可能性に基づくか。
- 数値的な価値。

保証された特典は、顧客にとってより価値が高いため、数値的な価値も高くなります。
すべてのホテルは、すべてのマスターレートにわたる特典の数値的価値の合計に基づく`特典スコア`を受け取ります。
お得なプランを探している旅行者は、特典スコアの高いホテルで並べ替えることができます。

特典は以下のための手段です：

- 客室 / レートプランの組み合わせを差別化する。
- より高い可視性を得る。*例：地域内の他のホテルよりも特典スコアのスケールで高得点を獲得する。*

:::tip
特典は、ホテルが自分たちを差別化し、`システムを攻略する`という欲求から生まれました。
前回それを行ったとき、彼らはより高いコミッションでOTAにすべての利益を渡してしまい、自らを苦しめました。
歴史的に、コミッションはホテルが攻略できる唯一の要素でした。

特典は、ホテルにとって実際のコストがほとんどない形で旅行者に報いる方法です。

もし他の種類の特典の提案があれば、[ぜひご連絡ください](mailto:hi@wink.travel)。
:::

販売チャネルの在庫を有効にする方法は2つあります：

- カード上で在庫の利用可能性を切り替える。*上の図を参照*
- メインナビバーの`Distribution > Inventory`に移動し、すべての販売チャネルの在庫を管理する。

## API

`Master Rates`を管理したい開発者は、[Developers > APIs > Monetize](/ja/developers/apis/#monetize-api)にアクセスしてください。
