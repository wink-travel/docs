料金プランは、ゲストルームをユーザーに提示するためのフィルターです。
また、特定のシナリオでの価格構造を制御する設定も含まれています。

料金プランでできること：

- キャンセルポリシー[およびポリシー例外]の設定。
- 追加の子供、大人、単独旅行者の料金設定。
- 食事の設定。
- 早めのチェックインおよび遅めのチェックアウト料金の追加。
- 追加料金の設定。
- 制限の設定。

料金プランを作成するには、`Create a new rate plan` ボタンをクリックしてください。

## 設定

開始時にデフォルトで選択されているのは `Configurations` タブです。

このセクションでは、料金プランの基本的なオプション設定について説明します。

- **料金プラン名** 内部で参照する料金プランの名前。*例：BAR RO*
- **有効** この料金プランをオンまたはオフに切り替えます。無効にすると、どの販売チャネルでも予約できなくなります。
- **前払い** この料金プランが予約時に前払いを必要とするかどうかを設定します。
- **ロイヤルティポイント** この料金プランでゲストがロイヤルティポイントを獲得するかどうか。*例：いいえ*
- **シングル占有料金修正** 複数人部屋で単独占有者に対する価格修正を設定。*例：通常価格の5%割引*
- **追加大人料金修正** 部屋に追加される大人1名あたりの価格修正を設定。*例：大人1名につき+10*
- **追加子供料金修正** 部屋に追加される子供1名あたりの価格修正を設定。*例：子供1名につき+5*

### キャンセルポリシー

- **キャンセルポリシー** この料金プランのキャンセルポリシーを設定。*例：返金可能*
- **例外** 特定の日付範囲に適用される追加のキャンセルポリシーを設定。*例：12月の期間は「追加条件付き返金可能」ポリシーを使用。*

### 食事

この料金プランに含まれる食事プランを選択します。以下のオプションがあります：

- **朝食**
- **ブランチ**
- **昼食**
- **夕食**
- **オールインクルーシブ**
- **オールインクルーシブ＋アルコール**

## 追加料金

料金プランに付随する追加料金を設定します。

- **早めのチェックイン** ゲストが早めにチェックインしたい場合の追加料金を設定。*これは情報提供のみです*
- **遅めのチェックアウト** ゲストが遅めにチェックアウトしたい場合の追加料金を設定。*これは情報提供のみです*

### 追加料金の詳細

料金プランに追加料金を設定します。

追加料金を作成するには、`Create extra charge` ボタンをクリックしてください。

- **料金タイプ** この料金の計算方法を選択します。利用可能なタイプ：
  - *滞在ごと* — 予約ごとに一度課金。
  - *日ごと* — 宿泊1泊ごとに課金。
  - *人数ごと / 滞在ごと* — ゲスト1人あたり予約ごとに課金。
  - *人数ごと / 日ごと* — ゲスト1人あたり1泊ごとに課金。
- **金額** 料金の固定金額を設定。*例：100*

この料金の名前と説明を入力してください。

:::note
説明は最低1つ英語で記述する必要があります。
:::

- **名前** 料金名。*例：一回限りの清掃料金*
- **説明** 料金について1〜2段落で説明。*例：ヴィラは到着前にプロの清掃チームによって清掃されます。*

## 制限

ゲストルームや料金プランの利用可否を制御する制限を設定します。

サポートされている制限：

- 宿泊日数
- 予約の事前期間
- 宿泊人数
- 年齢
- 販売日 / 宿泊日 / 利用可能日 / 到着日 / 出発日 / 必須日

### 宿泊日数

この制限を追加すると、ゲストに「最低○日以上」「最大○日以下」の宿泊を要求します。

*例：この部屋タイプは10日以上宿泊するゲストのみ利用可能です。注：最大宿泊日数は任意です。*

### 予約の事前期間

この制限を追加すると、ゲストに「最低○日前まで」「最大○日前まで」に予約することを要求します。

*例：このプロモーションは10日以上前に予約したゲストのみ利用可能です。注：最大事前予約期間も設定する必要があります。*

### 宿泊人数

この制限を追加すると、到着するグループの人数が「最低○人」「最大○人」であることを要求します。

### 年齢

この制限を追加すると、ゲストの年齢が「最低○歳」「最大○歳」であることを要求します。

### 販売日

この制限を追加すると、特定の日付範囲内でのみこの料金を予約可能にします。ゲストはこの日付範囲内の指定日に予約する場合にのみこの料金が表示されます。

### 宿泊日

この制限を追加すると、特定の日付範囲内でのみこの料金で宿泊可能にします。ゲストはこの日付範囲内の指定日に宿泊する場合にのみこの料金が表示されます。

### 利用可能曜日

:::note
この制限は「販売日制限」と組み合わせて使うと効果的で、日付範囲を曜日でさらに絞り込むことができます。
:::

この制限を有効にすると、指定した曜日のみこの料金の検索が可能になります。

### 到着日

:::note
この制限は「宿泊日制限」と組み合わせて使うと効果的で、日付範囲を曜日でさらに絞り込むことができます。
:::

この制限を有効にすると、ゲストはその曜日に到着する場合にのみこの料金が表示されます。

*例：月曜日を有効にすると、ゲストは月曜日に到着する必要があります。*

### 出発日

:::note
この制限は「宿泊日制限」と組み合わせて使うと効果的で、日付範囲を曜日でさらに絞り込むことができます。
:::

この制限を有効にすると、ゲストはその曜日に出発する場合にのみこの料金が表示されます。

*例：月曜日を有効にすると、ゲストは月曜日に出発する必要があります。*

### 必須日

:::note
この制限は「宿泊日制限」と組み合わせて使うと効果的で、日付範囲を曜日でさらに絞り込むことができます。
:::

この制限を有効にすると、ゲストは「少なくとも1日」これらの日に宿泊する必要があり、その場合にのみこの料金が表示されます。

*例：月曜日を有効にすると、ゲストは月曜日に宿泊する必要があります。*

## API

`Rate Plans` を管理したい開発者は、[Developers > APIs > Monetize](/ja/developers/apis/#monetize-api) にアクセスしてください。
