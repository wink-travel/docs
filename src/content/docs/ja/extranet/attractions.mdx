アトラクションは、施設内外でアトラクションを紹介・販売することができます。アトラクションを作成するには、`Create attraction` ボタンをクリックしてください。

以下のセクションでは、アトラクション管理のすべての部分を順を追って説明します。

## 設定

開始時にデフォルトで選択されているのは `Configurations` タブです。

このセクションでは、アトラクションの基本情報を設定します。

- **内部名** アトラクションの内部での呼称。*例: フライフィッシング*
- **ライフスタイル** 必要に応じて、このアトラクションに合うライフスタイルを選択。*例: スポーツ*
- **年齢** 必要に応じて、このアトラクションの推奨最低年齢層を選択。*例: 大人*
- **タイプ** 必要に応じて、最も適したアトラクションのタイプを選択。*例: 釣り*
- **予約必須** このアトラクションに予約が必要かどうか。*例: はい*

## スケジュール

このセクションでは、アトラクションの利用可能時間を管理します。

### 営業時間

- **開始時間** アトラクションが利用可能になる時間。*例: 09:00*
- **終了時間** アトラクションが利用不可になる時間。*例: 21:00*

### 営業日

このセクションでは、アトラクションが利用可能な曜日を設定します。

*スイッチを✅に切り替えて、その曜日が利用可能であることを示します。*

### 季節設定

:::note
ネイティブの「日/月」形式の入力フィールドはありません。この場合、日付セレクターから月と日だけを使用します。
:::

このセクションでは、年間を通じてアトラクションが利用可能な日付を設定します。

- **開始日** アトラクションが利用可能になる日付。*例: 2024年9月11日*
- **終了日** アトラクションが利用不可になる日付。*例: 2024年7月11日*

## アメニティ

:::note
このセクションは任意です。
:::

`Amenities` タブをクリックして続行してください。

施設のアメニティと同様に、アトラクションのアメニティは特定のアトラクションにのみ適用されます。該当するアメニティにチェックを入れてください。

## 説明

:::note
少なくとも1つの説明を英語で記入する必要があります。
:::

`Descriptions` タブをクリックして続行してください。

施設のウェルカムテキストと同様に、アトラクションのローカライズされた説明も作成できます。
希望するだけ多くの言語で記述可能です。人気のある言語の翻訳も追加します。
これは旅行者がアトラクションを見る際に表示されるテキストです。

- **名前** アトラクションの名前を付けてください。*例: フライフィッシング*
- **説明** アトラクションを1〜2段落で説明してください。*例: あなただけ…森…川…心の平穏が詰まったボトル。*

## ソーシャルメディア

:::note
このセクションは任意です。
:::

`Social media` タブをクリックして続行してください。

「なぜアトラクションに独自のソーシャルメディアセクションがあるのか？」と思うかもしれません。アトラクションは施設の近くにあるが敷地外で、独自のスタッフやInstagramプロフィールを持つ場合もあります。

ソーシャルメディアは、新規および既存の旅行者と継続的に連絡を取る手段です。
プロフィールにソーシャルネットワークアカウントを追加して、ゲストが簡単に連絡を取り、このアトラクションについて知ることができるようにしましょう。

<Steps>
1. 有効にしたいソーシャルネットワークのスイッチを切り替えます。
2. そのネットワーク上のアカウント名を入力します。
</Steps>

## 評判

:::note
このセクションは任意です。
:::

`Reputation` タブをクリックして続行してください。

`Social media` タブと同様に、このタブはアトラクションが別の評判スコアや特別な称賛を持っている場合に使用します。

評判はデジタル通貨のようなもので、どこへ行っても持ち運びたいものです。
評判エントリーは、施設が誇りに思う第三者評価や、シェフが金賞を受賞したコンテストなど、何でも構いません。
このセクションでは、インターネット上のあらゆるスコアやランキングを追加できます。

`Add reputation score` ボタンをクリックしてください。

<Steps>
1. **カテゴリ** 追加したいスコアの種類を選択。*例: 第三者レビュー*
2. **提供者** スコアを授与した団体名を入力。*例: BigOTA.com*
3. **評価タイプ** スコアの形式を選択。*例: 数値*
4. **日付** 任意でスコアを受け取った日付を入力。
5. **評価** 受け取ったスコアを入力。*例: 8.5*
6. **最大評価** 任意でこの賞の最大スコアを入力。*例: 10*
</Steps>

## 属性ベース販売

:::note
このセクションは任意です。
:::

`Attribute Based Selling` タブをクリックして続行してください。

すべての施設には独自のABSセクションがあります。このセクションでは、メインの在庫にアップセル機会を作成したり、非取引型の在庫タイプを取引可能にしたりできます。
予約プロセス中により高い価格帯で予約者とつながる可能性があるため、これらの機会を作成することを強くお勧めします。

:::tip
アトラクションはABSを通じて販売可能にできます。販売方法はさまざまです：

- 日帰り旅行。
- 3日間の旅行。
- +++ 想像力を働かせて追加収入を得ましょう！！
:::

- **価格帯** 旅行者にこのアトラクションの価格感を簡単に示します。
- **有効** このアトラクションをすべての販売チャネルで予約不可にしたい場合はスイッチを🛑に切り替えます。
- **コミッション対象** アフィリエイトがこの在庫でコミッションを得るかどうかを示します。
- **注目** このアトラクションをより目立つ位置に表示すべきかどうかを示します（オプションがある場合）。
- **属性ベース販売** この在庫でABSを有効にするにはスイッチを✅に切り替えます。*有効にしたら、少なくとも1つのABS商品を追加する必要があります*

### 付帯商品

ABSを有効にすると、付帯商品を追加して販売可能にできます。

*デフォルトで空のアイテムがすでに作成されています。`Add item for sale` ボタンをクリックしてさらに追加できます。*

- **名前** 付帯商品の名前を付けてください。*例: 日帰り旅行*
- **価格タイプ** 価格計算方法を示します。*例: `Per day` に設定して日数に基づく価格を示す。*
- **基本価格** この商品の通常価格を入力。*例: 200 / 日*
- **割引価格** Winkユーザーが予約時に得られる価格を入力。*例: 150 / 日*

### 制限

数量の最小/最大制限を設定して、この商品の利用可能性を制限できます。

- **最小数量** 予約が必要な最小数量を制限。*例: 1名*
- **最大数量** 予約可能な最大数量を制限。*例: 4名*

### 説明

:::note
少なくとも1つの説明を英語で記入する必要があります。
:::

- **名前** 付帯商品の名前を付けてください。*例: 日帰り旅行*
- **説明** 付帯商品を1〜2段落で説明してください。*例: 1日休みを取って川へ出かけましょう。*

## 近接情報

`Proximity` タブをクリックして続行してください。

すべての在庫が施設内にあるわけではありません。敷地外にある在庫がある場合は、ここで旅行者に知らせることができ、別の地図位置や任意の連絡先情報を設定できます。

*アトラクションがホテルから数キロ離れている場合に、旅行者がタクシー運転手に行き方を説明したいときに役立ちます。*

`Save` ボタンをクリックして続行してください。

`Attractions` を管理したい開発者は、[Developers > APIs > Experiences](/ja/developers/apis/#experiences-api) にアクセスしてください。
