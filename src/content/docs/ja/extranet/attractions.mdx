アトラクションでは、施設内外でアトラクションを宣伝・販売できます。アトラクションを作成するには、`Create attraction` ボタン。

次のセクションでは、アトラクションの管理のあらゆる部分を段階的に説明します。

## 構成

その`Configurations`起動すると、デフォルトでタブが選択されます。

このセクションでは、アトラクションの基本について説明します。

* **内部名**社内で呼ばれるアトラクションの名前。*例：フライフィッシング*
* **ライフスタイル**オプションで、このアトラクションに適したライフスタイルを選択してください。*例：スポーツ*
* **年**オプションで、このアトラクションの推奨最低年齢グループを選択します。*例：大人*
* **タイプ**オプションで、最も適したアトラクションの種類を選択します。*例：釣り*
* **予約が必要です**このアトラクションに予約が必要かどうか。*例：はい*

## スケジュール

このセクションでは、アトラクションがいつ利用可能かを管理できます。

### 営業時間

* **開く**アトラクションが利用可能になる時間。*例：09:00*
* **閉じる**アトラクションが利用できなくなる時間帯。*例：21:00*

### 営業日

このセクションでは、アトラクションを利用できる曜日を設定できます。

*その日が空いていることを示すには、スイッチを ✅ に切り替えます。*

### 季節限定

:::note
見栄えの良いネイティブの「日/月」フォームフィールドはありません。このシナリオでは、この日付セレクターから取得した月と日のみを使用します。
:::

このセクションでは、年間を通じてアトラクションを利用できる日付を設定できます。

* **開始日**アトラクションが利用可能になる日付。*例：2024年9月11日*
* **開始日**アトラクションが利用できなくなる日付。*例：2024年7月11日*

## アメニティ

:::注
このセクションは任意です。
:::

クリックして`Amenities`続行するにはタブをクリックします。

施設のアメニティと同様に、アトラクションのアメニティは特定のアトラクションにのみ適用されます。該当するアメニティにチェックを入れてください。

## 説明

:::注
説明文は少なくとも1つ、英語で記入してください。
:::

クリックして`Descriptions`続行するにはタブをクリックします。

宿泊施設のウェルカムテキストと同様に、アトラクションの説明もローカライズして作成できます。
ご希望の言語でご記入いただけます。主要な言語については、翻訳を追加いたします。
これは、旅行者がアトラクションを閲覧する際に表示されるテキストです。

* **名前**アトラクションに名前を付けます。*例：フライフィッシング*
* **説明**1～2 段落でその魅力を説明してください。*例えば、あなただけ...森...川...ボトルに入った心の平和。*

## ソーシャルメディア

:::注
このセクションは任意です。
:::

クリックして`Social media`続行するにはタブをクリックします。

「なぜアトラクションに独自のソーシャルメディアセクションがあるの？」と疑問に思う方もいるかもしれません。アトラクションは、施設の近くにあっても敷地外にあり、専属スタッフと独自のInstagramアカウントを持っている場合もあります。

ソーシャルメディアは、新規旅行者や既存の旅行者と継続的につながりを保つのに役立ちます。プロフィールにソーシャルネットワークのアカウントを追加すれば、ゲストが簡単に連絡を取り、このアトラクションについて知ることができます。

<Steps>
  1. 有効にするソーシャル ネットワークのスイッチを切り替えます。
  2. そのネットワーク上のアカウントの名前を入力します。
</Steps>


## 評判

:::注
このセクションは任意です。
:::

クリックして`Reputation`続行するにはタブをクリックします。

同様`Social media`タブでは、このアトラクションに別の評判スコアや、ここで強調表示したい追加の栄誉がある場合に、このタブを使用します。

あなたの評判はデジタル通貨のようなもの。どこへ行っても持ち歩きたいですよね。
評判エントリーには、第三者機関による評価や、シェフが金メダルを獲得したベーキングコンテストなど、あなたの施設が誇るべきあらゆる情報が含まれます。
このセクションでは、インターネット上のスコアやランキングを追加できます。

クリック`Add reputation score` ボタン。

<Steps>
  1. **カテゴリ** 追加するスコアの種類を選択します。 *例：第三者によるレビュー*
  2. **プロバイダー** スコアを付与した人を入力してください。 *例：BigOTA.com*
  3. **評価タイプ** スコアを受け取ったフォームを選択します。 *例：数値*
  4. **日付** 必要に応じて、スコアを受け取った日付を入力します。
  5. **評価** 受け取ったスコアを入力してください。 *例：8.5*
  6. **最大評価** オプションで、このタイプの賞の最大スコアを入力します。 *例：10*
</Steps>


## 属性ベースの販売

:::注
このセクションは任意です。
:::

クリックして`Attribute Based Selling`続行するにはタブをクリックします。

すべての施設に専用のABSセクションが設けられています。このセクションでは、メイン在庫のアップセル機会を作成したり、非トランザクション在庫をトランザクション対応にしたりできます。予約プロセス中に、より高い価格で予約者とつながる可能性を高めるために、これらの機会を作成することを強くお勧めします。

:::ヒント
アトラクションはABSを通じて販売できます。販売方法はいくつかあります。

* 日帰り旅行。

* 3日間の旅行。

* +++ 想像力を自由に働かせて、お小遣いを稼ぎましょう!!
  :::

* **価格帯**このアトラクションがどれくらい高価であるかを旅行者に簡単に知らせます。

* **アクティブ**このアトラクションをすべての販売チャネルで予約できないようにするには、スイッチを 🛑 に切り替えます。

* **手数料対象**アフィリエイトがこの在庫に対して手数料を受け取るかどうかを示します。

* **特集**オプションがある場合、このアトラクションをより目立つ場所に配置するかどうかを示します。

* **属性ベースの販売**このインベントリで ABS を有効にするには、スイッチを ✅ に切り替えます。*有効にしたら、少なくとも1つのABSアイテムを販売用に追加する必要があります。*

### 付属品

ABS を有効にすると、補助アイテムを追加して販売できるようになります。

*デフォルトでは、空のアイテムがすでに作成されています。クリックすると、さらにアイテムを追加できます。`Add item for sale` ボタン。*

* **名前**補助機能に名前を付けます。*例：日帰り旅行*
* **価格タイプ**価格を計算する方法を指定します。*例: 設定`Per day`価格は日数に基づいていることを示します。*
* **基本価格**この商品の通常価格を入力してください。*例：200 /日*
* **割引価格**Wink ユーザーがアイテムを予約するときに受け取る価格を入力します。*例：1日150円*

### 制限

数量の最小/最大制限を設定することで、このアイテムの入手可能性を制限できます。

* **最小数量**予約する必要がある最小数量を制限します。*例：参加者1名*
* **最大数量**予約する必要がある最大数量を制限します。*例：参加者4名*

### 説明

:::注
説明文は少なくとも1つ、英語で記入してください。
:::

* **名前**補助機能に名前を付けます。*例：日帰り旅行*
* **説明**補助的なものを 1 ～ 2 段落で説明してください。*例えば、一日休みを取って川へ出かけましょう。*

## 近接性

クリックして`Proximity`続行するにはタブをクリックします。

すべての在庫が敷地内にあるわけではありません。敷地外に在庫がある場合は、旅行者にここでお知らせください。地図上の場所を別途設定し、必要に応じて連絡先情報もご入力いただけます。

*これは、観光スポットがホテルから数キロ離れた場所にあり、旅行者がタクシーの運転手にそこへの行き方を説明したい場合に役立ちます。*

クリック`Save`続行するにはボタンをクリックしてください。
