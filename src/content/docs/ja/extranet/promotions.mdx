:::note
航空業界と同様に、ホテルでは価格設定や割引に関して多くの柔軟性が求められます。
プロモーションは、旅行者が追加の割引[またはプレミアム]を受けられる独自の条件をホテルが作成する方法です。
:::

プロモーションを管理するには、メインナビバーの `Monetize > Single Promotions` をクリックしてください。

新しいプロモーションを作成するには、`Create single promotion` ボタンをクリックします。

## 設定

このセクションでは、プロモーションの実際の金額を設定します。

- **名前** このプロモーションの名前を入力してください（あなたのみが見えます）。
- **タイプ** 割引を含むかプレミアムを含むかを選択してください。
- **価格タイプ** このバンドルの計算方法を選択してください。
    - `Per stay`: 金額は最終泊に適用されます。*例：100%オフ > 2泊すると3泊目が無料。*
    - `Per night`: 金額は宿泊数に掛けられます。
    - `Per use`: 追加の計算は行われません。
    - `Per person`: 金額は宿泊者数に掛けられます。
    - `Per person per night`: 金額は宿泊数と宿泊者数の両方に掛けられます。
- **金額タイプ** 金額の修正を固定額にするかパーセンテージにするか選択してください。
- **ステータス** スイッチを切り替えて、プロモーションがすべての販売チャネルで利用可能かどうかを制御します。

## 説明

:::note
少なくとも1つの説明を英語で記入する必要があります。
:::

`Descriptions` タブをクリックして続行してください。

ローカライズされた説明は、旅行者に対してこのプロモーションの内容をその言語で説明します。
必要なだけ多くの言語で記入できます。最も人気のある言語の翻訳を追加します。
これは旅行者がプロモーションを見る際に表示されるテキストです。

- **説明** プロモーションの名前を付けてください。*例：Early bird - 20% OFF*

## 制限

:::note
これは制限です… プロモーションのアクセスを制限したい場合のみ有効にしてください。
:::

`Restrictions` タブをクリックして続行してください。

Winkのプロモーションエンジンは、価格変更をトリガーするための20以上のユニークな方法を備えています。

以下の条件をサポートしています：

- 特定のアドオンにプロモーションを制限。
- 特定のマスターレートにプロモーションを制限。
- 特定のレートプランにプロモーションを制限。
- 特定のキャンセルポリシーにプロモーションを制限。
- 宿泊期間にプロモーションを制限。
- 事前予約にプロモーションを制限。
- 部屋数にプロモーションを制限。
- プロモコードにプロモーションを制限。
- 旅行者のIP範囲にプロモーションを制限。
- ラストミニットにプロモーションを制限。
- 旅行者の都市にプロモーションを制限。
- 旅行者の国にプロモーションを制限。
- 旅行者の大陸にプロモーションを制限。
- 旅行者のタイムゾーンにプロモーションを制限。
- 販売日(s)にプロモーションを制限。
- 宿泊日(s)にプロモーションを制限。
- 予約曜日にプロモーションを制限。
- 到着曜日にプロモーションを制限。
- 出発曜日にプロモーションを制限。
- 必須曜日にプロモーションを制限。

各制限の使い方は以下をお読みください。

### アドオン

:::note
この機能を使うには、少なくとも1つのアドオンを作成している必要があります。
:::

選択したアドオンのみにプロモーションを適用するには、以下の手順に従ってください：

<Steps>
1. `Add-on restriction` スイッチを有効にします。
2. 含めたい各アドオンのチェックボックスをオンにします。
</Steps>

### マスターレート

:::note
この機能を使うには、少なくとも1つのマスターレートを作成している必要があります。
:::

選択したマスターレートのみにプロモーションを適用するには、以下の手順に従ってください：

<Steps>
1. `Master rate restriction` スイッチを有効にします。
2. 含めたい各マスターレートのチェックボックスをオンにします。
</Steps>

### レートプラン

:::note
この機能を使うには、少なくとも1つのレートプランを作成している必要があります。
:::

選択したレートプランのみにプロモーションを適用するには、以下の手順に従ってください：

<Steps>
1. `Rate plan restriction` スイッチを有効にします。
2. 含めたい各レートプランのチェックボックスをオンにします。
</Steps>

### 返金可能

:::note
この機能を使うには、少なくとも1つのキャンセルポリシーを作成している必要があります。
:::

選択したキャンセルポリシーのみにプロモーションを適用するには、以下の手順に従ってください：

<Steps>
1. `Cancellation policy restriction` スイッチを有効にします。
2. 含めたい各キャンセルポリシーのチェックボックスをオンにします。
</Steps>

### 宿泊期間

:::note
この制限は通常「長期滞在」プロモーションに使います。
:::

この制限を追加すると、ゲストに「最低宿泊日数」と「最大宿泊日数」の条件を課します。

`Length of Stay` を有効にする手順：

<Steps>
1. `Length of stay restriction` スイッチを有効にします。
2. ゲストが最低限宿泊しなければならない日数を入力します。*例：7*
3. 任意でゲストが最大で宿泊できる日数を入力します。*例：90*
</Steps>

*例：このプロモーションは10日以上滞在するゲストのみ利用可能です。最大宿泊日数は任意です。*

### 事前予約

:::note
この制限は通常「早期予約」プロモーションに使います。
:::

この制限を追加すると、ゲストに「最低予約日数前」と「最大予約日数前」の条件を課します。

`Advance booking` を有効にする手順：

<Steps>
1. `Advance booking restriction` スイッチを有効にします。
2. ゲストが最低限何日前に予約する必要があるか入力します。*例：10*
3. 任意でゲストが最大で何日前に予約できるか入力します。*例：180*
</Steps>

*例：このプロモーションは10日以上前に予約したゲストのみ利用可能です。最大予約日数も必ず設定してください。*

### 部屋数

:::note
このプロモーションは、ゲストが最低部屋数を予約した場合のみ利用可能です。
:::

この制限を追加すると、ゲストに「最低部屋数」と「最大部屋数」の条件を課します。

`Number of rooms` を有効にする手順：

<Steps>
1. `Room restriction` スイッチを有効にします。
2. ゲストが最低限予約しなければならない部屋数を入力します。*例：3*
3. ゲストが最大で予約できる部屋数を入力します。*例：8*
</Steps>

*例：このプロモーションは3部屋以上予約したゲストのみ利用可能です。*

### プロモコード

:::note
ゲストに割引コードの入力を求めてこのプロモーションを解除したい場合に使います。
:::

`Promo code` を有効にする手順：

<Steps>
1. `Promotion restriction` スイッチを有効にします。
2. `Add promo code` ボタンをクリックします。
3. プロモコードを入力します。*例：ABC123*
</Steps>

*例：このプロモーションはプロモコード `ABC123` を入力したゲストのみ利用可能です。*

### IP範囲

:::note
モバイルキャリアや独自のIP範囲を持つ大企業とのクロスプロモーションキャンペーンに適した制限です。
:::

`IP range` を有効にする手順：

<Steps>
1. `IP range restriction` スイッチを有効にします。
2. `Add IP range restriction` ボタンをクリックします。
3. 開始IPアドレス（グローバルアクセス可能）を入力します。*例：203.0.113.50*
4. 終了IPアドレス（グローバルアクセス可能）を入力します。*例：203.0.113.59*
</Steps>

*例：このプロモーションはIPアドレスが `203.0.113.50` から `203.0.113.59` の範囲内のゲストのみ利用可能です。*

### ラストミニット

:::note
制限したいラストミニットの時間枠です。
:::

`Last minute` を有効にする手順：

<Steps>
1. `Last minute restriction` スイッチを有効にします。
2. `Add IP range restriction` ボタンをクリックします。
3. 到着日の何時間前かを入力します。*例：24時間*
</Steps>

*例：このプロモーションは到着日の24時間前に予約したゲストのみ利用可能です。*

### 都市

:::note
旅行者が現在いる都市で制限をかけられます。含めたい都市ごとに制限を作成してください。
:::

`City` を有効にする手順：

<Steps>
1. `City restriction` スイッチを有効にします。
2. 都市名を入力し始めます。
3. 検索結果から都市を選択します。*例：New York City*
</Steps>

*例：このプロモーションは `New York City` から予約したゲストのみ利用可能です。*

### 国

:::note
旅行者が現在いる国で制限をかけられます。含めたい国ごとに制限を作成してください。
:::

`Country` を有効にする手順：

<Steps>
1. `Country restriction` スイッチを有効にします。
2. 国名を入力し始めます。
3. 検索結果から国を選択します。*例：United States of America*
</Steps>

*例：このプロモーションは `United States of America` から予約したゲストのみ利用可能です。*

### 大陸

:::note
ゲストが現在いる大陸でレートを制限できます。含めたい大陸ごとに制限を作成してください。
:::

`Continent` を有効にする手順：

<Steps>
1. `Continent restriction` スイッチを有効にします。
2. 大陸名を入力し始めます。
3. 検索結果から大陸を選択します。*例：North America*
</Steps>

*例：このプロモーションは `North America` から予約したゲストのみ利用可能です。*

### タイムゾーン

:::note
ゲストが現在いるタイムゾーンでレートを制限できます。含めたいタイムゾーンごとに制限を作成してください。
:::

`Timezone` を有効にする手順：

<Steps>
1. `Timezone restriction` スイッチを有効にします。
2. タイムゾーン名を入力し始めます。
3. 検索結果からタイムゾーンを選択します。*例：America/New_York*
</Steps>

*例：このプロモーションは `America/New_York` から予約したゲストのみ利用可能です。*

### 販売日

:::note
特定の日付範囲で制限できます。この日付範囲内で予約した旅行者が対象となります。
:::

`Sell date` を有効にする手順：

<Steps>
1. `Sell date restriction` スイッチを有効にします。
2. `Add sell date restriction` ボタンをクリックします。
3. 開始日を入力します。*例：2024年9月19日*
4. 終了日を入力します。*例：2024年12月1日*
</Steps>

*例：このプロモーションは `2024年9月19日` から `2024年12月1日` の間に予約したゲストのみ利用可能です。*

### 宿泊日

:::note
特定の日付範囲で制限できます。この日付範囲内の宿泊に対して予約した旅行者が対象となります。
:::

`Stay date` を有効にする手順：

<Steps>
1. `Stay date restriction` スイッチを有効にします。
2. `Add stay date restriction` ボタンをクリックします。
3. 開始日を入力します。*例：2024年9月19日*
4. 終了日を入力します。*例：2024年12月1日*
</Steps>

*例：このプロモーションは `2024年9月19日` から `2024年12月1日` の間の宿泊に対して予約したゲストのみ利用可能です。*

### 予約曜日

:::note
この制限を有効にすると、予約が行われた曜日にのみこのレートが利用可能になります。

この制限は「販売日制限」と組み合わせて、日付範囲を曜日でさらに絞り込むのに適しています。
:::

`Booking days` を有効にする手順：

<Steps>
1. `Booking days restriction` スイッチを有効にします。
2. 曜日のスイッチを1つ有効にします。*例：月曜日*
</Steps>

*例：このプロモーションは `月曜日` に予約したゲストのみ利用可能です。*

### 到着曜日

:::note
この制限を有効にすると、ゲストはその曜日に到着しなければこのレートを見られません。

この制限は「宿泊日制限」と組み合わせて、日付範囲を曜日でさらに絞り込むのに適しています。
:::

`Arrival days` を有効にする手順：

<Steps>
1. `Arrival days restriction` スイッチを有効にします。
2. 曜日のスイッチを1つ有効にします。*例：月曜日*
</Steps>

*例：このプロモーションは `月曜日` に到着する宿泊に対して予約したゲストのみ利用可能です。*

### 出発曜日

:::note
この制限を有効にすると、ゲストはその曜日に出発しなければこのレートを見られません。

この制限は「宿泊日制限」と組み合わせて、日付範囲を曜日でさらに絞り込むのに適しています。
:::

`Departure days` を有効にする手順：

<Steps>
1. `Departure days restriction` スイッチを有効にします。
2. 曜日のスイッチを1つ有効にします。*例：月曜日*
</Steps>

*例：このプロモーションは `月曜日` に出発する宿泊に対して予約したゲストのみ利用可能です。*

### 必須曜日

:::note
この制限を有効にすると、ゲストは「少なくとも1日」この曜日を含む宿泊でなければこのレートを見られません。

この制限は「宿泊日制限」と組み合わせて、日付範囲を曜日でさらに絞り込むのに適しています。
:::

`Required days` を有効にする手順：

<Steps>
1. `Required days restriction` スイッチを有効にします。
2. 曜日のスイッチを1つ有効にします。*例：月曜日*
</Steps>

*例：このプロモーションは `月曜日` を含む宿泊に対して予約したゲストのみ利用可能です。*

## ブラックアウト日

`Blackout dates` タブをクリックして続行してください。

プロモーションが利用できない日付を指定して、さらに制御できます。

ブラックアウト日を追加するには、`Add blackout date` ボタンをクリックします。

- **開始日** ブラックアウト日の開始日を選択します。
- **終了日** ブラックアウト日の終了日を選択します。

予約日がこの日付範囲にかかる場合、プロモーションは適用されません。

旅行者は、長期滞在＋早期予約など複数のプロモーションの対象になることがあります。
バンドルプロモーションがない場合、旅行者は最もお得な1つのプロモーションのみ対象となります。

バンドルプロモーションを作成することで、両方のプロモーションの適用を認め、さらにカスタマイズできます。

バンドルプロモーションを作成するには、`Create bundled promotion` ボタンをクリックします。

### 設定

- **名前** バンドルプロモーションの名前を入力してください（あなたのみが見えます）。
- **ステータス** スイッチを切り替えて、バンドルがすべての販売チャネルで利用可能かどうかを制御します。
- **プロモーション** バンドルを構成する単一プロモーションを最低2つ選択します。

### 金額の上書き

合計割引額を上書きすることもできます。オフにすると、選択した各プロモーションの割引が合算されて1つの割引になります。

- **上書き金額** 割引額をすべて合算するか、自分の金額で上書きするか選択します。
- **タイプ** 価格を増加させるか減少させるか選択します。*例：価格を下げる場合は `Discount` に設定。*
- **価格タイプ** このバンドルの計算方法を選択します。
    - `Per stay`: 金額は最終泊に適用されます。*例：100%オフ > 2泊すると3泊目が無料。*
    - `Per night`: 金額は宿泊数に掛けられます。
    - `Per use`: 追加の計算は行われません。
    - `Per person`: 金額は宿泊者数に掛けられます。
    - `Per person per night`: 金額は宿泊数と宿泊者数の両方に掛けられます。
- **金額タイプ** 金額の修正を固定額にするかパーセンテージにするか選択してください。
