レストランでは、施設内外でレストランを紹介・販売できます。レストランを作成するには、`Create restaurant` ボタンをクリックしてください。

以下のセクションでは、レストラン管理の各ステップを順に説明します。

## 設定

開始時にデフォルトで選択されているのは `Configurations` タブです。

このセクションではレストランの基本情報を設定します。

- **内部名** レストランの内部での呼称。*例: Tsunami Sushi*
- **予約必須** このレストランが予約を必要とするかどうか。*例: はい*

### 収容人数

このセクションではレストランの収容人数を入力します。

- **最大座席数** 利用可能な総座席数。*例: 100*
- **最大グループ人数** 最大テーブル席数。*例: 20*

### 食事

このセクションではレストランが提供する食事の種類を設定します。

*スイッチを✅に切り替えて、対応している食事タイプをユーザーに知らせます。*

## スケジュール

このセクションではレストランの利用可能時間を管理します。

### 営業時間

- **開店時間** レストランが利用可能になる時間。*例: 09:00*
- **閉店時間** レストランが利用不可になる時間。*例: 21:00*

### 営業日

このセクションではレストランが営業する曜日を設定します。

*スイッチを✅に切り替えて、その曜日が利用可能であることを示します。*

### 季節営業

:::note
ネイティブの「日/月」形式の入力フィールドはありません。この場合、日付セレクターから月と日だけを使用します。
:::

このセクションでは、年間を通じてレストランが利用可能な日付を設定します。

- **開始日** レストランが利用可能になる日付。*例: 2024年9月11日*
- **終了日** レストランが利用不可になる日付。*例: 2024年7月11日*

## アメニティ

:::note
このセクションは任意です。
:::

`Amenities` タブをクリックして続行してください。

施設のアメニティと同様に、レストランのアメニティは特定のレストランにのみ適用されます。該当するアメニティにチェックを入れてください。

## 説明

:::note
少なくとも1つの説明文を英語で記入する必要があります。
:::

`Descriptions` タブをクリックして続行してください。

施設のウェルカムテキストと同様に、レストランのローカライズされた説明文を作成できます。
希望するだけ多くの言語で記述可能です。人気のある言語の翻訳を追加します。
これは旅行者がレストランを見る際に表示されるテキストです。

- **名前** レストランの名前を付けてください。*例: Tsunami Sushi*
- **説明** レストランについて1〜2段落で説明してください。*例: ここは世界で最高のボールルームです...*

## ソーシャルメディア

:::note
このセクションは任意です。
:::

`Social media` タブをクリックして続行してください。

「なぜレストランに独自のソーシャルメディアセクションがあるのか？」と思うかもしれません。レストランは施設に近いが敷地外にあり、独自のスタッフとInstagramプロフィールを持つ場合があります。

ソーシャルメディアは、新規および既存の旅行者と継続的に連絡を取る手段です。
プロフィールにソーシャルネットワークアカウントを追加して、ゲストが簡単に連絡を取り、このレストランについて知ることができるようにしましょう。

<Steps>
1. 有効にしたいソーシャルネットワークのスイッチを切り替えます。
2. そのネットワーク上のアカウント名を入力します。
</Steps>

## 評判

:::note
このセクションは任意です。
:::

`Reputation` タブをクリックして続行してください。

`Social media` タブと同様に、このタブはレストランが別の評判スコアや特別な称賛を持っている場合に使用します。

評判はあなたのデジタル通貨であり、どこへ行っても持ち運びたいものです。
評判エントリーは、3者評価やシェフが金賞を受賞したコンテストなど、施設が誇りに思うものなら何でも構いません。
このセクションでは、インターネット上のあらゆるスコアやランキングを追加できます。

`Add reputation score` ボタンをクリックしてください。

<Steps>
1. **カテゴリ** 追加したいスコアの種類を選択します。*例: 第三者レビュー*
2. **提供者** スコアを授与した団体名を入力します。*例: BigOTA.com*
3. **評価タイプ** スコアを受け取った形式を選択します。*例: 数値*
4. **日付** 任意でスコアを受け取った日付を入力します。
5. **評価** 受け取ったスコアを入力します。*例: 8.5*
6. **最大評価** 任意でこの賞の最大スコアを入力します。*例: 10*
</Steps>

## 属性ベース販売

:::note
このセクションは任意です。
:::

`Attribute Based Selling` タブをクリックして続行してください。

すべての施設には独自のABSセクションがあります。このセクションでは、メインの在庫にアップセル機会を作成したり、非取引型の在庫タイプを取引可能にしたりできます。
予約プロセス中により高い価格帯で予約者とつながる可能性があるため、これらの機会を作成することを強くお勧めします。

:::tip
レストランはABSを通じて販売可能にできます。販売方法はさまざまです：

- プリフィックス料金。
- グループビュッフェ。
- 食べ放題。
- アヒル一羽、丸ごとアヒル、それだけ。
- +++ 想像力を働かせて追加収入を得ましょう！！
:::

- **価格帯** 旅行者にこのレストランの価格感を簡単に示します。
- **有効** このレストランをすべての販売チャネルで予約不可にしたい場合はスイッチを🛑に切り替えます。
- **コミッション対象** アフィリエイトがこの在庫でコミッションを得るかどうかを示します。
- **注目** このレストランをより目立つ位置に表示すべきかどうかを示します（オプションがある場合）。
- **属性ベース販売** スイッチを✅に切り替えてこの在庫でABSを有効にします。*有効にしたら、少なくとも1つのABS販売アイテムを追加する必要があります*

### 付帯アイテム

ABSを有効にすると、付帯アイテムを追加して販売可能にできます。

*デフォルトで空のアイテムがすでに作成されています。`Add item for sale` ボタンをクリックしてさらに追加できます。*

- **名前** 付帯アイテムの名前を付けてください。*例: 食べ放題*
- **価格タイプ** 価格計算方法を示します。*例: `Per hour` に設定して時間単位の料金を示す。*
- **基本価格** このアイテムの通常価格を入力します。*例: 20 / 時間*
- **割引価格** Winkユーザーが予約時に得られる価格を入力します。*例: 15 / 時間*

### 制限

数量の最小/最大制限を設定して、このアイテムの利用可能性を制限できます。

- **最小数量** 予約が必要な最小数量を制限します。*例: 1名*
- **最大数量** 予約可能な最大数量を制限します。*例: 4名*

### 説明

:::note
少なくとも1つの説明文を英語で記入する必要があります。
:::

- **名前** 付帯アイテムの名前を付けてください。*例: 食べ放題*
- **説明** 付帯アイテムについて1〜2段落で説明してください。*例: 1時間の食べ放題。思いっきり楽しんでください。*

## 近接情報

`Proximity` タブをクリックして続行してください。

すべての在庫が施設内にあるわけではありません。施設外にある在庫がある場合は、ここで旅行者に知らせ、別の地図位置や任意の連絡先情報を設定できます。

*ホテルから数キロ離れた場所にあるレストランで、旅行者がタクシー運転手に行き方を説明したい場合に役立ちます。*

`Save` ボタンをクリックして続行してください。
