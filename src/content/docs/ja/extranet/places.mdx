場所は、施設内外であなたの場所を紹介し販売することができます。場所を作成するには、`Create place` ボタンをクリックしてください。

以下のセクションでは、場所の管理のすべての部分を順を追って説明します。

## 設定

開始時にデフォルトで選択されているのは `Configurations` タブです。

このセクションでは、場所の基本情報を扱います。

- **内部名** 場所の名前で、内部で参照するものです。*例: Haunted house*
- **ライフスタイル** 必要に応じて、この場所に合うライフスタイルを選択します。*例: Family*
- **タイプ** 必要に応じて、最も合う場所のタイプを選択します。*例: Museum*

## スケジュール

このセクションでは、場所が利用可能な時間を管理できます。

### 営業時間

- **開店時間** 場所が利用可能になる時間。*例: 09:00*
- **閉店時間** 場所が利用不可になる時間。*例: 21:00*

### 営業日

このセクションでは、場所が利用可能な曜日を設定できます。

*スイッチを✅に切り替えて、その曜日が利用可能であることを示します。*

### 季節設定

:::note
ネイティブの「日/月」形式のきれいな入力フィールドはありません。この場合、日付セレクターから月と日だけを使用します。
:::

このセクションでは、場所が年間を通じて利用可能な日付を設定できます。

- **開始日** 場所が利用可能になる日付。*例: 2024年9月11日*
- **終了日** 場所が利用不可になる日付。*例: 2024年7月11日*

## アメニティ

:::note
このセクションは任意です。
:::

続けて `Amenities` タブをクリックしてください。

物件のアメニティと同様に、場所のアメニティは特定の場所にのみ適用されます。該当するアメニティにチェックを入れてください。

## 説明

:::note
少なくとも1つの説明文を英語で記入する必要があります。
:::

続けて `Descriptions` タブをクリックしてください。

物件のウェルカムテキストと同様に、場所のローカライズされた説明文も作成できます。
希望するだけ多くの言語で記述可能です。最も人気のある言語の翻訳を追加します。
これは旅行者が場所を閲覧するときに見るテキストです。

- **名前** 場所の名前を付けてください。*例: Haunted house*
- **説明** 場所を1〜2段落で説明してください。*例: ゴーストが本当にいるか知りたいですか？*

## ソーシャルメディア

:::note
このセクションは任意です。
:::

続けて `Social media` タブをクリックしてください。

「なぜ場所に独自のソーシャルメディアセクションがあるのか？」と思うかもしれません。場所は施設内外の物件近くにあり、独自のスタッフやIGプロフィールを持つこともあります。

ソーシャルメディアは、新規および既存の旅行者と継続的に連絡を取る手段です。
プロフィールにソーシャルネットワークアカウントを追加して、ゲストが簡単に連絡を取り、この場所について知ることができるようにしましょう。

<Steps>
1. 有効にしたいソーシャルネットワークのスイッチを切り替えます。
2. そのネットワーク上のアカウント名を入力します。
</Steps>

## 評判

:::note
このセクションは任意です。
:::

続けて `Reputation` タブをクリックしてください。

`Social media` タブと同様に、この場所に別の評判スコアや特別な称賛がある場合に使用します。

評判はあなたのデジタル通貨であり、どこへ行っても持ち運びたいものです。
評判エントリーは、3者評価やシェフが金賞を受賞したベイクオフなど、物件が誇りたいものなら何でも構いません。
このセクションでは、インターネット上のあらゆるスコアやランキングを追加できます。

`Add reputation score` ボタンをクリックしてください。

<Steps>
1. **カテゴリ** 追加したいスコアの種類を選択します。*例: Third party review*
2. **提供者** スコアを授与した団体を入力します。*例: BigOTA.com*
3. **評価タイプ** スコアを受け取った形式を選択します。*例: Numeric*
4. **日付** 任意で、スコアを受け取った日付を入力します。
5. **評価** 受け取ったスコアを入力します。*例: 8.5*
6. **最大評価** 任意で、この種の賞の最大スコアを入力します。*例: 10*
</Steps>

## 属性ベース販売

:::note
このセクションは任意です。
:::

続けて `Attribute Based Selling` タブをクリックしてください。

すべての施設には独自のABSセクションがあります。このセクションでは、メインの在庫にアップセル機会を作成したり、非取引型の在庫タイプを取引可能にしたりできます。
予約プロセス中により高い価格帯で予約者とつながる可能性があるため、これらの機会を作成することを強くお勧めします。

:::tip
場所はABSを通じて販売可能にできます。販売方法はさまざまです：

- シングルチケット
- グループ旅行
- Meet-a-Ghost
- +++ 想像力を働かせて追加収入を得ましょう!!
:::

- **価格帯** 旅行者にこの場所の価格感を簡単に示します。
- **有効** すべての販売チャネルでこの場所の予約を不可にしたい場合はスイッチを🛑に切り替えます。
- **コミッション対象** アフィリエイトがこの在庫でコミッションを得るかどうかを示します。
- **注目** この場所をより目立つ位置に配置すべきかどうかを示します（オプションがある場合）。
- **属性ベース販売** スイッチを✅に切り替えてこの在庫でABSを有効にします。*有効にしたら、少なくとも1つのABS販売アイテムを追加する必要があります*

### 付帯アイテム

ABSを有効にすると、付帯アイテムを追加して販売可能にできます。

*デフォルトで空のアイテムがすでに作成されています。`Add item for sale` ボタンをクリックしてさらに追加できます。*

- **名前** 付帯アイテムの名前を付けてください。*例: Haunted house ticket*
- **価格タイプ** 価格の計算方法を示します。*例: `Per day` に設定すると日数に基づく価格になります。*
- **基本価格** このアイテムの通常価格を入力します。*例: 200 / 日*
- **割引価格** Winkユーザーが予約時に得られる価格を入力します。*例: 150 / 日*

### 制限

数量の最小/最大制限を設定して、このアイテムの利用可能性を制限できます。

- **最小数量** 予約が必要な最小数量を制限します。*例: 1名*
- **最大数量** 予約が可能な最大数量を制限します。*例: 4名*

### 説明

:::note
少なくとも1つの説明文を英語で記入する必要があります。
:::

- **名前** 付帯アイテムの名前を付けてください。*例: Group trip*
- **説明** 付帯アイテムを1〜2段落で説明してください。*例: 友達全員を怖がらせましょう。*

## 近接情報

続けて `Proximity` タブをクリックしてください。

すべての在庫が施設内にあるわけではありません。施設外にある在庫がある場合は、ここで旅行者に知らせることができ、別の地図位置や任意の連絡先情報を設定できます。

*場所がホテルから数キロ離れている場合など、旅行者がタクシー運転手に行き方を説明したいときに役立ちます。*

`Save` ボタンをクリックして続行してください。

`Places` を管理したい開発者は、[Developers > APIs > Experiences](/ja/developers/apis/#experiences-api) にアクセスしてください。
