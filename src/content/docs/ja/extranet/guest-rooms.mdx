すべての予約はゲストルームから始まります。ゲストルームには多くの機能があり、最初は少し圧倒されるかもしれません。
この記事では、Winkのゲストルームスペシャリストになるために必要なすべてのことをカバーします。

ゲストルーム、または業界用語で言うところのルームタイプは、施設内外でお部屋を紹介・販売するためのものです。ゲストルームを作成するには、`Create a new guest room` ボタンをクリックしてください。

以下のセクションでは、ゲストルーム管理の各部分を順を追って説明します。

## 設定

開始時にはデフォルトで `Configurations` タブが選択されています。

このセクションでは、どのようなゲストにこの部屋を販売できるかを把握するための基本情報を設定します。

- **ルームタイプ名** 内部で参照する部屋の名前。*例: Supreme deluxe*
- **禁煙** 部屋が禁煙かどうか。
- **標準料金** 割引や追加料金を含まない1泊の公示料金。*例: 100。これは販売チャネルに予想価格を表示したり、フォーム入力時に一部のフィールドを事前入力するためにのみ使用します。*
- **最良利用可能料金** 割引や追加料金を含まない1泊の最良利用可能料金。*例: 75。これも販売チャネルに予想価格を表示したり、フォーム入力時に一部のフィールドを事前入力するためにのみ使用します。*
- **数量** そのルームタイプのユニット数。*例: 40*

### 宿泊人数

:::caution
宿泊人数は、特定の旅程でゲストルームが利用可能かどうかに大きく影響します。これらのフィールドの多くは、最初に部屋を作成した際に自動入力されます。必ずルームタイプに合うように修正してください。
:::

- **最小宿泊人数** この部屋に宿泊可能な最小人数。*例: 1*
- **含まれる大人の人数** 1泊の料金に含まれる大人の人数。*例: 2。注：部屋がこれ以上の人数をサポートする場合は、料金プランの `Extra adult` 修正で計算されます。*
- **最大宿泊人数** この部屋に宿泊可能な最大人数（大人＋子供）。*例: 4*
- **最大大人宿泊人数** この部屋に宿泊可能な最大大人人数。*例: 3*
- **含まれる子供の人数** 1泊の料金に含まれる子供の人数。*例: 1。注：部屋がこれ以上の人数をサポートする場合は、料金プランの `Extra child` 修正で計算されます。*
- **最大子供宿泊人数** この部屋に宿泊可能な最大子供人数。*例: 2*

### その他

このセクションは情報提供のみを目的としています。大きな部屋やスイートを探している旅行者にとって興味深い情報です。

- **バスルーム数** このルームタイプのバスルーム数。*例: 1*
- **リビングルーム数** このルームタイプのリビングルーム数。*例: 1*
- **最大エキストラベッド数** このルームタイプでサポートされるエキストラベッドの数。*例: 1*
- **最大ベビーベッド数** このルームタイプでサポートされるベビーベッドの数。*例: 1*

### ルーム分類

- **セグメント** ゲストルームが分類されるセグメント。*例: Budget*
- **ロケーション** ゲストルームが分類されるロケーションタイプ。*例: 高層階*
- **眺望** このゲストルームから期待できる眺望の種類。*例: シティビュー*
- **分類** このゲストルームの分類。*例: エコノミー*
- **建築様式** このゲストルーム周辺の建築様式。*例: アジアン*
- **階数** このゲストルームがある（またはある可能性のある）階数（任意）。*例: 20階以上*

### 構成

構成は、ゲストルームが複数のゲストルームで構成されている場合にのみ適用されます。

- **構成** ゲストルームが複数の部屋で構成されている場合はスイッチを✅にする。*例: はい*
- **数** このゲストルームがいくつのゲストルームで構成されているかを示す。*例: 2*

### ホステル詳細

このセクションは、ドミトリー、ホステル、またはその他の共有ルームサービスを運営している場合にのみ適用されます。

- **共有ルーム** ゲストルームが共有ルームの場合はスイッチを✅にする。*例: はい*
- **性別** このゲストルームの性別を示す。*例: 男性*

## ベッドルーム

:::note
ベッドルームのレイアウトは無限にあります。施設に合わせてできるだけカスタマイズ可能にしています。
:::

`Bedrooms` タブをクリックして続行してください。

ベッドルームセクションでは、ベッドルームのレイアウトを指定し、それぞれに何があるかを示すことができます。

同じタイプのゲストルームでも複数のレイアウト、つまり `Bedding configurations` を持つことができます。ゲストルームには少なくとも1つのレイアウトが必要です。

- **構成名** 自分が理解しやすい名前を付ける。*例: Main*

1つのレイアウト内に複数のベッドルームを持つことができます。

- **ベッドルームタイプ** 各ベッドルームにはタイプがあります。*最初の1つはデフォルトで `Master` ですが、構成内で変更可能です。*

ベッドルーム内には、異なるタイプと数量のベッドを複数置くことができます。

- **ベッドタイプ** ベッドルーム内のベッドタイプを選択。*例: King*
- **数量** そのベッドタイプのベッド数を選択。*例: 1*

## アメニティ

:::note
このセクションは任意です。
:::

`Amenities` タブをクリックして続行してください。

施設のアメニティと同様に、ゲストルームのアメニティは特定のゲストルームにのみ適用されます。該当するアメニティにチェックを入れてください。

## 説明

:::note
少なくとも1つの説明を書き、英語で記述する必要があります。
:::

`Descriptions` タブをクリックして続行してください。

施設のウェルカムテキストと同様に、ゲストルームのローカライズされた説明も作成できます。
好きなだけ多くの言語で記述可能です。人気のある言語の翻訳は追加されます。
これは旅行者がゲストルームを閲覧する際に見るテキストです。

- **名前** ゲストルームに名前を付ける。*例: Supreme deluxe*
- **説明** ゲストルームを1〜2段落で説明する。*例: これは世界で最高のゲストルームです...*

## ソーシャルメディア

:::note
このセクションは任意です。
:::

`Social media` タブをクリックして続行してください。

「なぜゲストルームに独自のソーシャルメディアセクションがあるのか？」と思うかもしれません。ゲストルームは、施設の近くにあるが敷地外で独自のスタッフとIGプロフィールを持つヴィラである場合もあります。

ソーシャルメディアは、新規および既存の旅行者と継続的に繋がる手段です。
プロフィールにソーシャルネットワークアカウントを追加して、ゲストが簡単に連絡を取り、ゲストルームについて詳しく知ることができます。

<Steps>
1. 有効にしたいソーシャルネットワークのスイッチを切り替えます。
2. そのネットワークのアカウント名を入力します。
</Steps>

## 評判

:::note
このセクションは任意です。
:::

`Reputation` タブをクリックして続行してください。

`Social media` タブと同様に、このタブはゲストルームに別の評判スコアや特別な称賛を表示したい場合に使用します。

評判はデジタル通貨のようなもので、どこへ行っても持ち運びたいものです。
評判エントリーは、施設が誇りに思う第三者評価や、シェフが金賞を受賞したコンテストなど、何でも構いません。
このセクションでは、インターネット上のあらゆるスコアやランキングを追加できます。

`Add reputation score` ボタンをクリックしてください。

<Steps>
1. **カテゴリ** 追加したいスコアの種類を選択。*例: 第三者レビュー*
2. **提供者** スコアを授与した団体を入力。*例: BigOTA.com*
3. **評価タイプ** スコアの形式を選択。*例: 数値*
4. **日付** 任意でスコアを受け取った日付を入力。
5. **評価** 受け取ったスコアを入力。*例: 8.5*
6. **最大評価** 任意でこの評価の最大スコアを入力。*例: 10*
</Steps>

## 属性ベース販売

:::note
このセクションは任意です。
:::

`Attribute Based Selling` タブをクリックして続行してください。

すべての施設には独自のABSセクションがあります。このセクションでは、メインの在庫にアップセル機会を作成したり、非取引型の在庫タイプを取引型に変えることができます。
予約プロセス中により高い価格帯で予約者とつながる可能性があるため、これらの機会を作成することを強くお勧めします。

:::tip
ゲストルームには多くの付帯サービスを含めることができます：

- 高級マットレス
- デラックスバスルームキット
- Marshal Bluetoothスピーカー
- 快適な椅子付きの作業デスク
- 客室内朝食
- 客室内マッサージ
- 12本入りビールパック
- ワインボトル
- プレミアムチョコレート
- +++ 想像力を働かせて追加収入を得ましょう！！
:::

- **価格帯** 旅行者にこのゲストルームの価格帯を簡単に示す。そうです…ゲストルームの価格帯です。付帯サービスではありません。
- **有効** このゲストルームをすべてのマスターレートおよび販売チャネルで予約不可にしたい場合はスイッチを🛑にする。
- **コミッション対象** アフィリエイトがこの在庫でコミッションを得るかどうかを示す。
- **注目** このゲストルームをより目立つ位置に配置すべきかどうかを示す。
- **属性ベース販売** この在庫でABSを有効にするにはスイッチを✅にする。*有効にしたら、少なくとも1つのABS商品を販売用に追加する必要があります。*

### 付帯商品

ABSを有効にすると、付帯商品を追加して販売可能にできます。

*デフォルトで空の商品が1つ作成されています。`Add item for sale` ボタンをクリックしてさらに追加できます。*

- **名前** 付帯商品の名前を付ける。*例: Royceのシャンパンチョコレートトリュフ*
- **価格タイプ** 価格の計算方法を示す。*例: パドルボードレンタル。`Per hour` に設定すると、レンタル時間数に基づく価格になります。*
- **基本価格** この商品の通常価格を入力。*例: 20 / 時間*
- **割引価格** Winkユーザーが部屋と一緒に予約した場合の価格。*例: 15 / 時間*

### 制限

数量の最小/最大制限を設定して、この商品の利用可能性を制限できます。
例えば、パドルボードを時間単位でレンタルし、同時に最低2台の予約を必須にしたい場合は：

- **最小数量** 予約が必要な最小数量を制限。*例: パドルボード2台*
- **最大数量** 予約可能な最大数量を制限。*例: 最大制限なしの場合は空欄のまま*

### 説明

:::note
少なくとも1つの説明を書き、英語で記述する必要があります。
:::

- **名前** 付帯商品の名前を付ける。*例: スタンドアップパドルボード*
- **説明** 付帯商品を1〜2段落で説明する。*例: ただ座っているだけじゃなく、パドルを漕ごう！*

## 近接情報

`Proximity` タブをクリックして続行してください。

すべての在庫が施設内にあるわけではありません。敷地外にある在庫がある場合はここで旅行者に知らせることができ、別の地図位置や任意の連絡先情報を設定できます。

*例えば、ホテル所有のヴィラがホテルから数キロ離れている場合、旅行者がタクシー運転手に行き方を説明するのに役立ちます。*

`Save` ボタンをクリックして続行してください。

## API

`Guest Rooms` を管理したい開発者は、[Developers > APIs > Facilities](/ja/developers/apis/#facilities-api) にアクセスしてください。
