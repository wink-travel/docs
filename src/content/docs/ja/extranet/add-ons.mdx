アドオンは施設や体験の高度な形態です。アドオンを使うことで、施設内外でアドオンを紹介・販売できます。アドオンを作成するには、`Create add-on` ボタンをクリックしてください。

以下のセクションでは、アドオン管理の各ステップを順に説明します。

## 設定

開始時にデフォルトで選択されているのは `Configurations` タブです。

このセクションではアドオンの基本情報を設定します。

- **内部名** アドオンを内部で呼ぶ名前。*例: お化け屋敷*
- **ユニット数** 利用可能な数量を設定。*例: 10*
- **必須** 部屋の予約と一緒にこのアドオンが必須かどうか。*例: いいえ*
- **料金プラン** 制限を活用するために料金プランを任意で設定。

## スケジュール

このセクションではアドオンの利用可能時間を管理します。

### 営業時間

- **開始時間** アドオンが利用可能になる時間。*例: 09:00*
- **終了時間** アドオンが利用不可になる時間。*例: 21:00*

### 営業日

このセクションではアドオンが利用可能な曜日を設定します。

*スイッチを✅に切り替えて、その曜日が利用可能であることを示します。*

### 季節設定

:::note
ネイティブの「日/月」形式の入力フィールドはありません。この場合、日付セレクターから月と日だけを使用します。
:::

このセクションでは、アドオンが年間を通じて利用可能な日付を設定します。

- **開始日** アドオンが利用可能になる日付。*例: 2024年9月11日*
- **終了日** アドオンが利用不可になる日付。*例: 2024年7月11日*

## アメニティ

:::note
このセクションは任意です。
:::

`Amenities` タブをクリックして続行してください。

施設のアメニティと同様に、アドオンのアメニティは特定のアドオンにのみ適用されます。該当するアメニティにチェックを入れてください。

## 説明

:::note
少なくとも1つの説明を英語で記入する必要があります。
:::

`Descriptions` タブをクリックして続行してください。

施設のウェルカムテキストと同様に、アドオンのローカライズされた説明も作成できます。
希望するだけ多くの言語で記述可能です。人気のある言語の翻訳を追加します。
これは旅行者がアドオンを閲覧する際に表示されるテキストです。

- **名前** アドオンの名前を付けます。*例: 水上飛行機の送迎*
- **説明** アドオンについて1〜2段落で説明します。*例: 当島は水上飛行機でのみアクセス可能です...*

## ソーシャルメディア

:::note
このセクションは任意です。
:::

`Social media` タブをクリックして続行してください。

「なぜアドオンに独自のソーシャルメディアセクションがあるのか？」と思うかもしれません。アドオンは施設の近くにあるが敷地外で、独自のスタッフやInstagramプロフィールを持つ場合もあります。

ソーシャルメディアは新規および既存の旅行者と継続的に繋がる手段です。
プロフィールにソーシャルネットワークアカウントを追加して、ゲストが簡単に連絡を取り、このアドオンについて知ることができるようにしましょう。

<Steps>
1. 有効にしたいソーシャルネットワークのスイッチを切り替えます。
2. そのネットワークのアカウント名を入力します。
</Steps>

## 評判

:::note
このセクションは任意です。
:::

`Reputation` タブをクリックして続行してください。

`Social media` タブと同様に、このタブはアドオンが別の評判スコアや特別な称賛を持っている場合に使用します。

評判はデジタル通貨のようなもので、どこへ行っても持ち運びたいものです。
評判エントリーは、第三者の評価やシェフが金賞を受賞したコンテストなど、施設が誇りたいものなら何でも構いません。
このセクションでは、インターネット上のあらゆるスコアやランキングを追加できます。

`Add reputation score` ボタンをクリックしてください。

<Steps>
1. **カテゴリ** 追加したいスコアの種類を選択。*例: 第三者レビュー*
2. **提供者** スコアを授与した団体名を入力。*例: BigOTA.com*
3. **評価タイプ** スコアの形式を選択。*例: 数値*
4. **日付** 任意でスコアを受け取った日付を入力。
5. **評価** 受け取ったスコアを入力。*例: 8.5*
6. **最大評価** 任意でこの賞の最大スコアを入力。*例: 10*
</Steps>

## 属性ベース販売

:::note
このセクションは任意です。
:::

`Attribute Based Selling` タブをクリックして続行してください。

すべての施設には独自のABSセクションがあります。このセクションでは、メインの在庫にアップセル機会を作成したり、非取引型の在庫を取引型に変えたりできます。
予約プロセス中により高い価格帯で予約者とつながる可能性があるため、これらの機会を作成することを強くお勧めします。

:::tip
アドオンはABSを通じて販売可能にできます。販売方法はさまざまです：

- ゆっくりだが無料。
- 速いが高価。
- +++ 想像力を働かせて追加収入を得ましょう！！
:::

- **価格帯** 旅行者にこのアドオンの価格感を簡単に示します。
- **有効** このアドオンをすべての販売チャネルで予約不可にしたい場合はスイッチを🛑に切り替えます。
- **コミッション対象** アフィリエイトがこの在庫でコミッションを得るかどうかを示します。
- **注目** このアドオンを可能な場合により目立たせるかどうかを示します。
- **属性ベース販売** スイッチを✅に切り替えてこの在庫でABSを有効にします。*有効にしたら、少なくとも1つのABSアイテムを販売用に追加する必要があります*

### 付随アイテム

ABSを有効にすると、付随アイテムを追加して販売可能にできます。

*デフォルトで空のアイテムがすでに作成されています。`Add item for sale` ボタンをクリックしてさらに追加できます。*

- **名前** 付随アイテムの名前を付けます。*例: ゆっくりだが無料*
- **価格タイプ** 価格の計算方法を示します。*例: `Per person` に設定すると人数に基づく価格になります。*
- **基本価格** このアイテムの通常価格を入力。*例: 200 / 人*
- **割引価格** Winkユーザーが予約時に得られる価格を入力。*例: 150 / 人*

### 制限

数量の最小/最大制限を設定して、このアイテムの利用可能性を制限できます。

- **最小数量** 予約が必要な最小数量を制限。*例: 1名*
- **最大数量** 予約が可能な最大数量を制限。*例: 4名*

### 説明

:::note
少なくとも1つの説明を英語で記入する必要があります。
:::

- **名前** 付随アイテムの名前を付けます。*例: ゆっくりだが無料*
- **説明** 付随アイテムについて1〜2段落で説明します。*例: 古いプロペラ機を使って目的地へお連れします。*

## 近接情報

`Proximity` タブをクリックして続行してください。

すべての在庫が敷地内にあるわけではありません。敷地外にある在庫がある場合はここで旅行者に知らせることができ、別の地図位置や任意の連絡先情報を設定できます。

*アドオンがホテルから数キロ離れている場合に、旅行者がタクシー運転手に行き方を説明するのに役立ちます。*

`Save` ボタンをクリックして続行してください。
