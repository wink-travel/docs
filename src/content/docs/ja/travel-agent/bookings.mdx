Wink で予約が行われると、次のことが起こります。

<Steps>
  1. チャネル マネージャー、CRS、または PMS に通知されます。
  2. 確認メールが届きます *（予約デスク連絡先）* 新しい予約を管理するためのリンクが表示されます。
  3. 旅行者は、予約デスクへの連絡方法も記載された確認メールを受け取ります。
  4. 旅行代理店は旅行者と同じ予約確認を受け取ります。
  5. 開発者は、 `booking.created` webhook イベントに参加し、詳細な予約情報をリアルタイムで受け取ります。
</Steps>


メール内のリンクに従うか、`Account > Bookings`開始するには、メインナビゲーションバーから開始します。

## 予約グリッド

予約グリッドには、過去と将来のすべての予約が予約日順に昇順で表示されます。

### フィルター

フィルターを適用して、探している予約を見つけることができます。

* 予約日\[範囲]
* 到着日 \[範囲]
* 出発日 \[範囲]
* 予約コード
* 旅行者の姓
* 旅行者の名
* マスターレート名
* 物件名

### 輸出

現在のフィルター条件に一致する予約を CSV にエクスポートできます。

予約をエクスポートするには:

<Steps>
  1. 必要なフィルターがアクティブになっていることを確認してください。
  2. 予約グリッドが更新されるまで待ちます。
  3. 🗂️ (*ファイルのエクスポート*) アイコンボタンをクリックします。
</Steps>


## 予約

予約グリッドから操作したい予約を選択すると、`Booking`このページには、到着日にゲストをきちんとお迎えするために必要なすべての情報が記載されています。

このページには、お客様に適切なサービスを提供するために必要な重要なアクションもいくつか記載されています。利用可能なアクションはすべて`Actions`ページ上部のドロップダウン。

### メールを再送信

旅行者に確認メールを再送するには、`Actions > Resend confirmation email`。

### 予約をキャンセルする

宿泊施設として予約をキャンセルするには、`Actions > Cancel booking`。

次の操作を実行できるポップアップ ウィンドウが表示されます。

<Steps>
  1. キャンセルの種類を選択してください。 *例：日付が間違っている*
  2. キャンセルの理由を入力してください。
</Steps>


*このボタンはキャンセルポリシーのルールに従うため、常に利用できるとは限りません。*

## API

管理したい開発者`Bookings`旅行代理店として、[開発者 > API > 旅行代理店](/developers/apis/#travel-agent-api)。
