Winkで予約が行われると、以下のことが起こります：

<Steps>
1. あなたのチャネルマネージャー、CRS、またはPMSに通知されます。
2. 新しい予約を管理するためのリンクが記載された確認メール（*予約デスクの連絡先*）が届きます。
3. 旅行者には、予約デスクへの連絡方法も含まれた確認メールが届きます。
4. 旅行代理店にも旅行者と同じ予約確認メールが届きます。
4. 開発者は`booking.created`のWebhookイベントを受け取り、リアルタイムで詳細な予約情報を取得できます。
</Steps>

メール内のリンクをたどるか、メインナビバーの`アカウント > 予約`から開始してください。

## 予約グリッド

予約グリッドには、過去および今後のすべての予約が、予約日順（昇順）で表示されます。

### フィルター

目的の予約を見つけるためにフィルターを適用できます。

- 予約日 [範囲]
- 到着日 [範囲]
- 出発日 [範囲]
- 予約コード
- 旅行者の姓
- 旅行者の名
- マスターレート名
- 施設名

### エクスポート

現在のフィルター条件に合致する予約をCSV形式でエクスポートできます。

予約をエクスポートするには：

<Steps>
1. 希望のフィルターが有効になっていることを確認します。
2. 予約グリッドの更新を待ちます。
2. グリッドの右下にある🗂️（*ファイルエクスポート*）アイコンボタンをクリックします。
</Steps>

## 予約詳細

予約グリッドから操作したい予約を選択すると、`予約`ページにリダイレクトされます。
このページには、到着日にゲストを適切に迎えるために必要なすべての詳細が含まれています。

また、ゲストに適切なサービスを提供するために必要な重要な操作もいくつかあります。利用可能なすべての操作はページ上部の`アクション`ドロップダウンで確認できます。

### 確認メールの再送

旅行者に確認メールを再送するには、`アクション > 確認メールを再送`をクリックしてください。

### 予約のキャンセル

施設側で予約をキャンセルするには、`アクション > 予約をキャンセル`をクリックしてください。

ポップアップウィンドウが表示され、以下を行えます：

<Steps>
1. キャンセルの種類を選択します。*例：日付の誤り*
2. キャンセル理由を入力します。
</Steps>

*このボタンはキャンセルポリシーに従って動作するため、常に利用可能とは限りません。*

## API

旅行代理店として`予約`を管理したい開発者は、[Developers > API > Travel Agent](/ja/developers/apis/#travel-agent-api)をご覧ください。
