Winkのようなインテグレーターで、外部支払先をTripPayで使用するためにマッピングしたい場合、次の2つの方法のいずれかで行うことができます。

- 手動マッピング
- [Payment API](/ja/developers/apis/#payment-api) を使用する (*`Account Mappings`を参照*)

:::tip
同じ物件が少数だけの場合を除き、このステップはAPIを使って自動化することをお勧めします。
:::

以下では、外部支払先を手動でマッピングする方法を説明します。*内部的にはAPIを使用しています。*

<Steps>
1. [TripPay](https://pay.wink.travel)にログインします。
2. インテグレーターアカウントを選択します。
3. メインナビバーから `Settings > Mappings` をクリックします。
4. `Create mapping` ボタンをクリックします。マッピングフォームが表示されます。
5. **Local account** マッピングしたい既存のTripPayアカウント名を入力し始めます。
6. **External identifier** リモートシステムに存在するこのエンティティの外部IDを貼り付けます。
7. `Save` をクリックして続行します。マッピング一覧にリダイレクトされます。
</Steps>

マッピングが設定されると、TripPayは外部識別子を使って参照した際に、予約の受益者としてどのアカウントに入金すべきかを認識します。

## API

`Payments`を管理したい開発者は、[Developers > API > Payments](/ja/developers/apis/#payment-api)にアクセスしてください。
