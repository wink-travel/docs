Wink のようなインテグレーターで、TripPay で使用するために外部受取人をマッピングしたい場合は、次の 2 つの方法のいずれかで実行できます。

* 手動マッピング
* 使用方法[決済API](/developers/apis/#payment-api)（*見る `Account Mappings`*）

:::ヒント
少数の同じプロパティのみを扱う場合を除き、API を使用してこの手順を自動化することをお勧めします。
:::

以下では、外部受取人を手動でマッピングする手順を説明します。*内部的には API を使用します。*

<Steps>
  1. ログイン [トリップペイ](https://pay.wink.travel)。
  2. インテグレーター アカウントを選択します。
  3. クリック `Settings > Mappings` メインナビゲーションバーから。
  4. クリック `Create mapping` ボタンをクリックします。マッピングフォームが表示されます。
  5. **ローカルアカウント** マッピングする既存の TripPay の名前を入力し始めます。
  6. **外部識別子** リモート システムに存在するこのエンティティの外部 ID を貼り付けます。
  7. クリック `Save` 続行するには、マッピングのリストにリダイレクトされます。
</Steps>


マッピングが完了すると、TripPay は外部識別子を使用して参照したときに、どのアカウントを予約の受益者としてクレジットするかを認識できるようになります。

## API

管理したい開発者`Payments`、へ[開発者 > API > 支払い](/developers/apis/#payment-api)。
