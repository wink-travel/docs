アプリケーションを使用すると、WinkとTripPayの機能をフルに活用でき、FacebookやGoogleと同様に、OAuth2認証サーバーを使用してユーザーがサイトへの認証とログインを行うことができます。
Winkを使えば、独自のOTAを構築できます。まずはアプリケーションから始めましょう。

*WinkアフィリエイトアカウントまたはTripPayアカウントを新規登録すると、デフォルトでアプリが提供されます。`client ID`いくつかのシナリオの場合。*

*この例では、[ウィンクスタジオ](https://studio.wink.travel)。*

<Steps>
  1. 認証されると、右上隅にあるプロフィールアイコンをクリックすると、メニューが表示されます。
  2. クリック `Applications` リンク。
  3. アプリページにリダイレクトされます。
</Steps>


## アプリを作成

アクセスできないので`secret key`私たちが作成したアプリの代わりに、独自のアプリを作成する必要があります。

:::注意
アプリを作成する前に、Wink のアフィリエイトアカウントまたは TripPay のアカウントを作成する必要があります。アプリは特定のアカウントに紐付けられるためです。
:::

アプリケーションを作成する手順は次のとおりです。

<Steps>
  1. アプリページで、 `Create new application` ボタン。
  2. ***名前*** アプリに名前を付けます。 *例：クールアプリ*
  3. **エンティティ名** アプリを既存のアカウントの 1 つにリンクします。 *例：クールアカウント*
  4. **リダイレクトURI** 少なくとも1つのリダイレクトURIが必要です。これは、認証が成功した後にリダイレクトドメインを検証するために使用されます。 *例えばhttps://www.cool-site.com*
  5. クリック `Save` 続行するにはボタンをクリックしてください。
</Steps>


アプリケーションリストに戻ります。リストに新しいアプリが表示され、アプリのシークレットキーにアクセスできるのは今回のみです。必ずコピーして安全な場所に保存してください。

## アプリを更新する

リダイレクト URI のいずれかが変更された場合、または URI を追加または削除する場合は、アプリを更新する必要があります。

既存のアプリを更新するには、次の手順を実行します。

<Steps>
  1. クリック `Actions` 更新したいアプリのリンク。
  2. クリック `Update` \[アクション] の下のボタンをクリックします。
  3. アプリに変更を加えます。
  4. クリック `Save` 続行するにはボタンをクリックしてください。
</Steps>


アプリケーションのリストにリダイレクトされます。

## アプリを削除

:::注意
アカウントのすべての機能が動作することを確認するために自動作成されたアプリを削除しないでください。
:::

アプリを今後使用しない場合は、削除しても構いません。

アプリを削除するには、次の手順を実行します。

<Steps>
  1. クリック `Actions` 更新したいアプリのリンク。
  2. クリック `Remove` \[アクション] の下のボタンをクリックします。
  3. クリック `OK` 削除を確認するボタン。
</Steps>


アプリケーションのリストにリダイレクトされます。
