Winkで予約が作成、更新、またはキャンセルされると、プラットフォームはその変更を接続された外部システム（プロパティマネジメントシステム（PMS）、チャネルマネージャー、中央予約システム（CRS））と同期します。

## 仕組み

各予約の変更は、同期状態を追跡する**同期エントリ**を作成します。プラットフォームは同期エントリを非同期で処理し、失敗した場合は再試行して確実な配信を保証します。

## 同期のライフサイクル

すべての同期エントリは状態マシンに従います：

```
┌─────────────┐     success     ┌───────────┐
│  TO_CREATE   │ ──────────────→ │ PROCESSED │
└─────────────┘                  └───────────┘
       │
       │ failure
       ▼
┌──────────────────┐
│ TO_CREATE_FAILED │
└──────────────────┘
```

同じパターンが更新とキャンセルにも適用されます：

| アクション | 保留状態 | 成功状態 | 失敗状態 |
|---|---|---|---|
| 新規予約 | `TO_CREATE` | `PROCESSED` | `TO_CREATE_FAILED` |
| 予約更新 | `TO_UPDATE` | `PROCESSED` | `TO_UPDATE_FAILED` |
| キャンセル | `TO_CANCEL` | `PROCESSED` | `TO_CANCEL_FAILED` |

## 同期のトリガー

- **新規予約** — 旅行者がWinkまたは提携チャネルを通じて予約を行う。
- **予約更新** — 予約の詳細（日時、ゲスト情報、部屋割り）が変更される。
- **キャンセル** — 旅行者または施設によって予約がキャンセルされる。

## 監視

接続されたチャネルマネージャーやPMSのダッシュボードから同期状況を監視できます。失敗した同期エントリは自動的に再試行されます。同期が継続して失敗する場合は、調査のために`*_FAILED`状態のまま残ります。

## トラブルシューティング

予約がPMSに表示されない場合：

1. **チャネルマネージャーの接続を確認** — PMSまたはチャネルマネージャーの統合が有効で正しく設定されていることを確認してください。
2. **Webhook配信を検証** — Webhookを使用している場合、エンドポイントが`booking.create`および`booking.cancelled`イベントを受信しているか確認してください。詳細は[Webhook Events Catalog](/ja/developers/webhook-events#booking-events)をご覧ください。
3. **失敗したエントリを確認** — 失敗した同期エントリは、対象システムとの接続またはデータの問題を示しています。
4. **サポートに連絡** — 問題が解決しない場合は、Winkチームが失敗した同期エントリの調査を代行します。
