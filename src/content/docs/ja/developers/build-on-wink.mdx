> 「もし私がもっと遠くまで見ることができたのは、巨人の肩の上に立っていたからです。」

\--*アイザック・ニュートン卿*

このようなプラットフォームは、単独で構築されたものではありません。むしろ、古いものから新しいものまで、多くの可動部品の集合体です。私たちは、後を継ぐ人々にも貢献し続けたいと考えています。

> 「壊れた車輪を再発明する必要はない。」

旅行テクノロジーは断片化しており、時代遅れになっています。私たちは、古くからある良いものを取り入れようと努めてきました。多くのチャネルマネージャーや宿泊施設管理システムなどを統合し、新しい機能もいくつか追加しました。そして、これらはすべて、業界標準とプロトコルで標準化されており、使い慣れているので、お客様は何もする必要はありません。

:::note\[API 駆動型アプローチ]
つまり、私たちが自分たちで構築する機能は、大まかに言えば、次のようなものであるべきです。

* 理解できる
* フレキシブル
* スケーラブル
* 再利用可能で...
* 拡張可能

...他のユーザーもこれらの API を使用できるようになります。
:::

ウィンクはそのような`API-driven approach`この設計哲学により、Wink のすべての機能は、外部開発者であってもまるで自社システムであるかのように操作できます。また、Wink との連携において活用できるプラットフォームの側面も数多くあります。
