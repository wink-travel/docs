Google Analytics 4 プロパティを Wink に接続すると、部屋の閲覧から購入完了までの予約イベントが、Wink 独自の分析と並んで自動的にご自身の GA4 ダッシュボードに転送されます。コード不要、技術的な設定も不要です。

<Aside type="tip">
必要なのは GA4 の Measurement ID のみです。Wink ポータルに入力すれば、すぐに分析が開始されます。
</Aside>

---

## 仕組み

```
Wink ポータルに G-XXXXXXXXXX を入力
       ↓
Wink が予約イベントを GA4 プロパティに転送
       ↓
analytics.google.com にログインして予約状況を確認
```

---

## やること

<Steps>

1. **GA4 プロパティを作成する**

   [analytics.google.com](https://analytics.google.com) にアクセスし、管理 → プロパティを作成。タイムゾーンと通貨は市場に合わせて設定してください。

2. **ウェブデータストリームを作成する**

   管理 → データストリーム → ストリームを追加 → ウェブ。URL に `wink.travel` を入力。これで **Measurement ID**（`G-XXXXXXXXXX`）が生成されます。コピーしてください。

3. **Wink ポータルに Measurement ID を入力する**

   [my.wink.travel](https://my.wink.travel) にログインし、アカウントを選択して **カスタマイズ** タブへ。Measurement ID を貼り付けて保存。これで GA4 連携が有効になります。

4. **`purchase` を重要イベントに設定する**

   GA4 プロパティに戻り、管理 → イベント → `purchase` を探して **重要イベントとしてマーク** をオンにします。これで完了した予約がコンバージョンレポートに表示されます。

5. **データを確認する**

   [analytics.google.com](https://analytics.google.com) にログインすると、予約、収益、ゲストのファネルが Wink を通じて自動的に追跡されているのが確認できます。

</Steps>

<Aside type="note">
この連携のためにご自身のウェブサイトにトラッキングコードを設置する必要はありません。Wink がすべて処理します。Measurement ID は予約データの送信先に過ぎません。
</Aside>

---

## 追跡できる内容

接続後、GA4 プロパティは物件向けの拡張 Eコマースイベントの全セットを受信します。

### 発見段階

| イベント | 意味 |
|---|---|
| `view_item_list` | ゲストが検索やカテゴリ一覧で物件を見た |
| `select_item` | ゲストが一覧から物件をクリックした |
| `view_item` | ゲストがホテル詳細ページを閲覧した |

### 検討段階

| イベント | 意味 |
|---|---|
| `view_promotion` | ゲストが物件のプロモーションを見た |
| `select_promotion` | ゲストがプロモーションをクリックした |
| `add_to_wishlist` | ゲストが物件をウィッシュリストに保存した |

### 予約ファネル

| イベント | 意味 |
|---|---|
| `add_to_cart` | ゲストが部屋やパッケージを選択した |
| `view_cart` | ゲストが予約概要を見た |
| `remove_from_cart` | ゲストが部屋やパッケージを削除した |
| `begin_checkout` | ゲストがチェックアウトを開始した |
| `add_payment_info` | ゲストが支払い情報を入力した |
| `add_shipping_info` | ゲストがゲスト情報を入力した |
| `purchase` | ゲストが予約を完了した |
| `refund` | 予約がキャンセルされ返金された |

すべてのイベントには収益、通貨、部屋タイプ、予約金額などの関連ディメンションが含まれ、最初の接触から予約完了までのゲストの行動を完全に把握できます。`city_name`、`cancellation_policy`、`bed_type` などの物件・部屋レベルの追加ディメンションを利用するには、[GA4 のカスタムディメンション](/ja/guides/gtm-dual-tagging#custom-dimensions-in-ga4) をご覧ください。

## クロスドメイントラッキング（上級者向け）

上記の標準連携は Wink 内でのすべての動きを追跡します。しかし、ご自身のウェブサイトから Wink での予約完了までのゲストの行動を追跡し、元のトラフィックソース（Google 広告、メールキャンペーン、オーガニック検索など）を正しく帰属させたい場合は、双方で追加の設定が必要です。

<Aside type="caution">
このセクションはご自身のウェブサイトに GA4 を導入している場合のみ該当します。サイトで GA4 を使っていない場合は、上記の標準連携だけで十分です。
</Aside>

### なぜ重要か

クロスドメイントラッキングがないと、最初にあなたのウェブサイトを訪れてから Wink で予約したゲストは、GA4 上では元の訪問と切り離された **直接** セッションとして扱われます。Google 広告のクリックやメールキャンペーン、オーガニック検索など、予約に至るまでの流入元は正しく評価されません。

クロスドメイントラッキングを設定すると、ゲストの行動が一連の流れとして見えます：

```
google.com（有料広告）
       ↓
あなたのウェブサイト（ゲストが部屋を閲覧）
       ↓
wink.travel（ゲストが部屋を選択し予約）
       ↓
購入が元の Google 広告クリックに正しく帰属
```

### あなたがやること

GA4 プロパティで `wink.travel` を自分のドメインの一部として認識させ、外部リファラーとして扱わないように設定します。これにより GA4 は２つのセッションをつなげて一連のゲスト行動として扱います。

<Steps>
1. GA4 管理 → データストリーム → 対象のストリーム
2. タグ設定の構成 → **ドメインの構成**
3. `wink.travel` をリンクドメインとして追加
4. 保存
</Steps>

### Wink がやること

[Wink サポート](https://my.wink.travel) に連絡し、あなたのウェブサイトのドメイン（例：`yourhotel.com`）を伝えてください。Wink は Wink の GA4 プロパティのリファラー除外リストにあなたのドメインを追加します。これにより、あなたのサイトがリファラーとして記録されず、ゲストの元の帰属情報が上書きされるのを防ぎ、新しいセッションが始まるのを防止します。

<Aside type="tip">
標準連携はほとんどの物件に十分で、Wink 上の予約や収益をすぐに把握できます。クロスドメイントラッキングは GA4 のデータに慣れてきて、より詳細な帰属分析をしたい場合に設定を検討してください。
</Aside>
