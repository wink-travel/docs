カスタマイズは、旅行者に表示される在庫の見え方を制御します — テーマカラー、デフォルト通貨、言語、旅程などです。これらは共有可能なリンク、埋め込みカード、グリッド、地図、予約エンジン全体に適用されます。

## カスタマイズとは何ですか？

カスタマイズは、以下を定義する設定プロファイルです：

- **表示通貨** — 価格が表示される通貨。
- **表示言語** — コンテンツのデフォルト言語。
- **デフォルト旅程** — 事前に入力されたチェックイン/チェックアウト日とゲスト数。
- **テーマ** — 埋め込みコンポーネントに適用される色とブランディング。
- **設定** — 在庫の表示方法（レイアウト、並び替え）。

異なる市場やユースケース向けに複数のカスタマイズを作成できます。

## カスタマイズの作成

<Steps>
1. https://studio.wink.travel （または https://my.wink.travel のアカウント）にアクセスします。
2. サイドバーから `Customization` に移動します。
3. `Create a new customization` をクリックします。
4. 説明的な名前を付けます（例：「Europe - EUR」、「Japan Market」、「Default」）。
5. 表示通貨、言語、デフォルト旅程を設定します。
6. ブランドに合わせてテーマカラーを設定します。
7. `Save` をクリックします。
</Steps>

## カスタマイズの使用場所

作成後、カスタマイズはプラットフォーム全体で参照されます：

| 機能 | カスタマイズの適用方法 |
| -- | -- |
| **共有可能リンク** | リンクはカスタマイズの通貨、言語、旅程をデフォルトとして使用します |
| **埋め込みカード** | カードはテーマカラーと表示設定でレンダリングされます |
| **グリッドと地図** | 在庫グリッドと地図マーカーはカスタマイズに従います |
| **予約エンジン** | 予約フロー全体がカスタマイズの設定を使用します |
| **ソーシャル投稿** | AI生成投稿は価格表示に特定のカスタマイズを使用できます |

## 複数のカスタマイズ

一般的なパターンは、ターゲット市場ごとに1つのカスタマイズを作成することです：

- **「US Market」** — USD、英語、デフォルト2泊滞在。
- **「Europe」** — EUR、英語、デフォルト3泊滞在。
- **「Japan」** — JPY、日本語、デフォルト1泊滞在。

共有可能リンクを作成したりカードを埋め込む際に、使用するカスタマイズを選択します。これにより、同じ物件でも対象となる顧客によって異なる表示が可能です。

## URLによる上書き

共有可能リンクのURLパラメータを使ってカスタマイズのデフォルトを上書きすることもできます：

- `l=es` — 言語をスペイン語に上書き。
- `c=EUR` — 通貨をEURに上書き。
- `sd=2024-08-24&n=2` — 旅程を特定の日付と泊数に上書き。

URLパラメータの完全なリストは[The Book Now Button](/ja/guides/the-book-now-button)をご覧ください。
