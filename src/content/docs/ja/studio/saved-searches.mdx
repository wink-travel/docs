保存された検索機能は、その名の通り、旅行の在庫を検索する際に、検索クエリを保存して後で使用できる機能です。

## 検索クエリを保存しています

在庫の検索を開始すると（[検索](/studio/search)）では、検索フィルターを細かく調整することができます。
クエリフォームの下部には、`Save search`。

<figure>
  <CloudinaryImage width="350" height="329" resize="scale" quality="best" alt="検索を保存" src="academy/save-search" errorImage="noimage_opaque_nyrtl0.png" />

  <figcaption>*検索クエリを保存するためのリンクを表示します*</figcaption>
</figure>


このリンクをクリックすると、この検索に名前を付けることができます（*例: 魔法のホテルの私の図書館*) 後で思い出せるような内容にします。

<figure>
  <CloudinaryImage width="350" height="329" resize="scale" quality="best" alt="検索に名前を付ける" src="academy/name-save-search" errorImage="noimage_opaque_nyrtl0.png" />

  <figcaption>*検索に名前を付ける*</figcaption>
</figure>


名前を付けてクリックします`Save search`ボタンをクリックします。その後、`Inventory > Saves Searches`メインナビゲーションバーから、新しく作成されたレコードを表示します。

## リストアクション

保存した検索を使用すると、旅行の在庫をグリッドまたはマップの形式でユーザーに公開し、検索結果を希望どおりに表示することができます。

<figure>
  <CloudinaryImage width="700" height="939" resize="scale" quality="best" alt="保存された検索アクション" src="academy/saved-search-actions" errorImage="noimage_opaque_nyrtl0.png" />

  <figcaption>*フィルターが適用されていない保存された検索クエリ*</figcaption>
</figure>


保存されたすべての検索クエリには、実行できるアクションがあります。

* 検索クエリを[地図](/studio/maps)。
* 検索クエリを[グリッド](/studio/grids)。
* 保存した検索を削除します。

## API

保存した検索を管理したい開発者は、[開発者 > API > 閲覧](/developers/apis/#browse-api)。
