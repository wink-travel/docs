カスタマイズは、コードを一切書かずにWinkとの連携のあらゆる側面をコントロールできる強力な機能です。販売するすべての在庫をカスタマイズできます。機能の追加や削除、カラースキームの変更など、様々なカスタマイズが可能です。

乗っている間`Customization`ページで、`Actions`作業したいカスタマイズの下のボタンをクリックし、`Update`。*アカウント作成時に既にデフォルトのエントリが設定されています。このエントリは削除できません。*

するとカスタマイズフォームが開き、5～7個のタブが表示されます（*アカウントの種類に応じて*）。

## 一般的な

利用可能なオプションは次のとおりです。`General`タブ。

* **名前**カスタマイズに簡単に覚えられる名前を付けます。*例えば、私の旅行ブログの場合*
* **通貨を表示**予約可能な旅行在庫の横にユーザーに表示される初期通貨を選択します。*例：米ドル*
* **表示言語**ユーザーが最初に目にする言語を選択してください。*例：英語*
* **ページタイトルに追加**ページタイトルに追加される内容を制御できます（*ブラウザタブに表示されるタイトル*）。これは主に予約エンジンに使用されます。*例えば、<https://ota.wink.travel>デフォルトでは、`Home | Traveliko - Hotels with Benefits`*. '|' パイプの後のすべてを変更できます。

## テーマ

その`Theme`タブでは、[予約エンジン](/booking-engine/in-a-nutshell) Webサイト、 [ウィンクリンクス](/link-manager/wink-links)ウェブサイトとすべての[Webコンポーネント](/developers/web-components)。

### 構成

このセクションでは、デザインとカードのレイアウトを設定できます。現時点では、Wink というデザインが 1 つだけあります。

カードレイアウトは、ユーザーに表示する取引カードを横並びで表示するか縦並びで表示するかを制御します。表示される情報は同じです。

:::注
モバイル端末で取引カードを閲覧する場合、カードレイアウトに違いはありません。ただし、iPad miniや高解像度の端末では違いが確認できます。
:::

:::tip\[プロのヒント]
リスト内の旅行在庫（客室タイプなど）が4つ未満の場合、横長レイアウトを使用すると見栄えが良くなります。
:::

### 色

色は、UIコンポーネントやウェブサイトのユーザーへの見え方を制御します。既存のカラースキーム（もしあれば）に合わせて色を設定してください。

豊富な定義済みカラーをご用意しておりますので、ぜひお試しください。定義済みのテーマは、以下のフォームフィールドの色を変更します。また、完全に独自の色を設定することもできます。

## 旅程

その`Itinerary`選択した在庫の価格が確実に表示されるようにする点では、このタブが最も興味深いでしょう。
旅行在庫の価格設定はかなり複雑なため、Wink は正確な価格を表示する前にユーザーの旅程を把握する必要があります。

このセクションでは、次のようなすべてのデジタル不動産の価格を表示するためのデフォルトの旅程を設定できます。

* 価格オーバーレイを備えたソーシャルフレンドリーな画像。
* WinkLinks のグリッド、カード、リンク。
* Webコンポーネント
* 予約エンジン

### 日付

静的な日付範囲を設定するか、ローリングの日付範囲を設定するかを選択できます。

* 特定の日付に発生する特定のイベントに対してカスタマイズを使用する場合は、静的な日付範囲を選択します。
* 次のいずれかを行う場合は、ローリング日付範囲を選択します。
  1. デフォルトは今日 + 1 泊です。ほとんどの OTA ではこれが適用されます。
  2. 常に空きがある将来の日付をデフォルトに設定します。

### 部屋の構成

デフォルトでは、旅程は大人2名に設定されています。これは、ほとんどの旅行者にとって最も一般的な設定です。
これは簡単に変更でき、大人と子供の両方を含む複数の宿泊者が宿泊する複数の部屋を追加できます。

### プロモーション

プロモーションは、ユーザーの時間を節約できる便利な機能です。宿泊施設から友人と共有できるプロモーションコードが提供されている場合は、ここで入力できます。
あなたのリンクなどからアクセスしたユーザーの旅程にプロモーションコードが自動的に追加され、割引価格が自動的に適用されます。

## 統合

その`Integration`このタブは、Wink での予約の分配方法をより細かく制御したいホテル、インテグレーター、旅行代理店向けです。

### 通知

これにより、ホテルとインテグレーターは、予約が発生した際にWinkから関係者全員に通知を受け取るオプションを利用できるようになります。これにはチャネルマネージャーへの通知も含まれます。

デフォルトでは、これらのオプションはすべて有効になっています。インテグレーターがこれらの機能の1つ以上を無効にする必要があるユースケースは次のとおりです。

* 旅行代理店は API レベルで統合されており、独自の確認メールを送信したいと考えています。
* ホテルは統合をテストしており、営業スタッフに迷惑をかけたくないと考えています。
* ホテルは電子メールを気にしておらず、PMS 経由でのみ通知を受け取りたいと考えています。

### 電子メールのロゴ

弊社の電子メール通知サービスを使用したいインテグレーターは、その電子メールが自分たちから送信されたように見えるようにカスタマイズできます。

:::注
推奨サイズは幅300ピクセル x 高さ75ピクセルです。
:::

## ホテル固有

の下で`Hotel-specific`タブには、ホテルが予約エンジンのランディング ページに何を表示するか、またリクエストされた旅程に在庫がない場合に何を表示するかを制御できるオプションがあります。

* **ランキング**: まだ意味のあるランキングを獲得していないプロパティでは、その機能を無効にすることを選択する場合があります。

:::注
宿泊施設側では、アフィリエイトがホテルのランディングページをどのように表示するかを選択することはできません。これらは2つの異なる設定です。
:::

* **可用性**: 空きがない場合でも在庫を表示し続けるかどうかをプロパティで選択できます。

:::note
ほとんどの場合、プロパティはユーザーにクリック方法を提供する必要があります。`Book`ボタンを押すと、ランディングページに移動し、旅程を変更して空いている日付を探すことができます。
ユーザーが先へ進むのをブロックしたい場合は、このスイッチを有効にしてください。
:::

## 使い方

ほとんどの場合、`Customization`リンク、カード、マップ、グリッドに適用します。

手動で構成を適用する場合、次のような場合が考えられます。

* **手動オーバーライド**予約エンジンへのリンクを直接共有したいのですが、[共有可能なリンク](/studio/shareable-links)カスタム構成が適用された機能。
* **WordPressプラグイン**Winkに設定を追加する[WordPressプラグイン](/developers/wordpress) `Settings`。
* **カスタム統合**弊社のツールを使ってカスタムインテグレーターに手動で適用します。[ウィンクアプリローダー](/developers/web-components/#application-loader)Web コンポーネント。

### 手動オーバーライド

ここでは、すべての設定を含む構成 ID を当社のメイン予約エンジン URL に適用する方法の簡単な例を示します。

<h4 align="center">[https://ota.wink.travel?configurationId=MY-UNIQUE-ID](https://ota.wink.travel?configurationId=MY-UNIQUE-ID)</h4>


:::注
ユーザーがこのリンクをクリックすると、アフィリエイトアトリビューションCookieも設定されます。
:::

### WordPressプラグイン

に移動[ウィンクWordPressプラグイン](/developers/wordpress)詳細については、こちらをご覧ください。

### カスタム統合

ページ上の埋め込まれたすべてのWink Webコンポーネントをコントロールするには、[ウィンクアプリローダー](/developers/web-components/#application-loader)Web コンポーネント。

以下は、`MY-UNIQUE-ID`構成IDとして。

```html {11}
<html>
  <head>
    <link rel="stylesheet" href="https://elements.wink.travel/styles.css"></link>
  </head>
  <body>
    
    <script src="https://elements.wink.travel/elements.js" type="module" defer></script>

    <wink-app-loader
      client-id="YOUR CLIENT ID GOES HERE"
      configuration-id="MY-UNIQUE-ID"
    ></wink-app-loader>
  </body>
</html>
```
