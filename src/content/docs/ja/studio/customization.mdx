カスタマイズは、Winkとの統合のあらゆる側面をコードを書かずにコントロールできる強力な機能です。販売するすべての在庫はカスタマイズ可能です。機能の追加や削除、カラースキームの変更など、多彩な調整が可能です。

`Customization` ページで、操作したいカスタマイズの下にある `Actions` ボタンをクリックし、`Update` を選択します。  
*アカウント作成時にデフォルトのエントリーがすでに用意されています。このエントリーは削除できません。*

これにより、5～7つのタブ（*アカウントタイプによって異なります*）が表示されるカスタマイズフォームに移動します。

## 一般

`General` タブで利用可能なオプションは以下の通りです。

- **名前** カスタマイズにわかりやすい名前を付けます。*例：私の旅行ブログ用*
- **表示通貨** ユーザーが最初に表示される通貨を選択します。*例：USD*
- **表示言語** ユーザーが最初に遭遇する言語を選択します。*例：英語*
- **ページタイトルに追加** ページタイトル（ブラウザのタブに表示されるタイトル）に追加される内容を制御できます。主に当社の予約エンジンで使用されます。*例：https://ota.wink.travel にアクセスすると、デフォルトで `Home | Traveliko - Hotels with Benefits` と表示されます。'|' パイプ以降の部分を変更可能です。*

## テーマ

`Theme` タブでは、当社の[予約エンジン](/booking-engine/in-a-nutshell)サイト、[WinkLinks](/link-manager/wink-links)サイト、およびすべての[Web Components](/developers/web-components)の外観をカスタマイズできます。

### 設定

このセクションではデザインとカードレイアウトを設定できます。現在はWinkという1つのデザインのみです。

カードレイアウトは、ユーザーに横並びか縦並びのトランザクションカードを表示するかを制御します。情報は同じです。

:::note
モバイルデバイスで当社のトランザクションカードを表示する場合、カードレイアウトに違いはありません。ただし、iPad mini以上の解像度では違いが見られます。
:::

:::tip[プロのヒント]
リスト内の旅行在庫（例：部屋タイプ）が4つ未満の場合は横並びレイアウトを使うと見栄えが良くなります。
:::

### 色

色はユーザーが当社のUIコンポーネントやウェブサイトをどのように見るかを制御します。既存のカラースキームに合わせて使用してください（ある場合）。

すでに多くのプリセットカラーが用意されており、すぐに試すことができます。プリセットテーマは下のフォームフィールドの色を変更します。完全に独自の色も使用可能です。

## 分析

GA4分析を標準でサポートしています。強化されたeコマースのコンバージョンパス全体を追跡したい場合は、計測IDを用意し、`Analytics` タブのフィールドに入力してください。

<Aside type="caution" title="リマインダー">
  GA4プロパティのストリームに `i.trvl.as`、`book.wink.travel`、`ota.wink.travel` を追加するのを忘れないでください。
</Aside>

## マップ

当社のマップをサイトに埋め込むには、[WordPressプラグイン](/developers/wordpress)または[Web Components](/developers/web-components)を使用する場合でも、ご自身のGoogle Maps APIキーが必要です。

:::note
  [ここでGoogle Maps APIキーを作成](https://console.developers.google.com/projectselector2/apis/credentials?supportedpurview=project)
:::

キーを生成し、サイト用に設定したら、`Analytics` タブに入力してください。

## 旅程

`Itinerary` タブは、選択した在庫の価格表示を確実にするために最も重要な部分です。  
旅行在庫の価格設定は複雑なため、Winkは正確な価格を表示するためにユーザーの旅程を知る必要があります。

このセクションでは、以下のような当社のデジタル資産全体で価格を表示するためのデフォルト旅程を設定できます。

- 価格オーバーレイ付きのソーシャル向け画像
- WinkLinksのグリッド、カード、リンク
- Web Components
- 予約エンジン

### 日付

静的な日付範囲か、ローリング（日付が変動する）範囲かを選択できます。

- 特定の日付に開催されるイベント用にカスタマイズを使う場合は静的日付範囲を選択。
- ローリング日付範囲は以下のいずれかを選択：
    1. 多くのOTAが採用している「今日＋1泊」をデフォルトにする。
    2. 常に空室があるはずの将来の日付をデフォルトにする。

### 部屋の構成

デフォルトでは旅程は大人2名に設定されています。これは多くの旅行者にとって最も一般的な設定です。  
簡単に変更でき、複数の部屋や複数の宿泊者（大人・子供）を追加可能です。

### プロモーション

プロモーションはユーザーの時間を節約する便利な機能です。  
施設から提供されたプロモコードがあればここに入力できます。  
あなたのリンク経由で来たユーザーは自動的にプロモコードが旅程に追加され、割引価格が自動適用されます。

## 統合

`Integration` タブは、ホテル、インテグレーター、旅行代理店がWinkでの予約の配信方法をより細かく制御したい場合に使用します。

### 通知

ホテルやインテグレーターに、予約が発生した際に関係者全員に通知するオプションを提供します。  
チャネルマネージャーへの通知も含まれます。

デフォルトではこれらのオプションはすべて有効です。無効にしたいケースは以下の通りです。

- 旅行代理店がAPIレベルで統合されており、自社で確認メールを送信したい場合。
- ホテルが統合テスト中で営業スタッフに迷惑をかけたくない場合。
- ホテルがメール通知を気にせず、PMS経由のみで通知を受けたい場合。

### メールロゴ

インテグレーターが当社のメール通知サービスを利用する場合、メールの見た目を自社ブランドにカスタマイズできます。

:::note
  推奨サイズは幅300px、高さ75pxです
:::

## ホテル固有

`Hotel-specific` タブでは、ホテルが当社の予約エンジンのランディングページに表示する内容や、指定された旅程に在庫がない場合に表示する内容を制御できます。

- **ランキング** 意味のあるランキングをまだ獲得していない施設は、この機能を無効にすることができます。

:::note
  施設はアフィリエイトがホテルランディングページをどのように表示するかを制御できません。これらは別の設定です。
:::

- **空室状況** 在庫がない場合でも在庫を表示し続けるかどうかを施設が選択できます。

:::note
  多くの場合、施設はユーザーが `Book` ボタンをクリックしてランディングページに進み、旅程を変更して空室を探せるようにしたいと考えます。  
  ユーザーの継続をブロックしたい場合はこのスイッチを有効にしてください。
:::

## 使い方

ほとんどの場合、リンク、カード、マップ、グリッドに適用する `Customization` を選択します。

手動で設定を適用したい場合は以下のようなケースがあります。

- **手動オーバーライド** カスタム設定を適用した状態で、[共有可能リンク](/studio/shareable-links)機能を使わずに予約エンジンへのリンクを直接共有したい場合。
- **WordPressプラグイン** Winkの[WordPressプラグイン](/developers/wordpress)の `設定` に設定を追加する場合。
- **カスタム統合** 当社の [wink-app-loader](/developers/web-components/#application-loader) Web Componentを使ってカスタムインテグレーターに手動で適用する場合。

### 手動オーバーライド

以下は、設定ID（すべての設定を含む）を当社のメイン予約エンジンURLに適用する簡単な例です。

<h4 align="center">[https://ota.wink.travel?configurationId=MY-UNIQUE-ID](https://ota.wink.travel?configurationId=MY-UNIQUE-ID)</h4>

:::note
  ユーザーがこのリンクをクリックすると、アフィリエイトのアトリビューションクッキーも設定されます。
:::

### WordPressプラグイン

詳細は[Wink WordPressプラグイン](/developers/wordpress)をご覧ください。

### カスタム統合

ページ上のすべての埋め込みWink Web Componentsを [wink-app-loader](/developers/web-components/#application-loader) Web Componentで制御します。

以下は設定IDに `MY-UNIQUE-ID` を使ったサンプルです。

```html {11}
<html>
  <head>
    <link rel="stylesheet" href="https://elements.wink.travel/styles.css"></link>
  </head>
  <body>

    <script src="https://elements.wink.travel/elements.js" type="module" defer></script>

    <wink-app-loader
      client-id="YOUR CLIENT ID GOES HERE"
      configuration-id="MY-UNIQUE-ID"
    ></wink-app-loader>
  </body>
</html>
```
