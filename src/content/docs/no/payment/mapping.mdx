Hvis du er en integrator, som Wink, og ønsker å kartlegge dine eksterne betalingsmottakere for bruk med TripPay, kan du gjøre det på to måter:

- Manuell kartlegging
- Bruke [Payment API](/no/developers/apis/#payment-api) (*Se `Account Mappings`*)

:::tip
Med mindre du bare håndterer et fåtall av de samme eiendommene, anbefaler vi at du bruker API-en for å automatisere dette steget.
:::

Nedenfor viser vi deg den manuelle måten å kartlegge en ekstern betalingsmottaker på. *Den bruker API-en i bakgrunnen.*

<Steps>
1. Logg inn på [TripPay](https://pay.wink.travel).
2. Velg din integratorkonto.
3. Klikk `Settings > Mappings` i hovednavigasjonsmenyen.
4. Klikk på `Create mapping`-knappen. Du vil se kartleggingsskjemaet.
5. **Local account** Begynn å skrive navnet på den eksisterende TripPay-kontoen du vil kartlegge.
6. **External identifier** Lim inn din eksterne ID for denne enheten slik den finnes i det eksterne systemet.
7. Klikk `Save` for å fortsette. Du blir sendt tilbake til listen over kartlegginger.
</Steps>

Med kartleggingene på plass vet TripPay nå hvilke kontoer som skal krediteres som mottakere av en booking når du refererer til dem med dine eksterne identifikatorer.

## API

Utviklere som ønsker å administrere `Payments`, går til [Developers > API > Payments](/no/developers/apis/#payment-api).
