Ikke alle dager har samme pris. Arrangementer, høytider, lavsesong og siste-liten ledige rom krever ulike priser. Denne guiden viser deg hvordan du bruker spesialpriser og restriksjoner på prisplaner for å håndtere disse situasjonene.

## Pris for arrangementer

Når en konferanse, festival eller lokalt arrangement øker etterspørselen, vil du ønske å øke prisene for disse datoene.

<Steps>
1. Gå til `Monetize > Special Rates` i ditt Extranet.
2. Klikk `Create a new special rate`.
3. Velg masterprisen og romtypen som skal overstyres.
4. Sett datoperioden som dekker arrangementet.
5. Skriv inn de justerte (høyere) dagsprisene.
6. Klikk `Save`.
</Steps>

Spesialprisen overstyrer masterprisen kun for disse datoene. Din vanlige prising forblir uendret utenfor arrangementperioden.

## Sesongjusteringer

For lengre perioder (høysesong, skuldersesong, lavsesong) kan du enten:

- **Bruke spesialpriser** — Opprette overstyringer for hver sesong med justerte priser.
- **Oppdatere masterpriser** — Justere basisprisene direkte for disse periodene.

Spesialpriser er best for midlertidige justeringer, mens oppdatering av masterpriser gir mening for forutsigbare årlige mønstre.

## Siste-liten priser

For å fylle usolgte rom nær innsjekkingsdatoen:

<Steps>
1. Opprett en ny **prisplan** med en restriksjon for forhåndsbestilling (f.eks. må bookes senest 3 dager i forveien).
2. Koble den til en **spesialpris** som gir rabatt for de aktuelle datoene.
3. Denne prisplanen vil kun vises for reisende som booker innenfor siste-liten-vinduet.
</Steps>

## Svarteliste-datoer

For å blokkere visse datoer fra en spesifikk prisplan (f.eks. ingen kampanjepriser i høysesongen):

<Steps>
1. Gå til `Monetize > Rate Plans` og velg prisplanen.
2. Naviger til **Restrictions**-seksjonen.
3. Legg til en **Sell date**-restriksjon for å begrense når prisen kan bookes.
4. Eller bruk **Stay date**-restriksjoner for å begrense hvilke datoer prisen gjelder for.
</Steps>

For eksempel kan du begrense "Early Bird"-prisplanen slik at den ikke er tilgjengelig for booking i desember (høysesong), mens din vanlige prisplan forblir åpen.

## Kombinere restriksjoner

Restriksjoner kan kombineres for presis kontroll:

| Restriksjon | Hva den kontrollerer |
| -- | -- |
| **Sell dates** | Når prisen kan bookes |
| **Stay dates** | Hvilke oppholdsdatoer prisen gjelder for |
| **Available days** | Hvilke ukedager prisen vises |
| **Arrival days** | Hvilke dager gjesten må ankomme på |
| **Departure days** | Hvilke dager gjesten må reise fra |
| **Required days** | Gjesten må oppholde seg minst én av disse dagene |
| **Advance booking** | Hvor langt i forveien bookingen må gjøres |
| **Length of stay** | Minimum og maksimum antall netter |
