## Oversikt
Den hostede Wink MCP-serveren er tilgjengelig på:

```
https://docs.mcp.wink.travel/mcp
```

Pek kompatible AI-klienter (VS Code, Cursor, andre som implementerer MCP HTTP-transport) mot dette endepunktet for å la dem:
- Bla i produkt- og plattformsdokumentasjon (`wink://docs/...`)
- Bla i live OpenAPI-spesifikasjoner og operasjonsdokumentasjon (`wink://api/...`)
- Utføre lettvekts semantisk / nøkkelordssøk over dokumentasjon og live API-er

## Transport
Bruk det primære strømmende HTTP-endepunktet:

```
https://docs.mcp.wink.travel/mcp
```

Hvis klienten din kun støtter SSE, bruk:

```
https://docs.mcp.wink.travel/sse
```

### Claude Code
Offisiell dokumentasjon: [Koble Claude Code til verktøy via MCP](https://code.claude.com/docs/en/mcp)

Anthropic anbefaler fjern-HTTP-transport for skyhostede MCP-servere:
```bash
claude mcp add --transport http wink-docs https://docs.mcp.wink.travel/mcp
claude mcp list
```

I Claude Code, kjør `/mcp` for å inspisere serverstatus. Hvis du vil dele serveren med hele repoet, legg til `--scope project` slik at Claude Code skriver konfigurasjonen til `.mcp.json`.

### Codex
Offisiell dokumentasjon: [Docs MCP quickstart for Codex](https://developers.openai.com/resources/docs-mcp)

OpenAI dokumenterer MCP-oppsett gjennom Codex CLI, og nevner at konfigurasjonen deles med IDE-utvidelsen:
```bash
codex mcp add wink-docs --url https://docs.mcp.wink.travel/mcp
codex mcp list
```

### Cursor
Cursor MCP-dokumentasjon: https://cursor.com/docs/context/mcp
Oppsett: Legg til fjernserver via MCP-innstillinger.
```json
{
  "mcpServers": {
    "wink-docs": {
      "url": "https://docs.mcp.wink.travel/mcp",
    }
  }
}
```

### VS Code
Offisiell dokumentasjon: https://code.visualstudio.com/docs/copilot/customization/mcp-servers
Minimal fjernkonfigurasjon (bruker- eller arbeidsområde-`mcp.json`):
```json
{
  "servers": {
    "wink-docs": {
      "type": "http",
      "url": "https://docs.mcp.wink.travel/mcp"
    }
  }
}
```

Tips:
- Bruk Command Palette: “MCP: List Servers” for å bekrefte tilkobling og bla i verktøy.
- Verktøysvelger lar deg aktivere/deaktivere `api_search` og `docs_search`.
- Legg til kontekst: Chat-visning → Legg til kontekst → MCP Resources for å knytte dokumentasjonssider.

## Kjerne-URI-er for ressurser

| URI | Beskrivelse |
|-----|-------------|
| `wink://docs` | Dokumentasjonsindeks (gruppert etter kategori) |
| `wink://docs/{path}` | Spesifikk dokumentasjonsartikkel (Markdown / MDX) |
| `wink://api` | Live OpenAPI-katalog oppdaget fra Winks swagger-konfigurasjon |
| `wink://api/{apiName}` | Kompakt API-oversikt med autentisering, tagger, headere og eksempeloperasjoner |
| `wink://api/{apiName}/operations` | Full operasjonsliste gruppert etter tagg |
| `wink://api/{apiName}/operation/{key}` | Detaljer for individuell operasjon (parametere, forespørselskropp, svar) |
| `wink://api/{apiName}/schema/{schemaName}` | Løst komponent-skjema med nestede felt, typer og eksempler |
| `wink://api/{apiName}/operation/{key}/request` | Forespørselsfokusert visning med nødvendige input og minimal eksempelpayload |
| `wink://api/{apiName}/operation/{key}/response/{status}` | Svarfokusert visning for en spesifikk statuskode |
| `wink://api/{apiName}/operation/{key}/recipe` | Oppgaveorientert flyt for å kalle den operasjonen korrekt |

## Søkeverktøy (Rask referanse)
| Verktøy | Formål | Typiske spørringseksempler |
|---------|--------|----------------------------|
| `api_search` | Finn live API eller operasjon å integrere mot | `create affiliate`, `payment receipt`, `lookup inventory`, `invite manager` |
| `docs_search` | Finn konseptuelle dokumenter / guider | `authentication`, `webhooks`, `wordPress` |

La `query` være tom for å motta forslag. Hvert resultat inkluderer MCP-URI som du kan åpne.
For treff på operasjoner returnerer `api_search` også tilknyttede forespørsels-, svar-, skjema- og oppskrifts-URI-er slik at agenter kan hoppe direkte til den mest nyttige delressursen.

## Typisk bruksområde
1. Kjør `api_search` med din intensjon (f.eks. *"create affiliate"* eller *"payment receipt"*).
2. Åpne den returnerte `wink://api/...` URI-en, eller følg dens tilknyttede `request_uri`, `response_uris`, `request_schema_uris`, `response_schema_uris` og `recipe_uri` felt.
3. Bruk `docs_search` for støttende konsepter (autentisering, webhooks, arbeidsflyter).

## Notater
- Hver returnert URI kan åpnes av agenten for fullstendig innhold.
- `wink://api`-treet er live og følger den nåværende upstream OpenAPI-katalogen i stedet for en generert repo-indeks.
- Forslag vises automatisk når spørringer er tomme.
