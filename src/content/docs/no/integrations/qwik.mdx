Qwiks gjenopptakelsesmodell betyr at JavaScript kjører late — kun når brukeren interagerer. Bruk `useVisibleTask$` for å kjøre kode som må utføres i nettleseren når komponenten blir synlig.

/>

## Installer

```bash
npm install @wink/elements
```

## Last inn med useVisibleTask$

`useVisibleTask$` er Qwiks livssyklus-hook for nettleser-kode som skal kjøre tidlig. Plasser den i ditt rotoppsett:

```tsx
// src/routes/layout.tsx
import { component$, useVisibleTask$ } from '@builder.io/qwik';
import { load } from '@wink/elements';

export default component$(() => {
  useVisibleTask$(() => {
    load({ clientId: import.meta.env.PUBLIC_WINK_CLIENT_ID });
  });

  return (
    <main>
      <Slot />
    </main>
  );
});
```

<Aside type="tip" title="useVisibleTask$-strategi">
Som standard kjører `useVisibleTask$` når komponenten er synlig i viewporten. For et rotoppsett som alltid er synlig, skjer dette umiddelbart ved sideinnlasting — tilsvarende `onMount` i andre rammeverk.

Send med `{ strategy: 'document-ready' }` hvis du vil at skriptet skal lastes så tidlig som mulig:
```ts
useVisibleTask$({ strategy: 'document-ready' }, () => {
  load({ clientId: import.meta.env.PUBLIC_WINK_CLIENT_ID });
});
```
</Aside>

## Bruk på en side

```tsx
// src/routes/hotels/index.tsx
import { component$ } from '@builder.io/qwik';

export default component$(() => {
  return (
    <main>
      <wink-content-loader layout="HOTEL" id="YOUR_LAYOUT_ID" />
      <wink-lookup />
    </main>
  );
});
```

## TypeScript — deklarer JSX intrinsics

Qwik bruker sitt eget JSX-navnerom. Utvid det i en deklarasjonsfil:

```ts
// src/wink-elements.d.ts
import type {
  WinkContentLoaderAttributes,
  WinkLookupAttributes,
  WinkSearchButtonAttributes,
  WinkAccountButtonAttributes,
  WinkItineraryButtonAttributes,
  WinkShoppingCartButtonAttributes,
  WinkAppLoaderAttributes,
} from '@wink/elements';

declare module '@builder.io/qwik' {
  namespace JSX {
    interface IntrinsicElements {
      'wink-content-loader': WinkContentLoaderAttributes;
      'wink-lookup': WinkLookupAttributes;
      'wink-search-button': WinkSearchButtonAttributes;
      'wink-account-button': WinkAccountButtonAttributes;
      'wink-itinerary-button': WinkItineraryButtonAttributes;
      'wink-shopping-cart-button': WinkShoppingCartButtonAttributes;
      'wink-app-loader': WinkAppLoaderAttributes;
    }
  }
}
```

## Miljøvariabler

Qwik (basert på Vite) eksponerer variabler med prefikset `PUBLIC_` til nettleseren:

```
# .env
PUBLIC_WINK_CLIENT_ID=your-client-id
```

<Aside type="caution" title="Server-side rendering">
Qwik renderer til HTML på serveren. `<wink-*>`-taggene blir skrevet ut som statisk markup. CDN-skriptet og registreringen av egendefinerte elementer skjer på klientsiden etter at `useVisibleTask$` har kjørt. Elementene vil være i en uavklart tilstand en kort stund til CDN-pakken er lastet — dette er forventet oppførsel.
</Aside>
