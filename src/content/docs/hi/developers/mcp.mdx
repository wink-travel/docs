## अवलोकन
होस्ट किए गए Wink MCP सर्वर इस पर उपलब्ध है:

```
https://docs.mcp.wink.travel/mcp
```

संगत AI क्लाइंट्स (VS Code, Cursor, अन्य जो MCP HTTP ट्रांसपोर्ट लागू करते हैं) को इस एंडपॉइंट पर पॉइंट करें ताकि वे:
- उत्पाद और प्लेटफ़ॉर्म दस्तावेज़ ब्राउज़ कर सकें (`wink://docs/...`)
- लाइव OpenAPI स्पेक्स और ऑपरेशन दस्तावेज़ ब्राउज़ कर सकें (`wink://api/...`)
- दस्तावेज़ और लाइव APIs पर हल्का सेमांटिक / कीवर्ड खोज कर सकें

## ट्रांसपोर्ट
प्राथमिक स्ट्रीम करने योग्य HTTP एंडपॉइंट का उपयोग करें:

```
https://docs.mcp.wink.travel/mcp
```

यदि आपका क्लाइंट केवल SSE का समर्थन करता है, तो उपयोग करें:

```
https://docs.mcp.wink.travel/sse
```

### Claude Code
आधिकारिक दस्तावेज़: [MCP के माध्यम से Claude Code को टूल्स से कनेक्ट करें](https://code.claude.com/docs/en/mcp)

Anthropic क्लाउड-होस्टेड MCP सर्वरों के लिए रिमोट HTTP ट्रांसपोर्ट की सलाह देता है:
```bash
claude mcp add --transport http wink-docs https://docs.mcp.wink.travel/mcp
claude mcp list
```

Claude Code के अंदर, सर्वर स्थिति जांचने के लिए `/mcp` चलाएं। यदि आप पूरे रिपो के साथ सर्वर साझा करना चाहते हैं, तो `--scope project` जोड़ें ताकि Claude Code कॉन्फ़िग को `.mcp.json` में लिखे।

### Codex
आधिकारिक दस्तावेज़: [Codex के लिए Docs MCP क्विकस्टार्ट](https://developers.openai.com/resources/docs-mcp)

OpenAI MCP सेटअप को Codex CLI के माध्यम से दस्तावेज़ करता है, और नोट करता है कि कॉन्फ़िग IDE एक्सटेंशन के साथ साझा की जाती है:
```bash
codex mcp add wink-docs --url https://docs.mcp.wink.travel/mcp
codex mcp list
```

### Cursor
Cursor MCP दस्तावेज़: https://cursor.com/docs/context/mcp
सेटअप: MCP सेटिंग्स के माध्यम से रिमोट सर्वर जोड़ें।
```json
{
  "mcpServers": {
    "wink-docs": {
      "url": "https://docs.mcp.wink.travel/mcp",
    }
  }
}
```

### VS Code
आधिकारिक दस्तावेज़: https://code.visualstudio.com/docs/copilot/customization/mcp-servers
न्यूनतम रिमोट कॉन्फ़िग (यूजर या वर्कस्पेस `mcp.json`):
```json
{
  "servers": {
    "wink-docs": {
      "type": "http",
      "url": "https://docs.mcp.wink.travel/mcp"
    }
  }
}
```

टिप्स:
- कनेक्शन की पुष्टि और टूल्स ब्राउज़ करने के लिए कमांड पैलेट में “MCP: List Servers” का उपयोग करें।
- टूल्स पिकर आपको `api_search` और `docs_search` को सक्षम/अक्षम करने देता है।
- संदर्भ जोड़ें: चैट व्यू → Add Context → MCP Resources से दस्तावेज़ पेज संलग्न करें।

## कोर रिसोर्स URI

| URI | विवरण |
|-----|-------------|
| `wink://docs` | दस्तावेज़ सूचकांक (श्रेणी द्वारा समूहित) |
| `wink://docs/{path}` | विशिष्ट दस्तावेज़ लेख (Markdown / MDX) |
| `wink://api` | Wink के swagger कॉन्फ़िग से खोजा गया लाइव OpenAPI कैटलॉग |
| `wink://api/{apiName}` | प्रमाणीकरण, टैग, हेडर, और नमूना ऑपरेशंस के साथ कॉम्पैक्ट API सारांश |
| `wink://api/{apiName}/operations` | टैग द्वारा समूहित पूर्ण ऑपरेशन सूची |
| `wink://api/{apiName}/operation/{key}` | व्यक्तिगत ऑपरेशन विवरण (पैरामीटर, अनुरोध बॉडी, प्रतिक्रियाएँ) |
| `wink://api/{apiName}/schema/{schemaName}` | नेस्टेड फ़ील्ड, प्रकार, और उदाहरणों के साथ हल किया गया कंपोनेंट स्कीमा |
| `wink://api/{apiName}/operation/{key}/request` | आवश्यक इनपुट और न्यूनतम उदाहरण पेलोड के साथ अनुरोध-केंद्रित दृश्य |
| `wink://api/{apiName}/operation/{key}/response/{status}` | विशिष्ट स्थिति कोड के लिए प्रतिक्रिया-केंद्रित दृश्य |
| `wink://api/{apiName}/operation/{key}/recipe` | उस ऑपरेशन को सही ढंग से कॉल करने के लिए कार्य-उन्मुख फ्लो |

## खोज टूल (त्वरित संदर्भ)
| टूल | उद्देश्य | सामान्य क्वेरी उदाहरण |
|------|---------|-----------------------|
| `api_search` | एकीकृत करने के लिए लाइव API या ऑपरेशन खोजें | `create affiliate`, `payment receipt`, `lookup inventory`, `invite manager` |
| `docs_search` | वैचारिक दस्तावेज़ / गाइड खोजें | `authentication`, `webhooks`, `wordPress` |

सुझाव प्राप्त करने के लिए `query` खाली छोड़ें। हर परिणाम में MCP URI शामिल होता है जिसे आप खोल सकते हैं।
ऑपरेशन हिट्स के लिए, `api_search` लिंक किए गए अनुरोध, प्रतिक्रिया, स्कीमा, और रेसिपी URI भी लौटाता है ताकि एजेंट सबसे उपयोगी सबरिसोर्स में सीधे जा सकें।

## सामान्य उपयोग पैटर्न
1. अपनी मंशा के साथ `api_search` चलाएं (जैसे *"create affiliate"* या *"payment receipt"*)।
2. लौटाए गए `wink://api/...` URI खोलें, या इसके लिंक किए गए `request_uri`, `response_uris`, `request_schema_uris`, `response_schema_uris`, और `recipe_uri` फ़ील्ड का पालन करें।
3. सहायक अवधारणाओं (प्रमाणीकरण, वेबहुक्स, वर्कफ़्लोज़) के लिए `docs_search` का उपयोग करें।

## नोट्स
- प्रत्येक लौटाए गए URI को एजेंट द्वारा पूर्ण सामग्री के लिए खोला जा सकता है।
- `wink://api` ट्री लाइव है और वर्तमान अपस्ट्रीम OpenAPI कैटलॉग का अनुसरण करता है, न कि जेनरेटेड रिपो सूचकांक।
- जब क्वेरी खाली होती है तो सुझाव स्वचालित रूप से दिखाई देते हैं।
