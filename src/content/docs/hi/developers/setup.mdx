यह लेख आपको बताता है कि कैसे एक डेवलपर के रूप में जल्दी से सेटअप किया जाए और विंक पर हमारे एपीआई का परीक्षण शुरू किया जाए।

:::नोट
परीक्षण हमेशा हमारे यहां होना चाहिए[मंचन वातावरण](/getting-started/environments)इसमें बहुत कम डेटा है लेकिन आप उत्पादन वातावरण पर दबाव डाले बिना जितना चाहें उतना परीक्षण करने के लिए स्वतंत्र हैं।

एक बार जब आप लाइव होने के लिए तैयार हो जाते हैं, तो हम आपके खाते और आपके आवेदन को प्रमाणित करेंगे और आप हमारे उत्पादन वातावरण में लॉन्च कर सकते हैं।

इन चरणों का पालन करें:

<Steps>
  1. [पंजीकरण करवाना](/user/register) विंक पर आपका उपयोगकर्ता खाता
  2. एक बनाने के [ट्रैवल एजेंट खाता](/travel-agent/what-is-agent) या एक [सहबद्ध खाता](/studio/what-is-studio).
  3. [एक एप्लिकेशन बनाएं](/developer/applications) जहां आप इसे चरण 2 में बनाए गए खाते से संबद्ध करते हैं।
  4. एक बार जब आप अपना बना लेते हैं `Application`, क्लाइंट-आईडी और क्लाइंट सीक्रेट को सुरक्षित स्थान पर सहेजें।
</Steps>


## सीडीएन

विंक में, हम अपनी सभी छवि और वीडियो संपत्तियों के लिए क्लाउडिनरी का उपयोग करते हैं। जब आप हमारी यात्रा सूची तक पहुँचते हैं, तो आपको क्लाउडिनरी की सभी उन्नत सुविधाओं तक पूरी पहुँच प्राप्त होगी।

हमारा `cloud_name` है  `traveliko`.

### क्लाउडिनरी

* [शुरू करना](https://cloudinary.com/documentation/dev_kickstart)
* [एसडीके](https://cloudinary.com/documentation/cloudinary_sdks)
* [GitHub पर Cloudinary](https://github.com/cloudinary)
* [वीडियो ट्यूटोरियल](https://cloudinary.com/documentation/cloudinary_sdks_tutorials)

### उदाहरण

हमारे द्वारा समर्थित प्रत्येक यात्रा सूची प्रकार के लिए, एक नेस्टेड JSON सरणी होगी जिसे कहा जाता है`multimedias`मल्टीमीडिया प्रविष्टि के लिए JSON इस तरह दिखता है:

```json {3}
{
  "multimediaIdentifier": "multimedia-1",
  "identifier": "partners/radisson_blu_logo",
  "type": "IMAGE",
  "width": "1024",
  "height": "768",
  "category": "1",
  "descriptions": [
    {
      "name": "Lobby",
      "description": "Picture of the lobby",
      "language": "en"
    }
  ],
  "lifestyleType": "LIFESTYLE_BUSINESS",
  "attribution": [
    {
      "url": "https://www.hilton.com",
      "name": "Hilton"
    }
  ]
}
```

ऊपर हाइलाइट की गई लाइन क्लाउडिनरी की विशिष्ट आईडी दिखाती है`partners/radisson_blu_logo`. आपको अपनी इच्छानुसार किसी भी प्रारूप, गुणवत्ता और आकार में छवि प्राप्त करने के लिए बस इतना ही करना होगा।

## परीक्षण

हमारे पेज पर जाएँ[शहद की मक्खी](/developer/apis)अनुभाग पर जाएं और वह API चुनें जिसके साथ आप काम करना चाहते हैं.

### एपीआई दस्तावेज़

हमारी दस्तावेज़ीकरण साइट इस तरह से स्थापित की गई है कि आप साइट के भीतर से API कमांड चला सकते हैं।

इस उदाहरण में, आप उपयोग करना चाहते हैं`Search Lookups (OAuth2)`एपीआई कॉल.

:::टिप\[प्रो टिप]
उपयोग में आसानी के लिए, हम अनुशंसा करते हैं कि इंटीग्रेटर हमारे द्वारा निर्दिष्ट एंडपॉइंट का उपयोग करें`OAuth2`वे एंडपॉइंट एपीआई कॉलर्स की अपेक्षा करते हैं और पहले से ही जानते हैं कि आप किस खाते पर हैं, इसलिए आपको निर्दिष्ट करने की आवश्यकता नहीं है`companyIdentifier`अपने यूआरएल में.
:::

<Steps>
  1. उपयोग [लुकअप एपीआई](https://docs.wink.travel/lookup) गंतव्य या होटल पुनः प्राप्त करने के लिए.
  2. बाएं नेविगेशन बार में, पर क्लिक करें `Authentication`.
  3. अपना क्लाइंट-आईडी और क्लाइंट सीक्रेट दर्ज करें और क्लिक करें `Get TOKEN` बटन।
  4. यदि आपके क्रेडेंशियल वैध हैं, तो आप देखेंगे कि आपकी API कुंजी लागू कर दी गई है।
  5. अब आप लुकअप एंडपॉइंट का परीक्षण कर सकते हैं.
</Steps>

### डाकिया

आप पोस्टमैन के साथ भी यही काम कर सकते हैं।

<Steps>
  1. डाउनलोड करना [डाकिया](https://www.postman.com/downloads).
  2. पर क्लिक करें `Download OpenAPI spec` के ऊपर से [लुकअप एपीआई](https://docs.wink.travel/lookup) पृष्ठ.
  3. इसे पोस्टमैन में आयात करें.
  4. यदि आपने इसे एक नए संग्रह के रूप में आयात किया है `Wink API`, संग्रह नाम के आगे क्षैतिज एलिप्सिस (3 बिंदु) पर क्लिक करें और क्लिक करें `Edit`.
  5. में `Authorization tab`, सेट करें `Auth type` OAuth2 के लिए.
  6. फ़ील्ड सेट करें `Add auth data to` को `Request headers`.
  7. फ़ील्ड सेट करें `Header prefix` को `Bearer`.
  8. फ़ील्ड सेट करें `Token Name` आप पोस्टमैन में अपने सहेजे गए टोकन को जो भी नाम देना चाहें, उसे चुनें। जैसे: विंक स्टेजिंग।
  9. फ़ील्ड सेट करें `Grant type` को `Client Credentials`.
  10. फ़ील्ड सेट करें `Access token URL` https://staging-iam.wink.travel/oauth2/token पर जाएं.
  11. फ़ील्ड सेट करें `Client ID` आपके ऐप द्वारा जेनरेट की गई क्लाइंट आईडी पर.
  12. फ़ील्ड सेट करें `Client Secret` आपके ऐप द्वारा जेनरेट किए गए क्लाइंट सीक्रेट को.
  13. फ़ील्ड सेट करें `Scope` को `inventory.read inventory.write inventory.remove`ये सभी स्कोप हैं जिनकी आपको विंक के लिए आवश्यकता होगी।
  14. फ़ील्ड सेट करें `Client Authentication` को `Send as Basic Auth header`.
  15. क्लिक करें `Get New Access Token` अपना टोकन पुनः प्राप्त करने और उसे सहेजने के लिए बटन दबाएं।
  16. विंक API संग्रह के अंतर्गत सभी API कॉल के लिए, सुनिश्चित करें कि प्रमाणीकरण टैब कहता है `Inherit auth from parent` इसलिए इसमें हर कॉल के साथ आपका टोकन शामिल होता है।
</Steps>


:::टिप\[प्रो टिप]
सर्वर डोमेन, क्लाइंट आईडी और क्लाइंट सीक्रेट को एनवायरनमेंट वैरिएबल में बदलने की सलाह दी जाती है। इस तरह आप स्टेजिंग और प्रोडक्शन एनवायरनमेंट के बीच आसानी से स्विच कर सकते हैं और अपने द्वारा बनाए गए सभी ऑपरेशन का फिर से उपयोग कर सकते हैं।
:::
