यह लेख आपको बताता है कि एक डेवलपर के रूप में जल्दी से कैसे सेटअप करें और Wink पर हमारे APIs का परीक्षण कैसे शुरू करें।

:::note
परीक्षण हमेशा हमारे [staging environment](/hi/getting-started/environments) में होना चाहिए। वहाँ डेटा बहुत कम होता है लेकिन आप बिना प्रोडक्शन वातावरण पर प्रभाव डाले जितना चाहें परीक्षण कर सकते हैं।

जब आप लाइव जाने के लिए तैयार हों, तो हम आपके खाते और आपके एप्लिकेशन को प्रमाणित करेंगे और आप हमारे प्रोडक्शन वातावरण में लॉन्च कर सकते हैं।
:::

इन चरणों का पालन करें:

<Steps>
1. Wink पर अपना उपयोगकर्ता खाता [Register](/hi/my-account/register) करें।
2. [Create an affiliate account](/hi/my-account/create-account)। आपको इसे अपने Application से जोड़ने की आवश्यकता होगी। अधिक संदर्भ के लिए देखें [what is Studio](/hi/studio/what-is-studio)।
3. [Create an Application](/hi/my-account/applications) करें और इसे चरण 2 में बनाए गए affiliate खाते से जोड़ें। सर्वर-टू-सर्वर API इंटीग्रेशन के लिए, क्लाइंट प्रकार के रूप में **MACHINE_2_MACHINE** चुनें।
4. एक बार जब आपने अपना `Application` बना लिया, तो तुरंत client-id और secret key कॉपी करके सुरक्षित कर लें — secret key केवल एक बार दिखाई जाती है और इसे फिर से प्राप्त नहीं किया जा सकता।
</Steps>

## CDN

Wink में, हम अपने सभी छवि और वीडियो संसाधनों के लिए Cloudinary का उपयोग करते हैं। जब आप हमारे ट्रैवल इन्वेंट्री तक पहुँचते हैं, तो आपको Cloudinary की सभी उन्नत सुविधाओं तक पूर्ण पहुंच प्राप्त होगी।

हमारा `cloud_name` है `traveliko`।

### Cloudinary

- [शुरुआत कैसे करें](https://cloudinary.com/documentation/dev_kickstart)
- [SDKs](https://cloudinary.com/documentation/cloudinary_sdks)
- [GitHub पर Cloudinary](https://github.com/cloudinary)
-	[वीडियो ट्यूटोरियल्स](https://cloudinary.com/documentation/cloudinary_sdks_tutorials)

### उदाहरण

हम जिन प्रत्येक ट्रैवल इन्वेंट्री प्रकार का समर्थन करते हैं, उसके लिए एक नेस्टेड JSON एरे होगा जिसे `multimedias` कहा जाता है। एक मल्टीमीडिया प्रविष्टि के लिए JSON इस प्रकार दिखता है:

```json {3}
{
  "multimediaIdentifier": "multimedia-1",
  "identifier": "partners/radisson_blu_logo",
  "type": "IMAGE",
  "width": "1024",
  "height": "768",
  "category": "1",
  "descriptions": [
    {
      "name": "Lobby",
      "description": "Picture of the lobby",
      "language": "en"
    }
  ],
  "lifestyleType": "LIFESTYLE_BUSINESS",
  "attribution": [
    {
      "url": "https://www.hilton.com",
      "name": "Hilton"
    }
  ]
}
```

ऊपर हाइलाइट की गई पंक्ति Cloudinary का यूनिक ID `partners/radisson_blu_logo` दिखाती है। यही वह सब है जो आपको किसी भी फॉर्मेट, गुणवत्ता और आकार में छवि पुनः प्राप्त करने के लिए चाहिए।

## परीक्षण

हमारे [APIs](/hi/developers/apis) सेक्शन पर जाएं और उस API को चुनें जिसके साथ आप काम करना चाहते हैं।

### API दस्तावेज़

हमारी दस्तावेज़ साइट इस तरह से सेटअप है कि आप साइट के भीतर से API कमांड चला सकते हैं।

इस उदाहरण में, आप `Search Lookups (OAuth2)` API कॉल का उपयोग करना चाहते हैं।

:::tip[Pro Tip]
सुविधा के लिए, हम अनुशंसा करते हैं कि इंटीग्रेटर्स उन endpoints का उपयोग करें जिन्हें हम `OAuth2` के रूप में निर्दिष्ट करते हैं। वे endpoints API कॉलर्स की उम्मीद करते हैं और पहले से जानते हैं कि आप किस खाते पर हैं, इसलिए आपको अपने URLs में `companyIdentifier` निर्दिष्ट करने की आवश्यकता नहीं है।
:::

<Steps>
1. गंतव्य या होटलों को पुनः प्राप्त करने के लिए [Lookup API](https://developer.wink.travel/lookup) का उपयोग करें।
2. बाएं नेवबार में, `Authentication` पर क्लिक करें।
3. अपना client-id और client secret दर्ज करें और `Get TOKEN` बटन पर क्लिक करें।
4. यदि आपकी प्रमाण-पत्र मान्य हैं, तो आप देखेंगे कि आपका API key लागू हो गया है।
5. अब आप lookup endpoint का परीक्षण कर सकते हैं।
</Steps>

### Postman
आप Postman के साथ भी यही कर सकते हैं।

<Steps>
1. [Postman](https://www.postman.com/downloads) डाउनलोड करें।
2. [Lookup API](https://developer.wink.travel/lookup) पेज के शीर्ष से `Download OpenAPI spec` पर क्लिक करें।
3. इसे Postman में आयात करें।
4. यदि आपने इसे `Wink API` नामक एक नए संग्रह के रूप में आयात किया है, तो संग्रह नाम के बगल में क्षैतिज एलिप्सिस (3 डॉट) पर क्लिक करें और `Edit` पर क्लिक करें।
5. `Authorization` टैब में, `Auth type` को OAuth2 पर सेट करें।
6. `Add auth data to` फ़ील्ड को `Request headers` पर सेट करें।
7. `Header prefix` फ़ील्ड को `Bearer` पर सेट करें।
8. `Token Name` फ़ील्ड को उस नाम पर सेट करें जिसे आप Postman में अपने सहेजे गए टोकन के लिए रखना चाहते हैं। जैसे Wink Staging।
9. `Grant type` फ़ील्ड को `Client Credentials` पर सेट करें।
10. `Access token URL` फ़ील्ड को https://staging-iam.wink.travel/oauth2/token पर सेट करें।
11. `Client ID` फ़ील्ड को उस Client ID पर सेट करें जिसे आपके ऐप ने उत्पन्न किया है।
12. `Client Secret` फ़ील्ड को उस Client Secret पर सेट करें जिसे आपके ऐप ने उत्पन्न किया है।
13. `Scope` फ़ील्ड को `inventory.read inventory.write inventory.remove` पर सेट करें। ये सभी स्कोप्स हैं जो आपको Wink के लिए चाहिए होंगे।
14. `Client Authentication` फ़ील्ड को `Send as Basic Auth header` पर सेट करें।
15. अपने टोकन को प्राप्त करने और सहेजने के लिए `Get New Access Token` बटन पर क्लिक करें।
16. Wink API संग्रह के तहत सभी API कॉल्स के लिए, सुनिश्चित करें कि authentication टैब कहता है `Inherit auth from parent` ताकि यह हर कॉल के साथ आपका टोकन शामिल करे।
</Steps>

:::tip[Pro Tip]
यह अनुशंसा की जाती है कि सर्वर डोमेन, client ID और client secret को पर्यावरण चर (environment variables) में बदल दिया जाए। इस तरह आप आसानी से staging और production वातावरण के बीच स्विच कर सकते हैं और आपने जो सभी ऑपरेशंस बनाए हैं उन्हें पुनः उपयोग कर सकते हैं।
:::
