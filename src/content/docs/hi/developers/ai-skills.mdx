AI कौशल पोर्टेबल निर्देश पैकेज हैं जो कोडिंग एजेंट को Wink के साथ काम करना सिखाते हैं। प्रशिक्षण डेटा या पुरानी दस्तावेज़ीकरण पर निर्भर होने के बजाय, प्रत्येक कौशल एजेंट को [Wink MCP सर्वर](/hi/developers/mcp) का उपयोग API अनुबंधों और प्लेटफ़ॉर्म दस्तावेज़ों के लिए लाइव सत्य स्रोत के रूप में करने का निर्देश देता है।

कौशल किसी भी एजेंट क्लाइंट के साथ काम करते हैं जो उनका समर्थन करता है — Claude Code, Codex, Cursor, और अन्य।

## उपलब्ध कौशल

### wink-api-builder

Wink API एकीकरण को लागू करें, समीक्षा करें, या डिबग करें। यह कौशल एजेंट को एक संरचित कार्यप्रवाह के माध्यम से मार्गदर्शन करता है:

1. `api_search` के माध्यम से सही API ऑपरेशन खोजें
2. MCP सर्वर से लाइव अनुरोध/प्रतिक्रिया अनुबंध खोलें
3. दस्तावेज़ों से प्रमाणीकरण सेटअप प्राप्त करें
4. सत्यापित अनुबंध के खिलाफ कोड लिखें
5. मौजूदा कोड और वर्तमान API के बीच किसी भी असंगतियों को चिह्नित करें

उपयोगी है:
- किसी भी Wink API के लिए नए क्लाइंट मेथड बनाने के लिए
- अनुबंध असंगतियों को डिबग करने के लिए (जैसे HTTP 400 लौटाने वाला अनुरोध)
- लाइव स्पेक के खिलाफ मौजूदा एकीकरणों की समीक्षा करने के लिए

## इंस्टॉल करें

अपने एजेंट के कौशल निर्देशिका में कौशल फ़ोल्डर कॉपी करें।

:::tip
Wink MCP सर्वर को उसी वातावरण में कनेक्ट किया जाना चाहिए। सेटअप निर्देशों के लिए देखें [MCP Server](/hi/developers/mcp)।
:::

**Claude Code**

```bash
# रिपॉ-स्थानीय
cp -r skills/wink-api-builder .claude/skills/

# या उपयोगकर्ता-स्तर
cp -r skills/wink-api-builder ~/.claude/skills/
```

इसे इस तरह कॉल करें:
```
/wink-api-builder add a TypeScript client for the Wink affiliate create flow
```

**Codex**

```bash
# रिपॉ-स्थानीय
cp -r skills/wink-api-builder .agents/skills/

# या उपयोगकर्ता-स्तर
cp -r skills/wink-api-builder $CODEX_HOME/skills/
```

इसे इस तरह कॉल करें:
```
Use $wink-api-builder to add a TypeScript client for the Wink affiliate create flow.
```

**अन्य एजेंट**

जहां भी आपका एजेंट कौशल हल करता है, वहां कौशल फ़ोल्डर रखें। प्रवेश बिंदु `SKILL.md` है।
