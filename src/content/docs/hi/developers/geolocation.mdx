Wink स्थान-आधारित खोज, स्वचालित मुद्रा और भाषा पहचान, और भौगोलिक इन्वेंटरी फ़िल्टरिंग को सक्षम करने के लिए व्यापक रूप से भौगोलिक स्थिति का उपयोग करता है।

## GeoIP पहचान

जब कोई उपयोगकर्ता प्लेटफ़ॉर्म पर आता है, तो Wink स्वचालित रूप से उनके IP पते के आधार पर उनका स्थान पहचानता है। इससे सक्षम होता है:

- **स्वचालित मुद्रा चयन** — उपयोगकर्ता की स्थानीय मुद्रा में कीमतें दिखाना।
- **भाषा प्राथमिकता** — उपयोगकर्ता की संभावित भाषा में सामग्री सुझाना।
- **निकटवर्ती परिणाम** — उपयोगकर्ता के स्थान के करीब की इन्वेंटरी को प्राथमिकता देना।

## स्थानिक खोज

Wink चार स्थानिक क्वेरी प्रकारों का उपयोग करके भौगोलिक खोज का समर्थन करता है। ये पूरे प्लेटफ़ॉर्म पर मानचित्र-आधारित खोज, त्रिज्या खोज, और क्षेत्र-आधारित फ़िल्टरिंग को सक्षम करते हैं।

### बिंदु

एक एकल भौगोलिक निर्देशांक (अक्षांश, देशांतर)। किसी विशिष्ट स्थान को चिह्नित करने के लिए उपयोग किया जाता है — उदाहरण के लिए, किसी संपत्ति का पता या उपयोगकर्ता का वर्तमान स्थान।

```json
{
  "point": {
    "x": -73.9857,
    "y": 40.7484
  }
}
```

### वृत्त

एक केंद्र बिंदु और त्रिज्या। "मेरे पास खोजें" या त्रिज्या-आधारित क्वेरीज़ के लिए उपयोग किया जाता है — उदाहरण के लिए, "टाइम्स स्क्वायर से 10 किमी के भीतर होटल।"

```json
{
  "point": {
    "x": -73.9857,
    "y": 40.7484
  },
  "radius": 10000
}
```

### आयत

दक्षिण-पश्चिम और उत्तर-पूर्व कोनों द्वारा परिभाषित एक सीमा बॉक्स। मानचित्र-आधारित खोज के लिए उपयोग किया जाता है — जब कोई उपयोगकर्ता मानचित्र को पैन और ज़ूम करता है, तो दिखाई देने वाला क्षेत्र एक आयत क्वेरी को परिभाषित करता है।

```json
{
  "southWest": {
    "x": -74.0060,
    "y": 40.7128
  },
  "northEast": {
    "x": -73.9350,
    "y": 40.7831
  }
}
```

### बहुभुज

बिंदुओं की एक श्रृंखला द्वारा परिभाषित एक कस्टम आकृति। अनियमित भौगोलिक सीमाओं के भीतर खोज के लिए उपयोग किया जाता है — उदाहरण के लिए, एक पड़ोस, एक शहर का जिला, या मानचित्र पर कस्टम-ड्रॉ किया गया क्षेत्र।

```json
{
  "polygon": {
    "points": [
      { "x": -74.0060, "y": 40.7128 },
      { "x": -73.9350, "y": 40.7128 },
      { "x": -73.9350, "y": 40.7831 },
      { "x": -74.0060, "y": 40.7831 },
      { "x": -74.0060, "y": 40.7128 }
    ]
  }
}
```

:::note
बहुभुज बंद होना चाहिए — अंतिम बिंदु पहले बिंदु से मेल खाना चाहिए।
:::

## उपयोग के मामले

| उपयोग का मामला | आकृति प्रकार | उदाहरण |
|---|---|---|
| मेरे पास खोजें | वृत्त | मेरे स्थान से 5 किमी के भीतर होटल |
| मानचित्र दृश्य पोर्ट खोज | आयत | वर्तमान मानचित्र दृश्य में दिखाई देने वाली संपत्तियाँ |
| पड़ोस खोज | बहुभुज | एक विशिष्ट शहर जिले में होटल |
| संपत्ति स्थान | बिंदु | किसी संपत्ति सूची का सटीक स्थान |

## डेवलपर्स के लिए

स्थानिक क्वेरी पैरामीटर Lookup API और Inventory API एंडपॉइंट्स द्वारा स्वीकार किए जाते हैं। स्थान के अनुसार परिणामों को फ़िल्टर करने के लिए अपनी खोज अनुरोध में उपयुक्त GeoJSON आकृति पास करें।

एंडपॉइंट-विशिष्ट विवरण के लिए [APIs documentation](/hi/developers/apis) देखें।
