बुकिंग इंजन दो प्रकारों में आता है ताकि एक सहयोगी, एक संपत्ति प्रबंधन कंपनी और एक होटल की विभिन्न आवश्यकताओं का समर्थन किया जा सके।

## ऑनलाइन ट्रैवल एजेंसी

एक सहयोगी के लिए, ऑनलाइन ट्रैवल एजेंसी संस्करण का उपयोग करना सबसे अधिक समझदारी है। इसका कारण यह है कि साइट उन यात्रियों को समायोजित करने के लिए सेट की गई है जो कई संपत्तियों को देखना चाहते हैं।

आप इस एप्लिकेशन का होमपेज नीचे पा सकते हैं।

<LinkButton href="https://ota.wink.travel/home" variant="secondary" icon="external" target="_blank">
  Wink OTA
</LinkButton>

## इंटरनेट बुकिंग इंजन

एक होटल या संपत्ति प्रबंधन कंपनी के लिए, इंटरनेट बुकिंग इंजन संस्करण का उपयोग करना सबसे अधिक समझदारी है। इस प्रकार का उपयोगकर्ता एक एकल संपत्ति या प्रबंधन के तहत कई संपत्तियों को उजागर करने में रुचि रखता है।

आप इस एप्लिकेशन का होमपेज नीचे पा सकते हैं।

<LinkButton href="https://book.wink.travel/home?client-id=client-1&mp;configuration-id=d906a163-5cee-4198-a170-2bedf2407f5f" variant="secondary" icon="external" target="_blank">
  Wink IBE
</LinkButton>

## अनुकूलन

होटल और सहयोगी बुकिंग इंजन के `लुक और फील` को रंग पैलेट को अनुकूलित करके कस्टमाइज़ कर सकते हैं।

नीचे हमारे डेवलपर सैंडबॉक्स में अपने थीम को कस्टमाइज़ करके परीक्षण करें।

<LinkButton href="https://developer.wink.travel/web-component" variant="secondary" icon="external" target="_blank">
  थीमिंग
</LinkButton>


## कॉन्फ़िगरेशन

अधिकांश समय, इन दोनों एप्लिकेशन को सीधे साझा करने की आवश्यकता नहीं होती है। हमने आपके लिए यात्रा इन्वेंटरी साझा करने के कई तरीके बनाए हैं, जिन्हें आप [Wink Studio](/studio/what-is-studio) में या [WinkLinks](/link-manager/wink-links) के माध्यम से खोज सकते हैं।

हालांकि, यदि आप मैन्युअल रूप से इन वेबसाइटों में से किसी एक से लिंक करना चाहते हैं, तो सुनिश्चित करें कि आप URL में अपने क्लाइंट ID और कस्टमाइज़ेशन ID को पैरामीटर के रूप में जोड़ें।

आप अपना क्लाइंट ID इस प्रकार पा सकते हैं:

<Steps>
1. [Wink Extranet](/extranet/what-is-extranet) या [Wink Studio](/studio/what-is-studio) में लॉग इन करें।
2. ऊपर-दाएँ कोने में अपने प्रोफ़ाइल आइकन पर क्लिक करें और [Applications](/developers/applications) पर क्लिक करें।
3. उस एप्लिकेशन का उपयोग करें जो उस Wink खाते से जुड़ा हो जिसके साथ आप काम करना चाहते हैं।
4. उस एप्लिकेशन से क्लाइंट ID कॉपी करें और URL में अनुरोध पैरामीटर के रूप में पेस्ट करें। उदाहरण: ?client-id=MY_CLIENT_ID
</Steps>

### एक्सट्रानेट

आप एक संपत्ति के रूप में अपना कस्टमाइज़ेशन ID इस प्रकार पा सकते हैं:

<Steps>
1. [Wink Extranet](/extranet/what-is-extranet) में लॉग इन करें।
2. उस संपत्ति का चयन करें जिसके साथ आप काम करना चाहते हैं।
3. मुख्य नेविगेशन बार से, `Account > Branding` पर क्लिक करें।
4. कस्टमाइज़ेशन ID कॉपी करें और URL में अनुरोध पैरामीटर के रूप में पेस्ट करें। उदाहरण: ?configuration-id=MY_CUSTOMIZATION_ID
</Steps>

### स्टूडियो

आप एक सहयोगी के रूप में अपना कस्टमाइज़ेशन ID इस प्रकार पा सकते हैं:

<Steps>
1. [Wink Studio](/studio/what-is-studio) में लॉग इन करें।
2. उस खाते का चयन करें जिसके साथ आप काम करना चाहते हैं।
3. मुख्य नेविगेशन बार से, `Tools > Customization` पर क्लिक करें।
4. उस कस्टमाइज़ेशन से कस्टमाइज़ेशन ID कॉपी करें जिसे आप उपयोग करना चाहते हैं और URL में अनुरोध पैरामीटर के रूप में पेस्ट करें। उदाहरण: ?configuration-id=MY_CUSTOMIZATION_ID
</Steps>

## OTA

आपके पास अंत में ऐसा URL होगा:

https://ota.wink.travel?client-id=MY_CLIENT_ID&configuration-id=MY_CUSTOMIZATION_ID

यदि आप सीधे किसी होटल पेज पर जाना चाहते हैं, तो URL कुछ इस तरह होगा:

https://ota.wink.travel/hotel/hotel-x?client-id=MY_CLIENT_ID&configuration-id=MY_CUSTOMIZATION_ID

:::note
यदि आप क्लाइंट और/या कस्टमाइज़ेशन ID शामिल नहीं करते हैं, तो Wink OTA डिफ़ॉल्ट कॉन्फ़िगरेशन पर सेट हो जाएगा।
:::

## IBE

यदि आप एक संपत्ति प्रबंधन कंपनी हैं, तो आप Wink IBE होमपेज से लिंक करना चाहेंगे। होमपेज आपके निर्दिष्ट थीम के साथ-साथ [आप जिन संपत्तियों का प्रतिनिधित्व करते हैं](/studio/agency-representation) दिखाएगा।

आपके पास अंत में ऐसा URL होगा:

https://book.wink.travel?client-id=MY_CLIENT_ID&configuration-id=MY_CUSTOMIZATION_ID

यदि आप एक संपत्ति हैं, तो आप सीधे Wink IBE संपत्ति लैंडिंग पेज से लिंक करना चाहेंगे।

वह URL कुछ इस तरह होगा:

https://book.wink.travel/hotel/hotel-x?client-id=MY_CLIENT_ID&configuration-id=MY_CUSTOMIZATION_ID


:::note
यदि आप क्लाइंट और/या कस्टमाइज़ेशन ID शामिल नहीं करते हैं, तो Wink IBE लोड नहीं होगा। ये पैरामीटर आवश्यक हैं।
:::

## उदाहरण: संपत्ति पेज

Wink पर हर संपत्ति को अपनी अनूठी लैंडिंग पेज मिलती है, चाहे वह किसी भी वेबसाइट पर हो। सभी बाहरी लिंक यात्रियों को इन लैंडिंग पेजों में से किसी एक पर निर्देशित करते हैं।

लैंडिंग पेज में वह सारी जानकारी होती है जिसकी एक यात्री को यह निर्णय लेने के लिए आवश्यकता होती है कि क्या वह इस संपत्ति में ठहरना चाहता है।
