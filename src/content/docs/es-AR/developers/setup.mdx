Este artículo te muestra cómo configurarte rápidamente como desarrollador y comenzar a probar nuestras APIs en Wink.

:::note
Las pruebas siempre deben realizarse en nuestro [entorno de staging](/es-AR/getting-started/environments). Hay muchos menos datos allí, pero puedes probar todo lo que quieras sin afectar el entorno de producción.

Una vez que estés listo para lanzar, certificaremos tu cuenta y tu aplicación y podrás operar en nuestro entorno de producción.
:::

Sigue estos pasos:

<Steps>
1. [Registra](/es-AR/user/register) tu cuenta de usuario en Wink
2. Crea una [cuenta de agente de viajes](/es-AR/travel-agent/what-is-agent) o una [cuenta de afiliado](/es-AR/studio/what-is-studio).
3. [Crea una Aplicación](/es-AR/developer/applications) y asóciala con la cuenta que creaste en el Paso 2.
4. Una vez que hayas creado tu `Application`, guarda el client-id y el client secret en un lugar seguro.
</Steps>

## CDN

En Wink, usamos Cloudinary para todos nuestros recursos de imágenes y videos. Cuando accedas a nuestro inventario de viajes, tendrás acceso completo a todas las funciones avanzadas de Cloudinary.

Nuestro `cloud_name` es `traveliko`.

### Cloudinary

- [Primeros pasos](https://cloudinary.com/documentation/dev_kickstart)
- [SDKs](https://cloudinary.com/documentation/cloudinary_sdks)
- [Cloudinary en GitHub](https://github.com/cloudinary)
-	[Tutoriales en video](https://cloudinary.com/documentation/cloudinary_sdks_tutorials)

### Ejemplo

Para cada tipo de inventario de viajes que soportamos, habrá un arreglo JSON anidado llamado `multimedias`. El JSON para una entrada multimedia se ve así:

```json {3}
{
  "multimediaIdentifier": "multimedia-1",
  "identifier": "partners/radisson_blu_logo",
  "type": "IMAGE",
  "width": "1024",
  "height": "768",
  "category": "1",
  "descriptions": [
    {
      "name": "Lobby",
      "description": "Foto del lobby",
      "language": "en"
    }
  ],
  "lifestyleType": "LIFESTYLE_BUSINESS",
  "attribution": [
    {
      "url": "https://www.hilton.com",
      "name": "Hilton"
    }
  ]
}
```

La línea resaltada arriba muestra el ID único de Cloudinary `partners/radisson_blu_logo`. Eso es todo lo que necesitas para obtener la imagen en cualquier formato, calidad y tamaño que desees.

## Pruebas

Dirígete a nuestra sección de [APIs](/es-AR/developer/apis) y elige la API con la que quieras trabajar.

### Documentación de la API

Nuestro sitio de documentación está configurado para que puedas ejecutar comandos API desde dentro del sitio.

En este ejemplo, quieres usar la llamada API `Search Lookups (OAuth2)`.

:::tip[Consejo Profesional]
Para facilitar el uso, recomendamos que los integradores usen los endpoints que designamos como `OAuth2`. Esos endpoints saben que esperan llamadas API y ya saben a qué cuenta perteneces, por lo que no necesitas especificar el `companyIdentifier` en tus URLs.
:::

<Steps>
1. Usa la [API Lookup](https://developer.wink.travel/lookup) para obtener destinos o hoteles.
2. En la barra lateral izquierda, haz clic en `Authentication`.
3. Ingresa tu client-id y client secret y haz clic en el botón `Get TOKEN`.
4. Si tus credenciales son válidas, verás que tu clave API ha sido aplicada.
5. Ahora puedes probar el endpoint de lookup.
</Steps>

### Postman
Puedes hacer lo mismo con Postman.

<Steps>
1. Descarga [Postman](https://www.postman.com/downloads).
2. Haz clic en `Download OpenAPI spec` desde la parte superior de la página de la [API Lookup](https://developer.wink.travel/lookup).
3. Impórtalo en Postman.
4. Si lo importaste como una nueva colección llamada `Wink API`, haz clic en el elipsis horizontal (3 puntos) junto al nombre de la colección y selecciona `Edit`.
5. En la pestaña `Authorization`, configura el `Auth type` a OAuth2.
6. Configura el campo `Add auth data to` a `Request headers`.
7. Configura el campo `Header prefix` a `Bearer`.
8. Configura el campo `Token Name` con el nombre que quieras darle a tu token guardado en Postman. Por ejemplo, Wink Staging.
9. Configura el campo `Grant type` a `Client Credentials`.
10. Configura el campo `Access token URL` a https://staging-iam.wink.travel/oauth2/token.
11. Configura el campo `Client ID` con el Client ID que generó tu app.
12. Configura el campo `Client Secret` con el Client Secret que generó tu app.
13. Configura el campo `Scope` a `inventory.read inventory.write inventory.remove`. Esos son todos los scopes que necesitarás para Wink.
14. Configura el campo `Client Authentication` a `Send as Basic Auth header`.
15. Haz clic en el botón `Get New Access Token` para obtener tu token y guárdalo.
16. Para todas las llamadas API bajo la colección Wink API, asegúrate de que la pestaña de autenticación diga `Inherit auth from parent` para que incluya tu token en cada llamada.
</Steps>

:::tip[Consejo Profesional]
Se recomienda convertir el dominio del servidor, client ID y client secret en variables de entorno. Así podrás cambiar fácilmente entre los entornos de staging y producción y reutilizar todas las operaciones que hayas creado.
:::
