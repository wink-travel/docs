## Descripción general
El servidor MCP alojado de Wink está disponible en:

```
https://docs.mcp.wink.travel/mcp
```

Apuntá clientes de IA compatibles (VS Code, Cursor, otros que implementen transporte HTTP MCP) a este endpoint para que puedan:
- Navegar la documentación de productos y plataforma (`wink://docs/...`)
- Descubrir SDKs y profundizar en clases / modelos de API (`wink://sdk/...` y `wink://sdk/.../docs`)
- Realizar búsquedas semánticas / por palabras clave livianas sobre resúmenes de docs y SDKs

## Transporte
Usá el endpoint HTTP principal y transmisible:

```
https://docs.mcp.wink.travel/mcp
```

Si tu cliente solo soporta SSE, usá:

```
https://docs.mcp.wink.travel/sse
```

### Cursor
Documentación MCP de Cursor: https://cursor.com/docs/context/mcp  
Configuración: Agregá el servidor remoto vía ajustes MCP.
```json
{
  "mcpServers": {
    "wink-docs": {
      "url": "https://docs.mcp.wink.travel/mcp",
    }
  }
}
```

### VS Code
Documentación oficial: https://code.visualstudio.com/docs/copilot/customization/mcp-servers  
Configuración remota mínima (usuario o workspace `mcp.json`):
```json
{
  "servers": {
    "wink-docs": {
      "type": "http",
      "url": "https://docs.mcp.wink.travel/mcp"
    }
  }
}
```

Consejos:
- Usá la Paleta de Comandos: “MCP: List Servers” para confirmar conexión y navegar herramientas.
- El selector de herramientas permite habilitar/deshabilitar `sdk_search` y `docs_search`.
- Agregá contexto: Vista de chat → Agregar Contexto → Recursos MCP para adjuntar páginas de docs.

## URIs principales de recursos

| URI | Descripción |
|-----|-------------|
| `wink://docs` | Índice de documentación (agrupado por categoría) |
| `wink://docs/{path}` | Artículo específico de documentación (Markdown / MDX) |
| `wink://sdk` | Vista general de SDK (cada SDK + conteo de docs) |
| `wink://sdk/{sdkPath}` | README del SDK / carpeta de docs del módulo / doc específica del módulo |

### Documentación de módulos
Abrir una carpeta de docs de SDK (ej. `wink://sdk/python/wink-sdk-booking/docs`) devuelve agrupado:
- **Clases API** (cantidad de métodos + nombres de métodos de ejemplo)
- **Modelos y Tipos** (estructuras de datos usadas por esas APIs)

## Herramientas de búsqueda (Referencia rápida)
| Herramienta | Propósito | Ejemplos típicos de consulta |
|-------------|-----------|------------------------------|
| `sdk_search` | Encontrar qué SDK / clase API usar | `cancelar reserva`, `listar propiedades`, `inventario afiliado` |
| `docs_search` | Encontrar docs conceptuales / guías | `autenticación`, `webhooks`, `wordPress` |

Dejá `query` vacío para recibir sugerencias. Cada resultado incluye el URI MCP que podés abrir.

## Patrón típico de uso
1. Ejecutá `sdk_search` con tu intención (ej. *"listar propiedades"*).
2. Abrí el README del SDK retornado o su `docs_uri` para inspeccionar APIs / modelos.
3. Usá `docs_search` para conceptos de soporte (autenticación, webhooks).

## Notas
- Cada URI retornado puede ser abierto por el agente para contenido completo.
- Las sugerencias aparecen automáticamente cuando las consultas están vacías.
