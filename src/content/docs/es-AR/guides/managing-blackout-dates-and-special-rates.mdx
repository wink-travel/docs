No todos los días tienen el mismo precio. Eventos, feriados, temporadas bajas y espacios de último momento requieren precios diferentes. Esta guía te muestra cómo usar tarifas especiales y restricciones de planes tarifarios para manejar estos escenarios.

## Precios para eventos

Cuando una conferencia, festival o evento local genera demanda, querrás aumentar las tarifas para esas fechas.

<Steps>
1. Ve a `Monetize > Special Rates` en tu Extranet.
2. Haz clic en `Create a new special rate`.
3. Selecciona la tarifa maestra y el tipo de habitación a modificar.
4. Establece el rango de fechas que cubra el período del evento.
5. Ingresa los precios diarios ajustados (más altos).
6. Haz clic en `Save`.
</Steps>

La tarifa especial reemplaza la tarifa maestra solo para esas fechas. Tus precios regulares permanecen sin cambios fuera del período del evento.

## Ajustes estacionales

Para períodos más largos (temporada alta, media, baja), puedes:

- **Usar tarifas especiales** — Crear modificaciones para cada temporada con precios ajustados.
- **Actualizar tarifas maestras** — Ajustar las tarifas base directamente para esos períodos.

Las tarifas especiales son mejores para ajustes temporales, mientras que actualizar las tarifas maestras tiene sentido para patrones anuales previsibles.

## Tarifas de último momento

Para llenar habitaciones no vendidas cerca de la fecha de check-in:

<Steps>
1. Crea un nuevo **plan tarifario** con una restricción de reserva anticipada (por ejemplo, debe reservarse con no más de 3 días de anticipación).
2. Combínalo con una **tarifa especial** que ofrezca un descuento para las fechas objetivo.
3. Este plan tarifario solo aparecerá para viajeros que reserven dentro de la ventana de último momento.
</Steps>

## Fechas de bloqueo

Para bloquear ciertas fechas en un plan tarifario específico (por ejemplo, sin tarifas promocionales durante temporada alta):

<Steps>
1. Ve a `Monetize > Rate Plans` y selecciona el plan tarifario.
2. Navega a la sección de **Restrictions**.
3. Agrega una restricción de **Sell date** para limitar cuándo se puede reservar la tarifa.
4. O usa restricciones de **Stay date** para limitar cuándo cubre la tarifa.
</Steps>

Por ejemplo, puedes restringir tu plan tarifario "Early Bird" para que no esté disponible para reservas durante diciembre (temporada alta), mientras que tu plan tarifario regular permanece abierto.

## Combinando restricciones

Las restricciones pueden combinarse para un control preciso:

| Restricción | Qué controla |
| -- | -- |
| **Sell dates** | Cuándo se puede reservar la tarifa |
| **Stay dates** | Qué fechas de estadía cubre la tarifa |
| **Available days** | Qué días de la semana aparece la tarifa |
| **Arrival days** | En qué días debe llegar el huésped |
| **Departure days** | En qué días debe partir el huésped |
| **Required days** | El huésped debe alojarse al menos uno de estos días |
| **Advance booking** | Cuánto tiempo antes debe hacerse la reserva |
| **Length of stay** | Noches mínimas y máximas |
