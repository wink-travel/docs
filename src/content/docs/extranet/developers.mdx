The developer section lets you integrate your property systems with the Wink platform via API applications and webhooks.

## Applications

Applications let you connect to Wink APIs and SDKs, embed web components on your website, and enable authenticated sessions with the platform.

To manage your applications, navigate to `Developer > Applications`.

### Creating an application

Click `Create application` and fill in:

- **Name** — A unique name for your application.
- **Client type** — Web, SPA, Mobile, or Machine-to-Machine.
- **Redirect URIs** — URLs where users are sent after login.
- **Post-logout redirect URIs** — Where users are sent after logging out.

After creation, you will see your **Client ID** and **Secret Key**.

:::caution
The secret key is only displayed once. Copy and store it securely.
:::

### Revoking credentials

Click `Revoke` to generate a new Client ID and Secret Key. This immediately invalidates the current credentials.

## Webhooks

Webhooks deliver real-time event notifications when changes occur on your property — bookings, cancellations, rate updates, and more.

To manage your webhooks, navigate to `Developer > Webhooks`.

### Creating a webhook

- **Name** — A descriptive name for the webhook.
- **Webhook URL** — The endpoint on your server that receives notifications.
- **Events** — Select specific events or leave empty to receive all.
- **Enabled** — Toggle notifications on or off without deleting.

## API

Developers who want to manage property features programmatically can head over to [Developers > APIs > Extranet](/developers/apis/#extranet-api).
