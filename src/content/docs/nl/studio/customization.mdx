Aanpassing is een krachtige functie die je kunt gebruiken om elk aspect van de integratie met Wink te beheren zonder dat je code hoeft te schrijven. Elk stuk inventaris dat je verkoopt, kan worden aangepast. Voeg functies toe of verwijder ze, wijzig het kleurenschema en nog veel meer.

Terwijl je op de pagina `Aanpassing` bent, klik je op de knop `Acties` onder de aanpassing waarmee je wilt werken en klik je op `Bijwerken`.
*Je hebt al een standaardinvoer beschikbaar wanneer je je account aanmaakt. Deze invoer kan niet worden verwijderd.*

Dit brengt je naar het aanpassingsformulier waar je 5 - 7 tabbladen kunt zien (*afhankelijk van je accounttype*).

## Algemeen

Hier zijn de opties die beschikbaar zijn onder het tabblad `Algemeen`.

- **Naam** Geef je aanpassing een naam om deze gemakkelijk te onthouden. *bijv. Voor mijn reisblog*
- **Weergavevaluta** Selecteer de initiële valuta die gebruikers te zien krijgen naast boekbare reisvoorraad. *bijv. USD*
- **Weergavetaal** Selecteer de initiële taal die gebruikers zullen tegenkomen. *bijv. Engels*
- **Toevoegen aan paginatitel** Je kunt bepalen wat er wordt toegevoegd aan de paginatitel (*de titel die je in het browsertabblad ziet*). Dit wordt voornamelijk gebruikt voor onze boekingsengine. *bijv. Als je naar https://ota.wink.travel navigeert, zie je standaard `Home | Traveliko - Hotels with Benefits`*. Je kunt alles na de '|' pijp wijzigen.

## Thema

Het tabblad `Thema` laat je het uiterlijk en de uitstraling van onze [boekingsengine](/booking-engine/in-a-nutshell) website, [WinkLinks](/link-manager/wink-links) website en al onze [Web Components](/developers/web-components) aanpassen.

### Configuratie

Deze sectie laat je het ontwerp en de kaartlay-out configureren. Op dit moment is er slechts één ontwerp; namelijk Wink.

De kaartlay-out bepaalt of je wilt dat je gebruikers horizontaal of verticaal uitgelijnde transactieskaarten zien. De informatie is hetzelfde.

:::note
Er is geen verschil in kaartlay-out bij het bekijken van onze transactieskaarten vanaf een mobiel apparaat. Je kunt het verschil echter wel zien op een iPad mini en grotere resoluties.
:::

:::tip[Pro tip]
Gebruik de horizontale lay-out als je minder dan 4 reisvoorraad (bijv. kamertypes) in een lijst hebt... het ziet er gewoon beter uit.
:::

### Kleuren

Kleur bepaalt hoe je gebruikers onze UI-componenten en websites zien. Gebruik het om aan te sluiten bij je bestaande kleurenschema [indien aanwezig].

We hebben al een grote selectie vooraf gedefinieerde kleuren waar je mee kunt experimenteren. De vooraf gedefinieerde thema’s veranderen de kleuren in de onderstaande formulier velden. Je kunt ook volledig je eigen kleuren gebruiken.

## Analytics

We ondersteunen GA4 analytics standaard. Als je het volledige verbeterde e-commerce conversiepad wilt volgen, van begin tot eind, houd dan je meet-ID bij de hand en voer deze in het veld onder het tabblad `Analytics` in.

<Aside type="caution" title="Herinnering">
  Vergeet niet `i.trvl.as`, `book.wink.travel` en `ota.wink.travel` toe te voegen aan je GA4 property streams.
</Aside>

## Reisschema

Het tabblad `Reisschema` is waarschijnlijk het meest interessant om ervoor te zorgen dat prijzen worden weergegeven voor de inventaris die je hebt geselecteerd.
Omdat de prijsstelling van reisvoorraad vrij complex is, moet Wink het reisschema van de gebruiker kennen voordat een nauwkeurige prijs kan worden weergegeven.

Deze sectie laat je het standaard reisschema instellen om prijzen weer te geven voor al ons digitale vastgoed, zoals:

- Onze socialvriendelijke afbeeldingen met prijsoverlay.
- WinkLinks rasters, kaarten en links.
- Web Components
- Boekingsengine

### Data

Je kunt kiezen of je een statisch datumbereik of een rollend bereik wilt instellen.

- Kies een statisch datumbereik als je de aanpassing gebruikt voor een specifiek evenement dat op een bepaalde datum plaatsvindt.
- Kies een rollend datumbereik als je wilt:
    1. Standaard instellen op vandaag + 1 nacht, wat de meeste OTA’s doen OF.
    2. Standaard instellen op een datum in de toekomst die altijd beschikbaarheid moet hebben.

### Kamerconfiguratie

Standaard is het reisschema ingesteld op 2 volwassenen, omdat dit de meest voorkomende instelling is voor de meeste reizigers.
Dit kan eenvoudig worden gewijzigd en je kunt meerdere kamers toevoegen met meerdere bewoners; zowel volwassenen als kinderen.

### Promoties

Promoties is een handige functie die je gebruikers tijd bespaart. Als een accommodatie je een promotiecode heeft gegeven die je met je vrienden kunt delen, kun je ervoor kiezen deze hier in te voeren.
Iedereen die via jouw link of iets dergelijks binnenkomt, krijgt automatisch de promotiecode toegevoegd aan hun reisschema en ziet de korting automatisch toegepast.

## Integratie

Het tabblad `Integratie` is voor hotels, integrators en reisagenten die meer controle willen over hoe boekingen op Wink worden verdeeld.

### Meldingen

Dit geeft het hotel en de integrator de optie om Wink alle betrokken partijen te laten informeren wanneer een boeking plaatsvindt.
Dit omvat het informeren van de channel manager.

Standaard zijn deze opties allemaal ingeschakeld. Gebruikssituaties waarin een integrator een of meer van deze functies wil uitschakelen zijn:

- De reisagent is geïntegreerd op API-niveau en wil zijn eigen bevestigingsmails versturen.
- Het hotel test zijn integratie en wil het verkoopteam niet storen.
- Het hotel geeft niet om e-mails en wil alleen via hun PMS worden geïnformeerd.

### E-mail logo

Voor integrators die onze e-mailmeldingsservice willen gebruiken, kunnen ze de e-mail aanpassen zodat het lijkt alsof deze van hen afkomstig is.

:::note
  Aanbevolen afmetingen zijn 300px breed x 75px hoog
:::

## Hotelspecifiek

Onder het tabblad `Hotelspecifiek` zijn opties waarmee het hotel kan bepalen wat er op hun landingspagina van onze boekingsengine wordt weergegeven en wat er wordt getoond als er geen inventaris is voor het gevraagde reisschema.

- **Rankings**: Een accommodatie die nog geen betekenisvolle rankings heeft behaald, kan ervoor kiezen deze functie uit te schakelen.

:::note
  De accommodatie kan niet bepalen hoe een affiliate de hotel landingspagina weergeeft... Dit zijn twee verschillende configuraties.
:::

- **Beschikbaarheid**: Laat de accommodatie kiezen of de inventaris wordt weergegeven, ook al is er geen beschikbaarheid.

:::note
  In de meeste gevallen wil de accommodatie gebruikers een manier geven om op de knop `Boek` te klikken om door te gaan naar hun landingspagina waar ze hun reisschema kunnen wijzigen en proberen een beschikbare datum te vinden.
  Schakel deze schakelaar in als je wilt voorkomen dat de gebruiker doorgaat.
:::

## Hoe te gebruiken

In de meeste scenario’s kies je welke `Aanpassing` je toepast op je links, kaarten, kaarten en rasters.

Hier zijn de momenten waarop je je configuratie handmatig wilt toepassen:

- **Handmatige override** Je wilt een link naar onze boekingsengine direct delen, en niet onze [Deelbare Links](/studio/shareable-links) functie gebruiken, met je aangepaste configuratie toegepast.
- **WordPress plugin** Voeg de configuratie toe aan je Wink [WordPress Plugin](/developers/wordpress) `Instellingen`.
- **Aangepaste integratie** Pas het handmatig toe op je eigen integrator met behulp van onze [wink-app-loader](/developers/web-components/#application-loader) Web Component.

### Handmatige override

Hier is een eenvoudig voorbeeld van hoe je je configuratie-ID, met al je instellingen, toepast op onze hoofd-boekingsengine-URL’s.

<h4 align="center">[https://ota.wink.travel?configurationId=MY-UNIQUE-ID](https://ota.wink.travel?configurationId=MY-UNIQUE-ID)</h4>

:::note
  Wanneer een gebruiker op deze link klikt, stel je ook je affiliate attributiecookie in.
:::

### WordPress plugin

Ga naar de [Wink WordPress plugin](/developers/wordpress) om meer te leren.

### Aangepaste integratie

Beheer al je ingesloten Wink Web Components op een pagina met de [wink-app-loader](/developers/web-components/#application-loader) Web Component.

Hieronder een voorbeeld dat `MY-UNIQUE-ID` gebruikt als configuratie-ID.

```html {11}
<html>
  <head>
    <link rel="stylesheet" href="https://elements.wink.travel/styles.css"></link>
  </head>
  <body>

    <script src="https://elements.wink.travel/elements.js" type="module" defer></script>

    <wink-app-loader
      client-id="YOUR CLIENT ID GOES HERE"
      configuration-id="MY-UNIQUE-ID"
    ></wink-app-loader>
  </body>
</html>
```
