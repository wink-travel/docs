Dit artikel laat je zien hoe je snel aan de slag kunt als ontwikkelaar en onze API's op Wink kunt testen.

:::note
Testen moet altijd plaatsvinden in onze [stagingomgeving](/nl/getting-started/environments). Daar is veel minder data, maar je kunt zo veel testen als je wilt zonder de productieomgeving te belasten.

Zodra je klaar bent om live te gaan, zullen wij je account en je applicatie certificeren en kun je lanceren in onze productieomgeving.
:::

Volg deze stappen:

<Steps>
1. [Registreer](/nl/user/register) je gebruikersaccount op Wink
2. Maak een [reisagentaccount](/nl/travel-agent/what-is-agent) of een [affiliate-account](/nl/studio/what-is-studio) aan.
3. [Maak een Applicatie aan](/nl/developer/applications) en koppel deze aan het account dat je in stap 2 hebt aangemaakt.
4. Zodra je je `Applicatie` hebt aangemaakt, bewaar je de client-id en client secret op een veilige plek.
</Steps>

## CDN

Bij Wink gebruiken we Cloudinary voor al onze beeld- en video-assets. Wanneer je toegang krijgt tot onze reisvoorraad, heb je volledige toegang tot alle geavanceerde functies van Cloudinary.

Onze `cloud_name` is  `traveliko`.

### Cloudinary

- [Aan de slag](https://cloudinary.com/documentation/dev_kickstart)
- [SDK's](https://cloudinary.com/documentation/cloudinary_sdks)
- [Cloudinary op GitHub](https://github.com/cloudinary)
-	[Video Tutorials](https://cloudinary.com/documentation/cloudinary_sdks_tutorials)

### Voorbeeld

Voor elk type reisvoorraad dat we ondersteunen, is er een geneste JSON-array genaamd `multimedias`. De JSON voor een multimedia-item ziet er als volgt uit:

```json {3}
{
  "multimediaIdentifier": "multimedia-1",
  "identifier": "partners/radisson_blu_logo",
  "type": "IMAGE",
  "width": "1024",
  "height": "768",
  "category": "1",
  "descriptions": [
    {
      "name": "Lobby",
      "description": "Foto van de lobby",
      "language": "en"
    }
  ],
  "lifestyleType": "LIFESTYLE_BUSINESS",
  "attribution": [
    {
      "url": "https://www.hilton.com",
      "name": "Hilton"
    }
  ]
}
```

De gemarkeerde regel hierboven toont de unieke Cloudinary ID `partners/radisson_blu_logo`. Dat is alles wat je nodig hebt om de afbeelding in elk formaat, kwaliteit en grootte op te halen.

## Testen

Ga naar onze [API's](/nl/developer/apis) sectie en kies de API waarmee je wilt werken.

### API-documentatie

Onze documentatiesite is zo opgezet dat je API-commando's direct vanuit de site kunt uitvoeren.

In dit voorbeeld wil je de `Search Lookups (OAuth2)` API-aanroep gebruiken.

:::tip[Pro Tip]
Voor gebruiksgemak raden we integrators aan om de endpoints te gebruiken die wij aanduiden als `OAuth2`. Die endpoints weten dat ze API-aanroepen verwachten en weten al bij welk account je hoort, zodat je de `companyIdentifier` niet in je URL's hoeft op te geven.
:::

<Steps>
1. Gebruik de [Lookup API](https://developer.wink.travel/lookup) om bestemmingen of hotels op te halen.
2. Klik in de linker navigatiebalk op `Authentication`.
3. Voer je client-id en client secret in en klik op de knop `Get TOKEN`.
4. Als je inloggegevens geldig zijn, zie je dat je API-sleutel is toegepast.
5. Je kunt nu de lookup endpoint testen.
</Steps>

### Postman
Je kunt hetzelfde doen met Postman.

<Steps>
1. Download [Postman](https://www.postman.com/downloads).
2. Klik op `Download OpenAPI spec` bovenaan de [Lookup API](https://developer.wink.travel/lookup) pagina.
3. Importeer deze in Postman.
4. Als je het hebt ge√Ømporteerd als een nieuwe collectie genaamd `Wink API`, klik dan op de horizontale elipsis (3 puntjes) naast de collectienaam en klik op `Edit`.
5. Stel in het tabblad `Authorization` het `Auth type` in op OAuth2.
6. Stel het veld `Add auth data to` in op `Request headers`.
7. Stel het veld `Header prefix` in op `Bearer`.
8. Geef het veld `Token Name` een naam naar keuze voor je opgeslagen token in Postman, bijvoorbeeld Wink Staging.
9. Stel het veld `Grant type` in op `Client Credentials`.
10. Stel het veld `Access token URL` in op https://staging-iam.wink.travel/oauth2/token.
11. Stel het veld `Client ID` in op de Client ID die je app heeft gegenereerd.
12. Stel het veld `Client Secret` in op de Client Secret die je app heeft gegenereerd.
13. Stel het veld `Scope` in op `inventory.read inventory.write inventory.remove`. Dit zijn alle scopes die je voor Wink nodig hebt.
14. Stel het veld `Client Authentication` in op `Send as Basic Auth header`.
15. Klik op de knop `Get New Access Token` om je token op te halen en op te slaan.
16. Zorg ervoor dat bij alle API-aanroepen onder de Wink API-collectie het authenticatietabblad zegt `Inherit auth from parent`, zodat je token bij elke oproep wordt meegestuurd.
</Steps>

:::tip[Pro Tip]
Het wordt aanbevolen om de serverdomein, client ID en client secret als omgevingsvariabelen in te stellen. Zo kun je eenvoudig schakelen tussen staging- en productieomgevingen en hergebruik je alle gemaakte operaties.
:::
