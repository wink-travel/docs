[Webcomponenten](https://developer.mozilla.org/en-US/docs/Web/API/Web_components)is een standaard waarmee je complete Javascript-functies in je website kunt integreren zonder programmeerkennis.
Dankzij deze coole technologie kun je Wink Travel Inventory moeiteloos integreren. Dit artikel leidt je door onze collectie webcomponenten en laat je zien hoe je ze gebruikt.

## Wink insluiten

Er zijn drie dingen die u moet opnemen op elke pagina waarop u een van onze webcomponenten wilt weergeven.

1. Voeg ons stijlblad toe.
2. Gebruik onze Javascript.
3. Voeg onze applicatieloader toe.

### Stijlblad

Voeg onze CSS-stijlen toe aan de kop van uw document.

```html {3}
<html>
  <head>
    <link rel="stylesheet" href="https://elements.wink.travel/styles.css"></link>
  </head>
</html>
```

### Javascript

Voeg onze Javascript onderaan uw document toe.*Wij adviseren direct boven de eindteksttag*).

```html {3}
<html>
  <body>
    <script src="https://elements.wink.travel/elements.js" type="module" defer></script>
  </body>
</html>
```

Uw site is nu klaar en kan onze webcomponenten weergeven.

## Componenten

Lees hieronder meer over onze componentenbibliotheek.

### Applicatielader

De loader is verantwoordelijk voor het bijhouden van de status en het beheren van de interactie tussen onze componenten.*Voeg het toe onder onze Javascript*).

```html {3-6}
<html>
  <body>
    <wink-app-loader
      client-id="YOUR CLIENT ID GOES HERE"
      configuration-id="YOUR CUSTOMIZATION ID GOES HERE"
    ></wink-app-loader>
  </body>
</html>
```

### Inhoudslader

De content loader vormt de kern van onze webcomponentenbibliotheek. Deze is verantwoordelijk voor de weergave van uw inventaris (kaarten, rasters, kaarten).

Beschikbare kenmerken:

* indeling
* id
* soort

*Het kenmerk`sort`is alleen beschikbaar als de lay-out is`RANKED`en u geen bestaand gerangschikt raster wilt gebruiken. Laat in deze gevallen`id` leeg.*

Beschikbare lay-outtypen:

* `AD_BANNER`
* `MAP`
* `HOTEL`
* `GUEST_ROOM`
* `MEETING_ROOM`
* `SPA`
* `RESTAURANT`
* `ACTIVITY`
* `ATTRACTION`
* `PLACE`
* `ADD_ON`
* `LIST`
* `SEARCH`
* `RANKED`

Beschikbare sorteertypen:

* `MEMBER`
* `PRICE_LOW_TO_HIGH`
* `PRICE_HIGH_TO_LOW`
* `PRICE`
* `POPULARITY`
* `ECO`
* `EXPERIENCE`
* `PERK`
* `LOYALTY`
* `PACKAGE`

```html {3-6}
<html>
  <body>
    <wink-content-loader
       layout="GUEST_ROOM"
       id="2de7ee9c-61c9-11ef-9722-42004e494300"
    ></wink-content-loader>
  </body>
</html>
```

Er zijn voorbeelden te vinden voor[kaarten](/studio/cards),[roosters](/studio/grids) En [kaarten](/studio/maps).

## Componenten

### Betaling

Met het betaalonderdeel kunt u TripPay laten weten dat een reiziger iets wil kopen en wat de regels en prijzen voor die items zijn.

Er is één vereist kenmerk voor de widget:

* `id`De identificatie van het boekingscontract dat u wilt uitvoeren.

```html {3}
<html>
  <body>
    <trip-pay id="<INSERT_BOOKING_CONTRACT_ID>"></trip-pay>
  </body>
</html>
```

:::note
Leer hoe u het boekingscontract kunt genereren`id`, lees de gids:[Integratie met TripPay](/guides/integrating-with-trip-pay).
::

De widget bereidt het contract voor op uitvoering en geeft de betalingsgegevens wee&#x72;*(Figuur 1)*&#x61;an de gebruiker om de boeking af te ronden.
