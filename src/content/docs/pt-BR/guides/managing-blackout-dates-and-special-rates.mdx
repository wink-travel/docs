Nem todo dia tem o mesmo preço. Eventos, feriados, temporadas baixas e lacunas de última hora exigem preços diferentes. Este guia mostra como usar tarifas especiais e restrições de planos tarifários para lidar com esses cenários.

## Precificação para eventos

Quando uma conferência, festival ou evento local aumenta a demanda, você vai querer aumentar as tarifas para essas datas.

<Steps>
1. Vá para `Monetize > Special Rates` na sua Extranet.
2. Clique em `Create a new special rate`.
3. Selecione a tarifa mestre e o tipo de quarto para substituir.
4. Defina o intervalo de datas para cobrir o período do evento.
5. Insira os preços diários ajustados (mais altos).
6. Clique em `Save`.
</Steps>

A tarifa especial substitui a tarifa mestre apenas para essas datas. Sua precificação regular permanece inalterada fora do período do evento.

## Ajustes sazonais

Para períodos mais longos (alta temporada, meia temporada, baixa temporada), você pode:

- **Usar tarifas especiais** — Criar substituições para cada temporada com preços ajustados.
- **Atualizar tarifas mestres** — Ajustar diretamente as tarifas base para esses períodos.

Tarifas especiais são melhores para ajustes temporários, enquanto atualizar tarifas mestres faz sentido para padrões anuais previsíveis.

## Tarifas de última hora

Para preencher quartos não vendidos próximos à data de check-in:

<Steps>
1. Crie um novo **plano tarifário** com uma restrição de reserva antecipada (ex.: deve reservar com no máximo 3 dias de antecedência).
2. Combine com uma **tarifa especial** oferecendo desconto para as datas-alvo.
3. Esse plano tarifário aparecerá apenas para viajantes que reservarem dentro da janela de última hora.
</Steps>

## Datas bloqueadas

Para bloquear certas datas de um plano tarifário específico (ex.: sem tarifas promocionais durante a alta temporada):

<Steps>
1. Vá para `Monetize > Rate Plans` e selecione o plano tarifário.
2. Navegue até a seção **Restrictions**.
3. Adicione uma restrição de **Sell date** para limitar quando a tarifa pode ser reservada.
4. Ou use restrições de **Stay date** para limitar quando a tarifa cobre.
</Steps>

Por exemplo, você pode restringir seu plano tarifário "Early Bird" para que não esteja disponível para reservas durante dezembro (alta temporada), enquanto seu plano tarifário regular permanece aberto.

## Combinando restrições

As restrições podem ser combinadas para controle preciso:

| Restrição | O que controla |
| -- | -- |
| **Sell dates** | Quando a tarifa pode ser reservada |
| **Stay dates** | Quais datas de estadia a tarifa cobre |
| **Available days** | Quais dias da semana a tarifa aparece |
| **Arrival days** | Em quais dias o hóspede deve chegar |
| **Departure days** | Em quais dias o hóspede deve sair |
| **Required days** | O hóspede deve permanecer pelo menos em um desses dias |
| **Advance booking** | Com que antecedência a reserva deve ser feita |
| **Length of stay** | No mínimo e no máximo de noites |
