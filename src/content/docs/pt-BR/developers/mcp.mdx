## Visão Geral
O servidor MCP hospedado do Wink está disponível em:

```
https://docs.mcp.wink.travel/mcp
```

Aponte clientes de IA compatíveis (VS Code, Cursor, outros que implementam transporte HTTP MCP) para este endpoint para que eles possam:
- Navegar pela documentação de produtos e plataforma (`wink://docs/...`)
- Descobrir SDKs e explorar classes / modelos de API (`wink://sdk/...` & `wink://sdk/.../docs`)
- Realizar buscas semânticas / por palavra-chave leves sobre resumos de docs e SDKs

## Transporte
Use o endpoint HTTP principal e transmissível:

```
https://docs.mcp.wink.travel/mcp
```

Se seu cliente suporta apenas SSE, use:

```
https://docs.mcp.wink.travel/sse
```

### Cursor
Documentação MCP do Cursor: https://cursor.com/docs/context/mcp
Configuração: Adicione o servidor remoto via configurações MCP.
```json
{
  "mcpServers": {
    "wink-docs": {
      "url": "https://docs.mcp.wink.travel/mcp",
    }
  }
}
```

### VS Code
Documentação oficial: https://code.visualstudio.com/docs/copilot/customization/mcp-servers
Configuração remota mínima (usuário ou workspace `mcp.json`):
```json
{
  "servers": {
    "wink-docs": {
      "type": "http",
      "url": "https://docs.mcp.wink.travel/mcp"
    }
  }
}
```

Dicas:
- Use a Paleta de Comandos: “MCP: List Servers” para confirmar a conexão e navegar pelas ferramentas.
- O seletor de ferramentas permite ativar/desativar `sdk_search` e `docs_search`.
- Adicione contexto: Visualização de Chat → Adicionar Contexto → Recursos MCP para anexar páginas de docs.

## URIs Principais de Recursos

| URI | Descrição |
|-----|-------------|
| `wink://docs` | Índice de documentação (agrupado por categoria) |
| `wink://docs/{path}` | Artigo específico de documentação (Markdown / MDX) |
| `wink://sdk` | Visão geral do SDK (cada SDK + contagem de docs) |
| `wink://sdk/{sdkPath}` | README do SDK / pasta de docs do módulo / doc específico do módulo |

### Documentação de Módulos
Abrir uma pasta de docs do SDK (ex: `wink://sdk/python/wink-sdk-booking/docs`) retorna agrupado:
- **Classes de API** (contagem de métodos + nomes de métodos de exemplo)
- **Modelos & Tipos** (estruturas de dados usadas por essas APIs)

## Ferramentas de Busca (Referência Rápida)
| Ferramenta | Propósito | Exemplos Típicos de Consulta |
|------------|-----------|------------------------------|
| `sdk_search` | Encontrar qual SDK / classe de API usar | `cancelar reserva`, `listar propriedades`, `inventário afiliado` |
| `docs_search` | Encontrar docs conceituais / guias | `autenticação`, `webhooks`, `wordPress` |

Deixe `query` vazio para receber sugestões. Cada resultado inclui o URI MCP que você pode abrir.

## Padrão Típico de Uso
1. Execute `sdk_search` com sua intenção (ex: *"listar propriedades"*).
2. Abra o README do SDK retornado ou seu `docs_uri` para inspecionar APIs / modelos.
3. Use `docs_search` para conceitos de suporte (autenticação, webhooks).

## Notas
- Cada URI retornado pode ser aberto pelo agente para conteúdo completo.
- Sugestões aparecem automaticamente quando as consultas estão vazias.
