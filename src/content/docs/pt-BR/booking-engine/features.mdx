O motor de reservas está disponível em duas versões para atender às diferentes necessidades de um afiliado, uma empresa de gestão de propriedades e um hotel.

## Agência de viagens online

Para um afiliado, faz mais sentido usar a versão de agência de viagens online. O motivo é que o site é configurado para acomodar viajantes que desejam visualizar várias propriedades.

Você pode encontrar a página inicial desta aplicação abaixo.

<LinkButton href="https://ota.wink.travel/home" variant="secondary" icon="external" target="_blank">
  Wink OTA
</LinkButton>

## Motor de reservas pela internet

Para um hotel ou uma empresa de gestão de propriedades, faz mais sentido usar a versão do motor de reservas pela internet. Esse tipo de usuário está interessado em destacar uma única propriedade ou várias propriedades sob gestão.

Você pode encontrar a página inicial desta aplicação abaixo.

<LinkButton href="https://book.wink.travel/home?client-id=client-1&mp;configuration-id=d906a163-5cee-4198-a170-2bedf2407f5f" variant="secondary" icon="external" target="_blank">
  Wink IBE
</LinkButton>

## Configuração

Na maioria das vezes, não é necessário compartilhar diretamente nenhuma dessas aplicações. Criamos várias formas para você compartilhar o inventário de viagens, que você pode descobrir no [Wink Studio](/pt-BR/studio/what-is-studio) ou através do [WinkLinks](/pt-BR/link-manager/wink-links).

No entanto, se você quiser vincular manualmente a um desses sites, certifique-se de adicionar seu ID de cliente e ID de personalização à URL como parâmetros.

Você pode encontrar seu ID de cliente seguindo:

<Steps>
1. Faça login no [Wink Extranet](/pt-BR/extranet/what-is-extranet) ou no [Wink Studio](/pt-BR/studio/what-is-studio).
2. Clique no ícone do seu perfil no canto superior direito e clique em [Applications](/pt-BR/developers/applications).
3. Use a aplicação vinculada à conta Wink com a qual deseja trabalhar.
4. Copie o Client ID dessa aplicação e cole-o na URL como um parâmetro de requisição. Exemplo: ?client-id=MEU_CLIENT_ID
</Steps>

### Extranet

Você pode encontrar seu ID de personalização como propriedade seguindo:

<Steps>
1. Faça login no [Wink Extranet](/pt-BR/extranet/what-is-extranet).
2. Selecione a propriedade com a qual deseja trabalhar.
3. Na barra de navegação principal, clique em `Account > Branding`
4. Copie o Customization ID e cole-o na URL como um parâmetro de requisição. Exemplo: ?configuration-id=MEU_ID_DE_PERSONALIZAÇÃO
</Steps>

### Studio

Você pode encontrar seu ID de personalização como afiliado seguindo:

<Steps>
1. Faça login no [Wink Studio](/pt-BR/studio/what-is-studio).
2. Selecione a conta com a qual deseja trabalhar.
3. Na barra de navegação principal, clique em `Tools > Customization`
4. Copie o Customization ID da personalização que deseja usar e cole-o na URL como um parâmetro de requisição. Exemplo: ?configuration-id=MEU_ID_DE_PERSONALIZAÇÃO
</Steps>

## OTA

O que você terá é uma URL que se parece com isto:

https://ota.wink.travel?client-id=MEU_CLIENT_ID&configuration-id=MEU_ID_DE_PERSONALIZAÇÃO

Se você quiser ir diretamente para a página de um hotel, a URL seria algo assim:

https://ota.wink.travel/hotel/hotel-x?client-id=MEU_CLIENT_ID&configuration-id=MEU_ID_DE_PERSONALIZAÇÃO

:::note
Se você não incluir um ID de cliente e/ou de configuração, o Wink OTA usará uma configuração padrão.
:::

## IBE

Se você for uma empresa de gestão de propriedades, deseja vincular à página inicial do Wink IBE. A página inicial mostrará seu tema designado junto com as [propriedades que você representa](/pt-BR/studio/agency-representation).

O que você terá é uma URL que se parece com isto:

https://book.wink.travel?client-id=MEU_CLIENT_ID&configuration-id=MEU_ID_DE_PERSONALIZAÇÃO

Se você for uma propriedade, deseja vincular diretamente à página de destino da propriedade no Wink IBE.

Essa URL seria algo assim:

https://book.wink.travel/hotel/hotel-x?client-id=MEU_CLIENT_ID&configuration-id=MEU_ID_DE_PERSONALIZAÇÃO


:::note
Se você não incluir um ID de cliente e/ou de configuração, o Wink IBE NÃO será carregado. Esses parâmetros são obrigatórios.
:::

## Exemplo: página da propriedade

Cada propriedade no Wink recebe sua própria página de destino única em qualquer um dos sites. Todos os links externos direcionam os viajantes para uma dessas páginas de destino.

A página de destino contém todas as informações que um viajante precisa para fazer uma escolha informada sobre se hospedar ou não nessa propriedade.
