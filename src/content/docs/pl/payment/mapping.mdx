Jeśli jesteś integratorem, takim jak Wink, i chcesz zamapować zewnętrznych odbiorców płatności do wykorzystania w TripPay, możesz to zrobić na dwa sposoby:

* Mapowanie ręczne
* Korzystanie z[API płatności](/developers/apis/#payment-api)(*Widzieć `Account Mappings`*)

:::tip
Jeśli nie masz do czynienia tylko z kilkoma takimi samymi właściwościami, zalecamy użycie API do automatyzacji tego kroku.
:::

Poniżej przedstawiamy ręczny sposób mapowania zewnętrznego odbiorcy płatności.*Wykorzystuje wbudowaną funkcjonalność API.*

<Steps>
  1. Zaloguj się [Wypłata](https://pay.wink.travel).
  2. Wybierz swoje konto integratora.
  3. Trzask `Settings > Mappings` z głównego paska nawigacyjnego.
  4. Kliknij `Create mapping` przycisk. Zobaczysz formularz mapowania.
  5. **Konto lokalne** Zacznij wpisywać nazwę istniejącego TripPay, który chcesz zamapować.
  6. **Identyfikator zewnętrzny** Wklej swój zewnętrzny identyfikator dla tej jednostki, taki jaki istnieje w systemie zdalnym.
  7. Trzask `Save` aby kontynuować. Zostaniesz przekierowany z powrotem do listy mapowań.
</Steps>


Dzięki mapowaniu TripPay wie teraz, które konta należy uznać za beneficjentów rezerwacji, gdy odwołujesz się do nich za pomocą swoich zewnętrznych identyfikatorów.

## API

Programiści chcący zarządzać`Payments`, idź do[Deweloperzy > API > Płatności](/developers/apis/#payment-api).
