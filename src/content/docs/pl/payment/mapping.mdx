Jeśli jesteś integratorem, takim jak Wink, i chcesz zmapować swoich zewnętrznych odbiorców płatności do użytku z TripPay, możesz to zrobić na dwa sposoby:

- Mapowanie ręczne
- Korzystanie z [Payment API](/pl/developers/apis/#payment-api) (*Zobacz `Account Mappings`*)

:::tip
O ile nie masz do czynienia tylko z kilkoma tymi samymi nieruchomościami, zalecamy użycie API do automatyzacji tego kroku.
:::

Poniżej przeprowadzimy Cię przez ręczny sposób mapowania zewnętrznego odbiorcy płatności. *Pod spodem korzysta on z API.*

<Steps>
1. Zaloguj się do [TripPay](https://pay.wink.travel).
2. Wybierz swoje konto integratora.
3. Kliknij `Settings > Mappings` w głównym pasku nawigacyjnym.
4. Kliknij przycisk `Create mapping`. Pojawi się formularz mapowania.
5. **Local account** Zacznij wpisywać nazwę istniejącego konta TripPay, które chcesz zmapować.
6. **External identifier** Wklej swój zewnętrzny identyfikator tej jednostki, tak jak istnieje w zdalnym systemie.
7. Kliknij `Save`, aby kontynuować. Zostaniesz przekierowany z powrotem do listy mapowań.
</Steps>

Dzięki ustawionym mapowaniom TripPay wie teraz, które konta należy zasilić jako beneficjentów rezerwacji, gdy odwołujesz się do nich za pomocą swoich zewnętrznych identyfikatorów.

## API

Deweloperzy, którzy chcą zarządzać `Payments`, przejdź do [Developers > API > Payments](/pl/developers/apis/#payment-api).
