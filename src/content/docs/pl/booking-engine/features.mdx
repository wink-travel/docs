Silnik rezerwacji występuje w dwóch wersjach, aby wspierać różne potrzeby afilianta, firmy zarządzającej nieruchomościami oraz hotelu.

## Internetowa agencja turystyczna

Dla afilianta najbardziej sensowne jest korzystanie z wersji internetowej agencji turystycznej. Powodem jest to, że strona jest skonfigurowana tak, aby obsługiwać podróżnych chcących przeglądać wiele obiektów.

Poniżej znajdziesz stronę główną tej aplikacji.

<LinkButton href="https://ota.wink.travel/home" variant="secondary" icon="external" target="_blank">
  Wink OTA
</LinkButton>

## Silnik rezerwacji internetowej

Dla hotelu lub firmy zarządzającej nieruchomościami najbardziej sensowne jest korzystanie z wersji silnika rezerwacji internetowej. Tego typu użytkownik jest zainteresowany wyróżnieniem pojedynczego obiektu lub wielu obiektów pod zarządem.

Poniżej znajdziesz stronę główną tej aplikacji.

<LinkButton href="https://book.wink.travel/home?client-id=client-1&mp;configuration-id=d906a163-5cee-4198-a170-2bedf2407f5f" variant="secondary" icon="external" target="_blank">
  Wink IBE
</LinkButton>

## Konfiguracja

Zazwyczaj nie ma potrzeby bezpośredniego udostępniania którejkolwiek z tych aplikacji. Stworzyliśmy różne sposoby udostępniania oferty turystycznej, które możesz odkryć w [Wink Studio](/pl/studio/what-is-studio) lub za pomocą [WinkLinks](/pl/link-manager/wink-links).

Jeśli jednak chcesz ręcznie podlinkować do jednej z tych stron, upewnij się, że dodasz swój identyfikator klienta i identyfikator personalizacji do adresu URL jako parametry.

Swój identyfikator klienta znajdziesz poprzez:

<Steps>
1. Zaloguj się do [Wink Extranet](/pl/extranet/what-is-extranet) lub [Wink Studio](/pl/studio/what-is-studio).
2. Kliknij ikonę profilu w prawym górnym rogu i wybierz [Applications](/pl/developers/applications).
3. Użyj aplikacji powiązanej z kontem Wink, z którym chcesz pracować.
4. Skopiuj Client ID z tej aplikacji i wklej go w adres URL jako parametr zapytania. Przykład: ?client-id=MY_CLIENT_ID
</Steps>

### Extranet

Swój identyfikator personalizacji jako obiektu znajdziesz poprzez:

<Steps>
1. Zaloguj się do [Wink Extranet](/pl/extranet/what-is-extranet).
2. Wybierz obiekt, z którym chcesz pracować.
3. Z głównego paska nawigacji kliknij `Account > Branding`
4. Skopiuj Customization ID i wklej go w adres URL jako parametr zapytania. Przykład: ?configuration-id=MY_CUSTOMIZATION_ID
</Steps>

### Studio

Swój identyfikator personalizacji jako afiliant znajdziesz poprzez:

<Steps>
1. Zaloguj się do [Wink Studio](/pl/studio/what-is-studio).
2. Wybierz konto, z którym chcesz pracować.
3. Z głównego paska nawigacji kliknij `Tools > Customization`
4. Skopiuj Customization ID z personalizacji, której chcesz użyć, i wklej go w adres URL jako parametr zapytania. Przykład: ?configuration-id=MY_CUSTOMIZATION_ID
</Steps>

## OTA

Końcowy adres URL będzie wyglądał mniej więcej tak:

https://ota.wink.travel?client-id=MY_CLIENT_ID&configuration-id=MY_CUSTOMIZATION_ID

Jeśli chcesz przejść bezpośrednio do strony hotelu, adres URL będzie wyglądał mniej więcej tak:

https://ota.wink.travel/hotel/hotel-x?client-id=MY_CLIENT_ID&configuration-id=MY_CUSTOMIZATION_ID

:::note
Jeśli nie podasz identyfikatora klienta i/lub personalizacji, Wink OTA użyje domyślnej konfiguracji.
:::

## IBE

Jeśli jesteś firmą zarządzającą nieruchomościami, chcesz podlinkować do strony głównej Wink IBE. Strona główna pokaże Twój wyznaczony motyw wraz z [obiektami, które reprezentujesz](/pl/studio/agency-representation).

Końcowy adres URL będzie wyglądał mniej więcej tak:

https://book.wink.travel?client-id=MY_CLIENT_ID&configuration-id=MY_CUSTOMIZATION_ID

Jeśli jesteś obiektem, chcesz podlinkować bezpośrednio do strony docelowej obiektu Wink IBE.

Adres URL będzie wyglądał mniej więcej tak:

https://book.wink.travel/hotel/hotel-x?client-id=MY_CLIENT_ID&configuration-id=MY_CUSTOMIZATION_ID


:::note
Jeśli nie podasz identyfikatora klienta i/lub personalizacji, Wink IBE NIE załaduje się. Te parametry są wymagane.
:::
