Komponenty Wink to niestandardowe elementy działające tylko w przeglądarce — skrypty CDN muszą być ładowane po stronie klienta. Poniższe wzorce obejmują zarówno **App Router** (Next.js 13+), jak i starszy **Pages Router**.

/>

## Instalacja

```bash
npm install @wink/elements
```

## App Router (Next.js 13+)

Utwórz komponent klienta, który wywoła `load()` raz i wyrenderuj go w swoim głównym layoucie:

```tsx
// components/WinkLoader.tsx
'use client';
import { useEffect } from 'react';
import { load } from '@wink/elements';

export function WinkLoader() {
  useEffect(() => {
    load({ clientId: process.env.NEXT_PUBLIC_WINK_CLIENT_ID! });
  }, []);
  return null;
}
```

```tsx
// app/layout.tsx
import { WinkLoader } from '@/components/WinkLoader';

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body>
        <WinkLoader />
        {children}
      </body>
    </html>
  );
}
```

Używaj komponentów Wink w dowolnym **komponencie klienta**:

```tsx
// app/hotels/page.tsx
'use client';

export default function HotelsPage() {
  return (
    <main>
      <wink-content-loader layout="HOTEL" id="YOUR_LAYOUT_ID" />
    </main>
  );
}
```

<Aside type="caution" title="Komponenty serwerowe">
Tagi `<wink-*>` nie mogą być renderowane w komponentach serwerowych, ponieważ rejestr niestandardowych elementów działa tylko w przeglądarce. Zawsze dodawaj `'use client'` do każdego komponentu, który renderuje elementy Wink.
</Aside>

## Pages Router

Załaduj raz w `_app.tsx`:

```tsx
// pages/_app.tsx
import type { AppProps } from 'next/app';
import { useEffect } from 'react';
import { load } from '@wink/elements';

export default function App({ Component, pageProps }: AppProps) {
  useEffect(() => {
    load({ clientId: process.env.NEXT_PUBLIC_WINK_CLIENT_ID! });
  }, []);
  return <Component {...pageProps} />;
}
```

Następnie używaj komponentów Wink na dowolnej stronie:

```tsx
// pages/hotels.tsx
export default function HotelsPage() {
  return <wink-content-loader layout="HOTEL" id="YOUR_LAYOUT_ID" />;
}
```

## Zmienne środowiskowe

Dodaj do `.env.local`:

```
NEXT_PUBLIC_WINK_CLIENT_ID=your-client-id
```

Prefiks `NEXT_PUBLIC_` jest wymagany, aby zmienna była dostępna w pakiecie przeglądarki.
