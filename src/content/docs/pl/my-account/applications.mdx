Aplikacje dają Ci dostęp do wszystkich funkcji Wink jako deweloperowi. Możesz uzyskać dostęp do tych samych danych, a nawet rozszerzać funkcje Wink.
Możesz wybrać użycie serwera OAuth2 Wink do uwierzytelniania swojej strony podróżniczej tak samo, jak używasz Google lub Facebooka.
Dzięki temu możesz rozszerzyć funkcje Wink na własną stronę internetową lub aplikację.

:::note
Twoja pierwsza aplikacja jest już utworzona, gdy [tworzysz nowe konto](/pl/my-account/create-account). Używamy tej aplikacji w niektórych scenariuszach rezerwacji.

Nie możesz ponownie użyć tej aplikacji. Proszę twórz nowe aplikacje w miarę potrzeb.
:::

<Steps>
 1. Wybierz konto, z którym chcesz pracować, z [Konta](https://my.wink.travel).
 2. Kliknij `Akcje` na dole karty konta i wybierz `Zarządzaj kontem`.
 3. Kliknij zakładkę `Aplikacje` na kolejnej stronie.
 4. Ta sekcja wyświetli wszystkie Twoje aplikacje.
 5. Wybierz edycję istniejącej lub utwórz nową.
 </Steps>

## Utwórz aplikację

Ponieważ nie masz dostępu do `klucza sekretnego` aplikacji, którą dla Ciebie utworzyliśmy, musisz stworzyć własną.

:::caution
Musisz utworzyć konto afiliacyjne na Wink lub konto na TripPay, zanim stworzysz aplikację, ponieważ będziesz wiązać swoją aplikację z konkretnym kontem.
:::

### Typy klientów

Podczas tworzenia aplikacji wybierasz typ klienta, który określa sposób uwierzytelniania aplikacji:

- **SPA** (Single Page Application) — Dla aplikacji frontendowych działających w przeglądarce, które używają przepływu autoryzacji OAuth2 z kodem i PKCE. To najczęstszy typ dla aplikacji webowych.
- **MACHINE_2_MACHINE** — Dla integracji serwer-serwer, które uwierzytelniają się bezpośrednio za pomocą poświadczeń klienta. Używaj tego, gdy Twój backend musi wywoływać API Wink bez obecności użytkownika.

Oto kroki tworzenia aplikacji:

<Steps>
1. Na stronie aplikacji kliknij przycisk `Utwórz nową aplikację`.
2. ***Nazwa*** Nadaj aplikacji nazwę. *np. Cool App*
3. **Typ klienta** Wybierz `SPA` dla aplikacji webowych lub `MACHINE_2_MACHINE` dla integracji serwerowych.
4. **Nazwa podmiotu** Połącz aplikację z jednym z Twoich istniejących kont. *np. Cool Account*
5. **URI przekierowania** Wymagany jest co najmniej jeden URI przekierowania. Służy do weryfikacji domeny przekierowania po pomyślnym uwierzytelnieniu. *np. https://www.cool-site.com*
6. **URI przekierowania po wylogowaniu** *(Opcjonalne)* Adresy URL, na które użytkownicy mogą być przekierowani po wylogowaniu. Zapewnia to, że użytkownicy trafią na znaną stronę po zakończeniu sesji.
7. Kliknij przycisk `Zapisz`, aby kontynuować.
</Steps>

Zostaniesz przekierowany z powrotem do listy aplikacji. Zobaczysz swoją nową aplikację na liście i będziesz mieć dostęp do klucza sekretnego aplikacji tylko tym razem. Upewnij się, że go skopiowałeś i zapisałeś w bezpiecznym miejscu.

## Aktualizuj aplikację

Będziesz musiał zaktualizować aplikację, jeśli zmieni się któryś z URI przekierowania lub chcesz dodać lub usunąć URI.

Aby zaktualizować istniejącą aplikację, wykonaj następujące kroki:

<Steps>
1. Kliknij link `Akcje` przy aplikacji, którą chcesz zaktualizować.
2. Kliknij przycisk `Aktualizuj` w sekcji Akcje.
3. Wprowadź zmiany w aplikacji.
4. Kliknij przycisk `Zapisz`, aby kontynuować.
</Steps>

Zostaniesz przekierowany z powrotem do listy aplikacji.

## Usuń aplikację

:::caution
Prosimy, nie usuwaj aplikacji, którą automatycznie dla Ciebie utworzyliśmy, aby zapewnić działanie wszystkich funkcji dla Twojego konta.
:::

Jeśli nie potrzebujesz już aplikacji, możesz ją usunąć.

Aby usunąć aplikację, wykonaj następujące kroki:

<Steps>
1. Kliknij link `Akcje` przy aplikacji, którą chcesz usunąć.
2. Kliknij przycisk `Usuń` w sekcji Akcje.
3. Kliknij przycisk `OK`, aby potwierdzić usunięcie.
</Steps>

Zostaniesz przekierowany z powrotem do listy aplikacji.
