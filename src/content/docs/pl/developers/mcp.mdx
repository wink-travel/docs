## Przegląd
Hostowany serwer MCP Wink jest dostępny pod adresem:

```
https://docs.mcp.wink.travel/mcp
```

Wskaż kompatybilnych klientów AI (VS Code, Cursor, inne implementujące transport MCP HTTP) na ten punkt końcowy, aby mogli:
- Przeglądać dokumentację produktów i platformy (`wink://docs/...`)
- Przeglądać aktywne specyfikacje OpenAPI i dokumentację operacji (`wink://api/...`)
- Wykonywać lekkie wyszukiwanie semantyczne / słów kluczowych w dokumentacji i aktywnych API

## Transport
Użyj głównego strumieniowego punktu końcowego HTTP:

```
https://docs.mcp.wink.travel/mcp
```

Jeśli twój klient obsługuje tylko SSE, użyj:

```
https://docs.mcp.wink.travel/sse
```

### Claude Code
Oficjalna dokumentacja: [Połącz Claude Code z narzędziami przez MCP](https://code.claude.com/docs/en/mcp)

Anthropic zaleca zdalny transport HTTP dla hostowanych w chmurze serwerów MCP:
```bash
claude mcp add --transport http wink-docs https://docs.mcp.wink.travel/mcp
claude mcp list
```

W Claude Code wpisz `/mcp`, aby sprawdzić status serwera. Jeśli chcesz udostępnić serwer całemu repozytorium, dodaj `--scope project`, aby Claude Code zapisał konfigurację w `.mcp.json`.

### Codex
Oficjalna dokumentacja: [Szybki start MCP dla Codex](https://developers.openai.com/resources/docs-mcp)

OpenAI dokumentuje konfigurację MCP przez CLI Codex i zaznacza, że konfiguracja jest współdzielona z rozszerzeniem IDE:
```bash
codex mcp add wink-docs --url https://docs.mcp.wink.travel/mcp
codex mcp list
```

### Cursor
Dokumentacja MCP Cursor: https://cursor.com/docs/context/mcp
Konfiguracja: Dodaj zdalny serwer przez ustawienia MCP.
```json
{
  "mcpServers": {
    "wink-docs": {
      "url": "https://docs.mcp.wink.travel/mcp",
    }
  }
}
```

### VS Code
Oficjalna dokumentacja: https://code.visualstudio.com/docs/copilot/customization/mcp-servers
Minimalna zdalna konfiguracja (użytkownika lub workspace `mcp.json`):
```json
{
  "servers": {
    "wink-docs": {
      "type": "http",
      "url": "https://docs.mcp.wink.travel/mcp"
    }
  }
}
```

Wskazówki:
- Użyj palety poleceń: „MCP: List Servers”, aby potwierdzić połączenie i przeglądać narzędzia.
- Wybór narzędzi pozwala włączyć/wyłączyć `api_search` i `docs_search`.
- Dodaj kontekst: Widok czatu → Dodaj kontekst → Zasoby MCP, aby dołączyć strony dokumentacji.

## Podstawowe URI zasobów

| URI | Opis |
|-----|-------------|
| `wink://docs` | Indeks dokumentacji (grupowany według kategorii) |
| `wink://docs/{path}` | Konkretna artykuł dokumentacji (Markdown / MDX) |
| `wink://api` | Aktywny katalog OpenAPI odkryty z konfiguracji swagger Wink |
| `wink://api/{apiName}` | Zwięzłe podsumowanie API z autoryzacją, tagami, nagłówkami i przykładowymi operacjami |
| `wink://api/{apiName}/operations` | Pełna lista operacji pogrupowana według tagów |
| `wink://api/{apiName}/operation/{key}` | Szczegóły pojedynczej operacji (parametry, ciało żądania, odpowiedzi) |
| `wink://api/{apiName}/schema/{schemaName}` | Rozwiązany schemat komponentu z zagnieżdżonymi polami, typami i przykładami |
| `wink://api/{apiName}/operation/{key}/request` | Widok skoncentrowany na żądaniu z wymaganymi danymi i minimalnym przykładowym payloadem |
| `wink://api/{apiName}/operation/{key}/response/{status}` | Widok skoncentrowany na odpowiedzi dla konkretnego kodu statusu |
| `wink://api/{apiName}/operation/{key}/recipe` | Przepływ zorientowany na zadanie do poprawnego wywołania tej operacji |

## Narzędzia wyszukiwania (Szybkie odniesienie)
| Narzędzie | Cel | Typowe przykłady zapytań |
|------|---------|-----------------------|
| `api_search` | Znajdź aktywne API lub operację do integracji | `create affiliate`, `payment receipt`, `lookup inventory`, `invite manager` |
| `docs_search` | Znajdź dokumentację koncepcyjną / przewodniki | `authentication`, `webhooks`, `wordPress` |

Pozostaw `query` puste, aby otrzymać sugestie. Każdy wynik zawiera URI MCP, które możesz otworzyć.
Dla trafień w operacjach, `api_search` zwraca również powiązane URI żądań, odpowiedzi, schematów i przepisów, aby agenci mogli bezpośrednio przejść do najbardziej przydatnych podzasobów.

## Typowy wzorzec użycia
1. Uruchom `api_search` z twoim zamiarem (np. *"create affiliate"* lub *"payment receipt"*).
2. Otwórz zwrócone URI `wink://api/...` lub podążaj za powiązanymi polami `request_uri`, `response_uris`, `request_schema_uris`, `response_schema_uris` i `recipe_uri`.
3. Użyj `docs_search` dla wspierających koncepcji (autoryzacja, webhooks, workflowy).

## Umiejętności AI

Wstępnie zbudowane umiejętności agentów korzystają z tego serwera MCP jako źródła prawdy. Umiejętności dają twojemu agentowi kodującemu ustrukturyzowany przepływ pracy do budowania na bazie API Wink — zamiast zgadywać punkty końcowe, agent czyta na żywo kontrakty bezpośrednio z serwera MCP.

Zobacz [AI Skills](/pl/developers/ai-skills) dla dostępnych umiejętności i instrukcji instalacji.

## Uwagi
- Każde zwrócone URI może być otwarte przez agenta dla pełnej zawartości.
- Drzewo `wink://api` jest aktywne i podąża za aktualnym katalogiem OpenAPI zamiast generowanego indeksu repozytorium.
- Sugestie pojawiają się automatycznie, gdy zapytania są puste.
