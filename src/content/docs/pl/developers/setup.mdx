Ten artykuł pokazuje, jak szybko rozpocząć pracę jako deweloper i zacząć testować nasze API na Wink.

:::note
Testowanie powinno zawsze odbywać się w naszym [środowisku testowym](/getting-started/environments). Jest tam znacznie mniej danych, ale możesz testować dowolnie dużo, nie wpływając na środowisko produkcyjne.

Gdy będziesz gotowy do uruchomienia na żywo, zatwierdzimy Twoje konto i aplikację, a Ty będziesz mógł uruchomić je w środowisku produkcyjnym.
:::

Wykonaj następujące kroki:

<Steps>
1. [Zarejestruj się](/user/register) jako użytkownik na Wink
2. Utwórz [konto agenta turystycznego](/travel-agent/what-is-agent) lub [konto afiliacyjne](/studio/what-is-studio).
3. [Utwórz aplikację](/developer/applications), przypisując ją do konta utworzonego w kroku 2.
4. Po utworzeniu `Application` zapisz client-id i client secret w bezpiecznym miejscu.
</Steps>

## CDN

W Wink korzystamy z Cloudinary do wszystkich naszych zasobów obrazów i wideo. Gdy uzyskasz dostęp do naszej oferty turystycznej, będziesz mieć pełny dostęp do wszystkich zaawansowanych funkcji Cloudinary.

Nasza wartość `cloud_name` to `traveliko`.

### Cloudinary

- [Pierwsze kroki](https://cloudinary.com/documentation/dev_kickstart)
- [SDK](https://cloudinary.com/documentation/cloudinary_sdks)
- [Cloudinary na GitHub](https://github.com/cloudinary)
-	[Samouczki wideo](https://cloudinary.com/documentation/cloudinary_sdks_tutorials)

### Przykład

Dla każdego typu oferty turystycznej, którą obsługujemy, istnieje zagnieżdżona tablica JSON o nazwie `multimedias`. JSON dla wpisu multimedialnego wygląda tak:

```json {3}
{
  "multimediaIdentifier": "multimedia-1",
  "identifier": "partners/radisson_blu_logo",
  "type": "IMAGE",
  "width": "1024",
  "height": "768",
  "category": "1",
  "descriptions": [
    {
      "name": "Lobby",
      "description": "Zdjęcie lobby",
      "language": "en"
    }
  ],
  "lifestyleType": "LIFESTYLE_BUSINESS",
  "attribution": [
    {
      "url": "https://www.hilton.com",
      "name": "Hilton"
    }
  ]
}
```

Podświetlona linia powyżej pokazuje unikalny identyfikator Cloudinary `partners/radisson_blu_logo`. To wszystko, czego potrzebujesz, aby pobrać obraz w dowolnym formacie, jakości i rozmiarze.

## Testowanie

Przejdź do sekcji naszych [API](/developer/apis) i wybierz API, z którym chcesz pracować.

### Dokumentacja API

Nasza strona dokumentacji jest tak skonfigurowana, że możesz wykonywać polecenia API bezpośrednio na stronie.

W tym przykładzie chcesz użyć wywołania API `Search Lookups (OAuth2)`.

:::tip[Pro Tip]
Dla wygody zalecamy integratorom korzystanie z punktów końcowych oznaczonych jako `OAuth2`. Te punkty końcowe rozpoznają wywołujących API i już wiedzą, na jakim koncie jesteś, więc nie musisz podawać `companyIdentifier` w adresach URL.
:::

<Steps>
1. Użyj [Lookup API](https://developer.wink.travel/lookup), aby pobrać destynacje lub hotele.
2. W lewym pasku nawigacyjnym kliknij `Authentication`.
3. Wprowadź swój client-id i client secret, a następnie kliknij przycisk `Get TOKEN`.
4. Jeśli Twoje dane uwierzytelniające są poprawne, zobaczysz, że Twój klucz API został zastosowany.
5. Teraz możesz testować punkt końcowy lookup.
</Steps>

### Postman
To samo możesz zrobić za pomocą Postmana.

<Steps>
1. Pobierz [Postman](https://www.postman.com/downloads).
2. Kliknij `Download OpenAPI spec` na górze strony [Lookup API](https://developer.wink.travel/lookup).
3. Zaimportuj to do Postmana.
4. Jeśli zaimportowałeś to jako nową kolekcję o nazwie `Wink API`, kliknij poziome wielokropki (3 kropki) obok nazwy kolekcji i wybierz `Edit`.
5. W zakładce `Authorization` ustaw `Auth type` na OAuth2.
6. Ustaw pole `Add auth data to` na `Request headers`.
7. Ustaw pole `Header prefix` na `Bearer`.
8. Ustaw pole `Token Name` na dowolną nazwę, pod którą chcesz zapisać token w Postmanie, np. Wink Staging.
9. Ustaw pole `Grant type` na `Client Credentials`.
10. Ustaw pole `Access token URL` na https://staging-iam.wink.travel/oauth2/token.
11. Ustaw pole `Client ID` na Client ID wygenerowany przez Twoją aplikację.
12. Ustaw pole `Client Secret` na Client Secret wygenerowany przez Twoją aplikację.
13. Ustaw pole `Scope` na `inventory.read inventory.write inventory.remove`. To wszystkie zakresy, których potrzebujesz dla Wink.
14. Ustaw pole `Client Authentication` na `Send as Basic Auth header`.
15. Kliknij przycisk `Get New Access Token`, aby pobrać token i zapisać go.
16. Dla wszystkich wywołań API w kolekcji Wink API upewnij się, że zakładka uwierzytelniania mówi `Inherit auth from parent`, aby token był dołączany do każdego wywołania.
</Steps>

:::tip[Pro Tip]
Zaleca się, aby domenę serwera, client ID i client secret ustawić jako zmienne środowiskowe. Dzięki temu łatwo przełączysz się między środowiskami testowym i produkcyjnym oraz ponownie wykorzystasz wszystkie utworzone operacje.
:::
