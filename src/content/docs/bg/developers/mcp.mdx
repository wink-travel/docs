## Преглед
Хостваният Wink MCP сървър е достъпен на:

```
https://docs.mcp.wink.travel/mcp
```

Насочете съвместими AI клиенти (VS Code, Cursor, други, които имплементират MCP HTTP транспорт) към тази крайна точка, за да им позволите:
- Да разглеждат продуктова и платформа документация (`wink://docs/...`)
- Да разглеждат живи OpenAPI спецификации и документация на операции (`wink://api/...`)
- Да извършват лека семантична / ключова дума търсене в документацията и живите API-та

## Транспорт
Използвайте основната HTTP крайна точка за стрийминг:

```
https://docs.mcp.wink.travel/mcp
```

Ако клиентът ви поддържа само SSE, използвайте:

```
https://docs.mcp.wink.travel/sse
```

### Claude Code
Официална документация: [Свържете Claude Code с инструменти чрез MCP](https://code.claude.com/docs/en/mcp)

Anthropic препоръчва отдалечения HTTP транспорт за облачно хоствани MCP сървъри:
```bash
claude mcp add --transport http wink-docs https://docs.mcp.wink.travel/mcp
claude mcp list
```

В Claude Code изпълнете `/mcp`, за да проверите статуса на сървъра. Ако искате да споделите сървъра с цялото хранилище, добавете `--scope project`, за да запише конфигурацията в `.mcp.json`.

### Codex
Официална документация: [Docs MCP quickstart за Codex](https://developers.openai.com/resources/docs-mcp)

OpenAI документира настройката на MCP чрез Codex CLI и отбелязва, че конфигурацията се споделя с IDE разширението:
```bash
codex mcp add wink-docs --url https://docs.mcp.wink.travel/mcp
codex mcp list
```

### Cursor
Cursor MCP документация: https://cursor.com/docs/context/mcp
Настройка: Добавете отдалечен сървър чрез MCP настройките.
```json
{
  "mcpServers": {
    "wink-docs": {
      "url": "https://docs.mcp.wink.travel/mcp",
    }
  }
}
```

### VS Code
Официална документация: https://code.visualstudio.com/docs/copilot/customization/mcp-servers
Минимална отдалечена конфигурация (потребителски или работен `mcp.json`):
```json
{
  "servers": {
    "wink-docs": {
      "type": "http",
      "url": "https://docs.mcp.wink.travel/mcp"
    }
  }
}
```

Съвети:
- Използвайте Command Palette: “MCP: List Servers”, за да потвърдите връзката и да разгледате инструментите.
- Инструментът за избор ви позволява да активирате/деактивирате `api_search` и `docs_search`.
- Добавете контекст: Chat view → Add Context → MCP Resources, за да прикачите страници с документация.

## Основни URI за ресурси

| URI | Описание |
|-----|-------------|
| `wink://docs` | Индекс на документацията (групиран по категории) |
| `wink://docs/{path}` | Конкретна статия от документацията (Markdown / MDX) |
| `wink://api` | Жив каталог OpenAPI, открит от swagger конфигурацията на Wink |
| `wink://api/{apiName}` | Компактен API преглед с автентикация, тагове, заглавки и примерни операции |
| `wink://api/{apiName}/operations` | Пълен списък с операции, групирани по таг |
| `wink://api/{apiName}/operation/{key}` | Подробности за отделна операция (параметри, тяло на заявка, отговори) |
| `wink://api/{apiName}/schema/{schemaName}` | Разрешена схема на компонент с вложени полета, типове и примери |
| `wink://api/{apiName}/operation/{key}/request` | Преглед, фокусиран върху заявката с необходими входни данни и минимален примерен payload |
| `wink://api/{apiName}/operation/{key}/response/{status}` | Преглед, фокусиран върху отговора за конкретен статус код |
| `wink://api/{apiName}/operation/{key}/recipe` | Задача-ориентиран поток за правилно извикване на операцията |

## Инструменти за търсене (Бърза справка)
| Инструмент | Цел | Типични примери за заявки |
|------|---------|-----------------------|
| `api_search` | Намиране на жив API или операция за интеграция | `create affiliate`, `payment receipt`, `lookup inventory`, `invite manager` |
| `docs_search` | Намиране на концептуална документация / ръководства | `authentication`, `webhooks`, `wordPress` |

Оставете `query` празно, за да получите предложения. Всеки резултат включва MCP URI, който можете да отворите.
При попадения в операции, `api_search` връща и свързани URI-та за заявка, отговор, схема и рецепта, за да може агентите директно да преминават към най-полезния подресурс.

## Типичен модел на използване
1. Стартирайте `api_search` с вашето намерение (например *"create affiliate"* или *"payment receipt"*).
2. Отворете върнатия `wink://api/...` URI или следвайте свързаните му полета `request_uri`, `response_uris`, `request_schema_uris`, `response_schema_uris` и `recipe_uri`.
3. Използвайте `docs_search` за допълнителни концепции (автентикация, webhooks, работни потоци).

## AI умения

Предварително изградени умения на агенти използват този MCP сървър като източник на истината. Уменията дават на вашия кодиращ агент структуриран работен поток за изграждане върху Wink API — вместо да гадае крайни точки, агентът чете живите договори директно от MCP сървъра.

Вижте [AI Skills](/bg/developers/ai-skills) за налични умения и инструкции за инсталиране.

## Бележки
- Всеки върнат URI може да бъде отворен от агента за пълно съдържание.
- Дървото `wink://api` е живо и следва текущия upstream OpenAPI каталог, вместо генериран индекс на хранилището.
- Предложенията се появяват автоматично, когато заявките са празни.
