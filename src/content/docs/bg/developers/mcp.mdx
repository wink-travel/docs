## Преглед
Хостваният Wink MCP сървър е достъпен на:

```
https://docs.mcp.wink.travel/mcp
```

Насочете съвместими AI клиенти (VS Code, Cursor, други, които имплементират MCP HTTP транспорт) към тази крайна точка, за да им позволите:
- Преглед на продуктова и платформа документация (`wink://docs/...`)
- Откриване на SDK и разглеждане на API класове / модели (`wink://sdk/...` & `wink://sdk/.../docs`)
- Извършване на лека семантична / ключова дума търсене в документацията и резюмета на SDK

## Транспорт
Използвайте основната HTTP крайна точка с поток:

```
https://docs.mcp.wink.travel/mcp
```

Ако клиентът ви поддържа само SSE, използвайте:

```
https://docs.mcp.wink.travel/sse
```

### Cursor
Cursor MCP документация: https://cursor.com/docs/context/mcp
Настройка: Добавете отдалечен сървър чрез MCP настройките.
```json
{
  "mcpServers": {
    "wink-docs": {
      "url": "https://docs.mcp.wink.travel/mcp",
    }
  }
}
```

### VS Code
Официална документация: https://code.visualstudio.com/docs/copilot/customization/mcp-servers
Минимална отдалечена конфигурация (потребителски или работен `mcp.json`):
```json
{
  "servers": {
    "wink-docs": {
      "type": "http",
      "url": "https://docs.mcp.wink.travel/mcp"
    }
  }
}
```

Съвети:
- Използвайте Command Palette: “MCP: List Servers” за потвърждение на връзката и разглеждане на инструментите.
- Инструментът за избор ви позволява да активирате/деактивирате `sdk_search` и `docs_search`.
- Добавете контекст: Преглед на чат → Add Context → MCP Resources за прикачване на страници с документация.

## Основни URI за ресурси

| URI | Описание |
|-----|-------------|
| `wink://docs` | Индекс на документацията (групиран по категории) |
| `wink://docs/{path}` | Конкретна статия от документацията (Markdown / MDX) |
| `wink://sdk` | Преглед на SDK (всеки SDK + брой документи) |
| `wink://sdk/{sdkPath}` | README на SDK / папка с документация на модул / конкретен документ на модул |

### Документация на модули
Отварянето на папка с документация на SDK (напр. `wink://sdk/python/wink-sdk-booking/docs`) връща групирано:
- **API класове** (брой методи + примерни имена на методи)
- **Модели и типове** (структури от данни, използвани от тези API-та)

## Инструменти за търсене (Бърза справка)
| Инструмент | Цел | Примери за типични заявки |
|------------|-----|---------------------------|
| `sdk_search` | Намиране на кой SDK / API клас да се използва | `cancel booking`, `list properties`, `affiliate inventory` |
| `docs_search` | Намиране на концептуална документация / ръководства | `authentication`, `webhooks`, `wordPress` |

Оставете `query` празно, за да получите предложения. Всеки резултат включва MCP URI, който можете да отворите.

## Типичен модел на използване
1. Стартирайте `sdk_search` с вашето намерение (напр. *"list properties"*).
2. Отворете върнатия README на SDK или неговия `docs_uri`, за да разгледате API-та / модели.
3. Използвайте `docs_search` за допълнителни концепции (автентикация, webhooks).

## Забележки
- Всеки върнат URI може да бъде отворен от агента за пълен достъп до съдържанието.
- Предложенията се появяват автоматично, когато заявките са празни.
