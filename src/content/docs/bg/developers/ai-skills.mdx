AI уменията са преносими пакети с инструкции, които учат програмиращите агенти как да работят с Wink. Вместо да разчитат на тренировъчни данни или остаряла документация, всяко умение насочва агента да използва [Wink MCP сървъра](/bg/developers/mcp) като жив източник на истината за API договори и платформа документация.

Уменията работят с всеки агент клиент, който ги поддържа — Claude Code, Codex, Cursor и други.

## Налични умения

### wink-api-builder

Имплементирайте, преглеждайте или отстранявайте грешки в интеграции с Wink API. Умението води агента през структуриран работен процес:

1. Търсене на правилната API операция чрез `api_search`
2. Отваряне на живите договори за заявки/отговори от MCP сървъра
3. Извличане на настройките за удостоверяване от документацията
4. Писане на код спрямо проверения договор
5. Отбелязване на несъответствия между съществуващия код и текущото API

Полезно за:
- Създаване на нови клиентски методи за всеки Wink API
- Отстраняване на несъответствия в договорите (например заявка връщаща HTTP 400)
- Преглед на съществуващи интеграции спрямо живата спецификация

## Инсталация

Копирайте папката с умението в директорията за умения на вашия агент.

:::tip
Wink MCP сървърът трябва да е свързан в същата среда. Вижте [MCP Server](/bg/developers/mcp) за инструкции за настройка.
:::

**Claude Code**

```bash
# локално в репото
cp -r skills/wink-api-builder .claude/skills/

# или на ниво потребител
cp -r skills/wink-api-builder ~/.claude/skills/
```

Извикване с:
```
/wink-api-builder add a TypeScript client for the Wink affiliate create flow
```

**Codex**

```bash
# локално в репото
cp -r skills/wink-api-builder .agents/skills/

# или на ниво потребител
cp -r skills/wink-api-builder $CODEX_HOME/skills/
```

Извикване с:
```
Use $wink-api-builder to add a TypeScript client for the Wink affiliate create flow.
```

**Други агенти**

Поставете папката с умението там, откъдето вашият агент зарежда умения. Входната точка е `SKILL.md`.
