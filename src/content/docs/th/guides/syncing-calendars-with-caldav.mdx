หากคุณจัดการความพร้อมใช้งานของที่พักผ่านปฏิทินภายนอก การซิงค์ CalDAV จะช่วยให้ปฏิทิน Wink ของคุณอัปเดตโดยอัตโนมัติ การจองและวันที่ถูกบล็อกจะไหลระหว่าง Wink กับแอปปฏิทินของคุณ ช่วยให้คุณหลีกเลี่ยงการจองซ้ำซ้อน

## แอปปฏิทินที่รองรับ

CalDAV ทำงานร่วมกับแอปปฏิทินส่วนใหญ่ที่รองรับโปรโตคอล CalDAV:

- **Google Calendar**
- **Apple Calendar (iCal)**
- **Microsoft Outlook** (ที่รองรับ CalDAV)
- **ไคลเอนต์ที่รองรับ CalDAV ใดๆ**

## การตั้งค่า CalDAV

<Steps>
1. ไปที่ https://extranet.wink.travel และเลือกที่พักของคุณ
2. คลิกที่ `Distribution > Bookings` จากแถบนำทางหลัก
3. ค้นหาส่วน CalDAV และคัดลอก URL CalDAV ของคุณ
4. เปิดแอปปฏิทินของคุณและเพิ่มการสมัครสมาชิกปฏิทินใหม่โดยใช้ URL ที่คัดลอกมา
5. ปฏิทินภายนอกของคุณจะแสดงการจองและวันที่ถูกบล็อกของ Wink ตอนนี้
</Steps>

## วิธีการทำงานของการซิงค์

- **Wink → ปฏิทิน** — เมื่อมีการจองใน Wink จะปรากฏเป็นเหตุการณ์ที่ถูกบล็อกในปฏิทินภายนอกของคุณ
- **ปฏิทิน → Wink** — เมื่อคุณบล็อกวันที่ในปฏิทินภายนอก วันที่เหล่านั้นจะกลายเป็นวันที่ไม่ว่างใน Wink
- **ช่วงเวลาการซิงค์** — ปฏิทินภายนอกมักจะตรวจสอบการอัปเดตทุก 15–60 นาที ขึ้นอยู่กับแอป

:::tip
หากคุณต้องการซิงค์เกือบจะทันที ให้รีเฟรชการสมัครสมาชิกปฏิทินในแอปของคุณด้วยตนเองหลังจากทำการเปลี่ยนแปลง ตัวอย่างเช่น Google Calendar อาจใช้เวลาถึง 12 ชั่วโมงในการรีเฟรชอัตโนมัติ แต่คุณสามารถบังคับให้ซิงค์ได้
:::

## การหลีกเลี่ยงการจองซ้ำซ้อน

การซิงค์ CalDAV มีประโยชน์อย่างยิ่งหากคุณลงประกาศที่พักของคุณบนหลายแพลตฟอร์ม โดยการนำเข้าปฏิทินของแพลตฟอร์มอื่นๆ ของคุณเข้าสู่แอปปฏิทินและซิงค์แอปนั้นกับ Wink คุณจะสร้างแหล่งข้อมูลเดียวสำหรับความพร้อมใช้งาน

## การแก้ไขปัญหา

- **เหตุการณ์ไม่ปรากฏ** — ตรวจสอบให้แน่ใจว่า URL CalDAV ถูกต้องและแอปปฏิทินของคุณรองรับ CalDAV
- **ข้อมูลล้าสมัย** — บังคับรีเฟรชในแอปปฏิทินของคุณ แอปบางตัวเก็บแคชอย่างเข้มงวด
- **ความขัดแย้ง** — หากวันที่ถูกบล็อกในปฏิทินภายนอกแต่แสดงว่าว่างใน Wink ให้ตรวจสอบช่วงเวลาการซิงค์และตรวจสอบให้แน่ใจว่าการสมัครสมาชิกปฏิทินยังคงใช้งานได้
