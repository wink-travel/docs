โดยการเชื่อมต่อทรัพย์สิน Google Analytics 4 ของคุณกับ Wink เหตุการณ์การจองของคุณ — ตั้งแต่การดูห้องจนถึงการซื้อที่เสร็จสมบูรณ์ — จะถูกส่งต่อไปยังแดชบอร์ด GA4 ของคุณโดยอัตโนมัติพร้อมกับการวิเคราะห์ของ Wink เอง โดยไม่ต้องเขียนโค้ดหรือการตั้งค่าทางเทคนิคใดๆ จากฝั่งคุณ

<Aside type="tip">
สิ่งที่คุณต้องมีคือ GA4 Measurement ID เมื่อกรอกในพอร์ทัล Wink การวิเคราะห์ของคุณจะเริ่มทำงานทันที
</Aside>

---

## วิธีการทำงาน

```
คุณกรอก G-XXXXXXXXXX ในพอร์ทัล Wink
       ↓
Wink ส่งต่อเหตุการณ์การจองของคุณไปยังทรัพย์สิน GA4 ของคุณ
       ↓
คุณเข้าสู่ระบบที่ analytics.google.com และดูการจองของคุณ
```

---

## สิ่งที่คุณต้องทำ

<Steps>

1. **สร้างทรัพย์สิน GA4**

   ไปที่ [analytics.google.com](https://analytics.google.com): Admin → Create Property ตั้งค่าโซนเวลาและสกุลเงินให้ตรงกับตลาดของคุณ

2. **สร้าง Web Data Stream**

   Admin → Data Streams → Add Stream → Web กรอก `wink.travel` เป็น URL ระบบจะสร้าง **Measurement ID** (`G-XXXXXXXXXX`) ให้คุณ คัดลอกไว้

3. **กรอก Measurement ID ในพอร์ทัล Wink**

   เข้าสู่ระบบที่ [my.wink.travel](https://my.wink.travel) เลือกบัญชีของคุณ แล้วไปที่แท็บ **Customizations** วาง Measurement ID และบันทึก เพียงเท่านี้การผสานรวม GA4 ของคุณก็พร้อมใช้งาน

4. **ตั้งค่า `purchase` เป็นเหตุการณ์สำคัญ**

   กลับไปที่ทรัพย์สิน GA4 ของคุณ: Admin → Events → หา `purchase` → สลับเปิด **Mark as key event** เหตุการณ์นี้จะทำให้การจองที่เสร็จสมบูรณ์แสดงในรายงาน Conversions

5. **ดูข้อมูลของคุณ**

   เข้าสู่ระบบที่ [analytics.google.com](https://analytics.google.com) คุณจะเห็นการจอง รายได้ และเส้นทางของแขก — ทั้งหมดถูกติดตามโดยอัตโนมัติผ่าน Wink

</Steps>

<Aside type="note">
คุณไม่จำเป็นต้องติดตั้งโค้ดติดตามใดๆ บนเว็บไซต์ของคุณสำหรับการผสานรวมนี้ Wink จะจัดการทุกอย่าง Measurement ID ของคุณเป็นเพียงจุดหมายปลายทางที่ข้อมูลการจองถูกส่งไปเท่านั้น
</Aside>

## สิ่งที่คุณสามารถติดตามได้

เมื่อเชื่อมต่อแล้ว ทรัพย์สิน GA4 ของคุณจะได้รับชุดเหตุการณ์ Enhanced E-commerce ครบถ้วนสำหรับทรัพย์สินของคุณ:

### การค้นพบ

| เหตุการณ์ | ความหมาย |
|---|---|
| `view_item_list` | แขกเห็นทรัพย์สินของคุณในผลการค้นหาหรือรายการหมวดหมู่ |
| `select_item` | แขกคลิกที่ทรัพย์สินของคุณจากรายการ |
| `view_item` | แขกดูหน้ารายละเอียดโรงแรมของคุณ |

### การพิจารณา

| เหตุการณ์ | ความหมาย |
|---|---|
| `view_promotion` | แขกเห็นข้อเสนอโปรโมชั่นสำหรับทรัพย์สินของคุณ |
| `select_promotion` | แขกคลิกที่ข้อเสนอโปรโมชั่น |
| `add_to_wishlist` | แขกบันทึกทรัพย์สินของคุณลงในรายการโปรด |

### ช่องทางการจอง

| เหตุการณ์ | ความหมาย |
|---|---|
| `add_to_cart` | แขกเลือกห้องหรือแพ็กเกจ |
| `view_cart` | แขกดูสรุปการจองของตน |
| `remove_from_cart` | แขกลบห้องหรือแพ็กเกจออก |
| `begin_checkout` | แขกเริ่มกระบวนการชำระเงิน |
| `add_payment_info` | แขกกรอกรายละเอียดการชำระเงิน |
| `add_shipping_info` | แขกกรอกรายละเอียดผู้เข้าพัก |
| `purchase` | แขกทำการจองเสร็จสมบูรณ์ |
| `refund` | การจองถูกยกเลิกและคืนเงิน |

เหตุการณ์ทั้งหมดรวมมิติที่เกี่ยวข้อง เช่น รายได้ สกุลเงิน ประเภทห้อง และมูลค่าการจอง — ให้ภาพรวมเส้นทางของแขกตั้งแต่การรับรู้ครั้งแรกจนถึงการจองที่เสร็จสมบูรณ์ หากต้องการปลดล็อกมิติระดับทรัพย์สินและห้องเพิ่มเติม เช่น `city_name`, `cancellation_policy`, `bed_type` และอื่นๆ ดูที่ [Custom Dimensions in GA4](/th/guides/gtm-dual-tagging#custom-dimensions-in-ga4)

## การติดตามข้ามโดเมน (ขั้นสูง)

การผสานรวมมาตรฐานข้างต้นติดตามทุกอย่างที่เกิดขึ้นภายใน Wink อย่างไรก็ตาม หากคุณต้องการติดตามแขกตั้งแต่เว็บไซต์ของคุณจนถึงการจองที่เสร็จสมบูรณ์บน Wink — และระบุแหล่งที่มาของการเข้าชมเดิมอย่างถูกต้อง (Google Ads, แคมเปญอีเมล, การค้นหาแบบออร์แกนิก ฯลฯ) — จะต้องมีขั้นตอนเพิ่มเติมในแต่ละฝั่ง

<Aside type="caution">
ส่วนนี้ใช้ได้เฉพาะหากคุณติดตั้ง GA4 บนเว็บไซต์ของคุณเองเท่านั้น หากคุณไม่ได้ใช้ GA4 บนเว็บไซต์ของคุณ การผสานรวมมาตรฐานข้างต้นก็เพียงพอแล้ว
</Aside>

### ทำไมเรื่องนี้จึงสำคัญ

หากไม่มีการติดตามข้ามโดเมน แขกที่เข้าชมเว็บไซต์ของคุณก่อนแล้วจึงจองบน Wink จะปรากฏใน GA4 ของคุณเป็นเซสชัน **Direct** โดยไม่มีการเชื่อมโยงกับการเข้าชมก่อนหน้า การคลิก Google Ads แคมเปญอีเมล หรือการเข้าชมแบบออร์แกนิกที่นำไปสู่การจองจะไม่ได้รับเครดิตใดๆ

เมื่อกำหนดค่าการติดตามข้ามโดเมน คุณจะเห็นเส้นทางของแขกอย่างครบถ้วน:

```
google.com (โฆษณาชำระเงิน)
       ↓
เว็บไซต์ของคุณ (แขกเรียกดูห้อง)
       ↓
wink.travel (แขกเลือกห้องและจอง)
       ↓
การซื้อได้รับการระบุแหล่งที่มาถูกต้องจากการคลิก Google Ads เดิม
```

### สิ่งที่คุณต้องทำ

กำหนดค่าทรัพย์สิน GA4 ของคุณให้รู้จัก `wink.travel` เป็นส่วนหนึ่งของโดเมนของคุณเอง ไม่ใช่การอ้างอิงภายนอก ซึ่งจะทำให้ GA4 รวมเซสชันทั้งสองเป็นเส้นทางแขกที่ต่อเนื่องกัน

<Steps>
1. GA4 Admin → Data Streams → เลือกสตรีมของคุณ
2. Configure tag settings → **Configure your domains**
3. เพิ่ม `wink.travel` เป็นโดเมนที่เชื่อมโยง
4. บันทึก
</Steps>

### สิ่งที่ Wink ต้องทำ

ติดต่อ [Wink Support](https://my.wink.travel) และแจ้งโดเมนเว็บไซต์ของคุณ (เช่น `yourhotel.com`) Wink จะเพิ่มโดเมนของคุณในรายการยกเว้นการอ้างอิงในทรัพย์สิน GA4 ของ Wink เพื่อป้องกันไม่ให้เว็บไซต์ของคุณถูกบันทึกเป็นแหล่งอ้างอิง ซึ่งจะทำให้การระบุแหล่งที่มาของแขกถูกเขียนทับและเริ่มเซสชันใหม่

<Aside type="tip">
การผสานรวมมาตรฐานข้างต้นเพียงพอสำหรับทรัพย์สินส่วนใหญ่และให้คุณเห็นข้อมูลการจองและรายได้บน Wink ได้ทันที การติดตามข้ามโดเมนเป็นตัวเลือกขั้นสูงที่ควรตั้งค่าหลังจากที่คุณคุ้นเคยกับข้อมูล GA4 และต้องการข้อมูลเชิงลึกเกี่ยวกับการระบุแหล่งที่มาที่ลึกขึ้น
</Aside>
