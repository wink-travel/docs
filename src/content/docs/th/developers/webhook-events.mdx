หน้านี้แสดงรายการเหตุการณ์ webhook ทั้งหมดที่คุณสามารถสมัครรับบน Wink สำหรับคำแนะนำการตั้งค่า ดูที่ [คู่มือการรวม Webhook](/th/guides/webhook-integration) หรือ [หน้าการจัดการ Webhooks](/th/my-account/webhooks)

## วิธีการสมัครรับ

1. ไปที่ **Applications > Webhooks** ในพอร์ทัลของคุณ
2. สร้าง webhook และเลือกเหตุการณ์ที่คุณต้องการรับ
3. Wink จะส่ง HTTP POST ไปยัง endpoint ของคุณทุกครั้งที่เกิดเหตุการณ์นั้น

สำหรับขั้นตอนการตั้งค่าอย่างละเอียดและแนวทางปฏิบัติที่ดีที่สุด ดูที่ [Webhook Integration](/th/guides/webhook-integration)

## เหตุการณ์บัญชี

เหตุการณ์ที่เกี่ยวข้องกับวงจรชีวิตและการตั้งค่าบัญชี TripPay ของคุณ

| เหตุการณ์ | คีย์ | คำอธิบาย |
|---|---|---|
| สร้างบัญชี | `account.create` | มีการสร้างบัญชีใหม่ |
| บัญชีอยู่ระหว่างตรวจสอบ | `account.under_review` | บัญชีกำลังรอตรวจสอบ |
| บัญชีได้รับการอัปเดต | `account.updated` | รายละเอียดบัญชีมีการเปลี่ยนแปลง |
| สถานะบัญชีได้รับการอัปเดต | `account.status.updated` | สถานะบัญชีเปลี่ยนแปลง (เช่น อนุมัติ ระงับ) |
| ที่อยู่บัญชีได้รับการอัปเดต | `account.address.updated` | ที่อยู่บัญชีถูกแก้ไข |
| รายละเอียดบัญชีได้รับการอัปเดต | `account.details.updated` | ฟิลด์รายละเอียดบัญชีมีการเปลี่ยนแปลง |
| ข้อมูลบัญชีได้รับการอัปเดต | `account.information.update` | ข้อมูลทั่วไปของบัญชีได้รับการอัปเดต |
| ผู้จัดการบัญชีได้รับการอัปเดต | `account.managers.update` | รายชื่อผู้จัดการบัญชีมีการเปลี่ยนแปลง |
| รูปโปรไฟล์ได้รับการอัปเดต | `account.profile_picture.updated` | รูปโปรไฟล์บัญชีถูกเปลี่ยนแปลง |
| ตัวแทนได้รับการอัปเดต | `account.agent.updated` | ข้อมูลตัวแทนในบัญชีถูกแก้ไข |
| ชื่อ URL ได้รับการอัปเดต | `account.url-name.update` | URL ส่วนตัวของบัญชีเปลี่ยนแปลง |
| รหัสประจำตัวเฉพาะได้รับการอัปเดต | `account.unique-id.update` | ตัวระบุเฉพาะของบัญชีถูกแก้ไข |

### งานบัญชี

| เหตุการณ์ | คีย์ | คำอธิบาย |
|---|---|---|
| สร้างงาน | `account.task.create` | มีการเพิ่มงานใหม่ในบัญชี |
| งานได้รับการอัปเดต | `account.task.update` | งานในบัญชีถูกแก้ไข |
| งานถูกลบ | `account.task.remove` | งานในบัญชีถูกลบ |
| งานเสร็จสมบูรณ์ | `account.task.complete` | งานในบัญชีเสร็จสมบูรณ์ |
| งานตรวจสอบหมายเลขประจำตัวผู้เสียภาษีเสร็จสมบูรณ์ | `account.task.tax_id.complete` | งานตรวจสอบหมายเลขประจำตัวผู้เสียภาษีเสร็จสมบูรณ์ |

### ผู้รับเงินบัญชี

| เหตุการณ์ | คีย์ | คำอธิบาย |
|---|---|---|
| สร้างผู้รับเงิน | `account.acquirer.created` | มีการเพิ่มผู้รับเงินใหม่ในบัญชี |
| ผู้รับเงินได้รับการอัปเดต | `account.acquirer.updated` | ผู้รับเงินในบัญชีถูกแก้ไข |
| ผู้รับเงินถูกลบ | `account.acquirer.removed` | ผู้รับเงินถูกลบออกจากบัญชี |

## เหตุการณ์ผู้รับเงิน

เหตุการณ์ที่เกี่ยวข้องกับการจัดการผู้รับเงิน (ผู้รับชำระเงิน)

| เหตุการณ์ | คีย์ | คำอธิบาย |
|---|---|---|
| สร้างผู้รับเงิน | `payee.create` | สร้างผู้รับเงินใหม่สำเร็จ |
| การสร้างผู้รับเงินล้มเหลว | `payee.create.failure` | การสร้างผู้รับเงินล้มเหลว |
| ผู้รับเงินได้รับการอัปเดต | `payee.update` | ผู้รับเงินได้รับการอัปเดต |
| การอัปเดตผู้รับเงินล้มเหลว | `payee.update.failure` | การอัปเดตผู้รับเงินล้มเหลว |

## เหตุการณ์สัญญาการจอง

เหตุการณ์ติดตามวงจรชีวิตของสัญญาการจองผ่าน TripPay

| เหตุการณ์ | คีย์ | คำอธิบาย |
|---|---|---|
| ขอเริ่มต้น | `booking.contract.initialize.request` | มีการร้องขอเริ่มต้นสัญญาการจอง |
| เริ่มต้นเสร็จสมบูรณ์ | `booking.contract.initialize.complete` | การเริ่มต้นสัญญาการจองเสร็จสมบูรณ์ |
| สร้างสัญญา | `booking.contract.created` | สร้างสัญญาการจองใหม่ |
| กำลังดำเนินการสัญญา | `booking.contract.processing` | กำลังดำเนินการชำระเงินสัญญาการจอง |
| สัญญาล้มเหลว | `booking.contract.failed` | การชำระเงินสัญญาการจองล้มเหลว |
| ยกเลิกสัญญา | `booking.contract.cancelled` | สัญญาการจองถูกยกเลิก |
| คืนเงินบางส่วน | `booking.contract.refund.partial` | มีการคืนเงินบางส่วนในสัญญา |
| คืนเงินเต็มจำนวน | `booking.contract.refund.full` | มีการคืนเงินเต็มจำนวนในสัญญา |
| ขอคืนเงิน | `booking.contract.refund.request` | มีการส่งคำขอคืนเงิน |
| ข้อผิดพลาดคืนเงิน | `booking.contract.refund.error` | เกิดข้อผิดพลาดขณะดำเนินการคืนเงิน |
| สร้างการจ่ายเงิน | `booking.contract.payout.create` | มีการสร้างการจ่ายเงินสำหรับสัญญา |
| การจ่ายเงินเสร็จสมบูรณ์ | `booking.contract.payout.complete` | การจ่ายเงินเสร็จสมบูรณ์ |

## เหตุการณ์การชำระเงิน

เหตุการณ์ภายในที่เกี่ยวข้องกับการประมวลผลการชำระเงิน

| เหตุการณ์ | คีย์ | คำอธิบาย |
|---|---|---|
| ชำระเงินสำเร็จ | `internal.payment.succeeded` | การชำระเงินดำเนินการสำเร็จ |
| กำลังประมวลผลชำระเงิน | `internal.payment.processing` | กำลังดำเนินการชำระเงิน |
| ชำระเงินล้มเหลว | `internal.payment.failed` | การชำระเงินล้มเหลว |

## เหตุการณ์ผู้จัดการช่องทาง

เหตุการณ์ที่เกี่ยวข้องกับการอัปเดตอัตราและความพร้อมใช้งานของผู้จัดการช่องทาง

| เหตุการณ์ | คีย์ | คำอธิบาย |
|---|---|---|
| อัปเดตอัตรา | `channel-manager.update.rate` | ผู้จัดการช่องทางได้ส่งการอัปเดตอัตรา |
| อัปเดตความพร้อมใช้งาน | `channel-manager.update.availability` | ผู้จัดการช่องทางได้ส่งการอัปเดตความพร้อมใช้งาน |
| อัปเดตทั่วไป | `channel-manager.update` | การอัปเดตทั่วไปของผู้จัดการช่องทาง |

## เหตุการณ์หน่วยงานจัดการ

เหตุการณ์สำหรับหน่วยงานจัดการ (ทรัพย์สิน, บริษัทในเครือ, บริษัท) และการบริหารจัดการ

### การสมัครสมาชิก

| เหตุการณ์ | คีย์ | คำอธิบาย |
|---|---|---|
| การสมัครสมาชิกถูกจัดเตรียม | `managing_entity.subscription.provisioned` | มีการจัดเตรียมการสมัครสมาชิกใหม่ |
| การสมัครสมาชิกเปลี่ยนแปลง | `managing_entity.subscription.changed` | แผนการสมัครสมาชิกถูกเปลี่ยนแปลง |
| ใบแจ้งหนี้การสมัครสมาชิกเปลี่ยนแปลง | `managing_entity.subscription.invoice.changed` | ใบแจ้งหนี้การสมัครสมาชิกได้รับการอัปเดต |

### สถานะและการบริหาร

| เหตุการณ์ | คีย์ | คำอธิบาย |
|---|---|---|
| สถานะได้รับการอัปเดต | `managing_entity.status.updated` | สถานะของหน่วยงานเปลี่ยนแปลง |
| ลบหน่วยงาน | `managing_entity.deleted` | หน่วยงานจัดการถูกลบ |
| ตัวแทนได้รับการอัปเดต | `managing_entity.agent.updated` | ตัวแทนของหน่วยงานถูกแก้ไข |
| แจ้งเตือนทางอีเมลตัวแทน | `managing_entity.agent.email.notification` | ส่งอีเมลแจ้งเตือนไปยังตัวแทน |
| แจ้งเตือนทางอีเมลผู้จัดการ | `managing_entity.manager.email.notification` | ส่งอีเมลแจ้งเตือนไปยังผู้จัดการ |

### วงจรชีวิตผู้จัดการ

| เหตุการณ์ | คีย์ | คำอธิบาย |
|---|---|---|
| เพิ่มผู้จัดการ | `managing_entity.manager.added` | มีการเพิ่มผู้จัดการใหม่ |
| เชิญผู้จัดการ | `managing_entity.manager.invited` | ส่งคำเชิญผู้จัดการ |
| ลบผู้จัดการ | `managing_entity.manager.removed` | ลบผู้จัดการออก |
| ผู้จัดการยอมรับ | `managing_entity.manager.accepted` | ผู้จัดการยอมรับคำเชิญ |
| ผู้จัดการปฏิเสธ | `managing_entity.manager.rejected` | ผู้จัดการปฏิเสธคำเชิญ |
| ตั้งเจ้าของ | `managing_entity.manager.owner.set` | โอนกรรมสิทธิ์หน่วยงาน |

### ข้อมูลหน่วยงาน

| เหตุการณ์ | คีย์ | คำอธิบาย |
|---|---|---|
| ข้อมูลได้รับการอัปเดต | `managing_entity.information.updated` | ข้อมูลหน่วยงานได้รับการอัปเดต |
| ที่อยู่ได้รับการอัปเดต | `managing_entity.city.updated` | ที่อยู่/เมืองของหน่วยงานเปลี่ยนแปลง |
| ตัวแทนบัญชีได้รับการอัปเดต | `managing_entity.account_representative.updated` | ตัวแทนบัญชีถูกเปลี่ยนแปลง |
| รูปโปรไฟล์ได้รับการอัปเดต | `managing_entity.profile_picture.updated` | รูปโปรไฟล์หน่วยงานได้รับการอัปเดต |
| โลโก้ได้รับการอัปเดต | `managing_entity.logo.updated` | โลโก้หน่วยงานได้รับการอัปเดต |
| ชื่อ URL ได้รับการอัปเดต | `managing_entity.url_name.updated` | URL ส่วนตัวของหน่วยงานเปลี่ยนแปลง |
| รหัสประจำตัวเฉพาะได้รับการอัปเดต | `managing_entity.unique_id.updated` | ตัวระบุเฉพาะของหน่วยงานเปลี่ยนแปลง |
| ชื่อได้รับการอัปเดต | `managing_entity.name.updated` | ชื่อหน่วยงานถูกเปลี่ยนแปลง |
