ทักษะ AI คือชุดคำสั่งที่พกพาได้ซึ่งสอนตัวแทนเขียนโค้ดให้ทำงานกับ Wink แทนที่จะพึ่งพาข้อมูลการฝึกอบรมหรือเอกสารที่ล้าสมัย แต่ละทักษะจะชี้แนะตัวแทนให้ใช้ [Wink MCP server](/th/developers/mcp) เป็นแหล่งข้อมูลสดที่ถูกต้องสำหรับสัญญา API และเอกสารแพลตฟอร์ม

ทักษะเหล่านี้ทำงานร่วมกับไคลเอนต์ตัวแทนใดก็ได้ที่รองรับ — Claude Code, Codex, Cursor และอื่นๆ

## ทักษะที่มีให้ใช้งาน

### wink-api-builder

ใช้งาน, ตรวจสอบ หรือดีบักการผสานรวม Wink API ทักษะนี้จะนำทางตัวแทนผ่านเวิร์กโฟลว์ที่มีโครงสร้าง:

1. ค้นหาการดำเนินการ API ที่ถูกต้องผ่าน `api_search`
2. เปิดสัญญาการร้องขอ/ตอบกลับสดจากเซิร์ฟเวอร์ MCP
3. ดึงการตั้งค่าการตรวจสอบสิทธิ์จากเอกสาร
4. เขียนโค้ดตามสัญญาที่ได้รับการยืนยัน
5. แจ้งเตือนความไม่ตรงกันระหว่างโค้ดที่มีอยู่กับ API ปัจจุบัน

เหมาะสำหรับ:
- สร้างเมธอดไคลเอนต์ใหม่สำหรับ Wink API ใดก็ได้
- ดีบักความไม่ตรงกันของสัญญา (เช่น การร้องขอที่ส่งกลับ HTTP 400)
- ตรวจสอบการผสานรวมที่มีอยู่กับสเปคสด

## การติดตั้ง

คัดลอกโฟลเดอร์ทักษะไปยังไดเรกทอรีทักษะของตัวแทนของคุณ

:::tip
เซิร์ฟเวอร์ Wink MCP ต้องเชื่อมต่อในสภาพแวดล้อมเดียวกัน ดู [MCP Server](/th/developers/mcp) สำหรับคำแนะนำการตั้งค่า
:::

**Claude Code**

```bash
# repo-local
cp -r skills/wink-api-builder .claude/skills/

# or user-level
cp -r skills/wink-api-builder ~/.claude/skills/
```

เรียกใช้งานด้วย:
```
/wink-api-builder add a TypeScript client for the Wink affiliate create flow
```

**Codex**

```bash
# repo-local
cp -r skills/wink-api-builder .agents/skills/

# or user-level
cp -r skills/wink-api-builder $CODEX_HOME/skills/
```

เรียกใช้งานด้วย:
```
Use $wink-api-builder to add a TypeScript client for the Wink affiliate create flow.
```

**ตัวแทนอื่นๆ**

วางโฟลเดอร์ทักษะไว้ที่ใดก็ได้ที่ตัวแทนของคุณค้นหาทักษะ จุดเริ่มต้นคือ `SKILL.md`
