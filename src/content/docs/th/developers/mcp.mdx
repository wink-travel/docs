## ภาพรวม
เซิร์ฟเวอร์ MCP ของ Wink ที่โฮสต์อยู่พร้อมใช้งานที่:

```
https://docs.mcp.wink.travel/mcp
```

ชี้ไคลเอนต์ AI ที่รองรับ (VS Code, Cursor, และอื่น ๆ ที่ใช้ MCP HTTP transport) ไปยังจุดเชื่อมต่อนี้เพื่อให้พวกเขา:
- เรียกดูเอกสารผลิตภัณฑ์ & แพลตฟอร์ม (`wink://docs/...`)
- เรียกดูสเปก OpenAPI สดและเอกสารการทำงาน (`wink://api/...`)
- ทำการค้นหาคำสำคัญ / ความหมายเบา ๆ ในเอกสารและ API สด

## การขนส่ง
ใช้จุดเชื่อมต่อ HTTP แบบสตรีมหลัก:

```
https://docs.mcp.wink.travel/mcp
```

ถ้าไคลเอนต์ของคุณรองรับเฉพาะ SSE ให้ใช้:

```
https://docs.mcp.wink.travel/sse
```

### Claude Code
เอกสารอย่างเป็นทางการ: [เชื่อมต่อ Claude Code กับเครื่องมือผ่าน MCP](https://code.claude.com/docs/en/mcp)

Anthropic แนะนำให้ใช้การขนส่ง HTTP ระยะไกลสำหรับเซิร์ฟเวอร์ MCP ที่โฮสต์บนคลาวด์:
```bash
claude mcp add --transport http wink-docs https://docs.mcp.wink.travel/mcp
claude mcp list
```

ภายใน Claude Code ให้รัน `/mcp` เพื่อตรวจสอบสถานะเซิร์ฟเวอร์ หากคุณต้องการแชร์เซิร์ฟเวอร์กับทั้งรีโป ให้เพิ่ม `--scope project` เพื่อให้ Claude Code เขียนการตั้งค่าไปยัง `.mcp.json`

### Codex
เอกสารอย่างเป็นทางการ: [เริ่มต้นใช้งาน Docs MCP สำหรับ Codex](https://developers.openai.com/resources/docs-mcp)

OpenAI อธิบายการตั้งค่า MCP ผ่าน Codex CLI และระบุว่าการตั้งค่านี้ถูกแชร์กับส่วนขยาย IDE:
```bash
codex mcp add wink-docs --url https://docs.mcp.wink.travel/mcp
codex mcp list
```

### Cursor
เอกสาร MCP ของ Cursor: https://cursor.com/docs/context/mcp
การตั้งค่า: เพิ่มเซิร์ฟเวอร์ระยะไกลผ่านการตั้งค่า MCP
```json
{
  "mcpServers": {
    "wink-docs": {
      "url": "https://docs.mcp.wink.travel/mcp",
    }
  }
}
```

### VS Code
เอกสารอย่างเป็นทางการ: https://code.visualstudio.com/docs/copilot/customization/mcp-servers
การตั้งค่าระยะไกลขั้นต่ำ (ไฟล์ `mcp.json` ของผู้ใช้หรือเวิร์กสเปซ):
```json
{
  "servers": {
    "wink-docs": {
      "type": "http",
      "url": "https://docs.mcp.wink.travel/mcp"
    }
  }
}
```

เคล็ดลับ:
- ใช้ Command Palette: “MCP: List Servers” เพื่อตรวจสอบการเชื่อมต่อ & เรียกดูเครื่องมือ
- ตัวเลือกเครื่องมือช่วยให้คุณเปิด/ปิด `api_search` และ `docs_search`
- เพิ่มบริบท: มุมมองแชท → เพิ่มบริบท → MCP Resources เพื่อแนบหน้าหนังสือเอกสาร

## URI ทรัพยากรหลัก

| URI | คำอธิบาย |
|-----|-------------|
| `wink://docs` | ดัชนีเอกสาร (จัดกลุ่มตามหมวดหมู่) |
| `wink://docs/{path}` | บทความเอกสารเฉพาะ (Markdown / MDX) |
| `wink://api` | แคตตาล็อก OpenAPI สดที่ค้นพบจากการตั้งค่า swagger ของ Wink |
| `wink://api/{apiName}` | สรุป API แบบย่อพร้อมการยืนยันตัวตน, แท็ก, เฮดเดอร์ และตัวอย่างการทำงาน |
| `wink://api/{apiName}/operations` | รายการการทำงานทั้งหมดจัดกลุ่มตามแท็ก |
| `wink://api/{apiName}/operation/{key}` | รายละเอียดการทำงานแต่ละรายการ (พารามิเตอร์, ร่างคำขอ, การตอบสนอง) |
| `wink://api/{apiName}/schema/{schemaName}` | สคีมาคอมโพเนนต์ที่แก้ไขแล้วพร้อมฟิลด์ซ้อน, ประเภท และตัวอย่าง |
| `wink://api/{apiName}/operation/{key}/request` | มุมมองเน้นคำขอพร้อมอินพุตที่จำเป็นและตัวอย่าง payload ขั้นต่ำ |
| `wink://api/{apiName}/operation/{key}/response/{status}` | มุมมองเน้นการตอบสนองสำหรับรหัสสถานะเฉพาะ |
| `wink://api/{apiName}/operation/{key}/recipe` | โฟลวงานที่มุ่งเน้นการเรียกใช้งานนั้นอย่างถูกต้อง |

## เครื่องมือค้นหา (อ้างอิงด่วน)
| เครื่องมือ | จุดประสงค์ | ตัวอย่างคำค้นทั่วไป |
|------|---------|-----------------------|
| `api_search` | ค้นหา API สดหรือการทำงานเพื่อรวมเข้าด้วยกัน | `create affiliate`, `payment receipt`, `lookup inventory`, `invite manager` |
| `docs_search` | ค้นหาเอกสารแนวคิด / คู่มือ | `authentication`, `webhooks`, `wordPress` |

ปล่อย `query` ว่างเพื่อรับคำแนะนำ ผลลัพธ์แต่ละรายการจะรวม URI MCP ที่คุณสามารถเปิดได้
สำหรับผลลัพธ์การทำงาน, `api_search` ยังส่งคืน URI ที่เชื่อมโยงกับคำขอ, การตอบสนอง, สคีมา และสูตรทำงานเพื่อให้เอเจนต์สามารถกระโดดไปยังซับรีซอร์สที่มีประโยชน์ที่สุดได้โดยตรง

## รูปแบบการใช้งานทั่วไป
1. รัน `api_search` ด้วยเจตนาของคุณ (เช่น *"create affiliate"* หรือ *"payment receipt"*)
2. เปิด URI `wink://api/...` ที่ส่งกลับมา หรือทำตามฟิลด์ที่เชื่อมโยง `request_uri`, `response_uris`, `request_schema_uris`, `response_schema_uris` และ `recipe_uri`
3. ใช้ `docs_search` สำหรับแนวคิดสนับสนุน (การยืนยันตัวตน, webhooks, โฟลวงาน)

## หมายเหตุ
- URI ที่ส่งกลับแต่ละรายการสามารถเปิดโดยเอเจนต์เพื่อดูเนื้อหาเต็ม
- โครงสร้าง `wink://api` เป็นแบบสดและติดตามแคตตาล็อก OpenAPI ปัจจุบันแทนที่จะเป็นดัชนีรีโปที่สร้างขึ้น
- คำแนะนำจะปรากฏโดยอัตโนมัติเมื่อคำค้นว่าง
