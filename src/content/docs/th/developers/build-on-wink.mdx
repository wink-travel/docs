> “ถ้าฉันมองเห็นไกลกว่า นั่นก็เพราะฉันยืนอยู่บนบ่าของยักษ์”

-- *เซอร์ ไอแซก นิวตัน*

แพลตฟอร์มแบบนี้ไม่ได้ถูกสร้างขึ้นอย่างโดดเดี่ยว แต่มันคือผลรวมของหลายส่วนที่เคลื่อนไหว; ทั้งเก่าและใหม่ เราต้องการที่จะยังคงให้บริการแก่ผู้ที่ตามหลังเรา

> "ไม่จำเป็นต้องประดิษฐ์ล้อที่พังใหม่อีกครั้ง"

เทคโนโลยีการท่องเที่ยวมีความแตกแยกและเป็นระบบเก่า เราพยายามนำสิ่งที่ดีที่สุดจากของเก่าที่ใช้งานได้ดีมาใช้ เราได้รวมระบบจัดการช่องทางหลายตัว ระบบจัดการทรัพย์สิน ++ เราได้เพิ่มสิ่งใหม่ๆ และเราห่อหุ้มทั้งหมดนี้ด้วยมาตรฐานและโปรโตคอลในอุตสาหกรรมที่เป็นที่รู้จักและง่ายต่อการใช้งาน... เพื่อที่คุณจะได้ไม่ต้องทำเอง

:::note[แนวทางขับเคลื่อนด้วย API]
หมายถึงโดยคร่าวๆ ว่าคุณสมบัติใดๆ ที่เราสร้างขึ้นสำหรับตัวเราเองควรจะ:

- เข้าใจง่าย
- ยืดหยุ่น
- ขยายขนาดได้
- ใช้ซ้ำได้ และ...
- ขยายเพิ่มเติมได้

...โดยผู้อื่นที่ต้องการใช้ API เหล่านี้ด้วย
:::

Wink ถูกสร้างขึ้นโดยใช้ `แนวทางขับเคลื่อนด้วย API` ดังนั้นตามปรัชญาการออกแบบนี้ คุณสมบัติทั้งหมดของ Wink สามารถโต้ตอบได้ในฐานะนักพัฒนาภายนอกเหมือนกับว่าระบบเป็นของคุณเอง นอกจากนี้ยังมีแง่มุมอื่นๆ ของแพลตฟอร์มที่คุณสามารถใช้ประโยชน์ได้เมื่อรวมระบบกับเรา

## ตัวจัดการช่องทาง

เรามีจุดเชื่อมต่อเฉพาะสำหรับตัวจัดการช่องทางสำหรับระบบจัดการทรัพย์สิน ตัวจัดการช่องทาง และระบบสำรองห้องพักส่วนกลางที่ต้องการส่งอัตราและความพร้อมของทรัพย์สินให้เรา
