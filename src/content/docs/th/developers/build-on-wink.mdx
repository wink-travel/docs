> “ถ้าฉันมองเห็นไกลกว่า นั่นก็เพราะฉันยืนอยู่บนบ่าของยักษ์”

-- *เซอร์ ไอแซก นิวตัน*

แพลตฟอร์มแบบนี้ไม่ได้ถูกสร้างขึ้นอย่างโดดเดี่ยว แต่มันคือผลรวมของหลายส่วนที่เคลื่อนไหว; ทั้งเก่าและใหม่ เราต้องการที่จะยังคงให้บริการแก่ผู้ที่ตามหลังเรา

> "ไม่จำเป็นต้องประดิษฐ์ล้อที่พังใหม่"

เทคโนโลยีการท่องเที่ยวมีความกระจัดกระจายและเป็นระบบเก่า เราได้พยายามนำสิ่งที่ดีที่สุดจากของเก่าที่ใช้งานได้ดีมาใช้ เราได้รวมระบบจัดการช่องทางหลายตัว ระบบจัดการทรัพย์สิน ++ เราได้เพิ่มสิ่งใหม่ๆ และเราห่อหุ้มทั้งหมดนี้ด้วยมาตรฐานและโปรโตคอลในอุตสาหกรรมที่เป็นที่รู้จักและง่ายต่อการใช้งาน... เพื่อที่คุณจะได้ไม่ต้องทำเอง

:::note[แนวทางขับเคลื่อนด้วย API]
หมายความโดยประมาณว่า ฟีเจอร์ใดๆ ที่เราสร้างขึ้นสำหรับตัวเราเองควรจะ:

- เข้าใจง่าย
- ยืดหยุ่น
- ขยายได้
- ใช้ซ้ำได้ และ...
- ขยายเพิ่มเติมได้

...โดยผู้อื่นที่ต้องการใช้ API เหล่านี้ด้วย
:::

Wink ถูกสร้างขึ้นโดยใช้แนวทาง `ขับเคลื่อนด้วย API` ดังนั้นผลจากปรัชญาการออกแบบนี้ ฟีเจอร์ทั้งหมดของ Wink สามารถโต้ตอบได้ในฐานะนักพัฒนาภายนอกเหมือนกับว่าระบบเป็นของคุณเอง นอกจากนี้ยังมีแง่มุมอื่นๆ ของแพลตฟอร์มที่คุณสามารถใช้ประโยชน์เมื่อรวมระบบกับเรา

## Channel manager

เรามีจุดเชื่อมต่อ channel manager เฉพาะสำหรับระบบจัดการทรัพย์สิน, channel managers และระบบสำรองห้องพักกลางที่ต้องการส่งอัตราค่าห้องและความพร้อมให้เรา

## SDKs

:::note
ดู [Software Development Kit](https://en.wikipedia.org/wiki/Software_development_kit)
:::

เริ่มต้นการรวมระบบกับ Wink โดยใช้หนึ่งในภาษาการเขียนโปรแกรมที่คุณชื่นชอบ เรามีไลบรารีอย่างเป็นทางการสำหรับภาษาที่ใช้มากที่สุด หากเราไม่รองรับภาษาของคุณ คุณก็สามารถสร้างไลบรารีของตัวเองได้อย่างง่ายดาย
