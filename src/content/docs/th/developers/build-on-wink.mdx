> “หากฉันมองเห็นได้ไกลกว่านั้น ก็เพราะว่าฉันยืนอยู่บนไหล่ของยักษ์”

-*เซอร์ไอแซกนิวตัน*

แพลตฟอร์มแบบนี้ไม่ได้สร้างขึ้นมาอย่างโดดเดี่ยว แต่เป็นการรวมเอาส่วนประกอบต่างๆ ที่เคลื่อนไหวอยู่หลายส่วนเข้าด้วยกัน ทั้งส่วนเก่าและส่วนใหม่ เราต้องการที่จะให้บริการแก่ผู้ที่มาหลังจากเราต่อไป

> "ไม่จำเป็นต้องประดิษฐ์ล้อที่พังขึ้นมาใหม่"

เทคโนโลยีการเดินทางนั้นแตกแขนงออกไปและล้าสมัย เราพยายามใช้สิ่งที่ดีที่สุดจากเทคโนโลยีเก่าที่ใช้งานได้ เราได้บูรณาการระบบจัดการช่องทาง ระบบจัดการทรัพย์สิน และอื่นๆ เราได้เพิ่มสิ่งใหม่ๆ เข้าไป และเราได้รวมสิ่งเหล่านี้ไว้ในมาตรฐานและโปรโตคอลของอุตสาหกรรมที่เป็นที่รู้จักและใช้งานได้ง่าย... ดังนั้นคุณจึงไม่จำเป็นต้องทำเช่นนั้น

:::note\[แนวทางที่ขับเคลื่อนโดย API]
โดยคร่าวๆ แล้ว ฟีเจอร์ใดๆ ก็ตามที่เราสร้างขึ้นเองควรเป็นดังนี้:

* เข้าใจได้
* ยืดหยุ่นได้
* ปรับขนาดได้
* สามารถนำกลับมาใช้ซ้ำได้และ...
* ขยายได้

...โดยคนอื่นๆ ที่ต้องการใช้ API เหล่านี้เช่นกัน
:::

Wink ถูกสร้างขึ้นโดยใช้สิ่งนี้`API-driven approach`ด้วยปรัชญาการออกแบบนี้ ทำให้นักพัฒนาภายนอกสามารถโต้ตอบกับฟีเจอร์ทั้งหมดของ Wink ได้ราวกับว่าระบบนั้นเป็นของคุณเอง นอกจากนี้ยังมีแง่มุมอื่นๆ ของแพลตฟอร์มที่คุณสามารถใช้ประโยชน์ได้เมื่อผสานรวมกับเรา

## ผู้จัดการช่อง

เรามีผู้จัดการช่องทางการสิ้นสุดที่ทุ่มเทให้กับระบบการจัดการทรัพย์สิน ผู้จัดการช่องทาง และระบบการจองส่วนกลางที่ต้องการส่งอัตราและห้องว่างของทรัพย์สินให้กับเรา
