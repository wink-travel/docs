เมื่อมีการสร้าง อัปเดต หรือยกเลิกการจองบน Wink แพลตฟอร์มจะซิงค์การเปลี่ยนแปลงนั้นกับระบบภายนอกที่เชื่อมต่อของคุณ — ระบบจัดการทรัพย์สิน (PMS), channel managers และระบบสำรองห้องพักส่วนกลาง (CRS)

## วิธีการทำงาน

การเปลี่ยนแปลงการจองแต่ละครั้งจะสร้าง **รายการซิงค์** ที่ติดตามสถานะการซิงโครไนซ์ แพลตฟอร์มจะประมวลผลรายการซิงค์แบบอะซิงโครนัส โดยจะลองใหม่เมื่อเกิดข้อผิดพลาดเพื่อให้แน่ใจว่าการส่งข้อมูลสำเร็จอย่างน่าเชื่อถือ

## วงจรชีวิตการซิงค์

รายการซิงค์แต่ละรายการจะปฏิบัติตามเครื่องสถานะ:

```
┌─────────────┐     success     ┌───────────┐
│  TO_CREATE   │ ──────────────→ │ PROCESSED │
└─────────────┘                  └───────────┘
       │
       │ failure
       ▼
┌──────────────────┐
│ TO_CREATE_FAILED │
└──────────────────┘
```

รูปแบบเดียวกันนี้ใช้กับการอัปเดตและการยกเลิก:

| การกระทำ | สถานะรอดำเนินการ | สถานะสำเร็จ | สถานะล้มเหลว |
|---|---|---|---|
| การจองใหม่ | `TO_CREATE` | `PROCESSED` | `TO_CREATE_FAILED` |
| การอัปเดตการจอง | `TO_UPDATE` | `PROCESSED` | `TO_UPDATE_FAILED` |
| การยกเลิก | `TO_CANCEL` | `PROCESSED` | `TO_CANCEL_FAILED` |

## สิ่งที่กระตุ้นการซิงค์

- **การจองใหม่** — นักเดินทางทำการจองผ่าน Wink หรือช่องทางพันธมิตร
- **การอัปเดตการจอง** — รายละเอียดการจองเปลี่ยนแปลง (วันที่ ข้อมูลผู้เข้าพัก การจัดสรรห้อง)
- **การยกเลิก** — การจองถูกยกเลิกโดยนักเดินทางหรือที่พัก

## การตรวจสอบ

คุณสามารถตรวจสอบสถานะการซิงค์ผ่านแดชบอร์ด channel manager หรือ PMS ที่เชื่อมต่อของคุณ รายการซิงค์ที่ล้มเหลวจะถูกลองใหม่โดยอัตโนมัติ หากการซิงค์ยังล้มเหลวอยู่ จะคงอยู่ในสถานะ `*_FAILED` เพื่อการตรวจสอบ

## การแก้ไขปัญหา

หากการจองไม่ปรากฏใน PMS ของคุณ:

1. **ตรวจสอบการเชื่อมต่อ channel manager** — ตรวจสอบให้แน่ใจว่าการเชื่อมต่อ PMS หรือ channel manager ของคุณเปิดใช้งานและตั้งค่าอย่างถูกต้อง
2. **ตรวจสอบการส่ง webhook** — หากคุณใช้ webhook ให้ตรวจสอบว่า endpoint ของคุณได้รับเหตุการณ์ `booking.create` และ `booking.cancelled` ดูที่ [Webhook Events Catalog](/th/developers/webhook-events#booking-events)
3. **ตรวจสอบรายการที่ล้มเหลว** — รายการซิงค์ที่ล้มเหลวบ่งชี้ปัญหาการเชื่อมต่อหรือข้อมูลกับระบบเป้าหมาย
4. **ติดต่อฝ่ายสนับสนุน** — หากปัญหายังคงอยู่ ทีม Wink สามารถตรวจสอบรายการซิงค์ที่ล้มเหลวแทนคุณได้
