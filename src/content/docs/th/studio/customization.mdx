การปรับแต่งเป็นฟีเจอร์อันทรงพลังที่คุณสามารถใช้ควบคุมทุกแง่มุมของการผสานรวมกับ Wink ได้โดยไม่ต้องเขียนโค้ดใดๆ สินค้าคงคลังทุกชิ้นที่คุณขายสามารถปรับแต่งได้ เพิ่มหรือลบฟีเจอร์ เปลี่ยนรูปแบบสี และอื่นๆ อีกมากมาย

ในขณะที่อยู่บน`Customization`หน้าคลิ๊กที่`Actions`ปุ่มใต้การปรับแต่งที่คุณต้องการใช้งานและคลิก`Update`-*คุณมีรายการเริ่มต้นให้ใช้งานอยู่แล้วเมื่อคุณสร้างบัญชี ไม่สามารถลบรายการนี้ได้*

นี่จะนำคุณไปที่แบบฟอร์มการปรับแต่งซึ่งคุณจะเห็นแท็บ 5 - 7 แท็บ (*ขึ้นอยู่กับประเภทบัญชีของคุณ*-

## ทั่วไป

นี่คือตัวเลือกที่มีให้คุณภายใต้`General`แท็บ

* **ชื่อ**ตั้งชื่อให้กับการปรับแต่งของคุณเพื่อให้จดจำได้ง่าย*เช่น สำหรับบล็อกการเดินทางของฉัน*
* **แสดงสกุลเงิน**เลือกสกุลเงินเริ่มต้นที่ผู้ใช้จะเห็นแสดงถัดจากสินค้าคงคลังการเดินทางที่จองได้*เช่น USD*
* **ภาษาที่แสดง**เลือกภาษาเริ่มต้นที่ผู้ใช้จะพบ*เช่น ภาษาอังกฤษ*
* **ผนวกเข้ากับหัวเรื่องหน้า**คุณสามารถควบคุมสิ่งที่จะเพิ่มลงในหัวเรื่องของหน้าได้ (*ชื่อเรื่องที่คุณเห็นในแท็บเบราว์เซอร์*) ส่วนนี้ใช้สำหรับระบบจองของเราเป็นหลัก*เช่น หากคุณไปที่ https://ota.wink.travel ตามค่าเริ่มต้น คุณจะเห็น`Home | Traveliko - Hotels with Benefits`*. คุณสามารถเปลี่ยนแปลงทุกอย่างหลังท่อ '|' ได้

## ธีม

การ`Theme`แท็บช่วยให้คุณปรับแต่งรูปลักษณ์และความรู้สึกของเราได้[เครื่องมือจอง](/booking-engine/in-a-nutshell)เว็บไซต์,[วิงค์ลิงค์](/link-manager/wink-links)เว็บไซต์และทั้งหมดของเรา[ส่วนประกอบของเว็บไซต์](/developers/web-components)-

### การกำหนดค่า

ส่วนนี้ช่วยให้คุณกำหนดค่าการออกแบบและเค้าโครงของการ์ดได้ ในขณะนี้มีการออกแบบเพียงแบบเดียว คือ Wink

เค้าโครงของการ์ดจะควบคุมว่าคุณต้องการให้ผู้ใช้เห็นการ์ดธุรกรรมที่จัดวางในแนวนอนหรือแนวตั้ง ข้อมูลจะเหมือนกัน

:::หมายเหตุ
ไม่มีความแตกต่างในการจัดวางการ์ดเมื่อดูการ์ดธุรกรรมจากอุปกรณ์พกพา อย่างไรก็ตาม คุณจะเห็นความแตกต่างได้บน iPad mini และความละเอียดที่สูงกว่า
:::

:::tip\[เคล็ดลับจากผู้เชี่ยวชาญ]
ใช้เค้าโครงแนวนอนหากคุณมีรายการท่องเที่ยว (เช่น ประเภทห้อง) น้อยกว่า 4 รายการ... มันจะดูดีกว่า
:::

## แผนการเดินทาง

การ`Itinerary`แท็บนี้อาจเป็นแท็บที่น่าสนใจที่สุดสำหรับการตรวจสอบให้แน่ใจว่าราคาสินค้าที่คุณเลือกแสดงขึ้น
เนื่องจากการกำหนดราคาสินค้าสำหรับการเดินทางนั้นค่อนข้างซับซ้อน Wink จึงต้องทราบแผนการเดินทางของผู้ใช้เสียก่อนจึงจะสามารถแสดงราคาที่ถูกต้องได้

ในส่วนนี้คุณสามารถตั้งค่าแผนการเดินทางเริ่มต้นเพื่อแสดงราคาสำหรับอสังหาริมทรัพย์ดิจิทัลทั้งหมดของเรา เช่น:

* รูปภาพที่เป็นมิตรกับสังคมของเราพร้อมราคาซ้อนทับ
* กริด,การ์ด และลิงก์ WinkLinks
* ส่วนประกอบของเว็บไซต์
* ระบบจองคิว

### วันที่

คุณสามารถเลือกได้ว่าต้องการตั้งค่าช่วงวันที่คงที่หรือแบบต่อเนื่อง

* เลือกช่วงวันที่คงที่หากคุณจะใช้การปรับแต่งสำหรับเหตุการณ์เฉพาะที่เกิดขึ้นในวันที่กำหนด
* เลือกช่วงวันที่แบบต่อเนื่องหากคุณต้องการ:
  1. ค่าเริ่มต้นคือวันนี้ + 1 คืน ซึ่ง OTA ส่วนใหญ่ก็ทำแบบนั้น หรือ
  2. ค่าเริ่มต้นคือวันที่ในอนาคตซึ่งควรมีพร้อมใช้งานอยู่เสมอ

### การจัดวางห้อง

โดยค่าเริ่มต้น แผนการเดินทางจะกำหนดไว้ที่ผู้ใหญ่ 2 คน เนื่องจากเป็นการตั้งค่าทั่วไปสำหรับนักเดินทางส่วนใหญ่
สามารถเปลี่ยนได้ง่าย และคุณสามารถเพิ่มห้องหลายห้องที่มีผู้เข้าพักหลายคนได้ ทั้งผู้ใหญ่และเด็ก

### โปรโมชั่น

โปรโมชั่นเป็นฟีเจอร์ที่มีประโยชน์ซึ่งช่วยให้ผู้ใช้ประหยัดเวลา หากที่พักให้รหัสโปรโมชั่นแก่คุณเพื่อแบ่งปันกับเพื่อนๆ คุณสามารถเลือกใส่รหัสที่นี่ได้
ทุกคนที่เข้ามาโดยใช้ลิงก์ของคุณหรือลิงก์ที่คล้ายกัน จะได้รับรหัสโปรโมชั่นเพิ่มลงในแผนการเดินทางโดยอัตโนมัติ และจะเห็นราคาส่วนลดใช้โดยอัตโนมัติ

## การบูรณาการ

การ`Integration`แท็บนี้มีไว้สำหรับโรงแรม ผู้บูรณาการ และตัวแทนการท่องเที่ยวที่ต้องการควบคุมการกระจายการจองบน Wink มากขึ้น

### การแจ้งเตือน

วิธีนี้จะทำให้โรงแรมและผู้บูรณาการมีตัวเลือกในการให้ Wink แจ้งให้ทุกฝ่ายที่เกี่ยวข้องทราบเมื่อมีการจองเกิดขึ้น
ซึ่งรวมถึงการแจ้งผู้จัดการช่องทางด้วย

โดยค่าเริ่มต้น ตัวเลือกทั้งหมดเหล่านี้จะเปิดใช้งาน กรณีการใช้งานที่ผู้รวมระบบอาจต้องการปิดใช้งานคุณสมบัติหนึ่งหรือหลายคุณสมบัติเหล่านี้ ได้แก่:

* ตัวแทนการท่องเที่ยวมีการบูรณาการที่ระดับ API และต้องการส่งอีเมลยืนยันของตนเอง
* โรงแรมกำลังทดสอบการบูรณาการของระบบและไม่ต้องการรบกวนพนักงานขายของตน
* โรงแรมไม่สนใจอีเมล์และต้องการรับการแจ้งเตือนผ่านระบบ PMS เท่านั้น

### โลโก้อีเมล์

สำหรับผู้บูรณาการที่ต้องการใช้บริการการแจ้งเตือนทางอีเมลของเรา พวกเขาสามารถปรับแต่งอีเมลให้ดูเหมือนว่ามาจากตนเองได้

:::หมายเหตุ
ขนาดที่แนะนำคือกว้าง 300 พิกเซล x สูง 75 พิกเซล
:::

## เฉพาะโรงแรม

ภายใต้`Hotel-specific`แท็บคือตัวเลือกที่ให้โรงแรมควบคุมสิ่งที่จะแสดงบนเพจปลายทางในเครื่องมือจองห้องพักของเรา และสิ่งที่จะแสดงเมื่อไม่มีห้องพักว่างตามแผนการเดินทางที่ร้องขอ

* **อันดับ**:ทรัพย์สินที่ยังไม่ได้รับอันดับที่มีความสำคัญอาจเลือกปิดใช้งานคุณลักษณะนั้น

:::หมายเหตุ
ที่พักไม่สามารถควบคุมได้ว่าผู้ร่วมงานจะแสดงหน้า Landing Page ของโรงแรมอย่างไร... นั่นเป็นการกำหนดค่าสองแบบที่แตกต่างกัน
:::

* **ความพร้อมจำหน่าย**: ให้คุณสมบัติเลือกว่าจะแสดงสินค้าคงคลังต่อไปหรือไม่แม้ว่าจะไม่มีสินค้าอยู่ก็ตาม

:::หมายเหตุ
ในกรณีส่วนใหญ่ คุณสมบัติจะต้องการให้ผู้ใช้มีวิธีคลิก`Book`ปุ่มนี้ใช้เพื่อไปยังหน้าแรกซึ่งผู้ใช้สามารถเปลี่ยนแผนการเดินทางและค้นหาวันที่ว่างได้
เปิดใช้งานสวิตช์นี้เมื่อต้องการบล็อกผู้ใช้ไม่ให้ดำเนินการต่อ
:::

## วิธีการใช้งาน

ในสถานการณ์ส่วนใหญ่ คุณจะเลือกสิ่งที่`Customization`เพื่อใช้กับลิงก์ บัตร แผนที่ และตารางของคุณ

ต่อไปนี้คือเวลาที่คุณอาจต้องการใช้การกำหนดค่าด้วยตนเอง:

* **การควบคุมด้วยตนเอง**คุณต้องการแชร์ลิงก์ไปยังเครื่องมือจองของเราโดยตรงและไม่ใช้เครื่องมือของเรา[ลิงก์ที่สามารถแชร์ได้](/studio/shareable-links)คุณสมบัติที่มีการใช้การกำหนดค่าแบบกำหนดเองของคุณ
* **ปลั๊กอิน WordPress**เพิ่มการกำหนดค่าให้กับ Wink ของคุณ[ปลั๊กอิน WordPress](/developers/wordpress) `Settings`-
* **การบูรณาการแบบกำหนดเอง**นำไปใช้งานด้วยตนเองกับตัวรวมระบบแบบกำหนดเองของคุณโดยใช้ของเรา[โหลดแอพวิงค์](/developers/web-components/#application-loader)ส่วนประกอบเว็บไซต์

<h4 align="center">[https://ota.wink.travel?configurationId=รหัสประจำตัวของฉัน](https://ota.wink.travel?configurationId=MY-UNIQUE-ID)</h4>


:::หมายเหตุ
เมื่อผู้ใช้คลิกลิงก์นี้ คุณจะตั้งค่าคุกกี้การระบุแหล่งที่มาของพันธมิตรด้วย
:::

### การบูรณาการแบบกำหนดเอง

ควบคุมส่วนประกอบเว็บ Wink ที่ฝังไว้ทั้งหมดของคุณบนเพจด้วย[โหลดแอพวิงค์](/developers/web-components/#application-loader)ส่วนประกอบเว็บไซต์

ด้านล่างนี้เป็นตัวอย่างที่ใช้`MY-UNIQUE-ID`เป็นรหัสการกำหนดค่า

```html {11}
<html>
  <head>
    <link rel="stylesheet" href="https://elements.wink.travel/styles.css"></link>
  </head>
  <body>

    <script src="https://elements.wink.travel/elements.js" type="module" defer></script>

    <wink-app-loader
      client-id="YOUR CLIENT ID GOES HERE"
      configuration-id="MY-UNIQUE-ID"
    ></wink-app-loader>
  </body>
</html>
```
