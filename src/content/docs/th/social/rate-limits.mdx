เมื่อ Wink เผยแพร่เนื้อหาไปยังบัญชีโซเชียลมีเดียที่เชื่อมต่อของคุณ จะเคารพข้อจำกัดอัตราการใช้งานที่แต่ละแพลตฟอร์มกำหนดไว้ ซึ่งช่วยให้บัญชีของคุณอยู่ในสถานะที่ดีและเนื้อหาของคุณถูกส่งมอบอย่างน่าเชื่อถือ

## ข้อจำกัดของแพลตฟอร์ม

แต่ละแพลตฟอร์มโซเชียลมีเดียจะบังคับใช้ข้อจำกัดการเผยแพร่ของตนเอง Wink จะติดตามการใช้งานแบบเรียลไทม์และปรับลดการเผยแพร่โดยอัตโนมัติเมื่อคุณใกล้ถึงข้อจำกัดเหล่านี้

| แพลตฟอร์ม | ข้อจำกัดรายชั่วโมง | หมายเหตุ |
|---|---|---|
| **Instagram** | 180 คำขอ | รวมโพสต์, สตอรี่ และการโต้ตอบผ่าน API |
| **Facebook** | 4,500 คำขอ | รวมโพสต์บนเพจ, ความคิดเห็น และการโต้ตอบผ่าน API |

:::note
ข้อจำกัดเหล่านี้ถูกกำหนดโดยแพลตฟอร์มโซเชียลมีเดียเองและอาจมีการเปลี่ยนแปลง Wink จะปรับเปลี่ยนอัตโนมัติเมื่อข้อจำกัดของแพลตฟอร์มเปลี่ยนแปลง
:::

## วิธีที่ Wink จัดการข้อจำกัดอัตราการใช้งาน

Wink ใช้หน้าต่างเวลาที่เลื่อนเพื่อเก็บข้อมูลการใช้งาน API ของคุณในแต่ละแพลตฟอร์ม:

1. **แต่ละคำขอจะถูกประทับเวลาขณะส่งไปยังแพลตฟอร์ม**
2. **คำขอที่เก่ากว่า 1 ชั่วโมงจะถูกลบออกจากหน้าต่างโดยอัตโนมัติ**
3. **ก่อนส่งคำขอใหม่แต่ละครั้ง** Wink จะตรวจสอบว่าหน้าต่างเวลาปัจจุบันยังมีความจุเหลืออยู่หรือไม่
4. **หากถึงขีดจำกัด** Wink จะจัดคิวคำขอและส่งเมื่อมีความจุว่าง

ซึ่งหมายความว่าคุณไม่ต้องกังวลเกี่ยวกับการเกินข้อจำกัดของแพลตฟอร์มหรือการถูกติดธงบัญชี — Wink จะจัดการอย่างโปร่งใส

## แนวทางปฏิบัติที่ดีที่สุด

- **กระจายโพสต์ของคุณ** แทนที่จะตั้งเวลาหลายโพสต์พร้อมกัน ให้กระจายโพสต์ตลอดทั้งวัน วิธีนี้ดูเป็นธรรมชาติมากขึ้นสำหรับแพลตฟอร์มและผู้ชมของคุณ
- **ใช้ปฏิทิน** [Social Calendar](/th/social/calendar) ช่วยให้คุณเห็นภาพรวมตารางการโพสต์และหลีกเลี่ยงการโพสต์ที่กระจุกตัว
- **ติดตามวิเคราะห์** ตรวจสอบ [Social Analytics](/th/social/analytics) เพื่อดูจำนวนโพสต์ที่เผยแพร่และว่ามีโพสต์ใดถูกเลื่อนเนื่องจากข้อจำกัดอัตราการใช้งานหรือไม่
