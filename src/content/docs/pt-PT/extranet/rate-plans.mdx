Os planos tarifários são o filtro através do qual os quartos são apresentados aos utilizadores.
Contêm também definições que controlam a estrutura de preços em determinados cenários.

Os planos tarifários permitem-lhe:

- Definir uma política de cancelamento [e exceções à política].
- Preço para crianças extra, adultos e viajantes individuais.
- Definir refeições.
- Adicionar encargos por check-in antecipado e check-out tardio.
- Definir encargos adicionais.
- Definir restrições.

Para criar um plano tarifário, clique no botão `Create a new rate plan`.

## Configuração

A aba `Configurations` está selecionada por defeito quando começa.

Esta secção cobre as opções básicas para configurar o seu plano tarifário.

- **Nome do plano tarifário** O nome do plano tarifário, como o referencia internamente. *ex. BAR RO*
- **Pontos de fidelidade** Se o cliente acumula os seus pontos de fidelidade neste plano tarifário. *ex. Não*
- **Ocupante individual** Defina um modificador de preço para um ocupante individual num quarto de ocupação múltipla. *ex. 5% DE DESCONTO no preço regular*
- **Adulto extra** Defina um modificador de preço para um adulto extra no quarto. *ex. +10 / adulto*
- **Criança extra** Defina um modificador de preço para uma criança extra no quarto. *ex. +5 / criança*

### Política de cancelamento

- **Política de cancelamento** Defina a política de cancelamento para este plano tarifário. *ex. Reembolsável*
- **Exceções** Defina políticas de cancelamento adicionais que entram em vigor para intervalos de datas específicos. *ex. Para o mês de dezembro, use a política `Reembolsável com condições extra`.*

### Refeições

Defina as refeições que acompanham este plano tarifário. *ex. Pequeno-almoço incluído*

## Encargos

Defina encargos adicionais que acompanham o plano tarifário.

- **Check-in antecipado** Defina um encargo adicional quando o cliente quiser fazer check-in antecipado. *Isto é apenas para uso informativo*
- **Check-out tardio** Defina um encargo adicional quando o cliente quiser fazer check-out tardio. *Isto é apenas para uso informativo*

### Encargos extra

Defina encargos adicionais com o plano tarifário.

Para criar um encargo extra, clique no botão `Create extra charge`.

- **Tipo de preço** Escolha como este encargo é calculado. *ex. Por estadia*
- **Montante** Defina o valor fixo do encargo. *ex. 100*

Preencha o nome e a descrição deste encargo.

:::note
É necessário escrever pelo menos uma descrição e esta deve estar em inglês.
:::

- **Nome** Nome do encargo. *ex. Encargo único de limpeza*
- **Descrição** Descreva o encargo em um ou dois parágrafos. *ex. A vila será limpa pela nossa equipa profissional de limpeza antes da sua chegada.*

## Restrições

Defina restrições que controlam se um quarto / plano tarifário está disponível ou não.

Restrições suportadas:

- Duração da estadia
- Reserva antecipada
- Ocupação
- Idade
- Datas de venda / estadia / reserva / chegada / partida / obrigatórias

### Duração da estadia

Ao adicionar esta restrição, está a exigir que os clientes "fiquem pelo menos" e "no máximo" um número especificado de dias.

*Exemplo: Este tipo de quarto está disponível apenas para clientes que fiquem mais de 10 dias. Nota: O máximo da duração da estadia é opcional.*

### Reserva antecipada

Ao adicionar esta restrição, está a exigir que os clientes "reservem pelo menos" e "no máximo" um número especificado de dias com antecedência.

*Exemplo: Esta promoção está disponível apenas para clientes que reservem com mais de 10 dias de antecedência. Nota: Também é necessário definir uma reserva máxima antecipada.*

### Ocupação

Ao adicionar esta restrição, está a exigir que o grupo que chega tenha "pelo menos" e "no máximo" o número especificado de pessoas.

### Idade

Ao adicionar esta restrição, está a exigir que os clientes tenham "pelo menos" e "no máximo" a idade especificada.

### Datas de venda

Ao adicionar esta restrição, pretende tornar esta tarifa disponível para reserva apenas dentro de um intervalo de datas específico. Um cliente só verá esta tarifa se estiver a reservar um quarto NA data especificada dentro deste intervalo.

### Datas de estadia

Ao adicionar esta restrição, pretende tornar esta tarifa disponível para estadia apenas dentro de um intervalo de datas específico. Um cliente só verá esta tarifa se estiver a reservar um quarto PARA a data especificada dentro deste intervalo.

### Datas de reserva

:::note
Esta restrição funciona bem com a "restrição de datas de venda", onde pode limitar um intervalo de datas e restringir ainda mais os dias da semana usando esta restrição.
:::

Ao ativar esta restrição, está a permitir que esta tarifa seja pesquisada APENAS nos dias especificados.

### Datas de chegada

:::note
Esta restrição funciona bem com a "restrição de datas de estadia", onde pode limitar o intervalo de datas e restringir ainda mais os dias da semana usando esta restrição.
:::

Ao ativar esta restrição, os clientes DEVEM chegar nesse dia para ver esta tarifa.

*Exemplo: Ativar segunda-feira exige que o cliente chegue numa segunda-feira.*

### Datas de partida

:::note
Esta restrição funciona bem com a "restrição de datas de estadia", onde pode limitar o intervalo de datas e restringir ainda mais os dias da semana usando esta restrição.
:::

Ao ativar esta restrição, os clientes DEVEM partir nesse dia para ver esta tarifa.

*Exemplo: Ativar segunda-feira exige que o cliente parta numa segunda-feira.*

### Datas obrigatórias

:::note
Esta restrição funciona bem com a "restrição de datas de estadia", onde pode limitar o intervalo de datas e restringir ainda mais os dias da semana usando esta restrição.
:::

Ao ativar esta restrição, os clientes DEVEM permanecer "pelo menos um" destes dias para ver esta tarifa.

*Exemplo: Ativar segunda-feira exige que os clientes permaneçam numa segunda-feira.*
