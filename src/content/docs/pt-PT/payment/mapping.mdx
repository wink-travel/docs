Se é um integrador, como a Wink, e quer mapear os seus beneficiários externos para usar com o TripPay, pode fazê-lo de duas formas:

- Mapeamento manual
- Usando a [Payment API](/pt-PT/developers/apis/#payment-api) (*Veja `Account Mappings`*)

:::tip
A menos que esteja a lidar apenas com um pequeno número das mesmas propriedades, recomendamos que use a API para automatizar este passo.
:::

A seguir, mostramos-lhe o método manual para mapear um beneficiário externo. *Este método usa a API nos bastidores.*

<Steps>
1. Inicie sessão no [TripPay](https://pay.wink.travel).
2. Selecione a sua conta de integrador.
3. Clique em `Settings > Mappings` na barra de navegação principal.
4. Clique no botão `Create mapping`. Vai ver o formulário de mapeamento.
5. **Local account** Comece a escrever o nome da conta TripPay existente que quer mapear.
6. **External identifier** Cole o seu ID externo para esta entidade, tal como existe no sistema remoto.
7. Clique em `Save` para continuar. Será redirecionado de volta para a sua lista de mapeamentos.
</Steps>

Com os seus mapeamentos configurados, o TripPay sabe agora quais contas creditar como beneficiários de uma reserva quando as referir usando os seus identificadores externos.

## API

Os desenvolvedores que quiserem gerir `Payments` devem ir a [Developers > API > Payments](/pt-PT/developers/apis/#payment-api).
