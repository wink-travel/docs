Se for um integrador, como o Wink, e quiser mapear os seus beneficiários externos para utilização com o TripPay, pode fazê-lo de duas formas:

* Mapeamento manual
* Usando o [API de pagamento](/developers/apis/#payment-api) (*Ver `Account Mappings`*)

:::dica
A menos que esteja apenas a lidar com algumas das mesmas propriedades, recomendamos que utilize a API para automatizar esta etapa.
:::

Abaixo, mostraremos como mapear manualmente um beneficiário externo. *Usa a API por baixo dos panos.*

<Steps>
  1. Entrar [TripPay](https://pay.wink.travel).
  2. Selecione a sua conta de integrador.
  3. Clique `Settings > Mappings` na barra de navegação principal.
  4. Clique no `Create mapping` botão. Verá o formulário de mapeamento.
  5. **Conta local** Comece a digitar o nome do TripPay existente que pretende mapear.
  6. **Identificador externo** Cole o seu ID externo para esta entidade tal como existe no sistema remoto.
  7. Clique `Save` para continuar. Será redirecionado de volta para a sua lista de mapeamentos.
</Steps>


Com os seus mapeamentos em vigor, o TripPay sabe agora quais as contas a creditar como beneficiárias de uma reserva quando se refere a elas utilizando os seus identificadores externos.

Os desenvolvedores que desejam gerir `Payments`, Vá para [Programadores > API > Pagamentos](/developers/apis/#payment-api).
