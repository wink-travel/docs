Nem todos os dias têm o mesmo preço. Eventos, feriados, épocas de menor procura e lacunas de última hora exigem preços diferentes. Este guia mostra como usar tarifas especiais e restrições de planos tarifários para gerir estes cenários.

## Preços para eventos

Quando uma conferência, festival ou evento local aumenta a procura, vai querer aumentar as tarifas para essas datas.

<Steps>
1. Vá a `Monetize > Special Rates` no seu Extranet.
2. Clique em `Create a new special rate`.
3. Selecione a tarifa base e o tipo de quarto a substituir.
4. Defina o intervalo de datas para cobrir o período do evento.
5. Introduza os preços diários ajustados (mais altos).
6. Clique em `Save`.
</Steps>

A tarifa especial substitui a tarifa base apenas para essas datas. A sua tarifa regular mantém-se inalterada fora do período do evento.

## Ajustes sazonais

Para períodos mais longos (alta temporada, meia temporada, baixa temporada), pode:

- **Usar tarifas especiais** — Criar substituições para cada estação com preços ajustados.
- **Atualizar tarifas base** — Ajustar diretamente as tarifas base para esses períodos.

As tarifas especiais são melhores para ajustes temporários, enquanto atualizar as tarifas base faz sentido para padrões anuais previsíveis.

## Tarifas de última hora

Para preencher quartos não vendidos perto da data de check-in:

<Steps>
1. Crie um novo **plano tarifário** com uma restrição de reserva antecipada (por exemplo, deve reservar com no máximo 3 dias de antecedência).
2. Associe-o a uma **tarifa especial** que ofereça um desconto para as datas alvo.
3. Este plano tarifário só aparecerá para viajantes que reservem dentro da janela de última hora.
</Steps>

## Datas de bloqueio

Para bloquear certas datas num plano tarifário específico (por exemplo, sem tarifas promocionais durante a época alta):

<Steps>
1. Vá a `Monetize > Rate Plans` e selecione o plano tarifário.
2. Navegue até à secção **Restrictions**.
3. Adicione uma restrição de **Sell date** para limitar quando a tarifa pode ser reservada.
4. Ou use restrições de **Stay date** para limitar quando a tarifa é válida.
</Steps>

Por exemplo, pode restringir o seu plano tarifário "Early Bird" para que não esteja disponível para reservas durante dezembro (época alta), enquanto o seu plano tarifário regular permanece aberto.

## Combinar restrições

As restrições podem ser combinadas para um controlo preciso:

| Restrição | O que controla |
| -- | -- |
| **Sell dates** | Quando a tarifa pode ser reservada |
| **Stay dates** | Quais as datas de estadia que a tarifa cobre |
| **Available days** | Quais os dias da semana em que a tarifa aparece |
| **Arrival days** | Em que dias o hóspede deve chegar |
| **Departure days** | Em que dias o hóspede deve partir |
| **Required days** | O hóspede deve ficar pelo menos um destes dias |
| **Advance booking** | Com que antecedência a reserva deve ser feita |
| **Length of stay** | Número mínimo e máximo de noites |
