O Vue 3 precisa de uma dica ao compilador para tratar as tags `<wink-*>` como elementos personalizados nativos em vez de componentes Vue não resolvidos. Depois disso, funcionam como qualquer outro elemento HTML.

<LinkCard
  title="@wink/elements npm package"
  href="/integrations/npm"
  description="Instale e carregue o bundle CDN com tipos TypeScript."
/>

## Instalar

```bash
npm install @wink/elements
```

## Configurar o compilador

### Vite + Vue (`vite.config.ts`)

```ts
import { defineConfig } from 'vite';
import vue from '@vitejs/plugin-vue';

export default defineConfig({
  plugins: [
    vue({
      template: {
        compilerOptions: {
          // Tratar todas as tags que começam com "wink-" como elementos personalizados
          isCustomElement: (tag) => tag.startsWith('wink-'),
        },
      },
    }),
  ],
});
```

### Vue CLI (`vue.config.js`)

```js
module.exports = {
  chainWebpack(config) {
    config.module
      .rule('vue')
      .use('vue-loader')
      .tap((options) => ({
        ...options,
        compilerOptions: {
          isCustomElement: (tag) => tag.startsWith('wink-'),
        },
      }));
  },
};
```

## Carregar CDN uma vez no arranque da app

```ts
// src/main.ts
import { createApp } from 'vue';
import App from './App.vue';
import { load } from '@wink/elements';

load({ clientId: import.meta.env.VITE_WINK_CLIENT_ID });

createApp(App).mount('#app');
```

## Usar em templates

```vue
<template>
  <main>
    <wink-content-loader layout="HOTEL" :id="layoutId" />
    <wink-lookup />
  </main>
</template>

<script setup lang="ts">
const layoutId = 'YOUR_LAYOUT_ID';
</script>
```

## TypeScript — aumentar tipos globais de elementos

Vue 3 + TypeScript lê os tipos de elementos personalizados do `HTMLElementTagNameMap` global. Adicione um ficheiro de declaração:

```ts
// src/wink-elements.d.ts
import type {
  WinkContentLoaderAttributes,
  WinkLookupAttributes,
  WinkSearchButtonAttributes,
  WinkAccountButtonAttributes,
  WinkItineraryButtonAttributes,
  WinkShoppingCartButtonAttributes,
  WinkAppLoaderAttributes,
} from '@wink/elements';

declare global {
  interface HTMLElementTagNameMap {
    'wink-content-loader': HTMLElement & WinkContentLoaderAttributes;
    'wink-lookup': HTMLElement & WinkLookupAttributes;
    'wink-search-button': HTMLElement & WinkSearchButtonAttributes;
    'wink-account-button': HTMLElement & WinkAccountButtonAttributes;
    'wink-itinerary-button': HTMLElement & WinkItineraryButtonAttributes;
    'wink-shopping-cart-button': HTMLElement & WinkShoppingCartButtonAttributes;
    'wink-app-loader': HTMLElement & WinkAppLoaderAttributes;
  }
}
```

## Variáveis de ambiente

Adicione a `.env.local`:

```
VITE_WINK_CLIENT_ID=your-client-id
```

O Vite expõe apenas variáveis prefixadas com `VITE_` para o bundle cliente.
