O motor de reservas está disponível em duas versões para suportar as diferentes necessidades de um afiliado, uma empresa de gestão de propriedades e um hotel.

## Agência de viagens online

Para um afiliado, faz mais sentido usar a versão de agência de viagens online. A razão é que o site está configurado para acomodar viajantes que querem ver várias propriedades.

Pode encontrar a página inicial desta aplicação abaixo.

<LinkButton href="https://ota.wink.travel/home" variant="secondary" icon="external" target="_blank">
  Wink OTA
</LinkButton>

## Motor de reservas na internet

Para um hotel ou uma empresa de gestão de propriedades, faz mais sentido usar a versão do motor de reservas na internet. Este tipo de utilizador está interessado em destacar uma única propriedade ou várias propriedades sob gestão.

Pode encontrar a página inicial desta aplicação abaixo.

<LinkButton href="https://book.wink.travel/home?client-id=client-1&mp;configuration-id=d906a163-5cee-4198-a170-2bedf2407f5f" variant="secondary" icon="external" target="_blank">
  Wink IBE
</LinkButton>

## Configuração

Na maioria das vezes, não é necessário partilhar diretamente nenhuma destas aplicações. Criámos várias formas para partilhar inventário de viagens, que pode descobrir em [Wink Studio](/studio/what-is-studio) ou através de [WinkLinks](/link-manager/wink-links).

No entanto, se quiser ligar manualmente a um destes sites, certifique-se de adicionar o seu ID de cliente e o ID de personalização à URL como parâmetros.

Pode encontrar o seu ID de cliente ao:

<Steps>
1. Iniciar sessão em [Wink Extranet](/extranet/what-is-extranet) ou [Wink Studio](/studio/what-is-studio).
2. Clicar no ícone do seu perfil no canto superior direito e clicar em [Applications](/developers/applications).
3. Usar a aplicação associada à conta Wink com a qual quer trabalhar.
4. Copiar o ID de Cliente dessa aplicação e colá-lo na URL como parâmetro de pedido. Exemplo: ?client-id=MY_CLIENT_ID
</Steps>

### Extranet

Pode encontrar o seu ID de personalização como propriedade ao:

<Steps>
1. Iniciar sessão em [Wink Extranet](/extranet/what-is-extranet).
2. Selecionar a propriedade com a qual quer trabalhar.
3. Na barra de navegação principal, clicar em `Account > Branding`
4. Copiar o ID de Personalização e colá-lo na URL como parâmetro de pedido. Exemplo: ?configuration-id=MY_CUSTOMIZATION_ID
</Steps>

### Studio

Pode encontrar o seu ID de personalização como afiliado ao:

<Steps>
1. Iniciar sessão em [Wink Studio](/studio/what-is-studio).
2. Selecionar a conta com a qual quer trabalhar.
3. Na barra de navegação principal, clicar em `Tools > Customization`
4. Copiar o ID de Personalização da personalização que quer usar e colá-lo na URL como parâmetro de pedido. Exemplo: ?configuration-id=MY_CUSTOMIZATION_ID
</Steps>

## OTA

O que obtém é uma URL que se parece com isto:

https://ota.wink.travel?client-id=MY_CLIENT_ID&configuration-id=MY_CUSTOMIZATION_ID

Se quiser ir diretamente para a página de um hotel, a URL seria algo assim:

https://ota.wink.travel/hotel/hotel-x?client-id=MY_CLIENT_ID&configuration-id=MY_CUSTOMIZATION_ID

:::note
Se não incluir um ID de cliente e/ou de configuração, o Wink OTA usará uma configuração predefinida.
:::

## IBE

Se for uma empresa de gestão de propriedades, quer ligar à página inicial do Wink IBE. A página inicial mostrará o seu tema designado juntamente com as [propriedades que representa](/studio/agency-representation).

O que obtém é uma URL que se parece com isto:

https://book.wink.travel?client-id=MY_CLIENT_ID&configuration-id=MY_CUSTOMIZATION_ID

Se for uma propriedade, quer ligar diretamente à página de destino da propriedade no Wink IBE.

Essa URL seria algo assim:

https://book.wink.travel/hotel/hotel-x?client-id=MY_CLIENT_ID&configuration-id=MY_CUSTOMIZATION_ID


:::note
Se não incluir um ID de cliente e/ou de configuração, o Wink IBE NÃO será carregado. Estes parâmetros são obrigatórios.
:::

## Exemplo: Página da propriedade

Cada propriedade no Wink tem a sua própria página de destino única em qualquer um dos sites. Todos os links externos direcionam os viajantes para uma destas páginas de destino.

A página de destino contém toda a informação que um viajante precisa para tomar uma decisão informada sobre se deve ou não ficar nesta propriedade.
