AI veščine so prenosni paketi navodil, ki učijo kodirne agente, kako delati z Wink. Namesto da bi se zanašali na učne podatke ali zastarelo dokumentacijo, vsaka veščina usmerja agenta, da uporablja [Wink MCP strežnik](/sl/developers/mcp) kot živ vir resnice za API pogodbe in platformno dokumentacijo.

Veščine delujejo z vsakim agentnim odjemalcem, ki jih podpira — Claude Code, Codex, Cursor in drugi.

## Na voljo veščine

### wink-api-builder

Implementirajte, pregledujte ali odpravljajte napake pri integracijah Wink API-jev. Veščina vodi agenta skozi strukturiran potek dela:

1. Poiščite pravo API operacijo preko `api_search`
2. Odprite žive pogodbe zahtevkov/odgovorov s MCP strežnika
3. Pridobite nastavitve avtentikacije iz dokumentacije
4. Napišite kodo na podlagi preverjene pogodbe
5. Označite morebitna neskladja med obstoječo kodo in trenutnim API-jem

Uporabno za:
- Izgradnjo novih metod odjemalcev za kateri koli Wink API
- Odpravljanje napak neskladij pogodb (npr. zahtevek, ki vrača HTTP 400)
- Pregled obstoječih integracij glede na živo specifikacijo

## Namestitev

Kopirajte mapo veščine v imenik veščin vašega agenta.

:::tip
Wink MCP strežnik mora biti povezan v istem okolju. Za navodila o nastavitvi glejte [MCP Server](/sl/developers/mcp).
:::

**Claude Code**

```bash
# lokalno v repozitoriju
cp -r skills/wink-api-builder .claude/skills/

# ali na uporabniški ravni
cp -r skills/wink-api-builder ~/.claude/skills/
```

Zaženite z:
```
/wink-api-builder add a TypeScript client for the Wink affiliate create flow
```

**Codex**

```bash
# lokalno v repozitoriju
cp -r skills/wink-api-builder .agents/skills/

# ali na uporabniški ravni
cp -r skills/wink-api-builder $CODEX_HOME/skills/
```

Zaženite z:
```
Use $wink-api-builder to add a TypeScript client for the Wink affiliate create flow.
```

**Drugi agenti**

Postavite mapo veščine kamor koli vaš agent išče veščine. Vhodna točka je `SKILL.md`.
