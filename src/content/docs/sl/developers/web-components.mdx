[Spletne komponente](https://developer.mozilla.org/en-US/docs/Web/API/Web_components) so standard, ki vam omogoča integracijo celotnih funkcionalnosti JavaScript v vašo spletno stran, ne da bi morali znati programirati.
Zahvaljujoč tej kul tehnologiji lahko Wink potovalni inventar vdelate z zelo malo truda. Ta članek vas vodi skozi našo zbirko spletnih komponent in vam pokaže, kako jih uporabljati.

## Vdelajte Wink

Obstajajo tri stvari, ki jih morate vključiti na katerokoli stran, kjer želite prikazati eno od naših spletnih komponent.

1. Vključite naš slogovni list.
2. Vključite naš Javascript.
3. Vključite naš aplikacijski nalagalnik.

### Slogovni list

Vključite naše CSS stile v glavo vašega dokumenta.

```html {3}
<html>
  <head>
    <link rel="stylesheet" href="https://elements.wink.travel/styles.css"></link>
  </head>
</html>
```

### Javascript

Vključite naš Javascript na dno vašega dokumenta. (*Priporočamo takoj nad končnim &lt;/body&gt; tagom*).

```html {3}
<html>
  <body>
    <script src="https://elements.wink.travel/elements.js" type="module" defer></script>
  </body>
</html>
```

Vaša stran je zdaj pripravljena in sposobna prikazovati naše spletne komponente.

## Komponente

Preberite o naši knjižnici komponent spodaj.

### Aplikacijski nalagalnik

Nalagalnik je odgovoren za ohranjanje stanja in upravljanje interakcije med našimi komponentami. (*Vključite ga pod naš Javascript*).

```html {3-6}
<html>
  <body>
    <wink-app-loader
      client-id="VAŠ CLIENT ID TUKAJ"
      configuration-id="VAŠ ID PRILAGODITVE TUKAJ"
    ></wink-app-loader>
  </body>
</html>
```

### Nalagalnik vsebine

Nalagalnik vsebine je srce naše knjižnice spletnih komponent. Odgovoren je za prikaz vašega inventarja (kartice, mreže, zemljevide).

Na voljo so atributi:

- layout
- id
- sort

*Atribut `sort` je na voljo samo, ko je layout `RANKED` in ne želite uporabiti obstoječe rangirane mreže. V teh primerih pustite `id` prazno.*

Na voljo so tipi postavitve:

- `AD_BANNER`
- `MAP`
- `HOTEL`
- `GUEST_ROOM`
- `MEETING_ROOM`
- `SPA`
- `RESTAURANT`
- `ACTIVITY`
- `ATTRACTION`
- `PLACE`
- `ADD_ON`
- `LIST`
- `SEARCH`
- `RANKED`

Na voljo so tipi sortiranja:

- `MEMBER`
- `PRICE_LOW_TO_HIGH`
- `PRICE_HIGH_TO_LOW`
- `PRICE`
- `POPULARITY`
- `ECO`
- `EXPERIENCE`
- `PERK`
- `LOYALTY`
- `PACKAGE`

```html {3-6}
<html>
  <body>
    <wink-content-loader
       layout="GUEST_ROOM"
       id="2de7ee9c-61c9-11ef-9722-42004e494300"
    ></wink-content-loader>
  </body>
</html>
```

Vzorce lahko najdete za [kartice](/sl/studio/cards), [mreže](/sl/studio/grids) in [zemljevide](/sl/studio/maps).

### Iskanje

Iskanje odlično deluje skupaj z eno od vaših rangiranih mrež. Omogoča uporabnikom iskanje hotela in destinacije, mreža pa se posodobi glede na njihov rezultat iskanja.

```html {3}
<html>
  <body>
    <wink-lookup></wink-lookup>
  </body>
</html>
```

1. Vključite naš slogovni list.
2. Vključite naš Javascript.

### Slogovni list

Vključite naše CSS stile v glavo vašega dokumenta.

```html {3}
<html>
  <head>
    <link rel="stylesheet" href="https://elements.trippay.io/styles.css"></link>
  </head>
</html>
```

### Javascript

Vključite naš Javascript na dno vašega dokumenta. (*Priporočamo takoj nad končnim &lt;body&gt; tagom*).

```html {3}
<html>
  <body>
    <script src="https://elements.trippay.io/elements.js" type="module" defer></script>
  </body>
</html>
```

Vaša stran je zdaj pripravljena in sposobna prikazovati naše spletne komponente.

## Komponente

### Plačilo

Komponenta za plačilo omogoča, da TripPay obvestite, da potnik želi nekaj kupiti, ter pravila in cene za te izdelke.

Obvezni atribut za pripomoček je:

- `id` Identifikator pogodbe o rezervaciji, ki jo želite izvesti.

```html {3}
<html>
  <body>
    <trip-pay id="<INSERT_BOOKING_CONTRACT_ID>"></trip-pay>
  </body>
</html>
```

:::note
Za učenje, kako ustvariti `id` pogodbe o rezervaciji, preberite vodič: [Integracija s TripPay](/sl/guides/integrating-with-trip-pay).
:::

Pripomoček pripravi pogodbo za izvedbo in prikaže podrobnosti plačila *(Slika 1)* uporabniku za dokončanje rezervacije.

## Napake

Če vas ob vdelavi ene od naših spletnih komponent, bodisi na WinkLinks ali na vaši lastni spletni strani, pozdravi sporočilo o napaki, je lahko nekaj stvari šlo narobe:

### Razpoložljivost

Inventar morda ni več na voljo. Pojdite na [Wink Studio](/sl/studio/cards) in preverite, ali je status "kartice" na voljo.
Če ni, bo prikazana rdeča barva. V tem primeru lahko počakate in morda dobavitelj ponovno omogoči dostop ali pa jo odstranite s svojega seznama.

:::note
`Razpoložljivost` ne pomeni dnevne razpoložljivosti inventarja za rezervacijo. Pomeni, da je inventar popolnoma nedostopen za vas in/ali kogarkoli drugega.
:::

### Odstranjeno

Inventar je morda bil odstranjen. Pojdite na [Wink Studio](/sl/studio/cards) in preverite, ali je status "kartice" na voljo.
Če ni, bo prikazana rdeča barva. V tem primeru lahko počakate in morda dobavitelj ponovno omogoči dostop ali pa jo odstranite s svojega seznama.

### Prilagoditev

Morda ste po nesreči odstranili prilagoditev, ki ste jo definirali za vaš inventar. Prepričajte se, da je prilagoditev še vedno na voljo in nastavite novo, če katera manjka.

### Aplikacija

To je namenjeno samo spletnim razvijalcem. Če ste po nesreči odstranili [Aplikacijo](/sl/developer/applications) in s tem onemogočili dostop do client ID, prosimo, ustvarite novo aplikacijo in uporabite nov client ID za vdelavo naših spletnih komponent.
