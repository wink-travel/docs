## Pregled
Gostujoči Wink MCP strežnik je na voljo na:

```
https://docs.mcp.wink.travel/mcp
```

Usmerite združljive AI odjemalce (VS Code, Cursor, druge, ki uporabljajo MCP HTTP transport) na to končno točko, da jim omogočite:
- Brskanje po dokumentaciji izdelkov in platforme (`wink://docs/...`)
- Brskanje po živih OpenAPI specifikacijah in dokumentaciji operacij (`wink://api/...`)
- Izvajanje lahkih semantičnih / ključnih iskanj po dokumentaciji in živih API-jih

## Transport
Uporabite glavno pretočno HTTP končno točko:

```
https://docs.mcp.wink.travel/mcp
```

Če vaš odjemalec podpira samo SSE, uporabite:

```
https://docs.mcp.wink.travel/sse
```

### Claude Code
Uradna dokumentacija: [Povežite Claude Code z orodji prek MCP](https://code.claude.com/docs/en/mcp)

Anthropic priporoča oddaljeni HTTP transport za oblačno gostujoče MCP strežnike:
```bash
claude mcp add --transport http wink-docs https://docs.mcp.wink.travel/mcp
claude mcp list
```

V Claude Code zaženite `/mcp` za pregled stanja strežnika. Če želite strežnik deliti z vsem repozitorijem, dodajte `--scope project`, da Claude Code zapiše konfiguracijo v `.mcp.json`.

### Codex
Uradna dokumentacija: [Docs MCP hitri začetek za Codex](https://developers.openai.com/resources/docs-mcp)

OpenAI dokumentira nastavitev MCP prek Codex CLI in navaja, da je konfiguracija deljena z IDE razširitvijo:
```bash
codex mcp add wink-docs --url https://docs.mcp.wink.travel/mcp
codex mcp list
```

### Cursor
Cursor MCP dokumentacija: https://cursor.com/docs/context/mcp
Nastavitev: Dodajte oddaljeni strežnik prek MCP nastavitev.
```json
{
  "mcpServers": {
    "wink-docs": {
      "url": "https://docs.mcp.wink.travel/mcp",
    }
  }
}
```

### VS Code
Uradna dokumentacija: https://code.visualstudio.com/docs/copilot/customization/mcp-servers
Minimalna oddaljena konfiguracija (uporabniška ali delovna `mcp.json`):
```json
{
  "servers": {
    "wink-docs": {
      "type": "http",
      "url": "https://docs.mcp.wink.travel/mcp"
    }
  }
}
```

Namigi:
- Uporabite Command Palette: “MCP: List Servers” za potrditev povezave in brskanje po orodjih.
- Izbirnik orodij omogoča vklop/izklop `api_search` in `docs_search`.
- Dodajte kontekst: Pogled klepeta → Dodaj kontekst → MCP viri za pripenjanje strani dokumentacije.

## Osnovni URI-ji virov

| URI | Opis |
|-----|-------------|
| `wink://docs` | Indeks dokumentacije (združen po kategorijah) |
| `wink://docs/{path}` | Specifičen dokumentacijski članek (Markdown / MDX) |
| `wink://api` | Živi OpenAPI katalog, odkrit iz Winkove swagger konfiguracije |
| `wink://api/{apiName}` | Povzetek API-ja z avtentikacijo, oznakami, glavi in primeri operacij |
| `wink://api/{apiName}/operations` | Celoten seznam operacij združen po oznakah |
| `wink://api/{apiName}/operation/{key}` | Podrobnosti posamezne operacije (parametri, telo zahteve, odgovori) |
| `wink://api/{apiName}/schema/{schemaName}` | Razrešena komponenta sheme z gnezdenimi polji, tipi in primeri |
| `wink://api/{apiName}/operation/{key}/request` | Pogled osredotočen na zahtevo z obveznimi vnosi in minimalnim primerom |
| `wink://api/{apiName}/operation/{key}/response/{status}` | Pogled osredotočen na odgovor za določen statusni kodo |
| `wink://api/{apiName}/operation/{key}/recipe` | Naloga usmerjen potek za pravilno klicanje te operacije |

## Orodja za iskanje (hitri pregled)
| Orodje | Namen | Tipični primeri poizvedb |
|------|---------|-----------------------|
| `api_search` | Najdi živi API ali operacijo za integracijo | `create affiliate`, `payment receipt`, `lookup inventory`, `invite manager` |
| `docs_search` | Najdi konceptualno dokumentacijo / vodiče | `authentication`, `webhooks`, `wordPress` |

Pustite `query` prazno za prejem predlogov. Vsak rezultat vključuje MCP URI, ki ga lahko odprete.
Za zadetke operacij `api_search` vrne tudi povezane URI-je zahtev, odgovorov, shem in receptov, da lahko agenti neposredno skočijo na najbolj uporabne podvire.

## Tipičen vzorec uporabe
1. Zaženite `api_search` z vašo namero (npr. *"create affiliate"* ali *"payment receipt"*).
2. Odprite vrnjeni `wink://api/...` URI ali sledite njegovim povezanih poljem `request_uri`, `response_uris`, `request_schema_uris`, `response_schema_uris` in `recipe_uri`.
3. Uporabite `docs_search` za podporne koncepte (avtentikacija, webhooks, delovni tokovi).

## AI veščine

Vnaprej pripravljene veščine agentov uporabljajo ta MCP strežnik kot svoj vir resnice. Veščine dajejo vašemu kodirnemu agentu strukturiran potek dela za gradnjo na Wink API-jih — namesto ugibanja končnih točk agent bere žive pogodbe neposredno s MCP strežnika.

Oglejte si [AI Skills](/sl/developers/ai-skills) za razpoložljive veščine in navodila za namestitev.

## Opombe
- Vsak vrnjeni URI lahko agent odpre za celotno vsebino.
- Drevo `wink://api` je živo in sledi trenutnemu zgornjemu OpenAPI katalogu namesto generiranemu indeksu repozitorija.
- Predlogi se samodejno prikažejo, ko so poizvedbe prazne.
