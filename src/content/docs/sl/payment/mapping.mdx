Če ste integrator, kot je Wink, in želite svoje zunanje prejemnike plačil povezati s TripPayom, lahko to storite na enega od dveh načinov:

* Ročno kartiranje
* Uporaba[API za plačila](/developers/apis/#payment-api)(*Glej`Account Mappings`*)

:::nasvet
Razen če imate opravka le s peščico enakih lastnosti, priporočamo, da za avtomatizacijo tega koraka uporabite API.
:::

Spodaj vas bomo vodili skozi ročni postopek preslikave zunanjega prejemnika plačila.*Uporablja API pod pokrovom motorja.*

<Steps>
  1. Prijava [TripPay](https://pay.wink.travel).
  2. Izberite svoj integracijski račun.
  3. Kliknite `Settings > Mappings` iz glavne navigacijske vrstice.
  4. Kliknite na `Create mapping` gumb. Videli boste obrazec za preslikavo.
  5. **Lokalni račun** Začnite tipkati ime obstoječega TripPaya, ki ga želite preslikati.
  6. **Zunanji identifikator** Prilepite svoj zunanji ID za to entiteto, kot obstaja v oddaljenem sistemu.
  7. Kliknite `Save` za nadaljevanje. Preusmerjeni boste nazaj na seznam preslikav.
</Steps>


Ko so vaše povezave vzpostavljene, TripPay zdaj ve, katerim računom naj pripiše dobropis kot upravičencem rezervacije, ko se nanje sklicujete z zunanjimi identifikatorji.

## API

Razvijalci, ki želijo upravljati`Payments`, pojdi na[Razvijalci > API > Plačila](/developers/apis/#payment-api).
