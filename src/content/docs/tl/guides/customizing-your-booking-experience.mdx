Kinokontrol ng mga customization kung paano ipinapakita ang iyong imbentaryo sa mga biyahero — ang mga kulay ng tema, default na pera, wika, itinerary, at iba pa. Ito ay inilalapat sa mga shareable links, embedded cards, grids, maps, at ang booking engine.

## Ano ang isang customization?

Ang customization ay isang configuration profile na nagtatakda ng:

- **Display currency** — Anong pera ang ipinapakita sa mga presyo.
- **Display language** — Ang default na wika para sa nilalaman.
- **Default itinerary** — Pre-filled na mga petsa ng check-in/out at bilang ng mga bisita.
- **Theme** — Mga kulay at branding na inilalapat sa mga embedded na bahagi.
- **Configuration** — Paano ipinapakita ang imbentaryo (layout, pagsasaayos).

Maaari kang gumawa ng maraming customization para sa iba't ibang merkado o gamit.

## Paggawa ng isang customization

<Steps>
1. Pumunta sa https://studio.wink.travel (o ang iyong account sa https://my.wink.travel).
2. Mag-navigate sa `Customization` mula sa sidebar.
3. I-click ang `Create a new customization`.
4. Bigyan ito ng isang deskriptibong pangalan (hal., "Europe - EUR", "Japan Market", "Default").
5. Itakda ang display currency, wika, at default itinerary.
6. I-configure ang mga kulay ng tema upang tumugma sa iyong branding.
7. I-click ang `Save`.
</Steps>

## Saan ginagamit ang mga customization

Kapag nagawa na, ang mga customization ay tinutukoy sa buong platform:

| Feature | Paano inilalapat ang customization |
| -- | -- |
| **Shareable links** | Ginagamit ng link ang pera, wika, at itinerary ng customization bilang default |
| **Embedded cards** | Ang mga card ay nagre-render gamit ang iyong mga kulay ng tema at mga setting ng display |
| **Grids and maps** | Sinusunod ng mga grid ng imbentaryo at mga marker sa mapa ang customization |
| **Booking engine** | Ginagamit ng buong booking flow ang mga setting ng customization |
| **Social posts** | Maaaring gamitin ng AI-generated posts ang isang partikular na customization para sa pagpapakita ng presyo |

## Maramihang mga customization

Isang karaniwang pattern ang paggawa ng isang customization para sa bawat target na merkado:

- **"US Market"** — USD, English, default na 2-night stay.
- **"Europe"** — EUR, English, default na 3-night stay.
- **"Japan"** — JPY, Japanese, default na 1-night stay.

Kapag gumawa ka ng shareable link o nag-embed ng card, pipiliin mo kung aling customization ang gagamitin. Ibig sabihin nito, ang parehong property ay maaaring ipakita nang iba depende sa audience.

## Pag-override gamit ang URL

Maaari mo ring i-override ang mga default ng customization gamit ang mga URL parameter sa mga shareable link:

- `l=es` — I-override ang wika sa Spanish.
- `c=EUR` — I-override ang pera sa EUR.
- `sd=2024-08-24&n=2` — I-override ang itinerary sa isang partikular na petsa at bilang ng mga gabi.

Tingnan ang [The Book Now Button](/tl/guides/the-book-now-button) para sa buong listahan ng mga URL parameter.
