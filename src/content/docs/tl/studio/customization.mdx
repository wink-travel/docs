Ang pagpapasadya ay isang malakas na tampok na magagamit mo upang kontrolin ang bawat aspeto ng pagsasama sa Wink nang hindi kinakailangang sumulat ng anumang code. Ang bawat piraso ng imbentaryo na iyong ibinebenta ay maaaring ipasadya. Magdagdag o mag-alis ng mga feature, baguhin ang scheme ng kulay at marami pang iba.

Habang nasa `Customization` pahina, i-click ang `Actions` button sa ilalim ng customization na gusto mong gamitin at i-click `Update`.
*Mayroon ka nang default na entry na magagamit mo kapag ginawa mo ang iyong account. Hindi maalis ang entry na ito.*

Dadalhin ka nito sa form ng pagpapasadya kung saan makikita mo ang 5 - 7 tab (*depende sa uri ng iyong account*).

## Heneral

Narito ang mga opsyon na magagamit mo sa ilalim ng `General` tab.

* **Pangalan** Bigyan ang iyong pagpapasadya ng isang pangalan upang madaling matandaan. *hal. Para sa aking travel blog*
* **Ipakita ang pera** Piliin ang unang currency na makikita ng mga user na ipinapakita sa tabi ng mai-book na imbentaryo ng paglalakbay. *hal. USD*
* **Ipakita ang wika** Piliin ang unang wikang makakaharap ng mga gumagamit. *hal. Ingles*
* **Idagdag sa pamagat ng pahina** Makokontrol mo kung ano ang idaragdag sa pamagat ng pahina (*ang pamagat na nakikita mo sa tab ng browser*). Ito ay pangunahing ginagamit para sa aming booking engine. *hal. Kung magna-navigate ka sa https://ota.wink.travel, bilang default, makikita mo `Home | Traveliko - Hotels with Benefits`*. Maaari mong baguhin ang lahat pagkatapos ng '|' tubo.

## Tema

Ang `Theme` Hinahayaan ka ng tab na i-customize ang hitsura at pakiramdam ng aming [booking engine](/booking-engine/in-a-nutshell) website, [WinkLinks](/link-manager/wink-links) website at lahat ng aming [Mga Bahagi ng Web](/developers/web-components).

### Configuration

Hinahayaan ka ng seksyong ito na i-configure ang disenyo at ang layout ng card. Sa ngayon, mayroon lamang isang disenyo; si Wink.

Kinokontrol ng layout ng card kung gusto mong makita ng iyong mga user nang pahalang o patayo ang mga transactional card. Ang impormasyon ay pareho.

:::tandaan
Walang pagkakaiba sa layout ng card kapag tinitingnan ang aming mga transactional card mula sa isang mobile device. Gayunpaman, makikita mo ang pagkakaiba sa isang iPad mini at mas malalaking resolution.
:::

:::tip\[Pro tip]
Gumamit ng pahalang na layout kung mayroon ka lang < 4 na imbentaryo ng paglalakbay (i.e. mga uri ng kuwarto) sa isang listahan... mas maganda lang ang hitsura nito.
:::

### Mga kulay

Kinokontrol ng kulay kung paano nakikita ng iyong mga user ang alinman sa aming mga bahagi ng UI at mga web site. Gamitin ito upang tumugma sa iyong kasalukuyang scheme ng kulay \[kung mayroon man].

Mayroon na kaming malaking seleksyon ng mga paunang natukoy na kulay na maaari mong simulan upang paglaruan. Ang mga paunang natukoy na tema ay babaguhin ang mga kulay sa mga field ng form sa ibaba. Maaari mo ring gamitin ang iyong sariling mga kulay.

## Analytics

Sinusuportahan namin ang GA4 analytics out of the box. Kung gusto mong subaybayan ang iyong buong pinahusay na landas ng conversion ng ecommerce, mula simula hanggang katapusan, ihanda ang iyong measurement ID at ilagay ito sa field sa ilalim ng `Analytics` tab.

## Mga mapa

Upang mai-embed ang aming mga mapa sa iyong site sa pamamagitan ng paggamit ng aming [WordPress plugin](/developers/wordpress) o [Mga Bahagi ng Web](/developers/web-components), kailangan mong ibigay sa amin ang iyong sariling Google Maps API key.

:::tandaan
[Gumawa ng Google Maps API key dito](https://console.developers.google.com/projectselector2/apis/credentials?supportedpurview=project)
:::

Kapag nabuo mo na ang iyong key, at na-configure mo ito para sa iyong site, ilagay ito sa ilalim ng `Analytics` tab.

## Itinerary

Ang `Itinerary` Ang tab ay marahil ang pinakakawili-wili sa mga tuntunin ng pagtiyak na lalabas ang mga presyo para sa imbentaryo na iyong pinili.
Dahil medyo kumplikado ang pagpepresyo ng imbentaryo sa paglalakbay, kailangang malaman ng Wink ang itinerary ng user bago makapagpakita ng tumpak na presyo.

Hinahayaan ka ng seksyong ito na itakda ang default na itinerary upang ipakita ang mga presyo para sa lahat ng aming digital na real estate, gaya ng:

* Ang aming mga social-friendly na larawan na may overlay ng presyo.
* WinkLinks grids, card at link.
* Mga Bahagi ng Web
* Booking engine

### Mga petsa

Maaari mong piliin kung gusto mong magtakda ng static na hanay ng petsa o isang rolling.

* Pumili ng static na hanay ng petsa kung gagamitin mo ang pag-customize para sa isang partikular na kaganapan na magaganap sa isang partikular na petsa.
* Pumili ng rolling date range kung gusto mong:
  1. Default sa ngayon + 1 gabi, na ginagawa ng karamihan sa mga OTA na O.
  2. Default sa isang petsa sa hinaharap na dapat palaging may availability.

### Configuration ng kwarto

Bilang default, nakatakda ang itinerary sa 2 matanda, dahil ito ang pinakakaraniwang setting para sa karamihan ng mga manlalakbay.
Madali itong mababago at maaari kang magdagdag ng maraming silid na may maraming nakatira; parehong matatanda at bata.

### Mga promosyon

Ang mga promosyon ay isang magandang feature na nakakatipid sa iyong mga user ng ilang oras. Kung ang isang propery ay nagbigay sa iyo ng isang promo code na maaari mong ibahagi sa iyong mga kaibigan, maaari mong piliing ilagay ito dito.
Awtomatikong makukuha ng sinumang papasok sa iyong link, o katulad, ang promo code na idinagdag sa kanilang itinerary at makikita nilang awtomatikong nalalapat ang may diskwentong presyo.

## Pagsasama

Ang `Integration` tab ay para sa mga hotel, integrator at travel agent na gustong magkaroon ng higit na kontrol sa kung paano ipinamamahagi ang mga booking sa Wink.

### Mga abiso

Nagbibigay ito sa hotel at integrator ng opsyon na hayaan ang Wink na ipaalam sa lahat ng mga kasangkot na partido kapag may naganap na booking.
Kabilang dito ang pag-abiso sa tagapamahala ng channel.

Bilang default, pinagana lahat ang mga opsyong ito. Gumamit ng mga sitwasyon kung saan maaaring naisin ng isang integrator na huwag paganahin, isa o higit pa sa, ang mga feature na ito ay:

* Ang travel agent ay isinama sa API-level at gustong magpadala ng sarili nilang mga email sa pagkumpirma.
* Sinusubukan ng hotel ang pagsasama nito at ayaw niyang abalahin ang kanilang mga sales staff.
* Walang pakialam ang hotel sa mga e-mail at gusto lang maabisuhan sa pamamagitan ng kanilang PMS.

### Logo ng e-mail

Para sa mga integrator na gustong gamitin ang aming serbisyo sa pag-abiso sa e-mail, maaari nilang i-customize ang email para mukhang sa kanila ito nanggaling.

:::tandaan
Ang mga inirerekomendang dimensyon ay 300px ang lapad x 75px ang taas
:::

## Partikular sa hotel

Sa ilalim ng `Hotel-specific` Ang tab ay mga opsyon na nagbibigay-daan sa hotel na kontrolin kung ano ang ipinapakita sa kanilang landing page sa aming booking engine at kung ano ang ipapakita kapag walang imbentaryo para sa hiniling na itinerary.

* **Mga ranggo**: Maaaring piliin ng isang property, na hindi pa nakakakuha ng anumang makabuluhang ranggo, na huwag paganahin ang feature na iyon.

:::tandaan
Hindi makokontrol ng property kung paano pipiliin ng isang affiliate na ipakita ang landing page ng hotel... Dalawang magkaibang configuration ang mga iyon.
:::

* **Availability**: Hinahayaan ang property na pumili kung ipagpapatuloy ang pagpapakita ng imbentaryo kahit na walang availability.

:::tandaan
Sa karamihan ng mga kaso, nais ng property na bigyan ang mga user ng paraan upang i-click ang `Book` button upang magpatuloy sa kanilang landing page kung saan maaari nilang baguhin ang kanilang itinerary at subukang maghanap ng available na petsa.
Paganahin ang switch na ito kapag gusto mong harangan ang user sa pagpapatuloy.
:::

## Paano gamitin

Sa karamihan ng mga sitwasyon, pipiliin mo kung alin `Customization` upang ilapat sa iyong mga link, card, mapa at grids.

Narito ang mga oras na maaaring gusto mong manual na ilapat ang iyong configuration:

* **Manu-manong override** Gusto mong direktang magbahagi ng link sa aming booking engine, at hindi gamitin ang aming [Mga Naibabahaging Link](/studio/shareable-links) feature, gamit ang iyong custom na configuration na inilapat.
* **WordPress plugin** Idagdag ang configuration sa iyong Wink [WordPress Plugin](/developers/wordpress) `Settings`.
* **Custom na pagsasama** Manu-manong ilapat ito sa iyong custom na integrator sa paggamit ng aming [wink-app-loader](/developers/web-components/#application-loader) Web Component.

### Manu-manong override

Narito ang isang simpleng halimbawa kung paano ilapat ang iyong configuration ID, kasama ang lahat ng iyong mga setting, sa aming mga pangunahing URL ng engine sa pag-book.

<h4 align="center">[https://ota.wink.travel?configurationId=MY-UNIQUE-ID](https://ota.wink.travel?configurationId=MY-UNIQUE-ID)</h4>


:::tandaan
Kapag nag-click ang isang user sa link na ito, itatakda mo rin ang iyong affiliate attribution cookie.
:::

### WordPress plugin

Mag-navigate sa [Wink WordPress plugin](/developers/wordpress) para matuto pa.

### Custom na pagsasama

Kontrolin ang lahat ng iyong naka-embed na Wink Web Components sa isang page na may [wink-app-loader](/developers/web-components/#application-loader) Web Component.

Nasa ibaba ang isang sample na gumagamit `MY-UNIQUE-ID` bilang configuration ID.

```html {11}
<html>
  <head>
    <link rel="stylesheet" href="https://elements.wink.travel/styles.css"></link>
  </head>
  <body>

    <script src="https://elements.wink.travel/elements.js" type="module" defer></script>

    <wink-app-loader
      client-id="YOUR CLIENT ID GOES HERE"
      configuration-id="MY-UNIQUE-ID"
    ></wink-app-loader>
  </body>
</html>
```
