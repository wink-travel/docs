Ang Pagpapasadya ay isang makapangyarihang tampok na maaari mong gamitin upang kontrolin ang bawat aspeto ng integrasyon sa Wink nang hindi na kailangang magsulat ng anumang code. Bawat piraso ng imbentaryo na iyong ibinebenta ay maaaring ipasadya. Magdagdag o mag-alis ng mga tampok, baguhin ang scheme ng kulay at marami pang iba.

Habang nasa pahina ng `Customization`, i-click ang button na `Actions` sa ilalim ng pagpapasadya na nais mong trabahuhin at i-click ang `Update`.
*Mayroon ka nang default na entry na magagamit kapag gumawa ka ng iyong account. Hindi maaaring alisin ang entry na ito.*

Dadala ka nito sa form ng pagpapasadya kung saan makikita mo ang 5 - 7 tab (*depende sa uri ng iyong account*).

## Pangkalahatan

Narito ang mga opsyon na magagamit mo sa ilalim ng tab na `General`.

- **Pangalan** Bigyan ng pangalan ang iyong pagpapasadya upang madali itong maalala. *hal. Para sa aking travel blog*
- **Ipakitang pera** Piliin ang paunang pera na makikita ng mga gumagamit na ipinapakita sa tabi ng mga bookable na travel inventory. *hal. USD*
- **Ipakitang wika** Piliin ang paunang wika na makakasalubong ng mga gumagamit. *hal. English*
- **Idagdag sa pamagat ng pahina** Maaari mong kontrolin kung ano ang idinadagdag sa pamagat ng pahina (*ang pamagat na nakikita mo sa tab ng browser*). Pangunahing ginagamit ito para sa aming booking engine. *hal. Kung pupunta ka sa https://ota.wink.travel, sa default, makikita mo ang `Home | Traveliko - Hotels with Benefits`*. Maaari mong baguhin ang lahat pagkatapos ng '|' pipe.

## Tema

Pinapayagan ka ng tab na `Theme` na ipasadya ang hitsura at pakiramdam ng aming [booking engine](/booking-engine/in-a-nutshell) website, [WinkLinks](/link-manager/wink-links) website at lahat ng aming [Web Components](/developers/web-components).

### Konfigurasyon

Pinapayagan ka ng seksyong ito na i-configure ang disenyo at layout ng card. Sa kasalukuyan, iisa lamang ang disenyo; ang Wink.

Kinokontrol ng layout ng card kung nais mong makita ng iyong mga gumagamit ang mga transactional card na nakaayos nang pahalang o patayo. Pareho ang impormasyong ipinapakita.

:::note
Walang pagkakaiba sa layout ng card kapag tinitingnan ang aming mga transactional card mula sa isang mobile device. Gayunpaman, makikita mo ang pagkakaiba sa isang iPad mini at mas malalaking resolusyon.
:::

:::tip[Pro tip]
Gamitin ang pahalang na layout kung mayroon ka lamang < 4 na travel inventory (hal. mga uri ng kuwarto) sa isang listahan... mas maganda ang itsura nito.
:::

### Mga Kulay

Kinokontrol ng kulay kung paano nakikita ng iyong mga gumagamit ang alinman sa aming mga UI component at mga website. Gamitin ito upang itugma sa iyong umiiral na scheme ng kulay [kung mayroon man].

Mayroon na kaming malaking pagpipilian ng mga pre-defined na kulay na maaari mong simulan gamitin. Babaguhin ng mga pre-defined na tema ang mga kulay sa mga form field sa ibaba. Maaari ka ring gumamit ng ganap na sarili mong mga kulay.

## Analytics

Sinusuportahan namin ang GA4 analytics nang default. Kung nais mong subaybayan ang buong enhanced ecommerce conversion path, mula simula hanggang katapusan, ihanda ang iyong measurement ID at ilagay ito sa field sa ilalim ng tab na `Analytics`.

<Aside type="caution" title="Paalala">
  Huwag kalimutang idagdag ang `i.trvl.as`, `book.wink.travel` at `ota.wink.travel` sa iyong GA4 property streams.
</Aside>

## Mga Mapa

Upang ma-embed ang aming mga mapa sa iyong site gamit ang aming [WordPress plugin](/developers/wordpress) o [Web Components](/developers/web-components), kailangan mong ibigay sa amin ang iyong sariling Google Maps API key.

:::note
  [Gumawa ng Google Maps API key dito](https://console.developers.google.com/projectselector2/apis/credentials?supportedpurview=project)
:::

Kapag nagawa mo na ang iyong key, at na-configure ito para sa iyong site, ilagay ito sa ilalim ng tab na `Analytics`.

## Itinerary

Ang tab na `Itinerary` ay marahil ang pinaka-interesante pagdating sa pagtiyak na lumalabas ang mga presyo para sa imbentaryong iyong pinili.
Dahil medyo kumplikado ang pagpepresyo ng travel inventory, kailangan ng Wink na malaman ang itinerary ng gumagamit bago maipakita ang tamang presyo.

Pinapayagan ka ng seksyong ito na itakda ang default na itinerary upang ipakita ang mga presyo para sa lahat ng aming digital real estate, tulad ng:

- Aming mga social-friendly na larawan na may price overlay.
- WinkLinks grids, cards at links.
- Web Components
- Booking engine

### Mga Petsa

Maaari mong piliin kung nais mong magtakda ng static na saklaw ng petsa o rolling na saklaw.

- Pumili ng static na saklaw ng petsa kung gagamitin mo ang pagpapasadya para sa isang partikular na kaganapan na nagaganap sa isang takdang petsa.
- Pumili ng rolling na saklaw ng petsa kung nais mong:
    1. Default sa ngayon + 1 gabi, na karaniwang ginagawa ng karamihan sa mga OTA O.
    2. Default sa isang petsa sa hinaharap na dapat ay palaging may availability.

### Konfigurasyon ng Kuwarto

Sa default, ang itinerary ay nakatakda sa 2 matatanda, dahil ito ang pinakakaraniwang setting para sa karamihan ng mga biyahero.
Madaling mababago ito at maaari kang magdagdag ng maraming kuwarto na may maraming occupant; parehong matatanda at mga bata.

### Mga Promosyon

Ang mga promosyon ay isang kapaki-pakinabang na tampok na nakakatipid ng oras para sa iyong mga gumagamit. Kung ang isang property ay nagbigay sa iyo ng promo code na maaari mong ibahagi sa iyong mga kaibigan, maaari mong piliing ilagay ito dito.
Sinumang pumapasok sa pamamagitan ng iyong link, o katulad nito, ay awtomatikong makakakuha ng promo code na idinagdag sa kanilang itinerary at makikita nila ang awtomatikong paglalapat ng diskwentong presyo.

## Integrasyon

Ang tab na `Integration` ay para sa mga hotel, integrator at travel agent na nais magkaroon ng mas kontrol sa kung paano naipapamahagi ang mga booking sa Wink.

### Mga Abiso

Binibigyan nito ang hotel at integrator ng opsyon na ipaalam sa Wink na ipaalam sa lahat ng kasangkot na partido kapag may nangyaring booking.
Kasama dito ang pag-notify sa channel manager.

Sa default, lahat ng mga opsyon na ito ay naka-enable. Ang mga kaso kung saan maaaring nais ng integrator na i-disable ang isa o higit pa sa mga tampok na ito ay:

- Ang travel agent ay naka-integrate sa API-level at nais magpadala ng sarili nilang confirmation emails.
- Ang hotel ay sumusubok ng kanilang integrasyon at ayaw abalahin ang kanilang sales staff.
- Hindi interesado ang hotel sa mga e-mail at nais lamang makatanggap ng abiso sa pamamagitan ng kanilang PMS.

### Logo ng E-mail

Para sa mga integrator na nais gamitin ang aming serbisyo ng e-mail notification, maaari nilang ipasadya ang email upang magmukhang galing ito sa kanila.

:::note
  Inirerekomendang sukat ay 300px lapad x 75px taas
:::

## Para sa Hotel

Sa ilalim ng tab na `Hotel-specific` ay mga opsyon na nagpapahintulot sa hotel na kontrolin kung ano ang ipinapakita sa kanilang landing page sa aming booking engine at kung ano ang ipapakita kapag walang imbentaryo para sa hinihiling na itinerary.

- **Rankings**: Ang isang property na hindi pa nakakamit ng makabuluhang rankings ay maaaring piliing i-disable ang tampok na iyon.

:::note
  Hindi makokontrol ng property kung paano ipapakita ng isang affiliate ang hotel landing page... Magkaibang mga konfigurasyon ang mga iyon.
:::

- **Availability**: Pinapayagan ang property na piliin kung ipagpapatuloy ang pagpapakita ng imbentaryo kahit na walang availability.

:::note
  Sa karamihan ng mga kaso, nais ng property na bigyan ang mga gumagamit ng paraan upang i-click ang button na `Book` upang magpatuloy sa kanilang landing page kung saan maaari nilang baguhin ang kanilang itinerary at subukang maghanap ng available na petsa.
  I-enable ang switch na ito kapag nais mong hadlangan ang gumagamit na magpatuloy.
:::

## Paano Gamitin

Sa karamihan ng mga sitwasyon, pipiliin mo kung aling `Customization` ang ilalapat sa iyong mga link, card, mapa at grid.

Narito ang mga pagkakataon kung kailan maaaring nais mong manu-manong ilapat ang iyong konfigurasyon:

- **Manu-manong override** Nais mong magbahagi ng link sa aming booking engine nang direkta, at hindi gamitin ang aming [Shareable Links](/studio/shareable-links) na tampok, na may iyong custom na konfigurasyon na inilapat.
- **WordPress plugin** Idagdag ang konfigurasyon sa iyong Wink [WordPress Plugin](/developers/wordpress) `Settings`.
- **Custom integration** Manu-manong ilapat ito sa iyong custom integrator gamit ang aming [wink-app-loader](/developers/web-components/#application-loader) Web Component.

### Manu-manong override

Narito ang isang simpleng halimbawa kung paano ilapat ang iyong configuration ID, kasama ang lahat ng iyong mga setting, sa aming mga pangunahing URL ng booking engine.

<h4 align="center"> [https://ota.wink.travel?configurationId=MY-UNIQUE-ID](https://ota.wink.travel?configurationId=MY-UNIQUE-ID) </h4>

:::note
  Kapag nag-click ang isang gumagamit sa link na ito, itatakda mo rin ang iyong affiliate attribution cookie.
:::

### WordPress plugin

Pumunta sa [Wink WordPress plugin](/developers/wordpress) upang matuto pa.

### Custom integration

Kontrolin ang lahat ng iyong naka-embed na Wink Web Components sa isang pahina gamit ang [wink-app-loader](/developers/web-components/#application-loader) Web Component.

Narito ang isang halimbawa na gumagamit ng `MY-UNIQUE-ID` bilang configuration ID.

```html {11}
<html>
  <head>
    <link rel="stylesheet" href="https://elements.wink.travel/styles.css"></link>
  </head>
  <body>

    <script src="https://elements.wink.travel/elements.js" type="module" defer></script>

    <wink-app-loader
      client-id="YOUR CLIENT ID GOES HERE"
      configuration-id="MY-UNIQUE-ID"
    ></wink-app-loader>
  </body>
</html>
```
