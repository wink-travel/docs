[Wink IAM](https://iam.wink.travel) ang identity service para sa iyong Wink **user account** — na hiwalay sa mga **organization accounts** (mga property, affiliate accounts, atbp.) na pinamamahalaan mo sa [Portal](/tl/portal/overview).

## Ano ang nandito

- **`/my-account/general`** — Ang iyong user profile: unang pangalan, apelyido, username, email, larawan ng profile, password.
- **`/my-account/security`** — Passkeys (WebAuthn) para sa password-less na pag-sign-in.
- **`/my-account/apps`** — Mga OAuth2 application na pinahintulutan mong ma-access ang iyong Wink account. Maaari mong bawiin ang access anumang oras.

## Mga Setting ng User vs. Portal: ano ang pagkakaiba?

| Konsepto | Nandito sa | Mga Halimbawa |
|---|---|---|
| **User identity** (ikaw, ang tao) | `iam.wink.travel` | Email, password, passkeys, larawan ng profile |
| **Organization accounts** (mga property, affiliate org, atbp. na pinapatakbo mo) | `app.wink.travel` | Branding, mga manager, payouts, applications, webhooks |

Isang user ang maaaring magpatakbo ng maraming organization accounts; isang organization account naman ay maaaring patakbuhin ng maraming user (sa pamamagitan ng [managers](/tl/portal/settings/managers)).

## Daloy ng pag-sign-in

Laging nagsisimula ang pag-sign-in sa IAM. Kapag na-authenticate na, ikaw ay ireruta papunta sa Portal, kung saan pipiliin mo kung aling organization account ang gusto mong gamitin.
