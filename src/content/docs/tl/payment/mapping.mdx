Kung isa kang integrator, tulad ng Wink, at gusto mong imapa ang iyong mga external na nagbabayad para magamit sa TripPay, magagawa mo ito sa isa sa dalawang paraan:

* Manu-manong pagmamapa
* Gamit ang [API ng Pagbabayad](/developers/apis/#payment-api) (*Tingnan mo `Account Mappings`*)

:::tip
Maliban na lang kung kakaunti lang ang nakikitungo mo sa parehong mga katangian, inirerekomenda namin na gamitin mo ang API para i-automate ang hakbang na ito.
:::

Sa ibaba, dadalhin ka namin sa manu-manong paraan ng pagmamapa sa isang panlabas na nagbabayad. *Ginagamit nito ang API sa ilalim ng hood.*

<Steps>
  1. Mag-log in [TripPay](https://pay.wink.travel).
  2. Piliin ang iyong integrator account.
  3. I-click `Settings > Mappings` mula sa pangunahing nav bar.
  4. I-click ang `Create mapping` pindutan. Makikita mo ang mapping form.
  5. **Lokal na account** Simulan ang pag-type ng pangalan ng kasalukuyang TripPay na gusto mong imapa.
  6. **Panlabas na identifier** I-paste ang iyong external na ID para sa entity na ito dahil mayroon ito sa remote system.
  7. I-click `Save` upang magpatuloy. Ire-redirect ka pabalik sa iyong listahan ng mga pagmamapa.
</Steps>


Sa iyong mga pagmamapa, alam na ngayon ng TripPay kung aling mga account ang ikredito bilang mga benepisyaryo ng isang booking kapag tinukoy mo sila gamit ang iyong mga panlabas na pagkakakilanlan.

## API

Mga developer na gustong pamahalaan `Payments`, pumunta sa [Mga Developer > API > Mga Pagbabayad](/developers/apis/#payment-api).
