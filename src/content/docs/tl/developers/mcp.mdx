## Overview
Ang hosted Wink MCP server ay makikita sa:

```
https://docs.mcp.wink.travel/mcp
```

Ituro ang mga compatible na AI client (VS Code, Cursor, iba pa na nagpapatupad ng MCP HTTP transport) sa endpoint na ito upang payagan silang:
- Mag-browse ng mga dokumento ng produkto at platform (`wink://docs/...`)
- Mag-browse ng live OpenAPI specs at mga dokumento ng operasyon (`wink://api/...`)
- Magsagawa ng magaan na semantic / keyword search sa mga dokumento at live APIs

## Transport
Gamitin ang pangunahing streamable HTTP endpoint:

```
https://docs.mcp.wink.travel/mcp
```

Kung ang iyong client ay sumusuporta lamang sa SSE, gamitin:

```
https://docs.mcp.wink.travel/sse
```

### Claude Code
Opisyal na dokumentasyon: [Connect Claude Code to tools via MCP](https://code.claude.com/docs/en/mcp)

Inirerekomenda ng Anthropic ang remote HTTP transport para sa cloud-hosted MCP servers:
```bash
claude mcp add --transport http wink-docs https://docs.mcp.wink.travel/mcp
claude mcp list
```

Sa loob ng Claude Code, patakbuhin ang `/mcp` upang suriin ang status ng server. Kung nais mong ibahagi ang server sa buong repo, idagdag ang `--scope project` upang isulat ng Claude Code ang config sa `.mcp.json`.

### Codex
Opisyal na dokumentasyon: [Docs MCP quickstart for Codex](https://developers.openai.com/resources/docs-mcp)

Idinedetalye ng OpenAI ang MCP setup sa pamamagitan ng Codex CLI, at binabanggit na ang configuration ay ibinabahagi sa IDE extension:
```bash
codex mcp add wink-docs --url https://docs.mcp.wink.travel/mcp
codex mcp list
```

### Cursor
Cursor MCP docs: https://cursor.com/docs/context/mcp
Setup: Magdagdag ng remote server sa pamamagitan ng MCP settings.
```json
{
  "mcpServers": {
    "wink-docs": {
      "url": "https://docs.mcp.wink.travel/mcp",
    }
  }
}
```

### VS Code
Opisyal na dokumentasyon: https://code.visualstudio.com/docs/copilot/customization/mcp-servers
Minimal na remote config (user o workspace `mcp.json`):
```json
{
  "servers": {
    "wink-docs": {
      "type": "http",
      "url": "https://docs.mcp.wink.travel/mcp"
    }
  }
}
```

Mga Tip:
- Gamitin ang Command Palette: “MCP: List Servers” upang kumpirmahin ang koneksyon at mag-browse ng mga tool.
- Pinapayagan ng tools picker na i-enable/i-disable ang `api_search` at `docs_search`.
- Magdagdag ng konteksto: Chat view → Add Context → MCP Resources upang ikabit ang mga pahina ng dokumento.

## Core Resource URIs

| URI | Paglalarawan |
|-----|--------------|
| `wink://docs` | Index ng dokumentasyon (nakagrupo ayon sa kategorya) |
| `wink://docs/{path}` | Tiyak na artikulo ng dokumentasyon (Markdown / MDX) |
| `wink://api` | Live OpenAPI catalog na natuklasan mula sa swagger config ng Wink |
| `wink://api/{apiName}` | Compact na buod ng API na may auth, tags, headers, at mga sample na operasyon |
| `wink://api/{apiName}/operations` | Buong listahan ng operasyon na nakagrupo ayon sa tag |
| `wink://api/{apiName}/operation/{key}` | Detalye ng indibidwal na operasyon (params, request body, responses) |
| `wink://api/{apiName}/schema/{schemaName}` | Naresolbang component schema na may nested fields, types, at mga halimbawa |
| `wink://api/{apiName}/operation/{key}/request` | View na nakatuon sa request na may kinakailangang inputs at minimal na halimbawa ng payload |
| `wink://api/{apiName}/operation/{key}/response/{status}` | View na nakatuon sa response para sa isang partikular na status code |
| `wink://api/{apiName}/operation/{key}/recipe` | Task-oriented na daloy para sa tamang pagtawag ng operasyon na iyon |

## Search Tools (Quick Reference)
| Tool | Layunin | Karaniwang Halimbawa ng Query |
|------|---------|-------------------------------|
| `api_search` | Hanapin ang live API o operasyon na dapat i-integrate | `create affiliate`, `payment receipt`, `lookup inventory`, `invite manager` |
| `docs_search` | Hanapin ang mga konseptwal na dokumento / gabay | `authentication`, `webhooks`, `wordPress` |

Iwanang walang laman ang `query` upang makatanggap ng mga suhestiyon. Kasama sa bawat resulta ang MCP URI na maaari mong buksan.
Para sa mga operation hits, nagbabalik din ang `api_search` ng mga naka-link na request, response, schema, at recipe URIs upang makalundag nang direkta ang mga agent sa pinaka-kapaki-pakinabang na subresource.

## Typical Usage Pattern
1. Patakbuhin ang `api_search` gamit ang iyong layunin (hal. *"create affiliate"* o *"payment receipt"*).
2. Buksan ang ibinalik na `wink://api/...` URI, o sundan ang mga naka-link na `request_uri`, `response_uris`, `request_schema_uris`, `response_schema_uris`, at `recipe_uri` na mga field.
3. Gamitin ang `docs_search` para sa mga sumusuportang konsepto (authentication, webhooks, workflows).

## AI Skills

Ang mga pre-built na agent skills ay gumagamit ng MCP server na ito bilang kanilang pinagmumulan ng katotohanan. Nagbibigay ang mga skills ng istrukturadong workflow sa iyong coding agent para sa pagbuo laban sa Wink APIs — sa halip na hulaan ang mga endpoints, binabasa ng agent ang live contracts nang direkta mula sa MCP server.

Tingnan ang [AI Skills](/tl/developers/ai-skills) para sa mga available na skills at mga tagubilin sa pag-install.

## Notes
- Ang bawat ibinalik na URI ay maaaring buksan ng agent para sa buong nilalaman.
- Ang `wink://api` tree ay live at sumusunod sa kasalukuyang upstream OpenAPI catalog sa halip na isang generated repo index.
- Awtomatikong lumalabas ang mga suhestiyon kapag walang laman ang mga query.
