Wink berintegrasi dengan pengurus saluran terkemuka, sistem pengurusan hartanah (PMS), dan sistem tempahan pusat (CRS) untuk memastikan kadar dan ketersediaan anda diselaraskan merentasi semua saluran jualan anda.

## Cara ia berfungsi

Integrasi pengurus saluran membolehkan aliran data dua hala antara sistem pengurusan hartanah anda dan Wink:

1. **Penolakan kadar** — Pengurus saluran anda menghantar kemas kini kadar ke Wink. Apabila anda mengemas kini harga dalam PMS anda, perubahan tersebut secara automatik dipaparkan di Wink.
2. **Penyelarasan ketersediaan** — Pengurus saluran anda menghantar kemas kini ketersediaan ke Wink. Apabila bilik ditempah atau dilepaskan dalam PMS anda, inventori Wink dikemas kini secara masa nyata.
3. **Pemberitahuan tempahan** — Apabila tempahan dibuat di Wink, pengurus saluran anda diberitahu supaya tempahan tersebut muncul dalam PMS anda.

## Pengurus saluran yang disokong

Wink kini berintegrasi dengan sistem berikut:

| Pengurus Saluran | Penerangan |
|---|---|
| **Channex** | Platform pengedaran global dan pengurusan saluran. |
| **TravelClick** | Penyelesaian hotel dan pengurusan saluran (Amadeus). |
| **Omnibees** | Pengedaran dan pengurusan hasil untuk hotel. |
| **D-Edge** | Teknologi hospitaliti dan penyelesaian tempahan pusat. |
| **Allotz** | Peruntukan inventori dan pengurusan saluran. |
| **Ezee / Yanolja** | PMS awan dan pengurusan saluran yang memfokuskan pasaran Asia. |
| **RateGain** | Teknologi kecerdasan kadar dan pengedaran. |
| **RateTiger** | Pengedaran kadar dan ketersediaan masa nyata. |
| **CloudBeds** | PMS berasaskan awan, pengurus saluran, dan enjin tempahan. |
| **SiteMinder** | Pengurus saluran hotel terkemuka dan platform pengedaran. |
| **Exely** | Penyelarasan kadar dan ketersediaan untuk hotel. |

:::note
Jika pengurus saluran anda tidak disenaraikan, hubungi pasukan Wink. Integrasi baru boleh ditambah berdasarkan piawaian protokol OTA (Open Travel Alliance) yang disokong oleh Wink.
:::

## Untuk pembangun

Jika anda membina integrasi pengurus saluran atau PMS dengan Wink, gunakan titik akhir API Pengurus Saluran:

- **Kemas kini kadar** — Tolak perubahan kadar ke Wink.
- **Kemas kini ketersediaan** — Tolak perubahan ketersediaan ke Wink.
- **Pengambilan tempahan** — Tarik tempahan dari Wink ke dalam sistem anda.

API Pengurus Saluran mengikuti piawaian protokol OTA (SOAP/XML) untuk keserasian dengan sistem hospitaliti sedia ada.

## Pemberitahuan webhook

Anda boleh melanggan acara webhook pengurus saluran untuk menerima pemberitahuan masa nyata:

- `channel-manager.update.rate` — Kemas kini kadar diterima.
- `channel-manager.update.availability` — Kemas kini ketersediaan diterima.
- `channel-manager.update` — Kemas kini umum pengurus saluran.

Lihat [Katalog Acara Webhook](/ms/developers/webhook-events#channel-manager-events) untuk butiran.
