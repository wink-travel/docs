## Gambaran Keseluruhan
Pelayan MCP Wink yang dihoskan boleh diakses di:

```
https://docs.mcp.wink.travel/mcp
```

Arahkan klien AI yang serasi (VS Code, Cursor, lain-lain yang melaksanakan pengangkutan MCP HTTP) ke titik akhir ini untuk membolehkan mereka:
- Melayari dokumen produk & platform (`wink://docs/...`)
- Melayari spesifikasi OpenAPI langsung dan dokumen operasi (`wink://api/...`)
- Melakukan carian semantik / kata kunci ringan ke atas dokumen dan API langsung

## Pengangkutan
Gunakan titik akhir HTTP aliran utama:

```
https://docs.mcp.wink.travel/mcp
```

Jika klien anda hanya menyokong SSE, gunakan:

```
https://docs.mcp.wink.travel/sse
```

### Claude Code
Dokumen rasmi: [Sambungkan Claude Code ke alat melalui MCP](https://code.claude.com/docs/en/mcp)

Anthropic mengesyorkan pengangkutan HTTP jauh untuk pelayan MCP yang dihoskan di awan:
```bash
claude mcp add --transport http wink-docs https://docs.mcp.wink.travel/mcp
claude mcp list
```

Dalam Claude Code, jalankan `/mcp` untuk memeriksa status pelayan. Jika anda ingin berkongsi pelayan dengan seluruh repo, tambah `--scope project` supaya Claude Code menulis konfigurasi ke `.mcp.json`.

### Codex
Dokumen rasmi: [Docs MCP quickstart untuk Codex](https://developers.openai.com/resources/docs-mcp)

OpenAI mendokumentasikan tetapan MCP melalui CLI Codex, dan menyatakan bahawa konfigurasi dikongsi dengan sambungan IDE:
```bash
codex mcp add wink-docs --url https://docs.mcp.wink.travel/mcp
codex mcp list
```

### Cursor
Dokumen MCP Cursor: https://cursor.com/docs/context/mcp
Tetapan: Tambah pelayan jauh melalui tetapan MCP.
```json
{
  "mcpServers": {
    "wink-docs": {
      "url": "https://docs.mcp.wink.travel/mcp",
    }
  }
}
```

### VS Code
Dokumen rasmi: https://code.visualstudio.com/docs/copilot/customization/mcp-servers
Konfigurasi jauh minimum (pengguna atau ruang kerja `mcp.json`):
```json
{
  "servers": {
    "wink-docs": {
      "type": "http",
      "url": "https://docs.mcp.wink.travel/mcp"
    }
  }
}
```

Petua:
- Gunakan Command Palette: “MCP: List Servers” untuk mengesahkan sambungan & melayari alat.
- Pemilih alat membolehkan anda mengaktifkan/mematikan `api_search` dan `docs_search`.
- Tambah konteks: Paparan Sembang → Tambah Konteks → Sumber MCP untuk melampirkan halaman dokumen.

## URI Sumber Teras

| URI | Penerangan |
|-----|-------------|
| `wink://docs` | Indeks dokumentasi (dikelompokkan mengikut kategori) |
| `wink://docs/{path}` | Artikel dokumentasi khusus (Markdown / MDX) |
| `wink://api` | Katalog OpenAPI langsung yang ditemui dari konfigurasi swagger Wink |
| `wink://api/{apiName}` | Ringkasan API padat dengan pengesahan, tag, pengepala, dan operasi contoh |
| `wink://api/{apiName}/operations` | Senarai operasi penuh dikelompokkan mengikut tag |
| `wink://api/{apiName}/operation/{key}` | Butiran operasi individu (param, badan permintaan, respons) |
| `wink://api/{apiName}/schema/{schemaName}` | Skema komponen yang diselesaikan dengan medan bersarang, jenis, dan contoh |
| `wink://api/{apiName}/operation/{key}/request` | Paparan fokus permintaan dengan input wajib dan payload contoh minimum |
| `wink://api/{apiName}/operation/{key}/response/{status}` | Paparan fokus respons untuk kod status tertentu |
| `wink://api/{apiName}/operation/{key}/recipe` | Aliran berorientasikan tugas untuk memanggil operasi itu dengan betul |

## Alat Carian (Rujukan Pantas)
| Alat | Tujuan | Contoh Pertanyaan Biasa |
|------|---------|-----------------------|
| `api_search` | Cari API langsung atau operasi untuk diintegrasikan | `create affiliate`, `payment receipt`, `lookup inventory`, `invite manager` |
| `docs_search` | Cari dokumen konseptual / panduan | `authentication`, `webhooks`, `wordPress` |

Biarkan `query` kosong untuk menerima cadangan. Setiap hasil termasuk URI MCP yang boleh anda buka.
Untuk hasil operasi, `api_search` juga mengembalikan URI permintaan, respons, skema, dan resipi yang dipautkan supaya ejen boleh terus melompat ke sub-sumber yang paling berguna.

## Corak Penggunaan Biasa
1. Jalankan `api_search` dengan niat anda (contoh *"create affiliate"* atau *"payment receipt"*).
2. Buka URI `wink://api/...` yang dikembalikan, atau ikut medan `request_uri`, `response_uris`, `request_schema_uris`, `response_schema_uris`, dan `recipe_uri` yang dipautkan.
3. Gunakan `docs_search` untuk konsep sokongan (pengesahan, webhook, aliran kerja).

## Kemahiran AI

Kemahiran ejen yang telah dibina menggunakan pelayan MCP ini sebagai sumber kebenaran mereka. Kemahiran memberikan ejen pengekodan anda aliran kerja berstruktur untuk membina terhadap API Wink — bukannya meneka titik akhir, ejen membaca kontrak langsung dari pelayan MCP.

Lihat [AI Skills](/ms/developers/ai-skills) untuk kemahiran yang tersedia dan arahan pemasangan.

## Nota
- Setiap URI yang dikembalikan boleh dibuka oleh ejen untuk kandungan penuh.
- Pokok `wink://api` adalah langsung dan mengikuti katalog OpenAPI hulu semasa dan bukannya indeks repo yang dijana.
- Cadangan muncul secara automatik apabila pertanyaan kosong.
