Den här artikeln går igenom allt du behöver veta om hur priser fungerar på Wink.

## Funktioner

:::note
Wink hanterar priser på masterprisnivå och inte på gäst-rumsnivå. Detta ger dig större flexibilitet i hur du väljer att fördela rum.
:::

Ett pris på Wink stödjer följande datapunkter:

- **Antal** Antal tillgängliga enheter.
- **Prisbelopp** Grundpris.
- **Tillgänglig** Om priset är tillgängligt för det datumet eller inte.
- **Stängt vid ankomst (COA)** Om gäster kan anlända på detta datum.
- **Stängt vid avresa (COD)** Om gäster kan lämna på detta datum.
- **Minsta vistelselängd (Min LoS)** Om det finns ett krav på minsta vistelselängd.
- **Maximal vistelselängd (Max LoS)** Om det finns ett krav på maximal vistelselängd.
- **Minsta beläggning** Om det finns ett krav på minsta beläggning.
- **Maximal beläggning** Om det finns ett krav på maximal vistelselängd.

Nedanstående datapunkter är endast tillgängliga via en prisleverantör:

*Om din prisleverantör inte stödjer dessa datapunkter kan du fortfarande kontrollera dem via dina prisplaner på Wink.*

- Prismodifierare för ensamrum
- Prismodifierare för extra vuxen
- Prismodifierare för extra barn

Priser är kärnan i Wink. Boenden kan lägga till priser manuellt eller koppla sin prisleverantör.

## Manuell inmatning

:::caution
Skapa / uppdatera endast priser manuellt om du inte säljer dina rumstyper på någon annan plattform än Wink.
:::

:::caution

## Via prisleverantör

För att börja skicka priser till oss via din prisleverantör, följ dessa steg:

- Kontrollera först att vi stödjer din [prisleverantör](/sv/extranet/rate-providers).
- Ställ in din prisleverantör under `Distribution > Channel manager`.
- Logga in på din prisleverantör.
- Lägg till `Wink`-kanalen i din prisleverantörs dashboard.
- Koppla dina rumstyper / prisplaner till de du skapat i Wink.
- Initiera en fullständig prisuppdatering till Wink [om det inte görs automatiskt av prisleverantören].

Vänta cirka 3-5 minuter för att se dina Wink masterpris-kalendrar fyllas med priser.

## Verifierare

Du kan testa din prisinställning genom att navigera till `Distribution > Verifier` i huvudnavigeringsfältet.

På denna sida kan du simulera alla typer av prisförfrågningar som en resenär kan göra.

:::tip
Om du inte hittar inställningen du söker, klicka på knappen `Avancerad sökning` längst ner på sidan.
:::

### Exempel

Nedan är en exempelresa över en av dina befintliga försäljningskanaler:

- **Kanal** Välj den försäljningskanal du vill sälja genom. *t.ex. Traveliko OTA*.
- **Valuta** Välj visningsvaluta. *t.ex. USD*
- **Vistelsens startdatum** Ange ankomstdatum. *t.ex. 10 sept 2024*

Du kan testa hela ditt arbetsflöde från början till slut genom att skapa en testbokning.
En testbokning markeras som `test` och säkrar inte bokningen med kreditkort eller annan betalningsmetod.

Testbokningsalternativ inkluderar:

- **Meddela boendet via e-post** Detta triggar att boendet får en e-postbekräftelse.
- **Meddela channel manager via API** Detta triggar att bokningen skickas till din valda prisleverantör.
- **Meddela bokaren via e-post** Detta triggar att bokaren får en e-postbekräftelse.

Som standard är bokaren i detta scenario du, den autentiserade användaren. Du kan också välja att lägga till egna bokaruppgifter genom att aktivera reglaget `Lägg till egna bokaruppgifter` och fylla i formuläret.

*Om du valde att vi ska meddela din prisleverantör, kommer prisleverantören att behandla denna bokning som vilken annan bokning som helst och kommer inte att veta att det är ett test. Vi rekommenderar att testa med återbetalningsbara priser för enkel avbokning efter testet är klart.*
