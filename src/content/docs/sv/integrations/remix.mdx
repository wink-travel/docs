Remix är ett fullstack-ramverk som som standard server-renderar. Ladda Wink CDN endast på klienten med `useEffect` i rotkomponenten.

  title="@wink/elements npm package"

/>

## Installera

```bash
npm install @wink/elements
```

## Ladda i app/root.tsx

`useEffect` körs endast i webbläsaren, aldrig under server-rendering:

```tsx
// app/root.tsx
import { useEffect } from 'react';
import {
  Links, Meta, Outlet, Scripts, ScrollRestoration,
} from '@remix-run/react';
import { load } from '@wink/elements';

export default function App() {
  useEffect(() => {
    load({ clientId: import.meta.env.VITE_WINK_CLIENT_ID });
  }, []);

  return (
    <html lang="en">
      <head>
        <Meta />
        <Links />
      </head>
      <body>
        <Outlet />
        <ScrollRestoration />
        <Scripts />
      </body>
    </html>
  );
}
```

## Miljövariabler

Remix (Vite-baserat, v2.8+) stödjer `import.meta.env` för variabler exponerade i webbläsaren. Prefixa med `VITE_` för att exponera till webbläsarpaketet:

```
# .env
VITE_WINK_CLIENT_ID=din-klient-id
```

För äldre Remix-konfigurationer som använder Node.js-adaptern, returnera klient-ID från en root loader och läs in det i komponenten:

```tsx
// app/root.tsx
import { json } from '@remix-run/node';
import { useLoaderData } from '@remix-run/react';

export async function loader() {
  return json({ winkClientId: process.env.WINK_CLIENT_ID });
}

export default function App() {
  const { winkClientId } = useLoaderData<typeof loader>();

  useEffect(() => {
    if (winkClientId) {
      load({ clientId: winkClientId });
    }
  }, [winkClientId]);

  // ...
}
```

## Använd i routes

```tsx
// app/routes/hotels.tsx
export default function HotelsRoute() {
  return (
    <main>
      <wink-content-loader layout="HOTEL" id="YOUR_LAYOUT_ID" />
    </main>
  );
}
```

<Aside type="caution" title="Hydreringsavvikelse">
Remix hydrerar den server-renderade HTML:en på klienten. Om ett `<wink-*>`-element renderas på servern kan webbläsaren kortvarigt visa det oregistrerade custom elementet innan CDN-skriptet registrerar det. Detta är kosmetiskt — elementet uppgraderas korrekt när skriptet har laddats.
</Aside>
