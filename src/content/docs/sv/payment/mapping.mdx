Om du är en integratör, som Wink, och vill kartlägga dina externa betalningsmottagare för användning med TripPay, kan du göra det på två sätt:

- Manuell kartläggning
- Använda [Payment API](/sv/developers/apis/#payment-api) (*Se `Account Mappings`*)

:::tip
Om du inte bara hanterar ett fåtal av samma fastigheter rekommenderar vi att du använder API:et för att automatisera detta steg.
:::

Nedan guidar vi dig genom det manuella sättet att kartlägga en extern betalningsmottagare. *Det använder API:et i bakgrunden.*

<Steps>
1. Logga in på [TripPay](https://pay.wink.travel).
2. Välj ditt integratörskonto.
3. Klicka på `Settings > Mappings` i huvudnavigeringsfältet.
4. Klicka på knappen `Create mapping`. Du kommer att se kartläggningsformuläret.
5. **Local account** Börja skriva namnet på det befintliga TripPay-konto du vill kartlägga.
6. **External identifier** Klistra in din externa ID för denna enhet som den finns i det externa systemet.
7. Klicka på `Save` för att fortsätta. Du omdirigeras tillbaka till din lista över kartläggningar.
</Steps>

Med dina kartläggningar på plats vet TripPay nu vilka konton som ska krediteras som mottagare av en bokning när du hänvisar till dem med dina externa identifierare.

## API

Utvecklare som vill hantera `Payments`, gå till [Developers > API > Payments](/sv/developers/apis/#payment-api).
