Om du är en integratör, som Wink, och vill mappa dina externa betalningsmottagare för användning med TripPay, kan du göra det på ett av två sätt:

* Manuell mappning
* Använda[Betalnings-API](/developers/apis/#payment-api)(*Se`Account Mappings`*)

:::tip
Om du inte bara har att göra med ett fåtal av samma egenskaper rekommenderar vi att du använder API:et för att automatisera det här steget.
:::

Nedan guidar vi dig genom det manuella sättet att mappa en extern betalningsmottagare.*Den använder API:et under huven.*

<Steps>
  1. Logga in [TripPay](https://pay.wink.travel).
  2. Välj ditt integratorkonto.
  3. Klick `Settings > Mappings` från huvudnavigeringsfältet.
  4. Klicka på `Create mapping` knappen. Du kommer att se mappningsformuläret.
  5. **Lokalt konto** Börja skriva namnet på den befintliga TripPay-kontot som du vill mappa.
  6. **Extern identifierare** Klistra in ditt externa ID för den här entiteten så som den finns i fjärrsystemet.
  7. Klick `Save` för att fortsätta. Du omdirigeras tillbaka till din lista över mappningar.
</Steps>


Med dina mappningar på plats vet TripPay nu vilka konton som ska krediteras som mottagare av en bokning när du hänvisar till dem med dina externa identifierare.

## API

Utvecklare som vill hantera`Payments`, gå till[Utvecklare > API > Betalningar](/developers/apis/#payment-api).
