Kaikki päivät eivät ole hinnoiteltu samalla tavalla. Tapahtumat, juhlapyhät, sesongin ulkopuoliset ajat ja viime hetken vapaat paikat vaativat erilaisia hintoja. Tämä opas näyttää, miten käytät erikoishintoja ja hintasuunnitelmien rajoituksia näiden tilanteiden hallintaan.

## Tapahtumien hinnoittelu

Kun konferenssi, festivaali tai paikallinen tapahtuma lisää kysyntää, haluat nostaa hintoja kyseisinä päivinä.

<Steps>
1. Siirry Extranetissä kohtaan `Monetize > Special Rates`.
2. Klikkaa `Create a new special rate`.
3. Valitse ylin hinta ja huonetyyppi, jonka haluat korvata.
4. Aseta päivämääräalue kattamaan tapahtuman ajanjakso.
5. Syötä korotetut (korkeammat) päivähinnat.
6. Klikkaa `Save`.
</Steps>

Erikoishinta korvaa ylin hinnan vain noina päivinä. Tavallinen hinnoittelusi pysyy ennallaan tapahtuma-ajan ulkopuolella.

## Sesonkimuutokset

Pidempien ajanjaksojen (korkea sesonki, väli- tai matala sesonki) osalta voit joko:

- **Käyttää erikoishintoja** — Luo korvauksia jokaiselle sesongille säädetyillä hinnoilla.
- **Päivittää ylin hinta** — Säädä perushintoja suoraan noille ajanjaksoille.

Erikoishinnat sopivat paremmin tilapäisiin muutoksiin, kun taas ylin hinnan päivittäminen on järkevää ennustettavissa vuosittaisissa kausissa.

## Viime hetken hinnat

Myymättömien huoneiden täyttämiseksi lähellä saapumispäivää:

<Steps>
1. Luo uusi **hintasuunnitelma**, jossa on ennakko-varausrajoitus (esim. varaus on tehtävä enintään 3 päivää etukäteen).
2. Yhdistä se **erikoishintaan**, joka tarjoaa alennuksen kohdepäiville.
3. Tämä hintasuunnitelma näkyy vain matkustajille, jotka varaavat viime hetken ikkunan sisällä.
</Steps>

## Mustat päivät

Estääksesi tiettyjä päiviä tietystä hintasuunnitelmasta (esim. ei kampanjahintoja sesongin huippuna):

<Steps>
1. Siirry kohtaan `Monetize > Rate Plans` ja valitse hintasuunnitelma.
2. Mene **Restrictions**-osioon.
3. Lisää **Sell date** -rajoitus, jolla rajoitat, milloin hinta on varattavissa.
4. Tai käytä **Stay date** -rajoituksia, joilla rajoitat, milloin hinta kattaa oleskelun.
</Steps>

Esimerkiksi voit rajoittaa "Early Bird" -hintasuunnitelmasi niin, ettei sitä voi varata joulukuussa (sesongin huippu), kun taas tavallinen hintasuunnitelmasi pysyy avoimena.

## Rajoitusten yhdistäminen

Rajoituksia voi pinota tarkkaa hallintaa varten:

| Rajoitus | Mitä se ohjaa |
| -- | -- |
| **Sell dates** | Milloin hinta on varattavissa |
| **Stay dates** | Mitä oleskelupäiviä hinta kattaa |
| **Available days** | Mitkä viikonpäivät hinta näkyy |
| **Arrival days** | Milloin vieraan on saavuttava |
| **Departure days** | Milloin vieraan on lähdettävä |
| **Required days** | Vieraan on oltava vähintään yhtenä näistä päivistä |
| **Advance booking** | Kuinka kauan etukäteen varaus on tehtävä |
| **Length of stay** | Minimi- ja maksimiyöt |
