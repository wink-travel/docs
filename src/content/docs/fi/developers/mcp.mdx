## Yleiskatsaus
Isännöity Wink MCP -palvelin on saatavilla osoitteessa:

```
https://docs.mcp.wink.travel/mcp
```

Ohjaa yhteensopivat tekoälyasiakkaat (VS Code, Cursor, muut MCP HTTP -kuljetuksen toteuttavat) tähän päätepisteeseen, jotta ne voivat:
- Selailla tuotteen ja alustan dokumentaatiota (`wink://docs/...`)
- Selailla reaaliaikaisia OpenAPI-määritelmiä ja toimintadokumentteja (`wink://api/...`)
- Suorittaa kevyttä semanttista / avainsanahakua dokumentaatiossa ja reaaliaikaisissa API:ssa

## Kuljetus
Käytä ensisijaista suoratoistettavaa HTTP-päätepistettä:

```
https://docs.mcp.wink.travel/mcp
```

Jos asiakkaasi tukee vain SSE:tä, käytä:

```
https://docs.mcp.wink.travel/sse
```

### Claude Code
Viralliset ohjeet: [Yhdistä Claude Code työkaluihin MCP:n kautta](https://code.claude.com/docs/en/mcp)

Anthropic suosittelee etä-HTTP-kuljetusta pilvipalvelimilla isännöidyille MCP-palvelimille:
```bash
claude mcp add --transport http wink-docs https://docs.mcp.wink.travel/mcp
claude mcp list
```

Claude Codessa suorita `/mcp` tarkistaaksesi palvelimen tilan. Jos haluat jakaa palvelimen koko repositorion kanssa, lisää `--scope project`, jolloin Claude Code kirjoittaa konfiguraation tiedostoon `.mcp.json`.

### Codex
Viralliset ohjeet: [Docs MCP pika-aloitus Codexille](https://developers.openai.com/resources/docs-mcp)

OpenAI dokumentoi MCP:n asetuksen Codex CLI:n kautta ja mainitsee, että konfiguraatio jaetaan IDE-laajennuksen kanssa:
```bash
codex mcp add wink-docs --url https://docs.mcp.wink.travel/mcp
codex mcp list
```

### Cursor
Cursor MCP -dokumentaatio: https://cursor.com/docs/context/mcp
Asetus: Lisää etäpalvelin MCP-asetusten kautta.
```json
{
  "mcpServers": {
    "wink-docs": {
      "url": "https://docs.mcp.wink.travel/mcp",
    }
  }
}
```

### VS Code
Viralliset ohjeet: https://code.visualstudio.com/docs/copilot/customization/mcp-servers
Minimaalinen etäkonfiguraatio (käyttäjä- tai työtilan `mcp.json`):
```json
{
  "servers": {
    "wink-docs": {
      "type": "http",
      "url": "https://docs.mcp.wink.travel/mcp"
    }
  }
}
```

Vinkkejä:
- Käytä komentopalettia: “MCP: List Servers” vahvistaaksesi yhteyden ja selataksesi työkaluja.
- Työkalujen valitsin antaa mahdollisuuden ottaa käyttöön/poistaa käytöstä `api_search` ja `docs_search`.
- Lisää konteksti: Keskustelunäkymä → Lisää konteksti → MCP-resurssit liittääksesi dokumenttisivut.

## Keskeiset resurssi-URI:t

| URI | Kuvaus |
|-----|-------------|
| `wink://docs` | Dokumentaation indeksi (ryhmitelty kategorian mukaan) |
| `wink://docs/{path}` | Tietty dokumentaatioartikkeli (Markdown / MDX) |
| `wink://api` | Reaaliaikainen OpenAPI-katalogi, joka löydetty Wink:n swagger-konfiguraatiosta |
| `wink://api/{apiName}` | Tiivis API-yhteenveto autentikoinnilla, tageilla, otsikoilla ja esimerkkitoiminnoilla |
| `wink://api/{apiName}/operations` | Täydellinen toimintojen lista tagin mukaan ryhmiteltynä |
| `wink://api/{apiName}/operation/{key}` | Yksittäisen toiminnon tiedot (parametrit, pyyntörunko, vastaukset) |
| `wink://api/{apiName}/schema/{schemaName}` | Ratkaistu komponenttiskaema, jossa on sisäkkäiset kentät, tyypit ja esimerkit |
| `wink://api/{apiName}/operation/{key}/request` | Pyyntöön keskittyvä näkymä vaadituilla syötteillä ja minimiesimerkkipayloadilla |
| `wink://api/{apiName}/operation/{key}/response/{status}` | Vastausnäkymä tietylle tilakoodille |
| `wink://api/{apiName}/operation/{key}/recipe` | Tehtäväkeskeinen työnkulku kyseisen toiminnon oikeaan kutsumiseen |

## Hakutyökalut (Pikaviite)
| Työkalu | Tarkoitus | Tyypillisiä hakuesimerkkejä |
|------|---------|-----------------------|
| `api_search` | Löytää reaaliaikaisen API:n tai toiminnon integraatiota varten | `create affiliate`, `payment receipt`, `lookup inventory`, `invite manager` |
| `docs_search` | Löytää käsitteellisiä dokumentteja / oppaita | `authentication`, `webhooks`, `wordPress` |

Jätä `query` tyhjäksi saadaksesi ehdotuksia. Jokainen tulos sisältää MCP-URI:n, jonka voit avata.
Toimintohauissa `api_search` palauttaa myös linkitetyt pyyntö-, vastaus-, skeema- ja resepti-URI:t, jotta agentit voivat hypätä suoraan hyödyllisimpään aliresurssiin.

## Tyypillinen käyttötapa
1. Suorita `api_search` aikomuksellasi (esim. *"create affiliate"* tai *"payment receipt"*).
2. Avaa palautettu `wink://api/...` URI tai seuraa sen linkitettyjä `request_uri`, `response_uris`, `request_schema_uris`, `response_schema_uris` ja `recipe_uri` kenttiä.
3. Käytä `docs_search` tukevien käsitteiden (autentikointi, webhookit, työnkulut) löytämiseen.

## Huomautuksia
- Jokainen palautettu URI voidaan avata agentin toimesta täydellisen sisällön saamiseksi.
- `wink://api` -puu on reaaliaikainen ja seuraa nykyistä ylävirran OpenAPI-katalogia eikä generoituja repo-indeksejä.
- Ehdotukset ilmestyvät automaattisesti, kun hakukyselyt ovat tyhjiä.
