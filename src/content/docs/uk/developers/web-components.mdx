[Веб-компоненти](https://developer.mozilla.org/en-US/docs/Web/API/Web_components) — це стандарт, який дозволяє інтегрувати повноцінні функції на JavaScript у ваш вебсайт без необхідності знати програмування.
Завдяки цій крутій технології ви можете легко вбудувати інвентар Wink для подорожей. Ця стаття проведе вас через нашу колекцію веб-компонентів і покаже, як їх використовувати.

## Вбудування Wink

Існує три речі, які потрібно включити на будь-яку сторінку, де ви хочете відобразити один із наших веб-компонентів.

1. Включіть наш файл стилів.
2. Включіть наш Javascript.
3. Включіть наш завантажувач додатка.

### Файл стилів

Включіть наш CSS у секцію head вашого документа.

```html {3}
<html>
  <head>
    <link rel="stylesheet" href="https://elements.wink.travel/styles.css"></link>
  </head>
</html>
```

### Javascript

Включіть наш Javascript у нижній частині вашого документа. (*Рекомендуємо безпосередньо перед закриваючим тегом body*).

```html {3}
<html>
  <body>
    <script src="https://elements.wink.travel/elements.js" type="module" defer></script>
  </body>
</html>
```

Ваш сайт тепер готовий і може відображати наші веб-компоненти.

## Компоненти

Детальніше про нашу бібліотеку компонентів нижче.

### Завантажувач додатка

Завантажувач відповідає за збереження стану та керування взаємодією між нашими компонентами. (*Включайте його після нашого Javascript*).

```html {3-6}
<html>
  <body>
    <wink-app-loader
      client-id="YOUR CLIENT ID GOES HERE"
      configuration-id="YOUR CUSTOMIZATION ID GOES HERE"
    ></wink-app-loader>
  </body>
</html>
```

### Завантажувач контенту

Завантажувач контенту є серцем нашої бібліотеки веб-компонентів. Він відповідає за відображення вашого інвентарю (картки, сітки, карти).

Доступні атрибути:

- layout
- id
- sort

*Атрибут `sort` доступний лише коли layout встановлено в `RANKED` і ви не хочете використовувати існуючу ранжовану сітку. У таких випадках залиште `id` порожнім.*

Доступні типи layout:

- `AD_BANNER`
- `MAP`
- `HOTEL`
- `GUEST_ROOM`
- `MEETING_ROOM`
- `SPA`
- `RESTAURANT`
- `ACTIVITY`
- `ATTRACTION`
- `PLACE`
- `ADD_ON`
- `LIST`
- `SEARCH`
- `RANKED`

Доступні типи сортування:

- `MEMBER`
- `PRICE_LOW_TO_HIGH`
- `PRICE_HIGH_TO_LOW`
- `PRICE`
- `POPULARITY`
- `ECO`
- `EXPERIENCE`
- `PERK`
- `LOYALTY`
- `PACKAGE`

```html {3-6}
<html>
  <body>
    <wink-content-loader
       layout="GUEST_ROOM"
       id="2de7ee9c-61c9-11ef-9722-42004e494300"
    ></wink-content-loader>
  </body>
</html>
```

Приклади можна знайти для [карток](/studio/cards), [сіток](/studio/grids) та [карт](/studio/maps).

### Пошук

Пошук чудово працює разом з однією з ваших ранжованих сіток. Він дозволяє користувачам шукати готелі та напрямки, а сітка оновлюється відповідно до результатів пошуку.

```html {3}
<html>
  <body>
    <wink-lookup></wink-lookup>
  </body>
</html>
```

## Вбудування TripPay

Існує дві речі, які потрібно включити на будь-яку сторінку, де ви хочете використовувати веб-компонент оплати TripPay.

1. Включіть наш файл стилів.
2. Включіть наш Javascript.

### Файл стилів

Включіть наш CSS у секцію head вашого документа.

```html {3}
<html>
  <head>
    <link rel="stylesheet" href="https://elements.trippay.io/styles.css"></link>
  </head>
</html>
```

### Javascript

Включіть наш Javascript у нижній частині вашого документа. (*Рекомендуємо безпосередньо перед закриваючим тегом body*).

```html {3}
<html>
  <body>
    <script src="https://elements.trippay.io/elements.js" type="module" defer></script>
  </body>
</html>
```

Ваш сайт тепер готовий і може відображати наші веб-компоненти.

## Компоненти

### Оплата

Компонент оплати дозволяє повідомити TripPay, що мандрівник хоче щось придбати, а також правила і ціни для цих товарів.

Існує один обов’язковий атрибут для віджета:

- `id` Ідентифікатор контракту бронювання, який ви хочете виконати.

```html {3}
<html>
  <body>
    <trip-pay id="<INSERT_BOOKING_CONTRACT_ID>"></trip-pay>
  </body>
</html>
```

:::note
Щоб дізнатися, як згенерувати `id` контракту бронювання, прочитайте посібник: [Інтеграція з TripPay](/guides/integrating-with-trip-pay).
:::

Віджет готує контракт до виконання і відображає користувачу деталі оплати *(Рисунок 1)* для завершення бронювання.

## Помилки

Якщо при вбудовуванні одного з наших веб-компонентів, на WinkLinks або на вашому власному сайті, ви бачите повідомлення про помилку, можливі кілька причин:

### Доступність

Інвентар може бути більше недоступним. Перейдіть до [Wink Studio](/studio/cards) і перевірте, чи статус "картки" доступний.
Якщо ні, вона буде позначена червоним. У такому випадку ви можете почекати, можливо постачальник знову зробить її доступною, або видалити її зі свого списку.

:::note
`Доступність` не означає щоденну доступність інвентарю для бронювання. Це означає, що інвентар повністю недоступний для вас і/або будь-кого іншого.
:::

### Видалено

Інвентар міг бути видалений. Перейдіть до [Wink Studio](/studio/cards) і перевірте, чи статус "картки" доступний.
Якщо ні, вона буде позначена червоним. У такому випадку ви можете почекати, можливо постачальник знову зробить її доступною, або видалити її зі свого списку.

### Налаштування

Ви могли випадково видалити налаштування, які визначили для свого інвентарю. Переконайтеся, що налаштування все ще доступні, і встановіть нові, якщо якісь відсутні.

### Додаток

Це для веб-розробників. Якщо ви випадково видалили [Додаток](/developer/applications), через що client ID став недоступним, будь ласка, створіть новий додаток і використовуйте новий client ID для вбудовування наших веб-компонентів.
