## Огляд
Хостинговий MCP сервер Wink доступний за адресою:

```
https://docs.mcp.wink.travel/mcp
```

Направляйте сумісні AI-клієнти (VS Code, Cursor, інші, що реалізують MCP HTTP транспорт) на цю кінцеву точку, щоб вони могли:
- Переглядати документацію продукту та платформи (`wink://docs/...`)
- Знаходити SDK та досліджувати API класи / моделі (`wink://sdk/...` та `wink://sdk/.../docs`)
- Виконувати легкий семантичний / ключовий пошук по документації та резюме SDK

## Транспорт
Використовуйте основну HTTP кінцеву точку з потоковою підтримкою:

```
https://docs.mcp.wink.travel/mcp
```

Якщо ваш клієнт підтримує лише SSE, використовуйте:

```
https://docs.mcp.wink.travel/sse
```

### Cursor
Документація Cursor MCP: https://cursor.com/docs/context/mcp
Налаштування: Додайте віддалений сервер через налаштування MCP.
```json
{
  "mcpServers": {
    "wink-docs": {
      "url": "https://docs.mcp.wink.travel/mcp",
    }
  }
}
```

### VS Code
Офіційна документація: https://code.visualstudio.com/docs/copilot/customization/mcp-servers
Мінімальна віддалена конфігурація (користувацький або робочий `mcp.json`):
```json
{
  "servers": {
    "wink-docs": {
      "type": "http",
      "url": "https://docs.mcp.wink.travel/mcp"
    }
  }
}
```

Поради:
- Використовуйте Command Palette: “MCP: List Servers” для підтвердження підключення та перегляду інструментів.
- Вибір інструментів дозволяє увімкнути/вимкнути `sdk_search` та `docs_search`.
- Додайте контекст: Перегляд чату → Додати контекст → MCP Resources для прикріплення сторінок документації.

## Основні URI ресурсів

| URI | Опис |
|-----|-------------|
| `wink://docs` | Індекс документації (групований за категоріями) |
| `wink://docs/{path}` | Конкретна стаття документації (Markdown / MDX) |
| `wink://sdk` | Огляд SDK (кожен SDK + кількість документів) |
| `wink://sdk/{sdkPath}` | README SDK / папка з документацією модуля / конкретна документація модуля |

### Документація модулів
Відкриття папки документації SDK (наприклад, `wink://sdk/python/wink-sdk-booking/docs`) повертає згруповані:
- **API класи** (кількість методів + приклади назв методів)
- **Моделі та типи** (структури даних, що використовуються цими API)

## Інструменти пошуку (швидка довідка)
| Інструмент | Призначення | Типові приклади запитів |
|------------|-------------|------------------------|
| `sdk_search` | Знайти, який SDK / API клас використовувати | `скасувати бронювання`, `список об’єктів`, `партнерський інвентар` |
| `docs_search` | Знайти концептуальну документацію / керівництва | `авторизація`, `вебхуки`, `wordPress` |

Залиште `query` порожнім, щоб отримати пропозиції. Кожен результат містить MCP URI, який можна відкрити.

## Типовий сценарій використання
1. Запустіть `sdk_search` з вашим наміром (наприклад, *"список об’єктів"*).
2. Відкрийте повернутий README SDK або його `docs_uri` для ознайомлення з API / моделями.
3. Використовуйте `docs_search` для підтримуючих концепцій (авторизація, вебхуки).

## Примітки
- Кожен повернутий URI може бути відкритий агентом для повного вмісту.
- Пропозиції з’являються автоматично, коли запити порожні.
