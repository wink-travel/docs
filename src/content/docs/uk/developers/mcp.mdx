## Огляд
Хостинговий MCP сервер Wink доступний за адресою:

```
https://docs.mcp.wink.travel/mcp
```

Направляйте сумісні AI клієнти (VS Code, Cursor, інші, що реалізують MCP HTTP транспорт) на цю кінцеву точку, щоб вони могли:
- Переглядати документацію продукту та платформи (`wink://docs/...`)
- Переглядати живі OpenAPI специфікації та документацію операцій (`wink://api/...`)
- Виконувати легкий семантичний / ключовий пошук по документації та живих API

## Транспорт
Використовуйте основну HTTP кінцеву точку з потоковою підтримкою:

```
https://docs.mcp.wink.travel/mcp
```

Якщо ваш клієнт підтримує лише SSE, використовуйте:

```
https://docs.mcp.wink.travel/sse
```

### Claude Code
Офіційна документація: [Підключення Claude Code до інструментів через MCP](https://code.claude.com/docs/en/mcp)

Anthropic рекомендує віддалений HTTP транспорт для хмарних MCP серверів:
```bash
claude mcp add --transport http wink-docs https://docs.mcp.wink.travel/mcp
claude mcp list
```

У Claude Code виконайте `/mcp` для перевірки статусу сервера. Якщо хочете поділитися сервером з усім репозиторієм, додайте `--scope project`, щоб Claude Code записав конфіг у `.mcp.json`.

### Codex
Офіційна документація: [Швидкий старт Docs MCP для Codex](https://developers.openai.com/resources/docs-mcp)

OpenAI документує налаштування MCP через Codex CLI, зазначаючи, що конфігурація спільна з розширенням IDE:
```bash
codex mcp add wink-docs --url https://docs.mcp.wink.travel/mcp
codex mcp list
```

### Cursor
Документація Cursor MCP: https://cursor.com/docs/context/mcp
Налаштування: Додайте віддалений сервер через налаштування MCP.
```json
{
  "mcpServers": {
    "wink-docs": {
      "url": "https://docs.mcp.wink.travel/mcp",
    }
  }
}
```

### VS Code
Офіційна документація: https://code.visualstudio.com/docs/copilot/customization/mcp-servers
Мінімальна віддалена конфігурація (користувацький або робочий простір `mcp.json`):
```json
{
  "servers": {
    "wink-docs": {
      "type": "http",
      "url": "https://docs.mcp.wink.travel/mcp"
    }
  }
}
```

Поради:
- Використовуйте Command Palette: “MCP: List Servers” для підтвердження підключення та перегляду інструментів.
- Вибір інструментів дозволяє увімкнути/вимкнути `api_search` та `docs_search`.
- Додайте контекст: Перегляд чату → Додати контекст → MCP Resources для прикріплення сторінок документації.

## Основні URI ресурсів

| URI | Опис |
|-----|-------------|
| `wink://docs` | Індекс документації (групований за категоріями) |
| `wink://docs/{path}` | Конкретна стаття документації (Markdown / MDX) |
| `wink://api` | Живий каталог OpenAPI, виявлений з конфігурації swagger Wink |
| `wink://api/{apiName}` | Стисле резюме API з автентифікацією, тегами, заголовками та прикладами операцій |
| `wink://api/{apiName}/operations` | Повний перелік операцій, згрупованих за тегом |
| `wink://api/{apiName}/operation/{key}` | Деталі окремої операції (параметри, тіло запиту, відповіді) |
| `wink://api/{apiName}/schema/{schemaName}` | Розгорнута схема компонента з вкладеними полями, типами та прикладами |
| `wink://api/{apiName}/operation/{key}/request` | Перегляд, орієнтований на запит, з необхідними вхідними даними та мінімальним прикладом |
| `wink://api/{apiName}/operation/{key}/response/{status}` | Перегляд відповіді для конкретного коду статусу |
| `wink://api/{apiName}/operation/{key}/recipe` | Орієнтований на завдання потік для правильного виклику цієї операції |

## Інструменти пошуку (швидкий довідник)
| Інструмент | Призначення | Типові приклади запитів |
|------------|-------------|-------------------------|
| `api_search` | Знайти живий API або операцію для інтеграції | `create affiliate`, `payment receipt`, `lookup inventory`, `invite manager` |
| `docs_search` | Знайти концептуальну документацію / керівництва | `authentication`, `webhooks`, `wordPress` |

Залиште `query` порожнім, щоб отримати пропозиції. Кожен результат містить MCP URI, який можна відкрити.
Для збігів операцій `api_search` також повертає пов’язані URI запитів, відповідей, схем та рецептів, щоб агенти могли перейти безпосередньо до найкорисніших підресурсів.

## Типовий сценарій використання
1. Запустіть `api_search` з вашим наміром (наприклад, *"create affiliate"* або *"payment receipt"*).
2. Відкрийте повернутий URI `wink://api/...` або перейдіть за пов’язаними полями `request_uri`, `response_uris`, `request_schema_uris`, `response_schema_uris` та `recipe_uri`.
3. Використовуйте `docs_search` для підтримуючих концепцій (автентифікація, вебхуки, робочі процеси).

## AI Навички

Попередньо створені навички агентів використовують цей MCP сервер як джерело істини. Навички дають вашому кодувальному агенту структурований робочий процес для роботи з Wink API — замість здогадок про кінцеві точки агент читає живі контракти безпосередньо з MCP сервера.

Дивіться [AI Skills](/uk/developers/ai-skills) для доступних навичок та інструкцій з встановлення.

## Примітки
- Кожен повернутий URI може бути відкритий агентом для повного перегляду вмісту.
- Дерево `wink://api` живе і слідує за поточним upstream каталогом OpenAPI, а не згенерованим індексом репозиторію.
- Пропозиції з’являються автоматично, коли запити порожні.
