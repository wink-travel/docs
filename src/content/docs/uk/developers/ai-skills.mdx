Навички ШІ — це портативні пакети інструкцій, які навчають агентів кодуванню працювати з Wink. Замість того, щоб покладатися на навчальні дані або застарілу документацію, кожен навик направляє агента використовувати [Wink MCP server](/uk/developers/mcp) як актуальне джерело істини для контрактів API та документації платформи.

Навички працюють з будь-яким клієнтом агента, який їх підтримує — Claude Code, Codex, Cursor та іншими.

## Доступні навички

### wink-api-builder

Реалізуйте, перевіряйте або налагоджуйте інтеграції Wink API. Навик проводить агента через структурований робочий процес:

1. Пошук потрібної операції API через `api_search`
2. Відкриття живих контрактів запитів/відповідей з MCP сервера
3. Отримання налаштувань автентифікації з документації
4. Написання коду відповідно до перевіреного контракту
5. Позначення будь-яких невідповідностей між існуючим кодом і поточним API

Корисно для:
- Створення нових методів клієнта для будь-якого Wink API
- Налагодження невідповідностей контрактів (наприклад, запит, що повертає HTTP 400)
- Перевірки існуючих інтеграцій відповідно до живої специфікації

## Встановлення

Скопіюйте папку навичок у директорію навичок вашого агента.

:::tip
Wink MCP server має бути підключений у тому ж середовищі. Дивіться [MCP Server](/uk/developers/mcp) для інструкцій з налаштування.
:::

**Claude Code**

```bash
# локально в репозиторії
cp -r skills/wink-api-builder .claude/skills/

# або на рівні користувача
cp -r skills/wink-api-builder ~/.claude/skills/
```

Викликати так:
```
/wink-api-builder add a TypeScript client for the Wink affiliate create flow
```

**Codex**

```bash
# локально в репозиторії
cp -r skills/wink-api-builder .agents/skills/

# або на рівні користувача
cp -r skills/wink-api-builder $CODEX_HOME/skills/
```

Викликати так:
```
Use $wink-api-builder to add a TypeScript client for the Wink affiliate create flow.
```

**Інші агенти**

Розмістіть папку навичок там, де ваш агент шукає навички. Точка входу — `SKILL.md`.
