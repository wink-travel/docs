Коли бронювання створюється, оновлюється або скасовується у Wink, платформа синхронізує цю зміну з вашими підключеними зовнішніми системами — системами управління готелем (PMS), channel managers та центральними системами бронювання (CRS).

## Як це працює

Кожна зміна бронювання створює **запис синхронізації**, який відстежує стан синхронізації. Платформа обробляє записи синхронізації асинхронно, повторюючи спроби у разі невдачі, щоб забезпечити надійну доставку.

## Цикл синхронізації

Кожен запис синхронізації проходить через машину станів:

```
┌─────────────┐     success     ┌───────────┐
│  TO_CREATE   │ ──────────────→ │ PROCESSED │
└─────────────┘                  └───────────┘
       │
       │ failure
       ▼
┌──────────────────┐
│ TO_CREATE_FAILED │
└──────────────────┘
```

Такий самий шаблон застосовується до оновлень і скасувань:

| Дія | Очікуваний стан | Стан успіху | Стан невдачі |
|---|---|---|---|
| Нове бронювання | `TO_CREATE` | `PROCESSED` | `TO_CREATE_FAILED` |
| Оновлення бронювання | `TO_UPDATE` | `PROCESSED` | `TO_UPDATE_FAILED` |
| Скасування | `TO_CANCEL` | `PROCESSED` | `TO_CANCEL_FAILED` |

## Що запускає синхронізацію

- **Нове бронювання** — Мандрівник бронює через Wink або афілійований канал.
- **Оновлення бронювання** — Змінюються деталі бронювання (дати, інформація про гостя, призначення кімнати).
- **Скасування** — Бронювання скасовується мандрівником або власником.

## Моніторинг

Ви можете відстежувати статус синхронізації через панель вашого channel manager або PMS. Записи синхронізації, що не вдалися, автоматично повторюються. Якщо синхронізація продовжує зазнавати невдач, вона залишається у стані `*_FAILED` для подальшого розслідування.

## Усунення несправностей

Якщо бронювання не з’являються у вашому PMS:

1. **Перевірте підключення channel manager** — Переконайтеся, що інтеграція з PMS або channel manager активна і налаштована правильно.
2. **Перевірте доставку webhook** — Якщо ви використовуєте webhook, перевірте, що ваш endpoint отримує події `booking.create` та `booking.cancelled`. Див. [Каталог подій webhook](/uk/developers/webhook-events#booking-events).
3. **Перегляньте записи з помилками** — Записи синхронізації з помилками вказують на проблему з підключенням або даними у цільовій системі.
4. **Зверніться до служби підтримки** — Якщо проблеми тривають, команда Wink може розслідувати записи синхронізації з помилками від вашого імені.
