Система бронювання доступна у двох варіантах, щоб підтримувати різні потреби афілійованих осіб, компаній з управління нерухомістю та готелів.

## Онлайн туристична агенція

Для афілійованих осіб найбільш доцільно використовувати версію онлайн туристичної агенції. Причина в тому, що сайт налаштований для мандрівників, які хочуть переглядати кілька об’єктів нерухомості.

Нижче ви можете знайти головну сторінку цього застосунку.

<LinkButton href="https://ota.wink.travel/home" variant="secondary" icon="external" target="_blank">
  Wink OTA
</LinkButton>

## Інтернет-система бронювання

Для готелю або компанії з управління нерухомістю найбільш доцільно використовувати версію інтернет-системи бронювання. Цей тип користувачів зацікавлений у висвітленні одного об’єкта або кількох об’єктів під управлінням.

Нижче ви можете знайти головну сторінку цього застосунку.

## Налаштування

Спробуйте налаштувати вашу тему у нашій пісочниці для розробників нижче.

## Конфігурація

Зазвичай немає потреби безпосередньо ділитися цими застосунками. Ми створили різні способи для вас, щоб ділитися туристичним інвентарем, які ви можете знайти у [Wink Studio](/studio/what-is-studio) або через [WinkLinks](/link-manager/wink-links).

Однак, якщо ви хочете вручну створити посилання на один із цих сайтів, переконайтеся, що додали ваш client ID та customization ID у URL як параметри.

Ви можете знайти свій client ID, виконавши:

<Steps>
1. Увійдіть у [Wink Extranet](/extranet/what-is-extranet) або [Wink Studio](/studio/what-is-studio).
2. Натисніть на іконку профілю у верхньому правому куті та виберіть [Applications](/developers/applications).
3. Використовуйте застосунок, пов’язаний з обліковим записом Wink, з яким хочете працювати.
4. Скопіюйте Client ID з цього застосунку та вставте його у URL як параметр запиту. Приклад: ?client-id=MY_CLIENT_ID
</Steps>

### Extranet

Ви можете знайти свій customization ID як власник об’єкта, виконавши:

<Steps>
1. Увійдіть у [Wink Extranet](/extranet/what-is-extranet).
2. Виберіть об’єкт, з яким хочете працювати.
3. У головному навігаційному меню натисніть `Account > Branding`
4. Скопіюйте Customization ID і вставте його у URL як параметр запиту. Приклад: ?configuration-id=MY_CUSTOMIZATION_ID
</Steps>

### Studio

Ви можете знайти свій customization ID як афілійований користувач, виконавши:

<Steps>
1. Увійдіть у [Wink Studio](/studio/what-is-studio).
2. Виберіть обліковий запис, з яким хочете працювати.
3. У головному навігаційному меню натисніть `Tools > Customization`
4. Скопіюйте Customization ID з налаштування, яке хочете використати, і вставте його у URL як параметр запиту. Приклад: ?configuration-id=MY_CUSTOMIZATION_ID
</Steps>

## OTA

В результаті ви отримаєте URL, який виглядає приблизно так:

https://ota.wink.travel?client-id=MY_CLIENT_ID&configuration-id=MY_CUSTOMIZATION_ID

Якщо ви хочете перейти безпосередньо на сторінку готелю, URL виглядатиме приблизно так:

https://ota.wink.travel/hotel/hotel-x?client-id=MY_CLIENT_ID&configuration-id=MY_CUSTOMIZATION_ID

:::note
Якщо ви не включите client ID та/або configuration ID, Wink OTA використовуватиме конфігурацію за замовчуванням.
:::

## IBE

Якщо ви компанія з управління нерухомістю, ви хочете створити посилання на головну сторінку Wink IBE. Головна сторінка відобразить вашу призначену тему разом із [об’єктами, які ви представляєте](/studio/agency-representation).

В результаті ви отримаєте URL, який виглядає приблизно так:

https://book.wink.travel?client-id=MY_CLIENT_ID&configuration-id=MY_CUSTOMIZATION_ID

Якщо ви власник об’єкта, ви хочете створити посилання безпосередньо на сторінку об’єкта Wink IBE.

Цей URL виглядатиме приблизно так:

https://book.wink.travel/hotel/hotel-x?client-id=MY_CLIENT_ID&configuration-id=MY_CUSTOMIZATION_ID


:::note
Якщо ви не включите client ID та/або configuration ID, Wink IBE НЕ завантажиться. Ці параметри є обов’язковими.
:::
