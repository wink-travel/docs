Remix — це full-stack фреймворк, який за замовчуванням виконує серверний рендеринг. Завантажуйте Wink CDN лише на клієнті, використовуючи `useEffect` у кореневому компоненті.

<LinkCard
  title="@wink/elements npm package"

/>

## Встановлення

```bash
npm install @wink/elements
```

## Завантаження в app/root.tsx

`useEffect` виконується лише в браузері, ніколи під час серверного рендерингу:

```tsx
// app/root.tsx
import { useEffect } from 'react';
import {
  Links, Meta, Outlet, Scripts, ScrollRestoration,
} from '@remix-run/react';
import { load } from '@wink/elements';

export default function App() {
  useEffect(() => {
    load({ clientId: import.meta.env.VITE_WINK_CLIENT_ID });
  }, []);

  return (
    <html lang="en">
      <head>
        <Meta />
        <Links />
      </head>
      <body>
        <Outlet />
        <ScrollRestoration />
        <Scripts />
      </body>
    </html>
  );
}
```

## Змінні середовища

Remix (на базі Vite, v2.8+) підтримує `import.meta.env` для змінних, доступних у браузері. Додайте префікс `VITE_`, щоб зробити їх доступними у браузерному бандлі:

```
# .env
VITE_WINK_CLIENT_ID=your-client-id
```

Для старіших налаштувань Remix з Node.js адаптером повертайте client ID з root loader і читайте його в компоненті:

```tsx
// app/root.tsx
import { json } from '@remix-run/node';
import { useLoaderData } from '@remix-run/react';

export async function loader() {
  return json({ winkClientId: process.env.WINK_CLIENT_ID });
}

export default function App() {
  const { winkClientId } = useLoaderData<typeof loader>();

  useEffect(() => {
    if (winkClientId) {
      load({ clientId: winkClientId });
    }
  }, [winkClientId]);

  // ...
}
```

## Використання в маршрутах

```tsx
// app/routes/hotels.tsx
export default function HotelsRoute() {
  return (
    <main>
      <wink-content-loader layout="HOTEL" id="YOUR_LAYOUT_ID" />
    </main>
  );
}
```

<Aside type="caution" title="Несумісність гідратації">
Remix гідрує серверно відрендерений HTML на клієнті. Якщо елемент `<wink-*>` відрендерено на сервері, браузер може короткочасно показати нерозпізнаний кастомний елемент, перш ніж скрипт CDN його зареєструє. Це косметичне явище — елемент коректно оновиться після завантаження скрипта.
</Aside>
