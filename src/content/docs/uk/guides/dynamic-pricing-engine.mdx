Ціновий движок Wink виходить за межі простих нічних тарифів. Він динамічно обчислює кінцеву ціну, яку бачить мандрівник, поєднуючи кілька рівнів ціноутворення — основні тарифи, тарифні плани, акції та конфігурації номерів.

## Як працює ціноутворення

Ціновий движок оцінює кілька рівнів для обчислення кінцевого тарифу:

### 1. Основні тарифи

Основи вашого ціноутворення. Основні тарифи визначають базову ціну для кожного типу номера або одиниці інвентарю. Ви встановлюєте їх в Extranet, і вони слугують відправною точкою для всіх розрахунків цін.

<LinkButton href="/uk/extranet/master-rates">
  Налаштування основних тарифів
</LinkButton>

### 2. Тарифні плани

Тарифні плани змінюють основні тарифи для конкретних умов. Тарифний план може коригувати ціни на основі:

- **Завантаженості** — різні ціни для одно-, дво- або тримісного розміщення.
- **Тривалості проживання** — знижки за довше перебування.
- **Включення харчування** — лише номер, сніданок, напівпансіон або все включено.
- **Вікна бронювання** — раннє бронювання або ціни в останню хвилину.

<LinkButton href="/uk/extranet/rate-plans">
  Налаштування тарифних планів
</LinkButton>

### 3. Щоденні тарифи

Щоденні тарифи дозволяють перевизначати ціни для конкретних дат. Використовуйте їх для:

- **Сезонного ціноутворення** — вищі тарифи в пік сезону, нижчі — в міжсезоння.
- **Цін на події** — підвищені тарифи під час місцевих заходів або свят.
- **Дат блокування** — дати, коли певні тарифи недоступні.

<LinkButton href="/uk/extranet/rates">
  Керування щоденними тарифами
</LinkButton>

### 4. Акції

Акції застосовують знижки або спеціальні пропозиції поверх обчисленого тарифу:

- **Відсоткові знижки** — наприклад, 20% знижки за раннє бронювання.
- **Промоції з додатковою вартістю** — наприклад, безкоштовний сніданок, кредит на спа.
- **Пропозиції з обмеженням за датами** — акції, дійсні лише для певних дат подорожі або бронювання.

## Обчислення тарифу за проживання

Коли мандрівник шукає проживання на кілька ночей, движок обчислює загальну суму, виконуючи:

1. Пошук **основного тарифу** для типу номера.
2. Застосування відповідних коригувань **тарифного плану**.
3. Перевірку **перевизначень щоденних тарифів** на кожну ніч проживання.
4. Застосування будь-яких **акцій**, що відповідають умовам бронювання.
5. Підсумовування тарифів за кожну ніч у **загальний тариф за проживання**.

Це означає, що кожна ніч проживання може мати різний ефективний тариф, і мандрівник бачить точну загальну суму, що відображає всі ваші правила ціноутворення.

## Вплив політики скасування

Політики скасування також можуть впливати на відображену ціну. Гнучкі політики скасування можуть мати невелику надбавку порівняно з тарифами без можливості повернення, даючи мандрівникам вибір між ціною та гнучкістю.

## Поради

- **Використовуйте Rate Tester.** В Extranet є інструмент тестування тарифів, який дозволяє симулювати пошуки, щоб перевірити, чи ваші рівні ціноутворення дають очікувані результати.
- **Починайте з основних тарифів, потім додавайте шари.** Спочатку встановіть основні тарифи, потім додайте тарифні плани та акції. Це допомагає зберегти логіку ціноутворення керованою.
- **Переглядайте щоденні тарифи сезонно.** Оновлюйте перевизначення щоденних тарифів перед кожним сезоном, щоб ваша ціна залишалась конкурентоспроможною.

  Перевірте свої тарифи
</LinkButton>
