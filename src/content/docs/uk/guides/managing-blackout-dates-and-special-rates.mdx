Не кожен день має однакову ціну. Події, свята, міжсезоння та останні вільні дати вимагають різного ціноутворення. Цей посібник показує, як використовувати спеціальні тарифи та обмеження тарифних планів для таких випадків.

## Ціноутворення на події

Коли конференція, фестиваль або місцева подія збільшує попит, ви захочете підвищити тарифи на ці дати.

<Steps>
1. Перейдіть у `Monetize > Special Rates` у вашому Extranet.
2. Натисніть `Create a new special rate`.
3. Виберіть основний тариф і тип кімнати, які потрібно замінити.
4. Встановіть діапазон дат, що охоплює період події.
5. Введіть скориговані (вищі) щоденні ціни.
6. Натисніть `Save`.
</Steps>

Спеціальний тариф замінює основний тариф лише на ці дати. Ваше звичайне ціноутворення залишається незмінним поза періодом події.

## Сезонні коригування

Для тривалих періодів (високий сезон, міжсезоння, низький сезон) ви можете:

- **Використовувати спеціальні тарифи** — створювати заміни для кожного сезону з відкоригованими цінами.
- **Оновлювати основні тарифи** — безпосередньо коригувати базові тарифи для цих періодів.

Спеціальні тарифи краще підходять для тимчасових коригувань, тоді як оновлення основних тарифів має сенс для передбачуваних щорічних змін.

## Тарифи останньої хвилини

Щоб заповнити непродані кімнати ближче до дати заїзду:

<Steps>
1. Створіть новий **тарифний план** з обмеженням на попереднє бронювання (наприклад, бронювання не більше ніж за 3 дні).
2. Поєднайте його зі **спеціальним тарифом**, що пропонує знижку на цільові дати.
3. Цей тарифний план буде доступний лише мандрівникам, які бронюють у вікні останньої хвилини.
</Steps>

## Дати блокування

Щоб заблокувати певні дати для конкретного тарифного плану (наприклад, без промоційних тарифів у пік сезону):

<Steps>
1. Перейдіть у `Monetize > Rate Plans` і виберіть тарифний план.
2. Перейдіть до розділу **Restrictions**.
3. Додайте обмеження **Sell date**, щоб обмежити період, коли тариф можна забронювати.
4. Або використовуйте обмеження **Stay date**, щоб обмежити період перебування, який покриває тариф.
</Steps>

Наприклад, ви можете обмежити тарифний план "Early Bird", щоб він не був доступний для бронювань у грудні (пік сезону), тоді як ваш звичайний тарифний план залишатиметься відкритим.

## Поєднання обмежень

Обмеження можна комбінувати для точного контролю:

| Обмеження | Що контролює |
| -- | -- |
| **Sell dates** | Коли тариф можна забронювати |
| **Stay dates** | Які дати перебування покриває тариф |
| **Available days** | У які дні тижня тариф відображається |
| **Arrival days** | У які дні гість повинен приїхати |
| **Departure days** | У які дні гість повинен виїхати |
| **Required days** | Гість повинен залишатися принаймні один із цих днів |
| **Advance booking** | За скільки часу до заїзду має бути зроблено бронювання |
| **Length of stay** | Мінімальна та максимальна кількість ночей |
