Якщо ви інтегратор, наприклад Wink, і хочете зіставити своїх зовнішніх отримувачів платежів для використання з TripPay, ви можете зробити це одним із двох способів:

- Ручне відображення
- Використання [Payment API](/uk/developers/apis/#payment-api) (*Див. `Account Mappings`*)

:::tip
Якщо ви працюєте не лише з кількома однаковими об’єктами, ми рекомендуємо використовувати API для автоматизації цього кроку.
:::

Нижче ми крок за кроком покажемо вам ручний спосіб відображення зовнішнього отримувача платежів. *Він використовує API "під капотом".*

<Steps>
1. Увійдіть у [TripPay](https://pay.wink.travel).
2. Виберіть свій інтеграторський обліковий запис.
3. Натисніть `Settings > Mappings` у головному навігаційному меню.
4. Натисніть кнопку `Create mapping`. Ви побачите форму відображення.
5. **Local account** Почніть вводити назву існуючого облікового запису TripPay, який хочете зіставити.
6. **External identifier** Вставте свій зовнішній ідентифікатор для цієї сутності, як він існує у віддаленій системі.
7. Натисніть `Save`, щоб продовжити. Вас буде перенаправлено назад до списку відображень.
</Steps>

Зі створеними відображеннями TripPay тепер знає, які облікові записи кредитувати як бенефіціарів бронювання, коли ви посилаєтеся на них за допомогою зовнішніх ідентифікаторів.

## API

Розробники, які хочуть керувати `Payments`, переходьте до [Developers > API > Payments](/uk/developers/apis/#payment-api).
