Якщо ви інтегратор, як-от Wink, і хочете зіставити своїх зовнішніх одержувачів платежів для використання з TripPay, ви можете зробити це одним із двох способів:

* Ручне картографування
* Використання[API оплати](/developers/apis/#payment-api)(*Див.`Account Mappings`*)

:::tip
Якщо ви не маєте справу лише з кількома однаковими властивостями, ми рекомендуємо вам використовувати API для автоматизації цього кроку. :::

Нижче ми покроково розглянемо ручний спосіб зіставлення зовнішнього отримувача платежу.*Він використовує вбудований API.*

<Steps>
  1. Увійти [TripPay](https://pay.wink.travel).
  2. Виберіть свій обліковий запис інтегратора.
  3. Натисніть `Settings > Mappings` з головної панелі навігації.
  4. Натисніть на `Create mapping` кнопка. Ви побачите форму зіставлення.
  5. **Локальний обліковий запис** Почніть вводити назву існуючого TripPay, який ви хочете зіставити.
  6. **Зовнішній ідентифікатор** Вставте свій зовнішній ідентифікатор для цієї сутності, оскільки він існує у віддаленій системі.
  7. Натисніть `Save` щоб продовжити. Ви будете перенаправлені назад до списку зіставлень.
</Steps>


Завдяки вашим зіставленням TripPay тепер знає, які облікові записи зараховувати як бенефіціарів бронювання, коли ви звертаєтеся до них за допомогою своїх зовнішніх ідентифікаторів.

## API

Розробники, які хочуть керувати`Payments`, перейдіть до[Розробники > API > Платежі](/developers/apis/#payment-api).
