Webhookid võimaldavad teie süsteemidel saada reaalajas teavitusi, kui teie Wink kontol toimuvad sündmused — uued broneeringud, tühistamised, maksete uuendused ja palju muud. See juhend juhendab teid seadistamise ja parimate tavade osas.

## Sihtgrupp

See juhend on mõeldud arendajatele, kes integreerivad Winki väliste süsteemidega, nagu kinnisvarahaldussüsteemid (PMS), kanalihaldurid, CRM-id või kohandatud juhtpaneelid.

## Kuidas webhookid töötavad

1. Registreerite Winkis webhooki URL-i.
2. Kui toimub sündmus (nt uus broneering), saadab Wink teie URL-ile HTTP POST päringu.
3. Teie server töötleb andmepaketi ja vastab `200 OK`-ga.

## Webhooki seadistamine

<Steps>
1. Logige sisse oma kontole (Extranet, Studio või TripPay — kõik toetavad webhooke).
2. Minge jaotisse `Applications` ja seejärel `Webhooks`. Vaadake [Webhooks](/et/my-account/webhooks).
3. Klõpsake `Create webhook`.
4. Sisestage **nimi** (nt "PMS Broneeringu sünkroonimine").
5. Sisestage oma **webhooki URL** — HTTPS lõpp-punkt teie serveris.
6. **Valige sündmused** — valige konkreetsed sündmused, millele tellida, või jätke tühjaks, et saada kõik sündmused.
7. Lülitage **Enabled** sisse.
8. Klõpsake `Save`.
</Steps>

## Sündmuste tüübid

Wink toetab **üle 64 webhooki sündmuse tüübi** erinevates kategooriates, sealhulgas konto haldus, broneeringulepingud, maksed, kanalihalduri uuendused ja haldusüksuste juhtimine. Levinumad sündmused on:

| Kategooria | Näited |
|---|---|
| **Broneering** | `booking.create`, `booking.cancelled` |
| **Broneeringuleping** | `booking.contract.created`, `booking.contract.processing`, `booking.contract.cancelled` |
| **Makse** | `internal.payment.succeeded`, `internal.payment.failed` |
| **Tagasimaksed** | `booking.contract.refund.partial`, `booking.contract.refund.full` |
| **Kanalihaldur** | `channel-manager.update.rate`, `channel-manager.update.availability` |
| **Konto** | `account.create`, `account.status.updated` |

Kõigi sündmuste tüüpideta ja kirjeldustega leiate **Webhook Events Catalog**-ist.

</LinkButton>

:::tip
Alustage arendamise ajal kõigi sündmuste tellimisest. Kui olete andmepakette mõistnud, kitsendage valikut ainult teie süsteemile vajalikele sündmustele.
:::

## Parimad tavad

- **Kasutage HTTPS-i** — Wink saadab andmepakette ainult HTTPS lõpp-punktidele.
- **Vastake kiiresti** — Tagastage `200 OK` kohe, kui andmepakett on vastu võetud. Kõik mahukad protsessid tehke asünkroonselt.
- **Idempotentsus** — Teie töötleja peaks olema idempotentne. Wink võib proovida andmepaketti uuesti saata, kui `200` vastust ei saa.
- **Allika valideerimine** — Kontrollige, et sissetulevad päringud pärineksid Winkist, enne kui neid töödeldakse.
- **Logimine** — Logige iga vastu võetud webhooki andmepakett. See teeb integreerimisvigade silumise palju lihtsamaks.

## Peatamine ja kustutamine

Webhooki saab **keelata** ilma seda kustutamata. See peatab andmete edastamise, et saaksite tõrkeotsingut teha ilma konfiguratsiooni kaotamata. Kui olete valmis, lülitage see uuesti sisse.

Webhooki kustutamine eemaldab selle jäädavalt. Kõik integreerimised, mis sõltuvad sellest webhookist, lõpetavad teavituste saamise.
