Kui te viite läbi reklaamikampaaniaid Sojerniga, saate oma Sojern konto ühendada Winkiga, sisestades Wink portaali oma Sojern GTM konteineri ID. Kui ühendus on loodud, aktiveerub Sojerni jälgimispiksel automaatselt teie objekti jaoks — hõlmates kogu külaliste teekonda otsingust kuni broneeringu lõpuleviimiseni.

<Aside type="tip">
See eeldab, et teil on juba aktiivne Sojern konto ja olete saanud oma GTM konteineri ID Sojerni kontohaldurilt. Kui teil veel Sojern kontot ei ole, külastage [sojern.com/contact](https://www.sojern.com/contact), et alustada.
</Aside>

---

## Mida peate tegema

<Steps>

1. **Hankige oma GTM konteineri ID Sojernist**

   Võtke ühendust oma Sojerni kontohalduriga ja küsige oma **GTM konteineri ID-d**. See on vormingus `GTM-XXXXXXX`. See erineb teie Sojern Pixli ID-st — veenduge, et küsiksite just GTM konteineri ID-d.

2. **Sisestage see Wink portaali**

   Logige sisse [my.wink.travel](https://my.wink.travel), valige oma konto ja minge vahekaardile **Kohandused**. Kleepige oma Sojern GTM konteineri ID Sojern väljale ja salvestage. Teie integratsioon on nüüd aktiivne.

3. **Kinnitage oma Sojerni kontohalduriga**

   Teavitage oma Sojerni kontohaldurit, et integratsioon on aktiivne. Nad saavad oma poolt kontrollida, et piksel töötab korrektselt aadressil `wink.travel` teie objekti jaoks ja et andmed voolavad teie Sojern kontole.

</Steps>

<Aside type="note">
Selle integratsiooni jaoks ei ole vaja oma veebisaidile paigaldada ühtegi pikselikoodi. Wink laadib teie Sojern konteineri automaatselt kõigil lehtedel, kus teie objekt on aktiivne.

See konfiguratsioon töötab mis tahes kohandatud GTM konteineri ID-ga. Me kasutasime Sojerni näitena, kuna see on väga levinud integratsioon.
</Aside>

## Mida Wink meie pool teeb

Läbipaistvuse huvides on siin, mida Wink seadistab selle integratsiooni toimimiseks:

- **Sojern GTM konteineri ID väli** on saadaval Wink portaali Kohandused vahekaardil. Kui sisestate konteineri ID, kontrollib Wink selle vormingut ja salvestab selle turvaliselt teie objekti andmetesse.
- **Igal lehel, kus teie objekt on Winkis aktiivne**, renderdatakse teie Sojern GTM konteineri ID lehele ja Wink'i sildihaldussüsteem laadib dünaamiliselt teie Sojern konteineri koos oma omaga. See toimub automaatselt — Wink meeskonnal ei ole vaja objekti kohta käsitsi sekkuda.
- **Teie Sojern konteiner laaditakse asünkroonselt**, mis tähendab, et see ei mõjuta teie külaliste lehe laadimise kiirust.
- **Wink'i enda jälgimine ja teie Sojern piksel töötavad paralleelselt**, kasutades sama jagatud sündmuse andmeid. Iga broneerimissündmus, mida Wink jälgib, on samaaegselt kättesaadav teie Sojern konteinerile.
- **Kui teie objekti ei vaadata** — näiteks kui külaline sirvib Winkis mõnda teist hotelli — ei laadita teie Sojern konteinerit. See tagab, et Sojern saab ainult teie objekti jaoks asjakohaseid andmeid.

## DataLayer viide

Wink saadab GA4-ga ühilduvaid sündmusi `dataLayer`-isse kogu külaliste teekonna vältel. Teie GTM konteiner loeb samast jagatud `dataLayer`-ist, nii et iga teie seadistatud käivitaja või muutuja pääseb ligi kõigile allpool kirjeldatud omadustele.

Iga sündmus järgib standardset GA4 täiustatud e-kaubanduse struktuuri: ülemise taseme `event` stringi ja `ecommerce` objekti. Enne iga pushi tühjendab Wink eelmise e-kaubanduse koormuse, et vältida aegunud andmete kandumist uutesse käivitajatesse.

<Aside type="note">
**dataLayer lähtestamise muster**

Wink lähtestab e-kaubanduse objekti enne iga sündmuse pushi:

```js
dataLayer.push({ ecommerce: null });   // eelmine sündmus tühjendatud
dataLayer.push({ event: '...', ecommerce: { ... } });
```

Ärge eeldage, et e-kaubanduse omadused püsivad sündmuste vahel. Lugege neid alati käivitamise ajal samas sündmuses.
</Aside>

### Broneerimisteekonna sündmused

Need sündmused käivituvad automaatselt, kui külaline liigub broneerimislehtris. Seadistage oma GTM käivitajad kui **Kohandatud sündmuse** käivitajad, mis vastavad allpool toodud sündmuste nimedele.

| Samm | Sündmus | Millal käivitub |
|------|---------|-----------------|
| 1 | `view_item_list` | Hotelli inventari ruudustik või nimekiri laetakse |
| 2 | `view_item` | Külaline vaatab konkreetset objekti lehte |
| 3 | `add_to_cart` | Külaline lisab toa ostukorvi |
| 4 | `begin_checkout` | Kassa / makse leht laetakse |
| 5 | `add_payment_info` | Külaline valib makseviisi |
| 6 | `login` | Külaline autentib end SSO kaudu |
| 7 | `purchase` | Broneering kinnitatakse |
| 8 | `refund` | Broneering tühistatakse |

### Kohandatud mõõdikud GA4-s

GA4 ignoreerib kohandatud tooteparameetreid aruannetes, kuni te need registreerite kui tooteulatuses kohandatud mõõdikud. Registreerima peate ainult need parameetrid, mida soovite aruannetes kasutada — kõiki 18 ei ole vaja registreerida.

<Steps>

1. **Avage Kohandatud määratlused**

   GA4-s minge **Administraator** (vasak all hammasratta ikoon) → valige oma omadus → **Andmete kuvamine** → **Kohandatud määratlused**.

2. **Looge uus kohandatud mõõdik**

   Klõpsake **Loo kohandatud mõõdikud**.

3. **Täida mõõdiku andmed**

   - **Mõõdiku nimi** — inimloetav silt, mis kuvatakse GA4 aruannetes (vt allolevat tabelit)
   - **Ulatus** — valige **Item**
   - **Sündmuse parameeter** — täpne parameetri nimi, nagu see on dataLayer’is (vt allolevat tabelit)
   - **Kirjeldus** — valikuline, kuid soovitatav

4. **Salvesta ja korda**

   Klõpsake **Salvesta** ja korrake iga jälgitava parameetri puhul.

</Steps>

<Aside type="note">
Kohandatud mõõdikud võivad GA4 aruannetes ilmuda kuni 24 tunni jooksul pärast registreerimist. Enne registreerimist kogutud ajaloolisi andmeid ei täideta tagantjärele.

GA4 lubab standardsetel omadustel kuni 50 tooteulatuses kohandatud mõõdikut (360 omadused lubavad rohkem).
</Aside>

Kasutage allolevat tabelit viitena kohandatud mõõdikute loomisel. **Sündmuse parameetri** väärtus peab olema täpselt sama — GA4 on tõstutundlik.

| Mõõdiku nimi | Sündmuse parameeter | Tüüp |
|--------------|---------------------|------|
| Objekti ID | `property_id` | Tekst |
| Objekti nimi | `property_name` | Tekst |
| Objekti bränd | `property_brand` | Tekst |
| Objekti kett | `property_chain` | Tekst |
| Linna nimi | `city_name` | Tekst |
| Riigi kood | `country_code` | Tekst |
| Sisseregistreerimise kuupäev | `start_date` | Tekst |
| Väljaregistreerimise kuupäev | `end_date` | Tekst |
| Täiskasvanute arv | `num_adults` | Number |
| Laste arv | `num_children` | Number |
| Tühistamispoliitika | `cancellation_policy` | Tekst |
| Hinnaplaani ID | `rate_plan_id` | Tekst |
| Toa hinna ID | `room_rate_id` | Tekst |
| Toa hinna nimi | `room_rate_name` | Tekst |
| Lisade arv | `num_perks` | Number |
| Toa asukoht | `room_location` | Tekst |
| Toa vaade | `room_view` | Tekst |
| Toa klassifikatsioon | `room_classification` | Tekst |

### `virtualPageView`

See on kohandatud Wink sündmus — ei kuulu standardsete GA4 sündmuste hulka. See käivitub iga lehe navigeerimisel ja kannab seadme ning sessiooni konteksti. See ei sisalda `ecommerce` objekti.

```json
{
  "event": "virtualPageView",
  "device": "Desktop",
  "deviceAgent": "Mozilla/5.0 ...",
  "os": "MacOS",
  "path": "/hotels/grand-resort",
  "title": "Grand Resort — Wink",
  "userId": "user_abc123",
  "startDate": "2026-06-01",
  "endDate": "2026-06-05",
  "guests": "2",
  "rooms": "1"
}
```

| Omadus | Kirjeldus |
|--------|-----------|
| `device` | Tuvastatud seadme tüüp — `Desktop`, `Android`, `iPhone`, `iPad` või sarnane |
| `deviceAgent` | Kasutajaagendi string |
| `os` | Tuvastatud operatsioonisüsteem — `Windows`, `MacOS`, `Linux`, `UNIX` |
| `path` | Praeguse lehe rada |
| `title` | Praeguse lehe pealkiri |
| `userId` | Autentitud kasutaja identifikaator (puudub, kui pole sisse logitud) |
| `startDate` | Sisseregistreerimise kuupäev praegusest otsingusessioonist |
| `endDate` | Väljaregistreerimise kuupäev praegusest otsingusessioonist |
| `guests` | Külaliste arv praegusest otsingusessioonist |
| `rooms` | Tubade arv praegusest otsingusessioonist |
