Broneerimismootor on saadaval kahes variandis, et toetada sidusettevõtte, kinnisvarahaldusettevõtte ja hotelli erinevaid vajadusi.

## Veebipõhine reisibüroo

Partneri jaoks on kõige mõistlikum kasutada veebireisibüroo versiooni. Põhjus on selles, et sait on loodud nii, et see sobiks reisijatele, kes soovivad vaadata mitut majutuskohta.

Selle rakenduse avalehe leiate altpoolt.

<LinkButton href="https://ota.wink.travel/home" variant="secondary" icon="external" target="_blank">
  Wink OTA
</LinkButton>


## Interneti broneerimismootor

Hotelli või kinnisvarahaldusettevõtte jaoks on kõige mõistlikum kasutada internetipõhise broneerimismootori versiooni. Seda tüüpi kasutaja on huvitatud ühe või mitme hallatava kinnisvaraobjekti esiletõstmisest.

Selle rakenduse avalehe leiate altpoolt.

## Konfiguratsioon

Enamasti pole kumbagi rakendust otse vaja jagada. Oleme loonud reisivarustuse jagamiseks mitmesuguseid viise, millega saate tutvuda siin:[Wink Studio](/studio/what-is-studio)või läbi[WinkLinks](/link-manager/wink-links).

Kui aga soovite ühele neist veebisaitidest käsitsi linkida, lisage URL-ile kindlasti parameetritena oma kliendi ID ja kohandamise ID.

Kliendi ID leiate järgmiselt:

<Steps>
  1. Logi sisse kumbagi [Winki ekstranet](/extranet/what-is-extranet) või [Wink Studio](/studio/what-is-studio).
  2. Klõpsa oma profiiliikoonil paremas ülanurgas ja seejärel klõpsa nupul [Rakendused](/developers/applications).
  3. Kasutage rakendust, mis on seotud Winki kontoga, millega soovite töötada.
  4. Kopeeri kliendi ID sellest rakendusest ja kleebi see URL-ile päringuparameetrina. Näide: ?client-id=MY\_CLIENT\_ID
</Steps>


### Ekstranet

Oma kohandamise ID atribuudina leiate järgmiselt:

<Steps>
  1. Logi sisse [Winki ekstranet](/extranet/what-is-extranet).
  2. Valige omadus, millega soovite töötada.
  3. Klõpsake peamisel navigeerimisribal nuppu `Account > Branding`
  4. Kopeeri kohandamise ID ja kleebi see URL-ile päringuparameetrina. Näide: ?configuration-id=MY\_CUSTOMIZATION\_ID
</Steps>


### Stuudio

Partnerina leiate oma kohandamise ID järgmiselt:

<Steps>
  1. Logi sisse [Wink Studio](/studio/what-is-studio).
  2. Valige konto, millega soovite töötada.
  3. Klõpsake peamisel navigeerimisribal nuppu `Tools > Customization`
  4. Kopeeri soovitud kohanduse ID ja kleebi see URL-ile päringuparameetrina. Näide: ?configuration-id=MY\_CUSTOMIZATION\_ID
</Steps>


## OTA

Lõpptulemuseks on URL, mis näeb välja umbes selline:

https://ota.wink.travel?client-id=MY\_CLIENT\_ID\&configuration-id=MY\_CUSTOMIZATION\_ID

Kui soovite minna otse hotelli lehele, näeks URL välja umbes selline:

https://ota.wink.travel/hotel/hotel-x?client-id=MY\_CLIENT\_ID\&configuration-id=MY\_CUSTOMIZATION\_ID

:::note
Kui te ei lisa kliendi ja/või konfiguratsiooni ID-d, kasutab Wink OTA vaikimisi vaikekonfiguratsiooni.
:::

## IBE

Kui olete kinnisvarahaldusettevõte, soovite linkida Wink IBE avalehele. Avalehel kuvatakse teie määratud teema koos[teie esindatavad kinnisvaraobjektid](/studio/agency-representation).

Lõpptulemuseks on URL, mis näeb välja umbes selline:

https://book.wink.travel?client-id=MY\_CLIENT\_ID\&configuration-id=MY\_CUSTOMIZATION\_ID

Kui olete kinnisvaraettevõte, soovite lingida otse Wink IBE kinnisvara maandumislehele.

See URL näeks välja umbes selline:

https://book.wink.travel/hotel/hotel-x?client-id=MY\_CLIENT\_ID\&configuration-id=MY\_CUSTOMIZATION\_ID

:::note
Kui te ei lisa kliendi ja/või konfiguratsiooni ID-d, siis Wink IBE EI lae. Need parameetrid on kohustuslikud.
:::

## Näide: Kinnisvara leht

Igal Winki majutusasutusel on kummalgi veebisaidil oma unikaalne maandumisleht. Kõik välised lingid suunavad reisijad ühele neist maandumislehtedest.

Maandumisleht sisaldab kogu teavet, mida reisija vajab, et teha teadlik otsus selle majutusasutuse külastamise kohta.
