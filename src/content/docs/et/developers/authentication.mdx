:::tip
Wink IAM autentimisteenus toetab tavalisi OAuth2 lõpp-punkte:

- /oauth2/authorize
- /oauth2/token
- /oauth2/introspect
- /oauth2/revoke
- /oauth2/userinfo
:::

Rakendus annab sulle `Client ID` ja `Secret Key`, mida vajad autentitud OAuth2 sessiooni loomiseks, mida saad kasutada Wink ja TripPay API-dega suhtlemiseks.

Siin on sammud autentitud sessiooni loomiseks.

**Samm 1**. Hangi ligipääsu token testkeskkonnas või meie tootmiskeskkonnas:

Tõenäoliselt kasutad oma programmeerimiskeele jaoks võimsat OAuth2 teeki, mis teeb kogu raske töö sinu eest.
Meie näited näitavad kõige lihtsamat kasutust käsurealt `curl` abil.

### Testkeskkond

```bash

curl -X POST https://staging-iam.wink.travel/oauth2/token \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=client_credentials" \
  -d "client_id=<YOUR_CLIENT_ID>" \
  -d "client_secret=<YOUR_SECRET_KEY>"

```

### Tootmiskeskkond

```bash

curl -X POST https://iam.wink.travel/oauth2/token \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=client_credentials" \
  -d "client_id=<YOUR_CLIENT_ID>" \
  -d "client_secret=<YOUR_SECRET_KEY>"

```

See tagastab ligipääsu tokeni koos muu OAuth2 vastuse infoga:

```json
{
  "access_token": "abc123"
}
```

Kui teed kõne mõnele meie lõpp-punktile, lisa päisesse järgmised:

- `Wink-Version` = `2.0` Viimane - *Vaata meie API dokumentatsioonist teisi saadaolevaid versioone.*
- `Authentication` = `Bearer: ${access_token}` *Sisesta oma ligipääsu token.*

:::tip
Saad testida kõiki meie API-sid ilma ühtegi koodirida kirjutamata enne alustamist.

Hangi 'Client ID' ja 'Secret Key' mõnest oma rakendusest ning mine mõnele meie [API-le](/et/developers/apis).
:::
