Kui olete integreerija, nagu Wink, ja soovite kaardistada oma välised saajad TripPay kasutamiseks, saate seda teha kahel viisil:

- Käsitsi kaardistamine
- Kasutades [Payment API-d](/et/developers/apis/#payment-api) (*Vaata `Account Mappings`*)

:::tip
Kui te ei tegele ainult väheste samade kinnistutega, soovitame selle sammu automatiseerimiseks kasutada API-d.
:::

Allpool juhendame teid läbi välise saaja käsitsi kaardistamise. *See kasutab API-d taustal.*

<Steps>
1. Logige sisse [TripPay](https://pay.wink.travel).
2. Valige oma integreerija konto.
3. Klõpsake põhimenüüst `Settings > Mappings`.
4. Klõpsake nuppu `Create mapping`. Näete kaardistamisvormi.
5. **Kohalik konto** Hakake tippima olemasoleva TripPay konto nime, mida soovite kaardistada.
6. **Väline identifikaator** Kleepige selle üksuse väline ID, nagu see eksisteerib kaugemas süsteemis.
7. Klõpsake `Save`, et jätkata. Teid suunatakse tagasi oma kaardistuste nimekirja.
</Steps>

Kui teie kaardistused on paigas, teab TripPay nüüd, milliseid kontosid broneeringu kasusaajatena krediteerida, kui viitate neile oma väliste identifikaatorite kaudu.

## API

Arendajad, kes soovivad hallata `Payments`, minge [Developers > API > Payments](/et/developers/apis/#payment-api).
