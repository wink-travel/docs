## Oversigt
Den hostede Wink MCP-server er tilgængelig på:

```
https://docs.mcp.wink.travel/mcp
```

Peg kompatible AI-klienter (VS Code, Cursor, andre der implementerer MCP HTTP-transport) mod dette endepunkt for at lade dem:
- Gennemse produkt- og platformdokumentation (`wink://docs/...`)
- Gennemse live OpenAPI-specifikationer og operation-dokumenter (`wink://api/...`)
- Udføre letvægts semantisk / søgeordsbaseret søgning i dokumenter og live API’er

## Transport
Brug det primære streambare HTTP-endepunkt:

```
https://docs.mcp.wink.travel/mcp
```

Hvis din klient kun understøtter SSE, brug:

```
https://docs.mcp.wink.travel/sse
```

### Claude Code
Officielle dokumenter: [Connect Claude Code to tools via MCP](https://code.claude.com/docs/en/mcp)

Anthropic anbefaler den fjernede HTTP-transport til cloud-hostede MCP-servere:
```bash
claude mcp add --transport http wink-docs https://docs.mcp.wink.travel/mcp
claude mcp list
```

Inde i Claude Code, kør `/mcp` for at inspicere serverstatus. Hvis du vil dele serveren med hele repoet, tilføj `--scope project` så Claude Code skriver konfigurationen til `.mcp.json`.

### Codex
Officielle dokumenter: [Docs MCP quickstart for Codex](https://developers.openai.com/resources/docs-mcp)

OpenAI dokumenterer MCP-opsætning gennem Codex CLI og bemærker, at konfigurationen deles med IDE-udvidelsen:
```bash
codex mcp add wink-docs --url https://docs.mcp.wink.travel/mcp
codex mcp list
```

### Cursor
Cursor MCP-dokumenter: https://cursor.com/docs/context/mcp
Opsætning: Tilføj fjernserver via MCP-indstillinger.
```json
{
  "mcpServers": {
    "wink-docs": {
      "url": "https://docs.mcp.wink.travel/mcp",
    }
  }
}
```

### VS Code
Officielle dokumenter: https://code.visualstudio.com/docs/copilot/customization/mcp-servers
Minimal fjernkonfiguration (bruger- eller arbejdsområde `mcp.json`):
```json
{
  "servers": {
    "wink-docs": {
      "type": "http",
      "url": "https://docs.mcp.wink.travel/mcp"
    }
  }
}
```

Tips:
- Brug Command Palette: “MCP: List Servers” for at bekræfte forbindelse og gennemse værktøjer.
- Værktøjsvælgeren lader dig aktivere/deaktivere `api_search` og `docs_search`.
- Tilføj kontekst: Chat-visning → Tilføj kontekst → MCP Resources for at vedhæfte dokumentsider.

## Kerne Resource URIs

| URI | Beskrivelse |
|-----|-------------|
| `wink://docs` | Dokumentationsindeks (grupperet efter kategori) |
| `wink://docs/{path}` | Specifik dokumentationsartikel (Markdown / MDX) |
| `wink://api` | Live OpenAPI-katalog opdaget fra Winks swagger-konfiguration |
| `wink://api/{apiName}` | Kompakt API-oversigt med autentificering, tags, headers og eksempeloperationer |
| `wink://api/{apiName}/operations` | Fuld operationsliste grupperet efter tag |
| `wink://api/{apiName}/operation/{key}` | Detaljer om individuel operation (parametre, anmodningskrop, svar) |
| `wink://api/{apiName}/schema/{schemaName}` | Løst komponent-skema med indlejrede felter, typer og eksempler |
| `wink://api/{apiName}/operation/{key}/request` | Anmodningsfokuseret visning med nødvendige input og minimal eksempelpayload |
| `wink://api/{apiName}/operation/{key}/response/{status}` | Svarfokuseret visning for en specifik statuskode |
| `wink://api/{apiName}/operation/{key}/recipe` | Opgaveorienteret flow for korrekt kald af den operation |

## Søgeværktøjer (Hurtig reference)
| Værktøj | Formål | Typiske forespørgselseksempler |
|---------|--------|-------------------------------|
| `api_search` | Find live API eller operation til integration | `create affiliate`, `payment receipt`, `lookup inventory`, `invite manager` |
| `docs_search` | Find konceptuelle dokumenter / vejledninger | `authentication`, `webhooks`, `wordPress` |

Efterlad `query` tom for at modtage forslag. Hvert resultat inkluderer MCP URI, som du kan åbne.
For operationstræffere returnerer `api_search` også tilknyttede anmodnings-, svar-, skema- og opskrifts-URIs, så agenter kan hoppe direkte til den mest nyttige underressource.

## Typisk brugsmønster
1. Kør `api_search` med din hensigt (f.eks. *"create affiliate"* eller *"payment receipt"*).
2. Åbn den returnerede `wink://api/...` URI, eller følg dens tilknyttede `request_uri`, `response_uris`, `request_schema_uris`, `response_schema_uris` og `recipe_uri` felter.
3. Brug `docs_search` til understøttende koncepter (autentificering, webhooks, workflows).

## Noter
- Hver returneret URI kan åbnes af agenten for fuldt indhold.
- `wink://api` træet er live og følger det aktuelle upstream OpenAPI-katalog i stedet for et genereret repo-indeks.
- Forslag vises automatisk, når forespørgsler er tomme.
