AI-færdigheder er bærbare instruktionspakker, der lærer kodeagenter, hvordan man arbejder med Wink. I stedet for at stole på træningsdata eller forældet dokumentation, instruerer hver færdighed agenten i at bruge [Wink MCP-serveren](/da/developers/mcp) som den levende sandhedskilde for API-kontrakter og platformdokumentation.

Færdigheder fungerer med enhver agentklient, der understøtter dem — Claude Code, Codex, Cursor og andre.

## Tilgængelige færdigheder

### wink-api-builder

Implementer, gennemgå eller fejlret Wink API-integrationer. Færdigheden guider agenten gennem en struktureret arbejdsgang:

1. Søg efter den rette API-operation via `api_search`
2. Åbn de levende anmodnings-/svar-kontrakter fra MCP-serveren
3. Hent autentificeringsopsætning fra dokumentationen
4. Skriv kode mod den verificerede kontrakt
5. Marker eventuelle uoverensstemmelser mellem eksisterende kode og den aktuelle API

Nyttigt til:
- At bygge nye klientmetoder til enhver Wink API
- Fejlretning af kontrakt-uoverensstemmelser (f.eks. en anmodning, der returnerer HTTP 400)
- Gennemgang af eksisterende integrationer mod den levende specifikation

## Installation

Kopiér færdighedsmappen ind i din agents færdighedsmappe.

:::tip
Wink MCP-serveren skal være tilsluttet i samme miljø. Se [MCP Server](/da/developers/mcp) for installationsvejledning.
:::

**Claude Code**

```bash
# repo-lokal
cp -r skills/wink-api-builder .claude/skills/

# eller bruger-niveau
cp -r skills/wink-api-builder ~/.claude/skills/
```

Kald med:
```
/wink-api-builder add a TypeScript client for the Wink affiliate create flow
```

**Codex**

```bash
# repo-lokal
cp -r skills/wink-api-builder .agents/skills/

# eller bruger-niveau
cp -r skills/wink-api-builder $CODEX_HOME/skills/
```

Kald med:
```
Use $wink-api-builder to add a TypeScript client for the Wink affiliate create flow.
```

**Andre agenter**

Placer færdighedsmappen, hvor din agent henter færdigheder fra. Indgangspunktet er `SKILL.md`.
