[Webkomponenter](https://developer.mozilla.org/en-US/docs/Web/API/Web_components) er en standard, der lader dig integrere hele Javascript-funktioner på dit website uden at skulle kunne kode.
Takket være denne smarte teknologi kan du nemt integrere Wink rejseinventar. Denne artikel guider dig gennem vores samling af webkomponenter og viser, hvordan du bruger dem.

## Integrer Wink

Der er tre ting, du skal inkludere på enhver side, hvor du vil vise en af vores webkomponenter.

1. Inkluder vores stylesheet.
2. Inkluder vores Javascript.
3. Inkluder vores applikationsloader.

### Stylesheet

Inkluder vores CSS-stilarter i dokumentets head.

```html {3}
<html>
  <head>
    <link rel="stylesheet" href="https://elements.wink.travel/styles.css"></link>
  </head>
</html>
```

### Javascript

Inkluder vores Javascript nederst i dokumentet. (*Vi anbefaler lige over slutningen af body-tagget*).

```html {3}
<html>
  <body>
    <script src="https://elements.wink.travel/elements.js" type="module" defer></script>
  </body>
</html>
```

Dit site er nu klar til at vise vores webkomponenter.

## Komponenter

Læs om vores komponentbibliotek nedenfor.

### Applikationsloader

Loaderen er ansvarlig for at holde styr på tilstand og styre interaktionen mellem vores komponenter. (*Inkluder den under vores Javascript*).

```html {3-6}
<html>
  <body>
    <wink-app-loader
      client-id="DIN CLIENT ID HER"
      configuration-id="DIN TILPASNING ID HER"
    ></wink-app-loader>
  </body>
</html>
```

### Indholdsloader

Indholdsloaderen er kernen i vores webkomponentbibliotek. Den er ansvarlig for at vise dit inventar (kort, grids, kort).

Tilgængelige attributter:

- layout
- id
- sort

*Attributten `sort` er kun tilgængelig, når layout er `RANKED`, og du ikke ønsker at bruge et eksisterende rangeret grid. I disse tilfælde skal `id` være tom.*

Tilgængelige layout-typer:

- `AD_BANNER`
- `MAP`
- `HOTEL`
- `GUEST_ROOM`
- `MEETING_ROOM`
- `SPA`
- `RESTAURANT`
- `ACTIVITY`
- `ATTRACTION`
- `PLACE`
- `ADD_ON`
- `LIST`
- `SEARCH`
- `RANKED`

Tilgængelige sorterings-typer:

- `MEMBER`
- `PRICE_LOW_TO_HIGH`
- `PRICE_HIGH_TO_LOW`
- `PRICE`
- `POPULARITY`
- `ECO`
- `EXPERIENCE`
- `PERK`
- `LOYALTY`
- `PACKAGE`

```html {3-6}
<html>
  <body>
    <wink-content-loader
       layout="GUEST_ROOM"
       id="2de7ee9c-61c9-11ef-9722-42004e494300"
    ></wink-content-loader>
  </body>
</html>
```

Eksempler kan findes for [kort](/studio/cards), [grids](/studio/grids) og [kort](/studio/maps).

## Komponenter

### Betaling

Betalingskomponenten lader dig informere TripPay om, at en rejsende ønsker at købe noget, samt regler og priser for disse varer.

Der er én påkrævet attribut til widget’en:

- `id` Identifikatoren for den bookingkontrakt, du ønsker at udføre.

```html {3}
<html>
  <body>
    <trip-pay id="<INSERT_BOOKING_CONTRACT_ID>"></trip-pay>
  </body>
</html>
```

:::note
For at lære, hvordan du genererer bookingkontraktens `id`, læs guiden: [Integrering med TripPay](/guides/integrating-with-trip-pay).
:::

Widget’en forbereder kontrakten til udførelse og viser betalingsdetaljer *(Figur 1)* til brugeren for at færdiggøre bookingen.

## Fejl

Hvis du møder en fejlmeddelelse, når du integrerer en af vores webkomponenter, enten på WinkLinks eller på dit eget website, kan der være flere ting, der er gået galt:

### Tilgængelighed

Inventar kan være udgået. Gå til [Wink Studio](/studio/cards) og tjek, om status for "kortet" er tilgængeligt.
Hvis det ikke er, vises det som rødt. I så fald kan du vente og håbe, at leverandøren gør det tilgængeligt igen, eller fjerne det fra din liste.

:::note
`Tilgængelighed` betyder ikke den daglige tilgængelighed af inventaret til booking. Det betyder, at inventaret er helt utilgængeligt for dig og/eller alle andre.
:::

### Fjernet

Inventar kan være fjernet. Gå til [Wink Studio](/studio/cards) og tjek, om status for "kortet" er tilgængeligt.
Hvis det ikke er, vises det som rødt. I så fald kan du vente og håbe, at leverandøren gør det tilgængeligt igen, eller fjerne det fra din liste.

### Tilpasning

Du kan ved et uheld have fjernet den tilpasning, du har defineret til dit inventar. Sørg for, at tilpasningen stadig er tilgængelig, og sæt en ny, hvis en mangler.

### Applikation

Dette er kun for webudviklere. Hvis du ved et uheld har fjernet [Applikationen](/developer/applications), så klient-ID’et ikke længere er tilgængeligt, skal du oprette en ny applikation og bruge det nye klient-ID til at integrere vores Webkomponenter.
