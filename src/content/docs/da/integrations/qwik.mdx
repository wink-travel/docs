Qwiks resumability-model betyder, at JavaScript eksekveres doven — kun når brugeren interagerer. Brug `useVisibleTask$` til at køre kode, der skal eksekveres i browseren, når komponenten bliver synlig.

## Installér

```bash
npm install @wink/elements
```

## Indlæs med useVisibleTask$

`useVisibleTask$` er Qwiks lifecycle hook til browser-only, hurtigt eksekveret kode. Placer den i dit root layout:

```tsx
// src/routes/layout.tsx
import { component$, useVisibleTask$ } from '@builder.io/qwik';
import { load } from '@wink/elements';

export default component$(() => {
  useVisibleTask$(() => {
    load({ clientId: import.meta.env.PUBLIC_WINK_CLIENT_ID });
  });

  return (
    <main>
      <Slot />
    </main>
  );
});
```

<Aside type="tip" title="useVisibleTask$ strategi">
Som standard kører `useVisibleTask$` når komponenten er synlig i viewporten. For et root layout, der altid er synligt, sker dette straks ved sideindlæsning — svarende til `onMount` i andre frameworks.

Send `{ strategy: 'document-ready' }` hvis du vil have scriptet til at indlæses så tidligt som muligt:
```ts
useVisibleTask$({ strategy: 'document-ready' }, () => {
  load({ clientId: import.meta.env.PUBLIC_WINK_CLIENT_ID });
});
```
</Aside>

## Brug på en side

```tsx
// src/routes/hotels/index.tsx
import { component$ } from '@builder.io/qwik';

export default component$(() => {
  return (
    <main>
      <wink-content-loader layout="HOTEL" id="YOUR_LAYOUT_ID" />
      <wink-lookup />
    </main>
  );
});
```

## TypeScript — deklarér JSX intrinsics

Qwik bruger sit eget JSX namespace. Udvid det i en deklarationsfil:

```ts
// src/wink-elements.d.ts
import type {
  WinkContentLoaderAttributes,
  WinkLookupAttributes,
  WinkSearchButtonAttributes,
  WinkAccountButtonAttributes,
  WinkItineraryButtonAttributes,
  WinkShoppingCartButtonAttributes,
  WinkAppLoaderAttributes,
} from '@wink/elements';

declare module '@builder.io/qwik' {
  namespace JSX {
    interface IntrinsicElements {
      'wink-content-loader': WinkContentLoaderAttributes;
      'wink-lookup': WinkLookupAttributes;
      'wink-search-button': WinkSearchButtonAttributes;
      'wink-account-button': WinkAccountButtonAttributes;
      'wink-itinerary-button': WinkItineraryButtonAttributes;
      'wink-shopping-cart-button': WinkShoppingCartButtonAttributes;
      'wink-app-loader': WinkAppLoaderAttributes;
    }
  }
}
```

## Miljøvariabler

Qwik (Vite-baseret) eksponerer variabler med præfikset `PUBLIC_` til browseren:

```
# .env
PUBLIC_WINK_CLIENT_ID=your-client-id
```

<Aside type="caution" title="Server-side rendering">
Qwik renderer til HTML på serveren. `<wink-*>` tags outputtes som statisk markup. CDN-scriptet og registreringen af custom elementer sker klient-side efter `useVisibleTask$` aktiveres. Elementerne vil kortvarigt være i en uafklaret tilstand, indtil CDN-bundtet er indlæst — dette er forventet adfærd.
</Aside>
