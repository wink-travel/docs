Nuxt 3 renderer sider på serveren som standard. CDN-scripts og brugerdefinerede elementer skal kun indlæses på klientsiden. Et Nuxt klient-plugin er det reneste sted at gøre dette.

<LinkCard
  title="@wink/elements npm package"
  href="/integrations/npm"
  description="Installer og indlæs CDN-bundtet med TypeScript-typer."
/>

## Installér

```bash
npm install @wink/elements
```

## Opret et klient-plugin

`.client.ts`-endelsen fortæller Nuxt at køre dette plugin kun i browseren:

```ts
// plugins/wink.client.ts
import { load } from '@wink/elements';

export default defineNuxtPlugin(() => {
  load({ clientId: useRuntimeConfig().public.winkClientId });
});
```

## Konfigurer runtime-variablen

```ts
// nuxt.config.ts
export default defineNuxtConfig({
  runtimeConfig: {
    public: {
      winkClientId: process.env.NUXT_PUBLIC_WINK_CLIENT_ID,
    },
  },
});
```

Tilføj til `.env`:

```
NUXT_PUBLIC_WINK_CLIENT_ID=your-client-id
```

## Konfigurer Vue til brugerdefinerede elementer

Tilføj `isCustomElement` compiler-indstillingen, så Vue ikke advarer om ukendte `<wink-*>` tags:

```ts
// nuxt.config.ts
export default defineNuxtConfig({
  vue: {
    compilerOptions: {
      isCustomElement: (tag) => tag.startsWith('wink-'),
    },
  },
  runtimeConfig: {
    public: {
      winkClientId: process.env.NUXT_PUBLIC_WINK_CLIENT_ID,
    },
  },
});
```

## Brug i templates

Indpak Wink-komponenter i `<ClientOnly>` for at forhindre SSR hydration mismatch:

```vue
<template>
  <main>
    <ClientOnly>
      <wink-content-loader layout="HOTEL" :id="layoutId" />
    </ClientOnly>
  </main>
</template>

<script setup lang="ts">
const layoutId = 'YOUR_LAYOUT_ID';
</script>
```

<Aside type="tip">
`<ClientOnly>` renderer en pladsholder på serveren og monterer det rigtige element på klienten — dette er det anbefalede mønster for alle browser-only tredjeparts-embeds i Nuxt.
</Aside>
