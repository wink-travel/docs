يُظهر هذا المقال كيفية الإعداد السريع كمطور والبدء في اختبار واجهات برمجة التطبيقات الخاصة بنا على Wink.

:::note
يجب أن تتم الاختبارات دائمًا في [بيئة الاختبار](/ar/getting-started/environments). هناك بيانات أقل بكثير ولكن يمكنك الاختبار بقدر ما تريد دون التأثير على بيئة الإنتاج.

بمجرد أن تكون جاهزًا للانطلاق، سنقوم بتوثيق حسابك وتطبيقك ويمكنك الإطلاق في بيئة الإنتاج الخاصة بنا.
:::

اتبع هذه الخطوات:

<Steps>
1. [سجل](/ar/my-account/register) حساب المستخدم الخاص بك على Wink.
2. [أنشئ حساب تابع](/ar/my-account/create-account). ستحتاج إلى هذا لربط تطبيقك به. راجع [ما هو Studio](/ar/studio/what-is-studio) لمزيد من السياق.
3. [أنشئ تطبيقًا](/ar/my-account/applications) واربطه بحساب التابع الذي أنشأته في الخطوة 2. بالنسبة لتكاملات API من خادم إلى خادم، اختر **MACHINE_2_MACHINE** كنوع العميل.
4. بمجرد إنشاء `Application` الخاص بك، انسخ واحتفظ بمعرف العميل والمفتاح السري على الفور — يتم عرض المفتاح السري مرة واحدة فقط ولا يمكن استرجاعه مرة أخرى.
</Steps>

## CDN

في Wink، نستخدم Cloudinary لجميع أصول الصور والفيديو الخاصة بنا. عند وصولك إلى مخزون السفر لدينا، سيكون لديك وصول كامل إلى جميع ميزات Cloudinary المتقدمة.

اسم `cloud_name` الخاص بنا هو `traveliko`.

### Cloudinary

- [البدء](https://cloudinary.com/documentation/dev_kickstart)
- [مجموعات تطوير البرمجيات (SDKs)](https://cloudinary.com/documentation/cloudinary_sdks)
- [Cloudinary على GitHub](https://github.com/cloudinary)
-	[دروس فيديو](https://cloudinary.com/documentation/cloudinary_sdks_tutorials)

### مثال

لكل نوع من أنواع مخزون السفر التي ندعمها، سيكون هناك مصفوفة JSON متداخلة تسمى `multimedias`. يبدو JSON لمدخل الوسائط المتعددة كما يلي:

```json {3}
{
  "multimediaIdentifier": "multimedia-1",
  "identifier": "partners/radisson_blu_logo",
  "type": "IMAGE",
  "width": "1024",
  "height": "768",
  "category": "1",
  "descriptions": [
    {
      "name": "Lobby",
      "description": "Picture of the lobby",
      "language": "en"
    }
  ],
  "lifestyleType": "LIFESTYLE_BUSINESS",
  "attribution": [
    {
      "url": "https://www.hilton.com",
      "name": "Hilton"
    }
  ]
}
```

السطر المميز أعلاه يُظهر معرف Cloudinary الفريد `partners/radisson_blu_logo`. هذا كل ما تحتاجه لاسترجاع الصورة بأي تنسيق أو جودة أو حجم تريده.

## الاختبار

توجه إلى قسم [APIs](/ar/developers/apis) واختر واجهة برمجة التطبيقات التي تريد العمل معها.

### وثائق API

تم إعداد موقع التوثيق لدينا بطريقة تتيح لك تشغيل أوامر API من داخل الموقع.

في هذا المثال، تريد استخدام نداء API `Search Lookups (OAuth2)`.

:::tip[نصيحة احترافية]
لسهولة الاستخدام، نوصي بأن يستخدم المدمجون نقاط النهاية التي نحددها كـ `OAuth2`. تلك النقاط تعرف أنها تتوقع منادي API وتعرف بالفعل الحساب الذي تستخدمه لذلك لا تحتاج إلى تحديد `companyIdentifier` في عناوين URL الخاصة بك.
:::

<Steps>
1. استخدم [Lookup API](https://developer.wink.travel/lookup) لاسترجاع الوجهات أو الفنادق.
2. في شريط التنقل الأيسر، انقر على `Authentication`.
3. أدخل معرف العميل والمفتاح السري وانقر على زر `Get TOKEN`.
4. إذا كانت بيانات الاعتماد صحيحة، سترى أن مفتاح API الخاص بك قد تم تطبيقه.
5. يمكنك الآن اختبار نقطة نهاية البحث.
</Steps>

### Postman
يمكنك القيام بنفس الشيء باستخدام Postman.

<Steps>
1. قم بتنزيل [Postman](https://www.postman.com/downloads).
2. انقر على `Download OpenAPI spec` من أعلى صفحة [Lookup API](https://developer.wink.travel/lookup).
3. استوردها إلى Postman.
4. إذا استوردتها كمجموعة جديدة تسمى `Wink API`، انقر على النقاط الأفقية الثلاث بجانب اسم المجموعة ثم انقر على `Edit`.
5. في علامة تبويب `Authorization`، اضبط `Auth type` على OAuth2.
6. اضبط الحقل `Add auth data to` على `Request headers`.
7. اضبط الحقل `Header prefix` على `Bearer`.
8. اضبط الحقل `Token Name` على أي اسم تريد تسميته لرمزك المحفوظ في Postman. مثلاً Wink Staging.
9. اضبط الحقل `Grant type` على `Client Credentials`.
10. اضبط الحقل `Access token URL` على https://staging-iam.wink.travel/oauth2/token.
11. اضبط الحقل `Client ID` على معرف العميل الذي ساعد تطبيقك في إنشائه.
12. اضبط الحقل `Client Secret` على المفتاح السري الذي ساعد تطبيقك في إنشائه.
13. اضبط الحقل `Scope` على `inventory.read inventory.write inventory.remove`. هذه هي كل الصلاحيات التي ستحتاجها لـ Wink.
14. اضبط الحقل `Client Authentication` على `Send as Basic Auth header`.
15. انقر على زر `Get New Access Token` لاسترجاع الرمز وحفظه.
16. لجميع نداءات API تحت مجموعة Wink API، تأكد من أن علامة التوثيق تقول `Inherit auth from parent` بحيث يتضمن رمزك مع كل نداء.
</Steps>

:::tip[نصيحة احترافية]
يوصى بتحويل نطاق الخادم، معرف العميل، والمفتاح السري إلى متغيرات بيئية. بهذه الطريقة يمكنك التبديل بسهولة بين بيئة الاختبار والإنتاج وإعادة استخدام جميع العمليات التي أنشأتها.
:::
