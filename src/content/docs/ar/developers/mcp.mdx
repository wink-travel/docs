## نظرة عامة
خادم MCP المستضاف من Wink متاح على:

```
https://docs.mcp.wink.travel/mcp
```

وجه عملاء الذكاء الاصطناعي المتوافقين (VS Code، Cursor، وغيرهم الذين ينفذون نقل MCP عبر HTTP) إلى هذه النقطة النهائية للسماح لهم بـ:
- تصفح وثائق المنتج والمنصة (`wink://docs/...`)
- تصفح مواصفات OpenAPI الحية ووثائق العمليات (`wink://api/...`)
- إجراء بحث دلالي / بالكلمات المفتاحية خفيف الوزن عبر الوثائق وواجهات برمجة التطبيقات الحية

## النقل
استخدم نقطة النهاية الأساسية القابلة للبث عبر HTTP:

```
https://docs.mcp.wink.travel/mcp
```

إذا كان عميلك يدعم SSE فقط، استخدم:

```
https://docs.mcp.wink.travel/sse
```

### Claude Code
الوثائق الرسمية: [ربط Claude Code بالأدوات عبر MCP](https://code.claude.com/docs/en/mcp)

توصي Anthropic باستخدام نقل HTTP عن بُعد لخوادم MCP المستضافة سحابياً:
```bash
claude mcp add --transport http wink-docs https://docs.mcp.wink.travel/mcp
claude mcp list
```

داخل Claude Code، شغّل `/mcp` لفحص حالة الخادم. إذا أردت مشاركة الخادم مع كامل المستودع، أضف `--scope project` ليكتب Claude Code التكوين في `.mcp.json`.

### Codex
الوثائق الرسمية: [بدء سريع لوثائق MCP لـ Codex](https://developers.openai.com/resources/docs-mcp)

توثق OpenAI إعداد MCP عبر CLI الخاص بـ Codex، وتذكر أن التكوين مشترك مع امتداد IDE:
```bash
codex mcp add wink-docs --url https://docs.mcp.wink.travel/mcp
codex mcp list
```

### Cursor
وثائق Cursor MCP: https://cursor.com/docs/context/mcp
الإعداد: أضف الخادم البعيد عبر إعدادات MCP.
```json
{
  "mcpServers": {
    "wink-docs": {
      "url": "https://docs.mcp.wink.travel/mcp",
    }
  }
}
```

### VS Code
الوثائق الرسمية: https://code.visualstudio.com/docs/copilot/customization/mcp-servers
تكوين بعيد بسيط (ملف `mcp.json` للمستخدم أو مساحة العمل):
```json
{
  "servers": {
    "wink-docs": {
      "type": "http",
      "url": "https://docs.mcp.wink.travel/mcp"
    }
  }
}
```

نصائح:
- استخدم لوحة الأوامر: “MCP: List Servers” لتأكيد الاتصال وتصفح الأدوات.
- يتيح لك محدد الأدوات تمكين/تعطيل `api_search` و `docs_search`.
- أضف سياقًا: عرض الدردشة → إضافة سياق → موارد MCP لإرفاق صفحات الوثائق.

## عناوين URI الأساسية للموارد

| URI | الوصف |
|-----|-------------|
| `wink://docs` | فهرس الوثائق (مجمّع حسب الفئة) |
| `wink://docs/{path}` | مقالة وثائق محددة (Markdown / MDX) |
| `wink://api` | كتالوج OpenAPI الحي المكتشف من تكوين swagger الخاص بـ Wink |
| `wink://api/{apiName}` | ملخص API مضغوط مع المصادقة، العلامات، الرؤوس، وعينات العمليات |
| `wink://api/{apiName}/operations` | قائمة العمليات الكاملة مجمعة حسب العلامة |
| `wink://api/{apiName}/operation/{key}` | تفاصيل عملية فردية (المعلمات، جسم الطلب، الاستجابات) |
| `wink://api/{apiName}/schema/{schemaName}` | مخطط مكون محلل مع حقول متداخلة وأنواع وأمثلة |
| `wink://api/{apiName}/operation/{key}/request` | عرض مركز على الطلب مع المدخلات المطلوبة وحمولة مثال بسيطة |
| `wink://api/{apiName}/operation/{key}/response/{status}` | عرض مركز على الاستجابة لحالة رمز محددة |
| `wink://api/{apiName}/operation/{key}/recipe` | تدفق موجه بالمهام لاستدعاء تلك العملية بشكل صحيح |

## أدوات البحث (مرجع سريع)
| الأداة | الغرض | أمثلة استعلام نموذجية |
|------|---------|-----------------------|
| `api_search` | العثور على API الحي أو العملية للاندماج معها | `create affiliate`، `payment receipt`، `lookup inventory`، `invite manager` |
| `docs_search` | العثور على الوثائق المفاهيمية / الأدلة | `authentication`، `webhooks`، `wordPress` |

اترك `query` فارغًا لتلقي اقتراحات. كل نتيجة تتضمن URI MCP يمكنك فتحه.
بالنسبة لنتائج العمليات، يعيد `api_search` أيضًا روابط URI للطلب، الاستجابة، المخطط، والوصفة حتى يتمكن الوكلاء من القفز مباشرة إلى المورد الفرعي الأكثر فائدة.

## نمط الاستخدام النموذجي
1. شغّل `api_search` مع نيتك (مثلاً *"create affiliate"* أو *"payment receipt"*).
2. افتح URI `wink://api/...` المعاد، أو اتبع حقول `request_uri`، `response_uris`، `request_schema_uris`، `response_schema_uris`، و `recipe_uri` المرتبطة به.
3. استخدم `docs_search` للمفاهيم الداعمة (المصادقة، webhooks، سير العمل).

## مهارات الذكاء الاصطناعي

تستخدم مهارات الوكلاء المسبقة البناء هذا الخادم MCP كمصدر الحقيقة الخاص بها. تمنح المهارات وكيل البرمجة الخاص بك سير عمل منظم للبناء ضد واجهات برمجة التطبيقات الخاصة بـ Wink — بدلاً من التخمين في نقاط النهاية، يقرأ الوكيل العقود الحية مباشرة من خادم MCP.

راجع [مهارات الذكاء الاصطناعي](/ar/developers/ai-skills) للمهارات المتاحة وتعليمات التثبيت.

## ملاحظات
- يمكن للوكلاء فتح كل URI معاد للحصول على المحتوى الكامل.
- شجرة `wink://api` حية وتتبع كتالوج OpenAPI الحالي بدلاً من فهرس مستودع مولد.
- تظهر الاقتراحات تلقائيًا عند ترك الاستعلامات فارغة.
