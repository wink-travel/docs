مهارات الذكاء الاصطناعي هي حزم تعليمات قابلة للنقل تعلم وكلاء البرمجة كيفية العمل مع Wink. بدلاً من الاعتماد على بيانات التدريب أو الوثائق القديمة، توجه كل مهارة الوكيل لاستخدام [خادم Wink MCP](/ar/developers/mcp) كمصدر حي للحقيقة لعقود API ووثائق المنصة.

تعمل المهارات مع أي عميل وكيل يدعمها — مثل Claude Code و Codex و Cursor وغيرها.

## المهارات المتاحة

### wink-api-builder

تنفيذ أو مراجعة أو تصحيح تكاملات Wink API. ترشد المهارة الوكيل خلال سير عمل منظم:

1. البحث عن العملية الصحيحة عبر `api_search`
2. فتح عقود الطلب/الاستجابة الحية من خادم MCP
3. سحب إعداد المصادقة من الوثائق
4. كتابة الكود بناءً على العقد الموثق
5. الإشارة إلى أي تعارضات بين الكود الحالي وAPI الحالي

مفيد لـ:
- بناء طرق عملاء جديدة لأي Wink API
- تصحيح تعارضات العقود (مثل طلب يعيد HTTP 400)
- مراجعة التكاملات الحالية مقابل المواصفات الحية

## التثبيت

انسخ مجلد المهارة إلى دليل مهارات الوكيل الخاص بك.

:::tip
يجب توصيل خادم Wink MCP في نفس البيئة. راجع [خادم MCP](/ar/developers/mcp) للحصول على تعليمات الإعداد.
:::

**Claude Code**

```bash
# repo-local
cp -r skills/wink-api-builder .claude/skills/

# or user-level
cp -r skills/wink-api-builder ~/.claude/skills/
```

استدعِها بـ:
```
/wink-api-builder add a TypeScript client for the Wink affiliate create flow
```

**Codex**

```bash
# repo-local
cp -r skills/wink-api-builder .agents/skills/

# or user-level
cp -r skills/wink-api-builder $CODEX_HOME/skills/
```

استدعِها بـ:
```
Use $wink-api-builder to add a TypeScript client for the Wink affiliate create flow.
```

**وكلاء آخرون**

ضع مجلد المهارة في أي مكان يحل منه وكيلك المهارات. نقطة الدخول هي `SKILL.md`.
