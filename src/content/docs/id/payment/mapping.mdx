Jika Anda adalah integrator, seperti Wink, dan ingin memetakan penerima pembayaran eksternal Anda untuk digunakan dengan TripPay, Anda dapat melakukannya dengan dua cara:

- Pemetaan manual
- Menggunakan [Payment API](/id/developers/apis/#payment-api) (*Lihat `Account Mappings`*)

:::tip
Kecuali Anda hanya menangani beberapa properti yang sama, kami menyarankan Anda menggunakan API untuk mengotomatisasi langkah ini.
:::

Di bawah ini, kami memandu Anda melalui cara manual memetakan penerima pembayaran eksternal. *Ini menggunakan API secara internal.*

<Steps>
1. Masuk ke [TripPay](https://pay.wink.travel).
2. Pilih akun integrator Anda.
3. Klik `Settings > Mappings` dari bilah navigasi utama.
4. Klik tombol `Create mapping`. Anda akan melihat formulir pemetaan.
5. **Local account** Mulailah mengetik nama TripPay yang sudah ada yang ingin Anda petakan.
6. **External identifier** Tempelkan ID eksternal Anda untuk entitas ini seperti yang ada di sistem jarak jauh.
7. Klik `Save` untuk melanjutkan. Anda akan diarahkan kembali ke daftar pemetaan Anda.
</Steps>

Dengan pemetaan Anda yang sudah dibuat, TripPay sekarang tahu akun mana yang harus dikreditkan sebagai penerima manfaat dari pemesanan ketika Anda merujuknya menggunakan pengenal eksternal Anda.

## API

Pengembang yang ingin mengelola `Payments`, kunjungi [Developers > API > Payments](/id/developers/apis/#payment-api).
