Jika Anda seorang integrator, seperti Wink, dan ingin memetakan penerima pembayaran eksternal Anda untuk digunakan dengan TripPay, Anda dapat melakukannya dengan salah satu dari dua cara:

* Pemetaan manual
* Menggunakan [API Pembayaran](/developers/apis/#payment-api) (*Melihat `Account Mappings`*)

:::tip
Kecuali jika Anda hanya berurusan dengan beberapa properti yang sama, kami sarankan Anda menggunakan API untuk mengotomatiskan langkah ini.
:::

Di bawah ini, kami memandu Anda melalui cara manual untuk memetakan penerima pembayaran eksternal.*Ia menggunakan API di balik layar.*

<Steps>
  1. Masuk ke dalam [Pembayaran TripPay](https://pay.wink.travel).
  2. Pilih akun integrator Anda.
  3. Klik `Settings > Mappings` dari bilah navigasi utama.
  4. Klik tombol `Create mapping` tombol. Anda akan melihat formulir pemetaan.
  5. **Akun lokal** Mulailah mengetik nama TripPay yang ada yang ingin Anda petakan.
  6. **Pengenal eksternal** Tempelkan ID eksternal Anda untuk entitas ini seperti yang ada di sistem jarak jauh.
  7. Klik `Save` untuk melanjutkan. Anda akan diarahkan kembali ke daftar pemetaan Anda.
</Steps>


Dengan pemetaan yang sudah ada, TripPay kini mengetahui akun mana yang akan dikreditkan sebagai penerima pemesanan saat Anda merujuknya menggunakan pengenal eksternal Anda.

## API

Pengembang yang ingin mengelola`Payments`, pergi ke[Pengembang > API > Pembayaran](/developers/apis/#payment-api).
