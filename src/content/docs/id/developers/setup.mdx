Artikel ini menunjukkan cara cepat untuk memulai sebagai pengembang dan mulai menguji API kami di Wink.

:::note
Pengujian harus selalu dilakukan di [lingkungan staging](/getting-started/environments) kami. Data di sana jauh lebih sedikit tetapi Anda bebas menguji sebanyak yang Anda mau tanpa mengganggu lingkungan produksi.

Setelah Anda siap untuk live, kami akan memverifikasi akun dan aplikasi Anda dan Anda dapat meluncurkan di lingkungan produksi kami.
:::

Ikuti langkah-langkah berikut:

<Steps>
1. [Daftar](/user/register) akun pengguna Anda di Wink
2. Buat [akun agen perjalanan](/travel-agent/what-is-agent) atau [akun afiliasi](/studio/what-is-studio).
3. [Buat Aplikasi](/developer/applications) di mana Anda mengaitkannya dengan akun yang Anda buat di Langkah 2.
4. Setelah Anda membuat `Application`, simpan client-id dan client secret di tempat yang aman.
</Steps>

## CDN

Di Wink, kami menggunakan Cloudinary untuk semua aset gambar dan video kami. Saat Anda mengakses inventaris perjalanan kami, Anda akan memiliki akses penuh ke semua fitur canggih Cloudinary.

`cloud_name` kami adalah `traveliko`.

### Cloudinary

- [Memulai](https://cloudinary.com/documentation/dev_kickstart)
- [SDKs](https://cloudinary.com/documentation/cloudinary_sdks)
- [Cloudinary di GitHub](https://github.com/cloudinary)
-	[Video Tutorial](https://cloudinary.com/documentation/cloudinary_sdks_tutorials)

### Contoh

Untuk setiap jenis inventaris perjalanan yang kami dukung, akan ada array JSON bersarang yang disebut `multimedias`. JSON untuk entri multimedia terlihat seperti ini:

```json {3}
{
  "multimediaIdentifier": "multimedia-1",
  "identifier": "partners/radisson_blu_logo",
  "type": "IMAGE",
  "width": "1024",
  "height": "768",
  "category": "1",
  "descriptions": [
    {
      "name": "Lobby",
      "description": "Gambar lobi",
      "language": "en"
    }
  ],
  "lifestyleType": "LIFESTYLE_BUSINESS",
  "attribution": [
    {
      "url": "https://www.hilton.com",
      "name": "Hilton"
    }
  ]
}
```

Baris yang disorot di atas menunjukkan ID unik Cloudinary `partners/radisson_blu_logo`. Itu saja yang Anda butuhkan untuk mengambil gambar dalam format, kualitas, dan ukuran apa pun yang Anda inginkan.

## Pengujian

Langsung saja ke bagian [API](/developer/apis) kami dan pilih API yang ingin Anda gunakan.

### Dokumen API

Situs dokumentasi kami disusun sedemikian rupa sehingga Anda dapat menjalankan perintah API langsung dari dalam situs.

Dalam contoh ini, Anda ingin menggunakan panggilan API `Search Lookups (OAuth2)`.

:::tip[Pro Tip]
Untuk kemudahan penggunaan, kami menyarankan integrator menggunakan endpoint yang kami tetapkan sebagai `OAuth2`. Endpoint tersebut sudah mengenali pemanggil API dan sudah mengetahui akun Anda sehingga Anda tidak perlu menyebutkan `companyIdentifier` di URL Anda.
:::

<Steps>
1. Gunakan [Lookup API](https://developer.wink.travel/lookup) untuk mengambil destinasi atau hotel.
2. Di navbar kiri, klik `Authentication`.
3. Masukkan client-id dan client secret Anda lalu klik tombol `Get TOKEN`.
4. Jika kredensial Anda valid, Anda akan melihat bahwa kunci API Anda telah diterapkan.
5. Sekarang Anda dapat menguji endpoint lookup.
</Steps>

### Postman
Anda dapat melakukan hal yang sama dengan Postman.

<Steps>
1. Unduh [Postman](https://www.postman.com/downloads).
2. Klik `Download OpenAPI spec` dari bagian atas halaman [Lookup API](https://developer.wink.travel/lookup).
3. Impor ke Postman.
4. Jika Anda mengimpornya sebagai koleksi baru bernama `Wink API`, klik elipsis horizontal (3 titik) di sebelah nama koleksi dan klik `Edit`.
5. Di tab `Authorization`, atur `Auth type` ke OAuth2.
6. Atur bidang `Add auth data to` ke `Request headers`.
7. Atur bidang `Header prefix` ke `Bearer`.
8. Atur bidang `Token Name` ke nama apa pun yang ingin Anda berikan untuk token yang disimpan di Postman. Misalnya Wink Staging.
9. Atur bidang `Grant type` ke `Client Credentials`.
10. Atur bidang `Access token URL` ke https://staging-iam.wink.travel/oauth2/token.
11. Atur bidang `Client ID` ke Client ID yang dihasilkan aplikasi Anda.
12. Atur bidang `Client Secret` ke Client Secret yang dihasilkan aplikasi Anda.
13. Atur bidang `Scope` ke `inventory.read inventory.write inventory.remove`. Itu semua cakupan yang Anda butuhkan untuk Wink.
14. Atur bidang `Client Authentication` ke `Send as Basic Auth header`.
15. Klik tombol `Get New Access Token` untuk mengambil token Anda dan simpan.
16. Untuk semua panggilan API di bawah koleksi Wink API, pastikan tab otentikasi mengatakan `Inherit auth from parent` agar token Anda disertakan di setiap panggilan.
</Steps>

:::tip[Pro Tip]
Disarankan untuk menjadikan domain server, client ID, dan client secret sebagai variabel lingkungan. Dengan begitu Anda dapat dengan mudah beralih antara lingkungan staging dan produksi serta menggunakan kembali semua operasi yang telah Anda buat.
:::
