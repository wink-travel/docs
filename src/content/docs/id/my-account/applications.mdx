Aplikasi memberi Anda akses ke semua fitur Wink sebagai pengembang. Anda dapat mengakses semua data yang sama dan bahkan memperluas fitur Wink sendiri.
Anda dapat memilih untuk menggunakan server OAuth2 Wink untuk mengautentikasi situs perjalanan Anda dengan cara yang sama seperti menggunakan Google atau Facebook.
Dengan begitu Anda dapat memperluas fitur Wink ke dalam situs web atau aplikasi Anda sendiri.

:::note
Aplikasi pertama Anda sudah dibuat saat Anda [membuat akun baru](/id/my-account/create-account). Kami menggunakan aplikasi tersebut untuk beberapa skenario pemesanan.

Anda tidak dapat menggunakan kembali aplikasi ini. Silakan buat aplikasi baru sesuai kebutuhan Anda.
:::

## Buat aplikasi

Karena Anda tidak dapat mengakses `secret key` dari aplikasi yang kami buat untuk Anda, Anda perlu membuat aplikasi Anda sendiri.

:::caution
Anda perlu membuat akun afiliasi di Wink atau akun di TripPay sebelum membuat aplikasi karena aplikasi Anda akan terikat pada akun tertentu.
:::

### Jenis klien

Saat membuat aplikasi, Anda memilih jenis klien yang menentukan cara aplikasi mengautentikasi:

- **SPA** (Single Page Application) — Untuk aplikasi frontend berbasis browser yang menggunakan alur kode otorisasi OAuth2 dengan PKCE. Ini adalah jenis yang paling umum untuk aplikasi web.
- **MACHINE_2_MACHINE** — Untuk integrasi server-ke-server yang mengautentikasi langsung menggunakan kredensial klien. Gunakan ini ketika layanan backend Anda perlu memanggil API Wink tanpa kehadiran pengguna.

Berikut adalah langkah-langkah membuat aplikasi:

<Steps>
1. Dari halaman aplikasi, klik tombol `Create new application`.
2. ***Nama*** Beri nama aplikasi Anda. *misalnya Cool App*
3. **Jenis klien** Pilih `SPA` untuk aplikasi web atau `MACHINE_2_MACHINE` untuk integrasi server.
4. **Nama entitas** Hubungkan aplikasi Anda dengan salah satu akun yang sudah ada. *misalnya Cool Account*
5. **Redirect URIs** Minimal satu redirect URI diperlukan. Ini digunakan untuk memvalidasi domain redirect setelah autentikasi berhasil. *misalnya https://www.cool-site.com*
6. **Post-logout redirect URIs** *(Opsional)* URL tempat pengguna dapat diarahkan setelah logout. Ini memastikan pengguna mendarat di halaman yang diketahui setelah sesi mereka berakhir.
7. Klik tombol `Save` untuk melanjutkan.
</Steps>

Anda akan diarahkan kembali ke daftar aplikasi Anda. Anda akan melihat aplikasi baru Anda dalam daftar dan hanya sekali ini Anda dapat mengakses secret key aplikasi. Pastikan untuk menyalinnya dan menyimpannya di tempat yang aman.

## Perbarui aplikasi

Anda perlu memperbarui aplikasi jika salah satu redirect URI Anda berubah atau Anda ingin menambah atau menghapus URI.

Untuk memperbarui aplikasi yang sudah ada, lakukan hal berikut:

<Steps>
1. Klik tautan `Actions` untuk aplikasi yang ingin Anda perbarui.
2. Klik tombol `Update` di bawah Actions.
3. Lakukan perubahan pada aplikasi.
4. Klik tombol `Save` untuk melanjutkan.
</Steps>

Anda akan diarahkan kembali ke daftar aplikasi Anda.

## Hapus aplikasi

:::caution
Harap jangan hapus aplikasi yang kami buat otomatis untuk Anda agar semua fitur dapat berfungsi untuk akun Anda.
:::

Jika Anda tidak lagi membutuhkan aplikasi, Anda dapat menghapusnya.

Untuk menghapus aplikasi, lakukan hal berikut:

<Steps>
1. Klik tautan `Actions` untuk aplikasi yang ingin Anda hapus.
2. Klik tombol `Remove` di bawah Actions.
3. Klik tombol `OK` untuk mengonfirmasi penghapusan.
</Steps>

Anda akan diarahkan kembali ke daftar aplikasi Anda.
