Bu makale, bir geliştirici olarak hızlıca nasıl kurulacağınızı ve Wink üzerinde API’lerimizi test etmeye nasıl başlayacağınızı gösterir.

:::note
Testler her zaman [staging ortamımızda](/tr/getting-started/environments) yapılmalıdır. Orada çok daha az veri bulunur ancak üretim ortamını etkilemeden istediğiniz kadar test yapabilirsiniz.

Canlı yayına geçmeye hazır olduğunuzda, hesabınızı ve uygulamanızı sertifikalandıracağız ve üretim ortamımızda yayına başlayabilirsiniz.
:::

Aşağıdaki adımları izleyin:

<Steps>
1. Wink üzerinde kullanıcı hesabınızı [kaydettirin](/tr/my-account/register).
2. [Bir bağlı kuruluş hesabı oluşturun](/tr/my-account/create-account). Uygulamanızı bağlamak için buna ihtiyacınız olacak. Daha fazla bilgi için [Studio nedir](/tr/studio/what-is-studio) sayfasına bakın.
3. [Bir Uygulama oluşturun](/tr/my-account/applications) ve bunu 2. adımda oluşturduğunuz bağlı kuruluş hesabıyla ilişkilendirin. Sunucudan sunucuya API entegrasyonları için, istemci türü olarak **MACHINE_2_MACHINE** seçin.
4. `Application` oluşturduktan sonra, client-id ve gizli anahtarı hemen kopyalayıp kaydedin — gizli anahtar yalnızca bir kez gösterilir ve tekrar alınamaz.
</Steps>

## CDN

Wink’te tüm resim ve video varlıklarımız için Cloudinary kullanıyoruz. Seyahat envanterimize eriştiğinizde, Cloudinary’nin tüm gelişmiş özelliklerine tam erişiminiz olacak.

`cloud_name` değerimiz `traveliko`dur.

### Cloudinary

- [Başlarken](https://cloudinary.com/documentation/dev_kickstart)
- [SDK’lar](https://cloudinary.com/documentation/cloudinary_sdks)
- [Cloudinary GitHub’da](https://github.com/cloudinary)
-	[Video Eğitimleri](https://cloudinary.com/documentation/cloudinary_sdks_tutorials)

### Örnek

Desteklediğimiz her seyahat envanteri türü için, içinde `multimedias` adlı iç içe geçmiş bir JSON dizisi bulunur. Bir multimedya girdisinin JSON’u şöyle görünür:

```json {3}
{
  "multimediaIdentifier": "multimedia-1",
  "identifier": "partners/radisson_blu_logo",
  "type": "IMAGE",
  "width": "1024",
  "height": "768",
  "category": "1",
  "descriptions": [
    {
      "name": "Lobby",
      "description": "Picture of the lobby",
      "language": "en"
    }
  ],
  "lifestyleType": "LIFESTYLE_BUSINESS",
  "attribution": [
    {
      "url": "https://www.hilton.com",
      "name": "Hilton"
    }
  ]
}
```

Yukarıda vurgulanan satır, Cloudinary benzersiz ID’si `partners/radisson_blu_logo`’yu gösterir. İstediğiniz formatta, kalitede ve boyutta resmi almak için ihtiyacınız olan tek şey budur.

## Test Etme

[API’lerimiz](/tr/developers/apis) bölümüne gidin ve çalışmak istediğiniz API’yi seçin.

### API dokümanları

Dokümantasyon sitemiz, site içinden API komutlarını çalıştırabileceğiniz şekilde yapılandırılmıştır.

Bu örnekte, `Search Lookups (OAuth2)` API çağrısını kullanmak istiyorsunuz.

:::tip[Profesyonel İpucu]
Kullanım kolaylığı için, entegratörlerin `OAuth2` olarak belirttiğimiz uç noktaları kullanmalarını öneririz. Bu uç noktalar API çağrıcılarını bekler ve hangi hesapta olduğunuzu zaten bilir, böylece URL’lerinizde `companyIdentifier` belirtmeniz gerekmez.
:::

<Steps>
1. Hedefleri veya otelleri almak için [Lookup API](https://developer.wink.travel/lookup) kullanın.
2. Sol navigasyonda `Authentication` sekmesine tıklayın.
3. Client-id ve client secret bilgilerinizi girin ve `Get TOKEN` butonuna tıklayın.
4. Kimlik bilgileriniz geçerliyse, API anahtarınızın uygulandığını göreceksiniz.
5. Artık lookup uç noktasını test edebilirsiniz.
</Steps>

### Postman
Aynı işlemi Postman ile de yapabilirsiniz.

<Steps>
1. [Postman](https://www.postman.com/downloads) uygulamasını indirin.
2. [Lookup API](https://developer.wink.travel/lookup) sayfasının üstünden `Download OpenAPI spec` seçeneğine tıklayın.
3. Dosyayı Postman’a aktarın.
4. Eğer yeni bir koleksiyon olarak `Wink API` adıyla aktardıysanız, koleksiyon adının yanındaki yatay üç noktaya tıklayın ve `Edit` seçeneğini seçin.
5. `Authorization` sekmesinde, `Auth type` alanını OAuth2 olarak ayarlayın.
6. `Add auth data to` alanını `Request headers` olarak ayarlayın.
7. `Header prefix` alanını `Bearer` olarak ayarlayın.
8. `Token Name` alanına, Postman’da kaydedeceğiniz token için istediğiniz ismi verin. Örneğin Wink Staging.
9. `Grant type` alanını `Client Credentials` olarak ayarlayın.
10. `Access token URL` alanını https://staging-iam.wink.travel/oauth2/token olarak ayarlayın.
11. `Client ID` alanına uygulamanızın oluşturduğu Client ID’yi girin.
12. `Client Secret` alanına uygulamanızın oluşturduğu Client Secret’ı girin.
13. `Scope` alanına `inventory.read inventory.write inventory.remove` yazın. Bunlar Wink için gereken tüm kapsamlar.
14. `Client Authentication` alanını `Send as Basic Auth header` olarak ayarlayın.
15. `Get New Access Token` butonuna tıklayarak token’ınızı alın ve kaydedin.
16. Wink API koleksiyonundaki tüm API çağrılarında, kimlik doğrulama sekmesinin `Inherit auth from parent` olarak ayarlandığından emin olun; böylece token her çağrıyla gönderilir.
</Steps>

:::tip[Profesyonel İpucu]
Sunucu domaini, client ID ve client secret’ı ortam değişkenleri olarak ayarlamanız önerilir. Böylece staging ve üretim ortamları arasında kolayca geçiş yapabilir ve oluşturduğunuz tüm işlemleri yeniden kullanabilirsiniz.
:::
