[Web Bileşenleri](https://developer.mozilla.org/en-US/docs/Web/API/Web_components), kodlama bilmeden tüm Javascript özelliklerini web sitenize entegre etmenizi sağlayan bir standarttır.
Bu harika teknoloji sayesinde, Wink seyahat envanterini çok az zahmetle sitenize yerleştirmenize olanak tanır. Bu makale, web bileşenleri koleksiyonumuzu adım adım anlatır ve nasıl kullanacağınızı gösterir.

## Wink'i Yerleştirme

Web bileşenlerimizden birini göstermek istediğiniz herhangi bir sayfaya dahil etmeniz gereken üç şey vardır.

1. Stil sayfamızı dahil edin.
2. Javascript'imizi dahil edin.
3. Uygulama yükleyicimizi dahil edin.

### Stil Sayfası

CSS stillerimizi belgenizin head bölümüne ekleyin.

```html {3}
<html>
  <head>
    <link rel="stylesheet" href="https://elements.wink.travel/styles.css"></link>
  </head>
</html>
```

### Javascript

Javascript'imizi belgenizin en altına ekleyin. (*&lt;/body&gt; kapanış etiketinden hemen önce eklemenizi öneririz*).

```html {3}
<html>
  <body>
    <script src="https://elements.wink.travel/elements.js" type="module" defer></script>
  </body>
</html>
```

Artık siteniz web bileşenlerimizi gösterebilecek durumda.

## Bileşenler

Aşağıda bileşen kütüphanemiz hakkında bilgi edinin.

### Uygulama Yükleyicisi

Yükleyici, durumun korunmasından ve bileşenlerimiz arasındaki etkileşimin yönetilmesinden sorumludur. (*Javascript'imizin altına ekleyin*).

```html {3-6}
<html>
  <body>
    <wink-app-loader
      client-id="BURAYA MÜŞTERİ KİMLİĞİNİZİ GİRİN"
      configuration-id="BURAYA ÖZELLEŞTİRME KİMLİĞİNİZİ GİRİN"
    ></wink-app-loader>
  </body>
</html>
```

### İçerik Yükleyicisi

İçerik yükleyici, web bileşenleri kütüphanemizin kalbidir. Envanterinizi (kartlar, ızgaralar, haritalar) görüntülemekten sorumludur.

Mevcut öznitelikler:

- layout
- id
- sort

*`sort` özniteliği yalnızca layout `RANKED` olduğunda ve mevcut bir sıralı ızgara kullanmak istemediğinizde kullanılabilir. Bu durumlarda `id` boş bırakılmalıdır.*

Mevcut layout türleri:

- `AD_BANNER`
- `MAP`
- `HOTEL`
- `GUEST_ROOM`
- `MEETING_ROOM`
- `SPA`
- `RESTAURANT`
- `ACTIVITY`
- `ATTRACTION`
- `PLACE`
- `ADD_ON`
- `LIST`
- `SEARCH`
- `RANKED`

Mevcut sıralama türleri:

- `MEMBER`
- `PRICE_LOW_TO_HIGH`
- `PRICE_HIGH_TO_LOW`
- `PRICE`
- `POPULARITY`
- `ECO`
- `EXPERIENCE`
- `PERK`
- `LOYALTY`
- `PACKAGE`

```html {3-6}
<html>
  <body>
    <wink-content-loader
       layout="GUEST_ROOM"
       id="2de7ee9c-61c9-11ef-9722-42004e494300"
    ></wink-content-loader>
  </body>
</html>
```

Örnekler için [kartlar](/tr/studio/cards), [ızgaralar](/tr/studio/grids) ve [haritalar](/tr/studio/maps) sayfalarına bakabilirsiniz.

```html {3}
<html>
  <body>
    <wink-lookup></wink-lookup>
  </body>
```

## Bileşenler

### Ödeme

Ödeme bileşeni, bir gezginin bir şey satın almak istediğini ve bu öğeler için kurallar ile fiyatları TripPay'e bildirmenizi sağlar.

Bileşenin bir zorunlu özniteliği vardır:

- `id` Gerçekleştirilmesini istediğiniz rezervasyon sözleşmesinin tanımlayıcısı.

```html {3}
<html>
  <body>
    <trip-pay id="<INSERT_BOOKING_CONTRACT_ID>"></trip-pay>
  </body>
</html>
```

:::note
Rezervasyon sözleşmesi `id`'sinin nasıl oluşturulacağını öğrenmek için kılavuza bakın: [TripPay ile Entegrasyon](/tr/guides/integrating-with-trip-pay).
:::

## Hatalar

WinkLinks'te veya kendi web sitenizde web bileşenlerimizden birini yerleştirirken bir hata mesajı ile karşılaşırsanız, birkaç olası sorun olabilir:

### Kullanılabilirlik

Envanter artık mevcut olmayabilir. [Wink Studio](/tr/studio/cards) sayfasına gidin ve "kart" durumunun kullanılabilir olup olmadığını kontrol edin.
Eğer değilse, kırmızı olarak görünecektir. Bu durumda, tedarikçi tekrar kullanılabilir hale getirebilir veya listenizden kaldırabilirsiniz.

:::note
`Kullanılabilirlik`, envanterin günlük olarak rezerve edilebilir olması anlamına gelmez. Bu, envanterin tamamen size ve/veya başkalarına kapalı olduğu anlamına gelir.
:::

### Kaldırılmış

Envanter kaldırılmış olabilir. [Wink Studio](/tr/studio/cards) sayfasına gidin ve "kart" durumunun kullanılabilir olup olmadığını kontrol edin.
Eğer değilse, kırmızı olarak görünecektir. Bu durumda, tedarikçi tekrar kullanılabilir hale getirebilir veya listenizden kaldırabilirsiniz.

### Özelleştirme

Envanterinizle birlikte tanımladığınız özelleştirmeyi yanlışlıkla kaldırmış olabilirsiniz. Özelleştirmenin hala mevcut olduğundan emin olun ve eksikse yeni bir tane ayarlayın.

### Uygulama

Bu sadece web geliştiricileri içindir. Eğer yanlışlıkla [Uygulama](/tr/developer/applications) kaldırıldıysa ve müşteri kimliği artık mevcut değilse, lütfen yeni bir uygulama oluşturun ve yeni müşteri kimliğini kullanarak Web Bileşenlerimizi yerleştirin.
