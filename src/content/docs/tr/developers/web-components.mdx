[Web Bileşenleri](https://developer.mozilla.org/en-US/docs/Web/API/Web_components), kodlama bilmeden tüm Javascript özelliklerini web sitenize entegre etmenizi sağlayan bir standarttır.
Bu harika teknoloji sayesinde, Wink seyahat envanterini çok az zahmetle sitenize yerleştirebilirsiniz. Bu makale, web bileşenleri koleksiyonumuzu adım adım anlatır ve nasıl kullanacağınızı gösterir.

## Wink'i Yerleştirme

Web bileşenlerimizden birini göstermek istediğiniz herhangi bir sayfaya dahil etmeniz gereken üç şey vardır.

1. Stil sayfamızı dahil edin.
2. Javascript'imizi dahil edin.
3. Uygulama yükleyicimizi dahil edin.

### Stil Sayfası

CSS stillerimizi belgenizin `<head>` bölümüne ekleyin.

```html {3}
<html>
  <head>
    <link rel="stylesheet" href="https://elements.wink.travel/styles.css"></link>
  </head>
</html>
```

### Javascript

Javascript'imizi belgenizin en altına ekleyin. (*Bunu `</body>` kapanış etiketinden hemen önce yapmanızı öneririz*).

```html {3}
<html>
  <body>
    <script src="https://elements.wink.travel/elements.js" type="module" defer></script>
  </body>
</html>
```

Artık siteniz web bileşenlerimizi göstermek için hazırdır.

## Bileşenler

Aşağıda bileşen kütüphanemiz hakkında bilgi edinebilirsiniz.

### Uygulama Yükleyicisi

Yükleyici, durumun korunmasından ve bileşenlerimiz arasındaki etkileşimin yönetilmesinden sorumludur. (*Javascript'imizin altına ekleyin*).

```html {3-6}
<html>
  <body>
    <wink-app-loader
      client-id="BURAYA MÜŞTERİ KİMLİĞİNİZİ GİRİN"
      configuration-id="BURAYA ÖZELLEŞTİRME KİMLİĞİNİZİ GİRİN"
    ></wink-app-loader>
  </body>
</html>
```

### İçerik Yükleyicisi

İçerik yükleyici, web bileşeni kütüphanemizin kalbidir. Envanterinizi (kartlar, ızgaralar, haritalar) görüntülemekten sorumludur.

Kullanılabilir öznitelikler:

- layout
- id
- sort

*`sort` özniteliği yalnızca `RANKED` düzeni kullanıldığında ve mevcut bir sıralı ızgara kullanılmak istenmediğinde geçerlidir. Bu durumlarda `id` boş bırakılmalıdır.*

Kullanılabilir düzen türleri:

- `AD_BANNER`
- `MAP`
- `HOTEL`
- `GUEST_ROOM`
- `MEETING_ROOM`
- `SPA`
- `RESTAURANT`
- `ACTIVITY`
- `ATTRACTION`
- `PLACE`
- `ADD_ON`
- `LIST`
- `SEARCH`
- `RANKED`

Kullanılabilir sıralama türleri:

- `MEMBER`
- `PRICE_LOW_TO_HIGH`
- `PRICE_HIGH_TO_LOW`
- `PRICE`
- `POPULARITY`
- `ECO`
- `EXPERIENCE`
- `PERK`
- `LOYALTY`
- `PACKAGE`

```html {3-6}
<html>
  <body>
    <wink-content-loader
       layout="GUEST_ROOM"
       id="2de7ee9c-61c9-11ef-9722-42004e494300"
    ></wink-content-loader>
  </body>
</html>
```

Örnekler için [kartlar](/studio/cards), [ızgaralar](/studio/grids) ve [haritalar](/studio/maps) sayfalarına bakabilirsiniz.

## Bileşenler

### Ödeme

Ödeme bileşeni, TripPay'e bir yolcunun bir şey satın almak istediğini ve bu öğeler için kurallar ile fiyatları bildirmenizi sağlar.

Bileşenin zorunlu bir özniteliği vardır:

- `id` Gerçekleştirmek istediğiniz rezervasyon sözleşmesinin tanımlayıcısı.

```html {3}
<html>
  <body>
    <trip-pay id="<INSERT_BOOKING_CONTRACT_ID>"></trip-pay>
  </body>
</html>
```

:::note
Rezervasyon sözleşmesi `id`'sinin nasıl oluşturulacağını öğrenmek için kılavuza bakın: [TripPay ile Entegrasyon](/guides/integrating-with-trip-pay).

## Hatalar

Web bileşenlerimizden birini WinkLinks üzerinde veya kendi web sitenizde yerleştirirken bir hata mesajı ile karşılaşırsanız, aşağıdaki birkaç şey yanlış gitmiş olabilir:

### Kullanılabilirlik

Envanter artık kullanılamıyor olabilir. [Wink Studio](/studio/cards) sayfasına gidin ve "kart" durumunun kullanılabilir olup olmadığını kontrol edin.
Eğer değilse, kırmızı olarak gösterilir. Bu durumda, tedarikçi tekrar kullanılabilir hale getirebilir veya listenizden kaldırabilirsiniz.

:::note
`Kullanılabilirlik`, envanterin günlük olarak rezerve edilebilir olması anlamına gelmez. Bu, envanterin size ve/veya başkalarına tamamen kullanılamaz olduğu anlamına gelir.
:::

### Kaldırılmış

Envanter kaldırılmış olabilir. [Wink Studio](/studio/cards) sayfasına gidin ve "kart" durumunun kullanılabilir olup olmadığını kontrol edin.
Eğer değilse, kırmızı olarak gösterilir. Bu durumda, tedarikçi tekrar kullanılabilir hale getirebilir veya listenizden kaldırabilirsiniz.

### Özelleştirme

Envanterinizle birlikte tanımladığınız özelleştirmeyi yanlışlıkla kaldırmış olabilirsiniz. Özelleştirmenin hala mevcut olduğundan emin olun ve eksikse yeni bir tane ayarlayın.

### Uygulama

Bu sadece web geliştiricileri içindir. Eğer yanlışlıkla [Uygulama](/developer/applications) kaldırıldıysa ve müşteri kimliği artık mevcut değilse, lütfen yeni bir uygulama oluşturun ve yeni müşteri kimliğini Web Bileşenlerimizi yerleştirmek için kullanın.
