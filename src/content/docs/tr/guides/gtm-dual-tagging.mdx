Sojern üzerinden reklam kampanyaları yürütüyorsanız, Sojern GTM Container ID'nizi Wink portalına girerek Sojern hesabınızı Wink ile bağlayabilirsiniz. Bağlandıktan sonra, Sojern'in takip pikseli mülkünüz için otomatik olarak tetiklenir — aramadan tamamlanmış rezervasyona kadar tüm misafir yolculuğunu kapsar.

<Aside type="tip">
Bu, zaten aktif bir Sojern hesabınızın olduğu ve Sojern hesap yöneticinizden GTM Container ID'nizi aldığınız varsayımıyla yapılır. Henüz bir Sojern hesabınız yoksa, başlamak için [sojern.com/contact](https://www.sojern.com/contact) adresini ziyaret edin.
</Aside>

---

## Yapmanız Gerekenler

<Steps>

1. **Sojern'den GTM Container ID'nizi Alın**

   Sojern hesap yöneticinizle iletişime geçin ve **GTM Container ID** talep edin. Formatı `GTM-XXXXXXX` şeklindedir. Bu, Sojern Pixel ID'nizden farklıdır — özellikle GTM Container ID istediğinizden emin olun.

2. **Bunu Wink Portalına Girin**

   [my.wink.travel](https://my.wink.travel) adresine giriş yapın, hesabınızı seçin ve **Customizations** sekmesine gidin. Sojern alanına Sojern GTM Container ID'nizi yapıştırın ve kaydedin. Entegrasyonunuz artık aktif.

3. **Sojern Hesap Yöneticinizle Onaylayın**

   Entegrasyonun aktif olduğunu Sojern hesap yöneticinize bildirin. Onlar, mülkünüz için `wink.travel` üzerinde pikselin doğru şekilde tetiklendiğini ve verilerin Sojern hesabınıza aktığını kendi taraflarından doğrulayabilirler.

</Steps>

<Aside type="note">
Bu entegrasyon için kendi web sitenize herhangi bir piksel kodu yerleştirmenize gerek yoktur. Wink, mülkünüzün aktif olduğu tüm sayfalarda Sojern konteynerinizi otomatik olarak yükler.

Bu yapılandırma herhangi bir özel GTM Container ID için çalışır. Örnek olarak Sojern kullanıldı çünkü çok yaygın bir entegrasyondur.
</Aside>

## Wink Tarafında Neler Yapılır

Şeffaflık için, bu entegrasyonun çalışması için Wink'in yapılandırdıkları:

- **Sojern GTM Container ID alanı** Wink portalınızın Customizations sekmesinde mevcuttur. Container ID'nizi girdiğinizde, Wink formatı doğrular ve mülkünüze güvenli şekilde kaydeder.
- **Mülkünüzün aktif olduğu her sayfada** Wink, Sojern GTM Container ID'nizi sayfaya yerleştirir ve kendi etiket yönetim sistemiyle birlikte Sojern konteynerinizi dinamik olarak yükler. Bu otomatik gerçekleşir — Wink ekibinden mülk başına manuel bir işlem gerekmez.
- **Sojern konteyneriniz asenkron olarak yüklenir**, yani misafirleriniz için sayfa yükleme performansını etkilemez.
- **Wink'in kendi takibi ile Sojern pikseli paralel çalışır**, aynı paylaşılan etkinlik verisini okur. Wink'in takip ettiği her rezervasyon etkinliği aynı anda Sojern konteynerinizde de kullanılabilir.
- **Mülkünüz görüntülenmediğinde** — örneğin, misafir Wink'te başka bir oteli incelerken — Sojern konteyneri yüklenmez. Bu, Sojern'in sadece mülkünüze ait ilgili verileri almasını sağlar.

## DataLayer Referansı

Wink, misafir yolculuğu boyunca GA4 uyumlu etkinlikleri `dataLayer`'a iter. GTM konteyneriniz bu aynı paylaşılan `dataLayer`'ı okur, böylece yapılandırdığınız herhangi bir tetikleyici veya değişken aşağıda açıklanan tüm özelliklere erişebilir.

Her etkinlik standart GA4 Gelişmiş E-Ticaret yapısını takip eder: üst düzey bir `event` stringi ve bir `ecommerce` nesnesi. Her push öncesinde, Wink eski e-ticaret yükünü temizler, böylece eski veriler yeni tetikleyicilere sızmaz.

<Aside type="note">
**dataLayer sıfırlama deseni**

Wink her etkinlik push'undan önce ecommerce nesnesini sıfırlar:

```js
dataLayer.push({ ecommerce: null });   // önceki etkinliği temizle
dataLayer.push({ event: '...', ecommerce: { ... } });
```

E-ticaret özelliklerinin etkinlikler arasında kalıcı olduğunu varsaymayın. Her zaman tetikleme anında aynı etkinlik içinde okuyun.
</Aside>

### Rezervasyon Yolculuğu Etkinlikleri

Misafir rezervasyon hunisinde ilerledikçe bu etkinlikler otomatik tetiklenir. GTM tetikleyicilerinizi aşağıdaki etkinlik isimleriyle eşleşen **Custom Event** tetikleyicileri olarak yapılandırın.

| Adım | Etkinlik | Ne Zaman Tetiklenir |
|------|----------|---------------------|
| 1 | `view_item_list` | Otel envanter ızgarası veya listesi yüklendiğinde |
| 2 | `view_item` | Misafir belirli bir mülk sayfasını görüntülediğinde |
| 3 | `add_to_cart` | Misafir sepete bir oda eklediğinde |
| 4 | `begin_checkout` | Ödeme / checkout sayfası yüklendiğinde |
| 5 | `add_payment_info` | Misafir bir ödeme yöntemi seçtiğinde |
| 6 | `login` | Misafir SSO ile kimlik doğrulaması yaptığında |
| 7 | `purchase` | Rezervasyon onaylandığında |
| 8 | `refund` | Rezervasyon iptal edildiğinde |

### GA4'te Özel Boyutlar

GA4, özel ürün parametrelerini raporlarda görmezden gelir; bunları açıkça ürün kapsamlı özel boyut olarak kaydetmeniz gerekir. Raporlarda kullanmayı planladığınız parametreleri kaydetmeniz yeterlidir — tüm 18 parametreyi kaydetmeniz gerekmez.

<Steps>

1. **Özel Tanımları Açın**

   GA4'te, **Admin** (sol alt dişli simgesi) → mülkünüzü seçin → **Data display** → **Custom definitions**.

2. **Yeni özel boyut oluşturun**

   **Create custom dimensions** butonuna tıklayın.

3. **Boyut detaylarını doldurun**

   - **Dimension name** — GA4 raporlarında görünen okunabilir etiket (aşağıdaki tabloya bakın)
   - **Scope** — **Item** seçin
   - **Event parameter** — dataLayer'da göründüğü tam parametre adı (aşağıdaki tabloya bakın)
   - **Description** — isteğe bağlı ama önerilir

4. **Kaydedin ve tekrarlayın**

   **Save** tıklayın, ardından takip etmek istediğiniz her parametre için tekrarlayın.

</Steps>

<Aside type="note">
Özel boyutların GA4 raporlarında görünmeye başlaması kayıt işleminden sonra 24 saate kadar sürebilir. Kayıttan önce toplanan geçmiş veriler geriye dönük olarak eklenmez.

GA4, standart mülklerde mülk başına 50 ürün kapsamlı özel boyuta izin verir (360 mülklerde daha fazlası mümkündür).
</Aside>

Aşağıdaki tablo, özel boyut oluştururken referansınız olsun. **Event parameter** değeri tam olarak eşleşmelidir — GA4 büyük/küçük harfe duyarlıdır.

| Boyut adı | Event parameter | Tür |
|----------------|-----------------|------|
| Property ID | `property_id` | Metin |
| Property name | `property_name` | Metin |
| Property brand | `property_brand` | Metin |
| Property chain | `property_chain` | Metin |
| City name | `city_name` | Metin |
| Country code | `country_code` | Metin |
| Check-in date | `start_date` | Metin |
| Check-out date | `end_date` | Metin |
| Number of adults | `num_adults` | Sayı |
| Number of children | `num_children` | Sayı |
| Cancellation policy | `cancellation_policy` | Metin |
| Rate plan ID | `rate_plan_id` | Metin |
| Room rate ID | `room_rate_id` | Metin |
| Room rate name | `room_rate_name` | Metin |
| Number of perks | `num_perks` | Sayı |
| Room location | `room_location` | Metin |
| Room view | `room_view` | Metin |
| Room classification | `room_classification` | Metin |

### `virtualPageView`

Bu özel bir Wink etkinliğidir — standart GA4 etkinlik setinin parçası değildir. Her sayfa gezinmesinde itilir ve cihaz ile oturum bağlamını taşır. `ecommerce` nesnesi içermez.

```json
{
  "event": "virtualPageView",
  "device": "Desktop",
  "deviceAgent": "Mozilla/5.0 ...",
  "os": "MacOS",
  "path": "/hotels/grand-resort",
  "title": "Grand Resort — Wink",
  "userId": "user_abc123",
  "startDate": "2026-06-01",
  "endDate": "2026-06-05",
  "guests": "2",
  "rooms": "1"
}
```

| Özellik | Açıklama |
|----------|-------------|
| `device` | Algılanan cihaz türü — `Desktop`, `Android`, `iPhone`, `iPad` veya benzeri |
| `deviceAgent` | Ham kullanıcı ajanı dizisi |
| `os` | Algılanan işletim sistemi — `Windows`, `MacOS`, `Linux`, `UNIX` |
| `path` | Geçerli sayfa yolu |
| `title` | Geçerli sayfa başlığı |
| `userId` | Kimliği doğrulanmış kullanıcı tanımlayıcısı (giriş yapılmadıysa yok) |
| `startDate` | Geçerli arama oturumundan giriş tarihi |
| `endDate` | Geçerli arama oturumundan çıkış tarihi |
| `guests` | Geçerli arama oturumundan misafir sayısı |
| `rooms` | Geçerli arama oturumundan oda sayısı |
