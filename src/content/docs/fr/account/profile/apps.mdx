L’onglet **Apps** sur [iam.wink.travel/my-account/apps](https://iam.wink.travel/my-account/apps) liste toutes les applications OAuth2 auxquelles vous avez accordé l’accès à votre compte Wink, avec un moyen de révoquer cet accès.

## Comment une application arrive ici

Lorsque vous cliquez sur `Authorize` dans un outil tiers qui s’intègre à Wink (ou dans l’une des applications Wink), le flux de consentement enregistre l’autorisation. Dès lors, cette application peut agir en votre nom dans les limites des permissions que vous avez approuvées.

## Vérification des accès

Chaque entrée affiche :

- **Nom de l’application** et une courte description.
- **Permissions** auxquelles l’application peut accéder (par exemple lire l’inventaire, gérer les réservations).
- **Dernière utilisation** — la dernière fois que l’application a appelé l’API Wink en votre nom.

## Révoquer une application

Cliquez sur `Revoke` à côté de n’importe quelle entrée. Les jetons de l’application sont invalidés immédiatement, et toutes les tâches en arrière-plan qu’elle exécute sur votre compte commenceront à échouer avec des erreurs 401.

:::tip
La révocation n’est pas destructive — vous pouvez réautoriser l’application à tout moment en passant de nouveau par son flux de consentement. Utilisez cette option si une application se comporte mal, si vous changez vos identifiants, ou si vous n’utilisez plus l’intégration.
:::

## Applications OAuth que vous développez vous-même

Si vous souhaitez *créer* une application OAuth (et pas seulement en autoriser une), cette fonctionnalité est disponible au niveau du compte organisation — voir [Paramètres → Applications](/fr/portal/settings/applications).
