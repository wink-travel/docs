Si vous êtes un intégrateur, comme Wink, et que vous souhaitez mapper vos bénéficiaires externes pour les utiliser avec TripPay, vous pouvez le faire de deux manières :

* Cartographie manuelle
* En utilisant le [API de paiement](/developers/apis/#payment-api) (*Voir `Account Mappings`*)

:::tip
À moins que vous ne traitiez que quelques propriétés identiques, nous vous recommandons d'utiliser l'API pour automatiser cette étape.
:::

Ci-dessous, nous vous expliquons la méthode manuelle pour mapper un bénéficiaire externe.*Il utilise l'API sous le capot.*

<Steps>
  1. Connectez-vous à [TripPay](https://pay.wink.travel).
  2. Sélectionnez votre compte intégrateur.
  3. Cliquez `Settings > Mappings` depuis la barre de navigation principale.
  4. Cliquez sur le `Create mapping` bouton. Vous verrez le formulaire de mappage.
  5. **Compte local** Commencez à saisir le nom du TripPay existant que vous souhaitez mapper.
  6. **Identifiant externe** Collez votre ID externe pour cette entité telle qu'elle existe dans le système distant.
  7. Cliquez `Save` pour continuer. Vous êtes redirigé vers votre liste de mappages.
</Steps>


Grâce à vos mappages en place, TripPay sait désormais quels comptes créditer en tant que bénéficiaires d'une réservation lorsque vous y faites référence à l'aide de vos identifiants externes.
