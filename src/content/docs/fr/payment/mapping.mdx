Si vous êtes un intégrateur, comme Wink, et que vous souhaitez associer vos bénéficiaires externes pour une utilisation avec TripPay, vous pouvez le faire de deux manières :

- Cartographie manuelle
- Utilisation de la [Payment API](/fr/developers/apis/#payment-api) (*Voir `Account Mappings`*)

:::tip
Sauf si vous ne gérez qu’un petit nombre des mêmes propriétés, nous vous recommandons d’utiliser l’API pour automatiser cette étape.
:::

Ci-dessous, nous vous guidons à travers la méthode manuelle pour associer un bénéficiaire externe. *Elle utilise l’API en arrière-plan.*

<Steps>
1. Connectez-vous à [TripPay](https://pay.wink.travel).
2. Sélectionnez votre compte intégrateur.
3. Cliquez sur `Settings > Mappings` dans la barre de navigation principale.
4. Cliquez sur le bouton `Create mapping`. Vous verrez le formulaire de cartographie.
5. **Local account** Commencez à taper le nom du compte TripPay existant que vous souhaitez associer.
6. **External identifier** Collez votre ID externe pour cette entité tel qu’il existe dans le système distant.
7. Cliquez sur `Save` pour continuer. Vous êtes redirigé vers votre liste de cartographies.
</Steps>

Avec vos cartographies en place, TripPay sait désormais quels comptes créditer en tant que bénéficiaires d’une réservation lorsque vous les référez en utilisant vos identifiants externes.

## API

Les développeurs qui souhaitent gérer les `Payments` peuvent se rendre dans [Developers > API > Payments](/fr/developers/apis/#payment-api).
