La personnalisation est une fonctionnalité puissante que vous pouvez utiliser pour contrôler chaque aspect de l'intégration avec Wink sans avoir à écrire de code. Chaque élément d'inventaire que vous vendez peut être personnalisé. Ajoutez ou supprimez des fonctionnalités, changez le schéma de couleurs et bien plus encore.

Sur la page `Personnalisation`, cliquez sur le bouton `Actions` sous la personnalisation avec laquelle vous souhaitez travailler, puis cliquez sur `Mettre à jour`.
*Vous disposez déjà d'une entrée par défaut lors de la création de votre compte. Cette entrée ne peut pas être supprimée.*

Cela vous mènera au formulaire de personnalisation où vous pouvez voir 5 à 7 onglets (*selon votre type de compte*).

## Général

Voici les options disponibles sous l'onglet `Général`.

- **Nom** Donnez un nom à votre personnalisation pour vous en souvenir facilement. *ex. Pour mon blog de voyage*
- **Devise d'affichage** Sélectionnez la devise initiale que les utilisateurs verront affichée à côté de l'inventaire de voyage réservable. *ex. USD*
- **Langue d'affichage** Sélectionnez la langue initiale que les utilisateurs rencontreront. *ex. Anglais*
- **Ajouter au titre de la page** Vous pouvez contrôler ce qui est ajouté au titre de la page (*le titre que vous voyez dans l'onglet du navigateur*). Ceci est principalement utilisé pour notre moteur de réservation. *ex. Si vous naviguez sur https://ota.wink.travel, par défaut, vous verrez `Home | Traveliko - Hotels with Benefits`*. Vous pouvez modifier tout ce qui suit le caractère '|' (pipe).

## Thème

L'onglet `Thème` vous permet de personnaliser l'apparence de notre site [moteur de réservation](/booking-engine/in-a-nutshell), du site [WinkLinks](/link-manager/wink-links) et de tous nos [Composants Web](/developers/web-components).

### Configuration

Cette section vous permet de configurer le design et la disposition des cartes. Pour l'instant, il n'existe qu'un seul design : Wink.

La disposition des cartes contrôle si vous souhaitez que vos utilisateurs voient des cartes transactionnelles alignées horizontalement ou verticalement. Les informations sont les mêmes.

:::note
Il n'y a pas de différence dans la disposition des cartes lors de la consultation de nos cartes transactionnelles depuis un appareil mobile. Cependant, vous pouvez voir la différence sur un iPad mini et des résolutions plus grandes.
:::

:::tip[Astuce pro]
Utilisez la disposition horizontale si vous avez moins de 4 inventaires de voyage (c.-à-d. types de chambres) dans une liste... c'est simplement plus esthétique.
:::

La couleur contrôle la façon dont vos utilisateurs voient nos composants UI et sites web. Utilisez-la pour assortir votre schéma de couleurs existant [le cas échéant].

## Analyse

Nous prenons en charge l'analyse GA4 nativement. Si vous souhaitez suivre l'ensemble de votre parcours de conversion e-commerce amélioré, de bout en bout, ayez votre ID de mesure prêt et saisissez-le dans le champ sous l'onglet `Analyse`.

<Aside type="caution" title="Rappel">
  N'oubliez pas d'ajouter `i.trvl.as`, `book.wink.travel` et `ota.wink.travel` à vos flux de propriété GA4.
</Aside>

## Itinéraire

L'onglet `Itinéraire` est probablement le plus intéressant pour s'assurer que les prix s'affichent pour l'inventaire que vous avez sélectionné.
Comme la tarification de l'inventaire de voyage est assez complexe, Wink doit connaître l'itinéraire de l'utilisateur avant de pouvoir afficher un prix précis.

Cette section vous permet de définir l'itinéraire par défaut pour afficher les prix sur tous nos espaces digitaux, tels que :

- Nos images adaptées aux réseaux sociaux avec superposition de prix.
- Les grilles, cartes et liens WinkLinks.
- Composants Web
- Moteur de réservation

### Dates

Vous pouvez choisir de définir une plage de dates statique ou glissante.

- Choisissez une plage de dates statique si vous utilisez la personnalisation pour un événement particulier qui a lieu à une date donnée.
- Choisissez une plage de dates glissante si vous souhaitez soit :
    1. Par défaut aujourd'hui + 1 nuit, ce que font la plupart des OTA, OU.
    2. Par défaut une date dans le futur qui devrait toujours avoir de la disponibilité.

### Configuration des chambres

Par défaut, l'itinéraire est réglé sur 2 adultes, car c'est le réglage le plus courant pour la plupart des voyageurs.
Cela peut être facilement modifié et vous pouvez ajouter plusieurs chambres avec plusieurs occupants ; adultes et enfants.

### Promotions

Les promotions sont une fonctionnalité astucieuse qui fait gagner du temps à vos utilisateurs. Si une propriété vous a donné un code promo à partager avec vos amis, vous pouvez choisir de le saisir ici.
Toute personne arrivant via votre lien, ou similaire, verra automatiquement le code promo ajouté à son itinéraire et le prix réduit s'appliquera automatiquement.

## Intégration

L'onglet `Intégration` est destiné aux hôtels, intégrateurs et agents de voyage qui souhaitent avoir plus de contrôle sur la façon dont les réservations sur Wink sont distribuées.

### Notifications

Cela donne à l'hôtel et à l'intégrateur la possibilité de laisser Wink notifier toutes les parties impliquées lorsqu'une réservation est effectuée.
Cela inclut la notification du gestionnaire de canal.

Par défaut, ces options sont toutes activées. Les cas d'usage où un intégrateur pourrait vouloir désactiver une ou plusieurs de ces fonctionnalités sont :

- L'agent de voyage est intégré au niveau API et souhaite envoyer ses propres e-mails de confirmation.
- L'hôtel teste son intégration et ne veut pas déranger son personnel commercial.
- L'hôtel ne se soucie pas des e-mails et souhaite uniquement être notifié via son PMS.

### Logo e-mail

Pour les intégrateurs qui souhaitent utiliser notre service de notification par e-mail, ils peuvent personnaliser l'e-mail pour qu'il semble provenir d'eux.

:::note
  Les dimensions recommandées sont 300px de large x 75px de haut
:::

## Spécifique à l'hôtel

Sous l'onglet `Spécifique à l'hôtel` se trouvent des options qui permettent à l'hôtel de contrôler ce qui est affiché sur leur page d'accueil sur notre moteur de réservation et ce qui s'affiche lorsqu'il n'y a pas d'inventaire pour l'itinéraire demandé.

- **Classements** : Une propriété qui n'a pas encore obtenu de classements significatifs peut choisir de désactiver cette fonctionnalité.

:::note
  La propriété ne peut pas contrôler la façon dont un affilié choisit d'afficher la page d'accueil de l'hôtel... Ce sont deux configurations différentes.
:::

- **Disponibilité** : Permet à la propriété de choisir si elle souhaite continuer à afficher l'inventaire même s'il n'y a pas de disponibilité.

:::note
  Dans la plupart des cas, la propriété voudra donner aux utilisateurs un moyen de cliquer sur le bouton `Réserver` pour continuer vers leur page d'accueil où ils peuvent modifier leur itinéraire et essayer de trouver une date disponible.
  Activez ce commutateur lorsque vous souhaitez bloquer l'utilisateur pour qu'il ne puisse pas continuer.
:::

## Comment utiliser

Dans la plupart des scénarios, vous choisirez quelle `Personnalisation` appliquer à vos liens, cartes, cartes et grilles.

Voici les cas où vous pourriez vouloir appliquer manuellement votre configuration :

- **Surcharge manuelle** Vous souhaitez partager un lien vers notre moteur de réservation directement, sans utiliser notre fonctionnalité [Liens partageables](/studio/shareable-links), avec votre configuration personnalisée appliquée.
- **Plugin WordPress** Ajoutez la configuration à vos `Paramètres` du plugin Wink [WordPress](/developers/wordpress).
- **Intégration personnalisée** Appliquez-la manuellement à votre intégrateur personnalisé avec l'utilisation de notre composant Web [wink-app-loader](/developers/web-components/#application-loader).

### Surcharge manuelle

Voici un exemple simple de comment appliquer votre ID de configuration, avec tous vos paramètres, aux URL principales de notre moteur de réservation.

<h4 align="center"> [https://ota.wink.travel?configurationId=MY-UNIQUE-ID](https://ota.wink.travel?configurationId=MY-UNIQUE-ID) </h4>

:::note
  Lorsqu'un utilisateur clique sur ce lien, vous définirez également votre cookie d'attribution affilié.
:::

### Plugin WordPress

Rendez-vous sur le [plugin WordPress Wink](/developers/wordpress) pour en savoir plus.

### Intégration personnalisée

Contrôlez tous vos composants Web Wink intégrés sur une page avec le composant Web [wink-app-loader](/developers/web-components/#application-loader).

Voici un exemple utilisant `MY-UNIQUE-ID` comme ID de configuration.

```html {11}
<html>
  <head>
    <link rel="stylesheet" href="https://elements.wink.travel/styles.css"></link>
  </head>
  <body>

    <script src="https://elements.wink.travel/elements.js" type="module" defer></script>

    <wink-app-loader
      client-id="YOUR CLIENT ID GOES HERE"
      configuration-id="MY-UNIQUE-ID"
    ></wink-app-loader>
  </body>
</html>
```
