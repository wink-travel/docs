Wenn Sie ein Integrator wie Wink sind und Ihre externen Zahlungsempfänger für die Verwendung mit TripPay zuordnen möchten, haben Sie dafür zwei Möglichkeiten:

* Manuelle Zuordnung
* Mithilfe der[Zahlungs-API](/developers/apis/#payment-api)(*Sehen `Account Mappings`*)

:::Tipp
Sofern Sie nicht nur mit einer Handvoll identischer Eigenschaften arbeiten, empfehlen wir Ihnen, diesen Schritt über die API zu automatisieren.
:::

Nachfolgend führen wir Sie durch die manuelle Zuordnung eines externen Zahlungsempfängers.*Es verwendet die API im Hintergrund.*

<Steps>
  1. Melden Sie sich an [TripPay](https://pay.wink.travel).
  2. Wählen Sie Ihr Integratorkonto aus.
  3. Klicken `Settings > Mappings` aus der Hauptnavigationsleiste.
  4. Klicken Sie auf das `Create mapping` Schaltfläche. Das Zuordnungsformular wird angezeigt.
  5. **Lokales Konto** Beginnen Sie mit der Eingabe des Namens des vorhandenen TripPay, das Sie zuordnen möchten.
  6. **Externe Kennung** Fügen Sie Ihre externe ID für diese Entität ein, wie sie im Remote-System vorhanden ist.
  7. Klicken `Save` , um fortzufahren. Sie werden zurück zu Ihrer Zuordnungsliste geleitet.
</Steps>


Wenn Ihre Zuordnungen eingerichtet sind, weiß TripPay nun, welche Konten als Begünstigte einer Buchung gutgeschrieben werden müssen, wenn Sie mit Ihren externen Kennungen auf sie verweisen.
