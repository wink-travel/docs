Wenn Sie ein Integrator sind, wie Wink, und Ihre externen Zahlungsempfänger für die Verwendung mit TripPay zuordnen möchten, können Sie dies auf zwei Arten tun:

- Manuelle Zuordnung
- Verwendung der [Payment API](/de/developers/apis/#payment-api) (*Siehe `Account Mappings`*)

:::tip
Sofern Sie nicht nur mit einer Handvoll derselben Objekte zu tun haben, empfehlen wir, die API zu verwenden, um diesen Schritt zu automatisieren.
:::

Im Folgenden führen wir Sie durch die manuelle Methode zur Zuordnung eines externen Zahlungsempfängers. *Dabei wird die API im Hintergrund verwendet.*

<Steps>
1. Melden Sie sich bei [TripPay](https://pay.wink.travel) an.
2. Wählen Sie Ihr Integratorkonto aus.
3. Klicken Sie in der Hauptnavigationsleiste auf `Einstellungen > Zuordnungen`.
4. Klicken Sie auf die Schaltfläche `Zuordnung erstellen`. Sie sehen das Zuordnungsformular.
5. **Lokales Konto** Beginnen Sie, den Namen des vorhandenen TripPay-Kontos einzugeben, das Sie zuordnen möchten.
6. **Externe Kennung** Fügen Sie Ihre externe ID für diese Entität ein, wie sie im entfernten System existiert.
7. Klicken Sie auf `Speichern`, um fortzufahren. Sie werden zurück zu Ihrer Liste der Zuordnungen weitergeleitet.
</Steps>

Mit Ihren Zuordnungen weiß TripPay nun, welche Konten als Begünstigte einer Buchung gutgeschrieben werden sollen, wenn Sie diese mit Ihren externen Kennungen referenzieren.

## API

Entwickler, die `Payments` verwalten möchten, gehen zu [Developers > API > Payments](/de/developers/apis/#payment-api).
