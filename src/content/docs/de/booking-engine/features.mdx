Die Buchungsmaschine ist in zwei Varianten erhältlich, um die unterschiedlichen Anforderungen eines verbundenen Unternehmens, einer Immobilienverwaltungsgesellschaft und eines Hotels zu erfüllen.

## Online-Reisebüro

Für einen Partner ist die Nutzung der Online-Reisebüro-Version am sinnvollsten. Der Grund dafür ist, dass die Website so gestaltet ist, dass Reisende mehrere Unterkünfte besichtigen können.

Die Homepage zu dieser Anwendung finden Sie weiter unten.

<LinkButton href="https://ota.wink.travel/home" variant="secondary" icon="external" target="_blank">
  Wink OTA
</LinkButton>


## Internet-Buchungsmaschine

Für ein Hotel oder eine Hausverwaltung ist die Nutzung der Internet-Buchungsmaschine am sinnvollsten. Dieser Benutzertyp möchte ein einzelnes oder mehrere verwaltete Objekte hervorheben.

## Konfiguration

Meistens ist es nicht notwendig, eine dieser Anwendungen direkt zu teilen. Wir haben verschiedene Möglichkeiten für Sie geschaffen, Reiseinventar zu teilen. Entdecken Sie sie in[Wink Studio](/studio/what-is-studio)oder durch[WinkLinks](/link-manager/wink-links).

Wenn Sie jedoch manuell auf eine dieser Websites verlinken möchten, achten Sie darauf, dass Sie Ihre Client-ID und Anpassungs-ID als Parameter zur URL hinzufügen.

Sie finden Ihre Client-ID wie folgt:

<Steps>
  1. Melden Sie sich entweder bei [Wink Extranet](/extranet/what-is-extranet) oder [Wink Studio](/studio/what-is-studio).
  2. Klicken Sie oben rechts auf Ihr Profilsymbol und dann auf [Anwendungen](/developers/applications).
  3. Verwenden Sie die Anwendung, die mit dem Wink-Konto verknüpft ist, mit dem Sie arbeiten möchten.
  4. Kopieren Sie die Client-ID aus dieser Anwendung und fügen Sie sie als Anforderungsparameter in die URL ein. Beispiel: ?client-id=MY\_CLIENT\_ID
</Steps>


### Extranet

Sie können Ihre Anpassungs-ID als Eigenschaft wie folgt finden:

<Steps>
  1. Melden Sie sich an [Wink Extranet](/extranet/what-is-extranet).
  2. Wählen Sie die Eigenschaft aus, mit der Sie arbeiten möchten.
  3. Klicken Sie in der Hauptnavigationsleiste auf `Account > Branding`
  4. Kopieren Sie die Anpassungs-ID und fügen Sie sie als Anforderungsparameter in die URL ein. Beispiel: ?configuration-id=MY\_CUSTOMIZATION\_ID
</Steps>


### Studio

Sie können Ihre Anpassungs-ID als Partner folgendermaßen finden:

<Steps>
  1. Melden Sie sich an [Wink Studio](/studio/what-is-studio).
  2. Wählen Sie das Konto aus, mit dem Sie arbeiten möchten.
  3. Klicken Sie in der Hauptnavigationsleiste auf `Tools > Customization`
  4. Kopieren Sie die Anpassungs-ID der gewünschten Anpassung und fügen Sie sie als Anforderungsparameter in die URL ein. Beispiel: ?configuration-id=MY\_CUSTOMIZATION\_ID
</Steps>


## OTA

Das Ergebnis ist eine URL, die ungefähr so aussieht:

https://ota.wink.travel?client-id=MY\_CLIENT\_ID\&configuration-id=MY\_CUSTOMIZATION\_ID

Wenn Sie direkt zu einer Hotelseite gehen möchten, würde die URL ungefähr so aussehen:

https://ota.wink.travel/hotel/hotel-x?client-id=MY\_CLIENT\_ID\&configuration-id=MY\_CUSTOMIZATION\_ID

:::Hinweis
Wenn Sie keine Client- und/oder Konfigurations-ID angeben, verwendet Wink OTA standardmäßig die Standardkonfiguration.
:::

## IBE

Wenn Sie eine Immobilienverwaltung betreiben, möchten Sie auf die Wink IBE-Homepage verlinken. Die Homepage zeigt Ihr ausgewähltes Thema zusammen mit dem[Eigenschaften, die Sie vertreten](/studio/agency-representation).

Das Ergebnis ist eine URL, die ungefähr so aussieht:

https://book.wink.travel?client-id=MY\_CLIENT\_ID\&configuration-id=MY\_CUSTOMIZATION\_ID

Wenn Sie eine Immobilie besitzen, möchten Sie direkt auf die Wink IBE-Immobilien-Landingpage verlinken.

Diese URL würde ungefähr so aussehen:

https://book.wink.travel/hotel/hotel-x?client-id=MY\_CLIENT\_ID\&configuration-id=MY\_CUSTOMIZATION\_ID

:::Hinweis
Wenn Sie keine Client- und/oder Konfigurations-ID angeben, wird Wink IBE NICHT geladen. Diese Parameter sind erforderlich.
:::

## Beispiel: Eigenschaftenseite

Jede Unterkunft auf Wink erhält auf beiden Websites eine eigene Zielseite. Alle externen Links leiten Reisende auf eine dieser Zielseiten weiter.

Die Zielseite enthält alle Informationen, die ein Reisender benötigt, um eine fundierte Entscheidung darüber zu treffen, ob er in dieser Unterkunft übernachten möchte.
