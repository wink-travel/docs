[Web Components](https://developer.mozilla.org/en-US/docs/Web/API/Web_components) ist ein Standard, der es Ihnen ermöglicht, komplette JavaScript-Funktionen in Ihre Website zu integrieren, ohne programmieren zu müssen.
Dank dieser coolen Technologie können Sie Wink-Reiseinventar mit sehr wenig Aufwand einbetten. Dieser Artikel führt Sie durch unsere Sammlung von Webkomponenten und zeigt Ihnen, wie Sie sie verwenden.

## Wink einbetten

Es gibt drei Dinge, die Sie auf jeder Seite einbinden müssen, auf der Sie eine unserer Webkomponenten anzeigen möchten.

1. Binden Sie unser Stylesheet ein.
2. Binden Sie unser Javascript ein.
3. Binden Sie unseren Application Loader ein.

### Stylesheet

Binden Sie unsere CSS-Styles im Kopfbereich Ihres Dokuments ein.

```html {3}
<html>
  <head>
    <link rel="stylesheet" href="https://elements.wink.travel/styles.css"></link>
  </head>
</html>
```

### Javascript

Binden Sie unser Javascript am Ende Ihres Dokuments ein. (*Wir empfehlen direkt vor dem schließenden &lt;/body&gt;-Tag*).

```html {3}
<html>
  <body>
    <script src="https://elements.wink.travel/elements.js" type="module" defer></script>
  </body>
</html>
```

Ihre Seite ist nun bereit und kann unsere Webkomponenten anzeigen.

## Komponenten

Lesen Sie unten mehr über unsere Komponentenbibliothek.

### Application Loader

Der Loader ist verantwortlich für die Verwaltung des Zustands und die Steuerung der Interaktion zwischen unseren Komponenten. (*Binden Sie ihn unter unserem Javascript ein*).

```html {3-6}
<html>
  <body>
    <wink-app-loader
      client-id="IHRE CLIENT ID HIER EINFÜGEN"
      configuration-id="IHRE ANPASSUNGS-ID HIER EINFÜGEN"
    ></wink-app-loader>
  </body>
</html>
```

### Content Loader

Der Content Loader ist das Herzstück unserer Webkomponenten-Bibliothek. Er ist verantwortlich für die Anzeige Ihres Inventars (Karten, Raster, Kartenansichten).

Verfügbare Attribute:

- layout
- id
- sort

*Das Attribut `sort` ist nur verfügbar, wenn das Layout `RANKED` ist und Sie kein bestehendes Ranked Grid verwenden möchten. In diesen Fällen lassen Sie `id` leer.*

Verfügbare Layout-Typen:

- `AD_BANNER`
- `MAP`
- `HOTEL`
- `GUEST_ROOM`
- `MEETING_ROOM`
- `SPA`
- `RESTAURANT`
- `ACTIVITY`
- `ATTRACTION`
- `PLACE`
- `ADD_ON`
- `LIST`
- `SEARCH`
- `RANKED`

Verfügbare Sortier-Typen:

- `MEMBER`
- `PRICE_LOW_TO_HIGH`
- `PRICE_HIGH_TO_LOW`
- `PRICE`
- `POPULARITY`
- `ECO`
- `EXPERIENCE`
- `PERK`
- `LOYALTY`
- `PACKAGE`

```html {3-6}
<html>
  <body>
    <wink-content-loader
       layout="GUEST_ROOM"
       id="2de7ee9c-61c9-11ef-9722-42004e494300"
    ></wink-content-loader>
  </body>
</html>
```

Beispiele finden Sie für [Karten](/de/studio/cards), [Raster](/de/studio/grids) und [Kartenansichten](/de/studio/maps).

## Komponenten

### Payment

Die Payment-Komponente ermöglicht es Ihnen, TripPay mitzuteilen, dass ein Reisender etwas kaufen möchte, sowie die Regeln und Preise für diese Artikel.

Es gibt ein erforderliches Attribut für das Widget:

- `id` Die Kennung des Buchungsvertrags, den Sie ausführen möchten.

```html {3}
<html>
  <body>
    <trip-pay id="<INSERT_BOOKING_CONTRACT_ID>"></trip-pay>
  </body>
</html>
```

:::note
Um zu erfahren, wie Sie die Buchungsvertrags-`id` generieren, lesen Sie die Anleitung: [Integration mit TripPay](/de/guides/integrating-with-trip-pay).
:::

Das Widget bereitet den Vertrag zur Ausführung vor und zeigt dem Nutzer Zahlungsdetails *(Abbildung 1)* an, um die Buchung abzuschließen.

## Fehler

Wenn Sie beim Einbetten einer unserer Webkomponenten, entweder auf WinkLinks oder auf Ihrer eigenen Website, eine Fehlermeldung erhalten, könnten folgende Dinge schiefgelaufen sein:

### Verfügbarkeit

Das Inventar könnte nicht mehr verfügbar sein. Gehen Sie zu [Wink Studio](/de/studio/cards) und prüfen Sie, ob der Status der "Karte" verfügbar ist.
Wenn nicht, wird sie rot angezeigt. In diesem Fall können Sie warten, ob der Anbieter sie wieder verfügbar macht, oder sie aus Ihrer Liste entfernen.

:::note
`Verfügbarkeit` bedeutet nicht die tägliche Verfügbarkeit des Inventars zur Buchung. Es bedeutet, dass das Inventar für Sie und/oder alle komplett nicht verfügbar ist.
:::

### Entfernt

Das Inventar könnte entfernt worden sein. Gehen Sie zu [Wink Studio](/de/studio/cards) und prüfen Sie, ob der Status der "Karte" verfügbar ist.
Wenn nicht, wird sie rot angezeigt. In diesem Fall können Sie warten, ob der Anbieter sie wieder verfügbar macht, oder sie aus Ihrer Liste entfernen.

### Anpassung

Sie haben möglicherweise versehentlich die Anpassung entfernt, die Sie für Ihr Inventar definiert haben. Stellen Sie sicher, dass die Anpassung noch verfügbar ist, und legen Sie eine neue fest, falls eine fehlt.

### Anwendung

Dies ist nur für Webentwickler relevant. Wenn Sie versehentlich die [Application](/de/developer/applications) entfernt haben und die Client-ID nicht mehr verfügbar ist, erstellen Sie bitte eine neue Anwendung und verwenden Sie die neue Client-ID, um unsere Web Components einzubetten.
