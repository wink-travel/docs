Die Anpassung ist eine leistungsstarke Funktion, mit der Sie jeden Aspekt der Wink-Integration steuern können, ohne Code schreiben zu müssen. Jeder Artikel, den Sie verkaufen, kann individuell angepasst werden. Fügen Sie Funktionen hinzu oder entfernen Sie sie, ändern Sie das Farbschema und vieles mehr.

Während auf der`Customization`Seite, klicken Sie auf das`Actions`Klicken Sie unter der gewünschten Anpassung auf die Schaltfläche und klicken Sie auf`Update`.
*Beim Anlegen Ihres Kontos steht Ihnen bereits ein Standardeintrag zur Verfügung. Dieser kann nicht entfernt werden.*

Dadurch gelangen Sie zum Anpassungsformular, wo Sie 5 - 7 Registerkarten sehen können (*abhängig von Ihrem Kontotyp*).

## Allgemein

Hier sind die Optionen, die Ihnen unter dem`General`aus.

* **Name**Geben Sie Ihrer Anpassung einen leicht zu merkenden Namen.*z.B. Für meinen Reiseblog*
* **Anzeigewährung**Wählen Sie die Anfangswährung aus, die den Benutzern neben dem buchbaren Reisebestand angezeigt wird.*z. B. USD*
* **Anzeigesprache**Wählen Sie die Sprache aus, die den Benutzern als Erstes angezeigt wird.*zB Englisch*
* **An Seitentitel anhängen**Sie können steuern, was dem Seitentitel hinzugefügt wird (*der Titel, den Sie im Browser-Tab sehen*). Dies wird hauptsächlich für unsere Buchungsmaschine verwendet.*Wenn Sie beispielsweise zu<https://ota.wink.travel>, standardmäßig sehen Sie`Home | Traveliko - Hotels with Benefits`*. Sie können alles nach dem Pipe-Zeichen „|“ ändern.

## Thema

Der `Theme`Mit der Registerkarte können Sie das Erscheinungsbild unserer[Buchungsmaschine](/booking-engine/in-a-nutshell)Webseite,[WinkLinks](/link-manager/wink-links)Website und alle unsere[Webkomponenten](/developers/web-components).

### Konfiguration

In diesem Bereich können Sie das Design und das Kartenlayout konfigurieren. Aktuell ist nur ein Design verfügbar: Wink.

Das Kartenlayout steuert, ob Ihren Benutzern horizontal oder vertikal ausgerichtete Transaktionskarten angezeigt werden sollen. Die Informationen sind dieselben.

:::Hinweis
Beim Anzeigen unserer Transaktionskarten auf einem Mobilgerät gibt es keinen Unterschied im Kartenlayout. Auf einem iPad mini und höheren Auflösungen ist der Unterschied jedoch deutlich sichtbar.
:::

:::Tipp \[Profi-Tipp]
Verwenden Sie das horizontale Layout, wenn Ihre Liste nur weniger als vier Reiseangebote (z. B. Zimmertypen) enthält – das sieht einfach besser aus.
:::

### Farben

Die Farbe bestimmt, wie Ihre Benutzer unsere UI-Komponenten und Websites sehen. Passen Sie sie an Ihr vorhandenes Farbschema an (falls vorhanden).

Wir haben bereits eine große Auswahl an vordefinierten Farben, mit denen Sie experimentieren können. Die vordefinierten Designs ändern die Farben in den Formularfeldern unten. Sie können auch ganz eigene Farben verwenden.

## Route

Der `Itinerary`Der Tab ist wahrscheinlich der interessanteste, um sicherzustellen, dass die Preise für das ausgewählte Inventar angezeigt werden.
Da die Preisgestaltung für Reiseinventare recht komplex ist, muss Wink die Reiseroute des Benutzers kennen, bevor ein genauer Preis angezeigt werden kann.

In diesem Abschnitt können Sie die Standardroute festlegen, um die Preise für alle unsere digitalen Immobilien anzuzeigen, beispielsweise:

* Unsere sozialfreundlichen Bilder mit Preisüberlagerung.
* WinkLinks-Raster, -Karten und -Links.
* Webkomponenten
* Buchungsmaschine

### Termine

Sie können wählen, ob Sie einen statischen oder einen gleitenden Datumsbereich festlegen möchten.

* Wählen Sie einen statischen Datumsbereich, wenn Sie die Anpassung für ein bestimmtes Ereignis verwenden, das an einem bestimmten Datum stattfindet.
* Wählen Sie einen gleitenden Datumsbereich, wenn Sie Folgendes möchten:
  1. Standardmäßig ist es heute + 1 Nacht, was die meisten OTAs machen ODER.
  2. Standardmäßig wird ein Datum in der Zukunft verwendet, das immer verfügbar sein sollte.

### Raumkonfiguration

Standardmäßig ist die Reiseroute auf 2 Erwachsene eingestellt, da dies für die meisten Reisenden die gängigste Einstellung ist.
Dies lässt sich leicht ändern, und Sie können mehrere Zimmer mit mehreren Personen (Erwachsenen und Kindern) hinzufügen.

### Aktionen

Aktionen sind eine praktische Funktion, die Ihren Nutzern Zeit spart. Wenn Ihnen ein Anbieter einen Aktionscode zur Verfügung gestellt hat, den Sie mit Ihren Freunden teilen können, können Sie ihn hier eingeben.
Jeder, der über Ihren Link oder ähnliches auf Ihre Website gelangt, erhält den Aktionscode automatisch zu seiner Reiseplanung hinzugefügt und sieht den ermäßigten Preis automatisch.

## Integration

Der `Integration`Die Registerkarte ist für Hotels, Integratoren und Reisebüros gedacht, die mehr Kontrolle über die Verteilung von Buchungen auf Wink haben möchten.

### Benachrichtigungen

Dies gibt dem Hotel und dem Integrator die Möglichkeit, alle Beteiligten über eine Buchung von Wink benachrichtigen zu lassen.
Dazu gehört auch die Benachrichtigung des Channel Managers.

Standardmäßig sind alle diese Optionen aktiviert. Anwendungsfälle, in denen ein Integrator eine oder mehrere dieser Funktionen deaktivieren möchte, sind:

* Das Reisebüro ist auf API-Ebene integriert und möchte eigene Bestätigungs-E-Mails versenden.
* Das Hotel testet die Integration und möchte sein Verkaufspersonal nicht belästigen.
* Das Hotel legt keinen Wert auf E-Mails und möchte nur über sein PMS benachrichtigt werden.

### E-Mail-Logo

Integratoren, die unseren E-Mail-Benachrichtigungsdienst nutzen möchten, können die E-Mail so anpassen, dass es so aussieht, als käme sie von ihnen.

:::Hinweis
Empfohlene Abmessungen: 300 Pixel Breite x 75 Pixel Höhe
:::

## Hotelspezifisch

Unter dem`Hotel-specific`Die Registerkarte „Hotel“ bietet Optionen, mit denen das Hotel steuern kann, was auf der Zielseite unserer Buchungsmaschine angezeigt wird und was angezeigt werden soll, wenn für die angeforderte Reiseroute kein Bestand vorhanden ist.

* **Ranglisten**: Eine Immobilie, die noch keine aussagekräftigen Rankings erreicht hat, kann sich dafür entscheiden, diese Funktion zu deaktivieren.

:::Hinweis
Die Unterkunft hat keinen Einfluss darauf, wie ein Partner die Landingpage des Hotels anzeigt. Dies sind zwei verschiedene Konfigurationen.
:::

* **Verfügbarkeit**: Ermöglicht der Unterkunft die Auswahl, ob der Bestand auch dann weiterhin angezeigt werden soll, wenn keine Verfügbarkeit besteht.

:::Hinweis
In den meisten Fällen sollte die Eigenschaft den Nutzern die Möglichkeit geben, auf das`Book`Klicken Sie auf die Schaltfläche, um zur Zielseite zu gelangen. Dort können Sie Ihre Reiseroute ändern und versuchen, ein verfügbares Datum zu finden.
Aktivieren Sie diesen Schalter, wenn Sie den Benutzer daran hindern möchten, fortzufahren.
:::

## Anwendung

In den meisten Fällen wählen Sie, welche`Customization`zum Anwenden auf Ihre Links, Karten, Übersichten und Raster.

In den folgenden Fällen möchten Sie Ihre Konfiguration möglicherweise manuell anwenden:

* **Manuelle Übersteuerung**Sie möchten einen Link zu unserer Buchungsmaschine direkt teilen und nicht unsere[Teilbare Links](/studio/shareable-links)Funktion, mit Ihrer benutzerdefinierten Konfiguration.
* **WordPress-Plugin**Fügen Sie die Konfiguration zu Ihrem Wink hinzu[WordPress-Plugin](/developers/wordpress) `Settings`.
* **Benutzerdefinierte Integration**Wenden Sie es manuell auf Ihren benutzerdefinierten Integrator an, indem Sie unsere[Wink-App-Loader](/developers/web-components/#application-loader)Webkomponente.

### Manuelle Übersteuerung

Hier ist ein einfaches Beispiel, wie Sie Ihre Konfigurations-ID mit all Ihren Einstellungen auf die URLs unserer Hauptbuchungsmaschine anwenden.

<h4 align="center">[https://ota.wink.travel?configurationId=MEINE EINDEUTIGE ID](https://ota.wink.travel?configurationId=MY-UNIQUE-ID)</h4>


:::Hinweis
Wenn ein Nutzer auf diesen Link klickt, wird auch Ihr Affiliate-Attributions-Cookie gesetzt.
:::

### WordPress-Plugin

Navigieren Sie zum[Wink WordPress-Plugin](/developers/wordpress)um mehr zu erfahren.

### Benutzerdefinierte Integration

Steuern Sie alle eingebetteten Wink-Webkomponenten auf einer Seite mit dem[Wink-App-Loader](/developers/web-components/#application-loader)Webkomponente.

Unten sehen Sie ein Beispiel, das`MY-UNIQUE-ID`als Konfigurations-ID.

```html {11}
<html>
  <head>
    <link rel="stylesheet" href="https://elements.wink.travel/styles.css"></link>
  </head>
  <body>
    
    <script src="https://elements.wink.travel/elements.js" type="module" defer></script>

    <wink-app-loader
      client-id="YOUR CLIENT ID GOES HERE"
      configuration-id="MY-UNIQUE-ID"
    ></wink-app-loader>
  </body>
</html>
```
