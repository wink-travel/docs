Nicht jeder Tag wird gleich bepreist. Veranstaltungen, Feiertage, Nebensaison und Last-Minute-Lücken erfordern unterschiedliche Preise. Diese Anleitung zeigt Ihnen, wie Sie Sondertarife und Einschränkungen von Tarifplänen nutzen, um diese Szenarien zu bewältigen.

## Veranstaltungs-Preise

Wenn eine Konferenz, ein Festival oder eine lokale Veranstaltung die Nachfrage steigert, möchten Sie die Preise für diese Daten erhöhen.

<Steps>
1. Gehen Sie in Ihrem Extranet zu `Monetize > Special Rates`.
2. Klicken Sie auf `Create a new special rate`.
3. Wählen Sie den Master-Tarif und den Zimmertyp, die überschrieben werden sollen.
4. Legen Sie den Datumsbereich fest, der den Veranstaltungszeitraum abdeckt.
5. Geben Sie die angepassten (höheren) Tagespreise ein.
6. Klicken Sie auf `Save`.
</Steps>

Der Sondertarif überschreibt den Master-Tarif nur für diese Daten. Ihre regulären Preise bleiben außerhalb des Veranstaltungszeitraums unverändert.

## Saisonale Anpassungen

Für längere Zeiträume (Hauptsaison, Zwischensaison, Nebensaison) können Sie entweder:

- **Sondertarife verwenden** — Erstellen Sie Überschreibungen für jede Saison mit angepassten Preisen.
- **Master-Tarife aktualisieren** — Passen Sie die Basistarife direkt für diese Zeiträume an.

Sondertarife eignen sich besser für temporäre Anpassungen, während die Aktualisierung der Master-Tarife für vorhersehbare jährliche Muster sinnvoll ist.

## Last-Minute-Tarife

Um unverkaufte Zimmer kurz vor dem Check-in-Datum zu füllen:

<Steps>
1. Erstellen Sie einen neuen **Tarifplan** mit einer Vorausbuchungsbeschränkung (z. B. Buchung darf nicht mehr als 3 Tage im Voraus erfolgen).
2. Kombinieren Sie ihn mit einem **Sondertarif**, der für die Zieltermine einen Rabatt bietet.
3. Dieser Tarifplan wird nur Reisenden angezeigt, die innerhalb des Last-Minute-Zeitraums buchen.
</Steps>

## Sperrzeiten

Um bestimmte Daten für einen bestimmten Tarifplan zu blockieren (z. B. keine Aktionspreise während der Hauptsaison):

<Steps>
1. Gehen Sie zu `Monetize > Rate Plans` und wählen Sie den Tarifplan aus.
2. Navigieren Sie zum Abschnitt **Restrictions**.
3. Fügen Sie eine **Sell date**-Einschränkung hinzu, um zu begrenzen, wann der Tarif buchbar ist.
4. Oder verwenden Sie **Stay date**-Einschränkungen, um zu begrenzen, für welche Aufenthaltsdaten der Tarif gilt.
</Steps>

Beispielsweise können Sie Ihren "Early Bird"-Tarifplan so einschränken, dass er im Dezember (Hauptsaison) nicht buchbar ist, während Ihr regulärer Tarifplan offen bleibt.

## Kombination von Einschränkungen

Einschränkungen können kombiniert werden, um eine präzise Steuerung zu ermöglichen:

| Einschränkung | Was sie steuert |
| -- | -- |
| **Sell dates** | Wann der Tarif gebucht werden kann |
| **Stay dates** | Für welche Aufenthaltsdaten der Tarif gilt |
| **Available days** | An welchen Wochentagen der Tarif angezeigt wird |
| **Arrival days** | An welchen Tagen der Gast anreisen muss |
| **Departure days** | An welchen Tagen der Gast abreisen muss |
| **Required days** | Der Gast muss mindestens an einem dieser Tage bleiben |
| **Advance booking** | Wie weit im Voraus die Buchung erfolgen muss |
| **Length of stay** | Mindest- und Höchstdauer des Aufenthalts |
