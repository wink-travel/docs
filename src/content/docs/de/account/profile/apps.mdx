Der **Apps**-Tab unter [iam.wink.travel/my-account/apps](https://iam.wink.travel/my-account/apps) listet jede OAuth2-Anwendung auf, der Sie Zugriff auf Ihr Wink-Konto gewährt haben, mit einer Möglichkeit, diesen Zugriff zu widerrufen.

## Wie eine App hierher gelangt

Wenn Sie auf `Authorize` bei einem Drittanbieter-Tool klicken, das mit Wink integriert ist (oder einer der eigenen Apps von Wink), wird der Zustimmungsprozess die Erlaubnis speichern. Von da an kann diese App in Ihrem Namen innerhalb der genehmigten Bereiche handeln.

## Zugriff überprüfen

Jeder Eintrag zeigt:

- **App-Name** und eine kurze Beschreibung.
- **Scopes**, auf die die App zugreifen kann (z. B. Inventar lesen, Buchungen verwalten).
- **Zuletzt verwendet** — wann die App zuletzt die Wink-API in Ihrem Namen aufgerufen hat.

## Eine App widerrufen

Klicken Sie neben einem Eintrag auf `Revoke`. Die Tokens der App werden sofort ungültig, und alle Hintergrundprozesse, die sie für Ihr Konto ausführt, schlagen mit 401-Fehlern fehl.

:::tip
Der Widerruf ist nicht destruktiv — Sie können die App jederzeit erneut autorisieren, indem Sie den Zustimmungsprozess erneut durchlaufen. Verwenden Sie dies, wenn eine App sich fehlerhaft verhält, wenn Sie Zugangsdaten rotieren oder wenn Sie die Integration nicht mehr nutzen.
:::

## OAuth-Apps, die Sie selbst erstellen

Wenn Sie eine OAuth-App *erstellen* möchten (nicht nur eine autorisieren), ist das eine Funktion für Organisationskonten — siehe [Einstellungen → Anwendungen](/de/portal/settings/applications).
