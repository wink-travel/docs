Anpassung ist eine leistungsstarke Funktion, mit der Sie jeden Aspekt der Integration mit Wink steuern können, ohne Code schreiben zu müssen. Jedes Inventar, das Sie verkaufen, kann individuell angepasst werden. Fügen Sie Funktionen hinzu oder entfernen Sie sie, ändern Sie das Farbschema und vieles mehr.

So starten Sie:

<Steps>
 1. Wählen Sie das Konto aus, mit dem Sie arbeiten möchten, unter [Accounts](https://my.wink.travel).
 2. Klicken Sie unten auf der Kontokarte auf `Actions` und wählen Sie `Manage account`.
 3. Klicken Sie auf der folgenden Seite auf den Tab `Customizations`.
 4. In diesem Abschnitt werden alle Ihre Anpassungen aufgelistet.
 5. Wählen Sie eine bestehende Anpassung zur Bearbeitung aus oder erstellen Sie eine neue.
 </Steps>

Auf der Seite `Customization` klicken Sie auf die Schaltfläche `Actions` unter der Anpassung, mit der Sie arbeiten möchten, und dann auf `Update`.

:::note
Sie haben bereits einen Standardeintrag, der bei der Kontoerstellung verfügbar ist. Dieser Eintrag kann nicht entfernt werden.
:::

Dies führt Sie zum Anpassungsformular, in dem Ihnen die folgenden Funktionen zur Verfügung stehen.

## Allgemein

Hier sind die Optionen, die Ihnen unter dem Tab `General` zur Verfügung stehen.

- **Name** Geben Sie Ihrer Anpassung einen Namen, um sie leicht wiederzuerkennen. *z.B. Für meinen Reiseblog*
- **Anzeigewährung** Wählen Sie die anfängliche Währung aus, die Benutzer neben buchbarem Reiseinventar sehen. *z.B. USD*
- **Anzeigesprache** Wählen Sie die anfängliche Sprache, die Benutzer vorfinden. *z.B. Englisch*
- **An Seitentitel anhängen** Sie können steuern, was dem Seitentitel (*dem Titel, den Sie im Browser-Tab sehen*) hinzugefügt wird. Dies wird hauptsächlich für unsere Buchungsmaschine verwendet. *z.B. Wenn Sie https://ota.wink.travel aufrufen, sehen Sie standardmäßig `Home | Traveliko - Hotels with Benefits`*. Sie können alles nach dem '|' Pipe-Zeichen ändern.

## Design

Der Tab `Theme` ermöglicht es Ihnen, das Aussehen und Gefühl unserer [Buchungsmaschine](/de/booking-engine/in-a-nutshell) Website, der [WinkLinks](/de/link-manager/wink-links) Website und aller unserer [Web Components](/de/developers/web-components) anzupassen.

### Konfiguration

Dieser Abschnitt ermöglicht es Ihnen, das Design und das Kartenlayout zu konfigurieren. Derzeit gibt es nur ein Design, nämlich Wink.

Das Kartenlayout steuert, ob Ihre Nutzer horizontal oder vertikal ausgerichtete Transaktionskarten sehen. Die Informationen sind dieselben.

:::note
Es gibt keinen Unterschied im Kartenlayout, wenn unsere Transaktionskarten auf einem mobilen Gerät angezeigt werden. Den Unterschied sehen Sie jedoch auf einem iPad mini und größeren Auflösungen.
:::

:::tip[Profi-Tipp]
Verwenden Sie das horizontale Layout, wenn Sie weniger als 4 Reiseinventare (z.B. Zimmertypen) in einer Liste haben... es sieht einfach besser aus.
:::

Farben steuern, wie Ihre Nutzer unsere UI-Komponenten und Websites sehen. Verwenden Sie sie, um Ihr bestehendes Farbschema [falls vorhanden] anzupassen.

## Analytics

Wir unterstützen GA4-Analytics standardmäßig. Wenn Sie Ihren gesamten erweiterten E-Commerce-Konversionspfad von Anfang bis Ende verfolgen möchten, halten Sie Ihre Mess-ID bereit und geben Sie sie im Feld unter dem Tab `Analytics` ein.

<Aside type="caution" title="Erinnerung">
  Vergessen Sie nicht, `i.trvl.as`, `book.wink.travel` und `ota.wink.travel` zu Ihren GA4-Property-Streams hinzuzufügen.
</Aside>

Um unsere Karten in Ihre Website einzubetten, entweder mit unserem [WordPress-Plugin](/de/developers/wordpress) oder [Web Components](/de/developers/web-components), müssen Sie uns Ihren eigenen Google Maps API-Schlüssel bereitstellen.

:::note
  [Erstellen Sie hier einen Google Maps API-Schlüssel](https://console.developers.google.com/projectselector2/apis/credentials?supportedpurview=project)

Nachdem Sie Ihren Schlüssel generiert und für Ihre Website konfiguriert haben, geben Sie ihn unter dem Tab `Analytics` ein.

## Reiseroute

Der Tab `Itinerary` ist wahrscheinlich der interessanteste, um sicherzustellen, dass Preise für das von Ihnen ausgewählte Inventar angezeigt werden.
Da die Preisgestaltung von Reiseinventar recht komplex ist, muss Wink die Reiseroute des Nutzers kennen, bevor ein genauer Preis angezeigt werden kann.

Dieser Abschnitt ermöglicht es Ihnen, die Standard-Reiseroute festzulegen, um Preise für alle unsere digitalen Flächen anzuzeigen, wie z.B.:

- Unsere social-freundlichen Bilder mit Preisüberlagerung.
- WinkLinks-Gitter, Karten und Links.
- Web Components
- Buchungsmaschine

### Daten

Sie können wählen, ob Sie einen statischen oder einen rollierenden Datumsbereich festlegen möchten.

- Wählen Sie einen statischen Datumsbereich, wenn Sie die Anpassung für ein bestimmtes Ereignis verwenden, das an einem bestimmten Datum stattfindet.
- Wählen Sie einen rollierenden Datumsbereich, wenn Sie entweder:
    1. Standardmäßig heute + 1 Nacht verwenden möchten, wie es die meisten OTAs tun ODER.
    2. Standardmäßig ein zukünftiges Datum wählen möchten, das immer Verfügbarkeit haben sollte.

### Zimmerkonfiguration

Standardmäßig ist die Reiseroute auf 2 Erwachsene eingestellt, da dies die häufigste Einstellung für die meisten Reisenden ist.
Dies kann leicht geändert werden, und Sie können mehrere Zimmer mit mehreren Personen hinzufügen; sowohl Erwachsene als auch Kinder.

### Aktionen

Aktionen sind eine praktische Funktion, die Ihren Nutzern Zeit spart. Wenn eine Unterkunft Ihnen einen Promo-Code gegeben hat, den Sie mit Ihren Freunden teilen können, können Sie ihn hier eingeben.
Jeder, der über Ihren Link oder Ähnliches kommt, erhält den Promo-Code automatisch in seiner Reiseroute hinzugefügt und sieht den rabattierten Preis automatisch angewendet.

## Integration

Der Tab `Integration` richtet sich an Hotels, Integratoren und Reisebüros, die mehr Kontrolle darüber wünschen, wie Buchungen auf Wink verteilt werden.

### Benachrichtigungen

Dies gibt dem Hotel und Integrator die Möglichkeit, Wink alle beteiligten Parteien benachrichtigen zu lassen, wenn eine Buchung erfolgt.
Dies schließt die Benachrichtigung des Channel Managers ein.

Standardmäßig sind diese Optionen alle aktiviert. Anwendungsfälle, in denen ein Integrator eine oder mehrere dieser Funktionen deaktivieren möchte, sind:

- Das Reisebüro ist auf API-Ebene integriert und möchte eigene Bestätigungs-E-Mails senden.
- Das Hotel testet seine Integration und möchte sein Verkaufspersonal nicht stören.
- Das Hotel interessiert sich nicht für E-Mails und möchte nur über sein PMS benachrichtigt werden.

### E-Mail-Logo

Für Integratoren, die unseren E-Mail-Benachrichtigungsdienst nutzen möchten, können sie die E-Mail so anpassen, dass sie aussieht, als käme sie von ihnen.

:::note
  Empfohlene Abmessungen sind 300px breit x 75px hoch
:::

## Hotelspezifisch

Unter dem Tab `Hotel-specific` befinden sich Optionen, mit denen das Hotel steuern kann, was auf seiner Landingpage in unserer Buchungsmaschine angezeigt wird und was angezeigt wird, wenn für die angefragte Reiseroute kein Inventar verfügbar ist.

- **Rankings**: Eine Unterkunft, die noch keine aussagekräftigen Rankings erreicht hat, könnte diese Funktion deaktivieren.

:::note
  Die Unterkunft kann nicht steuern, wie ein Affiliate die Hotel-Landingpage anzeigt... Das sind zwei verschiedene Konfigurationen.
:::

- **Verfügbarkeit**: Ermöglicht der Unterkunft zu wählen, ob das Inventar weiterhin angezeigt wird, auch wenn keine Verfügbarkeit besteht.

:::note
  In den meisten Fällen möchte die Unterkunft den Nutzern ermöglichen, auf die Schaltfläche `Buchen` zu klicken, um auf ihre Landingpage zu gelangen, wo sie ihre Reiseroute ändern und nach einem verfügbaren Datum suchen können.
  Aktivieren Sie diesen Schalter, wenn Sie den Nutzer am Weiterklicken hindern möchten.
:::

## Verwendung

In den meisten Szenarien wählen Sie aus, welche `Customization` auf Ihre Links, Karten, Kartenraster und Karten angewendet wird.

Hier sind die Fälle, in denen Sie Ihre Konfiguration manuell anwenden möchten:

- **Manuelle Überschreibung** Sie möchten einen Link zu unserer Buchungsmaschine direkt teilen und nicht unsere Funktion [Shareable Links](/de/studio/shareable-links) mit Ihrer benutzerdefinierten Konfiguration verwenden.
- **WordPress-Plugin** Fügen Sie die Konfiguration in den `Settings` Ihres Wink [WordPress Plugins](/de/developers/wordpress) hinzu.
- **Benutzerdefinierte Integration** Wenden Sie sie manuell mit unserem [wink-app-loader](/de/developers/web-components/#application-loader) Web Component auf Ihren benutzerdefinierten Integrator an.

### Manuelle Überschreibung

Hier ein einfaches Beispiel, wie Sie Ihre Konfigurations-ID mit all Ihren Einstellungen auf unsere Haupt-Buchungsmaschinen-URLs anwenden.

<h4 align="center"> [https://ota.wink.travel?configurationId=MY-UNIQUE-ID](https://ota.wink.travel?configurationId=MY-UNIQUE-ID) </h4>

:::note
  Wenn ein Nutzer auf diesen Link klickt, wird auch Ihr Affiliate-Attributions-Cookie gesetzt.
:::

### WordPress-Plugin

Besuchen Sie das [Wink WordPress Plugin](/de/developers/wordpress), um mehr zu erfahren.

### Benutzerdefinierte Integration

Steuern Sie alle Ihre eingebetteten Wink Web Components auf einer Seite mit dem [wink-app-loader](/de/developers/web-components/#application-loader) Web Component.

Unten sehen Sie ein Beispiel, das `MY-UNIQUE-ID` als Konfigurations-ID verwendet.

```html {11}
<html>
  <head>
    <link rel="stylesheet" href="https://elements.wink.travel/styles.css"></link>
  </head>
  <body>

    <script src="https://elements.wink.travel/elements.js" type="module" defer></script>

    <wink-app-loader
      client-id="YOUR CLIENT ID GOES HERE"
      configuration-id="MY-UNIQUE-ID"
    ></wink-app-loader>
  </body>
</html>
```
