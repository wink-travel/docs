:::note
Ähnlich wie in der Fluggesellschaftsbranche benötigen Hotels viel Flexibilität bei Preisen und Rabatten.
Aktionen sind eine Möglichkeit für das Hotel, einzigartige Bedingungen zu schaffen, unter denen ein Reisender für einen zusätzlichen Preisnachlass [oder Aufschlag] berechtigt ist.
:::

Um Ihre Aktionen zu verwalten, klicken Sie in der Hauptnavigationsleiste auf `Monetize > Single Promotions`.

## Konfiguration

Dieser Abschnitt ist dafür zuständig, den tatsächlichen Geldwert der Aktion festzulegen.

- **Name** Geben Sie einen Namen für diese Aktion ein, den nur Sie sehen können.
- **Typ** Wählen Sie, ob diese Aktion einen Rabatt oder einen Aufschlag enthalten soll.
- **Preistyp** Wählen Sie, wie dieses Paket berechnet werden soll.
    - `Per stay`: Betrag wird auf die letzte Nacht angewendet. *z.B. 100 % Rabatt > Bleiben Sie 2 Nächte – die 3. Nacht ist kostenlos.*
    - `Per night`: Betrag multipliziert mit der Anzahl der Nächte.
    - `Per use`: Es erfolgt keine zusätzliche Berechnung.
    - `Per person`: Betrag multipliziert mit der Anzahl der Gäste.
    - `Per person per night`: Betrag multipliziert mit der Anzahl der Nächte + Anzahl der Gäste.
- **Betragstyp** Wählen Sie, ob der Betragsmodifikator ein fester Betrag oder ein Prozentsatz sein soll.
- **Status** Schalten Sie den Schalter um, um zu steuern, ob die Aktion über alle Vertriebskanäle verfügbar ist.

## Beschreibungen

:::note
Sie müssen mindestens eine Beschreibung schreiben, und diese muss in englischer Sprache sein.
:::

Klicken Sie auf den Tab `Descriptions`, um fortzufahren.

Eine lokalisierte Beschreibung erklärt dem Reisenden in seiner Sprache, wofür diese Aktion gedacht ist.
Sie können in so vielen Sprachen schreiben, wie Sie möchten. Wir fügen Übersetzungen für alle beliebtesten Sprachen hinzu.
Dies ist der Text, den Reisende sehen, wenn sie Ihre Aktion betrachten.

- **Beschreibung** Geben Sie der Aktion einen Namen. *z.B. Early bird - 20% OFF*

## Einschränkungen

:::note
Beachten Sie, dass dies Einschränkungen sind... Aktivieren Sie diese nur, wenn Sie Ihre Aktion weniger zugänglich machen möchten.
:::

Klicken Sie auf den Tab `Restrictions`, um fortzufahren.

Die Aktions-Engine von Wink bietet nicht weniger als 20 einzigartige Möglichkeiten, eine Preisänderung auszulösen.

Wir unterstützen die folgenden Kriterien:

- Aktion auf bestimmte Zusatzleistungen beschränken.
- Aktion auf bestimmte Master-Raten beschränken.
- Aktion auf bestimmte Tarifpläne beschränken.
- Aktion auf bestimmte Stornierungsbedingungen beschränken.
- Aktion auf Aufenthaltsdauer beschränken.
- Aktion auf Vorausbuchung beschränken.
- Aktion auf Anzahl der Zimmer beschränken.
- Aktion auf Aktionscode beschränken.
- Aktion auf IP-Bereich des Reisenden beschränken.
- Aktion auf Last Minute beschränken.
- Aktion auf Stadt des Reisenden beschränken.
- Aktion auf Land des Reisenden beschränken.
- Aktion auf Kontinent des Reisenden beschränken.
- Aktion auf Zeitzone des Reisenden beschränken.
- Aktion auf Verkaufsdatum(e) beschränken.
- Aktion auf Aufenthaltsdatum(e) beschränken.
- Aktion auf Buchungstage der Woche beschränken.
- Aktion auf Ankunftstage der Woche beschränken.
- Aktion auf Abreisetage der Woche beschränken.
- Aktion auf erforderliche Wochentage beschränken.

Lesen Sie weiter, um zu erfahren, wie Sie jede einzelne verwenden.

### Zusatzleistung

:::note
Um diese Funktion zu nutzen, müssen Sie mindestens eine Zusatzleistung erstellt haben.
:::

Damit eine Aktion nur auf die von Ihnen ausgewählten Zusatzleistungen angewendet wird, gehen Sie wie folgt vor:

<Steps>
1. Aktivieren Sie den Schalter `Add-on restriction`.
2. Aktivieren Sie das Kontrollkästchen neben jeder Zusatzleistung, die Sie einschließen möchten.
</Steps>

### Master-Rate

:::note
Um diese Funktion zu nutzen, müssen Sie mindestens eine Master-Rate erstellt haben.
:::

Damit eine Aktion nur auf die von Ihnen ausgewählten Master-Raten angewendet wird, gehen Sie wie folgt vor:

<Steps>
1. Aktivieren Sie den Schalter `Master rate restriction`.
2. Aktivieren Sie das Kontrollkästchen neben jeder Master-Rate, die Sie einschließen möchten.
</Steps>

### Tarifplan

:::note
Um diese Funktion zu nutzen, müssen Sie mindestens einen Tarifplan erstellt haben.
:::

Damit eine Aktion nur auf die von Ihnen ausgewählten Tarifpläne angewendet wird, gehen Sie wie folgt vor:

<Steps>
1. Aktivieren Sie den Schalter `Rate plan restriction`.
2. Aktivieren Sie das Kontrollkästchen neben jedem Tarifplan, den Sie einschließen möchten.
</Steps>

### Erstattbar

:::note
Um diese Funktion zu nutzen, müssen Sie mindestens eine Stornierungsbedingung erstellt haben.
:::

Damit eine Aktion nur auf die von Ihnen ausgewählten Stornierungsbedingungen angewendet wird, gehen Sie wie folgt vor:

<Steps>
1. Aktivieren Sie den Schalter `Cancellation policy restriction`.
2. Aktivieren Sie das Kontrollkästchen neben jeder Stornierungsbedingung, die Sie einschließen möchten.
</Steps>

### Aufenthaltsdauer

:::note
Diese Einschränkung wird normalerweise für Ihre „Langzeit“-Aktion verwendet.
:::

Wenn Sie diese Einschränkung hinzufügen, verlangen Sie von Gästen, dass sie mindestens und höchstens eine bestimmte Anzahl von Tagen bleiben.

So aktivieren Sie die Einschränkung `Length of Stay`:

<Steps>
1. Aktivieren Sie den Schalter `Length of stay restriction`.
2. Geben Sie die Mindestanzahl der Tage ein, die ein Gast bleiben muss. *z.B. 7*
3. Optional geben Sie die maximale Anzahl der Tage ein, die ein Gast bleiben darf. *z.B. 90*
</Steps>

*Beispiel: Diese Aktion ist nur für Gäste verfügbar, die länger als 10 Tage bleiben. Hinweis: Die maximale Aufenthaltsdauer ist optional.*

### Vorausbuchung

:::note
Diese Einschränkung wird normalerweise für Ihre „Early Bird“-Aktion verwendet.
:::

Wenn Sie diese Einschränkung hinzufügen, verlangen Sie von Gästen, dass sie mindestens und höchstens eine bestimmte Anzahl von Tagen im Voraus buchen.

So aktivieren Sie die Einschränkung `Advance booking`:

<Steps>
1. Aktivieren Sie den Schalter `Advance booking restriction`.
2. Geben Sie die Mindestanzahl der Tage ein, die ein Gast im Voraus buchen muss. *z.B. 10*
3. Optional geben Sie die maximale Anzahl der Tage ein, die ein Gast im Voraus buchen darf. *z.B. 180*
</Steps>

*Beispiel: Diese Aktion ist nur für Gäste verfügbar, die mehr als 10 Tage im Voraus buchen. Hinweis: Wir verlangen auch, dass Sie eine maximale Vorausbuchung angeben.*

### Anzahl der Zimmer

:::note
Die Aktion ist nur verfügbar, wenn der Gast die Mindestanzahl an Zimmern bucht.
:::

Wenn Sie diese Einschränkung hinzufügen, verlangen Sie von Gästen, dass sie mindestens und höchstens eine bestimmte Anzahl von Zimmern im Rahmen ihrer Buchung reservieren.

So aktivieren Sie die Einschränkung `Number of rooms`:

<Steps>
1. Aktivieren Sie den Schalter `Room restriction`.
2. Geben Sie die Mindestanzahl der Zimmer ein, die ein Gast buchen muss. *z.B. 3*
3. Geben Sie die maximale Anzahl der Zimmer ein, die ein Gast buchen darf. *z.B. 8*
</Steps>

*Beispiel: Diese Aktion ist nur für Gäste verfügbar, die mehr als 3 Zimmer buchen.*

### Aktionscode

:::note
Verwenden Sie diese Einschränkung, wenn der Gast einen Rabattcode eingeben muss, um diese Aktion freizuschalten.
:::

So aktivieren Sie die Einschränkung `Promo code`:

<Steps>
1. Aktivieren Sie den Schalter `Promotion restriction`.
2. Klicken Sie auf die Schaltfläche `Add promo code`.
3. Geben Sie den Aktionscode ein. *z.B. ABC123*
</Steps>

*Beispiel: Diese Aktion ist nur für Gäste verfügbar, die den Aktionscode `ABC123` eingeben.*

### IP-Bereich

:::note
Dies ist eine gute Einschränkung, wenn Sie eine Cross-Promotion-Kampagne mit Mobilfunkanbietern oder größeren Unternehmen durchführen möchten, die eigene IP-Bereiche haben.
:::

So aktivieren Sie die Einschränkung `IP range`:

<Steps>
1. Aktivieren Sie den Schalter `IP range restriction`.
2. Klicken Sie auf die Schaltfläche `Add IP range restriction`.
3. Geben Sie den Start-IP-Bereich [weltweit zugänglich] ein. *z.B. 203.0.113.50*
4. Geben Sie den End-IP-Bereich [weltweit zugänglich] ein. *z.B. 203.0.113.59*
</Steps>

*Beispiel: Diese Aktion ist nur für Gäste verfügbar, deren IP-Nummern zwischen `203.0.113.50` und `203.0.113.59` liegen.*

### Last Minute

:::note
Dies ist der Zeitraum des Last-Minute-Angebots, den Sie einschränken möchten.
:::

So aktivieren Sie die Einschränkung `Last minute`:

<Steps>
1. Aktivieren Sie den Schalter `Last minute restriction`.
2. Klicken Sie auf die Schaltfläche `Add IP range restriction`.
3. Geben Sie die Anzahl der Stunden vor dem Ankunftsdatum ein. *z.B. 24 Stunden*
</Steps>

*Beispiel: Diese Aktion ist nur für Gäste verfügbar, die 24 Stunden vor dem Ankunftsdatum buchen.*

### Stadt

:::note
Ermöglicht die Einschränkung auf die Stadt, in der sich der Reisende aktuell befindet. Erstellen Sie für jede Stadt, die Sie einschließen möchten, eine Einschränkung.
:::

So aktivieren Sie die Einschränkung `City`:

<Steps>
1. Aktivieren Sie den Schalter `City restriction`.
2. Beginnen Sie, den Namen der Stadt einzugeben.
3. Wählen Sie die Stadt aus den Suchergebnissen aus. *z.B. New York City*
</Steps>

*Beispiel: Diese Aktion ist nur für Gäste verfügbar, die aus `New York City` buchen.*

### Land

:::note
Ermöglicht die Einschränkung auf das Land, in dem sich der Reisende aktuell befindet. Erstellen Sie für jedes Land, das Sie einschließen möchten, eine Einschränkung.
:::

So aktivieren Sie die Einschränkung `Country`:

<Steps>
1. Aktivieren Sie den Schalter `Country restriction`.
2. Beginnen Sie, den Namen des Landes einzugeben.
3. Wählen Sie das Land aus den Suchergebnissen aus. *z.B. Vereinigte Staaten von Amerika*
</Steps>

*Beispiel: Diese Aktion ist nur für Gäste verfügbar, die aus den `Vereinigten Staaten von Amerika` buchen.*

### Kontinent

:::note
Ermöglicht die Einschränkung des Tarifs auf den Kontinent, auf dem sich der Gast aktuell befindet. Erstellen Sie für jeden Kontinent, den Sie einschließen möchten, eine Einschränkung.
:::

So aktivieren Sie die Einschränkung `Continent`:

<Steps>
1. Aktivieren Sie den Schalter `Continent restriction`.
2. Beginnen Sie, den Namen des Kontinents einzugeben.
3. Wählen Sie den Kontinent aus den Suchergebnissen aus. *z.B. Nordamerika*
</Steps>

*Beispiel: Diese Aktion ist nur für Gäste verfügbar, die aus `Nordamerika` buchen.*

### Zeitzone

:::note
Ermöglicht die Einschränkung des Tarifs auf die Zeitzone, in der sich der Gast aktuell befindet. Erstellen Sie für jede Zeitzone, die Sie einschließen möchten, eine Einschränkung.
:::

So aktivieren Sie die Einschränkung `Timezone`:

<Steps>
1. Aktivieren Sie den Schalter `Timezone restriction`.
2. Beginnen Sie, den Namen der Zeitzone einzugeben.
3. Wählen Sie die Zeitzone aus den Suchergebnissen aus. *z.B. America/New_York*
</Steps>

*Beispiel: Diese Aktion ist nur für Gäste verfügbar, die aus `America/New_York` buchen.*

### Verkaufsdatum

:::note
Ermöglicht die Einschränkung auf einen bestimmten Datumsbereich. Ein Reisender ist berechtigt, wenn er ein Zimmer an den innerhalb dieses Datumsbereichs angegebenen Daten bucht.
:::

So aktivieren Sie die Einschränkung `Sell date`:

<Steps>
1. Aktivieren Sie den Schalter `Sell date restriction`.
2. Klicken Sie auf die Schaltfläche `Add sell date restriction`.
3. Geben Sie das Startdatum ein. *z.B. 19. Sept. 2024*
4. Geben Sie das Enddatum ein. *z.B. 1. Dez. 2024*
</Steps>

*Beispiel: Diese Aktion ist nur für Gäste verfügbar, die zwischen dem `19. Sept. 2024` und dem `1. Dez. 2024` buchen.*

### Aufenthaltsdatum

:::note
Ermöglicht die Einschränkung auf einen bestimmten Datumsbereich. Ein Reisender ist berechtigt, wenn er ein Zimmer für die innerhalb dieses Datumsbereichs angegebenen Daten bucht.
:::

So aktivieren Sie die Einschränkung `Stay date`:

<Steps>
1. Aktivieren Sie den Schalter `Stay date restriction`.
2. Klicken Sie auf die Schaltfläche `Add stay date restriction`.
3. Geben Sie das Startdatum ein. *z.B. 19. Sept. 2024*
4. Geben Sie das Enddatum ein. *z.B. 1. Dez. 2024*
</Steps>

*Beispiel: Diese Aktion ist nur für Gäste verfügbar, die einen Aufenthalt zwischen dem `19. Sept. 2024` und dem `1. Dez. 2024` buchen.*

### Buchungstage der Woche

:::note
Wenn Sie diese Einschränkung aktivieren, ist dieser Tarif nur an den angegebenen Wochentagen verfügbar, an denen die Buchung erfolgt.

Diese Einschränkung passt gut zur „Sell date restriction“, bei der Sie einen Datumsbereich einschränken und diesen mit Wochentagen weiter begrenzen können.
:::

So aktivieren Sie die Einschränkung `Booking days`:

<Steps>
1. Aktivieren Sie den Schalter `Booking days restriction`.
2. Aktivieren Sie einen der Wochentagsschalter. *z.B. Montag*
</Steps>

*Beispiel: Diese Aktion ist nur für Gäste verfügbar, die an einem `Montag` buchen.*

### Ankunftstage der Woche

:::note
Wenn Sie diese Einschränkung aktivieren, müssen Gäste an diesem Tag ankommen, um diesen Tarif sehen zu können.

Diese Einschränkung passt gut zur „Stay date restriction“, bei der Sie den Datumsbereich einschränken und die Wochentage weiter begrenzen können.
:::

So aktivieren Sie die Einschränkung `Arrival days`:

<Steps>
1. Aktivieren Sie den Schalter `Arrival days restriction`.
2. Aktivieren Sie einen der Wochentagsschalter. *z.B. Montag*
</Steps>

*Beispiel: Diese Aktion ist nur für Gäste verfügbar, die einen Aufenthalt beginnen, der an einem `Montag` startet.*

### Abreisetage der Woche

:::note
Wenn Sie diese Einschränkung aktivieren, müssen Gäste an diesem Tag abreisen, um diesen Tarif sehen zu können.

Diese Einschränkung passt gut zur „Stay date restriction“, bei der Sie den Datumsbereich einschränken und die Wochentage weiter begrenzen können.
:::

So aktivieren Sie die Einschränkung `Departure days`:

<Steps>
1. Aktivieren Sie den Schalter `Departure days restriction`.
2. Aktivieren Sie einen der Wochentagsschalter. *z.B. Montag*
</Steps>

*Beispiel: Diese Aktion ist nur für Gäste verfügbar, die einen Aufenthalt beenden, der an einem `Montag` endet.*

### Erforderliche Wochentage

:::note
Wenn Sie diese Einschränkung aktivieren, müssen Gäste an „mindestens einem“ dieser Tage übernachten, um diesen Tarif sehen zu können.

Diese Einschränkung passt gut zur „Stay date restriction“, bei der Sie den Datumsbereich einschränken und die Wochentage weiter begrenzen können.
:::

So aktivieren Sie die Einschränkung `Required days`:

<Steps>
1. Aktivieren Sie den Schalter `Required days restriction`.
2. Aktivieren Sie einen der Wochentagsschalter. *z.B. Montag*
</Steps>

*Beispiel: Diese Aktion ist nur für Gäste verfügbar, die einen Aufenthalt buchen, der einen `Montag` berührt.*

## Sperrzeiten

Klicken Sie auf den Tab `Blackout dates`, um fortzufahren.

Steuern Sie die Aktion weiter, indem Sie Daten angeben, an denen diese Aktion nicht verfügbar sein soll.

Um ein Sperrdatum hinzuzufügen, klicken Sie auf die Schaltfläche `Add blackout date`.

- **Startdatum** Wählen Sie das Startdatum Ihres Sperrdatums.
- **Enddatum** Wählen Sie das Enddatum Ihres Sperrdatums.

Wenn die Buchungsdaten diesen Datumsbereich berühren, wird die Aktion nicht wirksam.

Ein Reisender könnte gleichzeitig für mehrere Aktionen berechtigt sein, z. B. Langzeit + Early Bird.
Ohne ein gebündeltes Angebot wäre der Reisende nur für eine davon berechtigt; nämlich das beste Angebot.

Durch die Erstellung eines gebündelten Angebots respektieren Sie die Berechtigung des Reisenden für beide Aktionen und können es weiter anpassen.

Um ein gebündeltes Angebot zu erstellen, klicken Sie auf die Schaltfläche `Create bundled promotion`.

### Konfiguration

- **Name** Geben Sie dem gebündelten Angebot einen Namen, den nur Sie sehen können.
- **Status** Schalten Sie den Schalter um, um zu steuern, ob das Paket über alle Vertriebskanäle verfügbar ist.
- **Aktionen** Wählen Sie mindestens 2 Einzelaktionen aus, die das Paket bilden.

### Betrag überschreiben

Sie können wählen, den insgesamt angewendeten Rabatt zu überschreiben. Wenn Sie es deaktiviert lassen, wird der Rabatt jeder ausgewählten Aktion zu einem Gesamt-Rabatt berechnet.

- **Betrag überschreiben** Ob Sie alle rabattierten Beträge aggregieren oder mit Ihrem eigenen Betrag überschreiben möchten.
- **Typ** Wählen Sie, ob der Preis erhöht oder verringert werden soll. *z.B. Auf `Discount` setzen für eine Preissenkung.*
- **Preistyp** Wählen Sie, wie dieses Paket berechnet werden soll.
    - `Per stay`: Betrag wird auf die letzte Nacht angewendet. *z.B. 100 % Rabatt > Bleiben Sie 2 Nächte – die 3. Nacht ist kostenlos.*
    - `Per night`: Betrag multipliziert mit der Anzahl der Nächte.
    - `Per use`: Es erfolgt keine zusätzliche Berechnung.
    - `Per person`: Betrag multipliziert mit der Anzahl der Gäste.
    - `Per person per night`: Betrag multipliziert mit der Anzahl der Nächte + Anzahl der Gäste.
- **Betragstyp** Wählen Sie, ob der Betragsmodifikator ein fester Betrag oder ein Prozentsatz sein soll.
