:::note
Sehen Sie sich unsere unterstützte Liste der [Preis-Anbieter](/de/extranet/rate-providers) an.

Wir sind integriert mit:

- Channel Managern (CM)
- Central Reservation Systems (CRS)
- Property Management Systems (PMS)

Um Ihre Preise zu verwalten, klicken Sie in der Hauptnavigationsleiste auf `Distribution > Master rate calendar`.

Dieser Artikel führt Sie durch alles, was Sie über die Funktionsweise von Preisen auf Wink wissen müssen.

## Funktionen

:::note
Wink verwaltet Preise auf Master-Preis-Ebene und nicht auf Gästezimmer-Ebene. Dies gibt Ihnen mehr Flexibilität bei der Zuweisung der Zimmer.
:::

Ein Preis auf Wink unterstützt folgende Datenpunkte:

- **Menge** Anzahl der verfügbaren Einheiten.
- **Preisbetrag** Grundpreis.
- **Verfügbar** Ob der Preis für dieses Datum verfügbar ist oder nicht.
- **Closed on Arrival (COA)** Ob Gäste an diesem Datum anreisen können.
- **Closed on Departure (COD)** Ob Gäste an diesem Datum abreisen können.
- **Mindestaufenthaltsdauer (Min LoS)** Ob eine Mindestaufenthaltsdauer erforderlich ist.
- **Maximale Aufenthaltsdauer (Max LoS)** Ob eine maximale Aufenthaltsdauer erforderlich ist.
- **Mindestbelegung** Ob eine Mindestbelegung erforderlich ist.
- **Maximale Belegung** Ob eine maximale Belegung erforderlich ist.

Die folgenden Datenpunkte sind nur über einen Preis-Anbieter verfügbar:

*Wenn Ihr Preis-Anbieter diese Datenpunkte nicht unterstützt, können Sie sie weiterhin über Ihre Preispläne auf Wink steuern.*

- Einzelbelegungs-Preisaufschlag
- Aufschlag für zusätzliche Erwachsene
- Aufschlag für zusätzliche Kinder

Preise sind das Herzstück von Wink. Unterkünfte können Preise manuell hinzufügen oder ihren Preis-Anbieter verbinden.

## Manuelle Eingabe

:::caution
Erstellen oder aktualisieren Sie Preise nur manuell, wenn Sie Ihre Zimmertypen nicht auf einer anderen Plattform als Wink verkaufen.
:::

:::caution
Wenn Sie Preise manuell aktualisieren, während Sie mit einem Preis-Anbieter verbunden sind, ändern sich die Preise erneut, sobald Ihr Preis-Anbieter uns ein Update für denselben Datumsbereich sendet.
:::

## Über Preis-Anbieter

Um uns Preise über Ihren Preis-Anbieter zu senden, gehen Sie wie folgt vor:

- Stellen Sie zunächst sicher, dass wir Ihren [Preis-Anbieter](/de/extranet/rate-providers) unterstützen.
- Legen Sie Ihren Preis-Anbieter unter `Distribution > Channel manager` fest.
- Melden Sie sich bei Ihrem Preis-Anbieter an.
- Fügen Sie den `Wink`-Kanal zu Ihrem Preis-Anbieter-Dashboard hinzu.
- Ordnen Sie Ihre Zimmertypen / Preispläne denjenigen zu, die Sie in Wink erstellt haben.
- Starten Sie einen vollständigen Preis-Upload zu Wink [falls nicht automatisch vom Preis-Anbieter durchgeführt].

Warten Sie etwa 3-5 Minuten, bis Ihre Wink-Master-Preiskalender mit Preisen gefüllt sind.

## Verifier

Sie können Ihre Preiseinrichtung testen, indem Sie in der Hauptnavigationsleiste zu `Distribution > Verifier` navigieren.

Auf dieser Seite können Sie jede Art von Preisanfrage simulieren, die ein Reisender stellen könnte.

:::tip
Wenn Sie die gesuchte Einstellung nicht finden, klicken Sie unten auf der Seite auf die Schaltfläche `Erweiterte Suche`.
:::

### Beispiel

Unten sehen Sie eine Beispiel-Reiseroute über einen Ihrer bestehenden Vertriebskanäle:

- **Kanal** Wählen Sie den Vertriebskanal, über den Sie verkaufen möchten. *z.B. Traveliko OTA*.
- **Währung** Wählen Sie die Anzeigewährung. *z.B. USD*
- **Anreisedatum** Geben Sie das Anreisedatum ein. *z.B. 10. Sept. 2024*
- **Nächte** Geben Sie die Anzahl der Nächte ein. *z.B. 1*
- **Gäste** Geben Sie die Anzahl der Erwachsenen / Kinder ein. *z.B. 2 Erwachsene*
- Klicken Sie auf die Schaltfläche `Suchen`, um fortzufahren.

Sie können Ihren gesamten Ablauf von Anfang bis Ende testen, indem Sie eine Testbuchung erstellen.
Eine Testbuchung ist als `Test` markiert und sichert die Buchung nicht mit einer Kreditkarte oder einer anderen Zahlungsmethode ab.

Testbuchungsoptionen umfassen:

- **Benachrichtigung der Unterkunft per E-Mail** Dies löst das Senden einer E-Mail-Bestätigung an die Unterkunft aus.
- **Benachrichtigung des Channel Managers per API** Dies löst das Senden der Buchung an Ihren ausgewählten Preis-Anbieter aus.
- **Benachrichtigung des Buchers per E-Mail** Dies löst das Senden einer E-Mail-Bestätigung an den Bucher aus.

Standardmäßig sind Sie in diesem Szenario der Bucher, also der authentifizierte Benutzer. Sie können auch benutzerdefinierte Bucherdaten hinzufügen, indem Sie den Schalter `Benutzerdefinierte Bucherdaten hinzufügen` aktivieren und das Formular ausfüllen.

*Wenn Sie ausgewählt haben, dass wir Ihren Preis-Anbieter benachrichtigen, behandelt der Preis-Anbieter diese Buchung wie jede andere Buchung und weiß nicht, dass es sich um einen Test handelt. Wir empfehlen, mit erstattungsfähigen Preisen zu testen, um eine einfache Stornierung nach Abschluss des Tests zu ermöglichen.*
