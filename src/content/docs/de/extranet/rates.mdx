:::Hinweis
Sehen Sie sich unsere unterstützten Produkte an[Tarifanbieter](/extranet/rate-providers).

Wir sind integriert mit:

* Channel Manager (CM)
* Zentrale Reservierungssysteme (CRS)
* Immobilienverwaltungssysteme (PMS)
  :::

Um Ihre Tarife zu verwalten, klicken Sie auf`Distribution > Master rate calendar`aus der Hauptnavigationsleiste.

In diesem Artikel erfahren Sie alles, was Sie über die Funktionsweise der Tarife bei Wink wissen müssen.

## Merkmale

:::Hinweis
Wink verwaltet Preise auf Hauptpreisebene und nicht auf Zimmerebene. Dies bietet Ihnen mehr Flexibilität bei der Zimmerzuteilung.
:::

Eine Bewertung auf Wink unterstützt die folgenden Datenpunkte:

* **Menge**Anzahl der verfügbaren Einheiten.
* **Ratenbetrag**Grundpreis.
* **Verfügbar**Ob der Preis für dieses Datum verfügbar ist oder nicht.
* **Bei Ankunft geschlossen (COA)**&#x4F;b Gäste an diesem Datum anreisen können.
* **Bei Abreise geschlossen (COD)**&#x4F;b Gäste an diesem Datum abreisen können.
* **Mindestaufenthaltsdauer (Min LoS)**&#x4F;b eine Mindest-LoS-Anforderung besteht.
* **Maximale Aufenthaltsdauer (Max LoS)**&#x4F;b eine maximale LoS-Anforderung besteht.
* **Mindestbelegung**Ob eine Mindestbelegungsanforderung besteht.
* **Maximalbelegung**Ob eine maximale LoS-Anforderung besteht.

Die folgenden Datenpunkte sind nur über einen Tarifanbieter verfügbar:

*Wenn Ihr Tarifanbieter diese Datenpunkte nicht unterstützt, können Sie sie trotzdem über Ihre Tarifpläne auf Wink steuern.*

* Modifikator für Einzelbelegungsraten
* Tarifmodifikator für zusätzliche Erwachsene
* Zusätzlicher Kindertarifmodifikator

Preise sind das Herzstück von Wink. Unterkünfte können Preise manuell hinzufügen oder ihren Preisanbieter anbinden.

## Manuelle Eingabe

:::Achtung
Erstellen/aktualisieren Sie Preise nur manuell, wenn Sie Ihre Zimmerkategorien ausschließlich auf Wink anbieten.
:::

:::Achtung
Wenn Sie Ihre Preise manuell aktualisieren, während Sie mit einem Preisanbieter verbunden sind, ändern sich die Preise erneut, sobald Ihr Preisanbieter uns eine Aktualisierung für denselben Zeitraum sendet.
:::

## Über den Tarifanbieter

Um uns Tarife über Ihren Tarifanbieter zu senden, folgen Sie diesen Schritten:

* Stellen Sie zunächst sicher, dass wir Ihre[Tarifanbieter](/extranet/rate-providers).
* Stellen Sie Ihren Tarifanbieter unter`Distribution > Channel manager`.
* Melden Sie sich bei Ihrem Tarifanbieter an.
* Fügen Sie die`Wink`Kanal zu Ihrem Tarifanbieter-Dashboard.
* Ordnen Sie Ihre Zimmertypen/Preispläne denen zu, die Sie in Wink erstellt haben.
* Leiten Sie einen vollständigen Tarif-Push an Wink ein \[sofern dies nicht automatisch vom Tarifanbieter erfolgt].

Warten Sie etwa 3–5 Minuten, bis sich Ihr Wink-Master-Tarifkalender mit Tarifen füllt.

## Prüfer

Sie können Ihre Tarifeinstellungen testen, indem Sie zu`Distribution > Verifier`aus der Hauptnavigationsleiste.

Auf dieser Seite können Sie jede Art von Preisanfrage simulieren, die ein Reisender stellen könnte.

:::Tipp
Wenn Sie die gewünschte Einstellung nicht finden können, klicken Sie auf das`Advanced search`Schaltfläche unten auf der Seite.
:::

### Beispiel

Nachfolgend sehen Sie ein Beispiel für eine Reiseroute über einen Ihrer vorhandenen Vertriebskanäle:

* **Kanal**Wählen Sie den Vertriebskanal, über den Sie verkaufen möchten.*z. B. Traveliko OTA*.
* **Währung**Wählen Sie die Anzeigewährung.*z. B. USD*
* **Beginndatum des Aufenthalts**Geben Sie das Ankunftsdatum ein.*z.B. 10. September 2024*
* **Nächte**Geben Sie die Anzahl der Nächte ein.*z.B. 1*
* **Gäste**Geben Sie den Besuch für Erwachsene/Kinder ein.*z.B. 2 Erwachsene*
* Klicken Sie auf das`Search`Schaltfläche, um fortzufahren.

### Testbuchung

<figure>
  <CloudinaryImage width="350" height="459" resize="scale" quality="best" alt="Buchungsmodal testen" src="academy/test-booking" errorImage="noimage_opaque_nyrtl0.png" />

  <figcaption>*Testen Sie das Buchungsbestätigungsfenster*</figcaption>
</figure>


Sie können Ihren gesamten Workflow durch eine Testbuchung testen.
Eine Testbuchung ist gekennzeichnet als`test`und sichert die Buchung nicht mit einer Kreditkarte oder einer anderen Zahlungsmethode.

Zu den Testbuchungsoptionen gehören:

* **Unterkunft per E-Mail benachrichtigen**Dadurch wird der Unterkunft eine E-Mail-Bestätigung zugesandt.
* **Channel-Manager über API benachrichtigen**Dadurch wird die Übermittlung der Buchung an Ihren ausgewählten Tarifanbieter ausgelöst.
* **Buchenden per E-Mail benachrichtigen**Dadurch wird der Versand der E-Mail-Bestätigung an den Bucher ausgelöst.

Standardmäßig sind Sie, der authentifizierte Benutzer, der Bucher in diesem Szenario. Sie können auch benutzerdefinierte Bucherdetails hinzufügen, indem Sie die Option`Add custom booker details`Schalter und Ausfüllen des Formulars.

*Wenn Sie Ihren Tarifanbieter benachrichtigen lassen, behandelt dieser diese Buchung wie jede andere Buchung und erkennt nicht, dass es sich um einen Test handelt. Wir empfehlen, mit erstattungsfähigen Tarifen zu testen, um eine einfache Stornierung nach Abschluss des Tests zu gewährleisten.*
